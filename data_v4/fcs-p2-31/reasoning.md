Two features of this matcher settle the approach before I choose an algorithm. The `*` operator stands for zero, one, or many characters, and a pattern may carry many of them, so the number of ways its `*`s could carve up `s` is combinatorial — precisely the structure that punishes any matcher resolving that ambiguity greedily or by local rewinding. And the empty string is a legal input on either side: since two empty tokens cannot share a whitespace-separated line, the contract encodes an empty token as a single hyphen `-`. I decode `-` to `""` for both `p` and `s` the moment I read them, before touching any length, or an empty case silently degrades into a one-character match against a literal hyphen. With `|p|, |s| <= 2000` under a 1-second limit anything `O(|p| * |s|) = 4 * 10^6` is trivially in budget, so the only real question is whether I reach for something cleverer and get burned.

Two routes are on the table. The first is two-pointer backtracking on `*`: walk `p` and `s` with two indices, consume one character on a literal or `?`, and on a `*` remember "last star was here, `s` was at this position", tentatively match nothing, and march on; on a later literal mismatch, rewind to that star, let it swallow one more character, and retry. It is short and carries no table. The second is a two-dimensional prefix DP — a boolean `dp[i][j]` filled by a fixed recurrence, `O(|p| * |s|)`, where the only work is getting the `*` transition and the empty-prefix boundary right.

I distrust the backtracking route on principle: `*` is a globally ambiguous operator, and any scheme resolving it with a single "last star" rewind pointer makes a local commitment about a global structure — the classic configuration where pointer tricks are either exponential or subtly wrong. So I try to break it before writing a line, on time and on correctness.

On time: "rewind the last `*` one more character" never remembers how far *earlier* stars were pushed; it only ever rewinds the most recent one. Take the failing shape `p = "a*a*a*...*a*b"` — a thousand `a`-blocks ending in a literal `b` — against `s = "aaaa...a"` of length 2000 with no `b` at all. Every prefix of stars matches, so the recursion happily descends, distributing the 2000 `a`s among the stars in combinatorially many ways, and only at the very end finds the `b` has nothing to match — then backtracks and tries another distribution, and another. Splitting a run of `a`s among many stars is combinatorial, so an unmemoized backtracker does astronomical work to reach a single `NO`. That is the textbook exponential blow-up of glob matching, and 2000 characters is ample rope: the clever route TLEs on an input I can write in one line.

On correctness: even the iterative single-rewind-pointer variant that is supposed to be near-linear is a minefield, and every fragile spot is a corner the hidden tests target. When `s` is exhausted but `p` still has characters I must skip *only* trailing `*`s and reject any leftover literal or `?`; a leading `*` with `s` empty must still match if the pattern's tail is all stars; consecutive `*`s must collapse without confusing which star to blame; and on a rewind I must advance the string anchor, not the pattern anchor. Each has a famous wrong variant, and getting all four right under budget is exactly the risk this problem is built to punish.

So I ship the thing I can prove. The DP is `4 * 10^6` operations with a recurrence I can write down and check line by line, and there is no ambiguity to resolve cleverly, because the table considers all ways a `*` can split `s` at once.

Let `dp[i][j]` be true iff the first `i` characters of `p` match the first `j` of `s`; I want `dp[|p|][|s|]`. For the boundary, `dp[0][0]` is true (empty matches empty) and `dp[0][j>=1]` is false. The first column asks whether a length-`i` pattern prefix can match the empty string — true only if every one of its characters is a `*` (each matching empty), which the `*` transition reproduces automatically. The transition for `i, j >= 1`, reading `pc = p[i-1]`:

- `pc == '*'`: the star matches the **empty** sequence — drop it, needing `dp[i-1][j]` — or absorbs `s[j-1]` and stays available, `dp[i][j-1]`. So `dp[i][j] = dp[i-1][j] || dp[i][j-1]`. This one OR is the whole reason DP beats backtracking: it folds "star eats more" and "star stops here" into a single memoized step, with no rewinding.
- `pc == '?'` or `pc == s[j-1]`: a single-character match, `dp[i][j] = dp[i-1][j-1]`.
- otherwise: `dp[i][j] = false`.

The `*` line is the only real transcription risk. On `p = "a*b"`, `s = "axxxb"`: the literal-`a` row is true only at `j=1`; the `*` row picks that up at `dp[2][1] = dp[1][1] || dp[2][0]` and, staying true once true, carries it across the whole row to `dp[2][5]` — the star has absorbed `xxx`; the literal-`b` row is true only where `s[j-1] == 'b'`, so `dp[3][5] = dp[2][4] = T` -> YES. Swap the trailing `b` for `c` and that final cell has no true predecessor -> NO, the accept/reject pair the contract names.

A full `2001 x 2001` `char` table is ~4 MB, comfortably inside 256 MB, but row `i` reads only row `i-1` and the current row to its left, so two rolling rows of length `|s|+1` suffice — a few kilobytes. The one subtlety this buys is that `cur[0]` (the first-column boundary) must be recomputed fresh at the top of each row, before the inner loop starts at `j=1` and reads `cur[j-1]`. The row loop mirroring the recurrence:

```
prev[0] = 1; for j in 1..m: prev[j] = 0;          // dp[0][*]
for i in 1..n:
    char pc = p[i-1];
    cur[0] = (pc == '*') ? prev[0] : 0;            // first-column boundary
    for j in 1..m:
        if (pc == '*')      cur[j] = prev[j] || cur[j-1];
        else if (pc=='?' || pc==s[j-1]) cur[j] = prev[j-1];
        else                cur[j] = 0;
    swap(prev, cur);
```

The rolling buffers introduce one hazard: after a `swap`, `cur` holds stale row `i-2` data that I overwrite left-to-right, so every read must hit either freshly-written `cur` to its left or the legitimate `prev`. The `*` case reads `cur[j-1]` (written this row) and `prev[j]` (the real row `i-1`); `cur[0]` reads `prev[0]`. The first column is where a stale `1` could leak, so trace `p = "*a"`, `s = "ba"`, expecting YES. `prev = [1,0,0]`. Row 1 `*`: `cur = [1,1,1]`; the swap leaves both rows `[1,1,1]`. Row 2 literal `a`: `cur[0] = 0` (not a star); `j=1`, `'a' != 'b'` -> `0`; `j=2`, `'a' == 'a'` -> `cur[2] = prev[1] = 1`. `cur = [0,0,1]` -> YES. The literal row overwrote every stale `1`, but only because `cur[0]` is written unconditionally: had the literal branch skipped it, that leftover `1` would falsely report the empty prefix as matchable mid-table. So `cur[0] = (pc == '*') ? prev[0] : 0` must fire on every row, literal branch included.

The other corner I trust least is `s` empty, because then the inner loop never runs and the whole answer rides on the first column. `p = "*?*"`, `s = ""`: `prev = [1]`; the `*` keeps `cur[0] = 1`; the `?` sets `cur[0] = 0`; the trailing `*` copies that `0`; answer `0` -> NO — right, the `?` demands a character `s` does not have. The complementary `p = "**"`, `s = ""` keeps `1` through both star rows -> YES. That first-column logic also settles the remaining edges: empty/empty -> YES; empty pattern versus nonempty `s` -> NO; nonempty pattern versus empty `s` -> YES iff the pattern is all stars; a trailing `*` matches an empty tail (`p = "a*"`, `s = "a"` -> YES); and the adversarial `*a*a*a*` versus `aa` propagates false to the final cell -> NO, in `O(|p| * |s|)` work with no rewinding to blow up.

Hand traces cover the cells I look at, not the thousands I do not, so I check against an independent oracle written straight from the matching definition: a top-down `match(i, j)` whose `*` case branches into "match empty: `match(i+1, j)`" OR "consume one: `match(i, j+1)`", memoized over `(i, j)`. It shares no code with the DP and resolves `*` by recursion rather than by a table, so agreement between them is real evidence, not a tautology. I differential-tested random patterns over `{a, b, c, ?, *}`, a star-heavy mode, and a "derived" mode that builds the string first and grows a usually-matching pattern (so YES cases are well represented, not just easy NOs), alongside a hand-picked corner bank — empty/empty, `*` versus empty, `?` versus empty, `*a*a*a*` versus `aaa` and versus `aa`, trailing/leading/consecutive stars, the `a*b` accept/reject pair. The YES fraction sat near half, so both verdicts are genuinely exercised, and there were zero mismatches; the full-size adversarial inputs — all-`a`, all-`*`, and the deep-rewind `NO` shape at length 2000 — each finished in well under a millisecond on a few megabytes, the exact inputs that would have detonated the backtracker.

So the file is a single self-contained C++ program: decode `-` to the empty string on both tokens, fill the two rolling DP rows with the unconditional `cur[0]` write, and print `prev[m] ? "YES" : "NO"`.
