The margins here span negatives, zeros, and positives, and the objective is max-over-routes of the *minimum* margin along the route — a widest-path / bottleneck problem, not an additive one. That sign mix is the whole game: with only positive weights this would be textbook, but zeros and negatives put two sign-corners in play — what the source's base value should be, and how to tell a genuinely-reachable but degraded node apart from an unreachable one. Scale first, since it fixes both the algorithm and the types: `n` up to `2*10^5`, `m` up to `5*10^5`, `|w|` up to `10^9`. Bellman-Ford relaxation (`O(nm)` ~ `10^11` ops) is hopeless at these sizes, so it has to be a heap-based Dijkstra. The metric never sums weights — a bottleneck always equals some single edge's margin — so the *answer* fits in 32 bits; but I carry the metric in `long long` anyway, because I want `+infinity` / `-infinity` sentinels sitting far outside `[-10^9, 10^9]` with no risk of `int` overflow. Node ids stay `int`.

The reflex "negative edges — Dijkstra is illegal" is about *additive* paths, where a later negative edge can retroactively undercut a finalized node. Here the metric is `min`, and extending a route is monotone non-increasing: `min(best[u], w) <= best[u]` for any sign of `w`. So once I pop the unfinished node `u` with the largest `best[u]`, no route can later reach `u` with a larger bottleneck — any such route runs through some still-unfinished node whose `best` is `<= best[u]`, and passing through it can only shrink the running minimum further. That is Dijkstra's exchange argument, resting on monotonicity, not on non-negativity. So a max-heap with relaxation `cand = min(best[u], w)`, keeping the max, is correct *with* negative and zero margins. (Simple paths suffice: dropping a cycle from a route removes edges, and fewer terms in a `min` can only raise it, so some simple route is always at least as good — the optimum is finite and attained.)

Now the source base value, the first sign-corner. `best[s]` is the bottleneck of the zero-length route `s -> s`, which uses no links; the `min` over an empty set of margins is the *identity* of `min`, `+infinity`. The additive reflex is to write `best[s] = 0`, and that is a real trap here rather than a harmless off-by-one: seeding `0` fabricates a phantom zero-margin link that every route inherits, clamping `best[v]` to `<= 0` everywhere. A first link `s -> v` with `w = 5` then gives `min(0, 5) = 0` instead of `5` — a blatant wrong answer on any all-positive graph. The identity of `min` is `+infinity`, so `best[s] = POS_INF`, and the first relaxation `min(POS_INF, w) = w` recovers the correct single-link bottleneck. I set `POS_INF = LLONG_MAX/4` and "not reached yet" to `NEG_INF = LLONG_MIN/4` — both far outside the real range and distinguishable from each other and from every real margin.

The second sign-corner is the output test. A reachable node can legitimately have a negative or zero bottleneck, so reachability cannot be read off the sign. Testing `best[v] <= 0 -> UNREACHABLE` collapses two genuinely different outcomes: a zero-margin link `s -> v` (`best = 0`, reachable, must print `0`) would report `UNREACHABLE`, and an all-negative graph where `best[2] = -5` would report `UNREACHABLE` for a node reachable with reliability `-5`. Reachability is exactly "did `best[v]` ever leave the `NEG_INF` sentinel" — a question about the sentinel, not the sign. So I gate `UNREACHABLE` on `best[v] == NEG_INF` exactly, print `INF` on the source line (its untouched `POS_INF`), and the integer `best[v]` otherwise.

A concrete graph to watch the mechanism run:
```
5 6 1
1 2 4 / 1 3 -2 / 2 3 5 / 2 4 0 / 3 4 7 / 4 5 3
```
By hand: node 3's best is `1->2->3 = min(4,5) = 4`, beating `1->3 = -2`; node 4 is `1->2->3->4 = 4`, beating `1->2->4 = 0` and `1->3->4 = -2`; node 5 is `min(4,3) = 3`. Running the algorithm: `best[1] = POS_INF`; pop 1, relax to `best[2] = 4`, `best[3] = -2`. Pop `(4,2)`: `2->3` gives `min(4,5) = 4 > -2` so `best[3] = 4`; `2->4` gives `0`. Pop `(4,3)`: `3->4` gives `min(4,7) = 4 > 0` so `best[4] = 4`. Pop `(4,4)`: `4->5` gives `3`. The stale heap entries `(0,4)`, `(-2,3)` get skipped. Final `INF / 4 / 4 / 4 / 3`. And the two bugs surface together on this same graph: seed `best[s] = 0` and the first relaxation writes `best[2] = min(0,4) = 0`, the clamp ripples to all-zero, and with the sign test every line then prints `UNREACHABLE`.

I use a lazy heap — push improved values, never decrease-key — so it holds outdated `(d, v)` pairs. `done[u]` alone already suffices for correctness: the max-heap pops larger values first, so if a larger entry for `u` existed it would have been popped and finalized `u` before this stale pop, which `done[u]` then skips. I keep an explicit `d != best[u]` guard alongside it so a node is only ever expanded on its up-to-date frontier value; it changes no answer but makes the invariant legible.

The corners the sign handling has to survive: `m = 0` (only the source pops, everyone else stays `NEG_INF`, so `INF` for `s` and `UNREACHABLE` elsewhere); `n = 1` (single line `INF`); an all-negative graph (reachable nodes print their negative bottleneck — the exact collision the sign test would have caused); a zero-margin link with an unreachable node alongside (`INF / 0 / UNREACHABLE`, the `0` and the `UNREACHABLE` kept distinct); self-loops and parallel edges (a self-loop relaxes `min(best[u], w) <= best[u]` against `best[u]` and can never improve it; parallel edges are just repeated relaxations, and the best one wins the `cand > best[v]` test). And overflow: since I never add weights, the metric stays in `[-10^9, 10^9]`, and the sentinels only pass through `min` or comparison (`min(POS_INF, w) = w` since `w << POS_INF`), never arithmetic that could overflow 64-bit.

Against an exhaustive DFS brute force enumerating all simple paths and taking the max-of-min, the Dijkstra solution agreed on every output line over 701 random cases spanning all-negative, all-zero, mixed, empty, self-loop, and parallel-edge inputs — enough to trust both sign-corner fixes and the sentinel-gated three-way output.
