The feasible region here is a *band* of total prices `[L, R]` with both endpoints inclusive, not the single cap of a textbook knapsack — and the whole problem lives at the two ends of that band, so it is really an exercise in getting the boundaries exactly right. Before any algorithm I fix the scale, because it forces the types: up to `2000` items, each with joy up to `10^9` in magnitude, means a subset's total joy can reach `2*10^3 * 10^9 = 2*10^12`, far past 32 bits. Every joy accumulator must be `long long`; an `int` accumulator would be a silent wrong answer on the large tests. Prices stay small (`s <= R <= 10^5`), but I carry them as `long long` too so the comparisons stay uniform.

Now the state. This is 0/1 knapsack, but I need a formulation I can read a band off cleanly. The textbook "capacity" form `dp[s]` = best joy at price `<= s` bakes the upper bound into the state and discards the exact price, so afterwards I cannot ask "is the price at least `L`?" — the lower edge of the band is lost. And with negative joys, "best at price `<= s`" is not monotone in a way that lets me recover the band by subtracting two capacities. So I use exact-price states instead: `dp[s]` = best joy of a subset whose total price is **exactly** `s`, for `s` in `0..R`. Items with `p[i] > R` can never fit and are dropped. The band then reads off as a plain scan — the answer is `max(dp[s])` over `s` in `[L, R]`, and `IMPOSSIBLE` means every such cell is unreachable.

The recurrence is standard. Base: before any item only price `0` is reachable (the empty subset, joy `0`); every other price I mark unreachable with a sentinel `NEG` far below any real joy. For item `i` with price `pi`, joy `vi`, a subset reaching price `s` either omits `i` (joy stays `dp[s]`) or includes it, meaning the rest reached `s - pi` (joy `dp[s - pi] + vi`): `dp[s] = max(dp[s], dp[s - pi] + vi)`. Keeping one array, I iterate `s` **downward** from `R` to `pi`, so `dp[s - pi]` still holds the item-excluded value and each item is used at most once.

The endpoints `L` and `R` never enter this recurrence. They enter in exactly two places, both inclusive-boundary decisions:

- The array spans prices `0..R` **inclusive** — that is `R + 1` cells, not `R`. Sizing it `R` drops index `R`, the cap itself, and the descending loop's `s = R` write then becomes an out-of-bounds heap overwrite (UB).
- The answer scan runs `s = L, L+1, ..., R` **inclusive on both ends**. Writing `s < R` silently discards any optimum whose price is exactly `R`; starting the scan at `L + 1` discards any optimum whose price is exactly `L`.

One concrete case pins both upper-edge slips at once. Take a single item priced exactly at the cap: `n=1, L=0, R=4`, item `(4, 7)`. The only nonempty in-band subset is `{item}` at price `4 = R`, joy `7`, and the empty subset (price `0`, joy `0`) is also in-band since `L = 0`, so the answer is `7`. That `7` lives only at `dp[4]` — which does not exist in a size-`4` array, and which an `s < R` scan would never read. With `dp` sized `R + 1 = 5` the item sets `dp[4] = dp[0] + 7 = 7`, and the inclusive scan reaches it. The lower edge is the mirror image: an input whose only in-band optimum sits at `s = L` — say `n=2, L=5, R=9`, items `(5,4),(3,-2)`, where `{a}` at price `5 = L` gives joy `4`, better than `{a,b}` at price `8` with joy `2` — would print the wrong `2` if the scan began at `L + 1`, so it must begin at `L`.

That leaves the corner cases to get right:

- `n = 0`: only `dp[0] = 0` is reachable. `L = 0` prints `0` (empty purchase legal); `L > 0` prints `IMPOSSIBLE`, since the empty subset's price `0` is below the band. `L = R = 0` prints `0`.
- `p[i] > R`: skipped by `if (pi > R) continue;`. The descending loop `s = R..pi` would not run anyway when `R < pi`, so the guard is for intent, not correctness — but it saves a wasted pass.
- All-negative joys with a forced in-band subset: e.g. `n=2, L=4, R=10`, items `(4,-3),(6,-5)` give reachable in-band joys `-3, -5, -8`, and the answer is the least-bad, `-3`. Since the answer can legitimately be negative, the sentinel has to sit far below any real joy: `NEG = LLONG_MIN/4 ≈ -2.3*10^18` is six orders of magnitude below the most negative reachable joy (`~ -2*10^12`), so the `!= NEG` reachability test never mistakes a real negative joy for "unreachable."
- No in-band subset (e.g. reachable prices `0` and `5` against band `[2, 2]`): the scan sees only `NEG` and prints `IMPOSSIBLE`.
- Overflow: `NEG` is only ever read behind the `dp[s - pi] != NEG` guard, so `NEG + vi` is never evaluated and the sentinel cannot underflow; the real joys fit in `long long` with room.

So the shape is settled: exact-price 0/1 knapsack, one array of `R + 1` cells filled by a descending update, then an inclusive `[L, R]` scan for the best joy or `IMPOSSIBLE`. `O(n*R)` time is about `2*10^8` simple operations at the limits — comfortable under 1 s.
