# Number Link — Maximum Connected Pairs (Flow-Free style routing)

## Research question

You are given a rectangular grid with `K` colored **endpoint pairs**. Each color has exactly two
endpoint cells. A *connection* for a color is a path that starts at one of its endpoints, ends at
the other, and moves between orthogonally-adjacent grid cells (unit rectilinear steps). The paths of
two different colors may **never share a cell** — not an interior cell and not an endpoint — so the
realized connections form a set of vertex-disjoint paths on the grid graph. Connecting *all* colors
simultaneously (the classic Number Link / Flow Free puzzle) is NP-complete, and on dense instances no
disjoint routing of every pair exists.

The task here is the **maximization relaxation**: connect *as many* pairs as possible. The objective
is the number of pairs that are simultaneously connected by cell-disjoint paths. This is the
maximum-disjoint-paths problem on a grid — the structure behind VLSI global routing and puzzle
solving — and it is the hard, no-exact-answer regime ALE-Bench targets.

## Input / output contract

**Instance (stdin).**

```
H W K
r0a c0a r0b c0b
r1a c1a r1b c1b
...
r(K-1)a c(K-1)a r(K-1)b c(K-1)b
```

- `H` rows, `W` columns, `K` colored pairs. All coordinates are 0-indexed: `0 <= r < H`,
  `0 <= c < W`.
- Line `i+1` gives the two endpoints of color `i`: `(ria, cia)` and `(rib, cib)`.
- All `2*K` endpoint cells are pairwise distinct.
- Typical sizes: `12 <= H, W <= 30`, with `K` chosen so the `2*K` endpoints occupy roughly 25%–40%
  of the cells (a regime where many — but not all — pairs can be routed at once).

**Solution (stdout).**

```
C
id_1 L_1 r c r c ... r c
id_2 L_2 r c r c ... r c
...
id_C L_C r c r c ... r c
```

- `C` is the number of pairs you claim to connect (`0 <= C <= K`).
- Each of the next `C` lines describes one connected pair: `id` is the color index (0-based, each id
  at most once), `L` is the number of cells on the path, followed by the `L` cells `(r, c)` in
  order from one endpoint to the other.
- A path must begin and end at color `id`'s two endpoints (either orientation), take unit
  rectilinear steps between consecutive cells, stay in bounds, and not revisit a cell.
- Emitting `C = 0` (connect nothing) is always legal.

## Background

This is **maximum vertex-disjoint paths on a grid** — equivalently, maximizing routed nets in a
single-layer global-routing instance. Two ideas frame the design space:

- **Greedy sequential routing.** Route the pairs one at a time with a shortest-path search (BFS / A*)
  over the cells not yet taken, claiming each found path. Simple and fast, but order-sensitive and
  myopic: an early pair can take a corridor a later, more constrained pair needed, so a pair that was
  routable in isolation becomes blocked.
- **Rip-up and reroute (negotiated congestion).** The established strong method for routing as many
  nets as possible. Route greedily, then for each *failed* pair, allow it to cut through cells owned
  by other pairs at a penalty, rip up exactly the pairs it crosses, claim it, and reroute the
  victims; a per-cell congestion price accumulates on chronically-contested cells so later attempts
  negotiate around them. This is the PathFinder idea, adapted to maximize the *count* of routed
  pairs rather than minimize total wire length.

## Evaluation settings

**Scoring rule.** A deterministic scorer reads the instance and the solution and returns the number
of validly connected, cell-disjoint pairs. A claimed path for color `id` is valid iff: `id` is a
real color index used at most once; the path has `>= 2` cells; its first and last cells are exactly
color `id`'s two endpoints (either order); every cell is in bounds; consecutive cells differ by a
single unit step (`|dr| + |dc| == 1`); and the path does not revisit a cell. Across all claimed
paths, **no grid cell may be used by more than one path** (endpoints included).

**Feasibility floor.** If *any* check fails — a malformed line, an out-of-range or duplicated id, a
non-unit step, an out-of-bounds cell, a wrong endpoint, a self-revisit, or **any cell shared between
two paths** — the whole solution is infeasible and the score is floored to **0**. Otherwise the score
is `C`, the number of connected pairs. (A solver should therefore always output a feasible set; the
worst safe answer is `C = 0`.)

**Normalization / baseline.** Scores are read against the trivial baseline named in the task: greedy
shortest-path-per-pair in input order, counting only the conflict-free pairs (route each pair with
BFS over free cells, skip a pair that cannot be routed). A strong solver must beat this baseline's
mean over the seed set.

**Instances.** Generated by `verify/gen.py <seed>`: a seed fixes `H, W` (12–30 each) and `K`, then
`2*K` distinct cells are sampled uniformly without replacement as the endpoints, paired up in order.
`K` is sized so endpoints fill ~25%–40% of the grid — dense enough that routing every pair at once is
generally impossible, sparse enough that a large fraction can be routed, so the maximization is
genuinely hard and discriminating.

**Time budget.** ~1.8 s of wall-clock per instance, single thread. The solver is deterministic
(fixed RNG seed).
