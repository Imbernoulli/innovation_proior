Problem:
-------
Given the following disjunctive graph:
• Nodes: operations j_k meaning “operation k of job j”; plus an artificial source s and sink t.  
• Source arcs: edges from s to every operation node with infinite capacity.  
• Sink arcs: edges from every operation node to t with infinite capacity.  
• Precedence arcs: for each job, connect its consecutive operations j_k → j_{k+1}.  
• Machine arcs: for each machine and each unordered pair of jobs, add exactly **one** directed edge between those two job’s operations (so the machine processes them one after another).

Input format (the file `jsplib.dat`):
• First line: two positive integers J and M separated by whitespace (no fixed column width).  
• Lines 2..(J+1): each describes one job as a sequence of 2M nonnegative integers: for each machine 0..M-1, the pair (processing time on that machine, job number). All numbers on a job line are space-separated.  
• After the description of all jobs there is a single blank line.  
• Line (J+3): two positive integers A and B separated by whitespace — these denote the weights of the **precedence arcs**.  
• Lines (J+4)..((J+3)+A*B*(B+3)/2): each contains one nonnegative integer K, the weight of a **machine arc**. There are exactly A*B*(B+3)/2 such lines, in arbitrary order.

Goal
----
Orient every machine arc so that the maximum flow from s to t has the greatest possible value (with respect to the arc weights). If several orientations yield the same maximal flow, break ties arbitrarily. Output the chosen orientation of all machine arcs, one per line, in machine order 0..M-1, then terminate.

Output format:
• One line per machine 0..M-1, containing exactly M distinct integers 0..M-1 (permutations of machine indices). Each integer lists a job assigned to that machine position.  
• No start/finish times.  
• Your output must describe **at least one optimal orientation** among those attaining the largest s-t flow; multiple solutions exist and any optimal one is accepted.

Constraints & Remarks
--------------------
• The official JSPLIB test set is challenging but not impossible: sizes reach ~ (J,M) ≈ (50,25), which implies up to ~1,250 operation nodes in the disjunctive graph.  
• Processing times are positive integers and may range widely (including very large values). Route structures include random, nearly-flow, block-flow, and strong-bottleneck cases.  
• Ties can occur when different arc-orientations give the same max-flow value. Any tie-breaking is permitted as long as the reported orientation is optimal.  
• Because many optimal orientations exist, the judging system allows multiple answers; it does not require a particular tie-break.

Scoring (Lower Is Better)
-------------------------
The checker computes your makespan P from your output. Each test case also carries two integers (B,T) from the answer file; here B is the makespan of a naive baseline schedule and T = 0 (a trivial lower bound). With this general scheme the score is

  Else:       score = clamp( (B - P) / (B - T), 0, 1 ).

Since T = 0 and B > 0, this simplifies to
