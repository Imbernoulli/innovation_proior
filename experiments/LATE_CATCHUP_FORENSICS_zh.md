# 后程反超取证:base+RL 为何在 s20 追上创新先验臂(2026-07-14)

**问题**:4 条 GRPO 臂(同配置、同 synthetic-500、20 步)中,创新先验臂在 s5–s15 的同步对照上领先,但 base 臂后程持续走高、s20 终点 FCS 反超(11.01 vs newmt 10.70)。为什么我们的臂"先强后被追上"?base 后程增益从哪来?我们的臂为什么"提前饱和"?

**数据源**(全部只读):
- 训练日志 `FrontierSmith/logs/rlfsx_*` (metric 行只存活于后半窗口:start/newmt s11–20,wd03/nom_a5 s16–20;前半窗口 .out 因 kill 时缓冲丢失。7/06 的 10728651 是旧 wave 数据的旧跑,未采用)
- rollout dumps `outputs/rl_frontiersmith_synth_rollout/fsx_*/{1..20}.jsonl`(每臂每步 32 题 × 8 样本,四臂同步同题——**同 step 跨臂严格配对**)
- 评测生成 `outputs/cc_eval_rlfsx_*_step{5,10,15,20}_thinking_32k_both_vllm` + step0 基线目录 + research 目录;dedup 键 (data_source, problem_idx, sample_idx) 取最后;**严格口径:每题 5 槽,缺失/error=0**;FCS=172 题按题平均
- 分析脚本与中间产物在本次会话 scratchpad(`eval_matrix.json`/`rollout_stats.json`/`train_metrics.json`)

|---|---|---|---|---|---|---|

先说三个被总表直接修正的印象:
1. "s5–s15 全面领先"不成立:每个同步点领先的**是不同的臂**(s5=wd03、s10/s15=newmt),base 对任一单臂在任何步都无显著差(见 H4);"我们最好的臂 vs 一条 base"自带 3 选 1 的选择偏置。
2. 按**自身起点的增益**看,newmt +4.81 是四臂最大(base +3.96);"被追上"的一半是**起点差**:base s0 就比 newmt 高 1.16。
3. base s20 的 11.01 是在 27 个 judge-infra error 记 0 的情况下拿到的(16 题受累);error 集中在本来就低分的题,修复后仅 +0.005,但各臂 error 数(27/4/12/1)本身提示评测 infra 抖动。

## H1 截断储备释放 —— **支持(主趋势),但"我们臂早饱和"只有弱微分信号**

**机制量化**(被截断样本得分≈0:各臂各步 rew|trunc = 0.1–1.1;未截断样本 rew|ok = 12.6–19.0):

s15→s20 窗口的"截断转化"记账(FCS 860 槽):

| arm | 截断数 s15→s20 | 转化槽×其题 rew|ok / 860 预测增益 | 实际增益 |
|---|---|---|---|
| start | 285→215 (−70) | ≈ +1.22 | **+1.12** |
| nom_a5 | 236→179 (−57) | ≈ +0.84 | +0.79 |
| newmt | 324→260 (−64) | ≈ +1.13 | +0.55 |
| wd03 | 274→286 (**+12,倒退**) | ≈ −0.2 | +0.36 |

- **base 后程增益几乎 100% 被"截断转化"记账解释**;nom 同样吻合;wd03 截断率停止下降(31.9%→33.3%,训练端其 response_length 也在回涨:s16 16.8k→s20 19.5k tok)→ FCS 同步停滞。这是 H1 的最强证据。
- 关键判别的答案:**没有哪条臂在 s20 真正"触底"**。截断率 s15→s20 除 wd03 外都仍在下降(base −8.1pt、nom −6.6pt、newmt −7.5pt)。nom 只是**水位最低**(20.8%),不是"更早停止下降"。
- 反事实上限(把所有截断槽替换为该题未截断均值):s20 base 12.40 > newmt 11.99 > nom 11.25 > wd03 10.92 —— 即使**完全中和截断**,base 仍最高。所以 H1 之外还有"每份完成答案的质量"差:s20 rew|ok base=15.1 vs nom=12.6(s0 时两者同为 ~18.9;匹配截断水平对比 base s20(25%)=15.1 vs nom s15(27%)=12.3,仍差 ~2.8,不全是"完成池被稀释"的成分效应)。
- 窗口级 corr(Δ截断, ΔFCS)=0.29(16 个臂×窗):截断释放解释**趋势**,窗口间起伏由彩票/题级噪声主导(见 H4)。

**结论**:所有臂的 FCS 增益主通道都是"学会在 32k 内写完"。base 的后程优势 = 仍有充足截断储备 + 完成答案质量保持在 15 分档;nom 储备水位更低且完成质量掉到 12.6 分档;wd03 后程截断反弹直接停滞。但"提前饱和"表述过强——我们的臂 s20 仍在释放,只是单位释放的兑换率低于 base。

## H2 训练分布上是否同样后程反超 —— **否定(训练端四臂重合)**

训练 reward(dump 复算,四臂同步同题,可配对):

```
step:      1     5    10    15    20
start   .126  .194  .163  .255  .249
wd03    .152  .178  .183  .275  .264
nom_a5  .144  .184  .168  .215  .228
newmt   .136  .190  .166  .282  .256
```
- 四臂曲线基本重合(0.13→0.25),**都在涨、无人平台、base 无后程独涨**。配对差(臂−base,32 题同题)幅度 ±0.03;唯一系统信号:nom_a5 末 6 步配对差持续为负(−0.02~−0.04)——与它 FCS 端 rew|ok 最低相互印证,幅度很小。
- 训练端零分率同步下降(~65%→27–35%),nom 略高(34.8% vs base 28.1% @s20)。

**结论**:eval 端的交叉**不在训练分布上发生**。这否定"优化动力学差异"(熵塌/步长/梯度枯竭)为主因,指向 synth→FCS 迁移 + 评测噪声。

## H4 增益题型分解 + bootstrap —— **支持"彩票+噪声"为交叉的第一主因**

**base s15→s20 的 +1.117 分解**(172 题配对差分):
- top-6 题占 **88%**;其中纯彩票(5 槽 0→恰 1 槽≥50):99/114/162 三题合计 **+0.345(31%)**;prob35(0→2 槽 84/88)+0.20(18%);prob12 是真巩固(3/5 槽 59.5 档 → 5/5 槽 91.9 档,+0.283,25%);prob68/147 为中度巩固。
- 每个窗口每条臂的**毛增益 +1.6~+3.2 / 毛损失 −0.9~−2.3**,净变化 ±1 全在题级 churn 量级内;base 这个窗口的特殊之处不是毛增益大,而是**毛损失端恰好小**(−1.40 vs newmt −1.99)。

**bootstrap(按题重采样 1 万次)**:
- s20 两两差:**全部不可分**。start−newmt +0.33 [−1.05,+1.73];start−nom +1.14 [−0.16,+2.48];start−wd03 +1.21 [−0.07,+2.54]。
- base 自身 s15→s20 的 +1.117:95%CI [−0.22,+2.41],**同样不显著**。
- 各臂对自己 s0 的增益从 s10 起全部显著(base +3.96 [2.33,5.72];newmt +4.81 [2.86,6.90] 最大)。
- 中程"领先"同样不显著:s5 wd03−start +1.17 [−0.21,+2.54];s10 newmt−start +0.72 [−0.42,+1.93]。

**结论**:**"先强后被追上"这个排序故事,在 5 槽/题的评测功率下不可证**。可证的只有:(a) 四臂都被 RL 显著推高;(b) 终点两两差都在噪声带内;(c) base 末窗 +1.11 里彩票+单题跃迁占 ~3/4。

- 与"同一 RL 把两类臂推向不同回报面"一致的证据链是:nom 的 FCS rew|ok 最低(12.6)+ 训练端末段配对差微负 + Research 保持最高;但每一环单独都在噪声边缘。

## H6 entropy/探索 —— **否定(不存在我们臂的熵塌/梯度枯竭)**

- entropy(可比窗口):base 反而**最低**且缓降(s11 0.683→s20 0.641);我们的臂更高(nom 0.69–0.72,newmt 0.69–0.70,wd03 0.66–0.70)。没有任何臂熵塌。
- "有梯度组占比"(32 组中组内 reward 非全同):四臂 s9 后稳定在 **94–100%**,全零组后期 ≤6%。**没有梯度枯竭**;GRPO 组内信号到 s20 都健康。
- 训练响应长度四臂同步从 ~100k chars(>20k tok)降到 58–73k chars(15.8–19.5k tok),clip 比降到 1–5%,写完率(``` 结尾)~97%:是健康的"学会收敛",不是 20k 时代的截断死亡螺旋。

## 机制定论(按证据权重排序)

1. **评测噪声 + 彩票题是"交叉"的第一解释**(H4,证据最硬)。s20 终点 base−newmt = +0.31,95%CI [−1.73,+1.05];base 自己的末窗 +1.11 也不显著,其中 ~31% 是 1/5 槽 0→100 型彩票、~25% 是单题(prob12)档位跃迁。中程"我们领先"同样不显著且每步领先臂不同(3 臂选 1 的选择偏置)。**在当前功率下,"先强后被追上"不构成需要机制解释的现象。**
2. **所有臂共享同一真实增益通道:截断储备释放;差异只在兑换率与储备节奏**(H1,趋势级支持)。base 的末窗增益 ≈100% 由"多写完 70 槽 × 每槽 ~15 分"记账;nom 水位最低(20.8%)且完成答案质量掉档(12.6 vs base 15.1,匹配截断水平后仍差 ~2.8);wd03 后程截断率反弹→停滞。反事实中和截断后 base 仍最高(12.40),即 base 从起点继承的 FCS 单样本质量优势(7.05 档)在 RL 后被保留。
3. **训练端无差异、KL/熵无差异 → 排除优化动力学类假设**(H2/H3/H6,干净否定)。四臂训练 reward/KL/熵/梯度组全程重合;交叉只发生在 eval 侧。加上 H5 的弱方向信号(nom 把增益更多兑换到 Research=15.16,FCS 兑换率最低),剩余的系统性成分是**迁移方向差**而非"提前饱和"。

## 下一轮怎么让我们的臂后程不饱和(可操作)

1. **先加评测功率,再谈机制**:把关键 sync 点(至少 s20/终点)的每题槽数 5→15(或 3 个独立 seed 的 5 槽合并),并在报表里把"彩票题(恰 1/5 槽≥50)"单列;两臂差 CI 收窄 ~√3 后 ±1 的差才可判。现在就下"被反超"结论会把噪声当机制。
2. **直接加步数,四臂都没到头**:s20 没有任何臂截断触底(20.8–33%),反事实上限还有 +1.1~+1.4/臂。优先跑到 s40(save_freq 允许),预算不够就只延 base+newmt 两臂对照。wd03 的截断回涨(训练长度 s16→s20 从 16.8k 回到 19.5k tok)是唯一的"退化苗头",延长时对它加 response-length 监控告警。
3. **起点选 newmt + 更多步**:newmt 增益最大(+4.81,CI 下界 2.86 也是四臂最高),s20 仍有 30% 截断储备,反事实上限 11.99≈base;它是"低起点高斜率"臂,20 步吃亏在起点。若目标是终点 FCS,新一轮用 newmt 起点跑 40 步是最有数据支撑的单一改动。
4. **不要为熵/KL 加复杂度**:熵塌、梯度枯竭、KL 锚差异均被数据否定(20 步尺度)。周期性熵正则/ref 更新解决的是不存在的问题;若延长到 40+ 步再复查 entropy(base 已在缓降,0.68→0.64)。
5. **若要 FCS 兑换率**:nom 的短板是"每份完成答案 12.6 分 vs base 15.1 分"。可在 RL 数据里混入少量 FCS 风格(判题器可跑的 C++ 构造题)或把 synth 里 heuristic-tier 题的奖励整形(target 归一已有,考虑 per-task hi 收紧),让完成质量而非完成率吃梯度。反过来若目标是 Research/MLS,nom 的兑换方向本来就是想要的——**同一条 RL,两个基准的"饱和"结论相反**,先定目标基准再定臂。
6. **评测 infra**:本轮 judge-infra error(base s20 27 个、research 端 5–29 个/次)虽经核对对headline影响 <0.01,但分布不均(27 vs 1);引用任何两臂差前先按 `samples.jsonl` error 数过闸(老规矩,eval-resume-error-zero 教训)。
