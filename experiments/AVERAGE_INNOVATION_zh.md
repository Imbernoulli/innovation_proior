## 0. 一页结论

- **(a) 没有丧失——但要分三层说。** 表层"创新文体"(Key Insight 计划体标记)在 α≤0.2 时确实几乎完全被稀释(保留率 ≈0–1%);但**行为层的创新倾向(提案敢偏离题面 baseline,Jaccard 指标)随 α 单调保留**(α=0.1 保留 ~38–41%、α=0.5 保留 ~81%),**结果层的"能兑现的发现收益"(MLS)在所有 α 上都 ≥ base(1.3–2.9×)**。真正"创新能兑换成分数"的区间恰恰是轻档 soup(α=0.05–0.2)——纯 SFT 的创新是空壳(提案最多、兑现率 1.3%)。
- **(b) SFT 直测差,机制上完全 make sense:不是评测假象,是"代码生成器被打断 + 思考塌缩 20 倍"。** 860 个零分样本里 **83.2% 是真实编译错误**(g++ 实测;干净代码块的编译通过率 17.1% vs base 58.1%),同时思考中位数从 94k 字符掉到 4.5k。抽取失败(0.1%)、截断(2.2%)、重复退化(0%)全部排除。SFT 模型交付纪律反而是最好的(74.9% 带完整代码块、87.8% 闭合 think)——**它交得又快又整齐,但交的是幻觉拼贴的坏代码**。FCS 是"编译不过=0"的兑现型基准,所以掉到 0.31;同一个模型在不要求精确交付的 MLS 上反而 0.081 > base 0.038。
- **(d) 定量曲线见 §2:能力(FCS)随 α 平滑塌(α=0.1 保住 base 的 91%、α=0.3 剩 64%、α=0.5 剩 24%、α=1 剩 4%);创新倾向随 α 单调上升;两者的交换比在 α=0.05–0.1 最划算。**
- **(e) α=0.5 不可作为交付点(在当前数据配方下)**:FCS 0.70–2.22(base 的 10–31%)、ALE 296–328(< base 357)、Research 6.8–7.2(flat 口径 base 10.63 的 64–68%),行为已 SFT 化(思考中位 5.4k 字符、66% 短思考)。唯一例外方向:MLS 0.109(base 2.9×,单种子噪声需谨慎)+ Jaccard 0.154(所有 soup 里最敢偏离)。**且 newmt(修好 maintain loss-mask)臂 α=0.5 仍有 FCS 4.23**——重档塌陷有相当一部分是数据工程 bug,不是 average 的宿命。

### 2.5 对 (a) 的分层回答

1. **表层创新文体:轻档 soup 下确实"看不见"了**(KI 保留 0–1% @ α≤0.2)。如果"创新能力"指"读起来像创新语料",那 α=0.1 的 soup 几乎没有。
2. **行为倾向:没有丧失,是按剂量稀释**(Jaccard 偏离 38–42% @ α=0.1–0.2,81% @ α=0.5)。
3. **可兑现的创新:不但没丧失,反而只有 soup 区间才有**——MLS 全 α ≥ base;ALE 最好档(wd03_a10 416、nom_a5 399)超 base;而 α=1 的"创新"兑现率只有 1.3%(FCS score>0 占比)。加上 RL 链路的既有结论(wd03_a10+RL 三线全超 base+RL,FCS 8.42/ALE 434/Research 14.01),**"SFT 注入 → soup 保全 → RL 放大"里,soup 不是创新的杀手,而是创新能活到 RL 阶段的前提**。
4. 诚实约束(SOUP_TRADEOFF 案例审计,老臂 40 题逐例读原文):在 FCS 上,**没有任何 α 的 soup 产出过"base 提不出的新想法且真得分"的样本**——soup 的 FCS 增益 = 撤销 SFT 损害 + 采样方差。创新先验的"净新增兑现"证据在发现类(MLS DR-learner +375% 案例、TeeMOEA/EBO-M 自命名提案、Jaccard 偏离)和 RL 之后(AHC025 在线学权重),不在 pre-RL 的竞赛题上。

### 3.3 四个假设的裁决

| 假设 | 裁决 | 证据 |
|---|---|---|
| (i) 风格/格式漂移导致抽取失败 | **否** | 抽取不到代码仅 0.1%;围栏率/闭合率反而高于 base |
| (ii) 长度膨胀→截断死亡 | **否(方向反了)** | SFT 截断 2.2%(base 64.5%);SFT 是"想太少",不是"想不完" |
| (iii) 真实生成能力退化 | **是,主因** | 83.2% 零分 = 真编译错误;干净代码编译率 58%→17%;think 20× 缩短且自审("Wait")密度 16.1→0.7 归零 |
| (iv) 数据/训练工件 | **是,显著次因** | ① maintain 数据缓冲:full 臂 SFT 2.42 vs nomaintain 0.31(编译率 29% vs 17%);② **maintain loss-mask 修复(newmt)后 α=0.5 仍有 FCS 4.23**(vs 0.70–2.22);③ LoRA 低秩同数据不塌(FCS 5.3–6.0 @s0.1,CLEAN §2)→ 塌陷 = 全参数大扰动 × 数据落点,而非数据内容不可救;④ 12.2% 样本从不闭合 think、1.8% 交 Python(研究轨迹文体渗漏) |

### 3.4 所以"不 make sense"其实 make sense

FCS 是**兑现型**基准:编译不过 = 0,过了还要打过测试。全参 SFT(lr 5e-6 × 1 epoch)把创新语料的分布"学满"——语料是方法计划 + 研究叙事,**没有"长链推导→可编译 C++ 交付"的闭环示范**——于是模型学会了"快速给出计划 + 立刻交码"的生成程序,把 instruct 模型原有的"边写边验、写 90k 字符再交"的程序覆盖了。能力不是消失而是**换了谱**:同一个 SFT 直测权重在 MLS(容忍不精确交付、奖励研究取向)上 0.081 > base 0.038。这也是为什么 soup 的角色是"把代码手还给它":α=0.1 的 soup 各项行为统计与 base 重合(§2.2),FCS 回到 6.4–6.7,同时 Jaccard 偏离保留 38%。

## 5. 诚实修正与方法学注意

1. **base Research 口径修正**:历史文档的 base=9.08 是"GPU 21 题均值(4.37)与 CPU 43 题均值(13.70)各占一半"的算法;各 soup 格用的是 64 题平权宏平均。**同口径(64 题平权)的 base = 10.63**。据此,"轻档 soup Research 明显 > base"应弱化为"≈ base(a5 11.12 略上,a10/a20 10.2–10.5 略下)";超 base 的 Research 增益要到 RL 阶段才成立(14.01 vs base+RL 11.48)。历史两目录合并核验:64 题集合与各格完全一致。
2. **§6.4/两腔指标是按文档词表的重实现**(原脚本未存档):老臂复算的形状与发表值一致(innov 2.58→1.41 vs 发表 2.17→1.24;KI 比值 12.8× vs 发表 11×),绝对值有 ~15–20% 实现差(rpartition vs split、词边界细节),**同一脚本内的 α 间比较不受影响**;Jaccard 用的是原脚本原函数,BASE=0.279 逐位复现。
3. MLS 单种子、单题波动大(CASE_STUDY §5 三个结构性问题),本文所有 MLS 结论只用于方向判断,且 a50 的 0.109 分摊在 6 个任务上(0.26–0.49),非单题翻盘。
4. FCS "compile_error" 桶对 base 有轻微高估(截断样本的兜底全文抽取会把思考文本当代码送编译),因此 §3 另给了"干净围栏代码编译率"(58.1% vs 17.1%),该对比不受此影响。
