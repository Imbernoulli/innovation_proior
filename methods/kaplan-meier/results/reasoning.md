I start with the survival target `S(t) = P(T > t)`, but I immediately lose the simple empirical fraction because some event times are not observed. If I count a censored time as an event, I make the curve too pessimistic. If I discard that subject, I throw away the fact that it survived to its censoring time. If I keep it in every later denominator, I invent follow-up that I never observed. The data force me to separate two facts: survival before the censoring time is observed, and survival after it is unknown.

So I stop asking for one global denominator at time `t`. The usable denominator has to be local. Just before an observed event time, I can list the subjects still under observation and not yet failed. That is the risk set. A censored subject belongs to earlier risk sets and then leaves later ones. This only makes sense if the censoring mechanism is not selectively removing subjects because of their hidden event times; otherwise the remaining risk set would no longer be representative for the next local question.

The older life-table idea gives me the right shape. Survival over several intervals is a product of conditional survivals: survive the first interval, then survive the second conditional on having reached it, and so on. I do not need to estimate the whole distribution at once. I need to estimate each local chance of not failing among the subjects for whom that local question is actually observed.

Now I refine the intervals until the observed event times themselves define the possible jumps. At a distinct event time `t_j`, let `n_j` be the number still at risk immediately before that time and let `d_j` be the number of observed events at that time. Among the current risk set, the empirical conditional failure fraction is `d_j / n_j`, so the empirical conditional survival factor is `1 - d_j / n_j`. Multiplying these factors through time gives the step curve. Censored-only times contribute a factor of one; mathematically I can omit them from a compressed jump table, but a software event table may keep them with the previous survival value so later denominators are auditable.

This also resolves ties. If several records share the same observed time, I cannot pretend there is a meaningful order inside that time. I count all observations at that time in the risk set just before the time. The observed events at that time determine the drop. Then I remove the whole group, including censored records, before moving to later times.

The no-censoring check is important. With distinct complete event times in a sample of size `m`, the factors are `(m - 1)/m`, then `(m - 2)/(m - 1)`, and so on. After `r` events the product telescopes to `(m - r)/m`, exactly the usual empirical survival fraction. That confirms that the new bookkeeping is not a different estimator in the complete-data case; it is the complete-data curve with the denominator corrected for incomplete follow-up.

One boundary remains: if the last observed lifetime is only a censoring time, the nonparametric curve is not determined beyond that last observation. I can report the value reached at the last observed time, but I should not invent tail behavior after it without extra assumptions or supplementary observation.

The implementation is then just the bookkeeping made explicit. I sort observed times, group equal values, keep `n_at_risk` before each group, multiply survival by `1 - d_j / n_at_risk`, and subtract the whole group from later risk sets. When `d_j` is zero the factor is one, so retaining that time in the output gives a flat row rather than a jump. The mathematical idea is the product of local conditional survival estimates; the code is the event table that keeps those local denominators honest.
