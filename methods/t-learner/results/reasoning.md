Let me start from what I actually have in front of me and what I actually want. I have a table of units: for each one a covariate vector X, a 0/1 treatment flag W, and a single observed outcome Y. What I want, for a brand-new unit with covariates x, is how much the treatment would move its outcome — not the population average, but the effect specific to units that look like x. So my target is tau(x) = E[Y(1) - Y(0) | X = x], where Y(1) and Y(0) are the two potential outcomes, treated and untreated. And immediately I run into the thing that makes this not an ordinary prediction problem: for any one unit I only ever see one of Y(1), Y(0) — the one matching the treatment it happened to get. The difference Y(1) - Y(0) for a single unit is never in my data at all.

So before I get clever, let me ask whether I should even be aiming at the per-unit difference. Call it D = Y(1) - Y(0). Can I estimate D itself? Let me try to construct a counterexample to convince myself one way or the other. Take X ~ Unif[0,1], a fair-coin treatment W ~ Bern(0.5) independent of X, and let the control outcome be a coin flip, P(Y(0)=1) = P(Y(0)=-1) = 0.5. Now I get to pick how Y(1) relates to Y(0), and I'll pick it two different ways. Way one: Y(1) = Y(0), so every unit's effect is D = 0. Way two: Y(1) = -Y(0), so every unit's effect is D in {-2, +2}. Now what does my data look like in each world? I observe Y^obs = Y(W). In both worlds Y^obs is a fair Rademacher draw independent of X and W — the marginal law of the observable triple (Y^obs, X, W) is identical across the two worlds. Yet the per-unit effects are completely different (all zero versus all ±2). No estimator built on observed data can tell these worlds apart, so the per-unit D is simply not identifiable. That settles it: I cannot chase D. But notice tau(x) = E[D|X=x] = 0 in both worlds — the conditional mean is the same, and it's estimable. The fundamental obstruction kills the unit effect but leaves the conditional mean alone.

And I should check that aiming at the conditional mean isn't a consolation prize even if I secretly care about the unit effect. Suppose I train some estimator tauhat on data independent of a fresh target unit, then score tauhat(x) against that unit's true D_i at X_i = x. Decompose the mean-squared error: E[(D_i - tauhat(x))^2 | X_i = x] = E[(D_i - tau(x) + tau(x) - tauhat(x))^2 | X_i = x]. The cross term vanishes because the target-unit residual D_i - tau(x) has conditional mean zero, while tau(x) - tauhat(x) is a function of the training sample and is independent of that fresh target-unit residual once x is fixed. So it splits cleanly: E[(D_i - tau(x))^2 | X_i = x] + E[(tau(x) - tauhat(x))^2]. The first term is the irreducible variance of the unit effect around its conditional mean — nothing I do touches it. The second is exactly the CATE error. So the tauhat that best predicts the unobservable unit effect is precisely the one that best estimates tau. Good — there is no tension; tau(x) is the right and only sensible target.

Now the second difficulty: this is observational data, not a randomized experiment. The treatment was assigned by some mechanism W ~ Bern(e(X)), where e(x) = P(W=1|X=x) is the propensity. If e depends on X — sicker patients more likely to be treated, say — then treated and control units have systematically different covariates, and just comparing the average treated outcome to the average control outcome confounds the effect of treatment with the effect of being the kind of unit that gets treated. I need assumptions that let me undo this. The standard pair: ignorability, (eps(0), eps(1)) ⟂ W | X, meaning once I condition on the covariates the assignment carries no extra information about the potential outcomes — within a covariate cell, treated and control units are exchangeable. And overlap, 0 < e_min < e(x) < e_max < 1 everywhere, so every covariate cell actually contains some treated and some control units; otherwise there are regions where one potential outcome is never observed and the effect there is unidentified.

Here is the lever those assumptions give me, and I want to write it out because I suspect the whole approach is going to hinge on it. Take the observed conditional mean among the treated: E[Y^obs | X = x, W = 1]. On the event W=1 we have Y^obs = Y(1), so this is E[Y(1) | X = x, W = 1]. By ignorability, conditioning further on W=1 adds nothing once we've conditioned on X, so it equals E[Y(1) | X = x] = mu_1(x). Identically, E[Y^obs | X = x, W = 0] = mu_0(x). So each potential-outcome response surface is just the ordinary conditional mean of the observed outcome *restricted to its own arm*. That is a striking simplification: under ignorability and overlap, estimating mu_1 is a plain regression problem on the treated rows, and estimating mu_0 is a plain regression problem on the control rows. No counterfactuals appear inside either regression — each arm only ever uses outcomes it actually observed.

So the most direct thing imaginable: regress the observed outcome on the covariates *separately within each arm*. Let muhat_0 be any regression estimator fit on the control rows {(X_i, Y_i) : W_i = 0}, and muhat_1 any regression estimator fit on the treated rows {(X_i, Y_i) : W_i = 1}, possibly a different kind of estimator for each, and then estimate the effect as their difference, tauhat(x) = muhat_1(x) - muhat_0(x). Each piece is identified by the argument above; their difference estimates mu_1(x) - mu_0(x) = tau(x). And what makes this appealing is that "any regression estimator" really means *any* — trees, forests, gradient boosting, OLS, a neural net, k-NN. The causal content would live entirely in the act of splitting by arm and differencing; the estimation of each surface gets handed off to whatever supervised method best fits that arm's data. So the candidate is to turn an off-the-shelf supervised learner into a CATE estimator by wrapping it in two-arm differencing.

Before I commit to that, I should take seriously the alternative that looks simpler and ask whether it isn't actually better: pool everything into one regression of Y^obs on (X, W) jointly, treating W as just another coordinate of the feature vector, and read off the effect as muhat(x, 1) - muhat(x, 0). That uses all N rows in one model, which sounds statistically more efficient, and it lets the model share whatever structure is common to both arms. So why would I split? My worry is this. If I hand W to a flexible learner as one feature among many, the learner is free to decide W doesn't matter much. Concretely, picture a tree-based learner where the covariates X are strongly predictive of the outcome and the treatment W is only weakly predictive — exactly the observational regime where the outcome is mostly driven by who the unit is. A greedy tree picks the split that most reduces outcome error at each node; if splitting on some X-coordinate buys more than splitting on W, it takes the X-coordinate, and whenever a path never splits on W, the predicted outcome is identical for w=0 and w=1, so the implied effect is exactly 0 on that region. That is a clean prediction I can test rather than just assert, so let me actually build the case and look at the numbers.

I set up N=2000 units, X uniform on [0,1], W a fair coin, and an outcome with a big smooth signal in X and a small constant treatment effect: Y = 3·sin(6X) + 0.5·W + tiny noise, so the true tau is 0.5 everywhere. I fit a single depth-4 regression tree on (X, W) and read off muhat(x,1) - muhat(x,0) on a grid. The pooled estimate comes back with mean tau_hat ≈ 0.059 against a truth of 0.5 — and when I look point by point, 87.5% of the grid points have tau_hat *exactly* 0. Inspecting which features the tree split on: it split on X twelve times and on W only three times. So the prediction was right and the magnitude is worse than I'd guessed — the pooled model didn't just shrink the effect, it zeroed it out across most of the covariate space, because W kept losing the greedy competition for splits and on any path that never splits on W the two predictions are bit-for-bit identical. As a control, I fit two depth-4 trees, one per arm, and difference them: mean tau_hat ≈ 0.493, essentially the truth. So separating the arms recovers the effect that pooling destroyed, on the very same data. That is concrete enough to make me trust the two-model route as the default rather than pooling.

But the same experiment tells me when pooling is *better*, and I don't want to overstate the case against it. The shrinkage I just watched is harmful only because the true effect is genuinely 0.5; if the true effect really were zero or nearly zero over much of the space, that same collapse toward zero would be regularizing toward the truth and would beat the two-model approach. So I'm not claiming the pooled model is wrong — I'm seeing precisely when each is right. The two-model approach forces the two surfaces to be estimated independently, so the collapse I measured cannot happen merely because W lost the split competition; that's its strength when the effect is genuinely present and complex. Its weakness is the mirror image: by refusing to pool, it can't borrow strength across arms to mimic structure that both surfaces share, and it pays full sampling cost in each arm separately.

Let me dwell on that weakness because it tells me when to be careful, not just that there's a cost. Suppose mu_0 and mu_1 both have, say, a sharp jump at the same place — some shared discontinuity driven by a covariate that affects the baseline outcome the same way in both arms. The pooled model can capture that jump once, from all the data, and have it cancel in the difference. The two-model approach has to discover the jump twice, independently, once in each arm, from each arm's data alone. If one arm is small, it may estimate the jump badly there, and the difference muhat_1 - muhat_0 will show a spurious jump in tau even though the true tau is smooth across it. That is the heart of the danger: artifacts in the two separately-estimated surfaces do not cancel, because nothing ties the two fits together. And the most common way one arm is small is the unbalanced design — administrative or online data give an enormous control group and a small treated group, or vice versa. There the small arm's surface dominates the error of the difference.

There's a practical corollary I should bank: because each arm is fit on its own sample, I must match each base learner's complexity to its own arm's size. If the treated arm has ten points I had better not fit it a wiggly high-variance model, even if the control arm has ten thousand and can support something complex. The freedom to use a *different* base learner per arm isn't a frill — it's the right response to the two arms typically having different sample sizes and possibly different surface complexity. Subject knowledge about each surface goes into each arm's learner independently.

Now I want a guarantee, because "fit two regressions and subtract" is so simple that I should be able to say exactly how good it is and exactly where its ceiling is. Let me set up the rate language. For a plain regression problem, the worst-case error over a family F of distributions is the minimax EMSE; say F is in the class S(a) if some estimator achieves sup over F of EMSE <= C N^{-a}. Parametric families (linear response, OLS) live in S(1), the N^{-1} rate. Lipschitz response functions on [0,1]^d live at the slower rate N^{-2/(2+d)}. I need a concrete instance, so let me derive that Lipschitz rate for k-nearest-neighbors, since I'll lean on it. For an L-Lipschitz mu on Unif([0,1]^d) with noise variance sigma^2, k-NN averages the outcomes of the k nearest points. The variance of that average is sigma^2/k — averaging k independent noises. The bias comes from the neighbors being a distance away: in d dimensions the k nearest of N points sit at radius on the order of (k/N)^{1/d}, and Lipschitz-ness turns distance into outcome error L·(k/N)^{1/d}, squared gives L^2 (k/N)^{2/d}. So E[||muhat - mu||^2] <= sigma^2/k + c L^2 (k/N)^{2/d}. Minimize over k: the two terms balance when sigma^2/k = L^2 (k/N)^{2/d}, i.e. k^{1+2/d} ~ (sigma^2/L^2) N^{2/d}, so k ~ N^{2/(2+d)}, and plugging back should give EMSE ~ N^{-2/(2+d)}.

Let me not just trust the algebra; let me actually minimize that bound numerically and read off the scaling. Take sigma^2 = c = L = 1, d = 1, and for each N grid over k = 1..N and find the argmin and the minimum value. At N = 100 the minimizing k is 17 (the formula N^{2/3} predicts 21.5); at N = 1000 it is 79 (predicted 100); at N = 10000 it is 368 (predicted 464); at N = 100000 it is 1710 (predicted 2154). So argmin k grows like N^{2/3} up to a constant factor, as it should. And the minimized bound: f* / N^{-2/3} comes out to 1.890 at *every* one of those N — a flat constant across four orders of magnitude. A constant ratio against N^{-2/3} is exactly what "the EMSE decays at rate N^{-2/3}" means, so the d=1 Lipschitz rate checks out and S(2/(2+d)) is the right slot for the Lipschitz-d family.

Now lift this to the causal problem. A distribution P of (Y(0), Y(1), X, W) — I'll say it's in the class S(a_mu, a_tau) if ignorability holds, the control regression family (law of (X, Y(0)) given W=0) is in S(a_mu), the treated regression family (law of (X, Y(1)) given W=1) is in S(a_mu), and the two effect-imputation families are in S(a_tau). The two-model approach only ever touches the first two; it has no step that learns from the effect-imputation families. So its general rate should be governed by a_mu, the response-surface rate, not by a possibly faster effect rate a_tau. Let me try to prove the bound EMSE(tauhat) <= 2C[(1-e_min)/(1-e_max) + e_max/e_min](n^{-a_mu} + m^{-a_mu}), where n is the number of treated and m the number of control units, and see whether that intuition survives a careful derivation.

Start with the algebraic split. For any reals, (a-b)^2 <= 2a^2 + 2b^2. Apply it to the error of the difference: with tauhat - tau = (muhat_1 - mu_1) - (muhat_0 - mu_0),

  (tauhat(X) - tau(X))^2 <= 2(muhat_1(X) - mu_1(X))^2 + 2(muhat_0(X) - mu_0(X))^2.

Take expectations, and define

  A = E[(muhat_1(X) - mu_1(X))^2 | sum_i W_i = n],
  B = E[(muhat_0(X) - mu_0(X))^2 | sum_i W_i = n].

Then EMSE(tauhat) <= 2A + 2B. So if I can show A = O(n^{-a_mu}) and B = O(m^{-a_mu}), the factor 2 from the square split is already accounted for at the end. Focus on A.

Here's a subtlety I didn't see coming and have to deal with. To state a clean rate in terms of n and m, I want to condition on the number of treated units being exactly n — otherwise n is itself random, and with small probability could be 0, which is degenerate. But conditioning on sum_i W_i = n breaks the independence of the rows: knowing one unit is treated changes the conditional probability that another is, since the total is fixed. So my n treated rows are no longer obviously i.i.d., and minimax rates are stated for i.i.d. samples. I can recover independence by conditioning not on the *count* n but on the *entire* assignment vector W = w (with sum w_i = n). Once the whole assignment is fixed, which rows are treated is no longer random, and I can ask about the joint law of the (X_i, Y_i) at the treated positions.

Let me prove they're i.i.d. from P_1, the law of (X, Y) given W=1. Two parts: independence and identical distribution. For independence, take any subset I of treated indices and events B_i. I want P(∩_{i in I} {(X_i,Y_i) in B_i} | W = w) to factor. Write it as a ratio with the event {W=w} = ∩_{j: w_j=1}{W_j=1} ∩ ∩_{k: w_k=0}{W_k=0}. Because the original rows (X_i, W_i, Y_i) are mutually independent under P, the numerator factors across indices, and the W-events on the indices not in I detach as their own product. After dividing by P(W=w), the terms for indices outside I cancel, and each treated i in I contributes P((X_i,Y_i) in B_i, W_i=1) / P(W_i=1) = P((X_i,Y_i) in B_i | W_i=1). That product over i is exactly the factorization I needed. For identical distribution, take a single treated index i and an event B. Because rows are mutually independent, conditioning on the full vector W=w is the same as conditioning on W_i=1 alone for that one row, so P((X_i,Y_i) in B | W=w) = P((X_i,Y_i) in B | W_i=1). And because the original rows are identically distributed, this is the same number for any two treated indices i, j. Hence every treated row has law P((X,Y) in · | W=1) = P_1, and they are i.i.d. Good — independence recovered by conditioning on the whole assignment.

Now muhat_1 depends only on the n treated rows, which are i.i.d. from P_1. By the assumption that the treated family is in S(a_mu), the minimax base learner estimates its conditional mean at rate n^{-a_mu} — but measured against the covariate law *inside* P_1, i.e. the distribution of X given W=1. The EMSE I actually want is against the *target* marginal of X (a fresh test point drawn from the population, not from the treated subpopulation). Those two covariate laws differ exactly because of confounding: treated units have a tilted covariate distribution. I need a change-of-measure bound to convert one to the other, and overlap is what makes it possible.

Let me prove the change-of-measure lemma. For any positive function g,

  E[g(X) | W = 1] = E[g(X) e(X)] / E[W],

since E[g(X) 1{W=1}] = E[g(X) E[1{W=1}|X]] = E[g(X) e(X)] by the definition of e. Now bound e(X) inside the numerator between e_min and e_max, and E[W] = E[e(X)] likewise between e_min and e_max. The lower bound: E[g(X)|W=1] >= E[g(X)] e_min / E[W] >= E[g(X)] e_min / e_max. The upper bound: E[g(X)|W=1] <= E[g(X)] e_max / E[W] <= E[g(X)] e_max / e_min. So

  (e_min/e_max) E[g(X)] <= E[g(X) | W = 1] <= (e_max/e_min) E[g(X)],

and a mirror-image argument with 1-e in place of e gives, for the control side,

  ((1-e_max)/(1-e_min)) E[g(X)] <= E[g(X) | W = 0] <= ((1-e_min)/(1-e_max)) E[g(X)].

I want to make sure I didn't drop a factor or flip an inequality, so let me put real numbers through it. Take X uniform on four points {0,1,2,3}, propensities e = (0.2, 0.4, 0.6, 0.8) so e_min=0.2, e_max=0.8, and a positive test function g(x) = (x+1)^2 = (1,4,9,16). Then E[g] = 7.5 and E[W] = E[e] = 0.5. Compute E[g|W=1] = E[g·e]/E[W] = (0.2+1.6+5.4+12.8)/4 / 0.5 = 5.0/0.5 = 10.0. The lemma's bracket is (e_min/e_max)·7.5 = 1.875 up to (e_max/e_min)·7.5 = 30.0, and 1.875 <= 10.0 <= 30.0 holds. For the control side, E[g|W=0] = E[g·(1-e)]/E[1-e] = 5.0, and the control bracket ((1-e_max)/(1-e_min))·7.5 = 1.875 up to ((1-e_min)/(1-e_max))·7.5 = 30.0 also contains 5.0. Both inequalities hold with room to spare, and the brackets came out symmetric between the two arms exactly as the algebra says they should — so I trust the constants e_max/e_min and (1-e_min)/(1-e_max).

Apply the lemma with g(X) = (muhat_1(X) - mu_1(X))^2: the EMSE of muhat_1 under the target marginal is at most (e_max/e_min) times its EMSE under P_1's covariate law, which is at most C n^{-a_mu}. Putting it back through the iterated expectation over W (the bound holds for every w, so it survives averaging over w),

  A <= (e_max/e_min) C n^{-a_mu}.

By the symmetric argument on the control arm, using the control change-of-measure constant,

  B <= ((1-e_min)/(1-e_max)) C m^{-a_mu}.

So altogether

  EMSE(tauhat) <= 2C (e_max/e_min) n^{-a_mu} + 2C ((1-e_min)/(1-e_max)) m^{-a_mu}
              <= 2C [ (1-e_min)/(1-e_max) + e_max/e_min ] (n^{-a_mu} + m^{-a_mu}).

So my intuition survived the derivation: the rate is the response-surface rate a_mu in each arm's own sample size, and the overlap constants are the price of confounding — they blow up as e approaches 0 or 1, i.e. as overlap fails, which is precisely when the tilt between the treated covariate law and the target law becomes extreme, and precisely where the change-of-measure brackets I just computed would widen toward infinity. To make it tangible, take Lipschitz response surfaces in dimension d, where a_mu = 2/(2+d): pick k-NN as the base learner in each arm with k_1 ~ n^{2/(2+d)} for the treated arm and k_0 ~ m^{2/(2+d)} for the control arm — exactly the bandwidths I verified numerically above — and the differenced estimator achieves O(n^{-2/(2+d)} + m^{-2/(2+d)}). That is the minimax-optimal rate for that family, so on Lipschitz surfaces the two-model approach is not just consistent, it is rate-optimal.

But now stare at the bound and feel where it's loose, because the "in general we cannot expect faster" is the real content. The rate is a_mu, the rate of estimating the response surfaces. It does not become a_tau, the rate of estimating the effect — and a_tau can be much faster. Why would the effect be easier to estimate than the surfaces? Because the effect is often simpler than either surface. Imagine both mu_0 and mu_1 are wildly nonlinear and complicated (slow rate a_mu), but they differ by a constant, or by a linear function — so tau is constant or linear (fast rate a_tau = 1). The two-model approach is blind to this: it estimates each complicated surface at the slow rate a_mu and differences them, and the slow errors do not magically cancel just because the true difference is simple. It is paying for the complexity of the surfaces when it should be paying only for the simplicity of the effect. And there is no knob inside the two-model recipe to fix this, because the two arms are estimated in total isolation — each subproblem is solved without any reference to the other, so there is nothing in the procedure that could notice "the difference is simpler than the parts" and exploit it. That blindness is structural, baked into the decision to never let the arms talk. It is most painful exactly in the unbalanced regime I flagged earlier: when m >> n, the control surface mu_0 is essentially known, the error is dominated by the treated-arm term n^{-a_mu}, and yet if tau is simple I still have no general route to the fast effect rate. The bound makes this precise as a guarantee: the two-model approach inherits the response-surface rate, and any faster behavior would have to come from extra structure outside this isolated two-regression recipe. I have to record that limitation as part of the method, not hide it behind the clean rate.

For now, though, I have a clean, general estimator — rate-optimal when the effect is as hard as the surfaces, and crucially one that converts any supervised learner into a CATE estimator. Let me also be honest about what I am not doing relative to the other things on the table. The single-pooled-model approach I already dissected and measured: it can attenuate a real effect nearly to zero, which separating the arms structurally avoids. The transformed-outcome approach builds a single response Y^* whose conditional mean is tau by reweighting with the propensity — Y^* = Y^obs (W - e(X)) / (e(X)(1 - e(X))) — and regresses it directly; clever, but it divides by e(1-e), so whenever the propensity is near 0 or 1 (exactly where overlap is weak) the transformed outcome has enormous variance, so I'm wary of it as a default. The causal-tree approach commits to a single honest tree-structured partition; it is elegant and gives valid intervals, but it ties me to one model class for the whole problem rather than letting me slot a different best-fit learner into each arm. The two-model differencing keeps maximal flexibility — one regressor per arm, each chosen freely — at the price of the no-pooling weakness I've characterized.

So let me land it as code, filling the open slot in the estimator I sketched. The contract was fit(X, T, Y) returning self and predict(X) returning one effect per row, with off-the-shelf regressors available. The whole method is: keep two regressors; in fit, mask the rows by arm and fit each regressor on its arm's observed outcomes; in predict, return the difference of the two predictions over all rows. Because each arm is fit separately, I clone a fresh learner for each arm instead of reusing one fitted object. A gradient-boosting regressor is a fine concrete base learner — flexible, handles nonlinear surfaces, and I can regularize it through the usual regression hyperparameters — but nothing in the method depends on that choice; any sklearn-style regressor drops in.

```python
import numpy as np
from sklearn.base import clone
from sklearn.ensemble import GradientBoostingRegressor


def make_base_regressor(seed):
    return GradientBoostingRegressor(random_state=seed)


class CATEEstimator:
    """T-learner ('T' = two): fit one outcome model per treatment arm and
    estimate the CATE as their difference.

    Under ignorability and overlap, E[Y^obs | X=x, W=w] = mu_w(x), so each
    response surface is a plain regression on that arm's observed rows:
      mu0_hat on the control rows {(X_i, Y_i): T_i = 0},
      mu1_hat on the treated rows {(X_i, Y_i): T_i = 1},
    and tau_hat(x) = mu1_hat(x) - mu0_hat(x). Any supervised regressor may be
    used as a base learner, possibly a different one per arm; here a gradient
    boosting regressor, with one independent instance per arm.
    """

    def __init__(self, base_regressor=None):
        self._base_regressor = (
            make_base_regressor(seed=42) if base_regressor is None else base_regressor
        )
        self._models = {}

    def fit(self, X, T, Y):
        X, T, Y = np.asarray(X), np.asarray(T), np.asarray(Y)
        mask0 = T == 0                     # control rows -> mu0
        mask1 = T == 1                     # treated rows -> mu1
        self._models[0] = clone(self._base_regressor)
        self._models[1] = clone(self._base_regressor)
        self._models[0].fit(X[mask0], Y[mask0])
        self._models[1].fit(X[mask1], Y[mask1])
        return self

    def predict(self, X):
        X = np.asarray(X)
        # tau_hat(x) = mu1_hat(x) - mu0_hat(x), evaluated at every row.
        return self._models[1].predict(X) - self._models[0].predict(X)
```
