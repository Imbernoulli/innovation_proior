Text is cheap and arrives in floods, but the thing that actually costs money is the human annotator who tells me which class each piece of text belongs to. The text is free; the labels are not. So the problem I am solving is not "fit a classifier" but "given that the oracle will only label a fixed number $n$ of examples, which examples do I ask about?" Every label I spend is a label I cannot spend elsewhere, and the naive answer — draw examples to label uniformly at random from the pool — spends them badly. The failure is sharpest exactly where it matters. If the category I care about is rare, say one example in a thousand is positive, then five hundred random labels buy me roughly five hundred negatives and essentially no positives, and I cannot teach a classifier to recognize a class from a sample that contains none of it. Even setting rarity aside, the marginal value of one more random label decays as the training set grows: most examples drawn at random are ones the current classifier already gets right, so labeling them only confirms what I know. The budget drains into the easy, settled part of the input space while the informative examples — the ambiguous cases near the boundary, the rare positives — almost never get drawn.

The clean theoretical account of "informative" comes from viewing learning as search through a version space, the set of all hypotheses in my model class still consistent with the labels seen so far. Each new label must agree with the true concept, so it carves that set down, and the question "which example is worth labeling" becomes "which example, once labeled, carves off the most." The object that pins this down is the region of uncertainty $R = \{x : \exists\, c_1, c_2 \in C \text{ consistent with every label so far},\ c_1(x) \neq c_2(x)\}$. If $x$ lies outside $R$, every consistent hypothesis already agrees on it, so labeling it contradicts none of them and the version space does not shrink — the label is wasted. If $x$ lies inside $R$, then whatever the true label turns out to be, it kills every consistent hypothesis that guessed wrong. So an example informs me only when it lies in $R$, and the mass $\alpha = \Pr[x \in R]$ does double duty: it bounds the error of any consistent hypothesis (disagreement with the target lives entirely inside $R$, so error $\leq \alpha$), and it is the probability a random draw lands somewhere useful. As I label more, $R$ shrinks, $\alpha$ shrinks, and a random draw lands in the useful region with probability $\alpha \to 0$ — that is the decaying efficiency, now named. The fix in principle is to query only points inside $R$. The trouble is that every direct route to "query inside $R$" breaks on a real text task. Membership queries, where the learner synthesizes the ideal point to ask about, are enormously query-efficient in theory but produce gibberish on text: a synthesized point in word-feature space is a meaningless bag of words no human can label. Computing $R$ exactly requires maintaining the entire version space and testing each candidate for two disagreeing consistent hypotheses — intractable for logistic-regression-style models over thousands of word features, recomputed every round, and it yields a binary in/out test rather than the ranking I need to spend a budget. Query-by-committee approximates $R$ by drawing two hypotheses from the version space and querying where they disagree, but it assumes noise-free realizability (a perfect consistent classifier exists), assumes I can sample the version space uniformly, and demands a committee on top of the single classifier I am already training — and on noisy, messy text none of the three hold; the version space can even be empty.

What the committee's disagreement actually measures is the key. Two consistent hypotheses split on $x$ precisely when the evidence so far does not determine its label — where, forced to commit, I would be guessing. But my single trained probabilistic classifier already reports that directly: its posterior is its own degree of belief about the label of $x$. Where two consistent hypotheses would split, a well-trained classifier sits near the fence; where they all agree, it is confident. So the classifier's own uncertainty about its prediction is a single-model, scalar surrogate for "$x$ is in the region of uncertainty," and it dodges all three cracks at once — one classifier, no realizability assumption (a posterior is defined whether or not a perfect classifier exists), and a number I can sort. I propose this acquisition rule, least-confidence sampling, the canonical instance of uncertainty sampling. Concretely, in the two-class case the classifier estimates $P(C \mid x)$, and under equal misclassification costs the decision boundary sits at $P(C \mid x) = 0.5$; the model is least certain exactly there, where the two outcomes are equiprobable, so I query the examples whose posterior is nearest $0.5$. To generalize past two classes, picture the classifier choosing $\hat{y} = \arg\max_y P(y \mid x)$ and acting with confidence $P(\hat{y} \mid x)$. It is most certain when that top-class mass is near $1$ and least certain when it is small — even the best class barely wins, the mass is spread, the model is near guessing. So the score is

$$\mathrm{uncertainty}(x) = 1 - P(\hat{y} \mid x) = 1 - \max_y P(y \mid x),$$

and I query the $n$ examples that maximize it, equivalently the $n$ with the smallest top-class posterior:

$$x^*_{\mathrm{LC}} = \arg\max_x \big[\, 1 - \max_y P(y \mid x) \,\big].$$

This is genuinely the same idea generalized, not a new heuristic: with two classes the larger posterior is $\max(p, 1-p) = 0.5 + |p - 0.5|$, so minimizing the top-class probability is exactly minimizing the distance from $0.5$. And $1 - \max_y P(y \mid x)$ has a reading I like even more than "distance to the boundary": if the model commits to $\hat{y}$ it is wrong whenever the true class differs, and under its own posterior that probability is exactly $1 - P(\hat{y} \mid x)$ — the model's own estimate of its 0/1 loss on $x$. I am querying the examples it believes it is most likely to get wrong, which are precisely the ones whose true labels would correct it, and it costs nothing beyond the forward pass I already run to classify.

I considered scoring with more of the distribution. The margin between the top two classes, $P(\hat{y}_1 \mid x) - P(\hat{y}_2 \mid x)$, uses the runner-up that least confidence discards and prefers examples where the model is torn between two specific classes; entropy $-\sum_y P(y \mid x)\log P(y \mid x)$ uses the whole distribution and extends to structured or multi-label outputs. In the binary case all three give the same ordering, since the top probability, the gap to the runner-up, and the entropy are all monotone in $|p - 0.5|$; they diverge only with three or more classes, where entropy is the more natural choice under log-loss while least confidence and margin tie more directly to decision uncertainty under 0/1 classification error. For the cleanest rule I want the minimal, most direct form — the one that scores nothing but the top-class probability the classifier already exposes and that equals the model's misclassification probability — so least confidence is the rule, with margin and entropy as refinements to reach for only if it leaves value on the table. The selection is only half of it: the classifier scoring the pool is the one trained on the labels I have so far, and after labeling this round's batch I retrain on the enlarged set and that classifier scores the next round. So it is sequential and self-correcting — a deficient classifier this round picks examples that, once labeled, tend to compensate for its deficiency next round, the exploratory feedback that makes it work. One caveat falls out of this: the initial classifier matters, because starting from nothing the uncertainty score is meaningless until the model stumbles onto a positive, so I seed the loop with a handful of known positives plus a few random examples to give the score something real to point at from the first round. Checked against the rare-class disaster that started everything, the rule does what I wanted: once the classifier has even a weak notion of the positive class, the examples it finds borderline cluster around the boundary between the rare positives and the confusable negatives, so the budget chases the contested region instead of re-confirming the bulk of obvious negatives.

```python
import numpy as np
from .strategy import Strategy


class LeastConfidence(Strategy):
    def __init__(self, X, Y, idxs_lb, net, handler, args):
        super(LeastConfidence, self).__init__(X, Y, idxs_lb, net, handler, args)

    def query(self, n):
        idxs_unlabeled = np.arange(self.n_pool)[~self.idxs_lb]
        probs = self.predict_prob(self.X[idxs_unlabeled],
                                  np.asarray(self.Y)[idxs_unlabeled])
        U = probs.max(1)[0]                       # confidence in the predicted class
        return idxs_unlabeled[U.sort()[1][:n]]    # n least confident (smallest max-prob)
```
