## Research question

Can a single learning system acquire good control policies **directly from high-dimensional raw sensory input** — the actual pixels on a screen — using nothing but a reward signal, with no hand-engineered features and no per-task tuning?

The concrete target is the Atari 2600 console viewed through an emulator. At each step the agent sees a 210×160 RGB frame at 60Hz and must pick one of up to 18 joystick/fire actions; the only feedback is the change in game score. The agent is not given the emulator's internal state, nor any game-specific knowledge — exactly the situation a human faces sitting down at an unfamiliar machine. A solution would have to:

- map raw pixels to actions without a human ever specifying what objects or features matter;
- cope with a reward that is sparse, noisy, and delayed by up to thousands of frames between an action and its scored consequence;
- use **one** architecture and **one** set of hyperparameters across many different games, so that the result is a general learning method rather than a hand-built solver for a single game.

Why it matters: every prior success at controlling agents from rich sensory streams leaned on hand-crafted features feeding a *linear* value function or policy. The performance ceiling of those systems is set by the quality of the features a human can invent. If a system could learn its own features from pixels, that ceiling would lift, and the same method would transfer across tasks.

## Background

**The deep-learning backdrop.** By 2012–2013, deep convolutional networks trained by stochastic gradient descent on raw inputs had overtaken hand-engineered pipelines in computer vision (Krizhevsky, Sutskever & Hinton 2012; Sermanet et al. 2013) and speech (Dahl et al. 2012; Graves et al. 2013). The lesson: given enough data and a deep enough network, learned representations beat handcrafted ones. These successes share three ingredients — raw inputs, lightweight SGD updates, and very large *labeled* training sets.

**The reinforcement-learning framework.** An agent interacts with an environment in a sequence of actions, observations, and rewards, seeking to maximize the future discounted return R_t = Σ_{t'≥t} γ^{t'-t} r_{t'}. The optimal action-value function

  Q*(s,a) = max_π E[ R_t | s_t=s, a_t=a, π ]

is the maximum expected return obtainable after seeing s and taking a. It obeys the Bellman optimality equation

  Q*(s,a) = E_{s'}[ r + γ max_{a'} Q*(s',a') | s,a ].

Treating this identity as an iterative assignment Q_{i+1}(s,a) = E[ r + γ max_{a'} Q_i(s',a') ] gives value iteration, which converges to Q* in the tabular case (Sutton & Barto 1998).

**Why partial observability forces a history.** A single Atari frame does not reveal velocities or directions, and many emulator states look identical on screen (perceptual aliasing). The task is therefore partially observed. One standard remedy is to define the state as the whole sequence of past frames and actions, s_t = x_1,a_1,…,a_{t-1},x_t; under this definition the problem is a large but finite Markov decision process, and ordinary MDP methods apply.

**The function-approximation problem and the stability danger.** Tabular value iteration keeps a separate estimate per state, with no generalization — hopeless when each state is a long visual history. The remedy is a parametric approximator Q(s,a;θ) ≈ Q*(s,a). The prevailing wisdom at the time, however, was that combining value-based RL with *nonlinear* approximators was dangerous:

- Tsitsiklis & Van Roy (1997) analyzed temporal-difference learning with function approximation and showed it can **diverge** when learning off-policy, while converging in the on-policy linear case.
- Baird (1995) exhibited a small counterexample on which off-policy TD with linear approximation sends the weights to infinity.

The shared diagnosis is that three properties together court divergence: **function approximation**, **bootstrapping** (updating an estimate toward a target built from other estimates), and **off-policy** learning (training on data generated by a different policy than the one being evaluated). Generalization is the double edge: an update at one state also shifts the value at neighboring states through shared parameters; when those shifted values feed back into the bootstrapped targets, the loop can amplify rather than settle. The community's response was largely to retreat to **linear** approximators with provable guarantees.

**Two deep-learning frictions specific to RL.** Beyond divergence, applying SGD-trained deep nets to RL hits problems that supervised vision/speech do not:

1. The data are a stream of **highly correlated** consecutive frames, violating the near-independence that SGD relies on.
2. The data distribution is **non-stationary**: as the policy improves, the states it visits change. Worse, the regression *targets* themselves move during training, because the target r + γ max_{a'} Q(s',a';θ) is built from the very weights being updated — unlike supervised learning, where labels are fixed before training begins. It is easy to see a feedback loop: if the greedy action favors moving left, the training data fill with left-side states; if the greedy action later flips to the right, the training distribution lurches with it, and parameters can get stuck or oscillate.

**A motivating precedent and its apparent limits.** The best-known RL success of the era was TD-Gammon (Tesauro 1995): a multilayer perceptron estimating the state-value V(s), trained *on-policy* by temporal-difference learning from self-play, reaching super-human backgammon. But attempts to reproduce the recipe in chess, Go, and checkers largely failed, fostering a belief that backgammon was a special case — the dice make the value function smooth and aid exploration (Pollack & Blair 1996). So the one shining example of neural-net RL was widely read as not generalizing.

**Diagnostic pressure on progress metrics.** Average episode reward is a very noisy training signal: tiny weight changes can swing the distribution of visited states and hence the score sharply. A steadier window onto learning would hold the states fixed and track the agent's own predicted action-value, for example the average of max_a Q(s,a) over a fixed set of states collected before training.

## Baselines

**Tabular Q-learning (Watkins & Dayan 1992).** The sample-based, model-free, off-policy realization of value iteration. From a single transition (s,a,r,s') it nudges the estimate toward the one-step target:

  Q(s,a) ← Q(s,a) + α [ r + γ max_{a'} Q(s',a') − Q(s,a) ].

It is *off-policy* (it learns about the greedy policy a=argmax_a Q while the behavior policy explores, typically ε-greedy) and *model-free* (it never builds a model of the environment). It converges to Q* with suitable step sizes — but only in the tabular setting, with one stored value per state. **Gap:** no generalization; cannot touch a pixel-sized state space.

**Q-learning with a parametric approximator (the loss view).** Replace the table by Q(s,a;θ). For a transition (s,a,r,s',done), the one-step target formed from the online parameters is

  y(θ) = r                                      if done,
  y(θ) = r + γ max_{a'} Q(s',a';θ)             otherwise.

The semi-gradient loss treats that target as a constant while differentiating the prediction:

  ℓ(θ) = 1/2 (sg[y(θ)] − Q(s,a;θ))²,
  ∇_θ ℓ = −(y − Q(s,a;θ)) ∇_θ Q(s,a;θ) = (Q(s,a;θ) − y) ∇_θ Q(s,a;θ).

Thus a gradient-descent step moves θ in the direction +(y − Q(s,a;θ))∇_θ Q(s,a;θ): if the prediction is below the target, the update raises it, and if it is above the target, the update lowers it. The target's dependence on parameters is deliberately not differentiated. **Gap:** with a nonlinear Q this combines function approximation, bootstrapping, and off-policy data — no convergence guarantee, and empirically prone to divergence — which is why the field had avoided it.

**Gradient-TD / GQ (Sutton, Maei et al. 2008–2010).** A family of methods derived as true gradients of the mean-squared projected Bellman error, restoring provable off-policy convergence under function approximation. Maei et al. (2009) extended convergence to smooth *nonlinear* approximators — but only for policy *evaluation* (a fixed policy). Maei et al. (2010), Greedy-GQ, reached *control* — but only with *linear* approximation and a restricted Q-learning variant. **Gap:** no member of the family handled nonlinear *control*, the case needed for a deep net that also chooses actions.

**Neural Fitted Q Iteration — NFQ (Riedmiller 2005), and Deep Fitted Q (Lange & Riedmiller 2010).** The closest prior in spirit. NFQ minimizes the same squared-error loss sequence, but as a *batch* method: at each iteration it re-fits the whole action-value approximator (with RPROP) on the entire collected dataset. It also parameterizes Q with the (state, action) pair as input and a scalar output, so computing the values of all actions costs one forward pass per action. DFQ added a deep autoencoder to first learn a low-dimensional representation, then ran NFQ on top. **Gaps:** the batch cost per iteration scales with the dataset, so it does not scale to large data or online operation; the (s,a)→scalar parameterization is wasteful when many actions must be compared; and DFQ's two-stage pipeline (autoencoder, then control) is not end-to-end from pixels, so the features are not shaped by what matters for choosing actions.

**TD-Gammon (Tesauro 1995).** MLP value function trained on-policy by TD from self-play; the proof that a neural net *can* be the value function. **Gap:** on-policy and online with every-sample updates (correlated data), and widely believed special to backgammon.

**Lin (1993).** An early study of reusing past transitions when training a small neural-network controller. **Gap:** demonstrated only from low-dimensional, hand-prepared state — not raw pixels — and not at the scale of a deep convolutional net.

**Prior agents on the Atari testbed.** The Arcade Learning Environment (Bellemare et al. 2013) established Atari as a benchmark and ran Sarsa with linear value functions over hand-engineered feature sets. *Contingency* (Bellemare et al. 2012) added a learned representation of the screen regions the agent controls. Both inject substantial visual prior knowledge — background subtraction and treating each of the 128 palette colors as its own channel (close to a per-object-type binary map). *HyperNEAT* (Hausknecht et al. 2013) evolved a separate network per game and could exploit deterministic frame sequences. **Gap:** all rely on engineered visual features or per-game evolution; none learn features end-to-end from raw pixels with one shared method.

## Evaluation settings

The natural yardstick is the Arcade Learning Environment (Bellemare et al. 2013) over a set of Atari 2600 games (e.g. Beam Rider, Breakout, Enduro, Pong, Q*bert, Seaquest, Space Invaders). The agent receives 210×160 RGB frames at 60Hz and the per-step reward is the change in game score. The protocol established for this testbed reports the average total reward per episode, computed by running a fixed ε-greedy policy (ε=0.05) for a fixed number of steps, averaged over games. Reference points include a uniform-random policy and an expert human player (median reward after a couple of hours of practice per game). A single architecture and a single hyperparameter setting are meant to be held fixed across all games so that what is measured is general competence, not per-game engineering. A complementary, lower-variance diagnostic tracks the agent's average predicted max_a Q over a fixed held-out set of states collected by a random policy before training.
