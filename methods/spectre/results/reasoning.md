I start with the spectral-signature picture because it gives me the only visible handle on a hidden trigger. Inside the target label, the examples are not one homogeneous population. Most are clean target-class examples with mean `mu_D` and covariance `Sigma_D`; a small fraction are poisoned examples with mean `mu_W` and covariance `Sigma_W`. If `F = (1 - epsilon)D + epsilon W`, then `mu_F = (1 - epsilon)mu_D + epsilon mu_W`, and expanding `E[(X - mu_F)(X - mu_F)^T]` gives `(1 - epsilon)Sigma_D + epsilon Sigma_W + epsilon(1 - epsilon)(mu_D - mu_W)(mu_D - mu_W)^T`. The sign of the mean difference does not matter because the extra term is an outer product. This explains why a top singular direction can find poisons at all: the attack adds variance along the clean-poison separation direction.

But that same formula shows the trap. A top-eigenvector filter only sees the direction of largest total variance in the combined target-label cloud. The poison term is large relative to clean variance only if `epsilon(1 - epsilon)||mu_D - mu_W||^2` beats the leading clean covariance directions. Real hidden representations are not isotropic; some clean directions carry ordinary object variation, lighting, pose, or subtype spread. If the trigger-induced shift sits in a low-clean-variance direction, the separation can be meaningful while still not being the top variance direction. Then the filter projects onto an unrelated clean direction and the poison scores blend into the clean scores.

A multi-way trigger makes this failure less accidental. If the poison examples are split across two or three trigger variants, the mean shift is no longer one clean rank-one spike. The poisoned subpopulation leaves several weaker directions or clusters. A squared top projection can catch the strongest one and miss the rest; a pure squared norm can dilute a one-direction signature among many ordinary directions. I need two changes: first, make clean directions comparable before looking for excess variance; second, score excess variance without committing to either the norm endpoint or the single-direction endpoint.

If I had the clean target-label covariance, the first change would be direct. I would transform a representation `h` into `Sigma_clean^{-1/2}(h - mu_clean)`. In that coordinate system, clean data has roughly identity covariance, so there is no high-variance clean direction for poison to hide behind. The contaminated directions are exactly the directions where the whitened covariance of all target-label examples rises above identity. The important detail is that I cannot whiten by the contaminated covariance itself, because that would flatten the poison bump too. The covariance estimate has to come from a robust clean-core estimate.

The robust-statistics guarantee gives me permission to do this in principle but not in the full feature dimension. The guarantee assumes enough samples for robust joint mean and covariance estimation, on the order of `d^2/epsilon^2` up to logarithmic factors for a `d`-dimensional Gaussian. A target class in CIFAR has about five thousand examples, while a hidden representation can have thousands of coordinates. Full-dimensional robust covariance estimation is therefore not just expensive; it is statistically underdetermined. So the first operational step must be dimension reduction: center the target-label representations, take a top-`k` SVD subspace, and run robust covariance estimation only in that `k`-dimensional coordinate system.

This creates a real choice of `k`. If `k` is too small, I may project away the poison separation before robust estimation starts. If `k` is too large, I include low-sample, heavy-tailed clean directions where the robust covariance filter is no longer reliable, and the whitening can amplify clean artifacts. A fixed moderate dimension is not safe because different attacks have different effective dimensions. The right way to choose `k` has to use only observable quantities, not poison labels.

The selection criterion should ask whether a candidate `k` leads to a cleaned set whose whitening still reveals a strong residual signature in the full candidate subspace. For each `k`, I run the detector and remove the `1.5 * epsilon * n` highest-scoring points. Then I take the survivors, estimate their covariance in a fixed top-`k_max` PCA coordinate system, whiten all target-label points by that survivor covariance, compute QUE scores on the whitened points, and average those scores. If the candidate `k` really found the poison, the removed set leaves a survivor covariance close to clean covariance, and whitening by it makes the remaining poisoned points high-score outliers. If `k` missed the poison, the average score is weaker. This is the `k`-Identifier logic used by `rcov_auto_quantum_filter`: the code uses a ten-point squared grid `[1, 4, 9, ..., 100]` and selects the `k` with the largest mean QUE objective, not the `k` with the largest raw top eigenvalue.

Now I need the robust covariance step itself. A tempting simplification is to iterate Mahalanobis distances and trim the farthest points, but that is not what the repository code does. The code uses the Diakonikolas-style covariance filter in `dkk17.jl`. After PCA centering, it computes the current empirical covariance, whitens by its inverse square root, first removes points with excessively large whitened norm, and otherwise searches over even quadratic polynomials by an implicit Kronecker operator. If the worst quadratic variance is within the robust threshold, it accepts the current Gaussian covariance. If not, it finds a tail threshold for that quadratic score and removes the points beyond it. The repository filter caps this robust-estimation removal at `round(2 * eps_count)` while using the corruption fraction `eps_count / n` inside the filter. That cap is separate from the final defense removal budget `round(1.5 * eps_count)`.

With a robust survivor set in the PCA subspace, whitening is straightforward. In the abstract algorithm, `RobustEst` returns both `Sigma_hat` and `mu_hat`, and the whitened vectors are `Sigma_hat^{-1/2}(\bar h_i - mu_hat)`. In the repository code path, the PCA helper has already centered the representation matrix, and `rcov_quantum_filter` estimates a covariance from the robustly selected centered PCA coordinates. For `k > 1`, it sets `Sigma = cov(reps_pca_selected', corrected=false)` and uses `Sigma^(-1/2) * reps_pca`; for `k == 1`, it skips robust covariance and uses the one-dimensional PCA coordinate directly with covariance `1`. That special case matters because a one-dimensional covariance filter and matrix exponential add no useful structure there.

The final score has to handle both concentrated and spread signatures. Let the whitened vectors be `h_tilde_i` in `R^k`, and let `Sigma_tilde = (1/n) sum_i h_tilde_i h_tilde_i^T`. The score matrix is `Q_alpha = exp(alpha * (Sigma_tilde - I) / (||Sigma_tilde||_2 - 1))`, and the score is `tau_i = h_tilde_i^T Q_alpha h_tilde_i / Tr(Q_alpha)`. The denominator inside the exponential is `||Sigma_tilde||_2 - 1`, not `||Sigma_tilde||_2`, because identity is the clean baseline after whitening and only the excess over one should set the scale. When `alpha = 0`, `Q_alpha = I`, so the trace-normalized score is `(1/k)||h_tilde_i||^2`. As `alpha` grows, the exponential concentrates on the largest excess-variance eigenspaces and approaches squared projection onto the leading whitened direction. With finite `alpha`, especially the default `alpha = 4`, the score is a soft maximum over excess covariance directions.

The sign convention is also fixed by this formula. Higher `tau_i` means more suspicious. The repository code implements "largest scores" by calling `k_lowest_ind` on negative quadratic scores, so the Boolean returned by `rcov_quantum_filter` is a keep-mask and `.! selected` is the removal mask. The PCA baseline uses the same ranking idea on squared or absolute top-projection scores; squared and absolute projection induce the same ordering, but the method's final QUE scores are nonnegative quadratic forms.

The complete detector is therefore target-label, representation-space, and two-stage. Given target-label representations, center them, project to a candidate PCA dimension, robustly select a clean covariance core with the covariance filter, whiten by that covariance, compute QUE scores, and remove `1.5` times the poison-count budget. Wrap that in the `k` identifier, which reruns the detector on the squared grid and chooses the candidate whose survivor-covariance whitening yields the largest average QUE score. This keeps the original spectral insight, but it removes the brittle assumption that the poison is already the loudest direction of the raw target-label covariance.
