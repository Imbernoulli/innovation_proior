Let me start from the thing that actually hurts. I want to pretrain a GPT-style language model, and the cost that dominates as I push context length is the attention layer: softmax attention forms `O = softmax(Q K^T / sqrt(d_k) + A) V`, and that `Q K^T` is an `L×L` object, so both compute and memory grow like `L^2`. For a fixed token budget that quadratic term is what eats the wall-clock and the memory, and at inference it is even worse in a different way — I have to keep the whole key/value cache around and attend the new token to all of it, so the per-token cost grows with how much I've already generated. I want a sequence mixer that is linear-time at inference with a fixed-size state, and subquadratic to train, but — and this is the part everyone trips on — without giving up language-model quality against a strong softmax Transformer trained on the same data. Cheapness is easy; cheapness that doesn't lose quality is the whole problem.

The obvious lever is linear attention. Softmax attention weights token `i` by `exp(q_t · k_i / sqrt(d_k))`, and the exponential is exactly what couples every query to every key through a normalization that can't be factored. So replace that exponential score with a plain kernel: `exp(q_t · k_i / sqrt(d_k)) → ⟨φ(q_t), φ(k_i)⟩` for some feature map `φ`. Then the output is `o_t = (Σ_{i≤t} φ(q_t) φ(k_i)^T v_i) / (Σ_{i≤t} φ(q_t) φ(k_i)^T)`, and now `φ(q_t)` pulls straight out of both sums. So I can carry a running summary `S_t = Σ_{i≤t} φ(k_i)^T v_i`, a `d×d` matrix, and a normalizer `z_t = Σ_{i≤t} φ(k_i)^T`, and write `S_t = S_{t-1} + φ(k_t)^T v_t`, `o_t = φ(q_t) S_t / (φ(q_t) z_t)`. That is a linear RNN whose hidden state is a *matrix*, updated by the outer product `φ(k_t)^T v_t` — old "fast weights" idea. Inference becomes `O(1)` per step with a fixed-size state; no growing cache. The variants already on the table can drop the normalizer and even take `φ = I`, leaving the bare `S_t = S_{t-1} + k_t^T v_t`, `o_t = q_t S_t`. I'll work with that clean unnormalized version; it keeps the algebra honest and the empirical evidence says the identity map is fine here.

So inference is solved. What about training? Non-causally I'd love to write `O = (Q K^T) V` and reassociate to `Q (K^T V)`, turning `O(L^2 d)` into `O(L d^2)`. But I'm autoregressive — token `t` may only see `i ≤ t` — so the causal parallel form is `O = ((Q K^T) ⊙ M) V` with `M` the lower-triangular mask, and that mask is applied *elementwise* to `Q K^T`. I cannot push `K^T V` inside before I mask, because the mask lives on the `L×L` score matrix, not on the contracted product. So the naive causal parallel form is right back to `O(L^2 d)`. Wall. The recurrence is linear-time but it's a scalar-by-scalar elementwise loop over `L` steps — terrible on a GPU, no matmuls, low arithmetic intensity. So I have a fast-but-unparallelizable form and a parallelizable-but-quadratic form, and I want neither.

The way out that's already in the literature is to chunk. Split the sequence into `N = L/C` blocks of length `C`. The matrix state is additive across the whole past, so let me carry it at the granularity of chunks: after processing chunk `i`, the state is `S_{[i]}`, and the inter-chunk recurrence is just `S_{[i+1]} = S_{[i]} + K_{[i]}^T V_{[i]}` — that `K_{[i]}^T V_{[i]}` is a `(d×C)(C×d)` matmul, a single tensor-core operation per chunk. Then within chunk `i+1` the output splits into a piece that reads the past through the carried state and a piece that handles the inside of the chunk locally: `O_{[i+1]} = Q_{[i+1]} S_{[i]} + ((Q_{[i+1]} K_{[i+1]}^T) ⊙ M) V_{[i+1]}`. The first term costs `O(C d^2)`, the second is the small `C×C` masked product, `O(C^2 d)`. Sum over `N = L/C` chunks: `O(L C d + L d^2)`, which beats `O(L^2 d)` whenever `L > d`. And `C` is a dial — `C = L` is the pure parallel form, `C = 1` is the pure recurrence, and in between I trade FLOPs against parallelism. Good. This is the skeleton.

But here's the uncomfortable fact I can't wave away: plain linear attention, trained like this, *loses* to softmax attention on language modeling, sometimes by a lot. So efficiency isn't the deliverable; quality parity is, and I have to figure out *why* it loses before I can fix it. Stare at the update: `S_t = S_{t-1} + k_t^T v_t`. Every step it does is *add* an outer product. It never subtracts, never scales down, never forgets. The fixed-size matrix state just keeps accumulating every key it's ever seen, and old, no-longer-relevant content sits there forever, diluting whatever's current. That smells exactly like the disease the gated-RNN people diagnosed decades ago. The single most important component of an LSTM is the forget gate — the multiplicative `f_t ⊙ c_{t-1}` that lets the cell *erase* its memory — and the analyses since (Gers and colleagues; the "unreasonable effectiveness of the forget gate" study) show it carries most of the model's capacity. Plain linear attention is precisely an RNN with *no forget gate*. That's the hole.

So someone already tried bolting decay on: RetNet and TransNormerLLM do `S_t = γ S_{t-1} + k_t^T v_t` with a single global `γ ∈ (0,1)`. This is linear attention with a recency bias — essentially ALiBi baked into the recurrence — and it does help, clearly, over no decay. And the reason they use *one fixed scalar* `γ` is not laziness; it's that a scalar pulls out of the cumulative product cleanly and keeps the whole attention-style parallel/chunkwise machinery intact, so training stays efficient. But think about what a single constant `γ` can express. It's the same forgetting rate for every token, every channel, every context. It cannot look at the content and decide "this token is a key fact, hold it" versus "this is filler, let it decay fast." That's exactly the thing a forget gate is supposed to do, and the lesson from 1D RNNs is unambiguous: the forget gate has to be *data-dependent*, computed from the input, or it isn't doing its job. A global `γ` is a data-independent gate. So the gap is sharp: I want a forget gate that depends on the input, not a constant.

Now, the moment I say "data-dependent gate," I have to be careful, because there's a tension that immediately bites. A forget gate's value in a classic RNN depends on both the previous hidden state and the current input, and the dependence on the previous hidden state is exactly what serializes the recurrence and kills parallel training. Martin and Cundy pointed out the fix: make the gate depend *only on the current input*, not on the previous state. Then the recurrence stays linear in the state — the gate is just a sequence of input-determined coefficients — and a linear recurrence with input-determined coefficients is still parallelizable (it's a scan, and more to the point for me, it still has cumulative-product structure I can exploit). HGRN showed this input-only gating works at moderate scale. So the constraint is: the gate is a function of `x_t` alone.

What shape should the gate be? This is the real design decision, and the trade-off is three-way: parameter efficiency, state size / expressivity, and training efficiency. Let me think about the extremes. The most general thing is a full gate matrix `G_t ∈ (0,1)^{d_k × d_v}` applied Hadamard to the state, `S_t = G_t ⊙ S_{t-1} + k_t^T v_t`. Maximally expressive — every entry of the state forgets at its own data-dependent rate. But mapping `x_t → G_t` directly needs a projection of size `d · d_k · d_v`, which is enormous; parameter-inefficient to the point of absurd. The other extreme is RetNet's scalar — `G_t = γ_t 1^T 1` if I let `γ_t` depend on `x_t` — cheap and trivially keeps the parallel form, but it's the weakest data-dependent gate, one number for the whole state.

There's a middle that Mao, and GateLoop, explored: a low-rank outer-product gate `G_t = α_t^T β_t`, with `α_t ∈ R^{1×d_k}`, `β_t ∈ R^{1×d_v}`. That's only `d·d_k + d·d_v` parameters, and it's a genuine 2D gate — fine-grained recency control per key channel *and* per value channel. So why isn't this just the answer? Because of training. Their algorithms, to handle the data-dependent matrix gate, end up materializing the full matrix-valued hidden state for *every* time step in HBM — that's `L · d_k · d_v` of state written to slow memory — and the gated update as they implement it doesn't reduce to tensor-core matmuls. So it's slow at scale, both from the I/O and from leaving the tensor cores idle. And there's a parallel cautionary tale in Mamba: it makes the state transition fully input-dependent and full-rank, which is even more expressive, but precisely because the full-rank selective update *can't* be written as a matrix multiply, it can't touch tensor cores and has to materialize per-step states; to keep them in SRAM it caps the state expansion around 16, which bounds the recurrent state size and shows up as weak recall. The pattern is clear and it's the pattern I have to break: expressivity that destroys the matmul/chunkwise structure buys you a slow, memory-bound trainer. I need a gate expressive enough to fix the no-forgetting problem but structured enough that the chunkwise matmul form survives.

Let me just try the outer-product gate `G_t = α_t^T β_t` and *see whether the chunkwise structure survives*, rather than assume it doesn't. Unroll the recurrence. With `G_t = α_t^T β_t`, the state is `S_t = Σ_{i≤t} ((∏_{j=i+1}^t G_j) ⊙ k_i^T v_i)`. The cumulative product `∏_{j=i+1}^t G_j` is the thing I'm afraid of. But it's an outer product of cumulative products: define `b_t = ∏_{j=1}^t α_j` and `d_t = ∏_{j=1}^t β_j` (elementwise products of the gate vectors). Then `∏_{j=i+1}^t G_j = ∏_{j=i+1}^t α_j^T β_j = (b_t/b_i)^T (d_t/d_i)` — the telescoping makes the product-of-outer-products collapse into an outer product of ratios, by the mixed-product property. So `(∏_{j=i+1}^t G_j) ⊙ (k_i^T v_i) = ((b_t/b_i) ⊙ k_i)^T ((d_t/d_i) ⊙ v_i)`. That's still a single outer product! Plug into the output:
`o_t = q_t S_t = Σ_{i≤t} (q_t ((b_t/b_i) ⊙ k_i)^T) ((d_t/d_i) ⊙ v_i)`. The scalar `q_t ((b_t/b_i)⊙k_i)^T = ⟨q_t ⊙ b_t, k_i / b_i⟩` by the identity `⟨a, b⊙c⟩ = ⟨a⊙b, c⟩`, and the value side `(d_t/d_i)⊙v_i` rearranges so the `d_t` factors out: `o_t = (Σ_{i≤t} ⟨q_t⊙b_t, k_i/b_i⟩ (v_i/d_i)) ⊙ d_t`. So the whole thing is *exactly* a linear-attention parallel form on transformed tensors: define `Q̃ = Q ⊙ B`, `K̃ = K / B`, `Ṽ = V / D`, compute `Õ = ((Q̃ K̃^T) ⊙ M) Ṽ`, and read off `O = Õ ⊙ D`. The data-dependent matrix gate did *not* destroy the matmul structure — it just preconditions Q, K, V by cumulative-product factors. A 2D *outer-product* gate keeps the chunkwise matmul form alive, where a full-rank data-dependent transition does not.

Now, do I actually need `β` too? The `d_t` on the value side and the `1/d_i` are extra cumulative products, extra log-space bookkeeping, extra failure modes. Let me check what fixing `β = 1` buys me. With `β=1` the gate is `G_t = α_t^T 1` — every row of the state shares the same per-key-channel decay `α_t`, broadcast across the value dimension. And the value-side transforms vanish: `Ṽ = V`, `O = Õ`. So the model is `S_t = (α_t^T 1) ⊙ S_{t-1} + k_t^T v_t = Diag(α_t) S_{t-1} + k_t^T v_t` — a per-key-channel data-dependent forget gate, which is exactly the fine-grained forgetting I argued for, and it halves the gating machinery. The value-side gate would give a second independent decay axis on the value channels, but that is the expensive part of the bookkeeping and it partly overlaps with the full-width value projection plus the per-head output normalization I'll add later. So I'll take the middle ground: `G_t = α_t^T 1`. More expressive than RetNet's scalar (it's per-channel and data-dependent), far cheaper than the full matrix, keeps the matmul chunkwise form, and avoids the value-side cumulative products that would make the stable implementation harder.

So the recurrent form is `S_t = Diag(α_t) S_{t-1} + k_t^T v_t`, `o_t = q_t S_t`, with `α_t ∈ (0,1)^{d_k}` a data-dependent gate computed from `x_t`. And the parallel form, specializing the above with `d_t = 1`: let `b_t = ∏_{j≤t} α_j` (the cumulative product of the gate, stacked into a matrix `B`), then `P = (Q ⊙ B)(K / B)^T`, `O = (P ⊙ M) V`. Beautiful — until I think about magnitudes. Each `α_j` is in `(0,1)`, so `b_t` is a cumulative product of numbers below one, and over a long sequence it decays toward zero, fast. Then `K / B` divides by something tiny and *explodes*. In fp16, or even fp32, this overflows and the whole thing is garbage. Wall. So the clean parallel form is numerically dead on arrival.

The fix is to do the multiplication in log space. The entry of the score matrix is `P_{ij} = Σ_k Q_{ik} K_{jk} (b_i)_k/(b_j)_k = Σ_k Q_{ik} K_{jk} exp(log B_{ik} − log B_{jk})` for `i ≥ j`, where `i` is the query position, `j` is the key position, and `k` indexes feature channels. The cumulative product becomes a cumulative *sum* of log-gates, `log b_t = Σ_{j≤t} log α_j`, which is well-behaved — it's a sum of negative numbers, no underflow. And the term inside the exponential, `log B_{ik} − log B_{jk}`, is a data-dependent function of the relative position `i − j` (it's the accumulated log-decay between `j` and `i`), which is a nice way to see it: the gate is acting as a *data-dependent relative position bias*, in the spirit of xPos. Computing `P` this way is numerically stable. But — and this is the next wall — `P_{ij} = Σ_k Q_{ik} K_{jk} exp(log B_{ik} − log B_{jk})` is *not* a matmul. The `exp` of a difference that depends on both `i` and `k` sits between the `Q` and `K` factors, so I can't express the whole `i,j` matrix as a single `Q' K'^T`. Which means I can't run it on tensor cores in half precision — exactly the efficiency I was trying to protect. So log space buys stability but throws away the matmul. I need both.

Let me look harder at where the instability actually bites, because maybe I don't need log space *everywhere*. The cumulative product `b_t` underflows only because it accumulates over the *whole* sequence. But within a single chunk of length `C`, the cumulative product runs over at most `C` steps — and `C` is small, a couple hundred at most. Over a short span the product of gates doesn't underflow. So the instability is a *long-range* phenomenon, and the chunkwise decomposition already isolates long range (across chunks) from short range (within a chunk). Let me redo the chunking *with the gate* and see where the dangerous cumulative products live.

Carry the chunk-level state `S_{[i]}`. Within chunk `i+1`, I need the cumulative gate measured *relative to chunk boundaries*, not from the start of the sequence, so the cumulative products stay short. Define, for position `iC + j` inside chunk `i+1`: `Λ_{iC+j} = b_{iC+j} / b_{iC}` — the decay from the *start of the chunk* up to position `j`; `Γ_{iC+j} = b_{(i+1)C} / b_{iC+j}` — the decay from position `j` to the *end of the chunk*; and `γ_{i+1} = b_{(i+1)C} / b_{iC}` — the total decay across the whole chunk. Each of these is a ratio of cumulative products *within one chunk's span*, so each is a product of at most `C` gates — bounded, no underflow. Now the inter-chunk recurrence: the previous state must be decayed by the whole-chunk gate before adding the chunk's contribution, and the chunk's keys must be weighted by how much they'll have decayed by the chunk's end:
`S_{[i+1]} = (γ_{i+1}^T 1) ⊙ S_{[i]} + (K_{[i+1]} ⊙ Γ_{[i+1]})^T V_{[i+1]}`,
and the inter-chunk part of the output reads the carried state through the from-chunk-start decay:
`O^inter_{[i+1]} = (Q_{[i+1]} ⊙ Λ_{[i+1]}) S_{[i]}`.
Intuitively `Λ` is the cumulative decay from the chunk's start (used to propagate the old state forward into the chunk), and `Γ` is the decay to the chunk's end (used to accumulate the chunk's keys into the next state). The intra-chunk part is the same masked parallel form as before, but with the chunk-local preconditioning: `O^intra_{[i+1]} = (((Q_{[i+1]} ⊙ Λ_{[i+1]})(K_{[i+1]} / Λ_{[i+1]})^T) ⊙ M) V_{[i+1]}`, computed in log space when the local product form is unsafe. Every cumulative product here spans at most one chunk, so the only place I'm forced into full-precision log-space arithmetic is the *intra-chunk* score computation — and that's a `C×C` block, much smaller than the whole `L×L`.

So I've localized the non-matmul, full-precision work to the intra-chunk blocks. Can I shrink it further? Yes — tile the chunk again. Subdivide each chunk of length `C` into sub-chunks of length `c`, exactly the FlashAttention tiling idea, but one level deeper. The `C×C` intra-chunk score matrix is now a grid of `c×c` sub-blocks. For the *off-diagonal* sub-blocks — interactions between distinct sub-chunks `[i]` and `[j]` with `i > j` — the two sub-chunks are separated, so I can factor the decay onto each side relative to its own sub-chunk boundary and write it as a half-precision matmul: `P_{[i][j]} = (Q_{[i]} ⊙ Λ_{[i]}) (K_{[j]} ⊙ Γ_{[j]} ⊙ b_{iC}/b_{(j+1)C})^T`, a clean `c×c` tensor-core product. Only the *diagonal* sub-blocks — the within-sub-chunk interactions — must use the log-space `Σ_k Q K exp(log B − log B)` form in full precision, because there the positions `i` and `j` are interleaved at the finest scale and there's no clean way to split the decay across the two sides. So with this two-level tiling, the full-precision non-matmul FLOPs are squeezed down to the small diagonal `c×c` blocks, and the bulk of the work — all the off-diagonal sub-block interactions, plus all the inter-chunk matmuls — runs on tensor cores in half precision. That's how I keep both stability and tensor cores.

Now the I/O side, because FLOPs aren't wall-clock. The naive recurrent form keeps low FLOPs but it's an elementwise scan with no matmuls and low arithmetic intensity — slow. And a basic chunkwise implementation that writes the chunk states `S_{[n]}` to HBM and reads them back pays a lot in memory traffic. I want to be I/O-aware the way FlashAttention is: tile so that when I load a block of `Q` into SRAM I do *all* the on-chip work that needs it before evicting it. Concretely, once `Q_{[n]}` is on chip I can compute both the inter-chunk term `Q_{[n]} S` and the intra-chunk term `(Q_{[n]} K_{[n]}^T ⊙ M) V_{[n]}` without re-loading `Q_{[n]}`. There are two ways to organize the chunk loop, and they trade memory for parallelism. The non-materialization version walks chunks left to right, keeping the single state `S` in SRAM and never writing per-chunk states to HBM — memory-frugal, parallel over batch × heads × head-dimension, but with *no* sequence-level parallelism. That's fine when the batch is large enough to fill the SMs. But in large-scale, long-sequence training the batch is small, and then with no sequence-level parallelism the SMs sit idle and occupancy tanks. So I also want the materialization version: first sweep the inter-chunk recurrence and store all the `S_{[n]}` in HBM, then compute every chunk's output *in parallel* — that recovers sequence-level parallelism and high occupancy at small batch. The cost is storing the states, about 10–20% more memory. But I can claw that back with recomputation, the other FlashAttention trick: throw away the materialized states after the forward pass and recompute them in the backward pass. Small runtime overhead, big memory saving — so I'll make materialization-plus-recomputation the default.

Recomputation forces me to confront the backward pass, and there's a specific landmine. The prior matrix-gate work claimed that to get the gradient of the gate, `dα_t`, you *must* materialize the per-step states, because `dα_t = (S_{t-1} ⊙ dS_t) 1` — it references `S_{t-1}` directly. If that were true, the whole memory-efficiency argument collapses: I'd be back to `L · d_k · d_v` states in HBM. So let me try to find a closed form for the gate gradient that *doesn't* need the states. Work in log space, since that's where the gate lives. The output, fully unrolled, is `o_t = Σ_{i≤t} (q_t ⊙ b_t)(k_i / b_i)^T v_i`, and writing `b_t = exp(log b_t)`, `1/b_i = exp(−log b_i)`, this is `o_t = Σ_{i≤t} (q_t ⊙ exp(log b_t))(k_i ⊙ exp(−log b_i))^T v_i`. I want the gradient with respect to `log b_t` (then sum to get `log α`). First the gradients with respect to the query and key vectors, which I'd compute anyway:
`dq_t = Σ_{i≤t} ⟨do_t, v_i⟩ (b_t ⊙ k_i / b_i)`, summing the contributions where `q_t` appears, and
`dk_i = Σ_{t≥i} ⟨do_t, v_i⟩ (q_t ⊙ b_t / b_i)`, summing where `k_i` appears.
`log b_t` enters the output through two places: multiplicatively as `q_t ⊙ exp(log b_t)` (the query side), and as `exp(−log b_i)` dividing the keys (the key side, with the opposite sign). For a factor that multiplies, `∂f(a ⊙ b)/∂ log b = a ⊙ ∂f/∂a` — because `∂/∂ log b = b · ∂/∂b` and `b · ∂(a⊙b)/∂b = a ⊙ b`-weighted derivative folds the `b` back in. For a factor that divides, `∂f(a/b)/∂ log b = − a ⊙ ∂f/∂a`, the same with a sign flip. Apply these: the query-side contribution to `d log b_t` is `q_t ⊙ dq_t`, and the key-side contribution is `− k_t ⊙ dk_t`. So
`d log b_t = q_t ⊙ dq_t − k_t ⊙ dk_t`,
a *closed form in the quantities I already have* — no `S_{t-1}` anywhere. And since `log b_t = Σ_{i≤t} log α_i`, the gradient with respect to `log α_t` is the running suffix sum, `d log α_t = Σ_{i≥t} d log b_i`. That's the whole gate gradient: compute `dq` and `dk` with the same chunkwise machinery as the forward, take `q ⊙ dq − k ⊙ dk` per position, and do one reverse cumulative sum. The claim that states must be materialized for `dα` was simply not exploiting the log-space identity. So recomputation is safe and the backward stays memory-efficient.

I have the layer's math. Now I have to make it an actual layer that competes, which means a few more decisions, and I want each one motivated, not pulled from a recipe. How do I parameterize `α_t`? It's a vector in `(0,1)^{d_k}`, data-dependent on `x_t`, so the natural thing is a linear map followed by a sigmoid. But a full `d × d_k` projection just for the gate adds real parameters, and I'm trying to keep the layer at roughly the same parameter count as softmax attention (about `4d^2`). So make the gate projection *low-rank*: `x_t → W_α^1 → W_α^2 → sigmoid`, with `W_α^1 ∈ R^{d×16}` and `W_α^2 ∈ R^{16×d_k}`. A rank-16 bottleneck is more than enough to choose a per-channel forget rate from content, and it's nearly free in parameters. Then there's a subtle but important knob: a freshly initialized sigmoid gate sits around `0.5`, meaning the state *halves* every step — that forgets far too aggressively, the model can't hold anything for more than a handful of tokens, and long-range capacity is dead before training even starts. I want the gate biased toward 1 (slow forgetting) early on. So raise it to a power: `α_t = σ(x_t W_α^1 W_α^2 + b_α)^{1/τ}` with a temperature `τ = 16`. Since `σ < 1`, taking the `1/16` power pushes it *toward* 1 — `0.5^{1/16} ≈ 0.96` — so the default behavior is slow forgetting, and the model has to actively decide to forget. In log space this is even cleaner: `log α_t = (1/τ) log σ(logits) = (1/16) logsigmoid(logits)`, a small-magnitude negative log-decay, which is exactly the well-conditioned cumulative-sum quantity I built the stable form around. So the temperature does double duty: it sets the prior toward long memory *and* keeps the log-gate small.

One more on the query/key side: I'll scale the queries by `1/sqrt(d_k)` before the score, the standard attention-logit scaling, so the magnitudes going into the (now linear) "attention" don't blow up with the key dimension. And I'll let the key and value dimensions differ from `d`: set `d_k = d/2` and `d_v = d`. Why an *asymmetric* split? The recurrent state is `d_k × d_v`, and its size is the model's memory capacity — bigger state, more it can recall, which matters because the whole reason linear attention loses is bounded memory. I want `d_v = d` to keep the value/output channels full width. But I don't need the key dimension as large, and keeping `d_k = d/2` controls the parameter count and the state size while still leaving a sizeable `d/2 × d` state. With full-rank `W_Q, W_K, W_V, W_O` and the low-rank gate, one GLA layer lands at roughly `4d^2` parameters — same budget as softmax attention, so any quality difference is about the mechanism, not the parameter count.

Now multi-head, and the output path, because a single-head matrix state is too coarse and the per-head outputs need taming. Split into `H` heads, each with its own `d'_k × d'_v` state, `d'_k = d_k/H`, `d'_v = d_v/H`, running the gated recurrence per head: `S_t^h = ((α_t^h)^T 1) ⊙ S_{t-1}^h + k_t^{hT} v_t^h`, `o_t^h = q_t^h S_t^h`. The per-head outputs of a linear-attention layer can have wildly different scales (there's no softmax normalizing them to a simplex), so I normalize each head's output before concatenating — `o'_t = concat(Norm(o_t^1), …, Norm(o_t^H))` — which stabilizes training; this per-head normalization plus output gating is the RetNet output recipe and it matters for quality. Then an output gate: `r_t = Swish(x_t W_r + b_r)`, applied multiplicatively, `y_t = (r_t ⊙ o'_t) W_O`. The Swish output gate is a gated nonlinearity on the layer's output — it lets the layer suppress or pass channels based on content, and empirically gated outputs help these linear models close the gap. Finally the block: pre-norm residuals, `Y^{(l)} = GLA(Norm(X^{(l)})) + X^{(l)}`, `X^{(l+1)} = SwiGLU(Norm(Y^{(l)})) + X^{(l)}`, with the SwiGLU FFN `(Swish(Z W_1) ⊙ Z W_2) W_3` — the modern LLaMA recipe, used identically to the softmax baseline so the comparison is fair. And note I don't need positional embeddings at all: the cumulative data-dependent decay *is* a relative position signal (a learned, content-dependent ALiBi), so absolute position embeddings are redundant and I drop them.

The layer code fills the token-mixer slot with the chunkwise gated recurrence I derived. The recurrence in code is the cleanest statement of the model and exactly what the chunkwise kernel computes in parallel; I'll show that recurrent kernel explicitly so the math is visible, then the layer module that wires up the low-rank log-gate, the multi-head split, the per-head norm, the output gate, and the projections. The gate is produced *in log space* — `log α_t = logsigmoid(low-rank logits) / τ`, with `τ = 16` — which is precisely `α_t = σ(·)^{1/16}` but kept as the well-conditioned log-decay the stable form wants.

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
from einops import rearrange


def gated_linear_attention_recurrent(q, k, v, log_alpha):
    """Reference recurrence the chunkwise kernel parallelizes.
    q, k, log_alpha: (B, H, T, d_k);  v: (B, H, T, d_v).
    log_alpha = log of the data-dependent forget gate (<= 0)."""
    B, H, T, d_k = q.shape
    d_v = v.shape[-1]
    scale = d_k ** -0.5                                   # 1/sqrt(d_k) logit scaling on q
    S = q.new_zeros(B, H, d_k, d_v)                       # matrix-valued state, the "memory"
    o = torch.zeros_like(v)
    for t in range(T):
        a_t = log_alpha[:, :, t].exp()                   # alpha_t = gate in (0,1)^{d_k}
        # S_t = Diag(alpha_t) S_{t-1} + k_t^T v_t   (forget, then add the outer product)
        S = S * a_t[..., None] + k[:, :, t][..., None] * v[:, :, t][..., None, :]
        # o_t = (q_t * scale) S_t
        o[:, :, t] = ((q[:, :, t] * scale)[..., None] * S).sum(-2)
    return o


class TokenMixer(nn.Module):
    """Gated Linear Attention (GLA): linear attention with a per-key-channel,
    data-dependent forget gate; trained with the chunkwise matmul form."""

    def __init__(self, config):
        super().__init__()
        d = config.n_embd
        self.n_head = config.n_head
        self.key_dim = d // 2                             # d_k = d/2  (state = d_k x d_v)
        self.value_dim = d                                # d_v = d    (full-width memory)
        self.head_k = self.key_dim // self.n_head
        self.head_v = self.value_dim // self.n_head
        self.tau = 16                                     # gate temperature: bias toward slow forgetting
        self.use_pos_emb = False                          # cumulative decay already encodes relative position

        self.q_proj = nn.Linear(d, self.key_dim, bias=False)
        self.k_proj = nn.Linear(d, self.key_dim, bias=False)
        self.v_proj = nn.Linear(d, self.value_dim, bias=False)
        self.g_proj = nn.Linear(d, self.value_dim, bias=False)   # output gate
        # low-rank gate projection: d -> 16 -> d_k  (rank-16 bottleneck keeps it ~free)
        self.gk_proj = nn.Sequential(nn.Linear(d, 16, bias=False),
                                     nn.Linear(16, self.key_dim, bias=True))
        self.o_proj = nn.Linear(self.value_dim, d, bias=False)
        self.head_norm = nn.RMSNorm(self.head_v)          # FLA uses per-head RMSNorm

    def forward(self, x):                                 # x: (B, T, d)
        q = rearrange(self.q_proj(x), 'b t (h d) -> b h t d', d=self.head_k)
        k = rearrange(self.k_proj(x), 'b t (h d) -> b h t d', d=self.head_k)
        v = rearrange(self.v_proj(x), 'b t (h d) -> b h t d', d=self.head_v)
        # log alpha_t = (1/tau) * logsigmoid(low-rank logits)  ==  log( sigmoid(.)^{1/tau} )
        log_alpha = F.logsigmoid(self.gk_proj(x)) / self.tau
        log_alpha = rearrange(log_alpha, 'b t (h d) -> b h t d', d=self.head_k)

        # the chunkwise kernel computes this in parallel on tensor cores; recurrence shown for clarity
        o = gated_linear_attention_recurrent(q, k, v, log_alpha)

        o = self.head_norm(o)                             # normalize each head, then concat
        o = rearrange(o, 'b h t d -> b t (h d)')
        o = o * F.silu(self.g_proj(x))                    # Swish output gate (gated nonlinearity)
        return self.o_proj(o)
```

In a real training run the `gated_linear_attention_recurrent` loop is replaced by the chunkwise kernel — the materialization-plus-recomputation version with secondary-level tiling — which computes exactly the same outputs but routes the off-diagonal sub-chunk interactions and the inter-chunk state propagation through half-precision matmuls on tensor cores, keeps only the diagonal sub-chunk blocks in full-precision log space for stability, and uses the closed-form `d log α_t = Σ_{i≥t}(q_i ⊙ dq_i − k_i ⊙ dk_i)` so the backward never materializes the per-step states.

Softmax attention is `O(L^2)` and grows a KV cache; linear attention fixes that with a matrix-valued state and a recurrent/chunkwise form, but it never forgets — the state only ever accumulates outer products — and so it loses to softmax on language modeling. The RNN literature says the cure is a forget gate, and that it must be data-dependent; RetNet's global scalar decay helps but is data-independent, one rate for everything. A fully data-dependent matrix gate is maximally expressive but, as the prior matrix-gate and selective-SSM work shows, it destroys the matmul/chunkwise structure and forces slow, memory-bound training. The escape is the *outer-product* gate `G_t = α_t^T 1`: unrolling it shows the cumulative gates telescope into a per-channel cumulative product `b_t`, so the layer reduces to plain linear attention on the preconditioned tensors `Q⊙B`, `K/B` — the matmul chunkwise form survives. The raw cumulative product underflows and makes `K/B` explode, so the scores are computed in log space; but log space breaks the half-precision matmul, so a two-level chunking confines the full-precision log-space work to small diagonal sub-blocks while the off-diagonal blocks and the inter-chunk recurrence stay on tensor cores. I/O-awareness — tiling, a materialization version for sequence-level parallelism at small batch, and recomputation in the backward — makes it fast in wall-clock, not just in FLOPs, and a closed-form log-space gate gradient (`q⊙dq − k⊙dk`, suffix-summed) keeps the backward memory-efficient. The gate is a low-rank projection through sigmoid with a temperature that biases it toward 1 (slow forgetting and a small log-decay), the state is kept wide (`d_v = d`, `d_k = d/2`) for memory capacity, and the layer is finished with multi-head split, per-head output normalization, a Swish output gate, and a SwiGLU pre-norm block — landing at the same `4d^2` parameter budget as softmax attention, with no positional embeddings because the cumulative decay already carries relative position.
