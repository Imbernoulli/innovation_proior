The problem is to show that every ideal in a polynomial ring is finitely generated when the base ring already has that property. At first sight this looks hopeless to verify directly: an ideal may be presented by infinitely many polynomials whose degrees grow without bound, and there is no obvious finite set of equations from which all algebraic consequences follow. Attempts that work equation by equation, or that try to enumerate all leading terms, fail because they chase an infinite collection of data. The missing step is to stop looking at the polynomials themselves and instead look at the coefficients of their highest-degree terms.

The canonical method is the Hilbert Basis Theorem. It states that if R is a Noetherian commutative ring, then the univariate polynomial ring R[x] is also Noetherian, and therefore every ideal in R[x_1, ..., x_n] is finitely generated whenever R is Noetherian. In particular, if k is a field, then k[x_1, ..., x_n] is Noetherian. The proof works by reducing the infinite polynomial ideal to two finite pieces: the ideal of leading coefficients inside the already-Noetherian base ring R, and a finite free R-module that captures all polynomials of bounded degree.

Here is how the reduction works. Take any ideal I in R[x]. Collect the leading coefficients of all nonzero polynomials in I; they form an ideal L in R. Because R is Noetherian, L has finitely many generators, and we can choose polynomials f_1, ..., f_r in I whose leading coefficients generate L. Let d be the maximum degree of the f_i. The polynomials in I whose degree is less than d form an R-submodule M of the finite free R-module R + Rx + ... + Rx^{d-1}; since R is Noetherian, M is also finitely generated, say by g_1, ..., g_s. The claim is that I is generated by the finite set f_1, ..., f_r, g_1, ..., g_s.

To see this, take any h in I. If deg h is less than d, then h lies in M and is an R-linear combination of the g_j. If deg h is at least d, its leading coefficient lies in L, so it can be written as a finite R-linear combination of the leading coefficients of the f_i. Multiply each f_i by the appropriate power of x so that all leading terms have degree equal to deg h, weight them by the same coefficients, and subtract the resulting polynomial from h. The leading terms cancel, so the remainder has strictly smaller degree and still belongs to I. Repeating this degree-lowering step must terminate because the degree is a nonnegative integer, so eventually the remainder falls below d and is captured by the g_j. Thus h belongs to the ideal generated by the finite set.

The same argument can be phrased by contradiction using Noetherian induction. If I were not finitely generated, one could repeatedly choose a nonzero element of minimal degree not yet generated. The leading coefficients of these minimal counterexamples generate an ascending chain of ideals in R, which must stabilize. Once it stabilizes, the latest leading coefficient is already generated by earlier ones, so the corresponding leading term can be eliminated using earlier polynomials, producing a lower-degree counterexample and contradicting minimality. This reformulation makes the engine of the proof explicit: any failure of finite generation in R[x] would propagate down to a non-stabilizing chain in R, which is forbidden by the Noetherian hypothesis.

The importance of the theorem is structural rather than algorithmic. It does not give an explicit Gröbner basis or a procedure for solving the equations; it guarantees that a finite set of generators exists. Because of this, affine algebraic sets, which a priori might be defined by infinitely many polynomial equations, are always defined by finitely many equations, and finitely generated algebras admit finite presentations. The theorem justifies the shift in algebra from manipulating individual equations to studying ideals, modules, and chain conditions.

```python
from typing import List
import math


class Poly:
    """Sparse univariate polynomial with integer coefficients."""

    def __init__(self, terms: dict):
        self.terms = {e: c for e, c in terms.items() if c != 0}

    def degree(self) -> int:
        return max(self.terms.keys()) if self.terms else -1

    def lc(self) -> int:
        d = self.degree()
        return self.terms.get(d, 0)

    def __add__(self, other: "Poly") -> "Poly":
        terms = dict(self.terms)
        for e, c in other.terms.items():
            terms[e] = terms.get(e, 0) + c
        return Poly(terms)

    def __neg__(self) -> "Poly":
        return Poly({e: -c for e, c in self.terms.items()})

    def __sub__(self, other: "Poly") -> "Poly":
        return self + (-other)

    def __mul__(self, other: "Poly") -> "Poly":
        terms = {}
        for e1, c1 in self.terms.items():
            for e2, c2 in other.terms.items():
                terms[e1 + e2] = terms.get(e1 + e2, 0) + c1 * c2
        return Poly(terms)

    def __rmul__(self, scalar: int) -> "Poly":
        return Poly({e: scalar * c for e, c in self.terms.items()})

    def __eq__(self, other: "Poly") -> bool:
        return self.terms == other.terms

    def __repr__(self) -> str:
        if not self.terms:
            return "0"
        parts = []
        for e in sorted(self.terms.keys(), reverse=True):
            c = self.terms[e]
            if e == 0:
                parts.append(f"{c}")
            elif e == 1:
                parts.append(f"{c}x")
            else:
                parts.append(f"{c}x^{e}")
        return " + ".join(parts)


def monomial(c: int, e: int) -> Poly:
    return Poly({e: c})


def reduce_by_generators(h: Poly, gens: List[Poly]) -> Poly:
    """Cancel leading terms of h using integer multiples of generators."""
    r = Poly(dict(h.terms))
    changed = True
    while changed and r.degree() >= 0:
        changed = False
        for g in gens:
            if r.degree() < g.degree() or g.lc() == 0:
                continue
            q, rem = divmod(r.lc(), g.lc())
            if rem == 0:
                power = r.degree() - g.degree()
                r = r - q * (g * monomial(1, power))
                changed = True
                break
    return r


def leading_coefficient_generators(polys: List[Poly]) -> List[Poly]:
    """
    Return a subset of polys whose leading coefficients generate the same
    ideal as all leading coefficients.  Over Z this ideal is principal.
    """
    if not polys:
        return []
    target = 0
    for p in polys:
        target = math.gcd(target, abs(p.lc()))
    gens, current = [], 0
    for p in polys:
        new_g = math.gcd(current, abs(p.lc()))
        if new_g != current:
            gens.append(p)
            current = new_g
            if current == target:
                break
    return gens


def hilbert_finite_generators(polys: List[Poly]) -> List[Poly]:
    """
    Return a finite generating set for the ideal generated by polys,
    illustrating the Hilbert basis theorem over Z.
    """
    if not polys:
        return []
    high_gens = leading_coefficient_generators(polys)
    d = max(p.degree() for p in high_gens)
    low_gens: List[Poly] = []
    for p in polys:
        for k in range(d):
            shifted = p * monomial(1, k)
            rem = reduce_by_generators(shifted, high_gens)
            if 0 <= rem.degree() < d and rem not in low_gens:
                low_gens.append(rem)
    return high_gens + low_gens


# Example: the ideal generated by x^2 + 1 and x + 1 over Z.
f1 = Poly({2: 1, 0: 1})
f2 = Poly({1: 1, 0: 1})
generators = hilbert_finite_generators([f1, f2])
h = f1 * monomial(1, 1) + f2 * monomial(5, 0)
print("Generators:", generators)
print("h =", h)
print("Remainder after reduction:", reduce_by_generators(h, generators))
```
