I start with a Transformer that is already useful, so I do not want to change the modeling story unless the resource arithmetic forces me to. The first arithmetic check is almost embarrassing: a very large reported Transformer layer has about $0.5$B parameters, which is about $2$GB in 32-bit floats. A batch of $8$ sequences of length $64\text{K}$ with width $1024$ is also about $64\text{K}\times 1\text{K}\times 8 \approx 0.5$B floats, another $2$GB. If I only had to hold one layer's parameters and one layer's activations, a long-sequence model should fit on a single accelerator. It does not, so the estimate is missing the real multipliers.

The missing terms are three different failures of the same hope. Backpropagation stores activations for every layer, so the single-layer activation cost is multiplied by $n_l$. The feed-forward sublayer is position-wise but wide, often $d_{ff}=4096$ when $d_{model}=1024$, so one intermediate is $[b,l,d_{ff}]$ rather than $[b,l,d_{model}]$. Attention then forms the $QK^\top$ scores; at $l=64\text{K}$ even one head and one sequence gives a $64\text{K}\times64\text{K}$ matrix, about $16$GB in fp32. I cannot fix one of these and declare victory. The useful target is to remove the $l^2$ attention term, the $n_l$ activation multiplier, and the $d_{ff}$ peak-memory multiplier.

The attention term is the most structural. For query $q_i$, dense attention computes a softmax over all dot products $q_i\cdot k_j$. The softmax is dominated by large logits, so most far keys contribute almost nothing. If I could find only the likely nearest keys for each query, attention would become sparse in a content-dependent way. But I cannot find those nearest keys by first computing all dot products; that is the quadratic work I am trying to avoid.

This is exactly the point of locality-sensitive hashing. I need a hash that groups vectors by angular similarity, because dot product after key normalization is a query norm times a cosine. A simple angular scheme is to draw $R\in\mathbb{R}^{d_k\times b/2}$, compute $xR$, concatenate the negation, and define
$$h(x)=\arg\max([xR;\,-xR]).$$
That gives $b$ signed-axis buckets from $b/2$ random directions. Nearby directions collide with high probability, far directions usually do not, and the whole operation is a batched matrix multiply plus an argmax.

There is a mismatch before hashing can be used inside a Transformer. In ordinary multi-head self-attention, $Q$ and $K$ come from different learned projections. A query and the key it should attend to can have a large dot product without living in a common hash geometry. The clean way to make "near in the hash" mean "near for attention" is to share the query-key projection. I produce one $QK$ tensor and a separate $V$ tensor. Then I use the query as the unnormalized query vector, but use a unit-normalized version of the same vector as the key: $k_j=q_j/\lVert q_j\rVert$. That keeps the key direction aligned with the angular hash while leaving $\lVert q_i\rVert$ as a per-query softmax temperature.

Now I can write the attention target for one position as
$$o_i=\sum_{j\in\mathcal{P}_i}\exp(q_i\cdot k_j-z(i,\mathcal{P}_i))v_j,$$
where $z(i,\mathcal{P}_i)=\log\sum_{j\in\mathcal{P}_i}\exp(q_i\cdot k_j)$. In causal attention the dense allowed set would be $\mathcal{P}_i=\{j:j\le i\}$. For hashed attention I want the content set $\mathcal{P}_i=\{j:h(q_i)=h(k_j)\}$, intersected with the causal mask when the model is autoregressive. For batching I may compute over a larger chunk-local set $\widetilde{\mathcal{P}}_i$ and subtract a mask term $m(j,\mathcal{P}_i)$, with $m=\infty$ outside the true set and $0$ inside it.

The next obstacle is not mathematical but mechanical: hash buckets are ragged. A GPU wants fixed-size matrix multiplications, not a variable number of keys for each query. I sort positions by the pair $(h(q_i),i)$, so items with the same bucket become contiguous and sequence order breaks ties. Then I cut the sorted sequence into chunks of length $m$ and let each chunk attend to itself and one previous chunk. In sorted coordinates the batch set is
$$\widetilde{\mathcal{P}}_i=\left\{j:\left\lfloor\frac{s_i}{m}\right\rfloor-1\le \left\lfloor\frac{s_j}{m}\right\rfloor\le \left\lfloor\frac{s_i}{m}\right\rfloor\right\}.$$
The one-previous-chunk case matters because a bucket can straddle a boundary. If every bucket has size less than $m$, then all true same-bucket partners are covered by this chunk plus one-back rule. Since the average bucket has size $l/n_{buckets}$, I choose $m=2l/n_{buckets}$, equivalently $n_{buckets}=2l/m$, so the chunk is twice the average bucket size. Sorting contributes the $\log l$ factor; the chunked dot products scale with $l$ times a fixed local window and the number of hash rounds.

Hashing can miss a real neighbor, so I should not trust one random rotation. I use $n_{rounds}$ independent hashes and define
$$\mathcal{P}_i=\bigcup_r \mathcal{P}_i^{(r)},\qquad \mathcal{P}_i^{(r)}=\{j:h^{(r)}(q_i)=h^{(r)}(q_j)\}.$$
The exact union has a double-counting problem: if the same key appears in several rounds, summing round outputs would count it several times. Let $N_{i,j}=|\{r:j\in\mathcal{P}_i^{(r)}\}|$. The exact set-union correction is to subtract $\log N_{i,j}$ inside the round mask, so each of the $N_{i,j}$ appearances receives weight $1/N_{i,j}$. The round outputs are then rebased by their log-partitions:
$$o_i=\sum_r \exp(z(i,\mathcal{P}_i^{(r)})-z(i,\mathcal{P}_i))\,o_i^{(r)}.$$
This sign is important: the local round denominator is removed by $o_i^{(r)}$, and the factor $\exp(z^{(r)}-z)$ moves that local softmax onto the union denominator.

The masks have two separate jobs. Causality cannot use sorted order, because sorting has scrambled positions, so I carry the original position index through the same permutation and compare original indices inside each chunk. The shared-QK self case is different. Since $q_i\cdot(q_i/\lVert q_i\rVert)=\lVert q_i\rVert$, the self logit is usually the largest one in its bucket. If I let a position attend to itself freely, the layer tends toward identity. I therefore subtract a large finite penalty, conventionally $10^5$, for $i=j$. It must be finite rather than $+\infty$ in the mask, because the first causal token may have no other legal key; in that case the finite self logit is still the only finite option and the softmax remains defined.

So the attention part becomes shared-QK, angular hashing, sorting by bucket and position, chunk-local attention with one previous chunk, causal and self masks from carried position indices, and multiple independent hashes. The dense $l^2$ score matrix is gone. With fixed chunk length and $n_{buckets}=2l/m$, the remaining attention work is linear in length apart from sorting and the number of rounds.

That still leaves the layer-storage multiplier. A usual residual layer $y=x+F(x)$ cannot be inverted without solving for $x$, but a RevNet block can. I split the state into two full-width halves and compute
$$y_1=x_1+F(x_2),\qquad y_2=x_2+G(y_1).$$
Given $(y_1,y_2)$, I recover
$$x_2=y_2-G(y_1),\qquad x_1=y_1-F(x_2).$$
For a Transformer block, $F$ is the attention sublayer and $G$ is the feed-forward sublayer, with normalization inside those residual functions. During the backward pass I reconstruct each layer's inputs from its outputs and immediately compute the local gradients. I do not need to cache every layer's activations; the $n_l$ activation multiplier disappears. If there is dropout or any other randomness inside $F$ or $G$, the recomputation must see the same random choices as the forward pass, so the implementation must make the stochastic layer deterministic across the recompute.

The feed-forward multiplier is easier because the FFN is independent across sequence positions. I do not need the $[b,l,d_{ff}]$ intermediate for all positions at once. I split the sequence axis into chunks, apply the same feed-forward network to one chunk at a time, and concatenate the results:
$$Y_2=\left[X_2^{(1)}+\mathrm{FF}(Y_1^{(1)});\ldots;X_2^{(c)}+\mathrm{FF}(Y_1^{(c)})\right].$$
This changes peak memory but not the function. The same chunking applies to the reverse computation, the backward pass, and the final large-vocabulary log-probability calculation.

Putting the pieces together, the irreducible activation scale is roughly $bld_{model}$, but the former multipliers around it have been removed. LSH attention removes the $l^2$ attention term, reversible layers remove the depth multiplier on activations, and feed-forward chunking removes the peak $d_{ff}$ intermediate. Parameters still grow with depth, so a very deep model may also stream inactive layer parameters from host memory; the long batch-times-length computation gives enough work per layer to amortize that transfer. The final design is still a Transformer in its modeling ingredients, but its memory behavior is no longer dictated by dense attention scores and stored per-layer activations.
