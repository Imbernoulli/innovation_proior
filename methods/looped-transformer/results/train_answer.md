A standard transformer encoder of $L$ layers performs exactly $L$ stages of computation and then halts. For tasks whose natural description is iterative — following a program counter, updating memory, running a numerical recurrence, making repeated passes over data while updating a model held in the input — this is the wrong computational model. The ordinary remedy, stacking more layers, makes depth scale with runtime, which is exactly what we do not want: a small computer does not grow a new arithmetic unit for each instruction, it reuses the same hardware and lets time carry the length of the computation. So the goal is constructive rather than statistical: not a trained network that happens to pass a benchmark, but an explicit set of fixed transformer weights whose effect can be verified algebraically, with depth that does not scale with the number of program steps, an input that carries enough mutable state to support read and write, primitives that include arithmetic and branching, and softmax approximation error that is bounded rather than swept under the rug. The prior options each supply a piece but not the whole. RASP and Tracr compile some sequence algorithms into weights, but the compiled transformer grows with the program and the language is not a general low-level machine. The in-context-learning constructions of Akyürek et al. and von Oswald et al. hardcode gradient-descent-like updates, but cover one narrow learning problem with a fixed number of update steps per forward pass. Turing-completeness results prove broad expressivity but give no compact recipe for compiling a particular arithmetic or learning program into a small fixed-depth block with explicit read/write. Universal Transformers reuse a block over many steps but leave the content of one step to training rather than specifying the weights that execute a known instruction.

I propose the Looped Transformer as a programmable computer: take a single fixed transformer block, treat it as the hardware of an instruction executor, and run it recurrently, $X \leftarrow \mathrm{TF}(W; X)$ for $t = 1,\dots,T$, so the input matrix $X \in \mathbb{R}^{d\times n}$ is the mutable machine state and one application of the block executes one instruction. The columns of $X$ are partitioned into a scratchpad area (where the current instruction and operands are copied, transformed, and cleared), a memory area (data), and a command area (instructions); every column also carries a $\pm 1$ binary address $p_i$ of length $\log n$, because a command cannot say "read cell $a$" unless attention has something to match against. The address code is what makes selection sharp: $p_i^\top p_i = \log n$, and if two addresses differ in $k \ge 1$ bits then $p_i^\top p_j = \log n - 2k$, so the matching address wins every nonmatch by a margin of at least two. One layer is
$$\mathrm{Attn}(X) = X + \sum_h V_h\,X\,\mathrm{softmax}\!\left(X^\top K_h^\top Q_h X\right),\qquad f(X) = \mathrm{Attn}(X) + W_2\,\mathrm{ReLU}\!\left(W_1\,\mathrm{Attn}(X) + b_1 \mathbf{1}^\top\right) + b_2 \mathbf{1}^\top.$$
Setting $K=Q$ to read the address rows turns the score matrix into a Gram matrix of address codes, so attention becomes an approximate address selector. With column-softmax temperature $\lambda \ge \log(n^3/\epsilon)$ in the simplified selector bound (and $\lambda > \log(G d n^3/\epsilon)$ in the full bounded-input read/write bound, which carries the extra $Gd$ factor for entries bounded by $G$), the softmax is within $\epsilon$ of the intended hard permutation — error is a knob driven down by raising temperature. A read is then attention-as-dereference: the command holds the address to fetch, every source column holds its own address, the matching column captures nearly all the mass, and a value matrix routes its contents into a buffer. Because of the residual connection one cannot simply "set" a slot by attention; the feedforward sublayer does the overwriting, gated by a scratchpad indicator bit so that on scratch columns the ReLU expression adds the new value and cancels the old one while off scratch columns a large negative gate closes the update. A write is the same with the gate firing on the destination memory column, and the buffer is cleared by $v \leftarrow v - \mathrm{ReLU}(v) + \mathrm{ReLU}(-v)$.

With read and write in place, arithmetic lives in the ReLU sublayer. Integers are stored as $N$ signed bits $[b_N,\dots,b_1]$ in a two's-complement-style code: if the sign bit $b_N = -1$ the value is $\sum_{i=1}^{N-1} 2^{i-1}(b_i+1)/2 \ge 0$, and if $b_N = +1$ it is $-2^{N-1} + \sum_{i=1}^{N-1} 2^{i-1}(b_i+1)/2 < 0$. Binary addition of two nonnegative integers is piecewise-linear, so a one-hidden-layer ReLU network with $8d$ hidden activations adds $d$-bit integers as long as the sum stays in range; subtraction is two's-complement negation (flip with $2\,\mathrm{ReLU}(-b)-1$, add one via that binary-add network) followed by addition, giving $\mathrm{mem}[b]-\mathrm{mem}[a]$. The branch test must be checked sign by sign, since this is where an error would silently corrupt the machine. A negative value has $b_N = +1$, so $\mathrm{ReLU}(b_N)$ contributes one; zero is the all-$-1$ pattern, detected by $\mathrm{ReLU}(1 - N - \sum_i b_i)$, whose argument is $1$ at all $-1$ and at most $-1$ once any bit flips to $+1$ (the sum jumps by at least two); for every positive value both terms vanish. Hence
$$\mathrm{flag} = \mathrm{ReLU}(b_N) + \mathrm{ReLU}\!\left(1 - N - \sum_i b_i\right),$$
equivalently the preactivation $-\sum_i b_i - N + 1$. The naive alternative $1 + N - \sum_i b_i$ is not a unit zero detector — on the all-$-1$ pattern it returns $1+2N$ — so it is rejected. Once $\mathrm{flag}\in\{0,1\}$, the program counter is updated by a coordinatewise ReLU multiplexer between the next sequential address $p_{\mathrm{inc}}$ and the jump target $p_{\mathrm{jump}}$,
$$p_{\mathrm{next}} = 2\,\mathrm{ReLU}(p_{\mathrm{inc}} - \mathrm{flag}\cdot\mathbf{1}) + 2\,\mathrm{ReLU}(p_{\mathrm{jump}} - (1-\mathrm{flag})\cdot\mathbf{1}) - 1,$$
with a residual-cancel term subtracting the old counter; at $\mathrm{flag}=0$ the first term rebuilds $p_{\mathrm{inc}}$ from its $\pm 1$ coordinates and the second is zero, and at $\mathrm{flag}=1$ the roles swap. Because softmax reads and writes are only approximate, accumulated selector error would compound over many loop iterations, so a cleanup layer maps small neighborhoods of $-1,0,1$ back to exact values; for $\epsilon < 1/2$,
$$b_{\mathrm{clean}} = \frac{\mathrm{ReLU}(b+1-\epsilon) - \mathrm{ReLU}(b+\epsilon)}{1 - 2\epsilon} + \frac{\mathrm{ReLU}(b-\epsilon) - \mathrm{ReLU}(b-1+\epsilon)}{1 - 2\epsilon} - 1,$$
which the three center checks confirm sends $-1\mapsto -1$, $0\mapsto 0$, $1\mapsto 1$ and is flat on each neighborhood.

These pieces assemble into the one-instruction set $\mathrm{SUBLEQ}(a,b,c)$: set $\mathrm{mem}[b] \leftarrow \mathrm{mem}[b]-\mathrm{mem}[a]$, and if the result is $\le 0$ jump to instruction $c$, else advance — a command being just $[p_a; p_b; p_c]$. By merging compatible feedforward work into neighboring attention layers, one instruction takes nine transformer layers and two heads at width $O(\log n + N)$, with valid range $[-2^{N-1}+1, 2^{N-1}-1]$; looping it for $T$ steps runs a $T$-instruction program with the depth fixed at nine and only $T$ growing. The end-of-program command is made a true fixed point by reserving two cells holding $0$ and $-1$: $\mathrm{EOF}$ subtracts the zero cell from the $-1$ cell (so memory is unchanged and the branch fires) and jumps to itself, so once the counter reaches $\mathrm{EOF}$ it stays. $\mathrm{SUBLEQ}$ proves universality but is an unpleasant way to write numerical code, so I generalize the instruction to $\mathrm{FLEQ}$: replace the fixed subtraction with a menu of hardcoded transformer-based function blocks, $\mathrm{mem}[c]\leftarrow f_m(\mathrm{mem}[a],\mathrm{mem}[b])$ with a branch on a flag cell. The two halves do not interfere — a fixed-positive flag gives a pure function call, dummy input/output pointers give a pure branch — and with blocks of depth $l_m$ and head count $h_m$ the unified executor has $9 + \max_m l_m$ layers, $\sum_m h_m$ heads, and width $O(Md + \log n)$. The function blocks get their nonlinearity from softmax rather than from deepening the ReLU net: arrange the score vector so the selected coordinate is a sigmoid of $a^\top x$, store slopes in the key/query construction and coefficients in the value matrices, and summing $m$ heads yields $\sum_{i=1}^m c_{ji}\,\sigma(a_{ji}^\top x)$ for the chosen function index $j$. Barron's theorem then makes this approximate any $f \in \Gamma_{C,B}$ with $f(0)=0$ to error $O(m^{-1/2})$ once the sharpness satisfies $\tau \ge m^{1/2}\log m$, giving a 3-layer, $m$-head block (the $m$ terms can be packed across dimension into one head). Padding the softmax with a large constant instead linearizes it, giving a 2-layer one-head block computing $A^\top B + \epsilon M$ with $\|M\|\le 1$, and a 4-layer one-head transpose via vectorization and a fixed permutation. With multiplication, transpose, and subtraction as function blocks, Newton matrix inversion, power iteration, linear-model SGD, and two-layer sigmoid-network SGD with backpropagation all become short programs whose executor stays at most 13 layers — the iteration count is paid in recurrent invocations, never in deeper weights.
