OK. I have a coupled-spin system and I want to drive it somewhere. The Hamiltonian is partly fixed and partly mine to shape: H(t) = H0 + Σ_k u_k(t) H_k, where H0 is the free evolution (couplings, chemical shifts) and the H_k are the rf control operators whose amplitudes u_k(t) I get to dial as functions of time over a fixed window [0,T]. Two concrete things I might want at t=T. One: starting from a density operator ρ0, land on a state ρ(T) that overlaps as much as possible with a target operator C — that's coherence/polarization transfer. Two: realize a whole target propagator U_F, i.e. implement a specified unitary on the system regardless of input — that's gate synthesis. Let me start with the first and see where it forces me.

The state obeys Liouville–von Neumann, ρ̇ = -i[H0 + Σ_k u_k(t) H_k, ρ]. Overlap of operators is the Hilbert–Schmidt inner product ⟨C|ρ⟩ = tr{C† ρ}. So the thing I want to maximize is

  Φ0 = ⟨C|ρ(T)⟩ = tr{C† ρ(T)}.

For hermitian ρ0 and C this is real, good — I'll worry about the non-hermitian case later because S⁻ and I⁻ aren't hermitian and that'll bite.

Now, u_k(t) is a function. An infinite-dimensional search. I can't optimize a continuum directly, so I discretize: chop [0,T] into N equal slices of width Δt = T/N, and hold each control constant on each slice. On slice j the amplitude of control k is u_k(j). Then the evolution over slice j is generated by a *constant* Hamiltonian, so it's a single matrix exponential:

  U_j = exp{ -i Δt (H0 + Σ_k u_k(j) H_k) }.

And the full evolution is just the ordered product of these slice propagators acting on ρ0:

  ρ(T) = U_N … U_1 ρ0 U_1† … U_N†,

so the objective is Φ0 = ⟨C | U_N … U_1 ρ0 U_1† … U_N† ⟩. Fine. Now the real question: how do I move the u_k(j) to increase Φ0? I need the gradient ∂Φ0/∂u_k(j) for every control k and every slice j. There are m·N of these, and m·N is going to be huge — if I want a freely shaped pulse I want N in the hundreds or thousands.

The obvious thing is finite differences: nudge u_k(j) by δ, recompute Φ0 by re-propagating ρ0 from 0 all the way to T, subtract, divide by δ. But that's one full propagation per control amplitude. With m controls and N slices that's about m·N+1 full time evolutions for a single gradient. Put numbers on it: N=500, m=4 gives 2001 full propagations of the density operator just to take one gradient step. That's why everyone has been boxing the pulse into a tiny family — a few composite-pulse flip/phase angles, a Gaussian cascade, a spline, a short Fourier series — a few dozen parameters, because finite differences over thousands of them is hopeless. I don't want to accept that box. There has to be a way to get all m·N derivatives without re-propagating m·N times.

Let me stare at the structure of Φ0. The product U_N…U_1 has the slice I care about, U_j, sitting in the middle. When I perturb u_k(j), only U_j moves; every other factor is untouched. So the derivative should be *local* to slice j — it shouldn't require redoing the whole evolution from scratch. The trace is cyclic, tr{ABC}=tr{CAB}, so let me cut the chain right at slice j and regroup. Write

  Φ0 = ⟨ U_{j+1}† … U_N† C U_N … U_{j+1} | U_j … U_1 ρ0 U_1† … U_j† ⟩.

Look at the two halves. The right half, U_j … U_1 ρ0 U_1† … U_j†, is just ρ propagated *forward* from 0 up to time jΔt — call it ρ_j. The left half, U_{j+1}† … U_N† C U_N … U_{j+1}, is the *target* C propagated *backward* from T down to time jΔt — call it λ_j. So at every cut point,

  Φ0 = ⟨ λ_j | ρ_j ⟩.

That's already suggestive. The target, carried backward, meets the state, carried forward, and their overlap is the (constant, j-independent) objective. λ_j is exactly the kind of object optimal control calls a co-state or adjoint: a backward-propagated companion to the forward state, with its terminal value set by the cost (here λ_N = C). I've imported the Pontryagin / Bryson–Ho adjoint picture into spin dynamics almost by accident, just by cutting the trace.

Now perturb u_k(j) → u_k(j) + δu_k(j) and see how Φ0 changes. Only U_j moves. I need δU_j to first order. U_j = exp{-iΔt(H0 + Σ_k u_k(j) H_k)}, and I'm bumping one coefficient inside the exponent. The exponent doesn't commute with H_k in general, so I can't just pull H_k out front — I have to use the proper derivative of a matrix exponential. The identity is

  d/dx e^{A + xB}|_{x=0} = e^A ∫_0^1 e^{Aτ} B e^{-Aτ} dτ.

With A = -iΔt(H0+Σu_k H_k) and B = -iΔt H_k (the bump multiplies H_k inside the exponent and carries the -iΔt), this gives

  δU_j = -iΔt δu_k(j) · H̄_k · U_j,   where H̄_k Δt = ∫_0^{Δt} U_j(τ) H_k U_j(-τ) dτ,

with U_j(τ) = exp{-iτ(H0+Σu_k(j)H_k)} the partial within-slice propagator. So H̄_k is H_k averaged over its rotation across the slice. Here's the lever I was hoping for: if the slice is short, Δt ≪ ‖H0+Σu_k H_k‖^{-1}, then U_j(τ)≈1 across the slice and H̄_k ≈ H_k. So to first order in Δt,

  δU_j ≈ -iΔt δu_k(j) H_k U_j.

I'll keep that small-Δt condition in my back pocket — it tells me how fine the discretization has to be for this gradient to be trustworthy. Now propagate the perturbation into ρ_j. ρ_j = U_j (U_{j-1}…U_1 ρ0 U_1†…U_{j-1}†) U_j† = U_j ρ_{j-1} U_j†. Bumping U_j by δU_j and keeping first order:

  δρ_j = δU_j ρ_{j-1} U_j† + U_j ρ_{j-1} δU_j†
       = (-iΔt δu_k H_k U_j) ρ_{j-1} U_j† + U_j ρ_{j-1} (U_j† (iΔt δu_k H_k))
       = -iΔt δu_k ( H_k ρ_j - ρ_j H_k )
       = -iΔt δu_k [H_k, ρ_j].

(Using δU_j† = +iΔt δu_k U_j† H_k since H_k is hermitian.) And the change in everything downstream of slice j is carried by λ_j, because the rest of the chain is exactly the backward-propagated target. So

  δΦ0 = ⟨ λ_j | δρ_j ⟩ = -⟨ λ_j | iΔt δu_k(j) [H_k, ρ_j] ⟩,

which means

  ∂Φ0/∂u_k(j) = -⟨ λ_j | iΔt [H_k, ρ_j] ⟩.

Now look at what this costs. To evaluate this for *all* k and *all* j, I need ρ_j for every j and λ_j for every j. But ρ_j is just the forward propagation of ρ0 — I get ρ_1, ρ_2, …, ρ_N in *one* forward sweep, storing each intermediate. And λ_j is the backward propagation of C from λ_N=C down — I get all of them in *one* backward sweep. Then the gradient at each (k,j) is a couple of matrix products and a trace, cheap. So the whole m·N gradient costs **two full propagations** — one forward of the state, one backward of the adjoint — instead of m·N+1. The 2001 collapses to 2. That's the entire game: the adjoint lets me get every partial derivative from one forward and one backward sweep. That's worth a name — gradient ascent pulse engineering.

So the loop writes itself. Guess controls u_k(j). Forward-propagate ρ0 to get all ρ_j. Backward-propagate C to get all λ_j. Form ∂Φ0/∂u_k(j) = -⟨λ_j|iΔt[H_k,ρ_j]⟩ for the whole m×N grid. Step uphill,

  u_k(j) ← u_k(j) + ε · ∂Φ0/∂u_k(j),

small ε. Repeat until Φ0 stops changing. Each step strictly increases Φ0 to first order, so it's genuine ascent. No guarantee of a global optimum from a local method, of course — but for a controllable system with enough control freedom the fidelity landscape is benign: the critical points one runs into tend to be global optima or saddles rather than real local traps, so plain ascent on the amplitudes generally lands where I want. If it stalls on a saddle I can jitter the controls with a little noise. And ε doesn't have to be a fixed scalar forever — once I trust the gradient I can feed it to a conjugate-gradient or quasi-Newton (L-BFGS) routine to get superlinear convergence instead of crawling.

Now the thing I flagged: ρ0 = I⁻ = I_x − iI_y and C = S⁻ = S_x − iS_y are not hermitian, so Φ0 = ⟨C|ρ(T)⟩ is complex and "maximize a complex number" is meaningless. I need a real-valued figure of merit. Two honest choices. If the phase of the overlap matters for the application, maximize its real part, Φ1 = Re⟨C|ρ(T)⟩. Split everything into hermitian and skew-hermitian parts, ρ_j = ρ_j^x + i ρ_j^y and λ_j = λ_j^x + i λ_j^y; the real part of the overlap is the sum of the real-pairing of the x-parts and of the y-parts, and the same per-slice perturbation argument applied to each piece gives

  ∂Φ1/∂u_k(j) = -⟨ λ_j^x | iΔt[H_k, ρ_j^x] ⟩ - ⟨ λ_j^y | iΔt[H_k, ρ_j^y] ⟩.

But for coherence-order-selective transfer the *phase* of the final coherence is irrelevant — what's measured is the magnitude |⟨C|ρ(T)⟩|. So the right objective is

  Φ2 = |⟨C|ρ(T)⟩|² = ⟨C|ρ(T)⟩ ⟨ρ(T)|C⟩.

This is a product of two overlaps, ⟨C|ρ(T)⟩ and ⟨ρ(T)|C⟩, so its derivative is a product rule. The derivative of the first factor is the Φ0 result, -⟨λ_j|iΔt[H_k,ρ_j]⟩, scaled by the *other* factor ⟨ρ_N|C⟩; the second factor is the complex conjugate of the first, scaled by ⟨C|ρ_N⟩. The two terms are conjugates of each other, so their sum is twice the real part:

  ∂Φ2/∂u_k(j) = -2 Re{ ⟨λ_j | iΔt[H_k, ρ_j]⟩ ⟨ρ_N|C⟩ }.

Same two-sweep cost — I just also carry the scalar ⟨ρ_N|C⟩ from the end of the forward sweep. Good. That handles non-hermitian coherence transfer.

What if there's relaxation? Then I'm not in Hilbert space with commutators; I'm in Liouville space with a superoperator, ρ̇ = L̂ρ where L̂ = -iĤ + Γ̂ (Hamiltonian superoperator plus relaxation superoperator). The slice "propagators" become L̂_j = exp{L̂Δt}, and ρ(T) = L̂_N … L̂_1 ρ0. But the structure I used didn't depend on the generator being a commutator — it depended only on the evolution being a product of slice maps and on the trace/cut trick. So nothing changes in form: Φ0 = ⟨λ_j|ρ_j⟩ with ρ_j forward-propagated by the L̂'s and λ_j backward-propagated, and ∂Φ0/∂u_k(j) = -⟨λ_j|iΔt[H_k,ρ_j]⟩ where now the only control-dependence sits in the coherent part Ĥ. The adjoint trick is robust to swapping the dynamics.

Now the second goal: synthesize a target *propagator* U_F, not a state. The propagator itself obeys U̇ = -i(H0+Σu_k H_k)U with U(0)=1, so over the slices U(T) = U_N … U_1, a product of unitaries — no ρ0, no sandwich. I want U(T) close to U_F. Distance in Hilbert–Schmidt norm:

  ‖U_F - U(T)‖² = ‖U_F‖² - 2 Re⟨U_F|U(T)⟩ + ‖U(T)‖².

The norms are fixed (unitaries), so minimizing the distance is maximizing

  Φ3 = Re⟨U_F|U(T)⟩ = Re⟨U_F | U_N … U_1⟩.

Same cut: regroup at slice j. Define the forward propagator X_j = U_j … U_1 (the gate built up to time jΔt) and the backward-propagated target P_j = U_{j+1}† … U_N† U_F. Then Φ3 = Re⟨P_j | X_j⟩ for every cut point. Perturb u_k(j): δX_j = δU_j X_{j-1} = -iΔt δu_k H_k U_j X_{j-1} = -iΔt δu_k H_k X_j (using δU_j = -iΔt δu_k H_k U_j again). So

  δΦ3 = Re⟨P_j | δX_j⟩ = -Re⟨ P_j | iΔt H_k X_j ⟩,   i.e.   ∂Φ3/∂u_k(j) = -Re⟨P_j | iΔt H_k X_j⟩.

Here X_j comes from one forward sweep of the propagator from the identity, and P_j from one backward sweep of U_F from the end — two sweeps again, identical bookkeeping.

But wait — Φ3 = Re⟨U_F|U(T)⟩ is over-strict. Physically the global phase of a propagator is unobservable: U(T) and e^{iϕ}U(T) implement the same operation on any density operator (the phase cancels in U ρ U†). So I shouldn't penalize a pulse that hits U_F up to a global phase. I want to minimize ‖U_F - e^{iϕ}U(T)‖² *over the free choice of ϕ*, which is equivalent to maximizing the phase-independent

  Φ4 = |⟨U_F|U(T)⟩|² = ⟨U_F|U(T)⟩⟨U(T)|U_F⟩ = ⟨P_j|X_j⟩⟨X_j|P_j⟩.

This quotients the global phase out — maximizing it drives U(T) to U_F up to a phase, which is exactly gate fidelity. Differentiate the product like I did for Φ2: the derivative of the first factor is the Φ3-type seed -⟨P_j|iΔt H_k X_j⟩ scaled by the second factor ⟨X_j|P_j⟩, plus the conjugate piece, so

  ∂Φ4/∂u_k(j) = -2 Re{ ⟨P_j | iΔt H_k X_j⟩ ⟨X_j|P_j⟩ }.

This Φ4 is the one I'd actually use for gate synthesis, and Φ3 is mostly of theoretical interest. Note I'd normalize the overlap by the Hilbert-space dimension d, ⟨U_F|U(T)⟩/d, so that perfect realization gives fidelity 1 and the number is comparable across system sizes.

Two practical knobs fall out cheaply. First, rf power: real coils heat the sample, so I'd like to discourage large amplitudes. Add a penalty Φrf = α Σ_{j,k} u_k(j)² Δt to whichever Φ_i I'm maximizing (with a weight α), subtract it, and its gradient just tacks on a term -2α u_k(j)Δt to ∂/∂u_k(j) — trivial, and it pulls big amplitudes back toward zero. If instead there's a hard ceiling on |u_k|, I can just clip any amplitude that exceeds the maximum back to the maximum after each update step. Second, robustness: real systems have a spread of chemical shifts and rf calibration errors, so I don't want a pulse that's optimal for one nominal system and fragile off it. Sample the uncertain parameter at discrete values ω_p and optimize the *sum* of performances, Φtot = Σ_p Φ(ω_p). Because the gradient is linear in Φ, the gradient is just the sum of the per-system gradients, Σ_p ∂Φ(ω_p)/∂u_k(j) — each from its own forward/backward sweep. So robustness costs more sweeps but no new math.

Let me also sanity-check the discretization choice against the validity condition Δt ≪ ‖H0+Σu_k H_k‖^{-1}. If I take Δt too large, H̄_k drifts away from H_k and the first-order gradient I derived is wrong; if I take it small enough, the linearization holds and N is just large. So I want N large enough to satisfy that bound — which is fine, because the whole point of the adjoint trick is that large N is cheap (still two sweeps). For a coherence-transfer run I'd pick Δt on the order of 10⁻³ s, giving thousands of control parameters, and initialize the controls as random values on a coarse grid smoothed by a cubic spline so the starting pulse is smooth but unbiased.

Now turn it into code. I'll do the unitary-synthesis version since that's the gate problem; the state version is the same skeleton with the sandwich. The structure: build the N slice propagators by matrix exponential; do one forward sweep accumulating X_j and one backward sweep accumulating the P_j-without-target; then for each (k,j) form P = P_j and Q = iΔt H_k X_j and read off the gradient — the phase-sensitive Φ3 gradient is -Re⟨P|Q⟩, the phase-insensitive Φ4 gradient is -2Re{⟨P|Q⟩⟨X_j|P⟩}; add the rf penalty term; step uphill.

```python
import numpy as np
from scipy.linalg import expm

def overlap(A, B):
    # Hilbert–Schmidt inner product tr(A† B), normalized by dimension d
    return np.trace(A.conj().T @ B) / A.shape[0]

def grape_unitary(U_target, H0, H_ops, times, R, eps,
                  phase_sensitive=False, alpha=None, u_start=None):
    """Shape controls so U(T)=U_N...U_1 realizes U_target.
       H_ops: list of m control operators H_k. times: slice grid (length M)."""
    M, J = len(times), len(H_ops)
    dt = times[1] - times[0]
    d = U_target.shape[0]
    u = np.zeros((R, J, M))
    if u_start is not None:
        for k, u0 in enumerate(u_start):
            u[0, k, :] = u0

    for r in range(R - 1):
        # slice propagators U_j = expm(-i dt (H0 + Σ_k u_k(j) H_k))
        def H_idx(idx):
            return H0 + sum(u[r, k, idx] * H_ops[k] for k in range(J))
        U_list = [expm(-1j * H_idx(idx) * dt) for idx in range(M - 1)]

        # one forward sweep -> X_j = U_j...U_1 ; one backward sweep -> the U_b that
        # gives P_j = U_{j+1}†...U_N† U_target. This is the adjoint trick: all the
        # gradient ingredients from two sweeps, not m*N propagations.
        X_list, Pb_list = [], []
        X = np.eye(d); Pb = np.eye(d)
        for n in range(M - 1):
            X = U_list[n] @ X
            X_list.append(X)                      # X_j (forward)
            Pb_list.insert(0, Pb)                 # accumulated backward map
            Pb = U_list[M - 2 - n].conj().T @ Pb

        for k in range(J):
            for j in range(M - 1):
                P = Pb_list[j] @ U_target          # P_j = backward-propagated target
                Q = 1j * dt * H_ops[k] @ X_list[j] # iΔt H_k X_j

                if phase_sensitive:                # Φ3 gradient
                    g = -overlap(P, Q)
                else:                              # Φ4 gradient
                    g = -2 * overlap(P, Q) * overlap(X_list[j], P)

                if alpha:                          # rf-power penalty gradient
                    g += -2 * alpha * u[r, k, j] * dt

                u[r + 1, k, j] = u[r, k, j] + eps * g.real
            u[r + 1, k, -1] = u[r + 1, k, -2]      # carry last slice

    return u[-1]
```

The causal chain, start to end: I want to steer a coupled-spin system to a target state or gate, and the controls are a function of time, so I discretize time into slices with piecewise-constant amplitudes and write the evolution as a product of slice propagators. Maximizing the overlap with the target requires the gradient with respect to every control amplitude, and finite differences cost m·N+1 propagations — prohibitive. But cutting the propagator product at slice j and using cyclic invariance of the trace shows the objective is the overlap of a forward-propagated state with a backward-propagated target (the adjoint/co-state); perturbing only slice j with the matrix-exponential derivative gives a clean per-slice gradient, -⟨λ_j|iΔt[H_k,ρ_j]⟩ for states (and the phase-quotiented -2Re{⟨P_j|iΔt H_k X_j⟩⟨X_j|P_j⟩} for gates), every ingredient of which comes from one forward plus one backward sweep — so the whole m·N gradient costs two propagations, and I just step the amplitudes uphill (with rf-power penalties, amplitude clipping, and robustness-averaging bolted on for free) until the fidelity converges.
