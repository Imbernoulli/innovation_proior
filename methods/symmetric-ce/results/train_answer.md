We want to train an accurate deep classifier when a known-but-unidentified fraction of the training labels have been flipped to wrong classes, with the test set still clean — and to do it without architectural surgery, without auxiliary clean data, and without a known noise-transition matrix, since those are exactly the resources practitioners lack. The instinctive framing, inherited from the memorization results, is that label noise hurts because high-capacity networks eventually overfit the flipped targets, so a good method should suppress late-stage memorization. But if I break cross entropy's behavior out by class rather than reading the aggregate accuracy curve, the picture changes. With clean labels the per-class test-accuracy curves already fan into a wide band: "easy" classes converge fast, "hard" classes crawl, because some classes simply have more separable patterns. Under, say, 40% uniform noise the band does not merely shift down — the easy classes climb high and then droop (the overfitting everyone talks about), while the hard classes plateau far below their clean ceiling and never recover. On the clean *portion* of a hard class — examples whose labels were never touched — the network's confidence on the correct class hovers near $0.5$, leaking a few percent onto visually similar classes. So the dominant cost is not the easy-class droop, which is a few points; it is the persistent *under-learning* of hard classes. Any fix must therefore do two opposing things at once: supply extra learning pressure aimed at the half-learned hard classes, and be noise-tolerant itself so that in pushing harder it does not simply memorize the flipped labels.

That tension is the whole design constraint, and it kills the obvious moves. Just turning CE up — $2\,\ell_{ce}$, $5\,\ell_{ce}$ — scales every gradient indiscriminately, including the pull toward the *observed* label, so on a flipped example it pulls five times harder toward the wrong class; upscaling CE accelerates exactly the memorization I am trying to avoid. The extra push cannot come from more CE, because CE has no notion that a target might be a lie. To make "robust" precise I use Ghosh, Kumar and Sastry's criterion: a loss $L$ is noise-tolerant if the global minimizer of the noisy risk is also a global minimizer of the clean risk, and a sufficient condition is *symmetry* — that the loss summed over all classes is a constant, $\sum_{i=1}^{K} L(f(x), i) = C$, the same for every $x$ and every $f$. The reason this buys robustness is a short piece of algebra I will reuse: under uniform noise at rate $\eta$ each label is kept with probability $1-\eta$ and flipped uniformly to one of the $K-1$ other classes otherwise, so the inner sum over wrong classes is $\sum_{k\neq y} L = C - L(f(x),y)$ precisely because the total is the constant $C$, giving
$$R^{\eta}(f) = (1-\eta)\,R(f) + \tfrac{\eta}{K-1}\bigl(C - R(f)\bigr) = \tfrac{C\,\eta}{K-1} + \Bigl(1 - \tfrac{\eta K}{K-1}\Bigr)R(f).$$
The noisy risk is an affine function of the clean risk with a positive multiplier whenever $\eta < 1 - 1/K$, and a positive affine map preserves argmin — robustness, distribution-free. Now CE fails this test outright: $\sum_k -\log p_k$ depends on $p$ and is unbounded as any $p_k \to 0$, so it carries no guarantee. Mean absolute error passes it — for a one-hot label $\ell_{mae} = 2(1-p_y)$ and $\sum_i \ell_{mae} = 2(K-1)$ — but its gradient on the true-class logit is $-2\,p_y(1-p_y)$, which vanishes both as $p_y \to 1$ (fine, that example is learned) and as $p_y \to 0$ (a disaster: a hard example the network is currently getting wrong gets almost no signal). GCE, $\ell_q = (1 - p_y^{\,q})/q$, interpolates CE and MAE with one knob $q$, but a single scalar must set how fast the model fits *and* how robust it is on a single axis; on an intrinsically hard-to-fit dataset I would need CE's convergence and a lot of robustness, and one $q$ cannot give both. So I am caught — CE has the gradients but not the robustness, MAE has the robustness but not the gradients — and I want two *separate* forces I can dial independently rather than one dial sliding between them.

I propose Symmetric Cross Entropy Learning (SCE, or SL): keep CE for what it is good at and *add* a robust, cross-entropy-shaped counterpart, instead of deforming CE into MAE. The route to that counterpart is to look at what CE is structurally. CE is $H(q,p) = -\sum_k q_k \log p_k$, and through $\mathrm{KL}(q\Vert p) = H(q,p) - H(q)$ with $H(q)$ constant, minimizing CE *is* minimizing $\mathrm{KL}(q\Vert p)$ — fitting $p$ toward the observed label $q$. That asymmetric KL treats $q$ as ground truth, which is exactly the wrong thing under noise, where a competent $p$ often reflects the truth better than a corrupted $q$. So I measure the discrepancy in the other direction too, $\mathrm{KL}(p\Vert q)$, where the prediction is now the reference, and symmetrize: $\mathrm{SKL} = \mathrm{KL}(q\Vert p) + \mathrm{KL}(p\Vert q)$. Transferring that symmetry from KL to cross entropy, the mirror of $H(q,p)$ is $H(p,q) = -\sum_k p_k \log q_k$ — the same formula with $p$ and $q$ swapped — which I call Reverse Cross Entropy,
$$\ell_{rce} = -\sum_{k=1}^{K} p_k \log q_k,$$
so the combined objective is the cross-entropy analogue of symmetric KL, $\ell_{ce} + \ell_{rce}$, with CE pulling $p$ toward $q$ and RCE pulling in the mirror direction.

RCE has $q$ inside the logarithm, and $q$ is one-hot, so for every wrong class $\log q_k = \log 0 = -\infty$ and the term is ill-defined. Rather than apologize for it, I *define* $\log 0 := A$ for a finite negative constant $A$ — the same spirit as the probability clipping every framework already does. With $q_y = 1$ (so $\log q_y = 0$) and $q_{k\neq y} = 0$ (so $\log q_k = A$), RCE collapses to a clean finite quantity,
$$\ell_{rce} = -\sum_{k\neq y} p_k\,A = -A\,(1 - p_y),$$
and this definition is not a band-aid but the load-bearing move, because it is exactly what makes RCE symmetric. Summing over which class plays the target, $\ell_{rce}(f(x), i) = -A(1 - p_i)$, and
$$\sum_{i=1}^{K} \ell_{rce}(f(x), i) = -A\sum_{i=1}^{K}(1 - p_i) = -A\,(K - 1),$$
a constant independent of $x$ and $f$. So RCE satisfies Ghosh's symmetry condition with $C = -(K-1)A$, and substituting that constant into the affine identity gives $R^{\eta}(f) = -A\eta + (1 - \tfrac{\eta K}{K-1})R(f)$, so $R^{\eta}(f^{*}) - R^{\eta}(f) = (1 - \tfrac{\eta K}{K-1})(R(f^{*}) - R(f)) \le 0$ for $\eta < 1 - 1/K$ — RCE's clean minimizer is its noisy minimizer, provably noise-tolerant. (The class-conditional case needs more: rewriting the true-label term via $\ell_{rce}(f,y) = -(K-1)A - \sum_{k\neq y}\ell_{rce}(f,k)$ leaves coefficients $1 - \eta_y - \eta_{yk}$, which are positive under diagonal dominance $\eta_{yk} < 1 - \eta_y$, and with zero clean RCE risk this forces the noisy minimizer to put zero mass on every wrong class, so $f^{*}_{\eta} = f^{*}$.) A reassuring landmark: RCE for a sample is $-A(1-p_y)$, the same function of $p_y$ as MAE's $2(1-p_y)$, so $A = -2$ makes RCE *exactly* MAE — RCE is a one-parameter generalization of MAE where $A$ is a steepness knob, which tells me I am on the right family rather than off in the weeds.

What makes RCE also fix the under-learning, not just add robustness, is its gradient. Taking $\alpha=\beta=1$ for the moment and using the softmax Jacobian $\partial p_k/\partial z_j = p_k(1-p_k)$ for $k=j$ and $-p_j p_k$ otherwise, the combined loss has
$$\frac{\partial \ell_{sl}}{\partial z_j} = (p_j - q_j) + p_j\Bigl(\sum_{k=1}^{K} p_k \log q_k - \log q_j\Bigr),$$
where $(p_j - q_j)$ is the familiar CE gradient. Read it in two cases. For the labeled class ($q_j = q_y = 1$, so $\log q_j = 0$ and $\sum_k p_k \log q_k = (1-p_j)A$) it becomes $(p_j - 1) + A\,p_j(1-p_j)$; since $A<0$ the RCE contribution is non-positive, so gradient descent raises $p_y$ faster, and its magnitude $(-A)\,p_j(1-p_j)$ is an inverted parabola maximal at $p_j = 0.5$ and zero at the ends. That is precisely the adaptive pacing the under-learning called for: the biggest boost goes to the half-learned hard classes sitting near $p_y \approx 0.5$, and it tapers off as $p_y \to 1$ so well-learned classes are not hammered into overfitting — and unlike $2\,\ell_{ce}$ the boost is gated by $p_j(1-p_j)$ rather than scaling everything indiscriminately. For a wrong class ($q_j = 0$, so $\log q_j = A$ and $\sum_k p_k \log q_k = (1-p_y)A$) it becomes $p_j - A\,p_j p_y$; the extra $-A\,p_j p_y \ge 0$ suppresses residual wrong-class mass in proportion to $p_y$, so if the network is confident the labeled class is right it sharpens hard, but if $p_y \approx 0$ — which is exactly what a flipped label looks like from the inside — the suppression vanishes. The robust term self-gates, sharpening only predictions the network already half-believes and going quiet on examples it implicitly flags as mislabeled.

So I keep both forces and give myself two decoupled coefficients,
$$\ell_{sl} = \alpha\,\ell_{ce} + \beta\,\ell_{rce},$$
rather than a single mixing $\alpha/(1-\alpha)$, because the knobs do genuinely different jobs: $\alpha$ scales CE, the source of overfitting, so *lowering* $\alpha$ eases memorization, while $\beta$ scales the robust pressure, so *raising* $\beta$ adds noise-tolerant signal. Easy-to-converge data like CIFAR-10 wants a small $\alpha$ (around $0.1$, with $\beta = 1.0$) and MNIST smaller still ($\alpha = 0.01$); a notoriously hard-to-converge dataset like CIFAR-100 actually needs *more* CE to converge at all, so $\alpha = 6.0$ with $\beta = 0.1$ — an asymmetry a single mixing weight could not express. Because $\beta\,\ell_{rce} = \beta(-A)(1-p_y)$, the product $\beta(-A)$ sets the RCE strength, so $\beta$ and $A$ overlap; in practice I fix $A$ to a sensible negative constant and tune the coefficients, and the result is fairly insensitive to $A$ once CE's overfitting is controlled by a small $\alpha$. I realize $\log 0 := A$ not by special-casing it but by clamping the one-hot label up to a small positive floor before its log, so a floor $f$ gives $A = \log f$; the implementation uses $f = 10^{-4}$, hence $A \approx -9.21$. I prefer this clamp to label smoothing, which would put $\epsilon/(K-1)$ mass on every class and bias the model at every point including the clean $q_y = 1$ ones, whereas clamping introduces a controlled, finite bias only at the $q_k = 0$ points — and that bias is the very thing that supplies the constant $A$ and makes the term symmetric. The prediction $p$ is also clipped away from zero inside CE, the standard guard, so neither cross entropy blows up.

```python
import tensorflow as tf


def symmetric_cross_entropy(alpha, beta):
    """ell = alpha * CE + beta * RCE.

    CE  = -sum_k q_k log p_k
    RCE = -sum_k p_k log q_k
    Clipping the one-hot label to 1e-4 realizes A = log(1e-4)."""

    def loss(y_true, y_pred):
        y_true_1 = y_true
        y_pred_1 = y_pred
        y_true_2 = y_true
        y_pred_2 = y_pred

        y_pred_1 = tf.clip_by_value(y_pred_1, 1e-7, 1.0)
        y_true_2 = tf.clip_by_value(y_true_2, 1e-4, 1.0)

        ce = tf.reduce_mean(-tf.reduce_sum(y_true_1 * tf.log(y_pred_1), axis=-1))
        rce = tf.reduce_mean(-tf.reduce_sum(y_pred_2 * tf.log(y_true_2), axis=-1))
        return alpha * ce + beta * rce

    return loss
```
