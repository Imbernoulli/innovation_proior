## Large-Scale Nonsmooth Convex Optimization

The problem is to minimize a convex function `f` over a closed convex set `X` in `R^n`, using only first-order oracle access. At a queried point `x`, the oracle returns a subgradient `g in partial f(x)`. The function can be nonsmooth, and the dimension can be large. The target accuracy is modest: find a point whose value is within `epsilon` of the minimum over `X`. In this regime, the key question is whether each iteration can be cheap and whether the number of oracle calls can avoid a severe dependence on the ambient dimension.

## First-Order Information and Norms

A subgradient is a linear functional. Convexity gives the basic inequality

`f(x) - f(x*) <= <g, x - x*>`

for any minimizer `x*` and any `g in partial f(x)`. To bound the right-hand side, a norm on primal directions must be paired with its dual norm on subgradients:

`<g, d> <= ||g||_* ||d||`.

Thus the size of the gradient depends on which geometry is assigned to the feasible set and which dual geometry is assigned to the oracle output.

## The Euclidean Baseline

The standard constrained first-order method uses the iteration

`x_{k+1} = Pi_X(x_k - t_k g_k)`,

where `Pi_X` is Euclidean projection. Equivalently, it chooses the next point by minimizing a linearized objective plus a squared Euclidean penalty:

`argmin_{x in X} { <g_k, x> + (1/(2 t_k)) ||x - x_k||_2^2 }`.

This method needs one subgradient and one projection. With a tuned diminishing or horizon-dependent step size, its worst-case guarantee has the usual `1/sqrt(k)` form, with a constant controlled by a Euclidean diameter and a Euclidean Lipschitz bound.

## Geometry the Simplex Already Carries

The probability simplex is a diagnostic domain because its natural scale is not Euclidean. Points are distributions, and the natural movement is measured by `l1`-type quantities and relative entropy. Loss vectors or subgradients in expert-style problems are often bounded coordinatewise, so their natural bound is in `l_infinity`, not in `l2`.

## Available Mathematical Pieces

Several pre-existing tools are already on the table. Strong convexity gives a way to make a tangent approximation leave a quadratic lower gap in a chosen norm. Convex conjugacy relates a strongly convex function to a smooth inverse mapping on dual variables. Bregman-type tangent gaps provide nonnegative, distance-like penalties generated by convex functions, and relative entropy is the canonical such gap on the simplex.
