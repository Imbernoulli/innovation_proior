I start with a classifier that learns in exactly the wrong place. Its categories live in fixed output weights, and the only way to add a category is to move weights with gradient descent. One labeled example is not enough evidence for that, and a few aggressive steps on a new class can overwrite the old representation. But the test condition does not let me fine-tune anyway. I need the support example itself to become usable immediately.

Nearest-neighbor methods already have that property. If I store a new example, it is instantly part of the classifier. The weakness is that all the intelligence sits in the distance function, and a fixed distance in pixel space or in a feature space trained for base-class softmax labels is only accidentally aligned with the new task. So I do not want to abandon non-parametric classification; I want to learn the space and the comparison rule while keeping the support set as the memory.

NCA gives me the first move. Hard nearest-neighbor classification is discontinuous in the metric, but a soft stochastic neighbor is smooth: `p_ij = exp(-||A x_i - A x_j||^2) / sum_{l != i} exp(-||A x_i - A x_l||^2)`, with `p_ii = 0`, and the probability of correct classification is the total mass on same-class neighbors. The sign is important: NCA uses negative squared distance because nearer points should receive larger mass. That solves differentiability, but not the whole problem. A fixed linear map `A` and a leave-one-out objective over the training set are not shaped like an episode with a fresh support set.

So I make the support set the reference set directly. For a query `x_hat` and support `S = {(x_i, y_i)}_{i=1}^k`, I write the prediction as `y_hat = sum_i a(x_hat, x_i) y_i`. If each `y_i` is one-hot and the weights `a_i` are nonnegative and sum to one, then `y_hat` is a distribution over the episode's labels. In the multi-shot case, the probability of class `n` is not a separate classifier output learned in weights; it is `sum_{i: y_i = e_n} a_i`, the total attention paid to support examples carrying that class label. In the one-shot case, that collapses to one attention weight per class. This handles every shot count without changing the parameterization.

The weights should be smooth, so I use a softmax over similarities: `a_i = exp(c(f(x_hat), g(x_i))) / sum_j exp(c(f(x_hat), g(x_j)))`. This is the NCA soft-neighbor trick moved from a fixed training set onto the episode's support set. The embeddings `f` and `g` can be deep, so I am no longer limited to a Mahalanobis metric. The readout also has the right memory interpretation: support labels are values bound to support inputs, and the query reads a convex combination of those values.

For the paper-level classifier, the similarity `c` is cosine. That choice removes norm from the comparison and makes the vote depend on direction in feature space rather than on whichever embedding has the largest magnitude. I have to be precise when I later implement it: the cited PyTorch reference normalizes the support embeddings before the final matrix multiply but leaves the query embeddings unnormalized, so the code computes `||f|| * cosine(f, g)` rather than literal symmetric cosine. That is a scaled-cosine vote with a query-dependent softmax temperature. It is reference-faithful, but it should not be described as exact cosine in code.

Now I need training to match the test condition. If I train a base-class softmax and then hope its hidden layer supports nearest-neighbor transfer, I am back to borrowed features. Instead I sample tasks during training. Draw a label set `L` from a task distribution, draw a support set `S` and a query batch `B` from those labels, and maximize `E_{L} E_{S,B} sum_{(x,y) in B} log P_theta(y | x, S)`. Equivalently, implementation minimizes the negative log-likelihood on the query labels. The sign is therefore consistent: the derivation is an argmax of log probability, while the loss is `-log P` through NLL.

At this point the classifier is already usable, but the embeddings are myopic. If each support vector `g'(x_i)` is computed independently, it cannot react to another support example of a different class sitting nearby. If the query vector `f'(x_hat)` is computed independently, it cannot emphasize the features that distinguish the particular candidate classes in this episode. The embedding functions themselves should be conditional on the support set.

For support embeddings, I run a bidirectional LSTM over the support features and add a residual connection: `g(x_i, S) = h_forward_i + h_backward_i + g'(x_i)`. The forward recurrence reads from the start up to `i`, the backward recurrence reads from the end down to `i`, and their sum gives each position context from the whole support sequence. The skip connection matters because the recurrent part only has to learn a contextual correction; the raw feature is preserved.

For query embeddings, I use an attention LSTM that repeatedly reads the support memory. At processing step `k`, it has previous hidden state `h_{k-1}` and previous readout `r_{k-1}`. It computes `h_hat_k, c_k = LSTM(f'(x_hat), [h_{k-1}, r_{k-1}], c_{k-1})`, sets `h_k = h_hat_k + f'(x_hat)`, and forms the readout `r_{k-1} = sum_i softmax_i(h_{k-1}^T g(x_i)) g(x_i)`. The softmax here is over support index `i`; its scores are dot products inside the refinement loop, not the final class vote. After `K` reads, `f(x_hat, S) = h_K`. The paper defines `K` as a fixed number of processing steps. The local port uses the number of support examples as the loop count, while the Facebook reference accepts `K` as a constructor argument.

The final tensor path is now clear. I contextualize the support features, one-hot the support labels, contextualize each query against that support memory, compute a support-normalized scaled-cosine score matrix, softmax over support examples, multiply by the one-hot support-label matrix, and take a log before NLL. The matrix multiply by labels is the multi-shot class aggregation; the log is valid because the attention vote is already a probability distribution over classes. The local implementation adds `1e-6` before the log as a numerical floor, which slightly breaks exact normalization but avoids `log(0)` when a class receives no mass.

So the design chain is complete. I keep the non-parametric ability to install a new classifier by changing the support set; I use a differentiable soft attention vote so the metric can be trained; I train on support/query episodes so the objective has the same shape as evaluation; and I condition both support and query embeddings on the support set so the comparison is not blind to the particular candidates in the episode. The result is a model whose learned parameters define the comparison machinery, while the episode's support set supplies the actual class-specific memory.
