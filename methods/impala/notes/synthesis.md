# IMPALA / V-trace — synthesis notes (Phase 1.5)

## Pain point
Single-task deep RL (A3C, UNREAL) needs ~1e9 frames / multiple days for ONE
domain. We want one agent on many tasks at once → need far more throughput.
Scaling is the binding constraint. Want to scale to thousands of machines
WITHOUT losing data efficiency or stability.

## Ancestors / load-bearing prior art
- **A3C (Mnih et al. 2016)**: asynchronous advantage actor-critic. Many actor
  threads each hold a copy of params, run env, compute *gradients* w.r.t. their
  local copy, push gradients to a central parameter server (Hogwild-style async
  SGD, NIPS 2012 Dean et al. DistBelief). On-policy: each worker's data matches
  its own (slightly stale) params and it corrects with on-policy n-step
  advantage. Limitations for scale: (1) communicating *gradients* is heavy and
  couples to model size; (2) the per-worker forward/backward is small-batch, bad
  GPU utilisation; (3) gradient staleness across many async workers degrades.
- **Gorila**: distributed DQN. Same async-SGD-of-gradients template.
- **Batched A2C (Clemente et al. 2017)**: synchronous; one learner steps a batch
  of envs together. Problem: the SLOWEST env in the batch gates the whole batch
  step → high env-time variance kills throughput (fine for cheap Atari render,
  bad for DMLab 3D). Two sync granularities: sync every step, or sync every n
  steps; both stall.
- **GA3C (Babaeizadeh et al. 2016)**: decouples acting (forward pass) from
  learning (backward) with dynamic batching on GPU — closest architecture.
  But the actor/learner asynchrony makes the data off-policy and it learns
  on-policy → instability; they only patch it by adding a small constant to
  action probs in the PG estimate ("ε-correction"). Ad hoc, not principled.
- **Importance sampling (IS)**: to evaluate target π from behaviour μ samples,
  reweight by ∏ π/μ. Unbiased but the product over a trajectory has variance
  that explodes with horizon and with how off-policy you are.
- **Retrace(λ) (Munos et al. 2016)**: off-policy multi-step correction with
  truncated traces c_i = λ min(1, π/μ). Safe (converges for any μ,π), efficient.
  BUT it corrects a state-ACTION value Q (it's built for Q). Many actor-critics
  (A3C) learn a state-value V, not Q. Also Retrace does NOT reduce to the plain
  on-policy n-step return in the on-policy case (its λ/trace machinery stays).
- Other off-policy: Precup et al. 2000/2001 eligibility traces & off-policy TD;
  Wawrzynski 2009; Geist & Scherrer 2014; O'Donoghue PGQ 2016; Harutyunyan
  Q(λ) 2016.

## Architecture (IMPALA)
- Decouple acting from learning. **Actors** each: copy latest learner policy π
  into local μ at start of trajectory, run μ for n steps, send the trajectory
  (x_t, a_t, r_t) AND the behaviour probs μ(a_t|x_t) (+ initial LSTM state) to
  the **learner** via a queue. Actors send *observations/trajectories*, NOT
  gradients (lower comms, decoupled from model size, fault tolerant).
- Learner: continuously SGD on batches of trajectories from many actors. GPU,
  big batch, parallelise all time-independent ops (fold time into batch for
  conv & output layer; LSTM fused). Throughput 250k fps, >30x single-machine A3C.
- Multi-learner: params sharded across learners, synchronous param update
  (Chen et al. 2016 — sync matters for data efficiency at scale).
- The catch: learner's π can be SEVERAL updates ahead of the actor's μ at
  gradient time → **policy-lag** → data is off-policy. Need an off-policy
  value correction. → V-trace.

## V-trace target (the core)
Trajectory (x_t,a_t,r_t) from t=s..s+n generated by μ. n-step V-trace target:
  v_s = V(x_s) + Σ_{t=s}^{s+n-1} γ^{t-s} (∏_{i=s}^{t-1} c_i) δ_t V
with
  δ_t V = ρ_t ( r_t + γ V(x_{t+1}) − V(x_t) )   [temporal difference]
  ρ_t   = min(ρ̄, π(a_t|x_t)/μ(a_t|x_t))         [truncated IS, in the TD]
  c_i   = min(c̄, π(a_i|x_i)/μ(a_i|x_i))          [truncated IS, the trace]
  ∏_{i=s}^{t-1} c_i = 1 when s=t. Assume ρ̄ ≥ c̄.

On-policy special case (π=μ, and c̄≥1 so all c_i=1, ρ_t=1): telescopes to the
n-step Bellman target v_s = Σ γ^{t-s} r_t + γ^n V(x_{s+n}). So same algo for on-
and off-policy (Retrace does NOT have this). Verify telescoping:
v_s = V(x_s) + Σ_{t=s}^{s+n-1} γ^{t-s}(r_t + γV(x_{t+1}) − V(x_t)); the −γ^{t-s}V(x_t)
of term t cancels the +γ^{(t-1)-s}·γ V(x_t)=γ^{t-s}V(x_t) of term t−1; only first
−V(x_s) (kills the leading V(x_s)) and last γ^n V(x_{s+n}) survive. ✓

Recursive form (used in code): v_s = V(x_s) + δ_s V + γ c_s (v_{s+1} − V(x_{s+1})).

## ρ̄ vs c̄ — DISTINCT ROLES (the crux)
- **ρ̄ sets the FIXED POINT (bias / which policy).** ρ_t sits inside δ_t V. The
  fixed point (δ_t V = 0 in expectation under μ) is the value function
  V^{π_{ρ̄}} of the policy
     π_{ρ̄}(a|x) = min(ρ̄ μ(a|x), π(a|x)) / Σ_b min(ρ̄ μ(b|x), π(b|x)).
  ρ̄=∞ → π_{ρ̄}=π → V^π (target). Finite ρ̄ → a policy between μ and π. ρ̄→0 →
  V^μ. Larger ρ̄ = smaller off-policy bias. Variance grows with ρ̄ but ρ_t is NOT
  taken as a product over time, so its variance does not explode with horizon.
- **c̄ controls VARIANCE & CONTRACTION SPEED, NOT the fixed point.** The c_i are
  the trace ("trace-cutting", like Retrace). Their product c_s…c_{t-1} weights
  how much a TD at time t affects the update at earlier time s. This PRODUCT is
  where variance explodes off-policy, so truncating with c̄ is the key variance
  reduction. c̄ enters the contraction modulus η (speed). The fixed point is
  INDEPENDENT of c̄ — depends on ρ̄ only. (Need c̄ ≤ ρ̄ for the proof.)
- Slogan: ρ̄ = which function we converge to; c̄ = how fast we get there.

## Contraction theorem (Appendix, full proof to be lived out)
Infinite-horizon operator:
  R V(x) = V(x) + E_μ[ Σ_{t≥0} γ^t (c_0…c_{t-1}) ρ_t (r_t+γV(x_{t+1})−V(x_t)) | x_0=x ].
Assume ρ̄≥c̄ and ∃ β∈(0,1] with E_μ ρ_0 ≥ β. Then R is an η-contraction in
sup-norm with unique fixed point V^{π_{ρ̄}}, where
  η = γ^{-1} − (γ^{-1}−1) E_μ[ Σ_{t≥0} γ^t (∏_{i=0}^{t-2} c_i) ρ_{t-1} ]
    ≤ 1 − (1−γ)β < 1.

Proof skeleton (must reproduce every step):
1. Rewrite R V(x) = (1−E_μρ_0) V(x) + E_μ[ Σ_{t≥0} γ^t (∏_{s=0}^{t-1} c_s)
   ( ρ_t r_t + γ(ρ_t − c_t ρ_{t+1}) V(x_{t+1}) ) ].
   (Group the −ρ_t V(x_t) of term t with the +γ c_{t-1} ρ_t V(x_t)... carefully:
   the V(x_t) terms; reindex.)
2. R V_1 − R V_2 = E_μ[ Σ_{t≥0} γ^t (∏_{s=0}^{t-2} c_s) (ρ_{t-1} − c_{t-1} ρ_t)
   (V_1(x_t) − V_2(x_t)) ], with c_{-1}=ρ_{-1}=1, empty product=1.
3. Coefficients α_t = ρ_{t-1} − c_{t-1} ρ_t are ≥0 in expectation:
   E_μ α_t = E[ρ_{t-1} − c_{t-1} ρ_t] ≥ E_μ[c_{t-1}(1 − ρ_t)] ≥ 0
   because c_{t-1} ≤ ρ_{t-1} (since c̄≤ρ̄ and same ratio), and E_μ ρ_t ≤ E_μ[π/μ]=1.
   (The first ≥: ρ_{t-1} − c_{t-1}ρ_t ≥ c_{t-1} − c_{t-1}ρ_t = c_{t-1}(1−ρ_t)
   when ρ_{t-1} ≥ c_{t-1}. Then E[c_{t-1}(1−ρ_t)]: c_{t-1}≥0 and E[1−ρ_t]≥0... need
   measurability; the bound E_μρ_t≤1 gives it.)
4. Sum of the non-negative coefficients telescopes:
   Σ γ^t E[(∏_{s=0}^{t-2}c_s)(ρ_{t-1}−c_{t-1}ρ_t)]
   = Σ γ^t E[(∏_{0}^{t-2}c_s)ρ_{t-1}] − γ^{-1}(Σ γ^t E[(∏_{0}^{t-2}c_s)ρ_{t-1}] − 1)
   = γ^{-1} − (γ^{-1}−1)·S, where S=Σ_{t≥0}γ^t E[(∏_{0}^{t-2}c_s)ρ_{t-1}] ≥ 1+γE_μρ_0.
   ⇒ ≤ 1 − (1−γ)E_μρ_0 ≤ 1−(1−γ)β < 1. (Since γ^{-1}>1 and S≥1+γβ... compute:
   γ^{-1}−(γ^{-1}−1)(1+γβ) = γ^{-1} − γ^{-1} − γ^{-1}γβ + 1 + γβ
   = 1 + γβ − β = 1 − β(1−γ). ✓)
   ⇒ ‖R V_1 − R V_2‖_∞ ≤ η ‖V_1−V_2‖_∞.
5. Fixed point is V^{π_{ρ̄}}: show E_μ[ρ_t(r_t+γV^{π_{ρ̄}}(x_{t+1})−V^{π_{ρ̄}}(x_t))|x_t]=0.
   Σ_a μ(a)min(ρ̄,π/μ)[r+γΣ_y p V −V] = [Σ_a π_{ρ̄}(a)(r+γΣpV−V)]·Σ_b min(ρ̄μ(b),π(b)).
   First factor = 0 (Bellman eq for V^{π_{ρ̄}}), since
   μ(a)min(ρ̄,π/μ)=min(ρ̄μ(a),π(a)) = π_{ρ̄}(a)·(normaliser). ⇒ R V^{π_{ρ̄}}=V^{π_{ρ̄}}.

## Online convergence (Thm 2)
Tabular, all states visited infinitely often, Robbins-Monro stepsizes
(Σα=∞, Σα²<∞) ⇒ V_k → V^{π_{ρ̄}} a.s. (stochastic approximation to contraction
fixed point; Dayan 1994 / Bertsekas-Tsitsiklis 1996 / Kushner-Yin 2003).

## Policy gradient (V-trace actor-critic)
On-policy PG: ∇V^μ(x_0) = E_μ[ Σ γ^s ∇logμ(a_s|x_s) Q^μ(x_s,a_s) ].
Off-policy: evaluate π_{ρ̄}, use IS ratio between π_{ρ̄} and μ:
  E_{a~μ}[ (π_{ρ̄}(a_s|x_s)/μ(a_s|x_s)) ∇logπ_{ρ̄}(a_s|x_s) q_s | x_s ],
with q_s = r_s + γ v_{s+1} (estimate of Q^{π_{ρ̄}}(x_s,a_s)). Note
ρ_s = min(ρ̄,π/μ) plays the role of that leading IS ratio (truncated). So the
PG direction in practice:
  ρ_s ∇_ω logπ_ω(a_s|x_s) (r_s + γ v_{s+1} − V_θ(x_s)),  with V(x_s) baseline.

Why q_s = r_s + γ v_{s+1} and NOT v_s as Q-target? Appendix: if V=V^{π_{ρ̄}} exactly,
E[q_s|x_s,a_s] = r_s + γ E[V^{π_{ρ̄}}(x_{s+1}) + δ_{s+1}V + γc_{s+1}δ_{s+2}V + …]
= r_s + γE[V^{π_{ρ̄}}(x_{s+1})] = Q^{π_{ρ̄}}(x_s,a_s)  (all δ's vanish in expectation).
Whereas E[v_s|x_s,a_s] = V^{π_{ρ̄}}(x_s)(1−ρ_s) + ρ_s Q^{π_{ρ̄}}(x_s,a_s) ≠ Q unless
V=Q. So v_s is biased as a Q-estimate; q_s is unbiased under perfect V. Use it.

Value loss: gradient descent on l2 (v_s − V_θ(x_s))² → direction
(v_s − V_θ(x_s)) ∇_θ V_θ(x_s). Entropy bonus −∇_ω Σ_a π log π (anti premature
convergence, like A3C). Total = weighted sum of the three.

## Code grounding (DeepMind scalable_agent vtrace.py + experiment.py)
- vtrace.from_importance_weights: rhos=exp(log_rhos); clipped_rhos=min(ρ̄,rhos)
  (ρ̄=clip_rho_threshold, default 1.0); cs=min(1.0,rhos) (c̄=1.0 in code);
  deltas = clipped_rhos*(r + γ·V_{t+1} − V); reverse tf.scan with
  acc = delta_t + γ·c_t·acc → vs_minus_v_xs; vs = that + values. ← exactly the
  recursive remark. pg_advantages = clipped_pg_rhos*(r + γ·vs_{t+1} − V) with
  clip_pg_rho_threshold (the ρ_s in the PG). stop_gradient on outputs.
- bootstrap_value = V at time T (last baseline) for the n-step truncation tail.
- experiment.py build_learner: policy_gradient_loss = Σ cross_entropy(logits,
  actions)·pg_advantages (cross_entropy = −logπ, so loss gradient gives
  −ρ_s ∇logπ · adv → ascends); baseline_loss = 0.5 Σ (vs − V)²;
  entropy_loss = −Σ entropy. total = pg + baseline_cost·baseline + entropy_cost·entropy.
  RMSProp, lr polynomial decay. Actors send behaviour logits μ; learner recomputes
  target logits π. discounts = (~done)·γ.
- Note code uses c̄ = ρ̄ = 1 (clip at 1) by default; clip_pg_rho_threshold=1.
