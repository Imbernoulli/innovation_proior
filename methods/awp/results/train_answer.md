Adversarial training succeeds at making models robust on the training set, but that robustness fails to generalize. On CIFAR-10 with a PreAct ResNet-18, PGD adversarial training can reach roughly 84% robust training accuracy while test robustness peaks around 43% and then declines as training continues. That 40-point train-test gap is robust overfitting, and it is the dominant bottleneck for adversarial robustness. Early stopping can recover a better test number, but only by halting before the model has learned much, so it trades away training robustness for a smaller gap. The real goal is to keep training robustness high while making it generalize.

The standard approaches attack the wrong surface. Vanilla PGD adversarial training, TRADES, and MART all reshape the input-side loss: they change which adversarial examples are generated or how the clean and adversarial losses are weighted. These redesigns improve the robustness-accuracy trade-off, yet they still overfit the robust objective late in training. What they share is that none of them directly controls the geometry of the loss as a function of the weights. When the weight loss landscape is measured properly, with adversarial examples regenerated on-the-fly for each perturbed model and directions rescaled to respect ReLU scale invariance, flatter landscapes correlate cleanly with smaller robust generalization gaps. The existing methods flatten this surface only as a side effect. The next step is to flatten it explicitly.

The method is Adversarial Weight Perturbation, or AWP. It adds a second, outer perturbation to the usual inner input perturbation of adversarial training, forming a double perturbation: inputs are perturbed toward their worst case, and weights are perturbed toward the worst case for the whole batch. The objective becomes min_w max_{v in V} rho(w + v), where rho is the adversarial loss and v is a weight perturbation drawn from a feasible region V. This looks like adding a flatness penalty rho(w + v) - rho(w) to the loss, but because the penalty and the original loss share rho(w), the objective collapses to simply minimizing the loss at the perturbed weights. The update descends from a deliberately high-loss nearby point, so sharp local rises in weight space are penalized without introducing a separate regularization coefficient.

A random weight perturbation would be a weak probe, since most random directions in high-dimensional weight space barely affect the loss. AWP therefore uses the worst-case perturbation, found by projected gradient ascent on the weights, just as PGD performs projected gradient ascent on the input. The worst-case sharpness also gives a conservative upper bound on the expected sharpness term in the PAC-Bayes flatness bound, so minimizing it squeezes the data-dependent part of the bound that controls generalization. The perturbation size must be defined per layer relative to the layer's own weight norm, because ReLU networks are scale invariant and a fixed absolute size would mean different things for differently scaled layers. The constraint is ||v_l|| <= gamma ||w_l|| for each weight tensor, where gamma is a single small dimensionless constant, typically around 5e-3. Under an L2 per-layer budget the steepest-ascent direction is the normalized gradient, so one proxy SGD step on the negated loss gives the ascent direction, and that direction is then rescaled to length ||w_l|| and multiplied by gamma.

The implementation is built on top of the KL-regularized TRADES base loss. For each minibatch, the adversarial input is first generated by maximizing KL(clean prediction || adversarial prediction) with PGD. Then a proxy copy of the model is loaded with the current weights, and one SGD step is taken on the negated TRADES loss to move the proxy in the direction that most increases the loss. The difference between proxy weights and model weights is renormalized per weight tensor to have norm ||w_l||, scaled by gamma, and added to the real model. The actual TRADES loss is computed and back-propagated at these perturbed weights, the optimizer steps, and then the gamma-scaled perturbation is subtracted to restore the center weights. The proxy copy protects the real model's batch-normalization running statistics from being polluted by the throwaway forward passes used to compute the worst-case direction.

```python
import copy
import torch
import torch.nn.functional as F
from collections import OrderedDict

EPS = 1e-20


def diff_in_weights(model, proxy):
    diff_dict = OrderedDict()
    model_state_dict = model.state_dict()
    proxy_state_dict = proxy.state_dict()
    for (old_k, old_w), (_, new_w) in zip(model_state_dict.items(),
                                          proxy_state_dict.items()):
        if len(old_w.size()) <= 1:
            continue
        if 'weight' in old_k:
            diff_w = new_w - old_w
            diff_dict[old_k] = old_w.norm() / (diff_w.norm() + EPS) * diff_w
    return diff_dict


def add_into_weights(model, diff, coeff=1.0):
    names_in_diff = diff.keys()
    with torch.no_grad():
        for name, param in model.named_parameters():
            if name in names_in_diff:
                param.add_(coeff * diff[name])


class TradesAWP:
    def __init__(self, model, proxy, proxy_optim, gamma):
        self.model = model
        self.proxy = proxy
        self.proxy_optim = proxy_optim
        self.gamma = gamma

    def calc_awp(self, inputs_adv, inputs_clean, targets, beta):
        self.proxy.load_state_dict(self.model.state_dict())
        self.proxy.train()

        loss_natural = F.cross_entropy(self.proxy(inputs_clean), targets)
        loss_robust = F.kl_div(
            F.log_softmax(self.proxy(inputs_adv), dim=1),
            F.softmax(self.proxy(inputs_clean), dim=1),
            reduction='batchmean',
        )
        loss = -1.0 * (loss_natural + beta * loss_robust)

        self.proxy_optim.zero_grad()
        loss.backward()
        self.proxy_optim.step()

        return diff_in_weights(self.model, self.proxy)

    def perturb(self, diff):
        add_into_weights(self.model, diff, coeff=self.gamma)

    def restore(self, diff):
        add_into_weights(self.model, diff, coeff=-self.gamma)


def perturb_input_trades(model, images, eps, step_size, perturb_steps):
    model.eval()
    adv_images = images.detach() + 0.001 * torch.randn_like(images)
    adv_images = torch.clamp(adv_images, 0.0, 1.0)
    for _ in range(perturb_steps):
        adv_images.requires_grad_()
        loss_kl = F.kl_div(
            F.log_softmax(model(adv_images), dim=1),
            F.softmax(model(images), dim=1),
            reduction='sum',
        )
        grad = torch.autograd.grad(loss_kl, [adv_images])[0]
        adv_images = adv_images.detach() + step_size * torch.sign(grad.detach())
        adv_images = torch.min(torch.max(adv_images, images - eps), images + eps)
        adv_images = torch.clamp(adv_images, 0.0, 1.0)
    return adv_images.detach()


def train_step(model, images, labels, optimizer, awp_adversary,
               eps, step_size, perturb_steps, beta=6.0):
    x_adv = perturb_input_trades(model, images, eps, step_size, perturb_steps)

    model.train()
    awp = awp_adversary.calc_awp(
        inputs_adv=x_adv,
        inputs_clean=images,
        targets=labels,
        beta=beta,
    )
    awp_adversary.perturb(awp)

    optimizer.zero_grad()
    logits_adv = model(x_adv)
    loss_robust = F.kl_div(
        F.log_softmax(logits_adv, dim=1),
        F.softmax(model(images), dim=1),
        reduction='batchmean',
    )
    logits = model(images)
    loss_natural = F.cross_entropy(logits, labels)
    loss = loss_natural + beta * loss_robust
    loss.backward()
    optimizer.step()

    awp_adversary.restore(awp)
    return {'loss': loss.item()}
```
