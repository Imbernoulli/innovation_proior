Let me start from what is actually bothering me, which is a list. Tautology-checking. Subgraph isomorphism. Hamiltonian circuit. Whether a system of linear inequalities has a 0/1 solution. Finding a minimal disjunctive normal form for a partial Boolean function. Graph isomorphism. For every one of these the only thing anyone can do is search — run through all the truth assignments, all the embeddings, all the subsets — and the running time blows up exponentially in the size of the input. People have been hammering on these individually for years. No polynomial algorithm for any of them, and no proof that there can't be one. I keep coming back to the fact that they *feel* the same. Same wall, over and over. The honest question is whether that feeling means anything, or whether each of these is just its own private accident of being hard.

I have a sharp enough notion of "easy" to even ask. Polynomial time. Cobham made the case that the functions computable within a polynomial time bound form the natural, machine-independent class of the feasible ones, and Edmonds, working on matching, called a polynomial-bounded method a "good" algorithm and explicitly contrasted it with the merely-finite exponential search — and conjectured that some problems have no good algorithm. So the line is drawn: polynomial = good, exponential search = the thing to escape. Fine. That gives me a yardstick. It does not, by itself, give me any way to relate one problem to another. The yardstick measures each problem alone.

So what is the actual shape of these problems? Let me look at them squinting. Tautology: is there a falsifying assignment? — better, dually, satisfiability: is there a satisfying assignment? Subgraph isomorphism: is there an embedding of one graph into the other? Hamiltonian circuit: is there a cyclic ordering of the vertices using only edges? 0/1 programming: is there a 0/1 vector satisfying the inequalities? Every single one is "does there exist a thing of modest size — an assignment, an embedding, an ordering, a vector — that passes a simple, fast check?" That's the unifying shape. Not the answers, the *form* of the question. Given x, is there a y, not much longer than x, with A(x,y) true, where A is something you can verify quickly?

And that form has a name in machine terms. A machine that may *guess* — a nondeterministic Turing machine — accepts an input exactly when *some* branch of its guessing leads to acceptance. If it runs in polynomial time, then the branch is a polynomial-length record of guesses, and a deterministic machine could check that record step by step in polynomial time. So "nondeterministic polynomial-time recognizable" is precisely "there is a short, poly-time-checkable certificate of a yes-answer." That is exactly my squinting-shape. The whole list lives in this class. Call it NP, the languages a nondeterministic polynomial-time machine accepts. Every problem on my list is in NP because the witness — the assignment, the embedding, the ordering — is short and checkable.

Now here is the thing I actually want. These problems all *being* in NP is just a restatement of their shape; it doesn't tie their difficulties together. What would tie them together is a *reduction*: a polynomial-time map that turns instances of one problem into equivalent instances of another. If I had a poly-time f with "x in A iff f(x) in B," then any fast algorithm for B gives a fast algorithm for A — run f, then the B-algorithm — because the composition of two polynomials is a polynomial. Reducibility is transitive, and it carries membership in P downward: if B is in P and A reduces to B then A is in P. That is the right glue. The question sharpens enormously: instead of "is each of these hard," ask "do they all reduce to one another?" And the sharpest possible version: is there a *single* problem in NP to which *every* problem in NP reduces? Because if there is — call it complete — then that one problem is at least as hard as the entire class, and the whole sprawling list collapses to one question: is *that* problem in P? If it is, P = NP and everything is easy; if it isn't, none of them are.

So I need a candidate. And I need to actually prove that an *arbitrary* member of NP reduces to it. Not a clever pairing of two problems I happen to understand — an arbitrary one, a machine I know nothing about except that it is nondeterministic and runs in time bounded by some polynomial.

What do I actually know about an arbitrary NP problem? Only this: it is recognized by some nondeterministic Turing machine M whose running time on inputs of length n is at most some polynomial p(n). Nothing else. So the reduction has to be built out of the *only* thing common to all of them — the bare fact that acceptance means "there is an accepting computation of M, of polynomial length, on this input." The map f, given input w, must produce an instance of my candidate problem that is a yes-instance exactly when such an accepting computation exists.

That phrase — "such an accepting computation exists" — is itself a guess-and-check existence statement. There exists a computation; check that it is a valid, accepting computation of M on w. If I could write the *checking* condition as an object of one of my problems, I'd be done. Which problem is built to express checking conditions? The propositional calculus. A Boolean formula is *nothing but* a conjunction of conditions over yes/no variables, and asking whether it can be satisfied is asking whether there's an assignment — a "thing" — making all the conditions hold. Satisfiability is the problem whose very nature is "is there an assignment passing this conjunction of local checks." If a computation can be turned into a conjunction of local checks over Boolean variables, then "does an accepting computation exist" becomes "is this formula satisfiable." That is why the target should be satisfiability. It isn't an arbitrary pick — it's the one problem on the list whose machinery *is* the encoding of constraints. And it's the very problem already sitting at the center of mechanical theorem proving, where the Davis–Putnam procedure decides CNF satisfiability and its dual decides tautology, and where nobody can say whether it needs more than polynomial time. The thing I've been circling and the thing the theorem-proving people can't pin down are the same thing.

So now the real work: take an arbitrary nondeterministic poly-time machine M and an input w, and manufacture a propositional formula that is satisfiable iff M accepts w, with the manufacturing done in time polynomial in |w|. Let me think about what a computation of M *is* concretely, because the encoding has to mirror it exactly.

M has a tape, a head, a control state. At each time step there's a configuration: the tape contents, the head position, and the current state. The computation is a sequence of these, step 0, step 1, …, each obtained from the one before by a single move of the transition function δ. If the running time is bounded by some polynomial p(n), I can choose a padded bound T = O(n^k) with T ≥ p(n)+n+2; this gives enough rows for p(n) moves plus the start row, and enough columns for the input plus every cell the head can reach. In T rows the head can visit only O(T) tape squares, so a table with T time rows and T interior columns is enough. I also want fixed boundary symbols at the two ends, because otherwise a local window at the edge could miss a bad move. So the columns are 0,1,…,T,T+1, with column 0 and column T+1 always holding #. A row is a configuration string: tape symbols, plus one state marker q placed immediately before the symbol currently scanned. The start row is #, q_0, w_1, …, w_n, blanks, #. Somewhere a row contains the accept marker. That table *is* the computation.

Why is laying it out as a grid the right move? Because validity becomes *local*. "This is a legal computation" sounds global — every step must follow from the one before — but a single δ-move is a tiny, local change. It rewrites the scanned cell, changes the state, and shifts the head by one. Everything else on the tape stays put. So whether row t+1 legally follows row t is determined cell-by-cell by looking at small neighborhoods. Locality is exactly what will let me write the condition as a *polynomial-size* conjunction of constant-size pieces. If validity were genuinely global I'd have no hope of a small formula.

Now, variables. A satisfying assignment has to *be* the table — that's the cleanest correspondence. So let me make one Boolean variable per (table position, possible content) pair. Let Σ_T be the finite set Γ ∪ Q ∪ {#}: tape symbols, state markers, and the boundary marker. For row i, column j, and symbol σ ∈ Σ_T, the variable x_{i,j,σ} means "position (i,j) contains σ." A truth assignment to all these variables is a way of writing one candidate configuration string in every row. The formula's job is to be true exactly on assignments that are *genuine accepting computations of M on w*.

I'll split that into four jobs and write a constraint group for each. Call the formula φ = φ_cell ∧ φ_start ∧ φ_move ∧ φ_accept.

First, φ_cell, because a raw assignment to the x's might put two symbols in one table position or none — that's not a table at all. I need every position to hold exactly one symbol. "At least one": the disjunction over σ of x_{i,j,σ}. "At most one": for every pair σ ≠ τ, the clause (¬x_{i,j,σ} ∨ ¬x_{i,j,τ}). Conjoin over all positions:
φ_cell = ⋀_{1≤i≤T, 0≤j≤T+1} [ (⋁_{σ∈Σ_T} x_{i,j,σ}) ∧ ⋀_{σ≠τ} (¬x_{i,j,σ} ∨ ¬x_{i,j,τ}) ].
There are T(T+2) positions and a constant number of symbols, so this is O(T^2) = O(n^{2k}) clauses of constant size. Good — polynomial.

Second, φ_start, pinning the first row to the actual start configuration of M on w and pinning the boundary columns for every row. Without this the formula would happily accept a "computation" beginning from some configuration that has nothing to do with w, or one that leaks through an edge. The start configuration string is #, q_0, w_1, …, w_n, then blanks, then #. So:
φ_start = (⋀_{i=1}^{T} (x_{i,0,#} ∧ x_{i,T+1,#})) ∧ x_{1,1,q_0} ∧ x_{1,2,w_1} ∧ x_{1,3,w_2} ∧ … ∧ x_{1,n+1,w_n} ∧ x_{1,n+2,␣} ∧ … ∧ x_{1,T,␣}.
A conjunction of O(T) unit clauses. This is where w actually enters the formula — different inputs give different φ_start, and that's the only group that depends on w beyond its length.

Fourth (I'll do acceptance before the hard one), φ_accept. M accepts iff the accept state appears somewhere in the table. So just demand that some position holds q_accept:
φ_accept = ⋁_{1 ≤ i ≤ T, 0 ≤ j ≤ T+1} x_{i,j,q_accept}.
One big disjunction over O(T^2) positions. It's an OR because "accepts" is "∃ a step where we're in the accept state," and disjunction is satisfiable iff some disjunct is. (I'll arrange M to idle in the accept state once it gets there, so reaching accept early doesn't break the move-rules in later rows.)

Now the one that carries the whole thing: φ_move, which has to force every row to follow from the one above by a legal δ-step. This is where the locality I noticed earlier has to be cashed out. My first instinct is to write, for each row-to-row transition, a constraint relating the entire row t to the entire row t+1. But that's a constraint over n^k variables at once; expressing "row t+1 is a legal successor of row t" as Boolean clauses over a whole row would explode — there are exponentially many possible rows. That can't be how I encode it. Let me back up.

The point of locality is that a δ-move changes almost nothing. Positions far from the head are *unchanged* from row t to row t+1. The only action is around the state marker and the scanned symbol: the scanned tape symbol is rewritten, the state marker changes, and the marker shifts one position left or right in the configuration string. So whether the table is consistent at a given spot is decided by a tiny window. How small a window do I need? Consider a position (i, j) and the position directly below it (i+1, j). What does (i+1, j) depend on? If the state marker is nowhere among columns j−1, j, j+1 at time i, then position j is outside the action and must be copied. If the marker is in that three-column neighborhood, then a move can rewrite or move into column j. So to decide column j in the next row I need to see columns j−1, j, j+1 in the current row, and I need the corresponding three columns in the next row to check the outcome. A window of three positions in row i sitting above three positions in row i+1 — a 2×3 stencil — is exactly enough to see the marker if it can affect the center column, and to confirm copying otherwise.

So here is the local condition. Call a 2×3 window — top positions (a, b, c), bottom positions (d, e, f) — *legal* if it could actually occur in two consecutive rows of some correct computation of M, with the boundary marker treated as immovable. There are only finitely many windows (constant number of symbols, six positions), so I can list, once and for all from M's δ, the set of legal windows. Concretely: if none of a, b, c is a state marker, then the head is not close enough to affect the center, so e must equal b. If, say, a is a state q and b is the scanned symbol, then every transition choice δ(q,b) contributes the bottom patterns that write the new symbol and place the next state marker one position left or right, with the overlapping unaffected positions copied. An illegal window is one no δ-step could produce — a symbol changing with no state marker nearby, a boundary marker moving, or a state marker appearing in the bottom row in a way no transition permits.

Now the move-formula says: *every* 2×3 window centered on a nonboundary column is legal.
φ_move = ⋀_{1 ≤ i < T, 1 ≤ j ≤ T} ( ⋁_{(a,b,c,d,e,f) legal} ( x_{i,j−1,a} ∧ x_{i,j,b} ∧ x_{i,j+1,c} ∧ x_{i+1,j−1,d} ∧ x_{i+1,j,e} ∧ x_{i+1,j+1,f} ) ).
For each window position, an OR over the constant list of legal six-tuples, each disjunct an AND of six variables. That is already a Boolean formula of polynomial size. If I insist on CNF at this point, I introduce a fresh selector y_{i,j,u} for each legal six-tuple u at this window, add clauses y_{i,j,u} → each of the six required cell literals, and add one clause saying at least one selector for this window is true. Because φ_cell makes each table position hold exactly one symbol, those clauses are satisfiable exactly when the displayed window matches one of the legal six-tuples. The selector expansion is still O(T^2), and any remaining wide OR is handled by the same width-reduction step below.

I need to be sure this local condition really implies the global one, because the whole reduction stands on it. The right proof is by induction down the rows. The first row has exactly one state marker because φ_start pins it. Suppose row i is a genuine configuration string with one state marker, just before the scanned symbol. For any column j whose three-column top neighborhood does not contain that marker, the legal-window definition forces the center to be copied, so (i+1,j) = (i,j). Around the marker, the overlapping legal windows all share cells, and φ_cell forces those shared cells to have one value; inconsistent transition choices cannot be mixed because they would demand different symbols in at least one shared bottom position. Thus the windows around the marker choose one allowed δ-move, rewrite the scanned symbol as that move says, place exactly one next state marker, shift it by one, and leave the rest copied. The boundary windows keep # fixed, so the head cannot escape the represented tape. Therefore row i+1 is exactly one legal δ-successor of row i and again has one state marker. By induction, every row is the next configuration of a single accepting branch. The local checks, conjoined with the pinned start row, are the global "valid computation" condition.

Put the four groups together: φ = φ_cell ∧ φ_start ∧ φ_move ∧ φ_accept. Suppose φ is satisfiable. The assignment fills every table position with one symbol (φ_cell), starts from M's start configuration on w with fixed boundaries (φ_start), has every step following the last by a legal δ-move (φ_move and the lemma), and reaches the accept state somewhere (φ_accept) — so it is an accepting computation of M on w, hence w is accepted. Conversely, if M accepts w, take an accepting computation of length ≤ T, extend it by the accepting idle move if it stops early, write it into the table, and set x_{i,j,σ} true exactly where position (i,j) holds σ; then all four groups are satisfied, so φ is satisfiable. Therefore φ is satisfiable iff M accepts w. The map f: w ↦ φ is the reduction.

Now I have to check f is computable in polynomial time, because otherwise the reduction is useless — the entire payoff is that difficulty is *transported*, not amplified. Count: variables x_{i,j,σ} number T(T+2)|Σ_T| = O(n^{2k}), each writable in O(log n) bits. φ_cell and φ_move each contribute O(T^2) constant-size pieces (or O(T^2) selector clauses in CNF form); φ_start contributes O(T) unit clauses; φ_accept one O(T^2) disjunction. So |φ| = O(n^{2k}) up to log factors, and every piece is mechanically read off from (M, w) by scanning indices. Writing it down takes time polynomial in n. So f is a polynomial-time reduction. Since A was an *arbitrary* language in NP — I used nothing about it but its nondeterministic poly-time machine M and its polynomial bound T = O(n^k) — every language in NP reduces in polynomial time to satisfiability.

And satisfiability is itself in NP: given a formula, a satisfying assignment is a certificate no longer than the formula, and a deterministic machine checks it in linear time by evaluating the formula tree, or each clause when the input is already in CNF. Guess the assignment, verify — that's a nondeterministic poly-time acceptance. So satisfiability is in NP, and everything in NP reduces to it. It is complete for NP. Reading off the consequence: if satisfiability were in P, then via the reductions every NP problem would be in P, so P = NP; and if any NP problem is outside P, satisfiability is too. The difficulty of the whole class is now concentrated in this one problem. The scattered list wasn't a coincidence — those problems are hard, if they are, for one and the same reason, and that reason has a single name.

Let me push on the robustness, because I want to know whether this hardness is an artifact of allowing arbitrary Boolean formulas or wide clauses. φ_move naturally contains ORs of six-variable ANDs; if I distribute naively, I create clauses with many literals. Does the hardness survive if I force clauses down to at most three literals? Take the formula and convert it to a conjunction of three-literal clauses preserving satisfiability — not logical equivalence, only the yes/no answer, which is all that matters for a reduction. The trick: walk the parse structure of φ and give each connective a fresh variable z standing for the value of that subformula, with short clauses tying z to its inputs. For an AND-gate z ⟺ (u ∧ v): in clauses, (¬z ∨ u) ∧ (¬z ∨ v) ∧ (z ∨ ¬u ∨ ¬v), all of width ≤ 3. For an OR-gate z ⟺ (u ∨ v): (z ∨ ¬u) ∧ (z ∨ ¬v) ∧ (¬z ∨ u ∨ v). Add a unit clause forcing the root variable true. And any single clause that is too wide, say (ℓ_1 ∨ ℓ_2 ∨ … ∨ ℓ_m), splits with fresh variables into a chain (ℓ_1 ∨ ℓ_2 ∨ z_1) ∧ (¬z_1 ∨ ℓ_3 ∨ z_2) ∧ … ∧ (¬z_{m−3} ∨ ℓ_{m−1} ∨ ℓ_m), satisfiable iff the original clause is, with width ≤ 3 throughout. The number of new variables and clauses is linear in the size of φ, so this is still a polynomial-time map. The result is a 3-CNF formula satisfiable iff the original is, hence iff M accepts w. So even 3-satisfiability — at most three literals per clause — is NP-complete. The hardness isn't a width artifact.

I want to record that the same construction can be stated dually, the way the theorem-proving frame makes natural. Instead of "φ is satisfiable iff M accepts w," negate: ¬φ, pushed through De Morgan into disjunctive normal form, is a *tautology* iff w is *not* accepted by M. Thus a theorem-proving procedure for tautology decides the complementary question, and with the answer flipped it decides the original satisfiability question. The construction can be carried out with proposition symbols read straight off the machine: a symbol P^i_{s,t} for "tape square s holds symbol σ_i at step t," a symbol Q^i_t for "the machine is in state q_i at step t," a symbol S_{s,t} for "square s is scanned at time t." The same jobs reappear as conjuncts. Uniqueness: at each step exactly one square scanned (B = ⋀_t [(S_{1,t} ∨ … ∨ S_{T,t}) ∧ ⋀_{i<j}(¬S_{i,t} ∨ ¬S_{j,t})]), exactly one symbol per square-time (C), exactly one state per step (D). Initialization: E = Q^0_1 ∧ S_{1,1} ∧ P^{i_1}_{1,1} ∧ … ∧ P^{i_n}_{n,1} ∧ P^1_{n+1,1} ∧ … ∧ P^1_{T,1}, with w = σ_{i_1}…σ_{i_n} and σ_1 the blank. Proper updating of the P's, Q's, S's: F, G, H. The transition group G is the explicit form of "a legal next state" in Cook's coordinates: for each time t and each state-index i and symbol-index j,
G^t_{i,j} = ⋀_{s=1}^{T} ( ¬Q^i_t ∨ ¬S_{s,t} ∨ ¬P^j_{s,t} ∨ Q^k_{t+1} ),
read as "if at time t the machine is in state q_i, square s is scanned, and that square holds σ_j, then at time t+1 the machine is in the next state q_k." The signs are exactly the implication signs: all three antecedent facts are negated, and the consequent Q^k_{t+1} is positive. If δ is written as a set-valued nondeterministic relation, I can either split the instruction table into constant-size choice states before applying this displayed clause, or replace the last literal by the constant-size disjunction over the permitted q_k's; F and H carry the same choice for the written symbol and head movement. F also forces unscanned squares to keep their symbols, and H forces the scanned square index to move by ±1 as δ says. Finally I = "an accepting state is reached at some time." The conjunction A(w) = B ∧ C ∧ D ∧ E ∧ F ∧ G ∧ H ∧ I is in conjunctive normal form, is satisfiable iff M accepts w, and is built in time polynomial in |w| — the same locality, in the theorem-prover's own coordinates.

And I can see now this isn't even tied to the propositional surface. The condition "there exists a short y with a fast-checkable A(x,y)" is a *search* problem — perebor — and the universality argument applies to any fixed target rich enough to express an arbitrary accepting computation. That is why a fixed handful of problems — set cover by bounded-size subsets, a DNF of given size for a partial Boolean function, deciding if a propositional formula is refutable, graph homomorphism, subgraph isomorphism, extending a partially-filled integer matrix under an adjacency condition — can each be made *universal*: every search problem reduces to each of those representatives. Solving any one of the universal representatives fast solves all search problems of this kind fast. Satisfiability is the cleanest representative in the propositional language, but the universality is the real content, and it is the same content whether I come at it through theorem-proving formulas or through the brute-force-search tradition.

The causal chain, start to finish: a pile of problems all resist everything but exponential search; squinting, they share one shape — "is there a short witness passing a fast check" — which is exactly nondeterministic polynomial time; polynomial-time reducibility is the glue that would let me prove they're hard *together*; so I look for one NP problem that everything in NP reduces to; an arbitrary NP problem is just "M has an accepting computation of polynomial length," and a computation laid out as a time×configuration table makes validity *local*; locality lets me encode "a valid accepting computation exists" as a polynomial-size conjunction of constant-size Boolean checks — one symbol per table position, the right first row and fixed boundaries, every 2×3 window legal, some position accepting — i.e. as a satisfiability instance; the map is poly-time, so every NP problem reduces to satisfiability, which is itself in NP; therefore satisfiability is NP-complete, even in 3-CNF form, and the theorem-proving tautology form gives the complementary decision view of the same construction.
