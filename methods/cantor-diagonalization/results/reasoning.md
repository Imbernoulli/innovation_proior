Let me get the question itself clean before I do anything with it. I want to know whether the positive integers and the positive reals can be matched up one against one — each integer to exactly one real, each real to exactly one integer, nothing left over on either side. That phrase, one-to-one correspondence, is the only thing I trust here. With finite collections I'd just count both and compare the counts, but for infinite collections "count" has no meaning until I say what it means, and the moment I try to define it I'm back to pairing elements. So pairing *is* the definition. Two collections have the same power exactly when their elements can be exhausted by such a pairing; a collection has the power of the integers — is enumerable — exactly when I can write it as a single list, first element, second element, third element, with every member of the collection turning up somewhere on the list. Good. That's the whole vocabulary.

Now what do I actually believe about this question? Everything I've managed to settle points one way, and it's the wrong way for the question to be interesting. The rationals — dense, packed everywhere, between any two there are infinitely many more — those I can list: order them so that only finitely many come before any given point and then read them off. So "looks bigger, sits denser" buys you nothing; the rationals are no more numerous than the integers. And it's worse than that. The algebraic numbers — every root of every integer polynomial, so √2 and the golden ratio and the root of any quintic you like, a collection that contains the rationals and a great deal more — Dedekind hands me a proof that *those* are enumerable too. The trick is the same one I'd use myself: attach to each polynomial a height, say the degree plus the sum of the absolute values of its coefficients, so that for each height only finitely many polynomials qualify, hence only finitely many new roots; then list height one, then height two, and so on, and every algebraic number eventually appears. So density fails to obstruct listing even for this enormous set.

This is starting to look like a pattern with no exceptions, and that's exactly what worries me. If *every* infinite set I can name turns out enumerable, then "same power as the integers" is just what "infinite" means, there's only one size of infinity, and the question I'm asking is empty. For the question to have a yes-or-no answer that's worth anything, I need either a proof that the reals list too — in which case infinity really is one thing — or a single infinite set that provably cannot be listed. I don't have a candidate technique for the second. Everything in my toolbox is a *positive* technique: to show a set is countable, produce the list. I have never once had to argue that no list can exist, because I've never met a set where one didn't.

So let me actually try to imagine a list of all the reals in some interval and see if anything goes wrong. Take [0,1]. Suppose I have it, ω₁, ω₂, ω₃, …, the claim being every real in [0,1] occurs. I want to catch this list in a lie — produce a real in [0,1] that I can prove is none of the ωₙ. How do I produce a specific real with a guaranteed property? The one reliable way I know is to trap it: squeeze it between an increasing sequence from below and a decreasing sequence from above and let them close on it. So let me try to build such a trap out of the list itself.

Here's a first attempt, and I can already feel it's going to be clumsy, but let me follow it. From the list I pull out an increasing run: take ω₁, then the next term larger than it, then the next term larger than that, and so on — that gives me an increasing sequence. The terms I skipped, I can comb through *them* the same way to pull out another increasing sequence, and again, so the whole list decomposes into infinitely many increasing sequences, call them (1), (2), (3), …. Now sequence (1) is increasing, so just past its first term there's a little interval [p,q] with no term of (1) in it — pick p, q between the first and second terms of (1). Some later sequence must poke a term into [p,q], or else [p,q] would contain reals missed by the whole list and I'd already be done; let (k) be the first sequence that does, with first such term inside [p,q]. Slip a smaller closed interval [p₁,q₁] strictly below that term, still inside [p,q]; now all of sequences (1) through (k−1) avoid [p₁,q₁]. Repeat: find the first sequence after that with a term inside [p₁,q₁], tuck [p₂,q₂] below it, and on and on. I get nested closed intervals [p,q] ⊃ [p₁,q₁] ⊃ [p₂,q₂] ⊃ …, and by the construction the n-th stage has thrown out everything from the early sequences. The left endpoints increase and are bounded above, so they converge; the right endpoints decrease and are bounded below, so they converge; the common limit (or any point of the limiting interval) sits inside every stage and is therefore missed by every sequence, hence by the whole list. Contradiction. The reals in [0,1] can't be listed.

It works. But look at what it cost: I had to shred the list into infinitely many monotone subsequences first, and only then run the interval-squeezing. The decomposition is doing real labour and it's ugly, and worse, it's distracting me from what's actually generating the contradiction. Let me stare at the engine. The engine is: nested intervals, each new one *placed so as to exclude more and more terms of the list*, closing on a point that every term has been excluded from. The decomposition was only a device to *find* where to place the next interval. Do I need it?

No. I don't need the list pre-sorted at all. Give me the bare list ω₁, ω₂, ω₃, … in whatever order it comes. If the original open interval (a,b) contains at most one list-term, I am already done: choose a different real in it. So assume there are at least two. Walk down the list and pick out the **first two terms that fall strictly inside (a,b)**; call their positions k₁ < k₂ and call the smaller value a₁ and the larger b₁. Now use the new interval (a₁,b₁) and again take the first two list-terms, in list order, that fall strictly inside it; call their positions k₃ < k₄ and sort their values as a₂ < b₂. Keep going: at stage n the chosen endpoints come from positions k_{2n-1} < k_{2n}, and the new open interval sits strictly inside the previous one. That's the whole construction — no sorting, no decomposition, just "first two that land inside the current interval."

This is the move. The argument was never about the reals being special; it's about an *arbitrary enumeration* and the fact that an interval has room to dodge it. Let me nail down exactly which terms each interval has shaken off, because that's where the contradiction will live. The clean statement is a little stronger than the one I need: after stage n, (aₙ,bₙ) excludes every term through position k_{2n}; since k_{2n} is at least 2n, it certainly excludes ω₁ through ω_{2n}. Induct. At the first stage, a₁ and b₁ are the first two list-terms inside (a,b). Every term before k₂ either failed to lie in (a,b) or is one of the two endpoints, so none of ω₁,…,ω_{k₂} lies in the open interval (a₁,b₁). Now suppose (aₙ,bₙ) misses ω₁,…,ω_{k_{2n}}. The next chosen terms are the first two terms of the list that lie inside (aₙ,bₙ), at positions k_{2n+1} and k_{2n+2}; there cannot be any earlier unexcluded term inside (aₙ,bₙ), or it would have been chosen earlier in the scan. The smaller interval (a_{n+1},b_{n+1}) sits inside (aₙ,bₙ), so it keeps out everything already kept out, and it also excludes its two new endpoint terms. Therefore it misses every term through k_{2n+2}, and hence at least ω₁ through ω_{2n+2}. The induction closes. In particular, every interval (aₙ,bₙ) excludes ωₙ itself. That last consequence is the one I'll use.

Now the endgame splits, and I have to handle every branch or I've proved nothing. Either the search terminates or it doesn't. Suppose it terminates: at some stage I can't find two more list-terms inside (a_L,b_L), meaning at most one list-term lies in that whole open interval. An open interval holds infinitely many reals, so there's a point η in (a_L,b_L) different from that lone possible term — and η, lying inside the final interval, lies inside every earlier one too, so by my lemma it equals no ωₙ. Done in this branch.

Otherwise the search never stops and I have infinitely many intervals. The left endpoints a₁ < a₂ < a₃ < … increase and stay below b; an increasing sequence bounded above converges, so a_∞ = lim aₙ exists — and I should be honest that this existence is precisely the completeness of the real line, the same continuity property Dedekind builds the reals on. Likewise b₁ > b₂ > … decreases and stays above a, so b_∞ = lim bₙ exists. Since aₙ < bₙ always, a_∞ ≤ b_∞, and there are two sub-branches. If a_∞ = b_∞, then this single common value η = a_∞ lies inside *every* (aₙ,bₙ): for each fixed n the later left endpoints are still below bₙ and above aₙ, so their limit is strictly above aₙ, and the symmetric argument puts the limit strictly below bₙ. Meanwhile ωₙ does *not* lie in (aₙ,bₙ) by the lemma. So for every n, η is in (aₙ,bₙ) and ωₙ is not, hence η ≠ ωₙ. The point η is a real in [a,b] on no rung of the list. If instead a_∞ < b_∞, then every η in [a_∞,b_∞] satisfies aₙ < η < bₙ for every n, so it lies inside every (aₙ,bₙ), while ωₙ does not lie in its own (aₙ,bₙ). Pick any such η. Every branch yields a real in [a,b] missed by ω₁,ω₂,…. 

So: given *any* sequence of reals and any interval, I can produce a real in that interval off the sequence. And now the uncountability is a one-line corollary. Suppose the reals in [a,b] could be listed. That list is a particular sequence of reals; feed it into what I just proved; out comes a real in [a,b] not on the list — contradicting that the list was supposed to contain every real in [a,b]. Therefore the reals in [a,b] cannot be enumerated. There exists an infinite set strictly un-listable. Infinity is not one thing.

And I get Liouville's transcendentals for free, which is delicious. The real algebraic numbers are enumerable (the height trick). The reals in [a,b] are not. If [a,b] held no transcendentals, every real in it would be algebraic, hence a sub-list of the enumerated algebraics, hence [a,b] would be enumerable — false. So [a,b] is full of transcendentals, with no approximation estimates anywhere in sight, just a clash of sizes. This is still not a Liouville-style formula, but it traps a non-algebraic point by counting rather than by approximation estimates.

Now let me sit with what I've got, because something about the proof bothers me even though it's correct. Three things, really. First, it leans on the limit a_∞ *existing* — that's completeness of ℝ, a deep analytic fact about the continuum. But the thing I proved is a counting fact: there's no bijection with the integers. Why should a counting fact need the full strength of the real line's continuity? That's a mismatch; the analysis feels smuggled in. Second, the proof has cases — terminating search, equal limits, unequal limits — and a separate existence-of-limit step. It's not one clean stroke; it's a small campaign. Third, and this is the one that really nags: the whole argument is *about the reals*. Order, intervals, limits — it's saturated with the structure of the number line. So it tells me exactly nothing about whether some *other* kind of set, not made of numbers at all, might also resist listing. I've answered one instance and built a machine that only runs on that instance.

Let me ask what the proof would look like if I stripped out everything that isn't pure counting. Where does the realness of the reals actually enter? Through the representation. A real in [0,1] can be controlled by its digits; the interval-squeezing is really just a roundabout way of controlling those digits one position at a time. So suppose I throw away the geometry and the analysis and keep only the strings. Forget reals; consider the set of all infinite sequences whose every entry is one of two fixed symbols — let me write them m and w to keep myself honest that they're just marks, not numbers. An element of this set M is E = (x₁, x₂, x₃, …) with each xᵥ either m or w. There are no intervals here, no order, no limits, no completeness — just sequences of two characters. If I can show *this* set can't be listed, using nothing but the symbols, then I've isolated the counting from the continuum entirely. And I can get [0,1] back afterward by encoding m and w as decimal digits 1 and 2; using only those digits avoids the usual double-expansion nuisance.

So, list of elements of M, and I want one that's on no rung. Write the supposed list down as an array:

  E₁ = (a₁,₁, a₁,₂, a₁,₃, …)
  E₂ = (a₂,₁, a₂,₂, a₂,₃, …)
  E₃ = (a₃,₁, a₃,₂, a₃,₃, …)
  …

with each a_{μ,ν} equal to m or w. I want to build E₀ = (b₁, b₂, b₃, …) in M that disagrees with every Eμ. To disagree with Eμ I only need them to differ in *one* coordinate. Which coordinate can I guarantee to control against Eμ, *simultaneously for every μ at once*? Look at the array as a table. Run my finger down the main diagonal: a₁,₁, a₂,₂, a₃,₃, …, the entry a_{μ,μ} being Eμ's own μ-th symbol. If I define b_ν to be the *other* symbol from a_{ν,ν} — if a_{ν,ν} = m then b_ν = w, and if a_{ν,ν} = w then b_ν = m — then b_ν ≠ a_{ν,ν} for every ν by construction. That single rule fixes all of E₀.

Now check E₀ against the list. Could E₀ = Eμ for some μ? If it did, then E₀ and Eμ would agree in *every* coordinate, in particular in the μ-th: b_μ = a_{μ,μ}. But I *built* b_μ to be the opposite of a_{μ,μ}. So b_μ ≠ a_{μ,μ}, and the equality E₀ = Eμ fails. This holds for every positive integer μ — whatever rung you point at, E₀ differs from it precisely at that rung's diagonal position. So E₀ is an element of M on no rung of the list. The list was arbitrary; therefore M cannot be put in the form E₁, E₂, E₃, …. M is uncountable.

One construction. No cases, no termination branches, no limits, no completeness, no order — I never even used that m and w are numbers. The disagreement is forced at coordinate μ for Eμ, all μ together, by reading down the diagonal and flipping. The thing that was a small campaign on the number line is now a single line on bare symbols. And it lands the reals cleanly: send E = (x₁,x₂,…) to the decimal 0.d₁d₂d₃… with d_ν = 1 when x_ν = m and d_ν = 2 when x_ν = w. If two sequences first differ at position k, the digit gap there is 10^{-k}, while the largest possible tail correction is 10^{-(k+1)} + 10^{-(k+2)} + … = (1/9)10^{-k}, too small to erase it. So distinct sequences give distinct reals in [0,1]. If [0,1] were enumerable, M would be enumerable through this injection, but M is not.

Here's what makes me sit up, though — it's not the simplicity, it's that *nothing in this argument knew it was about sequences of symbols*. Let me look again at what the diagonal flip actually used. It used: a list indexed by the positive integers; for each index μ, an element Eμ that, at "position μ," makes a binary choice I can read and reverse. That's all. The reals never mattered; the symbols barely mattered. So suppose I'm handed not the integers but *any* set L, and I want to outrun any attempt to list — no, to *index by L* — some other collection. What collection has, for each element of L, a readable-and-reversible binary choice at "position L"?

Functions on L taking two values. Let M now be the set of all functions f from L into {0,1} — equivalently, since a function into {0,1} is just an indicator, the set of all *subsets* of L. Each such f makes, at every point x of L, a binary choice f(x). Now imagine M *could* be indexed by L: that there's a way to write each element of M as f(x) = φ(x, z) where z ranges over L, every z giving an element of M and every element of M coming from exactly one z. This is the exact analog of "M is listed by the integers," with L in the role of the index set and z in the role of the list-position. I'll do to it precisely what I did on the diagonal.

The diagonal there was a_{μ,μ}: the listed element at position μ, evaluated at position μ. Here the listed element at position z is the function φ(·, z), and "evaluated at position z" means φ(z, z). So the diagonal is the function x ↦ φ(x, x), and the flip is: define g on L by letting g(x) be the value in {0,1} *different* from φ(x, x), at every x. Then g is a function from L into {0,1}, so g is an element of M. But can g equal φ(·, z₀) for some index z₀? If it did, the two would agree at every point of L, in particular at z₀: g(z₀) = φ(z₀, z₀). Yet I defined g(z₀) to be the opposite of φ(z₀, z₀). Contradiction. So g is in M but comes from no index z₀ — the supposed indexing of M by L misses g. There is no such correspondence; M cannot be indexed by L.

And M is at least as big as L, trivially — for each point x₀ of L take the function that is 1 at x₀ and 0 everywhere else; distinct points give distinct such functions, so L embeds into M. So M is not smaller than L, and not the same size as L, hence strictly bigger. For *every* set L, the set of all its subsets is strictly larger in power. Whatever L you hand me — the integers, the reals, the subsets-of-the-reals, anything — I produce a strictly larger set by the same single move. There is no largest size. The powers of well-defined sets have no maximum.

That is the thing worth having, and it's why the symbol version was worth finding even though the interval version already answered the original question. The nested-intervals proof was a key that fit exactly one lock. The diagonal flip is the *shape* of the key: list (or index) the objects, find the per-index binary choice, read the diagonal, reverse it, and the reversed object is forced off the list at every index at once. It runs on reals, on two-symbol strings, on functions, on subsets — on anything with that structure — because it never used anything else. The contradiction is always the same sentence: at position μ (or z), the constructed object was *built* to differ from the μ-th (or z-th) listed object, so it can't *be* the μ-th listed object, for any μ.

Let me write down the landing, both forms, so I have exactly what I've proved.

The diagonal theorem. Let M be the set of all infinite sequences over two symbols {m, w}. For any sequence E₁, E₂, E₃, … of elements of M, with Eμ = (a_{μ,1}, a_{μ,2}, …), define E₀ = (b₁, b₂, …) by

    b_ν = w if a_{ν,ν} = m,   and   b_ν = m if a_{ν,ν} = w.

Then E₀ ∈ M and E₀ ≠ Eμ for every μ, because E₀ = Eμ would force b_μ = a_{μ,μ}, contradicting b_μ's definition. Hence M is uncountable; through the decimal injection above, [0,1] is uncountable, and every non-degenerate interval follows by rescaling.

The general theorem. For any set L, let M be the set of all subsets of L (equivalently all functions L → {0,1}). Then |L| < |M|. Proof: the singleton map x₀ ↦ {x₀} injects L into M, so |L| ≤ |M|. And no surjection f from L onto M exists: given any f assigning to each x ∈ L a subset f(x) ⊆ L, form the "flip" subset

    T = { x ∈ L : x ∉ f(x) }.

For each x, either x ∈ T or not. If x ∈ T then by T's definition x ∉ f(x), so T and f(x) disagree at x, hence T ≠ f(x). If x ∉ T then x ∈ f(x), so again T and f(x) disagree at x, hence T ≠ f(x). So T equals no f(x): f is not onto, T is the witness it misses. Since some subset is always missed, no f is a surjection, so no bijection; with the embedding above, |L| < |M|. (T is exactly the diagonal flip in subset clothing: x is the index, "x ∈ f(x)?" is the diagonal entry a_{x,x}, and T reverses it — put x into T precisely when the index says to leave it out.) Therefore for every set there is a strictly larger one: no maximum power.

The whole chain in one breath: the only honest test of size is one-to-one correspondence; every set I could list lulled me into thinking all infinities are one; trying to list the reals, I trapped a missed point with nested intervals and broke that — but the proof leaned on the continuum's completeness, came in cases, and couldn't move off the real line; stripping it to bare two-symbol sequences, the trap became a single diagonal flip with no analysis, no cases, no order; and because the flip used nothing but "a list indexed by something, with a reversible binary choice at each index," the same one line, applied to the subsets of an arbitrary set, shows every set is dwarfed by its set of subsets — the sizes of infinity climb without end.
