I start with the only inequality I can trust. If I take any feasible flow and any cut separating the source from the sink, the flow value cannot exceed the cut capacity. I do not want to leave that as a picture in my head, because the exact slack in this inequality is going to matter. I sum the net outflow over all vertices on the source side `S`. Conservation erases every internal vertex, and every arc with both ends in `S` appears once positively and once negatively. What survives is `value(f) = sum_{S->T} f - sum_{T->S} f`. Then I drop the backward term, because it is nonnegative, and I replace forward flow by forward capacity. So `value(f) <= cap(S,T)`.

That calculation gives me more than an upper bound. It tells me exactly what equality would have to look like. The first inequality is tight only if every arc crossing backward from `T` into `S` carries zero flow. The second is tight only if every arc crossing forward from `S` into `T` is saturated. So if I ever find a cut whose forward arcs are all full and whose backward arcs are all idle, the flow value and cut capacity are equal. Then neither object can be improved: no larger flow can pass through that cut, and no smaller cut can be below that flow. The problem becomes very concrete. I am not trying to prove equality by magic; I am trying to manufacture a situation where a cut is full in one direction and empty in the other.

The obvious way to build a large flow is to find a source-to-sink route with unused capacity, push as much as the tightest arc allows, and repeat. But this is too rigid. I can make a bad early choice: send a unit through a middle diagonal that blocks the two clean parallel routes. The network still has a better total routing, but the greedy procedure sees only unused forward capacity. It has no operation that says, "take back the unit on this arc and send it another way." So the first wall is not capacity; it is irreversibility. If the method cannot undo its own routing choices, stopping does not mean optimality.

I need to encode undoing as a normal move. For a current flow `f`, a step from `u` to `v` should be possible in two ways. If there is a real arc `u->v` that is not full, I can push more through it, up to `c(u,v)-f(u,v)`. If there is a real arc `v->u` that currently carries flow, I can cancel some of that flow, up to `f(v,u)`. These two changes have the same net effect on the vertex balances: they move flow from `u` toward `v`. So I define the available room from `u` to `v` as `c_f(u,v) = c(u,v)-f(u,v)+f(v,u)`, treating missing arcs as zero. The graph of positive `c_f` values is the graph of changes I can still make.

Now a path in this change graph means something stronger than an ordinary route. Along each step I can either add forward flow, cancel reverse flow, or do both if both real arcs exist. If there is a source-to-sink path, I choose the smallest residual room on the path, call it `F`, and apply that same `F` to every step. Feasibility survives because `F` is no larger than the room at any step. Conservation survives because every interior vertex receives `F` units of net change from the previous step and sends `F` units of net change to the next. At the source, net outflow rises by `F`; at the sink, net inflow rises by `F`. So a residual source-to-sink path is not just a hint that improvement may be possible. It is a direct certificate that the current flow is not maximum.

That handles the improving case, but the important part is what happens when no such path exists. I should not treat failure to find a path as a computational dead end. I should ask what structure the failure forces. Let `S` be the vertices still reachable from the source in the residual graph, and let `T` be the rest. The sink is not reachable, so this is a real cut. Because `S` contains every residually reachable vertex, no residual edge leaves `S`.

Now I inspect an original arc `u->v` with `u in S` and `v in T`. If it had any unused forward capacity, then the residual edge `u->v` would leave `S`, making `v` reachable. That is impossible. So every forward arc across the cut is saturated. Next I inspect an original arc `v->u` going backward across the same cut, with `v in T` and `u in S`. If it carried positive flow, then I could cancel it by taking the residual step `u->v`, again making `v` reachable. That is impossible too. So every backward arc across the cut carries zero flow.

This is the moment the two halves meet. The exact equality conditions I extracted from the weak-duality calculation are forced by the absence of a residual path. No residual path gives me a reachable set. The reachable set gives me a cut. The lack of outgoing residual edges makes that cut saturated forward and idle backward. Therefore the flow value equals the cut capacity. The same failed search proves both that the current flow cannot be augmented and that the displayed cut cannot be cheaper.

So the method is constructive because it alternates between only two possible states. If the residual graph has a source-to-sink path, I augment and strictly increase the flow. If it has no such path, the source-reachable set is the minimum cut and the current flow is maximum. There is no third possibility where I am stuck but uncertified. The certificate is not an extra argument added after the algorithm; it is exactly the shape left behind when augmentation becomes impossible.

I still have to be careful about termination. With integer capacities, I start from the zero flow, and every residual capacity is an integer. Each augmentation uses the minimum of positive residual capacities, so it increases the value by at least one integer unit. The value is bounded above by any source cut, for example the capacity of all arcs leaving the source. Therefore the process stops, and every flow value on every arc remains integral. Rational capacities reduce to this by scaling.

But arbitrary choices of augmenting paths can still be computationally bad. A small middle edge can be used back and forth, causing one-unit improvements even though a large flow is available. With irrational capacities, careless choices can even fail to terminate. So I choose shortest augmenting paths in the residual graph, found by breadth-first search. Then residual distances from the source never decrease: any newly created residual edge is the reverse of a just-used shortest-path edge, and using it to make a vertex closer would contradict the old shortest-path layering. Once an edge is a bottleneck and disappears, it can become a bottleneck again only after a reverse use has pushed its endpoint distance up by at least two. Distances are bounded by the number of vertices, so each edge is critical only `O(V)` times. That gives `O(VE)` augmentations and `O(VE^2)` total time with breadth-first searches.

The final insight is therefore not merely that the largest flow equals the cheapest cut. It is that the residual graph is a live record of every local way the current flow can still change, including undoing previous choices. While a residual path reaches the sink, the flow is improvable. When no residual path reaches the sink, the vertices still reachable from the source form exactly the cut whose forward side has no slack and whose reverse side carries no flow. Augmenting paths and cut certificates meet at the stopping condition, and that meeting is optimality.
