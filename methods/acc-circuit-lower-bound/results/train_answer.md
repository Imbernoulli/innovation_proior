We want to prove that some explicit, large complexity class provably has no nonuniform ACC circuits of polynomial size, where ACC is the class of constant-depth, polynomial-size circuits over unbounded fan-in AND, OR, NOT together with MOD$_m$ gates for an arbitrary constant $m > 1$ (a MOD$_m$ gate fires iff $m$ divides the sum of its inputs), so that ACC $= \cup_m$ AC0$[m]$. The bottom-up program had genuine early wins — parity is hard for AC0 (Furst–Saxe–Sipser, Ajtai, then exponential bounds via Yao and Håstad), and AC0$[p]$ for prime $p$ fell to low-degree polynomial approximation over $\mathbb{F}_p$ (Razborov, Smolensky) — and then it stopped dead. For composite $m$ there is no field to support the polynomial method, and for over twenty years ACC has been a standing wall: it is not even known whether EXP$^{\mathrm{NP}}$ can be computed by depth-three circuits made of nothing but MOD$_6$ gates. So the natural moves are exhausted in two directions. Finding a *simple* function hard for ACC is exactly the bottom-up route that died, and any combinatorial property it would yield runs straight into the natural-proofs barrier (Razborov–Rudich): a constructive, large property useful against a class that can compute pseudorandom functions is self-defeating, and the AC0$[p]$ proofs are natural. Karp–Lipton–Meyer gives the right *shape* — under $\mathrm{P}=\mathrm{NP}$ (or a half-exponential-time Circuit-SAT algorithm) one gets EXP $\not\subset$ P/poly by guessing a circuit for a computation history and verifying it locally — but the algorithmic hypothesis is a fantasy nobody can supply, so it is vacuous as a route to an unconditional bound. And pure diagonalization, the one unconditional tool we trust, relativizes: there are oracles $A$ with NEXP$^A \subseteq$ ACC$^A$, so any argument surviving the addition of an oracle cannot separate NEXP from ACC; algebrization closes the obvious algebraic loophole too. The need is therefore for an argument that reaches *inside* circuits (so it does not relativize), never harvests a property of the hard function (so it is not natural), and rests its algorithmic demand on something achievable rather than on $\mathrm{P}=\mathrm{NP}$.

I propose the algorithmic method: invert the usual relationship between algorithms and lower bounds, so that a satisfiability algorithm for a circuit class $C$ that beats exhaustive search by even a tiny factor *implies* a circuit lower bound against $C$. Concretely, if $C$-Circuit-SAT on $n$-input circuits can be solved a $1/n^{\omega(1)}$ factor faster than the trivial $2^n$, then NEXP $\not\subset C$. The argument has two halves. The first half — call it the spinning half — takes any faster-than-$2^n$ SAT algorithm and spins it, through the nondeterministic time hierarchy, into the lower bound; it uses only that $C \supseteq$ AC0 and that $C$ is closed under composition, so it is generic across robust classes. The second half is the class-specific debt: an actual faster-than-$2^n$ satisfiability algorithm for ACC, delivered through the SYM$^+$ normal form.

The target must be chosen so that hardness for it drags the whole class down, hence an NEXP-complete problem, and the cleanest is SUCCINCT 3SAT: given a circuit $C$, is the 3-CNF whose $2^n$-bit truth table is $T(C)$ satisfiable? This is NEXP-complete (Papadimitriou–Yannakakis), and it is very NEXP-complete: by the efficient Cook–Levin theorem, the sharpened reductions (Cook 1988; Robson 1991; Fortnow–Lipton–van Melkebeek–Viglas 2005) give, for every $L \in$ NTIME$[2^n]$ and input $x$ of length $n$, a poly$(n)$-time-computable circuit $C_x$ with only $n + O(\log n)$ inputs and poly$(n)$ size such that $x \in L$ iff $F_{C_x}$ is satisfiable. The tight input count is the whole ballgame: a naive tableau would index a $2^n \times 2^n$ grid and need $2n$ or $3n$ inputs, but the quasi-linear $O(t\log^4 t)$ formula needs index $i$ of only $\log t + O(\log\log t)$ bits, so $n + O(\log n)$ inputs. This buys a strong time lower bound. If SUCCINCT 3SAT had a nondeterministic algorithm in $2^{n - \omega(\log n)}$ time on $n$-input poly-size circuits, every $L \in$ NTIME$[2^n]$ would land in NTIME$[2^{(n + O(\log n)) - \omega(\log n)}\,\mathrm{poly}(n)] =$ NTIME$[o(2^n)]$, which the nondeterministic time hierarchy theorem (Seiferas–Fischer–Meyer 1978; Žák 1983) forbids. With $3n + O(\log n)$ inputs we would only rule out $2^{n/3}$-beating algorithms — a vastly harder target — so the tight count is essential. The time hierarchy is the contradiction engine.

Now spin a circuit assumption into a fast algorithm. The naive move — guess an ACC circuit deciding SUCCINCT 3SAT and check it — fails because verifying correctness over all $2^n$ inputs is itself the SUCCINCT 3SAT problem, circular and too slow; and program-checking / MIP routes fail because they are randomized (unsound as nondeterministic algorithms, and NEXP $=$ MIP collapses to NP if the verifier is made nondeterministic) and because, being black-box, they throw away the one advantage we have: a *small ACC circuit can be cut open*. The fix is to stop guessing a *decider* and instead guess a *witness*. By easy witnesses (Impagliazzo–Kabanets–Wigderson 2002), if NEXP $\subseteq C \subseteq$ P/poly then every satisfiable succinct instance $C_x$ has a small witness circuit $W_x$ whose truth table is a satisfying assignment to $F_{C_x}$; and since $\mathrm{P} \subseteq C$ makes Circuit-Value $\in C$, $W_x$ can be taken in $C$. A satisfying assignment is *locally* checkable — every clause must have a satisfied literal — so we build a single circuit $D(i)$ that, on clause index $i$, runs one copy of the input circuit (which prints clause $i$'s three (index, sign) pairs) and three copies of $W_x$ (which read off the three assigned values), and outputs $0$ iff clause $i$ is satisfied. Then the witness is correct iff $D$ is *un*satisfiable: one SAT call replaces $2^n$ separate checks. If that SAT call runs in $2^{n - \omega(\log n)}$ time, the whole nondeterministic procedure does too, contradicting the time lower bound. So a SAT algorithm shaving a $1/n^{\omega(1)}$ factor off brute force already yields NEXP $\not\subset$ P/poly — the assumption has dropped from $\mathrm{P}=\mathrm{NP}$ to a minor improvement over exhaustive search.

The remaining obstruction is that under NEXP $\subseteq$ ACC the witness $W'_x$ is ACC, but the input circuit $x$ (here $C_x$) is *unrestricted*, so the circuit $D$ containing a copy of $x$ is not ACC and the ACC-SAT algorithm cannot run on it. What makes it work is to break the regress with redundancy, the error-correcting / arithmetization instinct: do not verify one output bit of $x$ (which forces recomputing $x$); force a guessed ACC circuit to commit to *all intermediate gate values* of $x$, which are locally checkable and, under $\mathrm{P} \subseteq$ ACC, themselves ACC-computable. Represent $x$ by its gate tuples $\langle j, j_1, j_2, g\rangle$ (type $g \in \{$AND, OR, NOT, INPUT$\}$, feeders $j_1, j_2$, fan-in $\le 2$). The gate-*connection* circuit $D(x, j)$, printing $(j_1, j_2, g)$ from $j$, computes a poly-time function so is ACC; crucially it is verified *directly* by evaluating on all poly-many gate indices — no SAT call, this is the regress-stopping base case. The gate-*value* circuit $E(x, i, j)$, outputting the value on gate $j$ when $x$ runs on input $i$, is also ACC; we verify it with the already-correct $D$ by a local consistency check $\mathrm{VALUE}(i,j)$: read $(j_1, j_2, g) \leftarrow D(x,j)$ and $v_1, v_2, v \leftarrow E(x,i,j_1), E(x,i,j_2), E(x,i,j)$, then output $0$ iff $E$'s claim at gate $j$ is consistent with type $g$ (for INPUT, the $j$-th bit of $i$ equals $v$; for NOT, $v = \neg v_1$; for AND, $v = v_1 \wedge v_2$; for OR, $v = v_1 \vee v_2$). Each is a constant overhead on $D$ and $E$, so VALUE is ACC, and it is unsatisfiable iff $E$ reports the true value of every gate of $x$ on every input — one ACC-SAT call certifies $E$. Then $x' = E(\cdot, j^*)$, with $j^*$ the output gate, is a genuine ACC equivalent of $x$, read off for free. The only properties of ACC used in this entire half are (a) ACC $\supseteq$ AC0, so the constant-size local checks and the ORs/ANDs of copies stay in the class, and (b) ACC is closed under composition, which keeps VALUE and $D'$ ACC. That is all — so this half works verbatim for any class $C$ with $C \supseteq$ AC0 closed under composition (TC0, NC1, formulas, P/poly), and the class-specific debt is quarantined entirely in the SAT algorithm.

That debt is paid by exploiting ACC structure. Every depth-$d$, size-$s$ ACC circuit with MOD$_m$ gates has an equivalent SYM$^+$ circuit — a symmetric function of ANDs of non-negated variables — of size $s^{O(\log^{f(d,m)} s)}$ with ANDs of $O(\log^{f(d,m)} s)$ fan-in, $f(d,m) \le m^{O(d)}$ (Yao 1990; Beigel–Tarui 1994; Allender–Gore 1991; Green et al. 1995). The construction makes the circuit a tree, randomizes AND/OR into MOD$_p$ of polylog-fan-in ANDs (Valiant–Vazirani), splits composite moduli by the Chinese Remainder Theorem and reduces MOD$_{p^e}$ to constant-fan-in ANDs of MOD$_p$, derandomizes into a MAJORITY top gate, then pushes each MOD$_p$ layer into the top symmetric gate with the modulus-amplifying polynomial $P_k(x) = 1 - (1-x)^k \left( \sum_{i=0}^{k-1} \binom{k+i-1}{i} x^i \right)$, which is $\equiv 0 \bmod p^k$ when $x \equiv 0 \bmod p$ and $\equiv 1 \bmod p^k$ when $x \equiv 1 \bmod p$; with $Q_k(x) = 1 - P_k(x^{p-1})$ and Fermat's little theorem, $Q_k(\sum y_i) \equiv 1 \bmod p^k$ iff $p \mid \sum y_i$, so a MOD$_p$ layer becomes a single sum mod $p^k$ of ANDs. The final symmetric function is $F(v) = \mathrm{MAJORITY}((\cdots((v \bmod p_1^{k_1}) \bmod p_2^{k_2}) \cdots \bmod p_{d'}^{k_{d'}}))$, still symmetric and evaluable in time comparable to the size. A symmetric top gate is exactly what we want because it depends only on the *count* of true bottom ANDs, which a counting primitive can produce in bulk. The Evaluation Lemma does so: index each AND$_j$ by its variable subset $G_j \subseteq [n]$, set $f(S) = \#\{j : G_j = S\}$, and note that the number of ANDs satisfied by the assignment $T$ is $g(T) = \sum_{S \subseteq T} f(S)$ — the zeta transform of $f$ — computed by Yates's 1937 dynamic program in $O(2^n \cdot \mathrm{poly}(n))$; apply $F$ to each count. (Two other routes give the same: Coppersmith's rectangular matrix multiplication splitting the inputs into halves to count satisfied ANDs per pair, and coefficient-to-point multilinear FFT.) All three cost $2^n$, but to get *below* $2^n$ we trade inputs for size with a $k$-blowup: OR over all $2^k$ settings of a $k$-subset, substituting each into $C$, yielding $C'$ with $n - k$ free inputs, size $\le 2^k \cdot s$, still ACC, equisatisfiable. Choosing $k = n^{1/(2f(d,m))}$ on a circuit of size $s = 2^{n^\varepsilon}$ keeps the monomial count $K \le 2^{n^{2/3}}$ and the running time $O(2^{n-k}\,\mathrm{poly}(n) + K) = 2^{n - \Omega(n^{1/(2f(d,m))})}$ — a $2^{\Omega(n^\delta)}$ savings, far more than the $1/n^{\omega(1)}$ the connection needs.

Combining the two halves: assume for contradiction NEXP $\subseteq$ ACC, so $\mathrm{P} \subseteq$ ACC; for any $L \in$ NTIME$[2^n]$ and input $x$, run the spinning procedure (guess and verify an ACC equivalent of $C_x$ via the $D$, $E$, VALUE construction, guess the ACC witness, decide the clause circuit $D'$) using the faster-than-$2^n$ ACC-SAT algorithm, which puts $L$ in NTIME$[o(2^n)]$ and contradicts the nondeterministic time hierarchy. Hence NEXP $\not\subset$ ACC. The barriers are threaded: no combinatorial property of the hard function is ever extracted (so nothing P-natural is produced — the argument leans on completeness and diagonalization), and the only non-black-box step is the ACC-SAT algorithm, which exploits the SYM$^+$ structure and therefore correctly does not relativize, consistent with the oracles $A$ for which NEXP$^A \subseteq$ ACC$^A$.

The final result, with the two algorithms exactly as constructed:

**Theorem 1 (lower bound for NEXP).** NTIME$[2^n]$ — hence NEXP — does not have nonuniform ACC circuits of polynomial size. The bound extends to quasipolynomial and even sub-third-exponential size, and down to grotesque classes just above NTIME$[2^n]$.

**Theorem 2 (exponential size–depth tradeoff for E$^{\mathrm{NP}}$).** For every $d$ and $m$ there is a $\delta > 0$ and a language in E$^{\mathrm{NP}}$ (= TIME$[2^{O(n)}]$ with an NP oracle) that has no nonuniform depth-$d$ ACC circuits of size $2^{n^\delta}$ with MOD$_m$ gates.

**Theorem 3 (metatheorem).** Let $C$ be any circuit class containing AC0 and closed under composition (e.g. ACC, TC0, NC1, formulas, P/poly). There is a $k > 0$ such that, if $C$-Circuit-SAT on $n$-variable, $n^c$-size circuits can be solved in $O(2^n / n^k)$ time for every $c$, then NTIME$[2^n]$ has no nonuniform polynomial-size $C$-circuits.

**Theorem 4 (ACC-SAT).** For every $d, m$ there is $\varepsilon \in (0,1)$ such that satisfiability of depth-$d$ ACC circuits with MOD$_m$ gates, $n$ inputs, and $2^{n^\varepsilon}$ size is decidable in $2^{n - \Omega(n^\delta)}$ time for some $\delta > \varepsilon$ depending only on $d$ and $m$.

The spinning algorithm (Part A — uses only AC0 $\subseteq C$ and closure under composition):

```
SATALG5(x):
    C_x ← efficient Cook–Levin circuit for x        # n + O(log n) inputs, poly(n) size, unrestricted
    guess ACC circuits D(x,·), E(x,·,·)             # gate-connection and gate-value circuits of C_x
                                                    #   (exist since they compute poly-time functions, P ⊆ ACC)
    verify D directly: evaluate D(x,·) on all |C_x| gate indices, check it matches C_x  # poly time
    build VALUE(i,j) from D, E:                      # local gate-consistency check
        (j1,j2,g) ← D(x,j);  v1,v2,v ← E(x,i,j1),E(x,i,j2),E(x,i,j)
        VALUE = 0  iff  E's claim at gate j on input i is consistent with type g
    if VALUE is satisfiable:  reject                 # one ACC-SAT call: certifies E correct
    x' ← E(·, j*)   where j* = output gate of C_x    # ACC circuit equivalent to C_x
    guess ACC witness W'_x                           # encodes a satisfying assignment (Ingredient 3)
    build D'(i) from x' and 3 copies of W'_x:        # iterate clauses by feeding index i to x'
        x'(i) prints clause i's 3 (index, sign) pairs; W'_x reads off the 3 assigned values
        D'(i) = 0  iff  clause i is satisfied by the assignment encoded by W'_x
    accept iff D' is unsatisfiable                   # one ACC-SAT call: certifies W'_x satisfies F
```

The ACC satisfiability algorithm (Part B — uses ACC structure):

```
ACCSAT(C):                                  # C: depth-d ACC, MOD_m, n inputs, size s = 2^{n^ε}
    k ← n^{1/(2 f(d,m))}
    C' ← OR over all 2^k settings of a k-subset of inputs, with that subset substituted into C
            # C': n − k free inputs, size ≤ 2^k · s, still ACC (one more depth level), sat iff C sat
    (F, {AND_j}) ← SYM⁺ normal form of C'   # Ingredient 4; K = 2^{O(k f(d,m) + log^{f(d,m)} s)} ANDs
    evaluate on all 2^{n−k} assignments via the zeta-transform DP   # Ingredient 5
    output "satisfiable" iff F(count) = 1 for some assignment
```
