## The pain point / research question
Photon IMRT (Bortfeld inverse planning) modulates fluence of exponentially-attenuating beams. Protons have a Bragg peak: near-zero exit dose, dose maximum at a depth set by energy, sharp distal falloff. A single pristine peak is too narrow to cover a tumor → SOBP stacks pristine peaks (weighted superposition at graded energies) to make a uniform high-dose plateau. Passive SOBP (single field uniform dose per field) wastes the proton advantage: every field must itself be uniform across the target. Question: can we instead let EACH field be deliberately INHOMOGENEOUS, modulating the weight of every individual Bragg spot in 3D, and only require the SUM over fields to be uniform on the target while sparing OARs? That's IMPT. It needs spot-scanning (magnetically steer a pencil beam laterally, change energy for depth) and inverse optimization over thousands of spot weights.

## Core derivation chain (for reasoning.md)
1. Pencil beam j at energy/position deposits a fixed spatial dose pattern d_j(x,y,z) (lateral Gaussian × depth Bragg curve), scaled linearly by its weight (number of protons / monitor units) w_j ≥ 0.
2. Linear superposition → dose to voxel i: D_i = Σ_j d_{ij} w_j, i.e. D = P·w where P (= "dij", dose-influence/dose-deposition matrix) is precomputed, P_{ij} = dose to voxel i per unit weight of spot j. This is the proton analogue of IMRT's beamlet dose matrix.
3. Inverse planning: choose w to make D uniform = D_pres on target, low on OARs. Underdetermined + competing → cast as optimization.
4. Objective: voxel-based penalized least squares. Target: squared deviation (D_i−D_pres)². OAR: squared OVERdosing only, max(D_i−D_max,0)² (one-sided; under-dosing an OAR is good). Target min: max(D_min−D_i,0)². Weighted sum over structures with penalties p_s. Heaviside one-sided operator for the max/min objectives.
   f(w) = Σ_s p_s/|V_s| Σ_{i∈V_s} [penalty term]_i².
5. Constraint w ≥ 0 (can't deliver negative protons). Convex quadratic (in d) objective, nonnegativity bound → bound-constrained nonlinear program. Solve by gradient-based optimizer (IPOPT interior point / L-BFGS / projected gradient).
6. Gradient via chain rule: ∂f/∂w = Pᵀ (∂f/∂D). ∂f/∂D is the per-voxel "delta" (2·penalty·deviation, one-sided for over/under). matRad: dose = dij.physicalDose*w; wGrad = (doseGrad' * dij.physicalDose)'. Backprojection through the SAME matrix.
7. WALL: protons' sharp distal falloff. Optimizer happily parks a Bragg peak's distal edge right behind an OAR (e.g. spinal cord) to exploit the steep gradient. Nominal plan looks great. But the actual proton range is uncertain: ~3.5% (CT HU→stopping-power conversion errors, anatomy/weight change, setup shift). A few-mm range error → the high-dose region slides INTO the OAR, or a cold spot opens in the target. Steep longitudinal gradients → fragile to range error; steep lateral gradients → fragile to setup error. Multi-field IMPT is WORSE than SFUD here because individual fields are non-uniform: their hot/cold patches are designed to cancel in the nominal scenario, and any misalignment breaks the cancellation.
8. FIX = robust optimization. Don't optimize the nominal dose; optimize over a set of error scenarios k=1..K (range over/undershoot, setup shifts ±). For each scenario a shifted/rescaled dose matrix P^(k), dose D^(k)=P^(k)w.
   - Probabilistic / stochastic (Unkelbach): minimize expected objective E_k[f(w,k)] = Σ_k π_k f(w; D^(k)). matRad STOCH/PROB; PROB uses expectation of dose + an integral-variance (Ω) penalty wᵀΩw.
   - Worst-case (Pflugfelder): minimize the worst scenario, min_w max_k f(w; D^(k)). Composite worst case (COWC): f = max_k Σ_s f_s(D^(k)); objective-wise worst case; voxel-wise worst case (VWWC, Lomax): build a single worst-case dose vector — min over scenarios in target, max over scenarios in OAR — then penalize that. matRad implements VWWC, VWWC_INV, COWC, OWC.
   - max is non-smooth → smooth it (log-sum-exp or p-norm) for gradient-based solvers (matRad useMaxApprox). Gradient of COWC = gradient of the (smoothed-)argmax scenario.
   - Robust gradient still backprojects: ∂f/∂w = Σ_k (P^(k))ᵀ (∂f/∂D^(k)) weighted by scenario weight / worst-scenario indicator.
9. Land on matRad-structured code: build dij (dose-influence) per scenario, penalized-LS objectives (squared dev / over / under), DoseProjection (d=Pw, wGrad=Pᵀ δ), objective wrapper that sums over structures and over robustness mode (nominal / expected / composite-worst-case), feed f and ∇f to a bound-constrained optimizer with w≥0.

## Design-decision → why (table)
- **Spot weights w as variables (not field shapes)**: spot scanning delivers each Bragg spot independently; weight = #protons is the natural, linearly-acting DOF. Alternative (passive scattering + apertures) can't modulate in depth per-field → can't do 3D IMPT.
- **Linear dose-influence matrix P**: dose is linear in fluence (each proton deposits independently to first order); precompute P once, every objective/gradient is then cheap matrix algebra. Alternative (re-run dose engine each iteration) is far too slow.
- **Quadratic (squared) penalties**: smooth, convex-in-dose, differentiable everywhere → gradient methods converge fast; least-squares matches "get close to prescription". Alternative (linear/L1) non-smooth; (max/DVH) non-convex, used only as added constraints.
- **One-sided (Heaviside) penalties for OAR/target-min**: only overdosing an OAR is bad, only underdosing the target min is bad — penalizing the good direction would fight coverage. The positive-part operator encodes the asymmetry.
- **Penalty/|V| normalization**: divide by #voxels so a big structure doesn't dominate by sheer count; per-structure penalty p_s sets the clinical trade-off (target coverage vs OAR sparing).
- **w ≥ 0 bound**: negative proton count is unphysical. Handled by a bound-constrained solver (IPOPT/L-BFGS-B/projected gradient), not unconstrained.
- **Why robust at all (not just margins)**: a static geometric margin (CTV→PTV) works for photons because dose is roughly shift-invariant; for protons a setup/range shift CHANGES the dose shape (peaks move in depth), so a static-margin PTV does not guarantee coverage. Must optimize over scenarios.
- **Probabilistic vs worst-case**: expected-value is less conservative, needs scenario probabilities, smooth; worst-case guarantees the bound but can be pessimistic. Worst-case chosen when you must protect a serial OAR under any plausible error.
- **VWWC vs COWC**: VWWC builds one pointwise worst dose (cheap, but the "worst per voxel" can be an over-pessimistic non-physical combination); COWC takes the worst *whole-scenario* objective (physical scenario, but a single scalar max → needs smoothing). Trade conservativeness vs realism.
- **Smoothing the max (log-sum-exp / p-norm)**: gradient solvers need differentiability; hard max has a kink. log-sum-exp / large-p p-norm → smooth upper bound on max, recovers max as sharpness→∞.
- **Scenario set = range ±%, setup shifts ±mm on each axis (+ nominal)**: these are the dominant, physically grounded error modes; range error ∝ water-equivalent depth (~3.5–5%), setup ~±3–5 mm.

## Canonical code (matRad, fetched into code/)
- objectives: matRad_SquaredDeviation.m, matRad_SquaredOverdosing.m, matRad_SquaredUnderdosing.m — f = 1/N (Δ'Δ), grad = 2/N Δ, one-sided via Δ(Δ<0)=0 etc.
- projection: matRad_DoseProjection.m — d = dij.physicalDose{scen}*w; wGrad = (doseGrad' * dij.physicalDose)'. Prob: dExp, dOmega (variance).
- assembly: matRad_objectiveFunction.m / matRad_objectiveGradient.m — loop structures, robustness switch (none/STOCH/PROB/VWWC/VWWC_INV/COWC/OWC), max-approx (logsumexp/pnorm/none).
- optimizer: IPOPT / fmincon, bound w≥0.
