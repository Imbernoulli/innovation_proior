I need to stop treating language understanding as a collection of separate supervised products. The usual recipe works only after I choose one task, collect labels for it, and specialize the model or at least specialize a head. Even the stronger transfer recipe still ends with supervised fine-tuning. That means the system is not really general; it is a reusable initialization followed by many small task systems.

The one objective I can train at scale without labels is language modeling. It estimates `p(x)` by the chain rule, `product_i p(s_i | s_1, ..., s_{i-1})`. A task looks different at first because it asks for `p(output | input)`, and a general system should really model `p(output | input, task)`. But the task name, the input, and the output can all be written in language. If the prefix says "translate to French" and then gives English text, the desired French text is just a continuation. If the prefix gives a passage, a question, and an answer marker, the answer is again just a continuation.

This is the key reduction. The supervised loss for one task is the same next-token loss, evaluated only on the answer portion of a longer sequence. At the exact distributional optimum, a model that predicts all tokens in such sequences also predicts the answer tokens for the induced tasks. I should be careful not to overclaim this: the argument does not say a finite model will discover every task, and it does not help if the training text never contains natural demonstrations of the task. It says the objective is not the obstacle. The practical obstacles are coverage, capacity, and optimization.

So the data becomes the first real wall. A single-domain corpus can make a strong language model for that domain, but it will not contain enough forms of translation, summarization, question answering, cloze completion, explanations, tables, and dialogue. The web has that variety, but raw web crawls are full of boilerplate, gibberish, duplicates, and pages no person would choose to read. I need a cheap human filter. A link that has been posted to Reddit and received at least three karma has already passed a weak crowd filter: someone found it interesting, useful, or amusing, and others agreed enough to upvote it. That gives me a way to scrape broad web text without starting from all of Common Crawl.

The corpus should be the text extracted from those outbound links, not the HTML around them. I can use content extractors, deduplicate, and do heuristic cleaning. I also need a cutoff so later evaluation datasets cannot leak backward through fresh links. Wikipedia must be removed because it is a common source for language-modeling and question-answering benchmarks; even if removing it costs useful text, it makes the zero-shot claim cleaner. The result I want is not a task dataset but a broad field of naturally occurring demonstrations.

Now I need the model to read any benchmark without task-specific preprocessing. Lowercasing, word tokenizers, and unknown tokens are all the wrong interface because they make the string space depend on preprocessing choices. Bytes solve coverage: UTF-8 can represent any Unicode string with a base alphabet of 256. But pure byte-level language modeling spends too much sequence length and capacity on low-level spelling and encoding patterns. I need byte coverage with larger useful units.

Byte-pair encoding gives the right direction because it turns frequent adjacent symbols into longer symbols while leaving rare material decomposable. The trap is that standard BPE is usually applied to Unicode code points. If I want full Unicode coverage before any merges, the base vocabulary alone is already above 130,000 symbols, larger than the useful vocabulary budget. Applying BPE after mapping bytes to reversible Unicode characters keeps the base alphabet at 256.

There is another trap. Greedy BPE over raw byte strings will happily spend vocabulary slots on variants such as a word followed by a period, the same word followed by an exclamation point, and the same word followed by a question mark. Those are not separate lexical discoveries; they are punctuation attachment. If I allow merges across every boundary, I waste vocabulary and capacity. The fix is to split the byte-mapped text by character category before applying merges: letters with letters, numbers with numbers, punctuation with punctuation, and whitespace as its own pattern. I make an exception for a leading space because English word boundaries are highly informative and compression improves. That gives a vocabulary around 50,257 tokens, no unknown token, and no lossy normalization.

The architecture can start from the decoder-only Transformer used for generative pretraining. It already has causal masked self-attention, learned token and position embeddings, a feed-forward sublayer with a four-times wider hidden dimension, and a tied output projection through the token embedding matrix. For language modeling, I do not need an encoder or cross-attention. I need a larger stack and a longer context.

Depth creates the next wall. In the older Transformer block, layer normalization sits after the residual addition. That means the identity path is not a clean identity path through many layers; every residual stream is immediately normalized. Very deep residual networks suggest a better arrangement: normalize the input to each sub-block, apply the sub-block, then add the result back. So the block becomes `x = x + attn(norm(x))`, then `x = x + mlp(norm(x))`. The residual stream stays an additive path through the stack.

Moving layer normalization before sub-blocks leaves the top of the stack unnormalized. Each block normalizes its own input, but after the last residual addition there is no post-block normalization. I therefore add a final layer normalization before the output projection. That is not a decorative extra layer; it is the counterpart of moving normalization off the residual path.

Initialization also changes with depth. A residual stack adds many layer outputs into one stream. If there are `N` residual layers and each contributes at the same initial scale, the accumulated residual signal grows like `sqrt(N)` under the usual independent-variance approximation. Scaling residual-layer weights by `1/sqrt(N)` at initialization keeps that accumulated scale controlled. Here `N` is the number of residual layers, meaning the attention and feed-forward residual branches across the stack, not merely the number of Transformer blocks if those are counted separately.

The actual model family should test whether capacity is the limiting variable. I keep the smallest model at the previous scale, 12 layers and width 768. Then I make log-spaced larger models: 24 layers at width 1024, 36 at width 1280, and 48 at width 1600. I increase the context from 512 to 1024 tokens because web documents and long-dependency benchmarks need more prefix. I tune only the learning rate per size on held-out corpus text, not on the downstream tasks, because downstream tuning would break the zero-shot claim.

I also need to keep evaluation honest. For language-modeling benchmarks I can compute the likelihood of the benchmark text directly because the tokenizer is reversible and does not require dataset-specific tokenization. For cloze and multiple-choice tasks I can score the candidate continuations under the language model. For generation-style tasks I can prompt with natural text markers, such as an answer marker or a short translation pattern, and decode without changing weights. The prompt is allowed because it is just context; learned task heads are not.

The risk is memorization. A broad web corpus will overlap with public benchmarks in small and annoying ways. Removing Wikipedia helps but does not prove enough. I need n-gram overlap checks between training text and benchmark test text, and I need to interpret gains under that shadow. If overlap changes the metrics only slightly, then the result is still a zero-shot transfer result rather than a memorized-test result. If overlap is large, the benchmark has to be reported differently.

The final shape is therefore simple but specific. Train a large causal Transformer on a Reddit-filtered, deduplicated, non-Wikipedia web corpus using a byte-level BPE tokenizer that preserves arbitrary strings and avoids wasteful cross-category merges. Use pre-layer-normalized residual blocks, a final layer normalization, a longer context, and residual-depth-aware initialization. Then evaluate by prompting or scoring continuations, with no supervised updates. The bet is not that a new task module appears. The bet is that many tasks are already written inside natural text, and a sufficiently strong next-token predictor must model those task-conditioned continuations to model the text well.
