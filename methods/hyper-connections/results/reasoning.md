I start from the residual connection because it is doing two jobs with one object. It gives gradients a clean route through depth, and it decides how much a new layer output matters after it is added. In Pre-Norm, `h_k = h_{k-1} + T_k(Norm(h_{k-1}))`, the first job is handled well: the identity path is untouched. But the second job becomes weak in deep layers because the branch always reads a normalized vector while the stream it writes into keeps accumulating unnormalized content. The branch output is not necessarily small by itself; it is small relative to the growing stream. So deep neighboring representations become too similar.

Post-Norm fixes that forward imbalance by normalizing after the addition, but then I lose the clean highway. The backward path now keeps crossing normalization Jacobians, and the depth product becomes the source of instability. I can tune scalar branch weights, but that still leaves one stream and one depth pattern. One coefficient cannot simultaneously mean "preserve a long clean route" and "give this new computation a strong local influence" when those demands point in opposite directions.

So I need more than a scalar. The first thought is to let the residual connection become a small linear connection rule. But if I keep a single hidden vector, a learned `2 x 2` rule is still one stream: it can reweight the old state and the new branch output, yet it cannot keep multiple depth patterns alive at once. The stream itself must have room to carry more than one pattern.

I therefore replicate the hidden state into `n` parallel streams, forming a hyper-hidden matrix `H in R^{n x d}`. The sublayer still wants one `d`-dimensional input, so I need one set of weights to read from the streams into that input. After the sublayer runs, I need another set of weights to write its output back into the streams. I also need the old streams to mix into the new streams, otherwise the copies are isolated lanes rather than a richer residual state.

This naturally gives a small `(n+1) x (n+1)` connection matrix. I reserve index zero for the branch output/input slot and indices `1..n` for the streams. In block form it is `HC = [[0, B], [A_m, A_r]]`. Here `A_m in R^{n x 1}` reads the streams into the branch input, `A_r in R^{n x n}` carries and mixes streams forward, and `B in R^{1 x n}` writes the branch output back into the streams. With row-vector conventions for the streams, the update becomes `h_0^T = A_m^T H`, `H' = A_r^T H`, and `H_hat = B^T T(h_0)^T + H'`.

This is the right factorization because it keeps the expensive part unchanged. Attention or the feed-forward network still sees one vector per token. The new work is only a small stream-mixing operation around that sublayer. The diagonal of `A_r` together with `B` describes the local depth connection; the full `[A_m A_r]` matrix describes how the streams exchange information. Those are exactly the two pieces the ordinary residual merges into one fixed rule.

Now I check whether this construction actually contains the old residual, because a replacement that cannot start as the baseline is too risky. For `n = 1`, the Pre-Norm residual is the matrix `[[0, 1], [1, 1]]`: read the one stream, run the sublayer, add the output and the old stream. Post-Norm can also be represented with weights depending on the input/output variances and covariance. So the new object is not a different sublayer architecture; it is a generalization of the connection rule around the same sublayer.

The important point is that `n = 1` still cannot break the seesaw. It can learn a different strength, but it cannot reserve separate depth patterns. With `n > 1`, one stream can preserve a clean long route while another stores a different recent-output pattern, and width mixing lets later layers read the useful combination. That is the structural reason for multiple streams.

The same matrix also explains sequential and parallel layouts. With `n = 2`, one fixed matrix makes the two streams behave like a sequential residual stack. A different odd/even pair of matrices makes two neighboring computations behave like a parallel transformer block. So the connection matrix is not merely a gate; it is a small representation of layer arrangement. If I learn it, I can learn mixtures between sequential and parallel depth patterns.

A static learned matrix is useful, but the best routing should depend on the token and current hidden state. I therefore let the connection weights be a static base plus a dynamic correction predicted from `H`. The predictor must be controlled. I normalize `H` before the projection so the routing predictor does not inherit the same scale growth that caused the original problem. I pass the projection through `tanh` so the correction is bounded. I multiply by a small learnable scale initialized to `0.01`, and I initialize the projection weights to zero so the dynamic correction is exactly zero at step zero.

The dynamic alpha predictor can be implemented as one matrix of shape `(d, n+1)` applied to each stream row. Its first output column corrects `A_m`; its remaining `n` columns correct that row of `A_r`. The dynamic beta predictor has shape `(d,)`, producing one write weight per stream row. This matches the formula view: `B(H)`, `A_m(H)`, and `A_r(H)` are the static matrices plus small normalized tanh corrections.

Initialization is the subtle part. I set `B = 1_{1 x n}`, `A_r = I_n`, and `A_m = e_{k mod n}` for the residual site index `k`; the dynamic projection weights are zero. Then the branch reads one stream, the old streams carry forward unchanged, and the branch output is added to every stream. Because all streams start as identical copies, every stream remains identical under this static base. The row-sum before the final normalization is therefore `n` times the ordinary Pre-Norm hidden vector, not literally the same vector. The following normalization removes global scale in the ideal formula, with only epsilon-level numerical differences in an implementation, so the model function starts as the Pre-Norm baseline. I must keep that scale caveat explicit.

The implementation should mirror this algebra exactly. `static_beta` is `ones(rate)`. `static_alpha` is the concatenation of the one-hot `A_m` column and the identity `A_r`. In `width_connection`, the dynamic path computes normalized `h`, then alpha and beta corrections separately, adds the static tensors with `[None, None, ...]` broadcasting, and forms `mix_h = alpha.transpose(-1, -2) @ h`. The first mixed row is the branch input; the remaining rows are the carried streams. In `depth_connection`, the branch output is distributed with `torch.einsum("blh,bln->blnh", h_o, beta)` and added to `mix_h[..., 1:, :]`.

The overhead is small for the right reason, not because the method is free. A static module has `n(n+2)` scalar parameters. A dynamic module has `|theta_norm| + d_model(n+2) + n(n+2) + 2` parameters: one beta projection, one alpha projection that covers `A_m` and `A_r`, the static matrices, and two scale parameters. The main extra computation is the width connection, `O(d_model * n * (n+1))` per token, which is tiny beside attention projections and the feed-forward network for small `n`.

Putting it together, the residual connection becomes a learnable, optionally input-dependent connection matrix over multiple streams. The Transformer sublayers do not change. The network begins as Pre-Norm up to the final normalization's scale invariance, then learns how strongly to carry old information, where to write new layer outputs, and when to behave more sequentially or more parallel. That is the path out of the original seesaw.
