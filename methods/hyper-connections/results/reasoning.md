I start from the residual connection because it is doing two jobs with one object. It gives gradients a clean route through depth, and it decides how much a new layer output matters after it is added. In Pre-Norm, `h_k = h_{k-1} + T_k(Norm(h_{k-1}))`, the first job is handled well: the identity path is untouched. But the second job becomes weak in deep layers because the branch always reads a normalized vector while the stream it writes into keeps accumulating unnormalized content. The branch output is not necessarily small by itself; it is small relative to the growing stream. So deep neighboring representations become too similar.

Post-Norm fixes that forward imbalance by normalizing after the addition, but then I lose the clean highway. The backward path now keeps crossing normalization Jacobians, and the depth product becomes the source of instability. I can tune a scalar branch weight, but that still leaves one stream and one depth pattern. A single coefficient has to mean both "preserve a long clean route" and "give this new computation a strong local influence," and in deep layers those two demands pull in opposite directions. Whatever value I pick is a compromise between them.

So I need more than a scalar. The first thought is to let the residual connection become a small learned linear rule instead of two fixed ones. With a single hidden vector that is a `2 x 2` mix of the old state and the new branch output. But I should check whether that actually buys anything over a tuned scalar. A `2 x 2` rule still maps one vector to one vector: at each block it produces a single combined state, and the next block reads that same single state. There is still exactly one accumulator. It can choose the carry-versus-write ratio more flexibly, but it cannot keep a clean long route and a strong local write *simultaneously available*, because both have to be collapsed into the one vector that gets passed on. The thing that is too small is not the number of mixing coefficients; it is the width of the state. The stream itself must have room to carry more than one pattern at a time.

I therefore replicate the hidden state into `n` parallel streams, forming a hyper-hidden matrix `H in R^{n x d}`. The sublayer still wants one `d`-dimensional input, so I need a set of weights to read from the streams into that input. After the sublayer runs, I need another set of weights to write its output back into the streams. I also need the old streams to be allowed to mix into the new streams, otherwise the copies are isolated lanes rather than a single richer residual state.

Counting the maps that gives: one read (streams to the branch input slot), one write (branch output to the streams), and one carry/mix (streams to streams). If I put the branch input/output on index zero and the streams on indices `1..n`, all three maps fit in one `(n+1) x (n+1)` matrix `HC = [[0, B], [A_m, A_r]]`. Here `A_m in R^{n x 1}` reads the streams into the branch input, `A_r in R^{n x n}` carries and mixes streams forward, and `B in R^{1 x n}` writes the branch output back into the streams. The top-left entry is zero because the branch input is never fed straight back into the branch output without passing through the sublayer. With row-vector conventions for the streams, the update is `h_0^T = A_m^T H`, `H' = A_r^T H`, and `H_hat = B^T T(h_0)^T + H'`.

What I like about this factorization is where the cost lands. Attention or the feed-forward network still sees exactly one vector per token: it acts only on `h_0`, the single mixed branch input. The new work is the stream mixing around the sublayer, which is the `(n+1) x (n+1)` matrix applied to a width-`n` stack — small for the small `n` I have in mind. The diagonal of `A_r` together with `B` describes the local depth connection of each stream; the off-diagonal of `A_r`, plus `A_m`, describes how streams exchange information. Those are exactly the two pieces the ordinary residual fuses into one fixed rule.

Before going further I want to know that this object actually *contains* the residual I am trying to replace — a replacement that cannot start as the baseline is too risky to train at scale. Take `n = 1`. Then `B`, `A_m`, `A_r` are all scalars, and the obvious values are `B = A_m = A_r = 1`, i.e. the matrix `[[0,1],[1,1]]`: read the one stream into the branch, run the sublayer, add both the output and the carried stream. I should make sure the algebra actually collapses to `h + T(h)` and not to something off by a term. Reading the formulas with `n=1`: `h_0 = A_m^T H = H`, `H' = A_r^T H = H`, and `H_hat = B^T T(h_0) + H' = T(H) + H`. That is the Pre-Norm update. I ran the reference module with `rate=1` against a plain `h + T(h)` baseline on a random input and the two outputs agreed to `0.0` max difference, so the reduction is exact, not just morally right. Post-Norm is also reachable with the weights set from the input/output variances, so the new object is not a different sublayer architecture; it is a strictly larger family of connection rules around the same sublayer.

But `n = 1` is exactly the case that cannot break the seesaw — it is one accumulator again, just confirming the embedding. The point of width is `n > 1`: one stream can hold a clean long route while another holds a recent-output-heavy pattern, and the mixing in `A_r`/`A_m` lets a later layer read whichever combination it needs. That is the structural reason to pay for more than one stream.

A second thing falls out of the same matrix, which makes me more confident this is the right level of generality and not just a gate. With `n = 2`, one fixed choice of `HC` makes the two streams behave like an ordinary sequential residual stack; a different pair of matrices alternating on odd/even sites makes two neighbouring computations behave like a parallel block, where attention and FFN read the same input and their outputs are added side by side. So `HC` is a small representation of the *layer arrangement*, not only the carry strength. If I learn it, I can learn mixtures between sequential and parallel depth, which a scalar weight cannot express at all.

A static learned matrix is already more than the baseline, but the routing a token needs probably depends on the token. So I want the connection weights to be a static base plus a correction predicted from the current `H`. The predictor needs to be controlled, and here I have to be careful, because the original failure was scale growth in the stream and a naive predictor would inherit it: if I feed raw `H` into a linear map, deep layers with large-norm streams produce large logits and the routing changes meaning with depth. So I normalize `H` before the projection. I pass the projection through `tanh` so each corrected weight moves by a bounded amount, and I multiply by a small learnable scale `s` initialized to `0.01`. I also initialize the projection weights to zero so the correction is exactly zero at step zero. The bound here matters: since `|tanh| <= 1`, the dynamic correction to any weight is at most `s`. I checked this directly — saturating the projection with large weights drives the correction to exactly `0.00999...`, i.e. it sits on the scale and never exceeds it. So the dynamic form starts as the static form and can only ever nudge the static matrix by `±s` per entry until `s` itself grows.

Concretely the alpha predictor is one matrix of shape `(d, n+1)` applied to each normalized stream row: its first output column corrects `A_m`, its remaining `n` columns correct that stream's row of `A_r`. The beta predictor is a vector of shape `(d,)`, giving one write weight per stream. This matches the formula view where `B(H)`, `A_m(H)`, `A_r(H)` are static matrices plus small normalized-tanh corrections.

Initialization is the subtle part, and this is where I want to actually trace it rather than trust it. I set `B = 1_{1 x n}` (write the output into every stream), `A_r = I_n` (carry each stream unchanged), and `A_m = e_{k mod n}` for the residual-site index `k` (read one specific stream into the branch). The dynamic projections are zero, so the dynamic correction vanishes and only the static base acts. I start `H` as `n` identical copies of `h_0`.

Now I trace one site. `A_m = e_{k mod n}` picks out one stream, but all streams are equal, so the branch input is exactly `h_0` regardless of which one is picked — I confirmed the branch input matched `h_0` to `0.0` on a random input. The sublayer produces `T(h_0)`. `A_r = I` carries every stream forward as `h_0`, and `B = 1` adds `T(h_0)` to *every* stream. So after the site each stream is `h_0 + T(h_0)` — identical, and identical to the Pre-Norm output of that one block. The module preserved the "all streams equal" invariant, which it must, or the streams would diverge for a reason unrelated to learning. I verified the post-site stream-to-stream difference is `0.0` and each stream equals the Pre-Norm step output to `0.0`.

The one place this is *not* literally the baseline is the read-out. At the end of the model the `n` streams are summed. With every stream equal to the Pre-Norm vector `v`, the sum is `n v`, not `v`. I initially expected the initialized model to be bit-identical to Pre-Norm and only noticed the factor when I summed the streams: the row-sum came out as exactly `n` times the single-stream Pre-Norm output (max difference `0.0` against `n * base_out`), not equal to it. So there is a global factor of `n` sitting in front of the whole hidden state at the final layer. A single multiplicative constant on the hidden state is removed by the final normalization in the mathematical rule — `norm(n v) = norm(v)` — up to epsilon-level numerical differences in an implementation, so the *model function* still starts as Pre-Norm even though the pre-norm activations are scaled. That is the honest statement: identical after the final norm, scaled by `n` before it.

The factor does not stay harmless everywhere, though. The streams are summed, so each branch-output module is effectively writing into something that will be added up `n` times relative to the single-stream case, and the initialization variance of those modules was tuned for the single-stream sum. To keep the initialized forward statistics matched I scale the initialization standard deviation of each residual branch-output module by `sqrt(n)` — concretely the FFN second linear layer and the attention output projection. This is separate from the connection matrix itself; it is the price of summing streams at the end. I keep both caveats explicit: the `n`-factor that the final norm absorbs, and the `sqrt(n)` init that compensates the summation for the branch outputs.

The implementation should mirror this algebra exactly. `static_beta` is `ones(n)` (the `B` row). `static_alpha` is the one-hot `A_m` column concatenated with the identity `A_r`, shape `(n, n+1)`. In `width_connection`, the dynamic path normalizes `h`, computes the alpha and beta corrections through their projections and `tanh`, scales them, and adds the static tensors with `[None, None, ...]` broadcasting; then `mix_h = alpha.transpose(-1, -2) @ h`. The first mixed row is the branch input `h_0`; the remaining rows are the carried/mixed streams `H'`. In `depth_connection`, the branch output is distributed to the streams with `torch.einsum("blh,bln->blnh", h_o, beta)` and added to `mix_h[..., 1:, :]`, giving `H_hat`.

Last, the cost, which I want to be honest about rather than call free. A static module holds `static_beta` (`n`) and `static_alpha` (`n(n+1)`), so `n + n(n+1) = n(n+2)` scalars. A dynamic module adds the layer-norm parameters `|theta_norm|`, the alpha projection `d(n+1)`, the beta projection `d`, and two scales, giving `|theta_norm| + d(n+2) + n(n+2) + 2`. I checked these against the actual module parameter counts: for `n=4, d=4` the static count is `24` and the dynamic count is `58`, both matching the formulas exactly. The dominant extra compute is the width connection, `O(d * n * (n+1))` per token, which is small beside the attention projections and FFN for the small expansion rates I have in mind — the main LLM setting being the dynamic form with `n = 4`.

Putting it together, the residual connection becomes a learnable, optionally input-dependent connection matrix over `n` streams. The sublayers do not change; they still see one vector. The network begins as Pre-Norm up to the final normalization's scale invariance, then is free to learn how strongly to carry old information, where to write new outputs, and when to behave more sequentially or more in parallel — the degrees of freedom the one fixed scalar residual never had, which is what the original Pre-Norm/Post-Norm seesaw was forcing me to trade away.
