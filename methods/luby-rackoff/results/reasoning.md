I start with the mismatch between the primitive I have and the primitive I need. A pseudorandom function gives me oracle answers that look random to every efficient adaptive observer, but a block cipher needs a permutation. If I use the PRF directly, I get no promise that distinct inputs have distinct outputs and no natural inverse. The first task is therefore not more randomness; it is randomness plus reversibility.

I then look at the Feistel step as the minimal reversible wrapper around an arbitrary function. If I map `(L, R)` to `(R, L xor f(R))`, I can invert the step because the new left half tells me which value of `f` to recompute, and xor recovers the old left half. This solves the mechanical inversion problem, but it plainly does not solve pseudorandomness: one round exposes the old right half as the new left half.

I try to understand why simply repeating the wrapper is not enough by itself. Two rounds already hide the raw right half, but a fixed algebraic relation still ties pairs of inputs with the same right half. The primary paper makes this concrete by showing a two-round distinguisher. In the stronger forward-and-inverse model, even three rounds leave a handle. So the issue is not whether Feistel can mix data; the issue is whether enough independent random-looking round values stand between the adversary and every visible relation.

I separate the problem into an ideal part and a computational part. For the ideal part, I pretend the round functions are truly random and ask what a bounded oracle algorithm can learn from a transcript. Each query pushes an input pair through hidden half-block variables. If the relevant hidden variables are all fresh, then the middle random functions answer at fresh points, and the resulting output pair has exactly the same probability it would have in the ideal experiment.

The only way this transcript comparison breaks is through collisions among those hidden half-block values. That is the central insight. The Feistel equations do not have to make every internal value globally independent; they only have to make the values touched by a polynomial number of queries fresh with high probability. The bad event is then a birthday event in an `n`-bit half-block space, so its probability is on the order of the square of the number of queries divided by `2^n`.

With three rounds in the forward-only setting, I can place the fresh random choices in the middle after the outside structure has randomized the relevant coordinates. The primary paper's main lemma formalizes this by comparing two experiments on oracle circuits and showing they differ only when a preserving condition fails. When preserving holds, the structured Feistel experiment and the random experiment produce the same transcript value by value.

I then move back from random functions to pseudorandom functions by a hybrid argument. I replace the round functions one at a time. If an efficient distinguisher notices the whole Feistel permutation changing during one replacement, I can use that distinguisher as a subroutine to tell a PRF oracle from a truly random function. That contradicts the PRF assumption. The proof therefore factors cleanly: combinatorial collision control for random functions, computational indistinguishability for replacing them with PRFs.

For inverse queries, I need to protect both ends of the transcript. A three-round construction can be probed from one side and then partially unwound from the other, revealing a relation that a random permutation almost never has. Adding the fourth independent Feistel round gives the inverse-query game the same kind of outer randomness that the forward-only proof needs. Now both encryption and decryption transcripts are screened by fresh half-block values except on the same kind of bounded collision events.

The method I arrive at is therefore a compiler from local unpredictability to permutation indistinguishability. PRFs supply values that look like random oracle answers at every queried point. Feistel wiring supplies invertibility without asking those functions to be invertible. A small number of independent rounds makes every efficient transcript look as if it came from a random permutation, unless the adversary hits an internal collision or one of the PRFs is distinguishable from random.
