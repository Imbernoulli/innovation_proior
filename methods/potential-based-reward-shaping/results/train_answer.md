In a sequential decision problem the entire task is carried by one object — the reward function $R$. Fix the MDP $M=(S,A,T,\gamma,R)$ and the optimal policy is pinned down; nothing else encodes the goal. Yet in practice we constantly add to the reward. To make a hard exploration problem tractable we sprinkle in extra *shaping* rewards $F$ on top of $R$, so the learner actually runs on the modified MDP $M'=(S,A,T,\gamma,R+F)$ and maximizes $R+F$ rather than $R$. This works, often spectacularly — but it can silently move the optimum, and the cautionary tales are concrete. A bicycle agent rewarded for progress toward its goal, but not penalized for moving away, learned to ride in tiny circles near the start, harvesting "progress" reward every time it happened to point goalward. A soccer robot rewarded for touching the ball learned to park beside it and vibrate, touching it as fast as possible. In each case the shaping reward, not the task, dictated the behavior, and the behavior was plainly terrible for the real objective. So the sharp question I want answered is exactly this: for which $F$ can I *guarantee* that every policy optimal under $R+F$ is still optimal under $R$ — policy invariance — and ideally guarantee it without knowing the dynamics $T$, since I am shaping precisely because the problem is hard and partly unknown.

The prior art does not settle this. The general additive form $R'=R+F$ with bounded $F:S\times A\times S\to\mathbb R$ is the right level of generality — it is even implementable model-free, since $M$ and $M'$ share $S,A,T$ and I can just report $R+F$ on whatever transition the environment hands me — but it is far too permissive and contains exactly the bugs. Distance-to-goal shaping (reward $r$ for any move closer to the goal) is not symmetric, so going out and coming back nets positive reward: the tiny-circle exploit. Subgoal/event shaping (reward for touching the ball, crossing a flag) pays out every time a repeatable event recurs: the vibration exploit, and with wrong magnitudes a subgoal detour can even beat the truly optimal path. The classical robustness bound — perturbing rewards by $\varepsilon$ keeps value within $2\varepsilon/(1-\gamma)$ of optimal — bounds damage but never characterizes the transformations that are *exactly* safe. Utility theory long ago answered the one-shot analogue, that under uncertainty only positive linear transformations of utility preserve the optimal choice, but the multi-step, sequential version had not been worked out.

The way in is to name the disease the bugs share. Each pathology is a *loop*: a closed cycle of states $s_1\to s_2\to\cdots\to s_n\to s_1$ around which the shaping reward sums to something positive, $F(s_1,a_1,s_2)+\cdots+F(s_n,a_n,s_1)>0$. Once such a cycle exists the agent has found a money pump — it can sit in the loop forever banking shaping reward with no relation to $R$ — and a learner optimizing $R+F$ will rationally exploit it. So the cure must make every cycle sum to exactly zero, and the construction that guarantees this is to attach a number $\Phi(s)$ to each state and pay the *difference* on each move: with $F$ a difference of a state quantity, summing around any closed loop telescopes and every intermediate $\Phi$ cancels, leaving zero. I propose **potential-based reward shaping**: the safe shaping rewards are precisely the differences of such a state *potential* $\Phi$. Killing cycles is only necessary, though, not obviously sufficient, and the naive undiscounted difference $\Phi(s')-\Phi(s)$ does not survive discounting — when the next state's value arrives weighted by $\gamma$, the per-step contributions no longer line up to cancel. So I derive the correct discounted form by demanding the property I actually want rather than guessing it.

What I want is for $F$ to be invisible to the *choice* of action. The policy is the greedy argmax over $Q^*(s,\cdot)$, so I am completely safe if adding $F$ shifts $Q^*$ at each state by an amount that is the *same across all actions at that state* — an offset depending only on $s$ cannot reorder the actions, whereas one depending on $a$ can. So I force $Q^*_{M'}(s,a)=Q^*_M(s,a)-\Phi(s)$ and read off the $F$ that makes it hold. Starting from the Bellman optimality equation for $M$,
$$Q^*_M(s,a)=\mathbb E_{s'\sim P_{sa}}\Big[R(s,a,s')+\gamma\max_{a'}Q^*_M(s',a')\Big],$$
subtract $\Phi(s)$ from both sides and define $\hat Q(s,a):=Q^*_M(s,a)-\Phi(s)$. Inside the max I have $Q^*_M(s',a')$ but I need $\hat Q(s',a')=Q^*_M(s',a')-\Phi(s')$, so I insert and repay $\gamma\Phi(s')$; because $\Phi(s')$ is constant across $a'$ it pulls straight through the max, giving $\gamma\max_{a'}Q^*_M(s',a')=\gamma\max_{a'}\hat Q(s',a')+\gamma\Phi(s')$. Substituting,
$$\hat Q(s,a)=\mathbb E_{s'}\Big[R(s,a,s')+\underbrace{\gamma\Phi(s')-\Phi(s)}_{F(s,a,s')}+\gamma\max_{a'}\hat Q(s',a')\Big].$$
The leftover terms that *had* to appear are $\gamma\Phi(s')-\Phi(s)$. So the discounted shaping reward is
$$F(s,a,s')=\gamma\,\Phi(s')-\Phi(s),$$
and with this choice the bracket becomes $R'(s,a,s')+\gamma\max_{a'}\hat Q(s',a')$, exactly the Bellman optimality equation for $M'$. The $\gamma$ has to sit on the *future* potential because in the discounted recursion the next state's value already arrives discounted by one step; the potential of where you are going must be discounted to match, while the potential of where you are is subtracted at full weight. At $\gamma=1$ this collapses back to the telescoping difference the cycle argument predicted, the consistency check I wanted, and notice $F$ does not depend on the action at all — a money pump is about the geography of the loop, not which lever was pulled.

Sufficiency then closes cleanly by uniqueness. The Bellman optimality equation has a unique solution (the contraction fixed point for $\gamma<1$, the unique optimal $Q$ in the proper undiscounted case), and I have just shown $\hat Q=Q^*_M-\Phi$ satisfies it for $M'$. For the undiscounted boundary I take $\Phi(s_0)=0$ without loss of generality — replacing $\Phi$ by $\Phi-\Phi(s_0)$ leaves $F=\Phi(s')-\Phi(s)$ unchanged when $\gamma=1$ — so $\hat Q(s_0,a)=0$ matches $M'$'s absorbing boundary; for $\gamma<1$ there is no boundary to check. Hence
$$Q^*_{M'}(s,a)=Q^*_M(s,a)-\Phi(s),\qquad V^*_{M'}(s)=V^*_M(s)-\Phi(s),$$
and because the shift $-\Phi(s)$ is identical across actions,
$$\pi^*_{M'}(s)\in\arg\max_a Q^*_{M'}(s,a)=\arg\max_a\big(Q^*_M(s,a)-\Phi(s)\big)=\arg\max_a Q^*_M(s,a).$$
Running the identical argument with $M$ and $M'$ swapped (shaping $-F$, potential $-\Phi$) gives the reverse inclusion, so the two MDPs have *exactly the same set* of optimal policies. Two refinements sharpen the "why." First, the identity $V^\pi_{M'}=V^\pi_M-\Phi$ holds for *every* $\pi$, not just the optimal one — the telescoping never used optimality — so a merely near-optimal policy in $M'$, within $\varepsilon$ of $V^*_{M'}$, is within the same $\varepsilon$ of $V^*_M$ once the common $-\Phi(s)$ cancels; near-optimality transfers, which matters because learners return approximate optima. Second, the deepest reason: a pure potential reward $R(s,a,s')=\gamma\Phi(s')-\Phi(s)$ gives every policy the same value $-\Phi(s)$ from each state and every action the same action-value, so it is by itself *indifferent to policy* — and adding something with no preferences of its own cannot impose new preferences on $R$.

Necessity is what turns this from a clever trick into a characterization. If $F$ is *not* potential-based, I can always build a world that ambushes it. If $F$ depends on the action — some $s,s',a,a'$ with $F(s,a,s')\ne F(s,a',s')$, set $\Delta:=F(s,a,s')-F(s,a',s')>0$ — make both actions go $s\to s'$ with probability 1 and set $R(s,a,s')=0$, $R(s,a',s')=\Delta/2$; then $M$ prefers $a'$ while $M'$, where the rewards become $F(s,a,s')$ and $F(s,a,s')-\Delta/2$, prefers $a$, opposite. If $F$ depends only on $(s,s')$ but still fails to telescope, pick a reference $\hat s_0$ (the absorbing $s_0$ when $\gamma=1$, else any fixed state, normalizing $F(\hat s_0,\hat s_0)=0$ by a constant offset for $\gamma<1$), define the candidate potential $\Phi(s):=-F(s,\hat s_0)$, and find the witnessing pair where it fails, $\Delta:=F(s_1,s_2)+\gamma F(s_2,\hat s_0)-F(s_1,\hat s_0)\ne 0$ — exactly the amount by which $F$ fails to telescope along $s_1\to s_2\to\hat s_0$ versus the direct edge. Build $M$ with a decision on that crack: from $s_1$, action $a\to\hat s_0$ and action $a'\to s_2$ (then on to $\hat s_0$), with $R(s_1,a,\hat s_0)=\Delta/2$ and zero elsewhere. Then $Q_M(s_1,a)=\Delta/2$, $Q_M(s_1,a')=0$, while $Q_{M'}(s_1,a)=F(s_1,s_2)+\gamma F(s_2,\hat s_0)-\Delta/2$ and $Q_{M'}(s_1,a')=F(s_1,s_2)+\gamma F(s_2,\hat s_0)$, so the preference reverses either way. Putting both directions together: fixing $S,A,\gamma$, a bounded $F$ guarantees optimal-policy consistency between $M'$ and $M$ for *all* proper $T$ and all $R$ if and only if $F$ is potential-based, $F(s,a,s')=\gamma\Phi(s')-\Phi(s)$ (with $S\setminus\{s_0\}=S$ when $\gamma<1$). This is the multi-step analogue of the utility-theory fact that under uncertainty only positive linear transforms preserve the single-step optimum; here the safe terms are exactly the discounted differences of a state potential.

One choice remains: which $\Phi$. Invariance holds for any $\Phi$, so I pick it to make learning fast. Since $V^*_{M'}=V^*_M-\Phi$, taking $\Phi\approx V^*_M$ flattens the shaped value function toward zero, the easiest thing to learn — and because invariance is guaranteed regardless, a crude estimate is enough and a wrong $\Phi$ can never corrupt the optimum, only help less. So the design rule is: let $\Phi$ be your best cheap guess at the value-to-go, and that guess is where domain knowledge enters. On an undiscounted minimum-cost-to-goal grid with $-1$ per step and 80/20 stochastic moves, a step toward the goal makes about $0.8$ of a Manhattan-step of progress per timestep, so $V^*(s)\approx-\mathrm{manhattan}(s,\text{goal})/0.8$, giving $\Phi(s)=-\mathrm{manhattan}(s,\text{goal})/0.8$. Through $F=\Phi(s')-\Phi(s)$ this rewards motion toward the goal but now *symmetrically* — moving away pays exactly the negative of moving back — so the bicycle's tiny-circle money pump is impossible by construction rather than by patch. For ordered subgoals the same reasoning fixes the magnitudes the old hacks guessed: with total time-to-go $t$ and $n_s$ of five subgoals collected, $\Phi(s)=-((5-n_s-0.5)/5)\,t$ jumps by $t/5$ at each crossing, so $F$ hands out exactly $t/5$ per subgoal — a principled bonus that refunds itself if a subgoal is crossed backward and so cannot be farmed. In code it is a one-line slot dropped into the ordinary tabular control loop: the shaping reward is the discounted potential difference, the potential is the cheap value estimate, and the only change to learning is the added term.

```python
def shaping_reward(s, a, s_next, gamma, Phi):
    # F(s,a,s') = gamma*Phi(s') - Phi(s):
    #   discounted difference of a state-only potential => shaped returns differ
    #   by the start-state offset -Phi(s), leaving argmax_a exactly unchanged.
    return gamma * Phi(s_next) - Phi(s)

def manhattan_potential(goal, p_intended=0.8):
    # Phi(s) ~ V*(s) for a -1/step minimum-cost-to-goal grid
    def Phi(s):
        return -manhattan(s, goal) / p_intended
    return Phi

def learn(mdp, episodes, Phi, alpha=0.02, eps=0.1):
    Q = {(s, a): 0.0 for s in mdp.states for a in mdp.actions}
    for _ in range(episodes):
        s = mdp.reset(); done = False
        while not done:
            a = eps_greedy(Q, s, mdp.actions, eps)
            s2, r_env, done = mdp.step(s, a)
            r = r_env + shaping_reward(s, a, s2, mdp.gamma, Phi)   # add F
            target = r + (0.0 if done else mdp.gamma * max(Q[(s2, b)] for b in mdp.actions))
            Q[(s, a)] += alpha * (target - Q[(s, a)])
            s = s2
    return Q
```
