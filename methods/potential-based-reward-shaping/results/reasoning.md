Let me start from the thing that keeps biting people in practice. We have a reinforcement-learning agent, and the whole task is carried by one object ‚Äî the reward function $R$. Fix the MDP $(S,A,T,\gamma,R)$ and the optimal policy is determined; nothing else carries the goal. And yet people constantly *add* to the reward. They hand the agent extra "shaping" rewards on top of $R$ to make it learn faster ‚Äî and it genuinely works, sometimes spectacularly, turning a hopeless exploration problem into an easy one. So we are running the learner not on $M=(S,A,T,\gamma,R)$ but on a modified MDP $M'=(S,A,T,\gamma,R+F)$, where $F$ is whatever extra reward we sprinkled in. And here is the discomfort: the agent maximizes $R+F$, not $R$. We changed the objective. Did we change the optimum?

The cautionary tales say yes, sometimes, and silently. The bicycle agent ‚Äî rewarded for making progress toward its goal, not penalized for moving away ‚Äî learned to ride in tiny circles near the start, harvesting "progress" reward every time it happened to point the right way, and never going anywhere. A soccer robot given a reward for *touching the ball* learned to park next to the ball and vibrate against it as fast as possible. In both cases the shaping reward, not the task, dictated the behavior, and the behavior is plainly terrible for the actual task. So shaping is powerful and also dangerous, and the question I actually care about is sharp: for which $F$ can I *guarantee* that whatever policy comes out optimal under $R+F$ is still optimal under $R$? Call that policy invariance. I want the exact class of safe $F$.

Notice the form of the modification is already as general as it can be, for additive memoryless shaping: any bounded $F:S\times A\times S\to\mathbb R$, with $R'(s,a,s')=R(s,a,s')+F(s,a,s')$. That's good and bad. Good, because it's implementable even when I only have sample access to the world ‚Äî I never need the model; whenever the environment hands me $R(s,a,s')$ I just report $R+F$ on the same transition, and since $M$ and $M'$ share states, actions and transition probabilities, any learning algorithm that ran on $M$ runs unchanged on $M'$. Bad, because this space of $F$ is enormous and clearly contains the bugs. So "which $F$ are safe" is the entire question.

Let me stare at the bugs and try to see what they have in common, because if I can name the disease I might guess the cure. The bicycle: it goes out a little, comes back, points goalward again, collects progress reward, repeats. The soccer robot: touch, lose contact, touch again, collect. In each case there is a *loop* of states the agent visits over and over ‚Äî $s_1\to s_2\to\cdots\to s_n\to s_1$ ‚Äî and going once around the loop nets *positive* shaping reward: $F(s_1,a_1,s_2)+\cdots+F(s_n,a_n,s_1)>0$. Once that's true, the agent has discovered a money pump. It can sit in the loop forever and keep banking shaping reward, with no relation whatsoever to $R$. That's the disease. Any $F$ that gives a closed cycle net positive value creates an incentive to cycle that has nothing to do with the task, and a learner optimizing $R+F$ will, quite rationally, exploit it.

So the cure, at least the first guess, writes itself: make it impossible to gain by going around a loop. I want every cycle to sum to exactly zero in $F$. What kind of $F$ has that property automatically? If $F$ on a transition is a *difference of a quantity attached to states* ‚Äî assign each state $s$ a number $\Phi(s)$, and let the reward for the move $s\to s'$ be $\Phi(s')-\Phi(s)$ ‚Äî then summing around any closed loop telescopes:
$$\big(\Phi(s_2)-\Phi(s_1)\big)+\big(\Phi(s_3)-\Phi(s_2)\big)+\cdots+\big(\Phi(s_1)-\Phi(s_n)\big)=0.$$
Every intermediate $\Phi$ cancels; you end where you started, so the net is zero. The cycle exploit is gone by construction. This is the undiscounted picture, $\gamma=1$: $F(s,a,s')=\Phi(s')-\Phi(s)$. Notice $F$ doesn't even depend on the action ‚Äî only on where you came from and where you went. That feels right: a money pump is about the geography of the loop, not which lever you pulled.

Now I have to be honest that killing cycles is necessary for safety but I haven't shown it's *sufficient* ‚Äî eliminating loop-farming doesn't yet prove the optimal policy is preserved. And before I trust this even on the no-loop axis, I should push on the discounted case, because the telescoping argument as I wrote it quietly used $\gamma=1$, and almost all real problems discount. With discounting the return weights a reward received $t$ steps from now by $\gamma^t$. If I add $\Phi(s')-\Phi(s)$ at each step, the contributions no longer line up to cancel, because the later terms get extra powers of $\gamma$. Let me not just say that; let me catch the naive form actually doing damage on a tiny example, so I know the worry is real and not bookkeeping.

Take $\gamma=0.9$ and four states. From $s_0$, action $a$ goes to a state $s_1$ with potential $\Phi(s_1)=0$, action $b$ goes to a state $s_2$ with potential $\Phi(s_2)=10$; the other potentials are $0$. Give the *task* a reason to prefer $a$: $R(s_0,a,s_1)=1$, every other reward $0$, and let $s_1,s_2$ be absorbing-ish (self-loops, no reward). In the unshaped $M$ the values are $Q_M(s_0,a)=1$, $Q_M(s_0,b)=0$, so the agent should pick $a$. Now shape with the *naive* $F=\Phi(s')-\Phi(s)$: stepping into $s_2$ pays $F=\Phi(s_2)-\Phi(s_0)=10$ once, and then nothing (the self-loop on $s_2$ pays $\Phi(s_2)-\Phi(s_2)=0$). So in $M'$, $Q_{M'}(s_0,b)=10$ while $Q_{M'}(s_0,a)=1+0=1$. I ran value iteration to be sure and it returns exactly these: argmax flips from $a$ to $b$. The naive difference handed out a one-time $+10$ for *arriving* at the high-potential state and never clawed it back, so the agent is bribed to go somewhere the task didn't want. The reason the loop argument missed this: a single arrival isn't a closed loop, so "cycles sum to zero" said nothing about it. Killing loops is not enough; I need the form that also leaves *non-cyclic* decisions alone. Let me find it by demanding the thing I actually want rather than guessing again.

What do I actually want? I want $F$ to be invisible to the *choice* of policy. The policy is the greedy argmax over $Q^*(s,\cdot)$ at each state. So I'd be completely safe if adding $F$ shifted $Q^*$ at each state by an amount that is the *same across all actions $a$ at that state* ‚Äî because then the argmax over $a$ is untouched. A shift that depends only on $s$, not on $a$, is harmless; a shift that depends on $a$ can reorder the actions and change the policy. So my target is: find $F$ such that $Q^*_{M'}(s,a)=Q^*_M(s,a)-(\text{something depending only on }s)$. The natural candidate for "something depending only on $s$" is $\Phi(s)$. Let me just *try to force* $Q^*_{M'}(s,a)=Q^*_M(s,a)-\Phi(s)$ and read off which $F$ makes it true.

Start from what I know rigidly: $Q^*_M$ satisfies the Bellman optimality equation,
$$Q^*_M(s,a)=\mathbb E_{s'\sim P_{sa}}\Big[R(s,a,s')+\gamma\max_{a'}Q^*_M(s',a')\Big].$$
Subtract $\Phi(s)$ from both sides:
$$Q^*_M(s,a)-\Phi(s)=\mathbb E_{s'}\Big[R(s,a,s')+\gamma\max_{a'}Q^*_M(s',a')-\Phi(s)\Big].$$
I want the right side to look like a Bellman equation for $M'$ in the variable $\hat Q(s,a):=Q^*_M(s,a)-\Phi(s)$. Inside the max I have $Q^*_M(s',a')$, but I want $\hat Q(s',a')=Q^*_M(s',a')-\Phi(s')$. So insert $\Phi(s')$ and pay it back: $\gamma\max_{a'}Q^*_M(s',a')=\gamma\max_{a'}\big(Q^*_M(s',a')-\Phi(s')\big)+\gamma\Phi(s')=\gamma\max_{a'}\hat Q(s',a')+\gamma\Phi(s')$, using that $\Phi(s')$ is constant across $a'$ so it pulls straight through the max. Substituting,
$$\hat Q(s,a)=\mathbb E_{s'}\Big[R(s,a,s')+\underbrace{\gamma\Phi(s')-\Phi(s)}_{=\,F(s,a,s')?}+\gamma\max_{a'}\hat Q(s',a')\Big].$$
There it is. The leftover terms that had to appear are $\gamma\Phi(s')-\Phi(s)$. If I define the shaping reward as
$$F(s,a,s')=\gamma\,\Phi(s')-\Phi(s),$$
then $R(s,a,s')+\gamma\Phi(s')-\Phi(s)=R(s,a,s')+F(s,a,s')=R'(s,a,s')$, and the equation becomes
$$\hat Q(s,a)=\mathbb E_{s'}\Big[R'(s,a,s')+\gamma\max_{a'}\hat Q(s',a')\Big],$$
which is precisely the Bellman optimality equation for $M'$. So the discounted form isn't $\Phi(s')-\Phi(s)$ ‚Äî it's $\gamma\Phi(s')-\Phi(s)$. The $\gamma$ has to sit on the *future* potential, because in the discounted recursion the next state's value already arrives discounted by one step; the potential of where you're going must be discounted to match, while the potential of where you are is subtracted at full weight. Set $\gamma=1$ and it collapses to $\Phi(s')-\Phi(s)$, the telescoping difference the cycle argument pointed at ‚Äî so the two routes to the form agree where they overlap. And this also explains the $+10$ bug above: there I used $\Phi(s')-\Phi(s)$ with $\gamma=0.9$, i.e. the $\gamma=1$ form on a $\gamma<1$ problem; the missing factor of $\gamma$ on $\Phi(s')$ is exactly the discrepancy that let arriving at $s_2$ pay an unrecovered bonus. With the correct $\gamma\Phi(s')-\Phi(s)$ the arrival bonus is $0.9\cdot 10-0=9$ on the way in, but the self-loop on $s_2$ now pays $0.9\cdot 10-10=-1$ every step forever, whose discounted tail from the next step on is $\sum_{t\ge 1}0.9^{t}(-1)=-9$ ‚Äî netting the $+9$ back to exactly $0$. The total shaping return along that path is $-\Phi(s_0)=0$, the same as along *any* path, so no bribe survives. That is the structural difference between a bribe and a potential.

Let me close the sufficiency argument properly rather than wave at it. The Bellman optimality equation has a *unique* solution ‚Äî it's the fixed point of a contraction (for $\gamma<1$) or, in the proper undiscounted case, the unique optimal $Q$. I just showed that $\hat Q(s,a)=Q^*_M(s,a)-\Phi(s)$ *satisfies* the Bellman optimality equation for $M'$. For the undiscounted case I also need the boundary to line up: the absorbing state $s_0$ has $Q^*_M(s_0,a)=0$, and I can take $\Phi(s_0)=0$ without loss of generality because replacing $\Phi$ by $\Phi-\Phi(s_0)$ leaves $F(s,a,s')=\Phi(s')-\Phi(s)$ unchanged when $\gamma=1$. With $\Phi(s_0)=0$, $\hat Q(s_0,a)=0-0=0$, matching $M'$'s boundary. For $\gamma<1$ there is no absorbing boundary condition to check. So $\hat Q$ solves $M'$'s Bellman equation with the right boundary convention where one is needed; by uniqueness it *is* $Q^*_{M'}$. Therefore
$$Q^*_{M'}(s,a)=Q^*_M(s,a)-\Phi(s),\qquad V^*_{M'}(s)=V^*_M(s)-\Phi(s),$$
the second following from $V^*=\max_a Q^*$ and $\Phi(s)$ pulling out of the max. And now the payoff: at each state the shift $-\Phi(s)$ is identical for every action, so
$$\pi^*_{M'}(s)\in\arg\max_a Q^*_{M'}(s,a)=\arg\max_a\big(Q^*_M(s,a)-\Phi(s)\big)=\arg\max_a Q^*_M(s,a),$$
which means every policy optimal in $M'$ is optimal in $M$. For the reverse inclusion ‚Äî every policy optimal in $M$ is optimal in $M'$ ‚Äî I just run the identical argument with the roles of $M$ and $M'$ swapped, using shaping function $-F$ (which is potential-based with potential $-\Phi$). So the two MDPs have *exactly the same set* of optimal policies.

This derivation forced the form on me algebraically, so before I lean on it I want to watch it work end-to-end on numbers, not just believe the symbol-pushing. I take a three-state MDP with $\gamma=0.9$, two actions, transitions and rewards picked arbitrarily (so $Q^*_M$ has no special structure ‚Äî at one state the optimal action even beats the other only narrowly, $5.5$ vs $4.455$), solve it by value iteration, then pick a deliberately lopsided potential $\Phi=(3,-2,1.5)$ ‚Äî nothing like the value function ‚Äî build $R'=R+\gamma\Phi(s')-\Phi(s)$, and solve $M'$. The predicted identity is $Q^*_{M'}(s,a)=Q^*_M(s,a)-\Phi(s)$. Running it, every entry matches to machine precision: e.g. at the narrow state $Q^*_M=(5.5,4.455)$ and $\Phi=-2$, so the prediction is $(7.5,6.455)$, and value iteration on $M'$ returns exactly $(7.5,6.455)$; across all six $(s,a)$ the largest discrepancy is $\sim 2\times10^{-15}$. And the argmax is identical at every state. So the shift really is a clean per-state offset, and a $\Phi$ chosen adversarially far from $V^*$ still didn't move a single optimal action ‚Äî which is the whole promise. The cycle intuition turns out to have been pointing at something stronger than "no money pumps": a potential-based $F$ shifts the entire value landscape by a state-only offset, invisible to the argmax.

Two things fall out that I should record because they sharpen the "why." First, robustness. Re-running the same telescoping on $V^\pi$ instead of $V^*$ ‚Äî nothing in it used optimality; the max just becomes "follow $\pi$" ‚Äî gives $V^\pi_{M'}(s)=V^\pi_M(s)-\Phi(s)$ for *every* policy $\pi$. I want to be sure I didn't quietly reuse optimality somewhere, so I check it on the same three-state MDP with a deliberately *suboptimal* policy $\pi=(b,b,a)$ (greedy would pick differently at the first two states): policy-evaluating $\pi$ in $M$ and in $M'$ and comparing, the values again differ by exactly $-\Phi(s)$ at every state to $\sim 10^{-15}$. Good ‚Äî the identity is genuinely policy-agnostic. So if I learn a merely *near*-optimal $\pi$ in $M'$, with $|V^*_{M'}(s)-V^\pi_{M'}(s)|<\varepsilon$, then subtracting the common $-\Phi(s)$ from both gives $|V^*_M(s)-V^\pi_M(s)|<\varepsilon$ ‚Äî near-optimal in $M'$ transfers to near-optimal in $M$. That matters, because learners don't return exact optima. Second, a cleaner way to see *why* it can't change the optimum: suppose the *original* reward were itself potential-based, $R(s,a,s')=\gamma\Phi(s')-\Phi(s)$. By the telescoping I just verified, every policy then has value $-\Phi(s)$ from state $s$ (with $\Phi(s_0)=0$ in the proper undiscounted case), and every action has action-value $-\Phi(s)$. A pure-potential reward gives the agent no reason to prefer any action over any other ‚Äî it is, by itself, *indifferent to policy*. So adding it to a real $R$ adds no preferences of its own, and a thing with no preferences can't reorder the preferences already in $R$. That is as compact a reason as I'll find for why it leaves the optimum where it was.

So potential-based $F$ is sufficient. Now the question that actually decides whether this is a characterization or just a nice sufficient trick: is it *necessary*? Could there be some other, non-potential $F$ that also happens to preserve the optimal policy? If I knew the dynamics $T$ and reward $R$ intimately I might cook up such an $F$ for that specific MDP. But the practitioner's situation is that they *don't* know $T$ ‚Äî they're shaping precisely because the problem is hard and partly unknown. So the right necessity claim is: if $F$ is *not* potential-based, then there *exists* some proper $T$ and some $R$ for which $F$ ruins the optimum. In other words, a non-potential $F$ cannot be safe *for all* MDPs; you can always be ambushed by a world in which it misleads you. If I want a guarantee with no knowledge of $T,R$, potential-based is forced.

Let me try to build that ambush. There are two ways $F$ can fail to be potential-based, and I'll handle them separately. The cleaner one first: $F$ depends on the *action*. Concretely, suppose there are states $s,s'$ and two actions $a,a'$ with $F(s,a,s')\ne F(s,a',s')$ ‚Äî taking $a$ versus $a'$ to the *same* destination pays differently. (A genuine potential difference $\gamma\Phi(s')-\Phi(s)$ can't tell $a$ from $a'$, so any action-dependence already disqualifies $F$.) Without loss of generality say $F(s,a,s')>F(s,a',s')$, and let $\Delta=F(s,a,s')-F(s,a',s')>0$. Now I get to *choose* the MDP to exploit this. Make both actions go from $s$ straight to $s'$ with probability 1, so they differ in nothing but the shaping they pay. Set the *task* rewards so that the agent's true preference is for $a'$: let $R(s,a,s')=0$ and $R(s,a',s')=\Delta/2$. Then in $M$, $Q_M(s,a)=0$ and $Q_M(s,a')=\Delta/2$, so the optimal task choice is $a'$. But in $M'$ the rewards become $R'(s,a,s')=0+F(s,a,s')=F(s,a,s')$ and $R'(s,a',s')=\Delta/2+F(s,a',s')=\Delta/2+F(s,a,s')-\Delta=F(s,a,s')-\Delta/2$, which is strictly *less* than $R'(s,a,s')$. So under shaping the agent prefers $a$. The optimal choices in $M$ and $M'$ are opposite ‚Äî no policy optimal in $M'$ is optimal in $M$. The action-dependent case is dead.

So I only have to worry about $F$ that depends on $(s,s')$ alone, $F(s,a,s')=F(s,s')$, but is *still* not a difference of potentials. I need to manufacture a world where this $F$ flips the optimum. Pick a reference state $\hat s_0$ ‚Äî take it to be the absorbing $s_0$ if $\gamma=1$, otherwise any fixed state. For $\gamma<1$ a constant offset in the reward can't change the optimal policy, so I may subtract $F(\hat s_0,\hat s_0)$ throughout and assume $F(\hat s_0,\hat s_0)=0$. Now *define* a candidate potential $\Phi(s):=-F(s,\hat s_0)$ ‚Äî the shaping reward for stepping into the reference state, negated. If $F$ *were* potential-based with this $\Phi$, then every $F(s,s')$ would equal $\gamma\Phi(s')-\Phi(s)$. By assumption it is not, so there must be a witnessing pair $s_1,s_2$ where this fails:
$$\gamma\Phi(s_2)-\Phi(s_1)\ne F(s_1,s_2).$$
Spell that out with $\Phi=-F(\cdot,\hat s_0)$: $\gamma\Phi(s_2)-\Phi(s_1)=-\gamma F(s_2,\hat s_0)+F(s_1,\hat s_0)$, so the inequality says $F(s_1,s_2)+\gamma F(s_2,\hat s_0)-F(s_1,\hat s_0)\ne 0$. Name that nonzero number
$$\Delta:=F(s_1,s_2)+\gamma F(s_2,\hat s_0)-F(s_1,\hat s_0)\ne 0.$$
$\Delta$ is exactly the amount by which $F$ fails to telescope along the little path $s_1\to s_2\to\hat s_0$ compared to the direct edge $s_1\to\hat s_0$. That mismatch is the crack I'll pry the optimum open with.

Build $M$ to put a decision right on that crack. From $s_1$ there are two actions. Action $a$ goes $s_1\to\hat s_0$ directly (probability 1). Action $a'$ goes $s_1\to s_2$ (probability 1); from $s_2$, and from $\hat s_0$ when $\gamma<1$, both actions just go to $\hat s_0$. So $a$ is the one-step route to the reference state and $a'$ is the two-step route through $s_2$. For the *task* reward, put a single nonzero reward on the direct edge: $R(s_1,a,\hat s_0)=\Delta/2$ and $R=0$ on everything else. By construction $V^*_M(\hat s_0)=V^*_{M'}(\hat s_0)=0$: in the undiscounted case $\hat s_0$ is absorbing and stops the return, and in the discounted case the only continuation from $\hat s_0$ has zero task reward and the normalized zero shaping reward. Now compute the action values in both MDPs.

In $M$ (task only): $Q_M(s_1,a)=R(s_1,a,\hat s_0)=\Delta/2$, and $Q_M(s_1,a')=0$ (the route through $s_2$ collects no task reward). So $\pi^*_M(s_1)=a$ if $\Delta>0$, and $a'$ if $\Delta<0$.

In $M'$ (task plus shaping), each edge's reward gains its $F$, and reaching $\hat s_0$ ends the accumulation. Along $a$: reward $R(s_1,a,\hat s_0)+F(s_1,\hat s_0)=\Delta/2+F(s_1,\hat s_0)$. Substituting $\Delta=F(s_1,s_2)+\gamma F(s_2,\hat s_0)-F(s_1,\hat s_0)$, that is $F(s_1,s_2)+\gamma F(s_2,\hat s_0)-\Delta/2$. So
$$Q_{M'}(s_1,a)=F(s_1,s_2)+\gamma F(s_2,\hat s_0)-\tfrac{\Delta}{2}.$$
Along $a'$: step $s_1\to s_2$ pays $F(s_1,s_2)$ (no task reward), then $s_2\to\hat s_0$ pays $\gamma F(s_2,\hat s_0)$ (discounted one step, and again no task reward), and then absorption. So
$$Q_{M'}(s_1,a')=F(s_1,s_2)+\gamma F(s_2,\hat s_0).$$
Compare: $Q_{M'}(s_1,a')-Q_{M'}(s_1,a)=\Delta/2$. So in $M'$ the preference is the *reverse* of $M$: $\pi^*_{M'}(s_1)=a'$ if $\Delta>0$ and $a$ if $\Delta<0$. Either way, whichever action is optimal in $M$ is *suboptimal* in $M'$ and vice versa.

The construction has enough moving parts ($\hat s_0$ normalization, the discounted middle edge, the sign of $\Delta$) that I don't fully trust the algebra until I've run one. Let me instantiate it. Take $\gamma=0.9$, states $\{s_1,s_2,\hat s_0\}=\{0,1,2\}$, and an action-independent $F$ that is *not* a potential difference: $F(s_1,\hat s_0)=2$, $F(s_1,s_2)=0$, $F(s_2,\hat s_0)=0$, with $F(\hat s_0,\hat s_0)=0$. Then $\Delta=F(s_1,s_2)+\gamma F(s_2,\hat s_0)-F(s_1,\hat s_0)=0+0-2=-2\ne0$, so this $F$ is indeed non-potential. With $\Delta<0$ the direct task reward is $R(s_1,a,\hat s_0)=\Delta/2=-1$. Solving $M$ by value iteration: $Q_M(s_1,a)=-1$ (the direct edge costs $1$), $Q_M(s_1,a')=0$, so $M$ prefers $a'$ ‚Äî go the long way and avoid the cost, as it should. Now shape and re-solve $M'$: $Q_{M'}(s_1,a)=-1+F(s_1,\hat s_0)=-1+2=+1$, while $Q_{M'}(s_1,a')=F(s_1,s_2)+\gamma F(s_2,\hat s_0)=0$. Value iteration confirms $(+1,0)$, so $M'$ prefers $a$. The optimum flipped from $a'$ to $a$ ‚Äî exactly opposite, just as the $\Delta/2$ gap predicted. The $+2$ bonus for stepping directly into $\hat s_0$, with nothing equal-and-discounted to offset it on the two-step route, is the whole mechanism. So a non-potential $F$ ‚Äî whether its non-potential-ness comes from action-dependence or from a telescoping mismatch in $(s,s')$ ‚Äî can always be ambushed; I've now seen both the symbolic argument and a worked instance agree.

Putting the two directions together, I have a clean characterization, not just a recipe. Fix $S,A,\gamma$ and a shaping reward $F$. Then $F$ guarantees that every optimal policy of $M'=(S,A,T,\gamma,R+F)$ is optimal in $M=(S,A,T,\gamma,R)$ for *all* proper $T$ and all $R$ **if and only if** $F$ is potential-based: there exists $\Phi:S\to\mathbb R$ with $F(s,a,s')=\gamma\Phi(s')-\Phi(s)$ (with $S\setminus\{s_0\}=S$ when $\gamma<1$). Sufficiency: $\Phi$ shifts $Q^*$ by the state-only offset $-\Phi(s)$, which the per-state argmax ignores, so the optimal-policy sets coincide exactly. Necessity: if $F$ is not of this form, there is a world in which it inverts a decision. This is the multi-step analogue of the old utility-theory fact that under uncertainty only positive linear transforms of utility preserve the optimal single-step choice; here, for additive sequential shaping, the safe terms are exactly the discounted differences of a state potential, with constant reward offsets in discounted continuing problems appearing as the constant-potential special case.

One more thing to settle so this is actually usable: which $\Phi$ should I pick? Invariance holds for *any* $\Phi$, so I'm free to choose $\Phi$ to make learning fast. The identity $V^*_{M'}(s)=V^*_M(s)-\Phi(s)$ tells me that if I could take $\Phi(s)=V^*_M(s)$, then $V^*_{M'}\equiv0$ ‚Äî the value function in the shaped problem is flat, the easiest possible thing to learn, and all that's left is to pin down the $Q$-values. Of course if I knew $V^*$ I wouldn't be learning at all; the point is that a *crude estimate* of $V^*$ is a good $\Phi$, and ‚Äî because invariance is guaranteed regardless ‚Äî I lose nothing by being wrong about it. A $\Phi$ that is far from $V^*$ still can't corrupt the optimum; at worst it just helps less. So the design rule is: let $\Phi$ be your best cheap guess at the value-to-go.

That guess is where domain knowledge enters. On an undiscounted minimum-cost-to-goal grid with $-1$ per step and 80/20 stochastic moves, take a step toward the goal and you make about $0.8$ of a Manhattan-step of progress per timestep (80% you move as intended; the 20% random move is, away from walls, as likely toward as away, so it averages out). So the expected steps-to-goal from $s$ is about $\mathrm{manhattan}(s,\text{goal})/0.8$, and since each step costs $1$, I'd guess $V^*(s)\approx-\mathrm{manhattan}(s,\text{goal})/0.8$. That "averages out" hand-wave makes me want to check it, since near walls the random move is *not* symmetric. So I solve a $10\times10$ grid exactly by value iteration and compare. The fit is close: at the start corner $(0,0)$ the exact $V^*=-22.0$ against the guess $-22.5$; at $(5,5)$, $-10.08$ vs $-10.0$; at $(8,8)$, $-2.58$ vs $-2.5$; the worst ratio I see is about $1.04$ (the guess slightly *over*-counts distance because it ignores that a stray move can occasionally help). A few percent off is completely fine here ‚Äî invariance doesn't depend on $\Phi$ being right, only learning speed does ‚Äî so this is a good cheap potential. That gives $\Phi(s)=-\mathrm{manhattan}(s,\text{goal})/0.8$, and through $F=\Phi(s')-\Phi(s)$ it rewards motion toward the goal ‚Äî but now *symmetrically*, because moving away pays exactly the negative of moving back, so the bicycle's tiny-circle money pump is impossible. The cure is structural, not a patch. For subgoal problems the same reasoning fixes the *magnitudes* the earlier hacks got wrong: estimate the total time-to-go $t$, and if having reached $n$ of, say, $5$ subgoals leaves roughly $((5-n-0.5)/5)\,t$ to go, set $\Phi(s)=-((5-n_s-0.5)/5)\,t$; then $\Phi$ jumps by $t/5$ each time a subgoal is crossed, so $F=\Phi(s')-\Phi(s)$ hands out exactly $t/5$ per subgoal ‚Äî a principled subgoal bonus rather than a guessed one, and one that can't be farmed because crossing a subgoal backward refunds it.

So the whole chain, start to finish: the task lives in $R$; shaping adds $F$ and silently optimizes $R+F$; the observed bugs are all positive-reward cycles that let the agent farm $F$ instead of doing the task; demanding that undiscounted cycles sum to zero points to a difference of a state potential; matching that to the *discounted* Bellman recursion fixes the form as $F(s,a,s')=\gamma\Phi(s')-\Phi(s)$; this shifts $Q^*$ by the action-independent $-\Phi(s)$, so the optimal-policy set is exactly preserved (and near-optimal too); and absent knowledge of $T,R$ no other $F$ can promise this, because any non-potential $F$ can be made to flip a decision. The landing result:

> **Theorem (potential-based shaping).** Fix $S,A,\gamma$ and a bounded shaping reward $F:S\times A\times S\to\mathbb R$. Say $F$ is *potential-based* if there is a (bounded) $\Phi:S\to\mathbb R$ with
> $$F(s,a,s')=\gamma\,\Phi(s')-\Phi(s)\qquad\text{for all }s\in S\setminus\{s_0\},\,a\in A,\,s'\in S$$
> (with $S\setminus\{s_0\}=S$ when $\gamma<1$). Then $F$ being potential-based is **necessary and sufficient** to guarantee consistency of the optimal policy between $M'=(S,A,T,\gamma,R+F)$ and $M=(S,A,T,\gamma,R)$:
> *(Sufficiency)* if $F$ is potential-based then $Q^*_{M'}(s,a)=Q^*_M(s,a)-\Phi(s)$, $V^*_{M'}(s)=V^*_M(s)-\Phi(s)$, and the sets of optimal policies of $M$ and $M'$ are identical (and near-optimal policies are preserved within the same $\varepsilon$).
> *(Necessity)* if $F$ is not potential-based, there exist proper $T$ and a reward $R$ such that no optimal policy of $M'$ is optimal in $M$.

And in code it's a one-line slot ‚Äî given any cheap potential $\Phi$, the shaping reward is just the discounted potential difference, dropped into the ordinary learning loop:

```python
def shaping_reward(s, a, s_next, gamma, Phi):
    # F(s,a,s') = gamma * Phi(s') - Phi(s):
    #   a discounted difference of a state-only potential, so the shaped return
    #   differs by the start-state offset -Phi(s), leaving argmax_a unchanged.
    return gamma * Phi(s_next) - Phi(s)

def manhattan_potential(goal, p_intended=0.8):
    # crude estimate of V*(s) for a -1/step minimum-cost-to-goal grid:
    #   ~p_intended Manhattan-steps of progress per timestep, each costing 1.
    def Phi(s):
        return -manhattan(s, goal) / p_intended
    return Phi
```
