I want to understand why a first-order theory cannot hold on to its intended infinite size. The obvious examples already make me uneasy. I can write sentences saying there are at least `1`, at least `2`, at least `3`, and so on, and together they force infinitude. But each sentence only talks about finitely many variables. Nothing in that list seems able to say "this infinity, not that one."

If I only want another model of the same sentences, compactness and Henkin constructions explain part of the story: first-order syntax is countable when the language is countable, so a completed Henkin theory can be made with countably many constants and then quotiented into a countable model. But that feels too detached from the structure I may already have. Suppose I am staring at a large model `M`. I do not merely want a new model somewhere else; I want to cut a small piece out of `M` and have first-order truth remain unchanged inside that piece. That is a sharper test of what first-order logic can and cannot see.

Start with a set `A` of elements I insist on keeping. The naive move is to close `A` under the function symbols of the language. That gives a substructure. If the language has a binary function `+`, then I must keep sums; if it has constants, I must keep them; if it has compositions of functions, closure handles the terms. Atomic formulas then make sense in the smaller structure because the functions and relations are inherited.

But existential formulas are exactly where this breaks. Maybe `M` satisfies `exists y phi(y,a)` for a parameter tuple `a` already in my small set, and every such witness lies outside the small set I have built. Then the smaller structure says the existential sentence is false while the big structure says it is true. So closure under the original functions is not the right closure. The missing operations are not just algebraic operations named in the language. They are witness choices for first-order formulas.

So for every formula `phi(y,x_1,...,x_n)`, I can make a choice function in the background of `M`: given a tuple `a` from `M`, if some `b in M` satisfies `phi(b,a)`, choose one such `b`; if no witness exists, choose anything harmless. I do not need this function to be definable in the original language. I only need it as a bookkeeping device while building the small subset. The point is brutally simple: if I close my set under these witness choices, then every existential fact over parameters from the set already has a witness in the set.

Now the size question becomes the whole theorem. How many such choices am I adding? A first-order language `L` has only `max(|L|, aleph_0)` formulas, because formulas are finite strings over the symbols of `L` plus logical notation and variables. For a countable language, that is countably many formulas. For a current set `A_i` of size at most `kappa = max(|A|, |L|, aleph_0)`, there are only `kappa` many finite tuples from it: `kappa^{<omega}=kappa` for infinite `kappa`. Applying all witness functions to all finite tuples therefore adds at most `kappa` new elements. Closing under the original functions does not change the bound, because there are at most `|L|` function symbols of finite arity and at most `kappa` finite tuples.

This suggests an omega-stage construction. Let `A_0` contain the original parameter set and the interpretations of the constant symbols. Given `A_i`, form `A_{i+1}` by adding all values of all original function symbols on tuples from `A_i`, and all chosen witnesses for all formulas with parameters from `A_i`. Each stage has size at most `kappa`. The union `N = union_i A_i` still has size at most `kappa`, because it is a countable union of sets of size at most `kappa`.

Why omega many stages? Because adding witnesses creates new parameters, and those new parameters create new existential questions. One round is not enough. But formulas have finite tuples of parameters, and any finite tuple from the union appears together in some stage. Once it appears, the next stage contains the witness for any existential formula over that tuple. That is exactly the closure property I need.

I also need `N` to be a substructure, not just a set. The construction includes the original function-symbol values stage by stage. If `f` is an `n`-ary function symbol and `a_1,...,a_n` are in `N`, all of them occur in some common `A_i`; then `f^M(a_1,...,a_n)` lies in `A_{i+1}`, hence in `N`. Constants were included at the start. Relations are just restricted from `M`. So `N` is an `L`-substructure of `M`.

Now I check elementarity. The Tarski-Vaught test says it is enough to handle existential witnesses as seen in `M`. Take any formula `phi(y,x_1,...,x_n)` and parameters `a_1,...,a_n` from `N`. Suppose `M |= exists y phi(y,a)`. The tuple `a` is finite, so all its entries lie in some `A_i`. The witness-choice function for `phi` applied to `a` was included in `A_{i+1}`. By how it was chosen, it gives some `b` with `M |= phi(b,a)`, and `b` lies in `N`. That is the Tarski-Vaught condition. Therefore `N prec M`.

This is the whole mechanism: first-order truth is preserved because every existential need over the small set is anticipated by a chosen witness, and size is controlled because first-order syntax and finite tuples do not generate more than `max(|A|, |L|, aleph_0)` obligations at each countable stage.

In a countable language, if `A` is countable, then `kappa=aleph_0`, so the elementary substructure `N` is countable. If `M` is uncountable, the smaller structure is not isomorphic to `M`, yet no first-order formula with parameters from `N` can separate `N` from `M`. That is exactly the expressive limitation. First-order logic can describe all the local finite-parameter witness patterns it asks for, but once those witnesses are supplied inside a countable closure, it cannot force the surrounding uncountable domain to remain.

This also dissolves the usual worry around set theory. A countable elementary model can satisfy sentences saying that some set is uncountable, because "there is no bijection from omega onto this set" is evaluated using functions that live inside that model. Externally, the model may be countable; internally, it may lack the bijection that the outside observer can see. The small model is not claiming to have the outside universe's full collection of functions. It is preserving first-order truth with respect to its own domain.

So the theorem lands as a closure theorem, not as a recombination theorem. I begin with the elements I care about, add the witnesses that first-order formulas can demand, repeat until finite parameter tuples are covered, and the result is an elementary submodel whose cardinality is bounded by the syntax and the starting set.

Let `L` be a first-order language, `M` an `L`-structure, and `A subseteq M`. Put `kappa = max(|A|, |L|, aleph_0)`. For every formula `phi(y,x_1,...,x_n)`, choose a function `F_phi:M^n -> M` such that, whenever `M |= exists y phi(y,a)`, `M |= phi(F_phi(a),a)`. Define `A_0` to be `A` together with all interpretations of constants of `L`. Given `A_i`, let `A_{i+1}` contain `A_i`, all values of all function symbols of `L` on tuples from `A_i`, and all values `F_phi(a)` for formulas `phi(y,x)` and finite tuples `a` from `A_i`. Let `N = union_{i<omega} A_i`.

Then `|N| <= kappa`, `N` is closed under the functions of `L`, and `A subseteq N`. If `M |= exists y phi(y,a)` with `a` from `N`, the finite tuple `a` lies in some `A_i`, so `F_phi(a)` lies in `A_{i+1} subseteq N` and witnesses `phi` in `M`. By Tarski-Vaught, the substructure on domain `N` is elementary in `M`. In particular, every infinite structure in a countable language has a countable elementary substructure containing any prescribed countable set.
