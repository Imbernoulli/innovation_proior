"this decoding algorithm is clearly suboptimal" — Viterbi did NOT realize it was ML/optimum. Forney later saw the path-merging property = a "trellis", and that the VA is exactly the shortest-path recursion on the trellis = optimum. Omura (1968) saw it as forward dynamic programming.
- 1999 oral history quote (via Forney): "came out of my teaching... I found information theory difficult to teach, so I started developing some tools... wrote the first paper in March '66... At one point I was actually discouraged from publishing the algorithm details. Fortunately, one of the reviewers, Jim Massey, encouraged me to include the algorithm. Nobody thought that it had any potential for practical value."

So the trace backbone: I am Viterbi, teaching info theory at UCLA ~1965, struggling to explain why convolutional codes beat block codes. I want to PROVE that superiority with a clean error bound. The decoding algorithm is a *device for the proof* — an analyzable suboptimal decoder whose error probability I can union-bound. The DP/survivor recursion drops out of wanting a decoder simple enough to analyze. I do NOT claim optimality (call it "asymptotically optimum"); the ML/DP recognition is Forney/Omura's, so it stays out of the in-frame trace (hindsight).

## Primary paper (refs/viterbi-1967-error-bounds-convolutional) — technical core
- Encoder (Sec II): q-ary convolutional/tree code. K-stage shift register, v inner-product computers over GF(q), generator matrix G. After each data symbol shifts in, produces v channel symbols. Tree code: q branches per node, v channel symbols each. Constraint length N = Kv (channel symbols), K = branch constraint length. L = tree length.
- Property A) (the key structural fact): "Two divergent paths of the tree code will converge (i.e. produce the same channel symbols) after the data symbols corresponding to the two paths have been identical for K consecutive branches." => the path-merging that makes the trellis finite-state: only the last K-1 data symbols matter for the future.
- Sec III lower bound: "magic genie" tells decoder all symbols except m consecutive branches; q^m alternate paths diverge at branch j+1 and reconverge at j+m+K; totally distinct over at most m+K-1 branches; reduces to a block-code lower bound (sphere-packing / SGB low-rate). Gives E_L(R).
- Sec IV — THE ALGORITHM: "a new probabilistic nonsequential decoding algorithm." Decodes L-branch tree by L repetitions of one basic step. Step 1: consider all q^K paths for first K branches, compute q^K likelihoods. Compare the q paths differing only in the first symbol (0,a2,...aK),(1,a2,...aK),...,(q-1,a2,...aK) for each of q^{K-1} vectors (a2,...,aK). Keep the max in each group = the SURVIVOR. Only q^{K-1} survivors kept; rest discarded. At each step: extend each survivor by q branches (population x q), then compare in groups of q and keep one survivor per (a2,...,aK) subsequence (population /q). Invariant: exactly one survivor per state (the K-1 trailing data symbols). Recursion is core.
- "While this decoding algorithm is clearly suboptimal, the optimal being a comparison of the likelihood functions of all q^L paths..." Then Sec V random-coding upper bound matches Yudkin -> "asymptotically optimum for R > R0".
- Closing: "Although this algorithm is rendered impractical by the excessive storage requirements, it contributes to a general understanding of convolutional codes and sequential decoding through its simplicity of mechanization and analysis."

## Antecedents (lineage)
- Elias 1955 ("Coding for noisy channels") — invented convolutional/tree codes; conjecture they beat block codes of same length.
- Yudkin 1964 (thesis) — first quantitative proof of an upper bound on error probability of optimal convolutional code, achieved by Fano sequential decoding.
- Fano 1963 ("A heuristic discussion of probabilistic decoding") — Fano sequential decoding algorithm; Wozencraft & Reiffen 1961 (Sequential Decoding) — original sequential decoding. These EXPLORE the code tree, backtracking; number of computations is a random variable (Pareto), unbounded mean for R>R0; works above R0 only.
- Gallager 1965 ("A simple derivation of the coding theorem") — random-coding union bound E_0(rho), used to bound the per-comparison error probability.
- Shannon-Gallager-Berlekamp — low-rate (expurgated/SGB) lower bound.
- Reiffen 1960 — proved property D) (independence of totally-distinct paths) within first K branches.
- (Hindsight, kept OUT of in-frame trace): Bellman dynamic programming; Forney trellis + ML proof (1967/1973); Omura DP (1968).

## The re-derivation (in-frame logic)
1. Teaching pain: I want to show convolutional codes beat block codes for a given decoding complexity. Sequential decoding (Fano/Wozencraft) is the known decoder but its analysis is via a *tree* and its computation is a random variable — messy to teach and its error bound (Yudkin) is hard. I want a decoder I can ANALYZE cleanly to get the error exponent.
2. The structural fact: in a convolutional code only the last K-1 data symbols (shift-register contents) determine the next output. So two paths that agree in the last K symbols are indistinguishable from then on — they MERGE (property A). The tree is really a finite-state graph that folds back on itself: a trellis.
3. The combinatorial wall: optimal (ML) decoding compares all q^L paths — exponential in tree length L, hopeless to analyze and to run.
4. The leap: because merged paths share an identical future, if two paths arrive at the same state (same last K-1 symbols), only the better one can ever be part of the optimal continuation — its worse competitor can be discarded forever. So at each branch level, per state, keep ONE survivor (the highest-likelihood path into that state). q^{K-1} states, one survivor each. Recursion: M_K(state) = max over predecessors {M_{K-1}(pred) + branch metric}.
5. Why this is exactly enough and not more: the discarded path can never beat the survivor on any common suffix, so discarding loses nothing for the per-state best — analyzable, and (proven later via union bound) asymptotically optimal.
6. Log-likelihood -> additive branch metric: ln L = sum of branch metrics; for BSC, branch metric reduces to Hamming distance between received and hypothesized branch symbols (so max-likelihood = min Hamming distance). Add-compare-select.
7. Error analysis (why I built it): correct path eliminated only if at some merge it loses to an adversary that diverged K+l branches back and is totally distinct. Union bound over adversaries -> per-step error <= block-code random-coding bound; sum over L steps -> P_E < [L(q-1)/(1-q^{-eps/R})] exp(-N E(R)), matching Yudkin's upper bound, and the computation per branch is ~q^K independent of rate (unlike sequential decoding's unbounded mean).
8. Land on code: convolutional encoder + Viterbi decoder (ACS + traceback), grounded in CommPy structure: next_state_table, output_table, path metrics, branch metric = Hamming distance (hard), survivor paths, traceback. Self-contained runnable.

## Design decisions -> why
- Keep one survivor per state (not per path): merged paths share future => dominated path can be dropped with no loss to the best path reaching that state. This is the whole saving (q^L -> q^{K-1} per level).
- Use log-likelihood (additive metrics): products of per-branch likelihoods underflow and aren't additive; logs make the path metric a SUM, so the recursion is add-compare-select.
- BSC -> Hamming distance: ln p(y|x) for BSC is affine in the number of disagreements, so maximizing likelihood = minimizing Hamming distance; integer metric, trivial to teach and to compute.
- Compare in groups of q sharing the same (a2..aK): those are exactly the paths entering the same state; comparing them is comparing the survivors into one state.
- Traceback / store path history: need the decoded SEQUENCE, not just the best final metric; store the surviving predecessor at each (state,time) and walk back. Forward-only (store history) vs backward pass — forward-only suits streaming.
- Termination with K-1 zeros: drive encoder back to all-zero state so the final state is known => single surviving path at the end.
- tb_depth (truncation): survivors merge within a few constraint lengths, so a finite traceback window (~5x memory) gives the same decision as full traceback — enables streaming and bounded memory.
