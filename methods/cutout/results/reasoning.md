I start from the fact that the usual regularizer is failing in a very specific place. Dropout is excellent when a dense hidden unit can be made unreliable and every other unit has to stand on its own, but a convolutional layer is not like that. Its parameters are shared, its local neighborhoods are correlated, and neighboring pixels or activations often carry almost the same evidence. If I remove one point, the nearby points still carry the missing signal. So in a convolutional stack, pointwise dropout is not really removing visual evidence; it is mostly adding noise and asking filters to be robust to that noise.

The feature-map variants point in the right direction but do not quite solve the problem I care about. Dropping a whole feature map is stronger than dropping scattered positions, and targeting the most active responses is also plausible, but these operations happen after the image has already been transformed into multiple channels. Evidence removed in one map may still exist in another. The model sees an inconsistent representation rather than an image where a cue is actually absent. That explains why these methods can help in isolation but lose their edge when batch normalization and ordinary augmentation are already in the system.

So I want the disappearance to happen before the feature maps are made. If I remove visual evidence at the input, then every downstream map has to live with the same missing evidence. There is no parallel channel that quietly preserves it. The network can only infer the missing part from the remaining context, which is exactly the behavior I want if the real failure mode is over-reliance on a small set of local cues.

The next question is how the input should be damaged. Scattered pixel corruption is too easy because neighboring pixels repair it statistically. Denoising autoencoders already show that input corruption can teach useful features, and context encoders show that a larger missing region forces more global understanding than isolated pixel noise. I do not need the reconstruction task, but I can keep the lesson: the corruption has to be spatially structured enough that local redundancy cannot simply undo it.

That gives the basic operation: during training, choose a location in the image and set one connected region to zero. A square is the conservative shape because the amount removed matters more than the exact outline, and a square leaves only one real knob, its length. The label stays unchanged because the training example is meant to be an occluded view of the same object, not a mixture with a different class.

I should not force every removed square to sit fully inside the image. That sounds cleaner at first, but it means every training image loses the same full-sized patch. The reference implementation instead samples the center uniformly from valid image coordinates and clips the patch at the image boundary. A near-edge center therefore removes only part of the nominal square, leaving a large fraction of the image visible. The model sees a range from strongly occluded to lightly occluded examples. The fully-contained alternative with application probability 0.5 works for the same reason: some training examples remain unmodified.

The exact mask arithmetic has one small case distinction that I need to keep faithful to the released code. For image height h and width w, sample y from 0,...,h-1 and x from 0,...,w-1. Then set y1=max(y - L//2, 0), y2=min(y + L//2, h), x1=max(x - L//2, 0), and x2=min(x + L//2, w), and zero mask[y1:y2, x1:x2]. Because Python slices exclude the upper endpoint, an unclipped even L removes exactly L by L pixels. If someone passes an odd L, the canonical code removes 2*(L//2) pixels per side, not L; the reported tuned lengths are even, so this edge case does not affect the stated settings. Near borders the actual removed area is (y2-y1)(x2-x1), which is deliberately smaller than the nominal square.

The fill value should be zero only after per-channel normalization. In the reference training script, tensor conversion and normalization happen before the mask transform, so multiplying by a zero mask replaces the removed pixels by the normalized dataset mean. That avoids creating a hard black patch in raw pixel space and keeps batch statistics from being distorted more than necessary. It also injects no new class information, unlike random noise or a patch from another image.

There is no test-time scaling. Dropout needs scaling because it is approximating the expectation of a multiplicative random network. Here the operation is data augmentation: training sees modified images and evaluation sees ordinary images, just like crops and flips. The network, loss, optimizer, labels, and learning-rate schedule do not need to change.

The early targeted version is worth rejecting explicitly. If I store the maximally activated feature map for each image, upsample it, threshold it, and hide that region in the next epoch, I can force the model off its favorite cue. But that adds a saliency bookkeeping loop, and random fixed-size regions work just as well. Once the random version matches the targeted version, the saliency machinery is not part of the method.

The final method is therefore a tiny input transform: build an all-ones h by w mask, sample one or more centers uniformly, zero the clipped square around each center, expand the mask across channels, and multiply the normalized image by it. It is applied only during training, usually with one hole, with the length selected on validation data. For the main reported settings, the lengths are 16 for CIFAR-10, 8 for CIFAR-100, and 20 for SVHN, with the usual CIFAR schedule multiplying the learning rate by 0.2 at epochs 60, 120, and 160. The reason it works is not a new loss or architecture; it is the placement and structure of the missing evidence.
