Start with the classical picture: polynomial equations cut out a set of solutions. If `I` is an ideal in `k[x_1,...,x_n]`, the variety is the set of points `a in k^n` where every polynomial in `I` vanishes. Then the coordinate ring `k[x_1,...,x_n]/I` is treated as the ring of polynomial functions on that set.

That order of construction is fragile. It assumes the base field supplies enough visible points, usually via algebraic closedness. It identifies functions that agree on all ordinary points, so nilpotent information disappears. It also makes arithmetic look external: reducing an equation modulo `p`, passing to characteristic zero, or comparing fibers over different primes feels like changing the universe rather than moving inside one geometric object.

The decisive reversal is to start from the ring. A commutative ring `A` already contains the functions, so the space should be the one naturally probed by those functions. A point should be a place where functions can be tested for vanishing, but not only at maximal ideals. Prime ideals are the correct level of generality because quotienting by a prime gives an integral domain, hence a coherent "irreducible" residue of the ring. Maximal ideals are closed classical points in good cases; nonmaximal primes are generic points of irreducible pieces and encode specialization relations.

Thus `Spec A` is not a decorative construction placed on top of a variety. It is the replacement for the variety in the affine case. Its points are all prime ideals. The topology records algebraic specialization: closed sets are `V(I)`, the primes containing an ideal `I`. A generic point sits inside the closure relation in exactly the way an irreducible subvariety should. This makes irreducible subobjects points of the same space, rather than external subsets one has to name separately.

But the topological space alone is still not enough. `Spec A` and `Spec A_red` have the same prime ideals, so the same underlying topological space. Nilpotents live in the functions, not in the prime set. This is why the scheme is a locally ringed space, not merely a spectrum as a set. The structure sheaf assigns local functions, and the stalk at a prime `p` is `A_p`, a local ring. If `A = k[epsilon]/(epsilon^2)`, the underlying space has one point, but its local ring contains the nonzero nilpotent `epsilon`. The scheme sees an infinitesimal thickening that a solution-set geometry cannot see.

The same definition also absorbs arithmetic. Consider `Spec Z`. Its points are `(0)` and `(p)` for prime numbers `p`. The generic point `(0)` carries characteristic zero behavior; the closed point `(p)` carries characteristic `p`. A scheme over `Spec Z` is therefore a single geometric object whose fibers include the generic fiber over `Q` and the reductions modulo every prime `p`. Arithmetic variation is not translated into a family of unrelated equations; it is literally a morphism to the base scheme `Spec Z`.

Families become the same idea one level up. A morphism `X -> S` is a family parameterized by the scheme `S`. The fiber over a point `s` is obtained by base change to `Spec k(s)`, where `k(s)` is the residue field at that point. Since points of `S` include closed points, generic points, and specialization relations, the language sees ordinary members, generic behavior, special fibers, and degeneration in one object. The mechanism is the same whether `S` is a curve, a parameter space, or `Spec Z`.

Gluing is what turns this affine insight into a global theory. Classical varieties can also be glued from affine patches, but schemes glue the whole locally ringed space: both the prime spectrum topology and the sheaf of local rings. A scheme is covered by open pieces `Spec A_i`, with the pieces identified along open subschemes. Because the local data is ringed, the gluing preserves the function rings and the local behavior at each prime. The object is not a subset of some ambient affine space; it is built intrinsically out of compatible affine spectra.

This is the fundamental reconstruction. The old question is "what are the solutions of these equations?" The scheme question is "what geometric object is represented by this ring of functions and all of its prime tests?" Passing from solutions to functions is already a duality: affine schemes reverse arrows, so a ring map `A -> B` gives a morphism `Spec B -> Spec A`. Geometry is recovered from algebra contravariantly.

The word "all" in all prime ideals is load-bearing. If one keeps only maximal ideals, one mostly gets closed points. If one keeps all primes, one gets generic points, irreducible components, embedded specialization, and arithmetic primes. If one keeps the sheaf of local rings, one gets nilpotents and infinitesimal neighborhoods. If one allows gluing, one gets global spaces not tied to one coordinate ring. These are not four extra features bolted onto classical varieties; they are consequences of defining the object as a locally ringed space locally modeled on `Spec A`.

So the distinctive Grothendieck move is not merely a broader definition. It changes the ontology of algebraic geometry. Equations no longer primarily define a set of visible solutions. They define a function ring; the function ring defines a prime spectrum with local rings; these affine spectra glue into schemes; and morphisms, fibers, and families are handled inside that same category. Nilpotents, arithmetic fibers, and families become forms of geometry because the definition was built to keep the algebra that solution sets throw away.
