We want a single deep network with one shared encoder feeding several heads — a depth regressor and a semantic classifier, or a fine-class head and a coarse-class head — trained together, because the shared representation is cheaper at test time and because the auxiliary task ought to regularise the features and help the task we actually care about. To train such a model we must collapse the per-task losses into one scalar, and the universal recipe is the weighted sum $L_{\text{total}} = \sum_i w_i L_i$. The weights are the problem. The losses are not even in the same units — depth error is in squared meters, cross-entropy is dimensionless nats, the instance-vector loss is in pixels — so their raw magnitudes differ by orders of magnitude, and the right relative weight depends on those arbitrary units *and* on how noisy each task's labels are. Sweeping the weight of a two-task model across its range produces the same picture every time: each task is best somewhere in the interior rather than at its single-task extreme, the two interiors do not coincide, and there is a narrow compromise band where both tasks beat their single-task baselines, flanked by regions where pushing the weight further wrecks one task. Locating that band costs a full training run per grid point — days each — and the search blows up combinatorially as tasks are added. So for three or more tasks one is stuck either guessing uniform weights, which lands outside the good band, or burning a week on grid search. The obvious fix — make the $w_i$ themselves trainable and let SGD set them — fails on inspection: $\partial L_{\text{total}}/\partial w_i = L_i \ge 0$, so gradient descent pushes every weight down, nothing pushes back, and the global optimum is $w_1 = w_2 = 0$, where the total loss is zero and the network has learned nothing. Any scheme that learns the weights must carry a term that resists shrinking them — a cost for declaring a task unimportant — and that coupling has to be principled, not a free knob.

I propose uncertainty weighting — homoscedastic, task-dependent uncertainty weighting — which derives that coupling from the observation that most of these losses are negative log-likelihoods in disguise, each carrying a per-task observation-noise scale $\sigma_i$ that we can simply learn. The depth $L_2$ loss $\|y - f^W(x)\|^2$ is, up to constants, the NLL of a Gaussian observation model $p(y \mid f^W(x)) = \mathcal{N}(f^W(x), \sigma^2)$, since $-\log\mathcal{N}(y;\mu,\sigma^2) = \tfrac{1}{2\sigma^2}\|y-\mu\|^2 + \tfrac{1}{2}\log\sigma^2 + \text{const}$. In single-task training $\sigma$ is a fixed constant folded into the learning rate and never written down — for one task a constant noise level is just an overall rescale. But that constant $\sigma$ is exactly a per-task scale: a task with noisier labels, or a loss on a larger numerical scale, has a bigger $\sigma$, and we can make it a free parameter. We want one constant $\sigma_i$ per task — the homoscedastic, task-dependent version rather than the input-dependent heteroscedastic $\sigma(x)$ of Nix & Weigend (1994) — because what varies across this problem is not the noise from pixel to pixel but the noise and scale from task to task. Writing the joint likelihood for two conditionally independent regression outputs, $p(y_1, y_2 \mid f^W(x)) = \mathcal{N}(y_1; f^W(x), \sigma_1^2)\,\mathcal{N}(y_2; f^W(x), \sigma_2^2)$, and minimising its negative log gives

$$L(W, \sigma_1, \sigma_2) = \frac{1}{2\sigma_1^2} L_1(W) + \frac{1}{2\sigma_2^2} L_2(W) + \log\sigma_1 + \log\sigma_2,$$

with $L_i(W) = \|y_i - f^W(x)\|^2$ the bare per-task losses. Two things fall out. The coefficient on each task is $1/(2\sigma_i^2)$ — an inverse-variance weighting: the noisier or larger-scale a task, the less it dominates the shared gradient. That is the statistically correct way to fuse measurements of differing precision, not a heuristic. And there is a $+\log\sigma_i$ term, which is precisely the anti-collapse penalty the bare learnable weighted sum lacked: if the optimiser tries to send $\sigma_i \to \infty$ to zero out the $1/(2\sigma_i^2)$ coefficient and make a task free, $\log\sigma_i$ shoots to $+\infty$ and punishes it. The same probabilistic model that weights the tasks also keeps the weights finite. Setting $\partial L/\partial\sigma_i = -L_i(W)/\sigma_i^3 + 1/\sigma_i = 0$ gives the fixed point $\sigma_i^2 = L_i(W)$: the learned variance tracks the current value of that task's loss, so a poorly-fit task is assigned a big $\sigma$ (low weight) and a well-fit task a small $\sigma$ (high weight), and neither term lets it run to zero or infinity.

To optimise $\sigma$ stably I reparameterise to the log-variance $s := \log\sigma^2$, since $\sigma$ appears as $1/\sigma^2$ (a divide-by-zero risk if it wanders to zero) and is constrained positive (awkward for unconstrained SGD). Then $1/\sigma^2 = \exp(-s)$, which is always strictly positive, and $s$ ranges over all of $\mathbb{R}$, so SGD can step it freely; the precision weight is manifestly positive for any real $s$. The per-task regression contribution becomes $\tfrac{1}{2}\exp(-s_i)L_i(W) + \tfrac{1}{2}s_i$.

The classifier needs its own treatment, because its loss is cross-entropy $-\log\text{Softmax}(f^W(x))_c$, not a squared error, and we cannot paste a Gaussian $\sigma$ in front of it. The softmax is a Boltzmann distribution over classes with the logits as energies, and the natural scale knob is temperature: scale the logits by $1/\sigma^2$ before the softmax, $p(y \mid f^W(x), \sigma) = \text{Softmax}\big((1/\sigma^2) f^W(x)\big)$, so $\sigma^2$ is the temperature — large $\sigma^2$ flattens toward uniform (high uncertainty), small $\sigma^2$ sharpens toward one-hot (high confidence). This gives $\sigma$ the same uncertainty meaning across regression and classification. Deriving the induced loss, the negative log-likelihood of the true class works out to

$$-\log p = \frac{1}{\sigma^2} L_2(W) + \log\!\left[ \frac{\sum_{c'} \exp\!\big((1/\sigma^2) f_{c'}^W(x)\big)}{\big(\sum_{c'} \exp f_{c'}^W(x)\big)^{1/\sigma^2}} \right],$$

with $L_2(W)$ the unscaled cross-entropy. The inverse-variance weight $1/\sigma^2$ on the cross-entropy is exactly parallel to the regression case, but the regulariser came out as an ugly log-ratio of two sums that depends on the logits — messy to optimise and conceptually wrong, since the penalty for being uncertain about a task should not depend on the particular image. Making the explicit approximation $(1/\sigma)\sum_{c'}\exp((1/\sigma^2)f_{c'}^W(x)) \approx \big(\sum_{c'}\exp f_{c'}^W(x)\big)^{1/\sigma^2}$ — which is exact at $\sigma \to 1$, where both sides reduce to $\sum_{c'}\exp f_{c'}$, and degrades smoothly away from it — makes the ratio inside the log equal to $\sigma$, collapsing the regulariser to $\log\sigma$ and removing the data-dependent input from it. The classification term is then $(1/\sigma_2^2) L_2(W) + \log\sigma_2$, and the mixed objective for one Gaussian output $y_1$ and one softmax output $y_2$ is

$$L(W, \sigma_1, \sigma_2) = \frac{1}{2\sigma_1^2} L_1(W) + \frac{1}{\sigma_2^2} L_2(W) + \log\sigma_1 + \log\sigma_2,$$

with $L_1 = \|y_1 - f^W(x)\|^2$ and $L_2 = -\log\text{Softmax}(y_2, f^W(x))$. Both tasks are weighted by inverse variance and regularised by $\log\sigma_i$, and the construction extends to any number of continuous and discrete outputs by adding terms. One honest asymmetry remains: the regression coefficient is $1/(2\sigma_1^2)$ and the classification coefficient is $1/\sigma_2^2$ — the factor of two is the $1/2$ in the Gaussian NLL, absent from the temperature scaling that puts the full $1/\sigma^2$ on the logits. For a compact module that consumes any already-reduced task loss, I adopt the canonical uniform log-variance convention: the term $\exp(-s_i)L_i + s_i$ per task. In the log-variance variable the regression term $\tfrac{1}{2}\exp(-s)L + \tfrac{1}{2}s$ multiplied by two becomes $\exp(-s)L + s$ without moving its optimum in $s$, and the classifier yields the same precision term after the $\sigma \to 1$ approximation; the probabilistic derivation still explains the factor-of-two asymmetry that this coding convention smooths over.

This beats a grid search for two structural reasons, not merely by automating it. A grid search samples the weighting space coarsely while the good band is narrow, so it can step over the best point; the learned $\sigma$ moves continuously and is not quantised. More importantly, a grid weight is static for the whole run, whereas the learned $\sigma$ is dynamic: early in training every loss is large, so every $\sigma_i$ is large (by $\sigma_i^2 = L_i$) and the weighting is roughly even; as the model masters an easy task its loss drops, its $\sigma$ drops, and its weight rises — a schedule of relative weights that evolves over training in a way no single fixed grid point can match. Robustness to initialisation is guaranteed by convexity: the per-task objective in $s$ is $\tfrac{1}{2}\exp(-s)L + s/2$ with $\partial/\partial s = -\tfrac{1}{2}\exp(-s)L + \tfrac{1}{2}$, zero at $s = \log L$, and second derivative $\tfrac{1}{2}\exp(-s)L > 0$, so it is strictly convex with a single minimum; the implemented $\exp(-s)L + s$ shares that minimiser and positive curvature. We can therefore initialise $s = 0$ — meaning $\sigma^2 = 1$, $\exp(-s) = 1$, every task weighted equally, the most neutral possible start — and let each scalar move toward its fixed point with no further hand-tuned hyperparameter.

The whole thing drops into the existing multi-task harness as one log-variance Parameter per task, registered so the optimiser trains it jointly with the network, and a short forward computing $\sum_i \exp(-s_i) L_i + s_i$:

```python
import torch
import torch.nn as nn


class MultiTaskLoss(nn.Module):
    """Homoscedastic uncertainty weighting of K task losses.

    Learns one log-variance s_i = log(sigma_i^2) per task. Each task loss is
    weighted by its precision exp(-s_i) and regularized by + s_i:
        L = sum_i  exp(-s_i) * L_i  +  s_i
    exp(-s_i) > 0 (no divide-by-zero); s_i is unconstrained so SGD steps it
    freely; + s_i forbids the sigma -> inf (weight -> 0) collapse that a bare
    learnable weight w_i in sum_i w_i L_i suffers."""

    def __init__(self, num_tasks=2):
        super().__init__()
        # s_i = log(sigma_i^2), init 0  =>  sigma_i^2 = 1  =>  equal weighting.
        # Objective is convex in s with optimum at s = log(L_i), so init is robust.
        self.log_vars = nn.Parameter(torch.zeros(num_tasks))

    def forward(self, fine_loss, coarse_loss, epoch, total_epochs):
        losses = [fine_loss, coarse_loss]
        total = sum(
            torch.exp(-self.log_vars[i]) * losses[i] + self.log_vars[i]
            for i in range(len(losses))
        )
        return total
```
