We have observational data on a set of units — patients, voters, customers — each a covariate vector $X \in \mathbb{R}^d$, a binary treatment flag $W \in \{0,1\}$, and a single observed outcome $Y$, and we want to know not whether the treatment helps on average but for whom and by how much: the conditional average treatment effect $\tau(x) = E[Y(1) - Y(0) \mid X = x]$ as a whole function over the covariate space. The obstacle is structural. For every unit we ever observe exactly one of its two potential outcomes, $Y_i^{\mathrm{obs}} = Y_i(W_i)$, and the counterfactual is gone forever, so the per-unit difference $D_i = Y_i(1) - Y_i(0)$ is never seen for even one row. In fact $D_i$ is not identifiable at all: take $X$ uniform, $W$ a fair coin, and a Rademacher control outcome $Y(0) = \pm 1$; the world where $Y(1) = Y(0)$ (so $D_i = 0$ everywhere) and the world where $Y(1) = -Y(0)$ (so $D_i \in \{-2, 2\}$) produce an identical joint law of $(Y, X, W)$, yet differ in $D_i$. The recoverable target is the conditional mean $\tau(x) = E[D \mid X = x]$, and retargeting it costs nothing: conditioning on the fitted estimator,
$$E[(D_i - \hat\tau(x))^2 \mid X_i = x, \hat\tau] = E[(D_i - \tau(x))^2 \mid X_i = x] + (\tau(x) - \hat\tau(x))^2,$$
because the cross term carries $E[D_i - \tau(x) \mid X_i = x] = 0$. The first piece is irreducible variation of the unit effect about its mean; the second is exactly the squared error in $\tau(x)$, so the estimator minimizing MSE for the unobservable individual effect is the very one minimizing MSE for the CATE. The metric of record is therefore $\mathrm{EMSE} = E[(\tau(X) - \hat\tau(X))^2]$.

To make $\tau$ estimable from observed data we lean on two standard assumptions. Ignorability, $(\varepsilon(0), \varepsilon(1)) \perp W \mid X$, says that conditional on the covariates who got treated is unrelated to the potential outcomes, which gives the bridge $E[Y^{\mathrm{obs}} \mid X = x, W = w] = \mu_w(x)$ — the observable within-arm regression equals the counterfactual response surface $\mu_w(x) = E[Y(w) \mid X = x]$. Overlap, $0 < e_{\min} < e(x) < e_{\max} < 1$, keeps both arms present at every $x$ so those conditional means are estimable everywhere. Together they collapse the causal target into $\tau(x) = \mu(x,1) - \mu(x,0)$, a difference of two ordinary conditional expectations — a problem we have a hundred trusted tools for. The mature workhorses (random forests, BART, gradient boosting, neural nets) predict $E[Y \mid \text{features}]$ in high dimensions with sensible built-in regularization and little tuning; none of them knows what a counterfactual is, but the bridge just told us the counterfactual surfaces are ordinary regressions. So the right move is not to invent a bespoke causal learner but to write a recipe that turns any black-box regressor into a CATE estimator by choosing how to slice the data and how to recombine the fits. The most direct slicing reads straight off the difference: fit $\hat\mu_0$ on the control rows, $\hat\mu_1$ on the treated rows, and subtract — the two-model (T-learner) recipe. It cannot lose the treatment, since which model is used is the treatment, and from $(a-b)^2 \le 2a^2 + 2b^2$ its error is bounded by $2E[(\hat\mu_1 - \mu_1)^2] + 2E[(\hat\mu_0 - \mu_0)^2]$. But it wastes statistical strength: each regression is tuned to fit its own response surface in isolation, never seeing the other arm, so when $\mu_0$ and $\mu_1$ share a complicated baseline dependence on $x$ but their difference $\tau$ is simple — the regularity the literature keeps reporting, since the covariates that drive the baseline cancel in the contrast — the procedure spends all its capacity chasing the complicated baseline twice, with two independent errors, and then subtracts them to recover a quantity that was simple all along. Worse, when the design is unbalanced and control data accrues cheaply while treated units are scarce ($n \ll m$), $\hat\mu_1$ is fit on a tiny sample and its error dominates. It never represents the effect, so it cannot exploit the fact that the effect is easier to learn than its pieces.

What I propose is the S-learner — "S" for single. Instead of splitting the data, pool all $N$ rows and hand the learner the treatment flag as one more feature sitting in the vector right next to the components of $x$, with no special status, and fit a single combined surface $\hat\mu(x, w)$ for $E[Y^{\mathrm{obs}} \mid X = x, W = w]$ on the whole dataset. By the ignorability bridge $\hat\mu(x,1)$ targets $\mu_1(x)$ and $\hat\mu(x,0)$ targets $\mu_0(x)$, so the effect at $x$ is read off by holding every covariate fixed and toggling only the flag,
$$\hat\tau_S(x) = \hat\mu(x, 1) - \hat\mu(x, 0).$$
The causal content lives entirely in this pooling-and-toggling, not in the learner, which stays a black box. Two things make it work. First, the single learner sees both arms at once, so wherever the response depends on $x$ in the same way regardless of treatment — the shared baseline — it models that dependence once with all $N$ rows of strength rather than twice with independent errors; and when $w$ is flipped at fixed $x$ that shared baseline is identical in both predictions and subtracts out exactly, so $\hat\tau_S(x)$ is the model's pure response to flipping the one feature. The cancellation I wanted is built into the architecture because both terms come from the same fitted function at the same $x$. Second, the single model can decline to use the treatment feature where it is irrelevant: in a region where $\tau(x) = 0$, a loss-driven learner finds that splitting on the flag reduces the squared-error loss not at all, so it does not split, and $\hat\mu(x,1) - \hat\mu(x,0)$ comes out exactly zero — it reports zero rather than manufacturing spurious heterogeneity from sampling noise, which the two-model recipe cannot do. There is a unifying picture that pins down exactly what has been traded: with a tree-ensemble base learner, ask where in the tree the model is allowed to split on $w$. The S-learner lets the ordinary squared-error criterion decide — anywhere or nowhere. The two-model recipe forces the split on $w$ at the root, after which the two subtrees are just the arm-models. The modified-splitting forest forces it last, right before the leaves. One family, parameterized by where the treatment split is permitted (anywhere, first, last); the S-learner is the least-constrained member, the one that lets the data decide whether and where the treatment matters.

I have to be honest about where this freedom cuts the wrong way, because it is the direct consequence of the design, not a patchable bug. The flag is exactly one feature competing against all $d$ covariates, and every trusted learner is regularized — it penalizes complexity and shrinks weak signals. When the treatment's effect is real but weak relative to how strongly the covariates predict the outcome, the regularizer down-weights or outright discards the flag in favor of the informative covariates; then $\hat\mu(x,1) \approx \hat\mu(x,0)$, the toggled difference collapses, and $\hat\tau$ is biased toward zero, in the extreme a flat zero everywhere. The same property that makes the S-learner correctly return zero when the effect truly is zero makes it biased toward zero when the effect is real but small. So the S-learner and the two-model recipe sit at opposite ends of one bias–variance knob — the single model pools and can shrink to zero (ideal for simple, sparse, near-zero, smooth effects; dangerous when the effect is real but under-weighted), the two-model recipe forces the treatment to matter (safe against bias-to-zero but unable to share strength and prone to overfit the small arm). Neither dominates. The structural ceiling is worth making quantitative on the arm-split route, since it is the warning for any estimator that gets the effect by estimating response values and subtracting. From $\hat\tau - \tau = (\hat\mu_1 - \mu_1) - (\hat\mu_0 - \mu_0)$ and $(a-b)^2 \le 2a^2 + 2b^2$, $\mathrm{EMSE} \le 2E[(\hat\mu_1 - \mu_1)^2] + 2E[(\hat\mu_0 - \mu_0)^2]$. Each arm-error is naturally an expectation over its own arm's covariate law, but the EMSE wants the marginal law of a fresh test point, and overlap supplies the bounded change of measure: from $E[g(X) \mid W = 1] = E[g(X)e(X)]/E[e(X)]$ with $e_{\min} < e(x) < e_{\max}$ one gets $(e_{\min}/e_{\max})E[g(X)] \le E[g(X)\mid W=1] \le (e_{\max}/e_{\min})E[g(X)]$, and symmetrically for the control arm with $1 - e(x)$. Each arm-error then becomes its minimax response rate, giving
$$\mathrm{EMSE} \le 2C\,(e_{\max}/e_{\min})\,n^{-a_\mu} + 2C\,\big((1-e_{\min})/(1-e_{\max})\big)\,m^{-a_\mu} = O(n^{-a_\mu} + m^{-a_\mu}).$$
The exponent is $a_\mu$, the smoothness of the response surfaces, never $a_\tau$, the often-greater smoothness of the effect: this whole family pays the response rate even when the effect is far simpler, and in the unbalanced case $n \ll m$ it is stuck at $n^{-a_\mu}$ in the small arm. That inability to cash in a smoother effect is the gap a difference-of-responses estimator leaves open. Within it the S-learner is the honest, simplest, least-constrained pooled member, and the right tool whenever the effect really is simple or near-zero.

The base learner I reach for is gradient-boosted trees: an additive ensemble of shallow trees fit to the squared-error residual. Shallow depth (four) lets each tree capture low-order interactions, including interactions between the treatment flag and the covariates — a split on a covariate beneath a split on the flag is exactly "the effect varies with that covariate," the mechanism by which the single model represents heterogeneity. A modest learning rate with many trees buys a smoother, lower-variance surface; row subsampling adds stochastic regularization; and a floor on leaf size stops any leaf from chasing a handful of points, which matters because the effect is a difference of two predictions and is therefore especially sensitive to leaf-level noise. None of these knobs is causal — they are the standard recipe for a low-variance nonparametric tabular regressor — and that is the point: the causal content is in how I slice and toggle, not in the learner. The estimator drops into the harness's `fit`/`predict` slot as three moves — append the flag, fit one pooled model, predict twice and subtract:

```python
import os
import numpy as np
from sklearn.ensemble import GradientBoostingRegressor


class CATEEstimator:
    """S-learner: a single outcome model on the pooled data with the treatment flag as an
    ordinary feature; CATE read off by toggling the flag.  tau_hat(x) = mu_hat(x,1) - mu_hat(x,0)."""

    def __init__(self):
        self._seed = int(os.environ.get("SEED", "42"))
        self._model = GradientBoostingRegressor(
            n_estimators=200,      # many shallow trees -> smooth, low-variance surface
            max_depth=4,           # depth > 1 lets flag x covariate interactions form (heterogeneity)
            learning_rate=0.1,     # modest per-tree shrinkage
            min_samples_leaf=20,   # leaf floor: the effect is a difference of predictions, leaf-noise sensitive
            subsample=0.8,         # stochastic row subsampling -> extra regularization
            random_state=self._seed,
        )

    def fit(self, X, T, Y):
        XT = np.column_stack([X, T.reshape(-1, 1)])   # pooled design: [covariates | treatment flag]
        self._model.fit(XT, Y)                         # single surface mu(x, w) = E[Y^obs | x, w]
        return self

    def predict(self, X):
        n = X.shape[0]
        X1 = np.column_stack([X, np.ones((n, 1))])     # toggle flag to 1: mu_hat(x, 1)
        X0 = np.column_stack([X, np.zeros((n, 1))])    # toggle flag to 0: mu_hat(x, 0)
        return self._model.predict(X1) - self._model.predict(X0)   # tau_hat(x)
```
