Let me start from where it actually hurts. I have a linear classifier — say the perceptron in its dual form — and I keep staring at how it touches the data. It maintains $w=\sum_t \alpha_t y_t x_t$ and decides by $\mathrm{sign}\,\langle w,x\rangle = \mathrm{sign}\sum_t \alpha_t y_t \langle x_t,x\rangle$. The point $x$ enters only through $\langle x_t,x\rangle$. Same with least squares: the normal equations live on $X X^\top$, a matrix of $\langle x_i,x_j\rangle$. I never need a coordinate of a data point on its own; I need the pairwise inner products. That is a very particular kind of blindness, and I want to know what it's worth.

What it costs me is curvature. A linear rule cuts the space with a flat sheet. If my positives and negatives are arranged in two concentric rings, no hyperplane separates them, and the perceptron just thrashes. The textbook escape is to manufacture nonlinear features: map $x\mapsto\phi(x)=(\phi_1(x),\phi_2(x),\dots)$ — squares, cross terms, higher monomials — and run the same linear machine on $\phi(x)$. A flat cut in $\phi$-space is a curved cut back in $x$-space; the rings become separable once I add the radius-squared coordinate. Fine in principle. But to get real richness $\phi(x)$ has to be huge. Degree-$d$ monomials in $p$ variables give me on the order of $p^d$ coordinates, and there are similarity notions I'd love to use whose natural feature space has *infinitely* many coordinates. I cannot store $\phi(x)$, I cannot optimize a weight vector that lives in $\mathbb R^{p^d}$, and I certainly cannot touch an infinite-dimensional one. So the explicit route is dead the moment the features get interesting.

Now hold those two observations next to each other. The algorithm only ever wants inner products. The thing I can't afford is the feature vector itself. So what if I never form $\phi(x)$ — what if I only ever need $\langle\phi(x),\phi(x')\rangle$? In the lifted space the perceptron decides by $\mathrm{sign}\sum_t \alpha_t y_t \langle\phi(x_t),\phi(x)\rangle$. Every appearance of the data is $\langle\phi(x_t),\phi(x)\rangle$. If I had some function $k(x,x')$ that equalled $\langle\phi(x),\phi(x')\rangle$ — and that I could evaluate directly, cheaply, without building $\phi$ — I could run the entire lifted algorithm by substituting $k$ for each inner product. The feature space could be as enormous as I like, even infinite, and I'd never visit it.

So the substitution is the prize: take any algorithm that touches data only through $\langle x_i,x_j\rangle$, and replace $\langle x_i,x_j\rangle\to k(x_i,x_j)$ everywhere. The question that decides whether this is real or wishful is: **which functions $k(x,x')$ are honestly equal to $\langle\phi(x),\phi(x')\rangle$ for some feature map $\phi$?** Because if I substitute an arbitrary $k$, I might be running an algorithm in a "space" that isn't a space at all — no consistent geometry, distances coming out negative — and the convergence and optimality guarantees that made the linear method worth using would evaporate.

Let me try to pin the condition down with the smallest possible example, where I can compute everything. Suppose my whole input domain is just a finite set $\{x_1,\dots,x_m\}$. Then $k$ is fully described by the matrix $K$ with $K_{ij}=k(x_i,x_j)$. I want a map $\Phi$ with $\langle\Phi(x_i),\Phi(x_j)\rangle = K_{ij}$. When can I build one? $K$ had better be symmetric — inner products are. A real symmetric matrix diagonalizes: $K=V\Lambda V^\top$, with $V$ orthogonal (columns are eigenvectors $v_t$) and $\Lambda$ diagonal (eigenvalues $\lambda_t$). Now just try the obvious feature map: let the $t$-th coordinate of $\Phi(x_i)$ be $\sqrt{\lambda_t}\,v_t^{(i)}$, i.e.
$$\Phi(x_i)=\big(\sqrt{\lambda_1}\,v_1^{(i)},\dots,\sqrt{\lambda_m}\,v_m^{(i)}\big).$$
Then $\langle\Phi(x_i),\Phi(x_j)\rangle = \sum_t \lambda_t v_t^{(i)} v_t^{(j)} = (V\Lambda V^\top)_{ij}=K_{ij}$. It works — *provided* every $\sqrt{\lambda_t}$ is real, i.e. provided $\lambda_t\ge 0$.

Is that nonnegativity a mere convenience of this construction, or is it forced? Suppose some eigenvalue $\lambda_s<0$ and a feature map still existed. Build the feature-space point $z=\sum_i v_s^{(i)}\Phi(x_i)$, using the components of the eigenvector $v_s$ as coefficients. Its squared length is
$$\|z\|^2=\sum_{i,j} v_s^{(i)}v_s^{(j)}\langle\Phi(x_i),\Phi(x_j)\rangle=\sum_{i,j}v_s^{(i)} K_{ij} v_s^{(j)} = v_s^\top K v_s = \lambda_s < 0.$$
A squared length cannot be negative. So a negative eigenvalue makes a feature map *impossible*, and nonnegative eigenvalues make one *available*. The two conditions coincide exactly: $k$ on a finite domain is an inner product of features iff $K$ has no negative eigenvalues, iff $c^\top K c\ge 0$ for every coefficient vector $c$. That last form — $\sum_{i,j}c_i c_j K_{ij}\ge 0$ for all $c$ — is the clean statement, and it doesn't reference eigenvectors at all. This is what gets called a *positive-definite* kernel.

I'll lean on that finite case as the definition even when the domain is infinite, because I can't write down one giant Gram matrix for an infinite $\mathcal X$. But I never need to: in practice I only ever see finitely many points at a time. So I'll demand that *every* finite Gram matrix $K=(k(x_i,x_j))$, for every choice of finitely many points, is positive semidefinite. That is exactly the leverage I have — a learning algorithm only ever forms a finite Gram matrix — and it's a condition I can check.

Before trusting this, let me make sure the geometry I'm importing is sound. With $k$ positive definite I should already be able to derive the basic inner-product inequalities without ever seeing $\phi$. The $1\times 1$ Gram matrix at a single point $x$ is just $k(x,x)\ge 0$ — lengths are nonnegative, good. For two points $u,v$ write $a=k(u,u)$, $b=k(u,v)=k(v,u)$, and $d=k(v,v)$. The $2\times 2$ Gram matrix $\begin{psmallmatrix}a&b\\b&d\end{psmallmatrix}$ is positive semidefinite, so $c^\top K c\ge 0$ for every $c$. Pick $c=(d,-b)^\top$ to make the mixed terms cancel. Expanding,
$$0\le c^\top K c = a d^2 - 2d b^2 + d b^2 = d(ad-b^2).$$
If $d>0$, division gives $b^2\le ad$. If $d=0$, positivity of $(t,1)K(t,1)^\top=a t^2+2bt$ for every real $t$ forces $b=0$, so the same inequality holds. Thus $k(u,v)^2\le k(u,u)\,k(v,v)$ — Cauchy–Schwarz, recovered purely from positive definiteness. The geometry is honest. So I can talk about lengths, angles, and orthogonality among the $\phi(x)$ while only ever evaluating $k$.

Now I want the infinite-dimensional version of the finite eigen-argument, because that's where the modelling power is. There's an old current of analysis exactly here. The name "kernel" comes from integral operators: $k$ defines $(T_k f)(x)=\int k(x,x')f(x')\,dx'$, and $k$ is the kernel of $T_k$. Mercer's theorem is the eigen-decomposition of that operator. For a continuous symmetric positive-definite kernel on a compact domain, $T_k$ has eigenfunctions $\psi_j$ (orthonormal in $L_2$) with nonnegative eigenvalues $\lambda_j$, and
$$k(x,x')=\sum_j \lambda_j\,\psi_j(x)\,\psi_j(x'),$$
uniformly convergent. This is the same statement as my finite $K=V\Lambda V^\top$, just with eigen*functions* in place of eigen*vectors*. And it hands me the feature map directly: set $\phi(x)=\big(\sqrt{\lambda_j}\,\psi_j(x)\big)_j$, a vector in $\ell_2$ with possibly infinitely many coordinates, and then $k(x,x')=\sum_j(\sqrt{\lambda_j}\psi_j(x))(\sqrt{\lambda_j}\psi_j(x'))=\langle\phi(x),\phi(x')\rangle$. The nonnegativity of the eigenvalues is, again, exactly the positive-definiteness condition — the very same wall the negative eigenvalue hit in the finite case. So positive-definite functions are inner products of features, in finite *or* infinite dimension, and I can wield that inner product by evaluating $k$ alone.

This is the substitution licensed at last. I'll take any inner-product-only algorithm and put $k$ in for each $\langle\cdot,\cdot\rangle$; positive definiteness of $k$ guarantees there's a genuine feature space — possibly infinite-dimensional — in which I am exactly running the original linear method. I never compute $\phi(x)$.

Let me sanity-check on a concrete kernel, because I want to see the trick bite. Take $k(x,z)=(\langle x,z\rangle_{\mathbb R^p})^2$, the square of the ordinary dot product, for $p=2$. Is that an inner product in some space? Expand: $\big(\sum_j x^{(j)}z^{(j)}\big)^2=\sum_{j,\ell}x^{(j)}x^{(\ell)}z^{(j)}z^{(\ell)}$. If I read off $\phi([x^{(1)},x^{(2)}])=\big((x^{(1)})^2,\,(x^{(2)})^2,\,x^{(1)}x^{(2)},\,x^{(2)}x^{(1)}\big)$ into $\mathbb R^4$, then $\langle\phi(x),\phi(z)\rangle=(x^{(1)})^2(z^{(1)})^2+(x^{(2)})^2(z^{(2)})^2+2x^{(1)}x^{(2)}z^{(1)}z^{(2)}=\langle x,z\rangle^2$. Yes. So the quadratic kernel silently maps into the space of degree-2 monomials, and I get curved boundaries for the price of one squaring. Push it further: $k(x,z)=(\langle x,z\rangle+c)^d$ for $c\ge 0$ expands into all monomials up to degree $d$, with $c$ tuning the relative weight of low- and high-order terms — polynomial decision boundaries, computed in $O(p)$ per evaluation no matter how astronomically large the implicit feature space is. And there are positive-definite kernels — the Gaussian among them — whose feature space is genuinely infinite-dimensional, so the only way to use them is the kernel substitution; there is no explicit $\phi$ to fall back on. The blindness of the algorithm has become its superpower.

There's still a loose end I want to close, because it makes everything cleaner and it'll matter for the optimization problem coming next. Mercer's eigen-expansion needed continuity and compactness and an actual measure to integrate against. But $\mathcal X$ might just be a *set* with no such structure. Can I get a feature space from positive definiteness alone, with no analysis on $\mathcal X$? Let me try to *build* the feature space by hand. The most economical idea: represent each point by its similarity to everything else. Define
$$\phi(x)=k(\cdot,x),$$
the function $x'\mapsto k(x',x)$. So $\phi(x)$ is itself a function on $\mathcal X$. To make these into a vector space I take finite linear combinations $f=\sum_i \alpha_i k(\cdot,x_i)$, and I need an inner product. The only sensible choice consistent with wanting $\langle\phi(x_i),\phi(x_j)\rangle=k(x_i,x_j)$ is, for a second function $f'=\sum_j \beta_j k(\cdot,x_j')$,
$$\big\langle f,\ f'\big\rangle := \sum_{i,j}\alpha_i\beta_j\, k(x_i,x_j').$$
I have to check this is even well-defined, since a function might have several expansions. Rewrite it as $\sum_{i,j}\alpha_i\beta_j k(x_i,x_j')=\sum_j \beta_j \big(\sum_i\alpha_i k(x_j',x_i)\big)=\sum_j\beta_j f(x_j')$. If I change the expansion of $f$ by adding a zero function $r$, the change in the proposed inner product is $\sum_j\beta_j r(x_j')=0$. Symmetrically, the same quantity is $\sum_i\alpha_i f'(x_i)$, so changing the expansion of $f'$ by a zero function changes nothing too. Good — the bilinear form depends only on the two functions, not on the particular sums used to write them. It's symmetric, it's bilinear, and it's positive: $\langle f,f\rangle=\sum_{i,j}\alpha_i\alpha_j k(x_i,x_j)\ge 0$ is exactly the positive-definiteness of $k$.

And now the property that this construction was secretly designed to produce drops out. Compute the inner product of a basis element $k(\cdot,x)$ with an arbitrary $f=\sum_i\alpha_i k(\cdot,x_i)$:
$$\langle k(\cdot,x),\,f\rangle=\sum_i \alpha_i\, k(x,x_i)=\sum_i\alpha_i k(x_i,x)=f(x).$$
The inner product of $f$ against the feature of $x$ just *reads off the value of $f$ at $x$*. In particular $\langle k(\cdot,x),k(\cdot,x')\rangle=k(x,x')$, so $\phi(x)=k(\cdot,x)$ is a bona fide feature map and the original $k$ is its inner product — no Mercer, no measure, $\mathcal X$ an arbitrary set. This is the *reproducing property*, and a space carrying it is a reproducing kernel Hilbert space; Aronszajn studied exactly these, and the correspondence is one-to-one — every positive-definite kernel yields a unique such space, and conversely every space where evaluation is continuous has a reproducing kernel. The last technical wrinkle: I should confirm $\langle f,f\rangle=0$ implies $f\equiv 0$, so the form is a true inner product and not just a seminorm. The same two-by-two Cauchy–Schwarz argument gives $|f(x)|^2=|\langle k(\cdot,x),f\rangle|^2\le k(x,x)\,\langle f,f\rangle$, so if $\langle f,f\rangle=0$ then $f(x)=0$ at every $x$. Complete the span in this norm and I have a Hilbert space $\mathcal H_k$.

So the picture is settled: a positive-definite $k$ gives me, for free, an inner-product space of functions in which $k$ both *is* the inner product of point-features and *reproduces* function values. I'll carry both readings — $\phi(x)=k(\cdot,x)$ as a vector, and $f(x)=\langle k(\cdot,x),f\rangle$ as evaluation — because together they're about to crack the second half of the problem.

Now to that second half, which is where the cost really threatens to be infinite. The substitution lets me *pose* a learning problem in $\mathcal H_k$, but $\mathcal H_k$ can be infinite-dimensional. Take the natural setup: estimating a function from data is ill-posed in an infinite-dimensional model space, so I regularize — minimize a fit term plus a roughness penalty,
$$\text{minimize over } f\in\mathcal H_k:\quad C\big((x_1,y_1,f(x_1)),\dots,(x_m,y_m,f(x_m))\big) + \lambda\,\|f\|_{\mathcal H_k}^2.$$
Notice the fit term touches $f$ *only* through its values $f(x_i)$ at the training points — it never asks for $f$ anywhere else. And the penalty is built from $\|f\|_{\mathcal H_k}$. This is an honest optimization over an infinite-dimensional space, and on its face it looks uncomputable. I do not want to search $\mathcal H_k$.

Let me look at what the objective can actually *see* of $f$. It sees the $m$ numbers $f(x_1),\dots,f(x_m)$, and it sees $\|f\|$. By the reproducing property, $f(x_i)=\langle f,\phi(x_i)\rangle$ — so the fit term sees only the projections of $f$ onto the $m$ directions $\phi(x_1),\dots,\phi(x_m)$. Everything about $f$ in directions orthogonal to those is invisible to the fit. And the penalty wants $\|f\|$ small. Those two facts are already pulling in a suggestive direction — anything orthogonal to the data directions costs norm and buys nothing.

Make it exact. The vectors $\phi(x_1),\dots,\phi(x_m)$ span a finite-dimensional subspace $S=\mathrm{span}\{\phi(x_i)\}\subseteq\mathcal H_k$, of dimension at most $m$. Any $f\in\mathcal H_k$ decomposes orthogonally into its projection onto $S$ and a remainder perpendicular to $S$:
$$f=\sum_{i=1}^m \alpha_i\,\phi(x_i) + v,\qquad \langle v,\phi(x_j)\rangle=0\ \text{ for all } j.$$
Now evaluate $f$ at a training point, using the reproducing property and the orthogonality of $v$:
$$f(x_j)=\langle f,\phi(x_j)\rangle=\Big\langle \sum_i\alpha_i\phi(x_i)+v,\ \phi(x_j)\Big\rangle=\sum_i\alpha_i\,\langle\phi(x_i),\phi(x_j)\rangle=\sum_i\alpha_i\,k(x_i,x_j).$$
The $v$ term contributes nothing — it's orthogonal to every $\phi(x_j)$. So *every* value $f(x_j)$ that the fit term reads is completely independent of $v$. The fit term cannot tell whether $v$ is there or not.

The penalty, however, can. Because $v\perp \sum_i\alpha_i\phi(x_i)$, Pythagoras gives
$$\|f\|^2=\Big\|\sum_i\alpha_i\phi(x_i)\Big\|^2+\|v\|^2\ \ge\ \Big\|\sum_i\alpha_i\phi(x_i)\Big\|^2,$$
with equality exactly when $v=0$. So $\|f\|$ is minimized, at fixed fit, precisely by killing $v$. Putting the two halves together: starting from any candidate $f$ with $v\ne 0$, set $v=0$. The fit term is *unchanged* (it only saw $f(x_j)$, which doesn't depend on $v$), and the squared norm drops by $\|v\|^2>0$, so any strictly increasing penalty in $\|f\|$ strictly decreases. The candidate with $v\ne0$ cannot have been a minimizer — there was a strictly better one sitting right next to it, with $v$ deleted. Hence at any minimizer $v=0$, and
$$f=\sum_{i=1}^m \alpha_i\,\phi(x_i)=\sum_{i=1}^m\alpha_i\,k(x_i,\cdot).$$

That is the whole game. The minimizer of a regularized risk in an infinite-dimensional RKHS lies in the finite span of the kernels at the training points. The orthogonal component is dead weight: it raises the regularizer without changing the fit. So an infinite-dimensional search collapses to optimizing $m$ scalars $\alpha_1,\dots,\alpha_m$. I'll substitute the form back in to see the reduced problem: with $f=\sum_i\alpha_i k(x_i,\cdot)$, the values are $f(x_j)=\sum_i\alpha_i k(x_i,x_j)=(K\alpha)_j$ and the norm is $\|f\|^2=\sum_{i,j}\alpha_i\alpha_j k(x_i,x_j)=\alpha^\top K\alpha$, so the problem becomes
$$\min_{\alpha\in\mathbb R^m}\ C\big((x_1,y_1,(K\alpha)_1),\dots,(x_m,y_m,(K\alpha)_m)\big)+\lambda\,\alpha^\top K\alpha,$$
a finite, $m$-dimensional optimization, expressed entirely through the Gram matrix $K$. Exactly the two things I wanted have arrived together: an infinitely rich feature space, and a finite computation.

Let me check what the argument truly used, because I want the theorem as wide as it honestly is. I used: (a) the fit term depends on $f$ only through the values $f(x_1),\dots,f(x_m)$ — any loss whatsoever, even one coupling the points, even one taking the value $+\infty$ to encode hard constraints, as long as it's a function of those values; and (b) the penalty is a function of $\|f\|$ that I can *strictly decrease* by decreasing $\|f\|$. I wrote $\lambda\|f\|^2$, but re-read the Pythagoras step: I only needed that replacing $\|f\|$ by the smaller $\|\sum_i\alpha_i\phi(x_i)\|$ does not increase the penalty, and strictly decreases it unless $v=0$. So any *strictly monotonically increasing* function $g$ of $\|f\|$ works:
$$g(\|f\|)=g\Big(\sqrt{\textstyle\|\sum_i\alpha_i\phi(x_i)\|^2+\|v\|^2}\Big)\ \ge\ g\Big(\|\textstyle\sum_i\alpha_i\phi(x_i)\|\Big),$$
equality iff $v=0$. So the regularizer need not be the squared norm — it can be $\|f\|$, or any strictly increasing transform, including ones lifted from a uniform-convergence risk bound. The representation $f=\sum_i\alpha_i k(x_i,\cdot)$ survives all of them. I also notice I needed *strict* monotonicity: if $g$ were merely nondecreasing, a flat stretch could let some $v\ne0$ tie, and then a minimizer might not have the finite form — though even then there would still *exist* a minimizer that does. Convexity, I notice, played no role in the representation at all; it only matters for whether the reduced $m$-dimensional problem has a unique global optimum rather than local ones.

There's an obvious extension I'll want, because real models often carry an unpenalized parametric part — a polynomial trend, an intercept. If the penalty is really a seminorm with a finite-dimensional null space $\mathrm{span}\{\psi_1,\dots,\psi_M\}$, I should not try to kill that null-space component; the penalty cannot see it, and the data may need it. I split the candidate into a null-space part plus a penalized part, then apply the same orthogonal argument only on the penalized complement. The component outside the training representers and outside the null space is invisible to the fit and pure cost to the penalty, so it dies, leaving
$$f(\cdot)=\sum_{i=1}^m\alpha_i\,k(x_i,\cdot)+\sum_{p=1}^M\beta_p\,\psi_p(\cdot).$$
A combination of data-kernels plus a low-dimensional parametric term. That's the semiparametric form, and it's how splines naturally appear: the penalty $\int(Lu)^2$ annihilates the polynomials of degree $<m$ (its null space), so those ride along unpenalized as the $\psi_p$, and the rest of the solution is a sum of reproducing-kernel terms — the representers of the evaluation functionals — at the data sites. Stated in the spline language: among functions $u$ in a Sobolev space with $u(t_i)=y_i$, the minimizer of $\int(Lu)^2$ is the natural spline, a finite combination of the Green's-function representers $K_1(t_i,\cdot)$ plus the null-space polynomials — which is the very same orthogonal-decomposition argument, since the constraints fix only the components along the representers and "clearly we must take the orthogonal part to be zero."

And I can see now that the kernel-perceptron from before was a special, algorithm-specific shadow of this. Its online rule produced $h(x)=\sum_t\alpha_t y_t k(x_t,x)$ — a finite combination over the data — but only because the perceptron's update *happens* to add a multiple of the current point's feature on each mistake. The representer argument explains why that finite form is not a lucky accident of the update but a structural necessity for *any* regularized risk of this shape: the solution lives in $\mathrm{span}\{k(x_i,\cdot)\}$ no matter how I get there. Likewise the support-vector machine's $\sum_i\alpha_i y_i k(x_i,\cdot)$, which I'd previously have justified only by staring at the Kuhn–Tucker conditions of its particular quadratic-program dual, is just this theorem applied to the hinge loss with $g(\|f\|)=\|f\|^2$ — and now I can swap in a different loss, or a regularizer borrowed from a risk bound where there is no clean dual to invoke, and the finite expansion still holds.

Let me say the whole causal chain back to myself. Linear methods see data only through inner products; that lets me run them in any feature space using just $\langle\phi(x),\phi(x')\rangle$. A function $k$ equals such an inner product exactly when it is positive definite — proven by diagonalizing the Gram matrix, where a negative eigenvalue would force a negative squared length and nonnegative eigenvalues hand back the feature map (Mercer in the continuous case, an explicit construction $\phi(x)=k(\cdot,x)$ with the reproducing property $f(x)=\langle k(\cdot,x),f\rangle$ in the general case). So I substitute $k$ for every inner product and work, for free, in a possibly infinite-dimensional space. Posing a regularized risk there looks uncomputable, but the fit term sees $f$ only through $\{f(x_i)\}$ and the penalty only through $\|f\|$; decomposing $f$ into its projection on $\mathrm{span}\{\phi(x_i)\}$ plus an orthogonal remainder, the remainder is invisible to the fit (reproducing property) and strictly costly to the penalty (Pythagoras + strict monotonicity), so it vanishes at any minimizer — leaving $f=\sum_i\alpha_i k(x_i,\cdot)$ and an $m$-dimensional solve in $\alpha$. Rich nonlinearity at the cost of pairwise similarities; an infinite-dimensional optimum pinned to finitely many points.

To make it concrete I'll land it on the simplest method whose every step I can name: kernel ridge regression. Squared-error fit, squared-norm penalty, the representer form substituted in. The penalized problem $\min_f \sum_i (y_i-f(x_i))^2+\lambda\|f\|^2$ becomes, with $f=\sum_i\alpha_i k(x_i,\cdot)$, the finite problem $\min_\alpha \|y-K\alpha\|^2+\lambda\,\alpha^\top K\alpha$. Since $K$ is symmetric positive semidefinite, the objective is convex. Its gradient is $-2K(y-K\alpha)+2\lambda K\alpha$, so stationarity is
$$K\big((K+\lambda I)\alpha-y\big)=0.$$
For $\lambda>0$, $K+\lambda I$ is positive definite, and the canonical choice $\alpha=(K+\lambda I)^{-1}y$ satisfies this stationarity condition because $y-K\alpha=\lambda\alpha$. If $K$ has a null space, other coefficient vectors can differ there without changing the represented function; this choice is the stable finite solve. Prediction at a new point is $f(x)=\sum_i\alpha_i k(x_i,x)$ — the kernel evaluated against the training set, never a feature vector in sight.

```python
import numpy as np

def gaussian_kernel(x, z, gamma):
    # A positive-definite k(x,z) = <phi(x), phi(z)> for an *infinite*-dimensional
    # feature map phi we never build. We only ever evaluate k.
    if gamma < 0:
        raise ValueError("gamma must be nonnegative")
    d = x - z
    return np.exp(-gamma * np.dot(d, d))

def gram_matrix(X, kernel):
    n = len(X)
    K = np.empty((n, n))
    for i in range(n):
        for j in range(n):
            K[i, j] = kernel(X[i], X[j])      # only pairwise similarities, never features
    return K

def is_positive_semidefinite(K, tol=1e-9):
    # The substitution k -> <phi,phi> is legitimate iff every Gram matrix is PSD:
    # a negative eigenvalue would mean a negative squared length in feature space.
    w = np.linalg.eigvalsh(0.5 * (K + K.T))
    return bool(np.all(w >= -tol))

def fit(X, y, kernel, lam):
    # Representer theorem: the minimizer of  sum_i (y_i - f(x_i))^2 + lam ||f||^2
    # over the (infinite-dim) RKHS lies in span{k(x_i, .)}, so f = sum_i alpha_i k(x_i, .)
    # and the infinite-dim search collapses to an m-dimensional solve in alpha.
    if lam <= 0:
        raise ValueError("lam must be positive")
    K = gram_matrix(X, kernel)
    assert is_positive_semidefinite(K)
    # Canonical KRR coefficient solve for the represented function.
    alpha = np.linalg.solve(K + lam * np.eye(len(X)), y)
    return alpha

def predict(alpha, X_train, x, kernel):
    # f(x) = sum_i alpha_i k(x_i, x): a new point enters only through its
    # similarities to the training points -- the representer expansion, evaluated.
    return sum(alpha[i] * kernel(X_train[i], x) for i in range(len(X_train)))
```
