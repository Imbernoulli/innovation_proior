I start with the obstacle that makes the linear method look too small and the nonlinear feature map look too large. If I write a separating function as a linear expression in features, I can make a hyperplane after mapping each input `x` to coordinates such as `phi_1(x), phi_2(x), ...`. But this appears to require choosing, storing, and computing every coordinate. The useful rectifying space may be very large, and in the cases I care about it may not even be convenient to list its coordinates.

I look more carefully at what the update rule actually uses. The correction rule does not need to inspect every coordinate separately if the only quantity it needs is the scalar product between the current example and earlier examples in the rectifying space. If a similarity has an expansion `k(x,y) = sum_r lambda_r phi_r(x) phi_r(y)` with `lambda_r >= 0`, then the lifted coordinate is `sqrt(lambda_r) phi_r(x)`, not `lambda_r phi_r(x)`. The scalar product is therefore `sum_r lambda_r phi_r(x) phi_r(y)`. Zero eigenvalues simply contribute no coordinate. The classifier after mistakes is a signed sum of such similarities centered at the examples that forced corrections, so I do not need a full table of coordinates; I need a reliable way to compute the inner products those coordinates would have produced.

This tells me the first half of the idea, but it also creates a new danger. I cannot replace an inner product by an arbitrary similarity score. In the real-valued case the finite matrix `K_ij = k(x_i,x_j)` must be symmetric, and every vector of coefficients must have nonnegative squared length `sum_i sum_j c_i c_j K_ij`. Otherwise the supposed feature-space norm can become negative. The two-point case already gives a useful guardrail: positive semidefiniteness implies `k(x,z)^2 <= k(x,x) k(z,z)`, and if `k(x,x)=0` then `k(x,z)=0` for every `z` in the sample. Conversely, if every finite Gram matrix is symmetric positive semidefinite, I can construct a feature space in which `k` acts like a dot product. The finite-sample calculation makes this concrete by diagonalizing the Gram matrix and using square roots of the nonnegative eigenvalues as coordinates. The general construction sends `x` to the function `k(.,x)` and defines inner products so that `<k(.,x), k(.,z)> = k(x,z)`.

Once I have this reproducing kernel Hilbert space, I can say the substitution without pretending that I know the hidden coordinates. The lifted feature vector for `x` is `k(.,x)`, and evaluating a function at `x` is the inner product `<f, k(.,x)>`. So any algorithm written only through lifted inner products can run from kernel evaluations. This is the coordinate-replacement insight: the data geometry is available as a finite Gram matrix even when the feature space itself is implicit or infinite-dimensional.

That still leaves a second problem. A regularized learning problem asks me to search over all functions in the Hilbert space. Even if predictions can be evaluated through inner products, why should the optimizer itself be finite? The missing move is to look at what the observations can see. In the Hilbert space, each bounded observation functional has a representer. I can decompose any candidate function into the part spanned by those representers and a remainder orthogonal to them. The constraints or fitted values only see the span part. The orthogonal remainder contributes to the penalty and contributes nothing to the observations.

I apply the same decomposition to the machine-learning objective. Given training inputs `x_1, ..., x_m`, I split any `f` as `f_parallel + f_perp`, where `f_parallel` lies in `span{k(.,x_i)}` and `f_perp` is orthogonal to every `k(.,x_i)`. By the reproducing property, `f(x_j) = <f, k(.,x_j)>`, so the orthogonal term vanishes at every training point. The loss is unchanged if I drop `f_perp`. The norm, however, follows Pythagoras: `||f||^2 = ||f_parallel||^2 + ||f_perp||^2`. If the regularizer is a strictly increasing function of the norm, keeping `f_perp` can only make the objective worse unless `f_perp` is zero.

Now the finite form is forced rather than guessed. Every minimizer under a strictly increasing norm penalty must lie in the data span, so it has the representation `f(.) = sum_i alpha_i k(.,x_i)`. If the penalty is only nondecreasing, I can still choose a minimizer in that span, but I cannot claim every minimizer has no invisible orthogonal component. This is more than a slogan about using kernels. The kernel substitution explains how to compute the geometry without explicit coordinates; the representer argument explains why the optimizer has no useful component outside the finite span generated by the observed inputs. Together they turn an infinite or implicit feature-space problem into a finite coefficient problem over the training set.

For squared-loss regression the coefficient equations need one more check, because the Gram matrix can be singular. With `f_train = K alpha` and `||f||_H^2 = alpha^T K alpha`, the objective `||y - K alpha||^2 + lambda alpha^T K alpha` has stationarity condition `K((K + lambda I) alpha - y)=0`. For `lambda > 0`, the canonical stable choice is `alpha = (K + lambda I)^(-1) y`; `K + lambda I` is positive definite even when `K` is only semidefinite. Prediction then uses the stored training inputs through `k(x, X_train)^T alpha`, which is exactly the finite coefficient form promised by the theorem.

I also see why the null-space extension is not optional in spline-like problems. If the penalty ignores a finite-dimensional subspace, such as low-degree polynomial terms, the solution keeps that unpenalized basis in addition to the kernel expansion. The same orthogonality logic removes only the invisible penalized remainder. The final learned function is therefore either a pure training-centered kernel expansion or that expansion plus a small explicit null-space basis, with rank conditions or side constraints supplied by the particular spline system to make the coefficients identifiable.
