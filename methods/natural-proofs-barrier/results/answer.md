# The Natural Proofs barrier

## The problem it addresses

Why has the combinatorial circuit-lower-bound program — which proved PARITY ∉ AC0 and MOD-`q` ∉ AC0[`p`] — stalled completely just above AC0[`p`], far short of `NC1` or `P/poly` and hence of `P ≠ NP`? Relativization (Baker–Gill–Solovay) explains why diagonalization can't separate `P` from `NP`, but the combinatorial methods are non-relativizing, so a different obstruction is needed. The Natural Proofs barrier supplies it: it identifies a structural feature shared by essentially all known nonmonotone lower bounds and proves that this feature is exactly what prevents them from reaching `P/poly`, assuming strong one-way functions exist.

## The key idea

Every known nonmonotone lower-bound proof, read literally, exhibits a property `C_n` of a function's `2^n`-bit truth table that is **constructive** (efficiently decidable from the truth table) and **large** (held by a non-negligible fraction of all functions), and shows it is **useful** (it certifies that any function having it lies outside the target class). But a property that is constructive, large, and useful against a class `C` *is* an efficient statistical test separating truly random functions from functions in `C` — i.e. a distinguisher. If `C` is strong enough to compute a pseudorandom function (one built, via the Goldreich–Goldwasser–Micali tree, from a hard pseudorandom generator), that distinguisher breaks the generator. So a "natural" proof against a powerful class is self-defeating: it hands you an algorithm refuting the very hardness assumption it presupposes.

## Definitions

Let `F_n` be all Boolean functions on `n` variables, identified with their `2^n`-bit truth tables. A combinatorial property `C_n ⊆ F_n` is **natural** if it has a subset `C*_n` with:

- **Constructivity.** The predicate `f ∈ C*_n` is decidable in time polynomial in the truth-table size `2^n` (the default benchmark; the actual proofs decide it in `AC0`, `NC2`, `TC0`, or `P`). The general `Γ`-natural version asks `C*_n` to be computable in a class `Γ`.
- **Largeness.** `|C*_n| ≥ 2^{−O(n)} · |F_n|` (the default density `2^{−O(n)}`; usually it is close to `1`).

`C_n` is **useful against** a class `Δ` if: for every sequence `{f_n}` with `f_n ∈ C_n` infinitely often, `{f_n} ∉ Δ`. (Default `Δ = P/poly`.) A lower-bound proof is **`Γ`-natural against `Δ`** if it proceeds by exhibiting such a `C_n`.

For a generator `G_k : {0,1}^k → {0,1}^{2k}`, its **hardness** `H(G_k)` is the least `S` for which some circuit `C` of size `≤ S` achieves
`|Pr_x[C(G_k(x)) = 1] − Pr_y[C(y) = 1]| ≥ 1/S`, `x` uniform on `{0,1}^k`, `y` uniform on `{0,1}^{2k}`.

## Main theorem and proof

**Theorem (barrier).** There is no `P/poly`-natural property useful against `P/poly`, unless `H(G_k) ≤ 2^{k^{o(1)}}` for every length-doubling generator `G_k` computable in `P/poly`. In particular, if `2^{n^ε}`-hard one-way functions exist for some `ε > 0`, then no natural proof can show `SAT ∉ P/poly`; hence no natural proof can prove `P ≠ NP`.

*Proof.* Suppose `C_n` is `P/poly`-natural and useful against `P/poly`; take `C_n = C*_n` (constructive, large). Let `G : {0,1}^k → {0,1}^{2k}` be any `P/poly`-computable length-doubling generator, `ε > 0` arbitrary, `n = ⌈k^ε⌉`. Split `G(s) = (G_0(s), G_1(s))` into the two `k`-bit halves, and build the GGM pseudorandom function family `f : {0,1}^k → F_n`: for seed `x` and input `y = y_1 … y_n`, set `G_y = G_{y_n} ∘ … ∘ G_{y_1}` and let `f(x)(y)` be the first bit of `G_y(x)`.

1. Each `f(x)` is computable by `poly(2^n)`-size circuits. If generated functions landed in `C_n` for infinitely many lengths, selecting one seed at each such length would give a `P/poly` sequence that hits `C_n` infinitely often, contradicting usefulness. Equivalently, from the pointwise lower-bound direction `C_n(f)=1 => f ∉ P/poly`, the contrapositive for this generated family is `f ∈ P/poly => C_n(f)=0`. Thus, for all large `k`, `C_n(f(x)) = 0` for every seed `x`, and `C_n ∩ {f(x) : x ∈ {0,1}^k} = ∅`.
2. By largeness, `Pr[C_n(f_n) = 1] ≥ 2^{−O(n)}` for a uniform random function `f_n`. With step 1,
   `|Pr[C_n(f_n)=1] − Pr[C_n(f(x))=1]| ≥ 2^{−O(n)}`, and by constructivity `C_n` is a size-`2^{O(n)}` test. So `C_n` distinguishes a random function from the GGM family with bias `2^{−O(n)}`.
3. **Hybrid down the tree.** The GGM tree `T` of height `n` has `2^n − 1` internal nodes `v_1, …, v_{2^n−1}` (each child after its parent). Hybrid `i` uses the forest formed by `v_1, …, v_i` together with all leaves: every forest root gets an independent uniform `k`-bit label, and labels below that root are generated by `G`. Hybrid `0` has independent leaf labels, hence a uniform random function; hybrid `2^n−1` has only the global root random, hence the GGM function `f(x)`. The endpoint bias `δ = 2^{−O(n)}` telescopes across `2^n−1` steps, so some adjacent pair is separated by at least `δ/(2^n−1) = 2^{−O(n)}`. Fix all labels except the changing node to preserve that adjacent bias. The adjacent hybrids differ only in whether the two child-subtree roots are independent uniform `k`-bit strings or the two halves of `G(x_v)` for a uniform parent label `x_v`. So the test becomes a size-`2^{O(n)}` circuit distinguishing `G` on a uniform seed from a uniform `2k`-bit string with bias `2^{−O(n)}`.
4. Hence `H(G_k) ≤ 2^{O(n)} = 2^{O(k^ε)}`. Since `ε > 0` was arbitrary, for every fixed `δ > 0` one can choose `ε < δ` and get `H(G_k) ≤ 2^{k^δ}` for all large `k`; equivalently `H(G_k) ≤ 2^{k^{o(1)}}`. If some generator is `2^{k^δ}`-hard (`δ > 0` fixed), contradiction. ∎

**Unconditional corollary (discrete log).** For a half-exponential `t` (nondecreasing, `t^{−1}(n^C) ≤ o(log t(n))` for all `C`), no large `P/poly`-constructive property useful against `SIZE(t(n))` holds for infinitely many discrete-log hard-bit functions `B_{p,g}(x) = [log_g x ≤ (p−1)/2]`. Let `k = ⌈log p⌉`. If such a property contained infinitely many `B_{p,g}`, usefulness gives `size(B_{p,g}) ≥ t(k)` along an infinite subsequence. For the standard Blum-Micali length-doubling generator `G : {0,1}^{2k} → {0,1}^{4k}` based on `B_{p,g}`, the hard-bit analysis gives `size(B_{p,g}) ≤ (H(G)+k)^{O(1)}`, hence `t(k) ≤ (H(G)+k)^{O(1)}`. Convert `G` to a GGM family on `n = t^{−1}(k^{C+1})+1` variables. The half-exponential condition gives `t(k)>k^{C+1}` for almost all `k`; otherwise `k ≤ t^{−1}(k^{C+1}) ≤ o(log t(k)) ≤ o(log k)`. Hence `n ≤ k`, and the GGM image has circuit size at most `(k+n)^C ≤ k^{C+1} < t(n)`, so usefulness rejects the image. Largeness accepts random functions, and the forest hybrid yields `H(G) ≤ 2^{O(n)}`. Since largeness also forces `t(n) ≤ 2^{O(n)}` for almost all `n`, `t(n)>k^{C+1}` gives `log k = O(n)`. Then `n = t^{−1}(k^{C+1})+1 ≤ o(log t(k)) ≤ o(log H(G)+log k) ≤ o(n)`, contradiction. So no natural proof shows discrete log needs more than half-exponential circuit size.

## Largeness is forced by formal complexity measures

A **formal complexity measure** is integer-valued, with `μ(f ∨ g), μ(f ∧ g) ≤ μ(f) + μ(g)` and cheap variables and negated variables (formula size is the prototype; approximation measures with an extra `+1` fit after shifting by `1`). **Theorem.** If `μ(f) = t` for one `f`, then `μ(g) ≥ t/4` for at least `1/4` of all `g` (and a `1−γ` fraction satisfy `μ(g) ≥ Ω(t/(n+log(1/γ))^2)−n`). Proof via Boolean-algebra representation: if `S = {g : μ(g) < t/4}` had `|S| > (3/4)|F_n|`, then any `f` is `(s_1 ∧ s_2) ∨ (s_3 ∧ s_4)` with `s_i ∈ S` (a union bound over the four uniform elements `b, ¬b ⊕ b_0, ¬b, b ⊕ b_0`, each outside `S` with probability `< 1/4`), giving `μ(f) ≤ μ(s_1∧s_2)+μ(s_3∧s_4) ≤ Σ_i μ(s_i) < t` — contradiction. Hence any property of the form "`μ(f)` is large" automatically satisfies largeness; only **constructivity** is the binding constraint, and it is the half we understand least outside exponential time.

## Why the known techniques fall on each side

- **AC0 / AC0[`p`] proofs are natural** — their `C_n` (high sensitivity; large rank of `M(f)`; weak degree; discrepancy) is decidable in `AC0` / `NC2` / `P` / `TC0` and holds for a random function. They succeed only because `AC0`, `AC0[p]` are too weak to host a pseudorandom function (Nisan's `AC0`-secure generator ⇒ no `AC0`-natural proof against `AC0[2]`, so the parity proofs provably cannot reach the `AC0[p]` bounds). Plausible pseudorandom-function candidates already press down around `TC0` and `NC1`, for example through subset-sum-based constructions — exactly where the lower bounds went silent.
- **Counting arguments** violate **constructivity** ("`f ∉ P/poly`" is large but presumably not decidable near exponential time; the theorem says no constructive large subset can exist if generators do).
- **Diagonalization** violates **largeness** (it targets the non-generic property of being EXP-complete) — which is why it evades this barrier, at the cost of relativizing.

## Takeaway

Any constructive, large lower-bound proof against a class `C` simultaneously proves `C` contains no strong pseudorandom function and yields an algorithm breaking any such function — a duality between impossibility proofs and algorithms. To separate `P` from `NP` one needs an *unnatural* proof: a property too special to be large (the diagonalization route) or too complex to be constructive. The barrier does not say lower bounds are hopeless; it says where not to look, and why the program stalled exactly at the threshold where circuits first become strong enough to compute pseudorandom functions.
