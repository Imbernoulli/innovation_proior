The argument begins with the halting problem because Busy Beaver inherits its force from the same source. A Turing machine can be described by a finite transition table. For any fixed number of states and symbols, there are finitely many possible tables. This finite enumeration makes the Busy Beaver question look deceptively ordinary: inspect all machines of size `n`, keep the ones that halt, and take the maximum output or runtime.

The obstruction is hidden in the phrase "keep the ones that halt." For any particular halting machine, simulation eventually verifies that it halts. For a non-halting machine, simulation never gives a final certificate by itself. Busy Beaver packages that asymmetry into a single number. `S(n)` is finite for every fixed `n`, but no algorithm can compute all its values.

The reduction is direct. Suppose an algorithm could compute `S(n)`. Given any `n`-state machine, simulate it for `S(n)` steps. If it halts within that bound, answer "halts." If it has not halted, answer "does not halt," because no halting `n`-state machine can run longer than the maximum. That would solve the halting problem for all finite machines after encoding size, which is impossible. So `S` cannot be computable. The same style of argument applies to `Sigma(n)`, since sufficiently much information about maximal finite output would also let us dominate computable prediction procedures.

The conceptual leap is that Busy Beaver turns a negative theorem into an extremal scale. The halting problem alone says no universal decision procedure exists. Busy Beaver asks how far finite computation can go before the undecidable part becomes unavoidable. It gives the boundary a numerical profile: at size `n`, there is some longest finite run, and knowing that number would settle every halting question in that finite class.

This is why Busy Beaver is more than a fast-growing function. Many functions grow quickly, but they remain computable if an algorithm can produce their values. Busy Beaver grows beyond every computable function because any computable upper prediction would eventually be beaten by a small halting machine engineered to outrun it and then stop. Its extremality is tied to description size, not just to large notation.

The useful phrasing for this method is: undecidability becomes comparable. Machines can be ranked by size; halting behavior can be summarized by maximal finite output or time; partial discoveries become lower bounds; exact values become finite classifications. The result is a concrete growth curve whose unknown values mark where computation, proof, and search stop being separable.
