I keep coming back to one nagging asymmetry, and I want to stare at it until it cracks. I have a counting random variable — say X is the number of copies of some configuration in a random structure, the number of k-cliques in a random graph, the number of distinct primes dividing a random integer. X is a nonnegative integer. I want to prove the configuration is *there*: Pr[X > 0] is close to 1. The tool everyone reaches for first is the expectation. Compute E[X] by linearity, and read off existence. And in one direction it's airtight. Markov says Pr(X ≥ α) ≤ E[X]/α; put α = 1 and Pr(X ≥ 1) ≤ E[X]. So if E[X] → 0 as my size parameter grows, then X is almost always 0 — the configuration almost surely does *not* appear. Clean, total, no fuss.

But the other direction is where I keep hitting the wall. I want: E[X] → ∞ implies X > 0 almost always. And that is just false. I have to be honest with myself about *why* it's false, not wave at it. Picture a variable that is gigantic on a rare event and zero otherwise. Concretely, let X be 0 with probability 1 − 1/n and equal to n² with probability 1/n. Then E[X] = n²·(1/n) = n → ∞, screaming to infinity, and yet Pr[X > 0] = 1/n → 0. The mean is large *because* of an enormous contribution from a vanishingly rare outcome; almost every time, X is just 0. So a positive expectation — even an exploding one — certifies nothing about whether the thing typically exists. Averaging tells me *some* outcome has X ≥ E[X], so an object at least as good as the mean exists somewhere in the sample space; it does not tell me the object is *usually* there. Existence-somewhere and existence-almost-surely are different claims, and the first moment only touches the first.

So what is the mean missing? In that bad example, the reason E[X] lies to me is that X is wildly spread out: it sits at 0 almost always and then spikes. If instead X were tightly clustered near its mean, and its mean is large, then it couldn't also be 0 most of the time — being 0 would be a *huge* deviation from a large mean, and a tightly clustered variable doesn't make huge deviations. So the missing quantity is the spread. The mean says where the mass would balance; I need a second number that says how far the mass typically strays from there. That's the variance, σ² = Var[X] = E[(X − µ)²], with µ = E[X].

Now I want the inequality that converts "small spread" into "stays near the mean," and it's a two-line consequence of Markov applied to the squared deviation. (X − µ)² is nonnegative, so Pr(|X − µ| ≥ α) = Pr((X − µ)² ≥ α²) ≤ E[(X − µ)²]/α² = Var[X]/α². Writing α = λσ that's Pr[|X − µ| ≥ λσ] ≤ 1/λ². There it is — Chebyshev. A variable is within λ standard deviations of its mean except with probability 1/λ². The only input beyond the mean is one number, the variance, and that's exactly the spread I said was missing.

Let me aim it directly at the question I care about. The event I'm afraid of is X = 0. If µ = E[X] > 0, then X = 0 means |X − µ| = µ, so X = 0 is contained in the event |X − µ| ≥ µ. Chebyshev with α = µ hands me

  Pr[X = 0] ≤ Pr[|X − µ| ≥ µ] ≤ σ²/µ² = Var[X]/E[X]².

That's the whole engine in one line. The probability the configuration is *absent* is at most the variance divided by the mean squared. So the criterion for existence is not "is the mean large" — it's "is the variance small *relative to the mean squared*." If Var[X] = o(E[X]²), then Pr[X = 0] → 0, i.e. X > 0 almost always. The mean being large is not even part of the statement; what matters is the ratio. (Of course in practice the mean has to be growing for the variance to be o of its square in any interesting count, but the *logical* content is the ratio.)

And I get more than nonzero for free. Run the same bound but on the event |X − µ| ≥ εµ for a fixed ε > 0: Pr[|X − E[X]| ≥ εE[X]] ≤ Var[X]/(ε²E[X]²). If Var[X] = o(E[X]²) this → 0 for every ε, which says X/E[X] → 1 in probability — X ∼ E[X] almost always. So the same hypothesis that forces existence also forces *concentration*: not just "at least one copy" but "about E[X] copies, almost surely." Good. So the entire program is: compute E[X], compute Var[X], check Var = o(E[X]²).

The catch is computing Var[X] for a combinatorial count, because the count is a sum of dependent indicators and dependence is the whole game. Write X = X₁ + ⋯ + X_m with Xᵢ = 1_{Aᵢ}, the indicator that the i-th potential copy is present, pᵢ = Pr[Aᵢ]. Then

  Var[X] = Σᵢ Var[Xᵢ] + Σ_{i≠j} Cov[Xᵢ, Xⱼ],   Cov[Y,Z] = E[YZ] − E[Y]E[Z],

the second sum over ordered pairs. Two simplifications fall out and they're the reason this is tractable. First, for an indicator Var[Xᵢ] = pᵢ(1 − pᵢ) ≤ pᵢ = E[Xᵢ], so the diagonal Σᵢ Var[Xᵢ] ≤ Σᵢ E[Xᵢ] = E[X] — the entire diagonal is dominated by the mean and costs me nothing. Second, if Aᵢ and Aⱼ are *independent* then Cov[Xᵢ, Xⱼ] = 0; so only *dependent* pairs survive in the off-diagonal sum. For dependent pairs, Cov[Xᵢ, Xⱼ] = E[XᵢXⱼ] − E[Xᵢ]E[Xⱼ] ≤ E[XᵢXⱼ] = Pr[Aᵢ ∧ Aⱼ]. Write i ∼ j when i ≠ j and Aᵢ, Aⱼ are dependent, and collect

  Δ = Σ_{i ∼ j} Pr[Aᵢ ∧ Aⱼ].

Then Var[X] ≤ E[X] + Δ. So the whole second-moment computation reduces to one quantity: the sum, over dependent ordered pairs, of the probability that *both* copies are present. The criterion Var = o(E[X]²) becomes: E[X] → ∞ and Δ = o(E[X]²). Under those two, Pr[X = 0] ≤ Var/E[X]² ≤ (E[X] + Δ)/E[X]² = 1/E[X] + Δ/E[X]² → 0, so X > 0 — and X ∼ E[X] — almost always.

This is already usable, but in the cases I care about there's a symmetry I should exploit, because computing Δ as a raw double sum is painful. The copies all "look the same": for any i ≠ j there's a measure-preserving map of the probability space that permutes the events and sends Aᵢ to Aⱼ. Then in Δ = Σ_{i∼j} Pr[Aᵢ ∧ Aⱼ] = Σᵢ Σ_{j∼i} Pr[Aᵢ] Pr[Aⱼ | Aᵢ], the inner sum Σ_{j∼i} Pr[Aⱼ | Aᵢ] doesn't depend on which i I fixed. Call it Δ* = Σ_{j∼i} Pr[Aⱼ | Aᵢ] for any fixed i. Then Δ = Σᵢ Pr[Aᵢ] Δ* = Δ* Σᵢ Pr[Aᵢ] = Δ*·E[X]. So Δ = o(E[X]²) becomes the much cleaner Δ* = o(E[X]): if E[X] → ∞ and Δ* = o(E[X]) then X > 0 almost always, and X ∼ E[X]. And Δ* has a meaning I can feel — fix one copy as present; Δ* is the expected number of *other* dependent copies that conditioning forces along with it. The condition says: conditioning on any one copy being there must not substantially inflate the expected count of the rest. If it doesn't, the count can't be concentrated on a rare spike, so it's positive almost surely.

Let me test the whole apparatus on something concrete and old, where I can check every factor — the number of distinct prime divisors of a random integer. Pick x uniformly from {1, …, n}. For a prime p, let X_p = 1 if p | x, else 0. The fraction of {1,…,n} divisible by p is ⌊n/p⌋/n, and since y − 1 < ⌊y⌋ ≤ y, E[X_p] = ⌊n/p⌋/n = 1/p + O(1/n). I don't want to sum over all primes up to n — the tiny ones near n contribute mush — so cut at M = n^{1/10} and set X = Σ_{p ≤ M} X_p. No x ≤ n has more than 10 prime factors exceeding M (since (n^{1/10})^{11} > n), so ν(x) − 10 ≤ X(x) ≤ ν(x); a deviation bound on X transfers to ν up to an additive 10, which is negligible against what's coming. By linearity, E[X] = Σ_{p ≤ M}(1/p + O(1/n)) = Σ_{p ≤ M} 1/p + O(1). And Σ_{p ≤ x} 1/p = ln ln x + O(1) (Mertens, the prime-harmonic sum), so E[X] = ln ln n + O(1). The mean is ln ln n — that's the Hardy–Ramanujan "normal order," dropping straight out of linearity.

Now the variance, which is where the method earns its keep. Var[X] = Σ_{p ≤ M} Var[X_p] + Σ_{p ≠ q} Cov[X_p, X_q]. The diagonal: Var[X_p] = (1/p)(1 − 1/p) + O(1/n), so Σ_{p ≤ M} Var[X_p] = Σ_{p ≤ M} 1/p + O(1) = ln ln n + O(1) — same size as the mean. The off-diagonal is the delicate part, and it's beautiful that it's *small*. For distinct primes p, q, X_p X_q = 1 iff p | x and q | x iff pq | x. So E[X_p X_q] = ⌊n/pq⌋/n and

  Cov[X_p, X_q] = ⌊n/pq⌋/n − (⌊n/p⌋/n)(⌊n/q⌋/n).

Bound it: ⌊n/pq⌋/n ≤ 1/pq, and ⌊n/p⌋/n ≥ (1/p − 1/n), ⌊n/q⌋/n ≥ (1/q − 1/n), so

  Cov[X_p, X_q] ≤ 1/pq − (1/p − 1/n)(1/q − 1/n) = 1/pq − 1/pq + (1/n)(1/p + 1/q) − 1/n² ≤ (1/n)(1/p + 1/q).

The leading 1/pq cancels — exactly what I'd hope, since X_p and X_q are "almost independent": divisibility by p and by q are nearly independent events for a random integer, and the small correlation is only the floor-function discretization. Summing,

  Σ_{p ≠ q} Cov[X_p, X_q] ≤ (1/n) Σ_{p ≠ q}(1/p + 1/q) ≤ (2M/n) Σ_{p ≤ M} 1/p,

since each 1/p appears with at most M − 1 < M partners q. And (2M/n)Σ_{p≤M}1/p = O((2 n^{1/10}/n)·ln ln n) = O(n^{−9/10} ln ln n) = o(1). I also need to know a negative covariance sum is not hiding at the same scale. Write ⌊n/r⌋/n = 1/r − θ_r/n with 0 ≤ θ_r < 1. Then

  Cov[X_p, X_q] = −θ_{pq}/n + θ_p/(nq) + θ_q/(np) − θ_pθ_q/n² ≥ −1/n − 1/n².

There are at most M² ordered pairs, so the total negative contribution is at worst O(M²/n) = O(n^{−4/5}) = o(1). The entire off-diagonal sum is therefore o(1): the dependencies contribute nothing. Var[X] = ln ln n + O(1). Now Chebyshev: Pr[|X − ln ln n| > λ√(ln ln n)] ≤ Var[X]/(λ²·ln ln n) = (ln ln n + O(1))/(λ² ln ln n) = λ^{−2} + o(1). And |ν − X| ≤ 10 ≪ √(ln ln n), so the same holds for ν. Almost all integers up to n have ν(x) within a few √(ln ln n) of ln ln n — Hardy–Ramanujan, but now the proof is just "compute E and Var, dependencies vanish, apply Chebyshev." The thing that made the hard 1917 argument is, in this language, the single line Σ Cov = o(1).

Now the structure I actually came for: random graphs. G(n,p) — each of the C(n,2) possible edges present independently with probability p. I want to know when a fixed configuration appears almost surely, and I suspect there's a *threshold*: a rate r(n) such that below it the configuration almost never appears and above it almost always does. The disappearance half is first moment; the appearance half is exactly the second-moment criterion I just built. Start small and explicit: when does G(n,p) almost surely contain a 4-clique?

For each 4-set S of vertices let A_S be the event "S is a clique" and X_S its indicator. A 4-clique needs all C(4,2) = 6 edges, so E[X_S] = Pr[A_S] = p⁶. Let X = Σ_{|S|=4} X_S be the number of 4-cliques; ω(G) ≥ 4 iff X > 0. Linearity: E[X] = C(n,4) p⁶ ∼ n⁴p⁶/24. To make E[X] = Θ(1) I balance n⁴ against p⁶: n⁴p⁶ = Θ(1) means p = Θ(n^{−4/6}) = Θ(n^{−2/3}). That's my candidate threshold. If p ≪ n^{−2/3}, then E[X] = o(1), so X = 0 almost always — no 4-clique. The disappearance half is done by the mean alone.

For the appearance half, take p ≫ n^{−2/3} so E[X] → ∞, and I need Δ* = o(E[X]). All 4-sets look the same, so the events are symmetric and Δ* is legitimate. Fix S. Which T are dependent on S? A_T and A_S share randomness exactly when S and T share an *edge*, i.e. |S ∩ T| ≥ 2; and if |S ∩ T| ∈ {0,1} they share no edge and are independent. So S ∼ T iff |S ∩ T| = 2 or 3. Split Δ* by overlap.

Overlap |S ∩ T| = 2: choosing T means choosing the 2 vertices outside S, so there are O(n²) such T. Given A_S, the shared edge (the one inside S ∩ T) is already present; T's clique needs its other 6 − 1 = 5 edges, each present with probability p, so Pr[A_T | A_S] = p⁵. Contribution O(n²p⁵).

Overlap |S ∩ T| = 3: T is fixed once I pick its 1 vertex outside S, so O(n) such T. Given A_S, the triangle on S ∩ T is already present — that's C(3,2) = 3 of T's edges — so T needs 6 − 3 = 3 more edges, Pr[A_T | A_S] = p³. Contribution O(np³).

So Δ* = O(n²p⁵) + O(np³). Compare each to E[X] ∼ n⁴p⁶. First: n²p⁵/(n⁴p⁶) = 1/(n²p). With p ≫ n^{−2/3}, n²p ≫ n²·n^{−2/3} = n^{4/3} → ∞, so this → 0. Second: np³/(n⁴p⁶) = 1/(n³p³) = 1/(np)³; np ≫ n·n^{−2/3} = n^{1/3} → ∞, so this → 0 as well. Hence Δ* = o(E[X]), the criterion holds, and X > 0 almost always: above n^{−2/3} the 4-clique appears. Together with the disappearance half, n^{−2/3} is the threshold for ω(G) ≥ 4. The whole thing turned on two ratios, and I checked both factors — the cancellation is real, not hoped for.

That calculation of Δ* looks fortuitous, so let me see what made it work and lift it to a general subgraph H with v vertices and e edges. The clean way to compare a config's external cost is its *density* ρ(H) = e/v. For each v-set S let A_S be the event that G restricted to S contains a copy of H, X_S its indicator, X = Σ_{|S|=v} X_S. A particular placement of H on S needs its e edges, probability pᵉ; there are at most v! placements, so pᵉ ≤ Pr[A_S] ≤ v! pᵉ, and E[X] = Σ_{|S|=v} Pr[A_S] = Θ(n^v pᵉ). Balancing n^v pᵉ = Θ(1) gives p = n^{−v/e} = n^{−1/ρ(H)} as the candidate threshold. Below it E[X] = o(1) ⇒ X = 0 a.a.s.

Above it, E[X] → ∞ and I need Δ* = o(E[X]). Fix S; S ∼ T iff S, T share an edge, i.e. |S ∩ T| = i with 2 ≤ i ≤ v − 1. Split by overlap:

  Δ* = Σ_{i=2}^{v−1} Σ_{|T∩S|=i} Pr[A_T | A_S].

For each i there are O(n^{v−i}) choices of T. Now the crucial step, and the place where I'd hit a wall if I weren't careful. Given A_S, how cheap is A_T? I should not pretend the conditioning names a single copy of H inside S; A_S is a union over O(1) placements. But Pr[A_S] = Θ(pᵉ), and a union bound over the O(1) pairs of placements in S and T only changes constants. For a fixed pair of placements, the T-copy has some of its edges inside S ∩ T — those are "free," already paid for by the S-copy — and the rest, outside, cost a factor of p each. If H had a very dense piece sitting inside the overlap, many edges would be free and the conditional probability would be too large, blowing up Δ*. This is exactly where I need **H balanced** — every subgraph H′ of H has density ρ(H′) ≤ ρ(H) = e/v. The part of the T-copy inside the i shared vertices is a subgraph of H on at most i vertices, so by balance it has at most i·(e/v) = ie/v edges inside S ∩ T. Hence at least e − ie/v edges lie outside and must be paid for: Pr[A_T | A_S] = O(p^{e − ie/v}). Therefore

  Δ* = Σ_{i=2}^{v−1} O(n^{v−i} p^{e − ie/v}) = Σ_{i=2}^{v−1} O((n^v pᵉ)^{1 − i/v}),

because n^{v−i} p^{e − ie/v} = (n^v pᵉ)^{1 − i/v}: the exponent of n is v − i = v(1 − i/v) ✓, and the exponent of p is e − ie/v = e(1 − i/v) ✓. Since n^v pᵉ → ∞ and each i ≥ 2 gives an exponent 1 − i/v < 1, every term is o(n^v pᵉ) = o(E[X]). So Δ* = o(E[X]), the criterion applies, and the copy of H appears almost surely above p = n^{−v/e}. The 4-clique calculation also shows what this general estimate is doing. For i = 2 it gives O((n⁴p⁶)^{1/2}) = O(n²p³), not the sharper O(n²p⁵), because balance only says a 2-vertex overlap has at most ie/v = 3 free edges; the actual K₄ overlap has exactly 1. For i = 3 it gives O((n⁴p⁶)^{1/4}) = O(np^{3/2}), again looser than the explicit O(np³). Both coarse ratios still vanish above p ≫ n^{−2/3}: n²p³/(n⁴p⁶) = 1/(n²p³) → 0 and np^{3/2}/(n⁴p⁶) = 1/(n³p^{9/2}) → 0. The explicit clique count just uses more structure than the all-balanced-graphs bound.

I should check the balanced hypothesis is *necessary*, not just convenient — otherwise n^{−v/e} might still be the threshold for unbalanced H and I'm over-restricting. Suppose H is *not* balanced: it has a subgraph H₁ with v₁ vertices, e₁ edges, and strictly larger density e₁/v₁ > e/v. Pick α with v₁/e₁ < α < v/e, and set p = n^{−α}. The expected number of copies of H₁ is Θ(n^{v₁} p^{e₁}) = Θ(n^{v₁ − α e₁}); since α > v₁/e₁, the exponent v₁ − α e₁ < 0, so this → 0, and almost surely G has *no* copy of H₁. But H contains H₁, so no copy of H₁ means no copy of H. Yet α < v/e means p = n^{−α} ≫ n^{−v/e} — I'm *above* the would-be threshold and H still doesn't appear. So n^{−v/e} is not the threshold when H is unbalanced; the densest subgraph governs. The actual threshold for general H is n^{−v₁/e₁} where H₁ is the densest subgraph — the bottleneck is the hardest-to-build dense piece, which has to show up first. That's the Erdős–Rényi subgraph-appearance answer, and it explains *why* "balanced" was the right hypothesis: for balanced H the whole graph is its own densest piece, so the global density e/v is the bottleneck and n^{−v/e} is exactly the threshold.

And above threshold I get the count, not just appearance. For H balanced with a automorphisms, label the v vertices of a placement; the indicator that an ordered v-tuple (x₁,…,x_v) carries a copy is I_{x₁,…,x_v}, and identifying tuples that differ by an automorphism of H, X = number of copies = Σ over one representative per class. There are (n)_v / a classes (falling factorial over automorphisms), each placement has probability pᵉ, so E[X] = ((n)_v / a) pᵉ ∼ n^v pᵉ / a. The same Δ* = o(E[X]) argument (the overlap-i terms are o(E[X]) exactly as above, plus the |T| = |S| term where T is another placement on the same vertex set, contributing only O(1) = o(E[X])) gives X ∼ n^v pᵉ / a almost always. Above its threshold, the number of copies is concentrated at n^v pᵉ / a. And more generally, for *any* fixed H, if p is large enough that E[X_{H′}] → ∞ for every subgraph H′ (so every dense piece is well above its own threshold), then the count of copies of H is concentrated at its mean — the overlap-i contribution to Δ* is, up to constants, n^{v−w} p^{e−f} = Θ(E[X_H]/E[X_{H′}]) for the subgraph H′ on the overlap, which is o(E[X_H]) precisely because E[X_{H′}] → ∞.

There's still a regime the o(E[X]²) criterion can't quite reach, and I want to name it before I leave the topic. Chebyshev forces X > 0 with probability → 1 only when Var = o(µ²). What if Var is merely O(µ²) — comparable to, not smaller than, the mean squared? Then σ²/µ² doesn't tend to 0, and Chebyshev gives nothing better than "X > 0 with probability at least 1 − C for some constant C < 1" — possibly vacuous if C ≥ 1. But I shouldn't need the *full* second-moment cancellation just to get a *constant* probability of existence. Let me push on the nonnegativity of X directly. X ≥ 0, and I'll split its mean by where it sits relative to a fraction t of its mean. Take any t ∈ [0,1]:

  E[X] = E[X·1_{X ≤ tE[X]}] + E[X·1_{X > tE[X]}].

The first term is at most tE[X], since on that event X ≤ tE[X]. For the second term, Cauchy–Schwarz on the product X·1_{X > tE[X]}:

  E[X·1_{X > tE[X]}] ≤ √(E[X²]) · √(E[1²_{X > tE[X]}]) = √(E[X²]) · √(Pr[X > tE[X]]).

Put them together: E[X] ≤ tE[X] + √(E[X²])·√(Pr[X > tE[X]]), so √(E[X²])·√(Pr[X > tE[X]]) ≥ (1 − t)E[X], and squaring,

  Pr[X > tE[X]] ≥ (1 − t)² E[X]² / E[X²].

That's the Paley–Zygmund inequality — a reverse Chebyshev, lower-bounding the chance X is a constant fraction of its mean using only the first two moments. Let me sanity-check the t = 0 case: Pr[X > 0] ≥ E[X]²/E[X²]. And E[X²] = Var[X] + E[X]², so Pr[X > 0] ≥ µ²/(µ² + σ²) = 1/(1 + σ²/µ²). When σ² = o(µ²) this → 1, recovering the Chebyshev conclusion. But when σ² = O(µ²), say σ² ≤ Cµ², this gives Pr[X > 0] ≥ 1/(1 + C) — a *constant*, bounded-away-from-zero probability of existence, even though Chebyshev couldn't promise high probability. That's the refinement I wanted: small variance relative to the mean squared forces existence with high probability; merely *comparable* variance still forces it with constant probability. For a graph property that constant-probability foothold is often the start of a sharper argument that boosts it to probability → 1.

So the picture is whole. A positive — even a diverging — expectation, by itself, guarantees nothing about whether an object typically exists; the cautionary spike with E[X] → ∞ and Pr[X > 0] → 0 makes that precise. The one extra statistic that closes the gap is the variance: Chebyshev turns Var[X] = o(E[X]²) into Pr[X = 0] ≤ Var/E[X]² → 0, hence X > 0 and in fact X ∼ E[X] almost always. For a count X = Σ 1_{Aᵢ} the variance reduces to E[X] plus the dependency sum Δ = Σ_{i∼j} Pr[Aᵢ ∧ Aⱼ], and under symmetry to Δ*·E[X], so the criterion becomes the legible condition Δ* = o(E[X]): conditioning on one copy must not inflate the expected number of others. Setting E[X] = Θ(1) pins the threshold — n^{−v/e} for a balanced subgraph H, n^{−2/3} for the 4-clique — with the first moment killing the property below and the second moment switching it on above. And when the variance is only comparable to the mean squared, Paley–Zygmund still extracts a constant probability of existence from the same two moments. That is the engine behind appearance thresholds: not the mean, but the mean *together with* a variance that is small beside it.
