So what exactly governs "the surrogate stops being valid"? Distance between π_θ and π_old. Let me dig into why, because the answer should point at the fix. Kakade and Langford worked this out years ago: how does performance change when you swap π_old for a new π? There is an exact identity, η(π) − η(π_old) = E_{τ∼π}[ Σ_t γ^t A_{π_old}(s_t, a_t) ] — the expected advantage of the *old* policy, but taken under the *new* policy's own trajectory distribution. The advantage telescopes: A_{π_old}(s,a) = E_{s'}[ R + γ V_{π_old}(s') − V_{π_old}(s) ]. Sum γ^t of that along a trajectory drawn from π and the consecutive value terms cancel in a telescope — −V(s_0) + γV(s_1) − γV(s_1) + γ²V(s_2) − γ²V(s_2) + ... — leaving Σ γ^t R_t minus V_{π_old}(s_0); take the expectation over π's start state and trajectories and the first sum is η(π) while the leftover is E[V_{π_old}(s_0)] = η(π_old). So the identity is real. But it is useless directly, because the expectation is over π's *own* state distribution — the very thing I am trying to produce, which I do not have yet. So they approximate: evaluate that expected advantage under the *old* state distribution instead, giving the surrogate L_{π_old}(π) = η(π_old) + E_{s∼ρ_old, a∼π}[ A_{π_old}(s,a) ]. The two agree exactly to first order at π = π_old, because at that point the state distributions coincide and their derivatives match. And — this is the part that makes the swap usable — the error of the swap is bounded by a term that grows with how far π strays from π_old, a total-variation, hence KL, distance, times a constant: η(π) ≥ L_{π_old}(π) − C·max_s KL[π_old(·|s), π(·|s)]. So if I improve the surrogate by more than that penalty while keeping the KL small, the true η actually goes up — a genuine *lower bound* on improvement, not just a first-order proxy. Closeness is then not a heuristic safety rail; it is the precise condition under which improving the surrogate provably improves η. So the real job is to turn "stay close" into something I can compute, and, with luck, into something first-order.

Let me rewrite the surrogate into a form I can estimate from the samples I actually have. The "a∼π" part is annoying because my data was drawn from π_old. This is exactly the problem TRPO already solved, so let me look hard at how, and at why I do not just want to use it. TRPO maximizes that same Ê_t[ r_t(θ) Â_t ] but slaps on a hard constraint, Ê_t[ KL[π_old(·|s_t), π_θ(·|s_t)] ] ≤ δ. That KL ball is the trust region; it directly bounds the distributional move, which is what keeps the surrogate trustworthy. And how does it solve a constrained problem over a million-parameter net? F is gigantic so you never form it; you run conjugate gradient to solve F x = g using only Fisher-vector products, each of which is a couple of extra backprops, and then you do a backtracking line search to make sure the *actual* KL constraint holds and the surrogate truly improved. It works, it is reliable. But look at everything I just listed: conjugate gradient, Fisher-vector products, a line search, a second-order approximation. That is a custom solver bolted onto my training loop. Worse, the KL/Fisher story gets murky the instant my policy and value function *share* a network, or there is dropout, or auxiliary heads — the KL is defined on the policy's output distribution, and sharing and stochastic units smear exactly that distribution, so the Fisher matrix no longer measures what I think it measures. TRPO itself hints at an escape and then closes it, which is interesting. Let me not abandon the KL-penalty idea entirely, though, because it is an honest alternative and I want to know whether I am leaving something on the table. Now the surrogate is only half a training step — I still need the advantages Â_t, I need to fit the value function, and I should think about exploration. For the advantage I want low variance without too much bias, and I want something that works with short fixed-length rollouts so I can run many actors in parallel the way A3C/A2C does: launch N actors, each rolls its own environment for a fixed T steps with T much shorter than an episode, and I pool the NT samples into one batch. The value function: I will regress V_θ(s) toward the returns with a squared-error loss L^{VF} = (V_θ(s_t) − V_t^{targ})², where the target is just V_t^{targ} = Â_t + V_old(s_t), the advantage added back to the old value estimate (so the target is consistent with the GAE I just computed). There is a small thing nagging me about the value loss that is worth a minute. The whole reason the policy surrogate is clipped is that, across K epochs, the new network drifts from the data-generating one and large moves are destructive. But the value head is on the *same* drifting network and gets the *same* K epochs — so a single minibatch could yank V_θ(s) far from the V_old(s) the rollout recorded, an analogous overshoot. By symmetry with the policy clip, I can clip the value update too: instead of plain (V_θ − V^{targ})², take the max of the unclipped squared error and a version where V_θ is constrained to V_old ± ε before squaring, max((V_θ − V^{targ})², (clip(V_θ, V_old−ε, V_old+ε) − V^{targ})²). Same pessimism logic — never let the clipped form *reduce* the loss, only cap how far the value prediction is rewarded for moving per update. It mirrors the policy clip exactly and costs nothing to add, so I will include it; whether it actually helps is the kind of thing only experiments will settle, but the symmetry is clean enough that it belongs in the default.

A few more knobs I should pin down with reasons, not just numbers, because on RL these "details" are the difference between learning and not. Initialization: orthogonal, but with layer-specific gains. Hidden layers get gain √2 — that is the gain that preserves the variance of activations and gradients through a tanh/ReLU-type nonlinearity, so a two-hidden-layer net does not see its signal shrink or blow up before it has learned anything. For the *policy output* layer I want the agent to begin by exploring roughly uniformly rather than being born over-confident about some arbitrary action and then having to claw back out of a bad deterministic basin — so I want the initial action distribution near-uniform / centered, all logits (or means) close together, which means a tiny output gain, 0.01. The *value output* layer gets gain 1.0 — no reason to shrink it, it is a plain regression head. Adam's ε: RL gradients can get very small, and a denominator floor that is too tiny lets the effective step blow up when the second-moment estimate is near zero, so I want a larger floor — I set ε to 1e-5 rather than the framework default 1e-8 to keep the update well-conditioned. Learning-rate annealing: linearly decay the step from its initial value to 0 over the run, so early on, when the policy is bad and any reasonable direction helps, I take big steps, and late, when I am fine-tuning near a good policy, the steps shrink so I do not jitter back out. Advantage normalization: subtract the mean and divide by the std of Â within each minibatch before the surrogate — this keeps the effective gradient scale (and hence the effective step size) roughly constant across tasks and across training, no matter how the raw reward scale or the value-function accuracy changes, which is the same robustness ε buys me on the ratio side. I do it per minibatch rather than per whole batch just because that is where the loss is computed; it makes the update's advantage signal unit-scale locally. And a global gradient-norm clip at 0.5: even with the surrogate clip, an occasional minibatch can produce a spiky gradient, and rescaling the whole gradient vector to a max norm is a cheap last line of defense against one bad step — it does not change the direction, only caps the length. The Gaussian's log-std is *state-independent*, a single learned vector shared across all states, rather than a head that outputs σ(s): a state-dependent σ would let the policy quietly drive exploration to zero in specific states early, defeating exploration where it might matter, whereas one global σ that the optimizer anneals keeps the exploration honest and is one fewer thing to destabilize.

```python
def layer_init(layer, std=np.sqrt(2), bias_const=0.0):
    # orthogonal init: sqrt(2) on hidden layers preserves signal through tanh;
    # the caller passes std=0.01 for the policy head (start near-uniform/centered)
    # and std=1.0 for the value head (plain regression output).
    nn.init.orthogonal_(layer.weight, std)
    nn.init.constant_(layer.bias, bias_const)
    return layer

# policy + value network (separate heads; 2x64 tanh MLP, the standard for MuJoCo)
class Agent(nn.Module):
    def __init__(self, obs_dim, act_dim):
        super().__init__()
        self.critic = nn.Sequential(
            layer_init(nn.Linear(obs_dim, 64)), nn.Tanh(),
            layer_init(nn.Linear(64, 64)), nn.Tanh(),
            layer_init(nn.Linear(64, 1), std=1.0))
        self.actor_mean = nn.Sequential(
            layer_init(nn.Linear(obs_dim, 64)), nn.Tanh(),
            layer_init(nn.Linear(64, 64)), nn.Tanh(),
            layer_init(nn.Linear(64, act_dim), std=0.01))
        # state-independent log std: the Gaussian policy's own exploration knob,
        # one vector for all states so exploration can't silently die per-state.
        self.actor_logstd = nn.Parameter(torch.zeros(1, act_dim))

    def get_value(self, x):
        return self.critic(x)

    def get_action_and_value(self, x, action=None):
        mean = self.actor_mean(x)
        std = self.actor_logstd.expand_as(mean).exp()
        dist = Normal(mean, std)
        if action is None:
            action = dist.sample()
        # sum over action dims: log pi(a|s), entropy, and V(s)
        return action, dist.log_prob(action).sum(1), dist.entropy().sum(1), self.critic(x)

# eps=1e-5 (not the 1e-8 default): a larger denominator floor keeps Adam steps
# well-conditioned when RL gradients get small.
agent = Agent(obs_dim, act_dim)
optimizer = torch.optim.Adam(agent.parameters(), lr=3e-4, eps=1e-5)

# ---- one iteration ----
# learning-rate anneal: big steps early, small steps late.
frac = 1.0 - (iteration - 1.0) / num_iterations
optimizer.param_groups[0]["lr"] = frac * 3e-4

# rollout already filled obs/actions/logprobs/rewards/dones/values for N*T steps.

# 1) advantages via truncated GAE: A_t = sum_l (gamma*lam)^l delta_{t+l},
#    delta_t = reward + gamma*V'*(1-done) - V ; (1-done) masks the bootstrap across resets.
with torch.no_grad():
    next_value = agent.get_value(next_obs).reshape(1, -1)
    advantages = torch.zeros_like(rewards)
    lastgaelam = 0
    for t in reversed(range(num_steps)):
        if t == num_steps - 1:
            nextnonterminal = 1.0 - next_done
            nextvalues = next_value
        else:
            nextnonterminal = 1.0 - dones[t + 1]
            nextvalues = values[t + 1]
        delta = rewards[t] + gamma * nextvalues * nextnonterminal - values[t]
        advantages[t] = lastgaelam = delta + gamma * gae_lambda * nextnonterminal * lastgaelam
    returns = advantages + values          # value targets V_t^targ = A_t + V_old(s_t)

# flatten the N*T batch
b_obs, b_logprobs = obs.reshape(-1, obs_dim), logprobs.reshape(-1)
b_actions, b_advantages = actions.reshape(-1, act_dim), advantages.reshape(-1)
b_returns, b_values = returns.reshape(-1), values.reshape(-1)

# 2) K epochs of minibatch updates on the SAME batch -- the data reuse that started all this
b_inds = np.arange(batch_size)
for epoch in range(update_epochs):
    np.random.shuffle(b_inds)
    for start in range(0, batch_size, minibatch_size):
        mb = b_inds[start:start + minibatch_size]
        _, newlogprob, entropy, newvalue = agent.get_action_and_value(b_obs[mb], b_actions[mb])

        # probability ratio r_t(theta) = pi_theta / pi_old ; r = 1 on the first epoch.
        # work in log space then exp for numerical stability.
        logratio = newlogprob - b_logprobs[mb]
        ratio = logratio.exp()

        # normalize advantages per minibatch: keeps the effective step scale constant
        # across tasks/training regardless of reward or value scale.
        mb_adv = b_advantages[mb]
        mb_adv = (mb_adv - mb_adv.mean()) / (mb_adv.std() + 1e-8)

        # clipped surrogate L^CLIP = E[min(r*A, clip(r,1-e,1+e)*A)];
        # we minimize the negative, so the pessimistic min becomes a max of two negatives:
        pg_loss1 = -mb_adv * ratio
        pg_loss2 = -mb_adv * torch.clamp(ratio, 1 - clip_coef, 1 + clip_coef)
        pg_loss = torch.max(pg_loss1, pg_loss2).mean()   # the trust region, in one line

        # value loss, clipped by symmetry with the policy clip: cap how far V can move per update
        newvalue = newvalue.view(-1)
        v_unclipped = (newvalue - b_returns[mb]) ** 2
        v_clipped = (b_values[mb] + torch.clamp(newvalue - b_values[mb], -clip_coef, clip_coef)
                     - b_returns[mb]) ** 2
        v_loss = 0.5 * torch.max(v_unclipped, v_clipped).mean()

        entropy_loss = entropy.mean()   # c2 * S[pi] keeps exploring (0 is fine on MuJoCo)

        # combined objective L^{CLIP+VF+S} (sign: minimize -L)
        loss = pg_loss - ent_coef * entropy_loss + vf_coef * v_loss

        optimizer.zero_grad()
        loss.backward()
        nn.utils.clip_grad_norm_(agent.parameters(), 0.5)  # cap gradient length, not direction
        optimizer.step()   # Adam -- first-order, nothing else

# theta_old <- theta is implicit: the next rollout uses the updated agent to produce new b_logprobs
```

So the KL leash becomes a flat spot in an ordinary loss: clip the ratio, take the pessimistic min against the unclipped term so overshoots still get corrected, and the same batch can now stand K epochs of minibatch Adam instead of one fragile step — TRPO's reliability without conjugate gradient, Fisher-vector products, or a line search anywhere in sight.
