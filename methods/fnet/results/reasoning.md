I start by separating the cost of attention from the role it plays. In a Transformer encoder, the feed-forward sublayer acts independently at each token and mixes only hidden features. The self-attention sublayer is the part that moves information across token positions. Its dot products, softmax, heads, and value projections make a powerful content-dependent mixer, but the structural job I need from the sublayer is simpler: after it runs, each position should have access to the rest of the sequence.

If that is the job, approximating attention is not the only route. I can replace the attention sublayer with another linear operation that mixes positions, then let the same residual, layer norm, and feed-forward machinery do the nonlinear modeling. The most direct trial is a learned dense matrix along the sequence axis, possibly paired with a learned dense matrix along the hidden axis. That is a useful sanity check because it removes query-key attention while still giving every output position a learned combination of every input position.

But a learned sequence matrix exposes the next wall immediately. It is `N x N`, so it still scales quadratically in sequence length, and its entries are tied to the chosen maximum length. It removes attention, but it does not give me a length-friendly replacement. I need a fixed structured linear map whose outputs are dense sums over the input positions and whose multiplication has a fast algorithm.

The discrete Fourier transform has exactly that shape. With the forward convention used by NumPy and JAX, a length-`N` vector is mapped by

`X_k = sum_{n=0}^{N-1} x_n exp(-2*pi*i*n*k/N)`.

The sign matters: this is the negative-exponent forward transform, the one computed by `jnp.fft.fftn` and `np.fft.fftn`. Every output frequency is a fixed-coefficient sum over every input position, so the operation is a dense global mixer with no learned parameters. Direct multiplication is quadratic, but the FFT computes the same transform in `O(N log N)` for the usual composite lengths.

The encoder input is not a single vector; it is an array with sequence and hidden axes. I first need the sequence-axis transform because that is what replaces attention's position mixing. I can also transform the hidden axis. The feed-forward sublayer already mixes hidden features, so hidden-axis mixing is not the essential move, but applying both one-dimensional transforms gives a separable two-dimensional transform, and the two axes commute. The candidate sublayer is therefore

`y = Re(F_seq(F_hidden(x)))`.

The complex output is the next practical wall. I do not want to make the feed-forward layers, output heads, and losses complex-valued. Taking the real part once, after both axes have been transformed, keeps the rest of the encoder unchanged while preserving the complex intermediate information long enough for the second transform to use it. Taking the real part after the first transform would discard information before the sequence transform; taking an absolute value changes the operation more aggressively. The real-at-the-end form is the clean drop-in replacement.

I also have to be precise about normalization. A Vandermonde DFT matrix is sometimes written with a `1/sqrt(N)` factor as a unitary convention, but the canonical implementation tests its matrix path against `np.fft.fftn`, and `scipy.linalg.dft` defaults to the unnormalized forward DFT. If I insert `1/sqrt(N)` on the sequence axis and `1/sqrt(d)` on the hidden axis, the matrix path is scaled by `1/sqrt(N*d)` relative to the FFT path. For code faithfulness, the cached matrix path must use the unnormalized DFT matrices unless I also deliberately rescale the FFT path.

Now the architecture can remain BERT-shaped. The mixer ignores the padding mask because the transform mixes every slot, including padded slots, and the released checkpoints are trained at a fixed maximum length. Hybrid variants can still pass a padding mask to attention layers, but the pure transform layer deletes it. The block is post-norm: add the mixer output to the input, layer-normalize with epsilon `1e-12`, run the unchanged `Dense(d_ff) -> GELU -> Dense(d_model)` feed-forward layer, add it, and layer-normalize again.

I should still check whether the chosen structured transform is uniquely forced. It is not. The requirement is fixed, global, structured mixing, and other transforms are plausible. A cosine transform stays real but loses accuracy. A Hadamard transform is very fast but also loses accuracy. A Hartley transform is real-valued and ties the Fourier transform in the reported GLUE result. That means the insight is not "complex numbers are magic"; the insight is that a fast structured global basis can replace token-dependent attention surprisingly well. The Fourier transform remains the natural implementation because it is standard, parameter-free, globally mixing, and has excellent FFT kernels.

Adding learned parameters back into the mixer is also tempting, but it undercuts the point. Elementwise scales, learned sequence/hidden matrices around the transform, inverse-transform sandwiches, and complex learned DFT weights either hurt, do nothing useful, or slow the model. So I keep the transform sublayer parameter-free and let the surrounding embeddings, feed-forward layers, and task heads carry the learned modeling capacity.

The resulting chain is: attention is expensive because it materializes all-pairs token scores; its necessary encoder role is global token mixing; a learned dense mixer confirms that attention's dot-product form is not the only way to mix tokens but keeps quadratic length cost; a fixed fast structured transform gives dense global mixing without parameters; the two-dimensional forward DFT over hidden and sequence axes, with the real part taken once at the end, is the simplest faithful drop-in mixer. The final encoder is still a post-LayerNorm BERT-style encoder, but the self-attention sublayer has become a deterministic global transform.
