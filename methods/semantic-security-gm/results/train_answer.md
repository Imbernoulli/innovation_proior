The problem that drives everything here is a card game played over a telephone line. Two people deal cards without a deck by exchanging encrypted cards over the wire, and I have public-key encryption to do it — Diffie and Hellman gave the concept, RSA and Rabin gave concrete trapdoors. So I can encrypt a card and send it, and yet the game can still be broken. Suppose my opponent, looking only at the ciphertext of a card he is not supposed to see, can compute its color: red or black, one bit. He never recovers the card, never inverts the encryption; he learns one bit, and that bit ruins the game. Lipton showed exactly this for mental poker built on RSA. So the inherited notion of security is failing me, and when I look at what that notion actually says, the failure is structural. Diffie–Hellman, and RSA and Rabin after them, define security as a work-factor on *inversion*: it is hard to recover the whole plaintext or the key. RSA rests on the hardness of taking $s$-th roots mod $n$; Rabin even sharpens this into a beautiful "break implies factor" theorem, since two square roots $x \neq \pm y$ of the same residue give $\gcd(n, x\pm y)$ as a factor. But every one of these protects the *whole message* and says nothing about the color of the card. The gap between "cannot invert" and "cannot learn anything" is precisely where the game dies.

The gap is not an accident of RSA; it is what determinism does. Take any deterministic encryption $E$. If some visible output bit varies with $m$ — parity, say — that bit is a nonconstant predicate of the plaintext, readable from the ciphertext for free, no trapdoor needed: a deterministic encoding turns features of $E(m)$ into features of $m$. And it gets worse once I remember the key is public. Equal plaintexts give equal ciphertexts, so the ciphertext leaks message *equality*; and the adversary, holding $E$, can take any guess $m_0$, compute $E(m_0)$ himself, and compare it against the intercepted ciphertext — "is this card the ace of spades?" becomes encrypt-the-guess-and-compare. The reason this works is that for the adversary $E(m)$ is a fixed target, one plaintext to one ciphertext. If instead each plaintext had many possible ciphertexts and encryption tossed coins to choose among them, his own encryption of a correct guess would almost surely not match the intercepted sample, and the attack dissolves. So I take this as a hard conclusion: secure encryption cannot be a function; it must be probabilistic.

Before building anything I have to settle what "secure" should *mean*. I want the ciphertext to give the adversary no help in computing *anything* about the plaintext — color, suit, a hash, the identity, any predicate, recursive or not — relative to what he could already do without it, since he always knows the message distribution and can guess the most likely value. So for any function $f$ on the message space let $p_f = \max_v \Pr[f(m)=v]$ be the best a-priori guessing probability; if $f$ may depend on the public algorithm $E$, write $p_E$ for the same maximum after $E$ is fixed. I call the scheme **semantically secure** if for every such $f$, no polynomially bounded adversary, *given* the ciphertext, computes $f(m)$ with probability non-negligibly above $p_f$. I phrase it as a game and am generous to the adversary: he may even *choose* $f$ after seeing $E$, then receive a ciphertext and be asked for $f(m)$ — and still he cannot beat $p_f$ by more than $1/\mathrm{poly}(k)$. The function $f$ need not even be recursive, because it is the *adversary's* job of computing it that must be hard. This is the polynomially bounded shadow of Shannon's perfect secrecy — a-posteriori equal to a-priori probabilities — but against bounded adversaries, which is exactly what lets the key be short.

The trouble with this definition is that it quantifies over *all* functions $f$, which is a nightmare to certify against. So I look for an equivalent single-game test, and the right one is that the adversary cannot tell two messages apart. Let a poly-bounded message-finder produce two messages $m_0, m_1$; encrypt one at random; hand the ciphertext to a poly-bounded line-tapper and ask which it was. The scheme is **indistinguishable** if no finder/tapper pair wins non-negligibly above one-half: with $p_b^T = \Pr[T(E,a)=1 : a \in E(m_b)]$, no $T$ achieves $|p_0^T - p_1^T| > 1/\mathrm{poly}(k)$ for a findable pair. One game, no quantifier over functions — and it instantly explains the determinism failure, since a deterministic scheme is never indistinguishable: the finder outputs any $m_0 \neq m_1$, their ciphertexts differ as fixed strings, and the tapper wins trivially.

These two definitions are the same thing, and the equivalence is the lever, so I prove it both ways. Semantic security implies indistinguishability by contrapositive: if a finder/tapper pair separates $m_0, m_1$ with gap $\delta > 1/\mathrm{poly}$, put the distribution uniformly on $\{m_0, m_1\}$, set $f(m) = [m = m_1]$ so $p_f = 1/2$, run the tapper $T$ (or $1-T$ if the sign is reversed) as the guess, and succeed with probability $1/2 + \delta/2$ — semantic security fails. The other direction, indistinguishability implies semantic security, is the one that matters. Suppose a poly-size circuit $C$, given $E$ and $a \in E(m)$, computes $f_E(m)$ with probability exceeding $p_E + \varepsilon$, $\varepsilon = 1/Q(k)$. For a message $m$ and value $v$ write $r_v^m = \Pr[C(E, E(m)) = v]$ for $C$'s output distribution on fresh encryptions of $m$. If $C$ produced the same output distribution on every message, against any fixed reference $s_v$ its success would be at most $\sum_v \Pr[f_E(m)=v]\, s_v \le p_E$, so the extra $\varepsilon$ must come from messages where $C$'s output actually moves. Fix a random reference message $\mu$ and call $\zeta$ *good* if some value $v$ has $|r_v^\zeta - r_v^\mu| > \varepsilon/10$. The **mass lemma**: if $\zeta$ is not good then $r_{f_E(\zeta)}^\zeta \le r_{f_E(\zeta)}^\mu + \varepsilon/10$, so writing $G$ for the good set of probability $\alpha$, the total success is at most $\sum_{\zeta \in G}\Pr[\zeta]\cdot 1 + \sum_{\zeta \notin G}\Pr[\zeta]\bigl(r_{f_E(\zeta)}^\mu + \varepsilon/10\bigr) \le \alpha + p_E + \varepsilon/10$; since this exceeds $p_E + \varepsilon$, the good mass $\alpha > 9\varepsilon/10 > \varepsilon/10$. The **sampling lemma**: $E$ is public, so for a good $\zeta$ I draw $O(k^c/\varepsilon^2)$ fresh encryptions of $\zeta$ and of $\mu$, run $C$, tabulate the empirical output frequencies, and by the weak law a value whose true gap exceeds $\varepsilon/10$ surfaces — accepting one whose empirical gap exceeds $3\varepsilon/40$ yields, with high probability, a true gap above $\varepsilon/20$. Now the distinguisher assembles itself: the finder picks random $\mu, \zeta$ (good with probability $\ge \varepsilon/10$), uses sampling to find the witnessing $v$, and outputs $(m_0, m_1) = (\zeta, \mu)$ with $v$; the tapper outputs $[C(E, x) = v]$, firing at rate $r_v^\zeta$ on $\zeta$ and $r_v^\mu$ on $\mu$, a gap above $\varepsilon/20$ — indistinguishability fails. Both directions hold, so I will *define* security by semantic security but *prove* schemes secure through indistinguishability.

Now I can build, and I know the scheme must be randomized with the smallest randomized unit being a single bit — the card game itself says one bit is the whole battle. The wrong shape for this is a trapdoor *function*, because the embedded-bit attacks fail exactly the way the color attack does: hide a bit as the $i$-th bit of a random $r$ and send $E(r)$, and hardness of recovering $r$ says nothing about that bit. The sharp witness is $g^x \bmod p$, which is one-way (inverting it is discrete log) yet leaks the last bit of $x$ for free, since $x$ is even precisely when $g^x$ is a quadratic residue mod the prime $p$, decidable in polynomial time by Euler's criterion $a^{(p-1)/2} \bmod p$. Whole-message hardness simply does not contain one-bit hardness. So I need a primitive whose hardness *is* the hardness of one bit. I propose to encrypt bit by bit with what I call an **unapproximable trapdoor predicate**: a family $\{B_i : \Omega_i \to \{0,1\}\}$ such that one can efficiently sample a uniform $x$ with $B_i(x)=0$ or with $B_i(x)=1$ (so I can encrypt either bit), a secret trapdoor $\sigma(i)$ lets the receiver compute $B_i(z)$ for any $z$ (so he can decrypt), no poly-size circuit computes $B_i$ on more than a $1/2 + 1/\mathrm{poly}$ fraction (the bit is genuinely hidden, by construction rather than as a hoped-for byproduct), and the pair $(i, \sigma(i))$ is samplable in poly time essentially uniformly (constructibility — otherwise an adversary could resample $(i,\sigma)$ pairs until he regenerates my published $i$ and recovers $\sigma(i)$). Given such a predicate, encryption writes itself: the public key is the index $i$; to encrypt bit $b$, sample a random $x$ with $B_i(x) = b$ and send it; to encrypt a message, encrypt each bit independently into a tuple; to decrypt, apply the trapdoor to each component.

Per-bit hiding is not obviously whole-message hiding, since the adversary sees the whole tuple, so I prove the bit-by-bit scheme indistinguishable by a hybrid argument. Suppose a tapper $T$ distinguishes $l$-bit messages $m_0, m_1$ with advantage above $\varepsilon$. View messages as vertices of the hypercube $\{0,1\}^l$, each labeled by the frequency $T$ outputs 1 on a random encryption of that vertex. The endpoints differ by more than $\varepsilon$, so along a one-bit-flip path from $m_0$ to $m_1$ (length $\le l$) some single step between adjacent words $s, t$, differing only in position $d$, changes the label by more than $\varepsilon/l$. To turn this against $B$ I must preserve the distribution defining the labels, so I sample each of the other $l-1$ coordinates from the public sampler conditioned on its agreed bit, splice the challenge $y$ into position $d$, and feed the tuple to $T$: if $B(y) = s_d$ the tuple has $s$'s encryption distribution, if $B(y) = t_d$ it has $t$'s. The sign of the gap says whether to output $T$'s bit or its complement, giving an $\varepsilon/(5l)$-approximator of $B$ (a small constant loss covers the case of an unbalanced predicate, where the majority value would already approximate $B$, ruled out by unapproximability), contradicting unapproximability. Per-bit security lifts to the whole message at a $1/l$ cost.

It remains to exhibit a concrete unapproximable trapdoor predicate, and the discrete-log example that broke me is suggestive: residuosity is *easy* mod a prime, so I move to a composite modulus where it is not. Let $n = p\cdot q$, a product of two distinct $k$-bit primes, and define $Q_n(x) = 1$ if $x$ is a quadratic residue mod $n$, else $0$. With the factorization this is trapdoor-evaluable: $x$ is a residue mod $n$ iff it is a residue mod $p$ and mod $q$, each tested by Euler's criterion and combined by CRT in $O(|n|^3)$. Without the factorization, the Jacobi symbol $(x/n)$ is poly-time computable; when $(x/n) = -1$, $x$ is a nonresidue for free and useless for hiding, so I work entirely inside the Jacobi-$+1$ elements, where exactly half are residues and half nonresidues and *deciding which* is the Quadratic Residuosity Problem, hard since Gauss singled it out in the *Disquisitiones*. I *assume* it is hard — the QR assumption — but that is an assumption about *some* inputs, whereas my reduction needs *unapproximability*, hardness on the average. The bridge is **random self-reducibility**, the prettiest part. Suppose an oracle $O$ decides $Q_n$ on a $1/2 + \varepsilon$ fraction of Jacobi-$+1$ inputs, I-know-not-which. To test an unknown $q$, generate many random residues $x_i$ (square random elements) and form $q\cdot x_i \bmod n$: multiplication by a random residue is a bijection within the residue class or within the nonresidue coset, so if $q$ is a residue these are random residues, and if $q$ is a Jacobi-$+1$ nonresidue these are random Jacobi-$+1$ nonresidues — I have turned $q$ into fresh random samples of its own class. Calibrate by estimating the rate $\alpha$ at which $O$ says "residue" on genuine squares, then compare $O$'s "residue" rate on the $q\cdot x_i$; the overall advantage forces $\alpha - \beta > 2\varepsilon$ between the two classes, and a poly-size sample separates them by the weak law, driving the confidence arbitrarily high. So residuosity is everywhere-hard-or-everywhere-easy, and under the QR assumption no poly-size circuit even approximates $Q_n$.

Two loose ends finish the predicate. To encrypt a $1$ I must sample a random Jacobi-$+1$ nonresidue, but without the factorization I cannot even recognize one — squaring only produces residues. So I publish, as part of the key, one fixed Jacobi-$+1$ nonresidue $y$ (a *pseudosquare*); then $y\cdot(\text{random residue})$ is a uniform Jacobi-$+1$ nonresidue, making both bit-values samplable. Publishing $y$ is safe: revealing a *random* Jacobi-$+1$ nonresidue leaves QR exactly as hard, because any procedure that decides residuosity using such an auxiliary $y$ can be converted, via the same random-square cosets and self-reducibility amplification, into one that decides residuosity without it. And constructibility holds: $4k$ coin flips give candidate primes $p, q$ (accepted with good probability by the Prime Number Theorem and a poly-time primality test) and a pseudosquare $y$, the index $n\#y$ and secret $(p,q)$ generated together, essentially uniformly, in random poly time. Dropping this predicate into the bit-by-bit architecture gives the **Goldwasser–Micali cryptosystem**. Key generation picks $k$-bit primes $p, q$, sets $n = pq$, and picks a Jacobi-$+1$ nonresidue $y$, publishing $(n, y)$ and keeping $(p, q)$ secret. To encrypt bit $b$, pick random $x \in Z_n^*$ and send
$$e = y^{b}\cdot x^{2} \bmod n,$$
so $b=0$ sends a random residue and $b=1$ a random Jacobi-$+1$ nonresidue; a whole message becomes a tuple of such $e$'s, one plaintext bit expanding to $k$ ciphertext bits. To decrypt $e$, the holder of $(p,q)$ reads $b=0$ if $e$ is a residue mod $n$ (residue mod $p$ and mod $q$), else $b=1$. The scheme is randomized, so determinism's parity-, equality-, and re-encrypt-and-compare leaks are gone; the hypercube hybrid lifts per-bit hiding to the whole tuple; indistinguishability is equivalent to semantic security; and any break would decide quadratic residuosity. Ciphertexts are even re-randomizable by multiplying each component by a fresh random square — the same self-reducibility at work.

```python
from math import gcd
import random

def jacobi(a, n):
    a %= n; result = 1
    while a:
        while a % 2 == 0:
            a //= 2
            if n % 8 in (3, 5): result = -result
        a, n = n, a
        if a % 4 == 3 and n % 4 == 3: result = -result
        a %= n
    return result if n == 1 else 0

def is_qr_mod_prime(a, p):                       # Euler's criterion
    return pow(a % p, (p - 1) // 2, p) == 1

def keygen(p, q):                                # public (n, y), secret (p, q)
    n = p * q
    while True:                                  # y: nonresidue mod BOTH primes => Jacobi(y/n) = +1
        y = random.randrange(2, n)
        if gcd(y, n) == 1 and not is_qr_mod_prime(y, p) and not is_qr_mod_prime(y, q):
            return (n, y), (p, q)

def encrypt_bit(b, n, y):                        # b=0 -> x^2 (QR); b=1 -> y*x^2 (Jacobi-+1 QNR)
    x = random.randrange(1, n)
    while gcd(x, n) != 1: x = random.randrange(1, n)
    return (pow(y, b, n) * pow(x, 2, n)) % n

def decrypt_bit(e, p, q):                        # QR mod n iff QR mod p and mod q
    return 0 if (is_qr_mod_prime(e, p) and is_qr_mod_prime(e, q)) else 1

def encrypt(bits, pub): n, y = pub;        return [encrypt_bit(b, n, y) for b in bits]
def decrypt(ct, sec):   p, q = sec;        return [decrypt_bit(e, p, q) for e in ct]
```
