I start with the shape of the detection problem, because the awkward part is not the definition. Given an image, I want every object, each with a class and a tight box. The awkward part is that the best systems around me do not look like that simple mapping. They look like pipelines.

DPM is a sliding-window model over a HOG pyramid: a root filter, part filters, deformation costs, then box prediction and non-max suppression. It has a real spatial model, but the features, classifier, geometry correction, and post-processing are separate pieces. R-CNN replaces exhaustive sliding windows with Selective Search proposals, then warps each proposal, runs a CNN, scores with per-class SVMs, adjusts boxes, and suppresses duplicates. Fast R-CNN shares the convolutional pass and trains the proposal classifier and box regressor together, but the proposals still come from a separate algorithm. Faster R-CNN moves proposal generation into a neural network, but the design is still propose first, classify and refine second.

Two pressures keep showing up. One is speed. If I have to generate thousands of regions or classify each crop separately, real-time video is out of reach. Sharing convolution helps, but a proposal stage and a second per-region decision stage still impose latency. The other pressure is trainability. A pipeline optimizes proposal recall, classification accuracy, box regression, and duplicate suppression as separate objectives. I cannot simply backpropagate through the whole detector as one function of the image and the final detection targets.

There is also a structural context problem. A region classifier sees a cropped patch. If the patch locally resembles an object, the classifier has little access to the surrounding scene that might reveal it as background. This is not a claim that any particular model must fail on every such crop; it is a consequence of deciding locally after proposals have already been cut out. If every box prediction is instead made from features of the full image, the score can depend on both local appearance and global context.

So I want one network, one image-sized forward pass, and one detection objective. But the moment I say that, I hit the first wall: a detector must output a variable-size set, while a neural network head naturally emits a fixed-size tensor. If I just allocate a fixed list of boxes, I get an assignment problem. Which output slot owns which object? Two slots can chase the same central object, one object can be ignored, and the symmetry between slots gives the optimizer no clean division of labor.

The way out is to make the division of labor spatial. I divide the image into an `S x S` grid and assign each object to the cell containing its center. That rule gives me a fixed set of output locations, but it also breaks the symmetry: a cell owns only the objects whose centers fall inside it, and neighboring cells should not all chase the same object. It also reduces duplicate detections before any post-processing, because the intended owner of an object is unique.

Inside a cell, one box predictor is too rigid. Across the training set, the same cell position may need to represent a tall person, a wide car, or a squat object. A single regressor will average incompatible shapes. If the cell predicts `B` boxes, the predictors can specialize. I still want the output small, so `B=2` is the natural minimal choice for the VOC setting: enough to let predictors separate by shape or size, not enough to recreate a huge proposal set.

Now I need a responsibility rule within the cell. For a cell with an object, I choose the currently best matching box predictor, measured by IOU with the ground-truth box, and only that predictor receives the coordinate gradient for that object. That is a self-reinforcing specialization rule. If one predictor is already better for a tall object, it gets selected, improves on tall objects, and remains the tall-object specialist. If all candidate boxes have zero overlap with the truth early in training, I still need a deterministic tie-breaker; choosing the smallest coordinate RMSE keeps exactly one responsible predictor.

The confidence target needs to mean more than "there is an object somewhere in this cell." A useful detection score should be high only when a box contains an object and the box is well localized. So I define confidence as `Pr(Object) * IOU(pred, truth)`. In an empty cell, `Pr(Object)=0`, so every box confidence should be zero. In an object cell, the responsible box should predict its current IOU with the ground truth, not merely 1. A poorly placed box around a real object should report low confidence because it is not a good detection yet.

The class prediction should not be duplicated per box. The two boxes in a cell are competing geometric hypotheses for the same cell-owned object, not independent objects with independent labels. I therefore predict one set of conditional class probabilities per cell, `Pr(Class_i | Object)`, shared by all `B` boxes. At test time the multiplication is clean: the class-conditional probability from the cell times each box confidence gives `Pr(Class_i) * IOU`, a class-specific score that combines label belief and localization quality.

For VOC, the fixed output becomes `S x S x (B*5 + C) = 7 x 7 x 30`: twenty class probabilities per cell plus two boxes, each with coordinates and confidence. That is only 98 boxes per image. It is a severe compression compared with thousands of proposals, and that is exactly why the model can be fast.

The backbone should be a strong classifier feature extractor, but detection data is too scarce to learn everything from scratch. I pretrain the convolutional layers on ImageNet at `224 x 224`, then switch to `448 x 448` for detection because localization needs finer spatial detail. GoogLeNet suggests the useful compute-saving pattern: use `1 x 1` reductions before expensive `3 x 3` convolutions. I do not need full inception modules; a stack of `1 x 1` and `3 x 3` convolutional layers followed by two fully connected layers gives the fixed detection tensor. The hidden layers use leaky ReLU with slope 0.1, and the final layer is linear.

Now the loss. Plain sum-squared error over the whole tensor is easy to optimize, but it is badly balanced. It treats localization and classification as equally important numeric coordinates, even though a detector lives or dies by its boxes. It also sees many more empty cells than object cells. If I let every empty-cell confidence term push hard toward zero, the easy solution is to predict low confidence everywhere and swamp the rare positive signal.

I introduce two weights. `lambda_coord = 5` multiplies the coordinate terms, so localization gets a larger gradient. `lambda_noobj = 0.5` multiplies confidence terms for boxes that are not responsible for an object, so empty and non-responsible boxes do not dominate training. The exact cases matter. In an empty cell, both box confidences are no-object terms and no class or coordinate loss is applied. In an object cell, the responsible box gets coordinate loss and object-confidence loss; the other box or boxes in that same cell still get no-object confidence loss with target zero. Class loss is applied once for the cell, because the class distribution is conditional on an object being present in that cell.

There is a third mismatch in squared error. The same absolute width error is much worse on a small object than on a large one. A 10-pixel error on a bird can destroy IOU; the same error on a bus may barely matter. I need a cheap way to compress large sizes so equal absolute errors hurt less for large boxes. I make the network predict `sqrt(w)` and `sqrt(h)` rather than `w` and `h` directly. At decoding time those channels are squared to recover the actual width and height. In the coordinate loss, the target is `sqrt(w*)` and `sqrt(h*)`.

With the indicators in place, the objective is straightforward. `1_i^obj` marks cells containing an object. `1_ij^obj` marks the one responsible predictor in such a cell. `1_ij^noobj` marks every predictor that is not responsible for an object, including both predictors in empty cells and the unused predictor in an object cell. Then the coordinate loss is applied only under `1_ij^obj`, the responsible confidence target is IOU, the no-object confidence target is zero under `1_ij^noobj`, and the class loss is applied under `1_i^obj`.

That also tells me what the implementation must do. It cannot treat only empty cells as no-object cases, because then the unused box in an object cell would be untrained for confidence. It also cannot use a target confidence of 1 for the responsible box; it has to use the responsible box's IOU. The width and height output channels should live in square-root coordinates already, not raw widths that are later square-rooted inside the loss.

At inference, the procedure is intentionally small. One forward pass gives the fixed tensor. For each cell and box, I decode the center by adding the cell offset and dividing by `S`, square the predicted root-width and root-height, multiply box confidence by the cell's class probabilities, threshold the class-specific scores, and apply a light NMS pass for the remaining duplicate cases. The grid does most of the duplicate prevention; NMS is cleanup for large or boundary-straddling objects.

The same choices impose the main limitations. A cell has only two boxes and one class distribution, so several small objects whose centers fall into the same cell cannot all be represented. The final prediction grid is coarse, so small objects and unusual aspect ratios are difficult. The square-root size parameterization helps the loss care more about small boxes, but it does not make squared error identical to average precision or IOU. Those are the costs of making the detector a single fixed tensor from one full-image pass.
