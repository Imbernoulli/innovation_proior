I am trying to decide, for formulas of the first-order functional calculus whose quantifier prefix is entirely universal, whether such a formula can be satisfied over an infinite domain — one strand of the Entscheidungsproblem posed by Hilbert and Ackermann. With finitely many predicate symbols of bounded arity, the truth-pattern realized by any fixed tuple of $r$ individuals is one of finitely many "types," and a universal formula's satisfiability is governed entirely by how these types distribute across all $r$-tuples of the domain: once a single type recurs uniformly on every $r$-tuple drawn from some large sub-domain, the formula collapses to a single readable condition. So the logic reduces to a purely combinatorial question with the syntax stripped away: partition all the $r$-element subsets of a set, however one likes, into finitely many colour classes — must there always be a large, or in the infinite case an infinite, sub-set all of whose $r$-element subsets fall into a single class? Call such a sub-set homogeneous. For $r=1$ this is nothing but the pigeonhole principle — sorting points into $\mu$ boxes, a box with $h$ points is itself a homogeneous $h$-set — so $r=1$ is settled trivially, with $\mu(h-1)+1$ points the least forcing size. But raising $r$ to $2$ already breaks the naive approach: two-colouring the pairs of a huge point set and applying pigeonhole to the pairs themselves only bounds a *global tally* of, say, red pairs, and a large pile of red pairs can be spread thin across many points with no three of them forming a uniformly red triangle. Homogeneity is a nested, internal-structure demand that one global count cannot see. The syntactic partial solutions on the table — Behmann's decision procedure for monadic predicates, and Bernays and Schönfinkel's for formulas with only two individual variables under the prefix $\exists^*\forall^*$ — sidestep this combinatorial question entirely by restricting the *shape* of the formula rather than answering it; the six-person folklore fact, that any two-colouring of the pairs among six people contains a monochromatic triple, shows the phenomenon is real in the smallest nontrivial case but supplies no general argument and no bound. I want the general statement — any $r$, any finite number of colours $\mu$, both infinite and finite domains — and for the finite domain I want an explicit, computable threshold, since a decision procedure needs a number to search up to, not a bare existence claim. The existence side already tells me how good such a bound can possibly be: a "generic" random colouring of the pairs of an $N$-point set avoids a monochromatic $s$-set as long as $N$ is below roughly $2^{s/2}$, since $\binom{N}{s}\cdot 2^{1-\binom{s}{2}}$ then drops below one, so no forcing bound can beat single-exponential in the target size, and the real question is how close a genuine proof's bound comes to that floor.

I propose to settle both the infinite and the finite question with a single idea used over and over: apply the pigeonhole principle not once, globally, but *locally and repeatedly* — fix a point, retreat to the sub-population that agrees in colour with it, and recurse. This is Ramsey's theorem, and it is best understood as the pigeonhole principle lifted from points to $r$-subsets.

Start with the infinite case, two colours, pairs ($r=2$), because there I never run out of room and the mechanism shows itself cleanly. Take an infinite set $\Gamma$ and pick any point $a_1$. It sits in infinitely many pairs $\{a_1,x\}$, each red or blue; pigeonhole on these gives an infinite $B_1\subseteq\Gamma\setminus\{a_1\}$ all of whose pairs with $a_1$ share one colour $c_1$. Recurse inside $B_1$: pick $a_2\in B_1$, get an infinite $B_2\subseteq B_1\setminus\{a_2\}$ monochromatic to $a_2$ in colour $c_2$; continue forever, choosing infinitely often and each time passing to an infinite reservoir — a use of the axiom of choice that the finite version below will not need, since there only finitely many choices are made. The colour of any pair $\{a_i,a_j\}$, $i<j$, is now determined entirely by the smaller index, since $a_j\in B_j\subseteq\cdots\subseteq B_i$ and $B_i$ is all-$c_i$ to $a_i$: it equals $c_i$. That leaves an infinite sequence of colours $c_1,c_2,\dots$, and pigeonhole strikes once more, at the meta level: infinitely many of the $c_n$ agree, say all $c_n$ for $n$ in an infinite index set $S$ are red, and then $\{a_n:n\in S\}$ is an infinite homogeneous red set. Three uses of pigeonhole — fixing each vertex's local colour, letting a pair's colour reduce to its lower endpoint, and homogenizing the sequence of vertex-colours at the end — do the whole job.

General $r$ follows by induction on $r$, with $r=1$ as the base. Given a $2$-colouring of the $p$-subsets of $\Gamma$, the same greedy construction is attempted: find $a_1$ and an infinite $\Gamma_1$ with every $p$-subset $\{a_1\}\cup(p-1 \text{ of } \Gamma_1)$ landing in colour $C_1$ — possible because colouring a $(p-1)$-subset $F$ of $\Gamma\setminus\{a_1\}$ by the colour of $\{a_1\}\cup F$ is exactly a $(p-1)$-uniform colouring, which the induction hypothesis on $r$ already knows how to homogenize. If this greedy step never fails, the resulting $a_1,a_2,\dots$ is homogeneous in $C_1$ by the same least-index argument as before. If it fails at some stage — no point of the current reservoir has an infinite all-$C_1$ neighbourhood — that failure is itself informative: pick any $y_1$ there, colour the $(p-1)$-subsets of the rest by whether adjoining $y_1$ gives $C_1$ or $C_2$, and the induction hypothesis returns an infinite set on which this induced colouring is constant; that constant cannot be $C_1$, because an all-$C_1$ neighbourhood through $y_1$ is precisely what the failure ruled out, so it is $C_2$. Repeating inside that set with $y_2,y_3,\dots$ forces every one of them into an all-$C_2$ neighbourhood, and $\{y_1,y_2,\dots\}$ is an infinite $C_2$-homogeneous set. Either branch of the case split lands on a homogeneous set, so the induction on $r$ closes. Lifting from two colours to any finite $\mu$ is a second induction, this time on $\mu$: merge the last two colours into one, apply the $(\mu-1)$-colour theorem, and if the resulting homogeneous set lands in the merged class, split it again with the $\mu=2$ case applied inside it. This gives, for $\Gamma$ infinite and $r,\mu$ positive integers: any partition of the $r$-subsets of $\Gamma$ into $\mu$ classes $C_1,\dots,C_\mu$ leaves an infinite sub-class $\Delta\subseteq\Gamma$ all of whose $r$-subsets lie in one $C_i$ (using the axiom of choice for the infinitely many selections).

For the decision procedure I need the finite statement with an actual number, and here compactness is a trap. Gluing hypothetical finite counterexamples into a single bad infinite colouring — the standard König-lemma packaging: agree on the $r$-subsets of $\{1,\dots,r\}$, then of $\{1,\dots,r+1\}$, and so on, always possible for infinitely many of the counterexamples since only finitely many colourings of each initial block exist — does prove a threshold $m_0$ exists, by contradicting the infinite theorem, but it is purely existential and hands back no value at all. The bound has to be built by a direct recursion that carries the size along, and the naive version of that recursion gets stuck for a structural reason: absorbing one vertex into the growing homogeneous set consumes the freedom to absorb the next, and in a finite reservoir that freedom runs out, unlike the infinite case where there is always more reservoir left. The fix is to strengthen the statement being proved so the induction feeds itself: instead of asking only for a homogeneous $\Delta_n$, ask for $\Delta_n$ *together with a reservoir* $\Lambda_k$ of extra points such that every $r$-subset of $\Delta_n\cup\Lambda_k$ that touches $\Delta_n$ is also one colour — the reservoir being exactly the stock the next promoted vertex is drawn from. This asymmetric $(n,k)$ form sits strictly between demanding a homogeneous $n$-set and demanding a homogeneous $(n+k)$-set, and that slack is what lets the recursion propagate.

Writing $f(r,n,k)$ for the threshold this $(n,k)$ statement needs, with two colours, the recursion falls out of tracking exactly what each step consumes. The base $r=1$ is pigeonhole plus a physical fit condition: $f(1,n,k)=\max(2n-1,\,n+k)$. Peeling a single vertex $x$ at general $r$ turns the incident $r$-subsets into an induced $(r-1)$-uniform colouring of the rest, and I want the resulting $(r-1)$-homogeneous set to come out *fully* monochromatic rather than merely core-plus-reservoir — which happens automatically by feeding the $(r-1)$-form the parameters $n=k-r+2$, $k=r-2$, since then the reservoir has only $r-2<r-1$ members, too few to fill an $(r-1)$-subset on its own, forcing every $(r-1)$-subset of the resulting $k$-set to touch the core and hence be monochromatic:
$$f(r,1,k) = f(r-1,\,k-r+2,\,r-2) + 1,$$
the $+1$ being $x$ itself. Building $\Delta_n$ one vertex at a time from $\Delta_{n-1}$ needs an auxiliary bookkeeping function $g(r,n,k)$: it is the reservoir size to demand at the $(n-1)$-stage so that, after converting one reservoir vertex into the $n$-th core point via the $n=1$ machinery, a reservoir of size $k$ still survives. Converting one vertex costs one application of $f(r,1,\cdot)$, so
$$g(r,0,k)=\max(r-1,k), \qquad g(r,n,k)=f\bigl(r,1,\,g(r,n-1,k)\bigr) \ (n\ge 1),$$
$$f(r,n,k) = f\bigl(r,n-1,\,g(r,n,k)\bigr) \ (n>1).$$
The reason this closes: given $\Delta_{n-1}$ with reservoir $g(r,n,k)$, promoting a vertex $x_1$ from the reservoir either lands its neighbourhood in the original colour $C_1$ — in which case $\Delta_{n-1}\cup\{x_1\}$ becomes the new core, with reservoir left over exactly as in the infinite construction's successful branch — or in $C_2$, in which case $x_1$ is peeled aside as the seed of a $C_2$-homogeneous set and the same promotion is retried on the surviving reservoir; $g$ is sized precisely so this can fail up to $n$ times and still leave enough reservoir, and if it fails every time, $x_1,\dots,x_n$ themselves form a $C_2$-homogeneous core by the same least-index argument as the infinite proof's failure branch. Either outcome proves the $(n,k)$ statement at threshold $f(r,n,k)$.

An outright homogeneous $n$-set for two colours falls out of this asymmetric form by choosing the reservoir just barely too small to fill an $r$-subset on its own: with core size $n-r+1$ and reservoir size $r-1$, any $r$-subset of the combined $n$-point set must include a core point and is therefore already known to be monochromatic, giving $h(r,n,2)=f(r,\,n-r+1,\,r-1)$. Lifting to $\mu$ colours repeats the infinite proof's merging trick, but now composing thresholds instead of merely re-applying an existence statement:
$$h(r,n,\mu) = h\bigl(r,\,h(r,n,\mu-1),\,2\bigr) \quad (\mu>2),$$
merging colours $C_2,\dots,C_\mu$ into one super-colour, using the two-colour bound to land either in $C_1$ (done, with room to spare) or in the super-colour, and then invoking the $(\mu-1)$-colour theorem inside that super-coloured set to peel out a genuinely monochromatic $n$-set.

This machine is completely general, but traced through for the plainest and most-used case — pairs, $r=2$, two colours — it produces a threshold on the order of $2^{n(n-1)/2}$, wildly larger than the $2^{n/2}$ existence floor. Pairs deserve a direct argument. A local pigeonhole gives a factorial bound: with $k$ colours, $k\cdot(n+1)!$ points force a homogeneous $n$-set, by inducting on $n$ so that a reservoir of $k(n+1)$ points left over from an $(n-1)$-core either contains a point whose $k$ same-colour partners let it join the core with a fresh reservoir of size $k$, or, failing that, up to $n$ points can be peeled off one at a time — each guaranteed by pigeonhole to have at most $k-1$ same-colour partners among the remaining reservoir, hence forced into the other colour with everyone still to come — yielding a homogeneous set in that other colour instead. Sharper still, in the two-colour case, is to package the statement as the Ramsey number $R(s,t)$: the least $N$ such that every red/blue colouring of the pairs of $N$ points contains a red clique of size $s$ or a blue clique of size $t$. Trivially $R(s,t)=R(t,s)$, and $R(2,t)=t$, since $t$ vertices either contain one red pair or are all blue among themselves, while $t-1$ all-blue vertices have neither. Fixing a vertex $v$ among $R(s-1,t)+R(s,t-1)$ vertices and splitting the rest by their colour to $v$, pigeonhole guarantees at least $R(s-1,t)$ red neighbours or at least $R(s,t-1)$ blue neighbours — falling short of both would leave only $R(s-1,t)-1+R(s,t-1)-1$ neighbours in total, one short of what is available — and either a red $K_{s-1}$ among the red neighbours extends by $v$ to a red $K_s$, or a blue $K_t$ is already there; symmetrically on the blue side. That gives the recursion
$$R(s,t) \le R(s-1,t) + R(s,t-1),$$
and unrolling it against the base cases with Pascal's identity closes it into a single closed form:
$$R(s,t) \le \binom{s+t-2}{\,s-1\,}, \qquad R(s,s) \le \binom{2s-2}{s-1} \le \frac{4^s}{\sqrt s} < 4^s,$$
a single-exponential bound whose shape matches the existence floor: any two-colouring of the pairs of $N$ points contains a monochromatic set of size at least roughly $\tfrac12\log_2 N$. The smallest interesting instance checks out by hand: $R(3,3)\le R(2,3)+R(3,2)=3+3=6$, and six people always do contain three who pairwise agree — fix one person $P$; of the other five, $P$ relates the same way to at least three of them by pigeonhole ($\lceil 5/2\rceil=3$); among those three, either two of them agree with each other, closing a monochromatic triple with $P$, or all three disagree with $P$'s relation pairwise, which is itself a monochromatic triple. Five is not enough: colouring a $5$-cycle's edges red and its diagonals blue leaves neither a red nor a blue triangle, so $R(3,3)=6$ exactly.

The result is a theorem, not a program, and needs no implementation; but the threshold and the smallest hand-checked instance are cheap to confirm by brute force on tiny cases:

```python
from itertools import combinations, product

def is_homogeneous(subset, r, coloring):
    """True iff every r-subset of `subset` gets one colour."""
    return len({coloring[s] for s in combinations(subset, r)}) == 1

def has_homogeneous_set(vertices, r, n, coloring):
    return any(is_homogeneous(c, r, coloring) for c in combinations(vertices, n))

def forces_homogeneous_set(m, r, n, mu):
    """Does EVERY mu-colouring of the r-subsets of an m-set have a mono n-set?
    Brute force over all colourings — feasible only for tiny m, r, mu."""
    verts = list(range(m))
    subs = list(combinations(verts, r))
    for assign in product(range(mu), repeat=len(subs)):
        coloring = dict(zip(subs, assign))
        if not has_homogeneous_set(verts, r, n, coloring):
            return False   # found a colouring with no homogeneous n-set
    return True

# R(3,3) = 6:  6 forces a mono triangle (pairs, 2 colours, target 3); 5 does not.
assert forces_homogeneous_set(6, r=2, n=3, mu=2) is True
assert forces_homogeneous_set(5, r=2, n=3, mu=2) is False
```
