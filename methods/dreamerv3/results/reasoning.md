OK, let me think about what actually hurts. I have an agent family that learns a latent world model and then learns behavior entirely inside that model's imagination — an RSSM with a deterministic recurrent state $h_t$ and a stochastic state $z_t$, a prior $p(z_t\mid h_t)$ that dreams forward without observations and a posterior $q(z_t\mid h_t,x_t)$ that filters real frames, an actor and a critic that train on imagined rollouts with $\lambda$-returns. On continuous control it works. On Atari it works. The problem is that "it works" each time only after I sit down and retune. Moving from a video game to a robot, from dense scores to a sparse milestone, from pixels to proprioception — every move forces me to re-find the KL weight, the reward scale, the entropy coefficient, the value parameterization. That retuning is the actual bottleneck. So the thing I want is not a higher score on any one benchmark; it's *one fixed set of hyperparameters that learns across all of them*. If I had that, RL would be applicable out of the box, which is the whole point.

So let me reframe the project as a hunt for brittleness. Walk the agent end to end and ask, at every place there's a number I currently retune: *why* does the right value of that number depend on the domain, and what would make it not depend on the domain? Every such number is a knob, and a knob is a failure of scale-invariance. If I can make each objective in the agent self-balancing — invariant to the arbitrary scale and shape of rewards, returns, and observations — then one configuration spans everything.

Where do the scales actually differ? Rewards can be dense and in the hundreds, like an arcade score, or sparse and binary, like reaching a milestone once in thousands of steps. Returns inherit that and can be unimodal or multimodal. Observations can be pixels where a single pixel matters for the task, or 3D scenes drowning in irrelevant texture, or low-dimensional sensor vectors. A loss scale, a KL weight, an entropy coefficient that's right in one of these regimes is wrong in another. Good — that's the list. Let me take them one at a time, and let each fix drop out of the failure it's fixing.

Start with the most basic: I have neural nets that have to *predict* scalars — the decoder reconstructs observations, a reward head predicts $r_t$, the critic predicts returns. These targets span orders of magnitude across domains, and even within a run they can grow as the agent gets better. What loss do I use? The reflex is squared error. But picture a target that's suddenly in the thousands: the squared loss is in the millions, its gradient is enormous, and the optimizer takes a giant step and diverges. So a raw squared loss on large targets is a wall. The classic patch is an absolute or Huber loss, which caps the gradient magnitude — but now look at what that costs me: once the error is past the Huber threshold the gradient is *constant*, so when the target is genuinely large the net crawls toward it at a fixed tiny rate and learning stagnates. Another patch is to normalize the target by a running mean/std — but that makes the regression target *move* as the statistics update, which is non-stationarity injected straight into the optimization, and non-stationarity is exactly what I'm trying to avoid in a system that already has a moving world model and a moving policy. A third, adjusting the output-layer weights whenever a new extreme value shows up, is intrusive and couples the architecture to the value range. None of these is a fixed knob; each is a knob.

What I actually want is a *fixed* transform that compresses large magnitudes so the squared loss never explodes, but leaves small targets essentially alone so it doesn't distort learning when values are already small, and crucially doesn't truncate — I still want to be able to move the prediction to a large value quickly when the target really is large. Compression of magnitude says logarithm. But $\log$ can't handle the sign — rewards and returns go negative — and it blows up near zero. I need something that's symmetric around the origin, defined for negatives, and roughly the identity near zero. Take the log of $|x|+1$ to fix the singularity at zero, and reattach the sign:
$$\symlog(x)\doteq\sign(x)\,\ln(|x|+1),\qquad \symexp(x)\doteq\sign(x)\big(\exp(|x|)-1\big).$$
Check the properties. Near zero, $\ln(1+|x|)\approx|x|$, so $\symlog(x)\approx x$ — identity, small targets untouched. For large $|x|$ it grows like $\ln|x|$ — heavy compression, so a target of $1000$ becomes about $\sign\cdot 6.9$ and the squared loss is tame. It's odd, $\symlog(-x)=-\symlog(x)$, so signs are preserved and it's symmetric. And $\symexp$ is its exact inverse: $\symexp(\symlog(x))=\sign(x)(\exp(\ln(|x|+1))-1)=\sign(x)(|x|+1-1)=x$. So I train the net to predict $f(x,\theta)=\symlog(y)$ under a plain squared loss, $\tfrac12(f-\symlog(y))^2$, and read predictions back out with $\hat y=\symexp(f)$. No divergence, no stagnation, no non-stationarity, no truncation, and no knob. I'll feed vector observations through symlog on the way into the encoder and have the decoder predict symlog'd targets too, for the same reason — large sensor readings shouldn't produce large reconstruction gradients.

Now the harder targets: rewards and returns aren't just large, they can be *multimodal*. In a randomized environment some episodes have far higher achievable return than others, so the return distribution has multiple bumps; squeezing that through a single scalar regression — even a symlog'd one — throws away the structure and, more to the point, the gradient of a squared regression still scales with the residual, so a far-off mode still produces a scale-dependent gradient. I want the gradient size to be *completely* decoupled from the target size. The way to do that is to stop regressing a scalar and instead *classify*. Lay down a fixed set of bins $B$ over the value axis and have the head output a softmax distribution over them, trained by cross-entropy. Cross-entropy's gradient depends only on the predicted *probabilities* versus the target label — it has nothing to do with how large the bin values are. That's the decoupling I want. And to span orders of magnitude with few bins, space the bins exponentially: $B=\symexp([-20,\dots,20])$, so the support reaches enormous magnitudes (and negatives) while staying dense near zero where most values live.

But the target is a continuous scalar and the head is a classifier — what's the label? A one-hot on the nearest bin would quantize, and the readout would be stuck at bin centers. Instead split the scalar's mass between the two nearest bins, linearly by distance — a *twohot* label. If $b_k\le y< b_{k+1}$, put weight $\tfrac{|b_{k+1}-y|}{|b_{k+1}-b_k|}$ on bin $k$ and $\tfrac{|b_k-y|}{|b_{k+1}-b_k|}$ on bin $k+1$, zero elsewhere; the two weights sum to one, with more mass on the closer bin. Read the prediction back as the probability-weighted average of bin positions, $\hat y=\softmax(f)^{\!\top}B$ — which can land *between* bins, so the discreteness of the support doesn't quantize the output. Train with the soft-target cross-entropy $-\,\twohot(y)^{\!\top}\log\softmax(f)$. Now reward and return prediction is multimodal-capable, scale-decoupled, and continuous. I'll use this symexp-twohot head for the reward predictor and for the critic, and the plain symlog squared loss for the decoder and other deterministic scalar reads.

One trap I can already foresee with the twohot heads. At initialization the output logits are random, so the head predicts some arbitrary distribution over bins, and after $\symexp$ that can be a *large* predicted reward or value. A large hallucinated value at step zero sends the actor chasing phantom returns and delays the onset of real learning. Easy fix: initialize the output weight matrix of the reward predictor and critic to *zero*, so at init they predict the zero-centered bin (value zero) and don't hallucinate. Costs nothing, removes a stall.

Next knob: the world-model KL. The model is fit by reconstructing inputs, rewards, and continuation flags from the model state, plus a KL that ties the prior $p(z_t\mid h_t)$ to the posterior $q(z_t\mid h_t,x_t)$ — because at imagination time I sample the prior, so the prior had better predict where the posterior would land. In the previous generation that KL had to be scaled per domain, and I now understand *why*: in a texture-rich 3D world there's a flood of detail irrelevant to control, so I want a strong regularizer to throw it away and keep the latent simple and predictable; in a game with a static background where individual pixels carry task information, I want a weak regularizer so the latent retains fine detail. A single weight can't be both strong and weak. That's the domain dependence to kill.

Two things to fix here, and they're separate. First, the KL is between two distributions I'm *both* learning, and I don't want them to meet in the middle. The posterior, having seen the frame, is the richer, more informative distribution; the prior has to chase it. If I optimize one symmetric KL, the gradient pulls both ways and the prior can "win" by dragging the posterior toward its own ignorance — toward a lazy, uninformative latent. So split the KL into two copies with stop-gradients on opposite sides and different weights. Write
$$\mathcal L_{\mathrm{dyn}}=\KL\!\big[\sg\big(q(z_t\mid h_t,x_t)\big)\,\big\|\,p(z_t\mid h_t)\big],\qquad
\mathcal L_{\mathrm{rep}}=\KL\!\big[q(z_t\mid h_t,x_t)\,\big\|\,\sg\big(p(z_t\mid h_t)\big)\big].$$
The dynamics loss moves only the prior toward a frozen posterior — the prior learns to predict the representation. The representation loss moves only the posterior toward a frozen prior — it nudges the representation to be *predictable*. Weight the dynamics term fully ($\beta_{\mathrm{dyn}}=1$) and the representation term lightly ($\beta_{\mathrm{rep}}=0.1$): the prior chases hard, the posterior yields only gently. That asymmetry is the point — the prior does most of the moving, so the representation stays informative.

Second, even with that balance, a small fixed weight will sometimes drive the KL to near zero, and a KL near zero is a degenerate solution: the dynamics become trivially predictable precisely because the latent stopped carrying information about the input — posterior collapse. I want the optimizer to *stop* pushing the KL down once it's already small, and spend its effort on the prediction loss instead. So floor each KL term: $\max\!\big(1,\,\KL[\cdots]\big)$, clipping below one nat ($\approx 1.44$ bits). Below one nat the term is constant, its gradient is zero, and the model is freed to focus on reconstruction; above one nat the KL is active. These "free bits" mean the latent is *required* to carry at least a nat of information, which structurally prevents collapse. Free bits plus a small representation weight together resolve the strong-vs-weak-regularizer dilemma without a per-domain knob: the floor guarantees enough information for the static-background case, and the light representation pressure plus symlog'd inputs keep the texture-rich case from drowning in irrelevant detail. So the world-model loss is
$$\mathcal L(\phi)=\mathbb E_{q_\phi}\Big[\textstyle\sum_t \beta_{\mathrm{pred}}\mathcal L_{\mathrm{pred}}+\beta_{\mathrm{dyn}}\mathcal L_{\mathrm{dyn}}+\beta_{\mathrm{rep}}\mathcal L_{\mathrm{rep}}\Big],$$
with $\mathcal L_{\mathrm{pred}}=-\ln p(x_t\mid z_t,h_t)-\ln p(r_t\mid z_t,h_t)-\ln p(c_t\mid z_t,h_t)$, $\beta_{\mathrm{pred}}=\beta_{\mathrm{dyn}}=1$, $\beta_{\mathrm{rep}}=0.1$, and the two KLs free-bit-clipped at $1$.

While I'm in the latent, there's a stability bug to head off. The encoder and dynamics predictor output *categorical* distributions over the stochastic state (a vector of softmaxes, with straight-through gradients through the sample). A softmax can drift arbitrarily close to deterministic, and when a categorical posterior goes near-deterministic the KL can spike — the same pathology reported for deep variational autoencoders — and a one-hot-ish distribution also produces zero probabilities and infinite log-probs. So mix in a floor of uniform mass: each encoder, dynamics-predictor, and actor categorical becomes $0.99$ times the network's softmax plus $0.01$ times uniform. The $1\%$ unimix makes it *impossible* for these distributions to become deterministic, which keeps the KL well-behaved and the log-probs finite.

Now the behavior side, which has its own scale problem and it's the subtlest one. The actor maximizes return and explores via an entropy regularizer, and the regularizer needs a coefficient $\eta$ trading exploration against exploitation. The right $\eta$ depends on the reward scale *and* frequency — and worse, the *interaction* I actually want is: explore more when rewards are sparse, exploit more when they're dense or nearby, while being completely invariant to someone multiplying all rewards by a constant. A fixed $\eta$ against raw returns can't do that, because raw return magnitude is arbitrary. So I have to normalize the return scale before it meets a fixed $\eta$ — but normalize it in a way that *preserves information about reward frequency*, otherwise I lose the explore-when-sparse behavior.

Let me see why the standard normalizations fail at exactly this. Normalize advantages, as the on-policy yardstick does: that puts a *fixed* amount of emphasis on maximizing return over entropy regardless of whether any reward is within reach. Under sparse rewards the advantages are mostly noise from function approximation, and scaling them up to unit magnitude amplifies that noise until it outweighs the entropy term and exploration stagnates — the opposite of what I want when rewards are sparse. Normalize returns by their standard deviation: under sparse rewards the std is near zero, so dividing by it blows the returns up arbitrarily, again drowning the entropy regularizer regardless of the actual reward size. Target a fixed average entropy by constrained optimization: robust, but it explores too slowly under sparse rewards (it's not pushed to seek reward) and converges lower under dense rewards (it's forced to stay stochastic). I couldn't find a fixed setting of any of these that holds across domains.

What survives all three failure modes? I want to scale *down* large return magnitudes so a fixed $\eta$ isn't swamped, but I must *not* scale *up* small returns, because scaling up small/sparse returns is precisely the noise-amplification trap. So divide returns by a range $S$, but only when $S$ is large — floor the divisor at one: divide by $\max(1,S)$ with threshold $L=1$. Small returns ($S<1$) pass through untouched, so under sparse rewards the entropy term keeps its grip and the agent keeps exploring; large returns get compressed into roughly the unit interval so a fixed $\eta$ behaves consistently. And the range $S$ must be robust to the multimodal/outlier returns I worried about earlier — using the min-to-max range would let a single lucky high-return episode crush everything else. So measure the range between the $5$th and $95$th percentiles of the returns in the batch, and smooth it with an exponential moving average so it doesn't jitter:
$$S\doteq\EMA\!\big(\Per(R^\lambda,95)-\Per(R^\lambda,5),\,0.99\big).$$
Note I only need to *divide* by the range, not subtract an offset, because subtracting a state-independent constant from the return leaves the policy gradient unchanged — the baseline already handles the offset. With this, a single fixed entropy scale $\eta=3\times10^{-4}$ works everywhere, and the explore-when-sparse / exploit-when-dense behavior comes for free because I deliberately left small returns unscaled.

Now, what gradient do I use for the actor? My earlier instinct in this line of work was the pathwise gradient: the dynamics are differentiable, so backpropagate the imagined return through the transition into the actions and read off $\partial(\text{return})/\partial(\text{action})$. That's lovely for continuous actions, but it needs a clean differentiable path, and for discrete actions there isn't one. If I want *one* estimator across discrete and continuous domains — and I do, that's the whole fixed-hyperparameter creed — then I should use the estimator that doesn't care about the action type. That's the score-function / Reinforce estimator: weight the log-probability of each action by a scalar advantage, $\mathbb E[A\,\nabla_\theta\log\pi_\theta(a\mid s)]$. Its usual weakness is variance, but I've already built the two things that tame it — a value baseline (so $A=R^\lambda_t-v(s_t)$ subtracts the state value) and the return normalization above (so the advantage scale is controlled). With those in hand, Reinforce becomes a single code path that works for discrete *and* continuous actions, which is worth more across domains than the lower variance of the pathwise gradient on the continuous subset. So the actor surrogate is
$$\mathcal L(\theta)=-\textstyle\sum_t\Big[\sg\!\Big(\tfrac{R^\lambda_t-v_\psi(s_t)}{\max(1,S)}\Big)\,\log\pi_\theta(a_t\mid s_t)+\eta\,\mathrm H\!\big[\pi_\theta(a_t\mid s_t)\big]\Big],$$
where the advantage is stop-gradiented (it's a multiplier, not a path) and divided by the floored range.

The critic. It predicts the distribution of the $\lambda$-return for each model state, using the symexp-twohot head from before. The $\lambda$-return blends $k$-step bootstraps so a finite imagination horizon $H$ can still account for reward beyond it — the value soaks up the tail — and it carries the predicted continuation flag $c_t\in\{0,1\}$ as the per-step discount so an imagined step that probably ends the episode is weighted down:
$$R^\lambda_t\doteq r_t+\gamma\,c_t\big((1-\lambda)\,v_t+\lambda\,R^\lambda_{t+1}\big),\qquad R^\lambda_T\doteq v_T,$$
with $v_t=\mathbb E[p_\psi(\cdot\mid s_t)]$ read out as the mean of the critic's distribution. Let me make sure this recursion is the bias/variance dial I think it is by unrolling one step: $R^\lambda_t=r_t+\gamma c_t(1-\lambda)v_t+\gamma c_t\lambda R^\lambda_{t+1}$; the piece $r_t+\gamma c_t v_t$ is the one-step bootstrap, carried with weight from the $(1-\lambda)$ branch, while the $\lambda$ branch recurses into the next step's return, and continuing the substitution puts geometric weight $(1-\lambda)\lambda^{k-1}$ on the $k$-step bootstrap with the leftover mass on the full-horizon return — small $\lambda$ leans on short low-variance bootstraps, $\lambda\to1$ on the long return. I'll use $\lambda=0.95$, $\gamma=0.997$ (a discount horizon of about $333$), and an imagination horizon of about $H=15$ steps from each replayed start state. The critic loss is the twohot cross-entropy regressing $R^\lambda_t$ as a soft label.

There's a self-reference problem in the critic that I have to handle: $R^\lambda_t$ is built from the critic's own bootstrap values $v_t$, so the target depends on the network being trained. The textbook fix is a frozen target network, but a frozen copy means I'd have to compute returns with stale values. Instead, keep training the critic against the *current* network's returns, but add a regularizer pulling the critic toward an exponential moving average of its own past parameters (decay $0.98$). That stabilizes the bootstrap — the EMA is the slow, stable reference — while still letting me compute returns with the live critic. And to get more value signal in environments where rewards are hard to predict, apply the critic loss not only to imagined trajectories ($\beta_{\mathrm{val}}=1$) but also to replayed real trajectories ($\beta_{\mathrm{repval}}=0.3$), using the imagination returns at the rollout start states as on-policy value annotations and then computing $\lambda$-returns over the real replay rewards.

Two more knobs, both about the optimizer, both currently coupled to loss scale. I'm clipping gradients to keep training stable, but a global gradient-norm threshold has to be retuned whenever I change a loss weight or a loss function, because the gradient magnitude scales with the loss. That coupling is itself a knob. Decouple it: clip each tensor's gradient relative to *its own weight matrix's* norm — adaptive gradient clipping, clip a per-tensor gradient when it exceeds $30\%$ of the L2 norm of the corresponding weights (with a small $\epsilon=10^{-3}$). Now the clip threshold is dimensionless and scale-free; I can change loss scales without touching it. And the optimizer itself: Adam computes both the momentum and the normalizer from the raw gradients, which I found occasionally unstable and which forces a not-too-small $\epsilon$. Swap to LaProp, which first normalizes the gradient by an RMSProp denominator and *then* applies momentum to the already-normalized gradient. Because the momentum sees a pre-normalized signal, I can take $\epsilon$ all the way down to $10^{-20}$ without the instabilities, with the usual $\beta_1=0.9,\beta_2=0.99$ and a learning rate of $4\times10^{-5}$.

A couple of architecture choices, each forced by a concrete constraint rather than taste. The sequence model is a GRU, but I want a large recurrent memory and a dense GRU's recurrent weight matrix grows quadratically with the number of units — too many parameters and FLOPs. Make the recurrent weights *block-diagonal* with 8 blocks, so memory capacity grows without the quadratic blow-up; feed the GRU a linear embedding of the sampled latent, the action, and the recurrent state so information still mixes across blocks. For normalization and activation, RMSNorm with SiLU throughout. Images are encoded with stride-2 convolutions down to a small spatial resolution and decoded with transposed stride-2 convolutions ending in a sigmoid; vector inputs are symlog'd then run through small MLPs; the actor and critic are 3-layer MLPs, the reward and continue predictors 1-layer MLPs.

Stepping back: every change I've made is the resolution of a specific brittleness, and each one removes a knob. Symlog squared loss removes the loss-scale knob on deterministic predictions; symexp-twohot removes it on stochastic reward/value predictions and decouples gradient size from target size; KL balancing plus free bits removes the per-domain representation-loss scale and forecloses posterior collapse; $1\%$ unimix removes KL spikes and infinite log-probs; percentile return normalization with a floored divisor removes the entropy-scale knob while preserving explore-when-sparse; the Reinforce estimator gives one actor update across discrete and continuous; the EMA-regularized distributional critic removes the target-network staleness; adaptive gradient clipping decouples the clip threshold from loss scale; and LaProp removes the Adam-$\epsilon$ fragility. The world model, the imagination, and the $\lambda$-return are the inherited skeleton; the contribution is the robustness layer that lets one configuration sit on top of all of it.

Let me land it on code.

```python
# --- robust scalar transform: fixed, sign-preserving, ~identity near 0 ------
def symlog(x):  return jnp.sign(x) * jnp.log1p(jnp.abs(x))
def symexp(x):  return jnp.sign(x) * jnp.expm1(jnp.abs(x))

# --- twohot soft label over exponentially spaced bins -----------------------
BINS = symexp(jnp.linspace(-20., 20., 255))            # dense near 0, huge reach, signed

def twohot(y, bins=BINS):
    y = jnp.clip(y, bins[0], bins[-1])
    k = (bins <= y[..., None]).sum(-1) - 1              # lower bin index
    k = jnp.clip(k, 0, len(bins) - 2)                   # keep an upper neighbor
    lo, hi = bins[k], bins[k + 1]
    w_hi = jnp.abs(y - lo) / jnp.abs(hi - lo)           # linear interp weight
    w_lo = jnp.abs(hi - y) / jnp.abs(hi - lo)
    return onehot(k, len(bins)) * w_lo + onehot(k + 1, len(bins)) * w_hi

def twohot_mean(probs, bins=BINS):
    neg = bins < 0
    neg_sum = jnp.flip(probs[..., neg] * bins[neg], -1).sum(-1)
    pos_sum = (probs[..., ~neg] * bins[~neg]).sum(-1)
    return neg_sum + pos_sum

class TwoHotHead:                                        # reward predictor and critic
    def __init__(self, net):
        self.net = net                                   # output weights init to ZERO
    def pred(self, feat):
        return twohot_mean(jax.nn.softmax(self.net(feat)))       # weighted-avg readout
    def loss(self, feat, target):                        # scale-decoupled cross-entropy
        logits = jax.nn.log_softmax(self.net(feat))
        return -(twohot(target) * logits).sum(-1)

class SymlogMSEHead:                                     # decoder / deterministic reads
    def pred(self, feat):  return symexp(self.net(feat))
    def loss(self, feat, target):
        return 0.5 * (self.net(feat) - symlog(target)) ** 2

def unimix(probs, alpha=0.01):                           # 1% uniform floor on categoricals
    return (1 - alpha) * probs + alpha / probs.shape[-1]

# --- world model loss: prediction + KL-balanced, free-bit-floored dynamics --
def world_model_loss(model, batch):
    post, prior = model.rssm.observe(model.encode(batch['x']), batch['action'])
    feat = model.rssm.get_feat(post)
    L_pred = (model.dec.loss(feat, batch['x'])               # symlog MSE on vector inputs
              + model.rew.loss(feat, batch['reward'])        # twohot cross-entropy
              - model.con.logp(feat, batch['cont']))         # logistic regression NLL
    kl_dyn = jnp.maximum(1., kl(sg(post), prior))            # free bits: clip below 1 nat
    kl_rep = jnp.maximum(1., kl(post, sg(prior)))            # prior chases hard, post yields soft
    return (1.0 * L_pred + 1.0 * kl_dyn + 0.1 * kl_rep).mean()

# --- lambda-return with continuation flags as per-step discount -------------
def lambda_return(reward, value, cont, bootstrap, lam=0.95, gamma=0.997):
    R = bootstrap                                            # bootstrap: R_T = v_T
    out = []
    for t in reversed(range(len(reward))):
        R = reward[t] + gamma * cont[t] * ((1 - lam) * value[t] + lam * R)
        out.append(R)
    return jnp.stack(out[::-1])

# --- actor + critic on an imagined rollout ----------------------------------
def actor_critic_loss(traj, actor, critic, slow_critic, retnorm):
    feat = traj['feat']
    value = critic.pred(feat)
    ret = lambda_return(traj['reward'][1:], value[1:], traj['cont'][1:], value[-1])

    # percentile return normalization: scale DOWN large returns, floor at L=1
    S = retnorm.update(percentile(ret, 95) - percentile(ret, 5))   # EMA(., 0.99)
    adv = (ret - value[:-1]) / jnp.maximum(1.0, S)

    # actor: Reinforce (one estimator for discrete AND continuous) + fixed entropy
    logpi = actor.dist(feat[:-1]).log_prob(traj['action'][:-1])
    actor_loss = -(sg(adv) * logpi).mean() - 3e-4 * actor.dist(feat[:-1]).entropy().mean()

    # critic: twohot cross-entropy to the (sg) lambda-return + EMA self-regularizer
    critic_loss = (critic.loss(feat[:-1], sg(ret))
                   + 1.0 * critic.loss(feat[:-1], sg(slow_critic.pred(feat[:-1])))).mean()
    return actor_loss, critic_loss

# --- gradient application: scale-free clip + LaProp -------------------------
def apply_gradients(grads, params, opt):
    grads = adaptive_grad_clip(grads, params, clip=0.3, eps=1e-3)  # per-tensor: 30% of weight norm
    opt.step(grads)             # LaProp: RMSProp-normalize, THEN momentum; eps=1e-20, lr=4e-5
```

Tracing the causal chain once more: the scarce resource is not data here, it's *human tuning*, so the target is one fixed configuration across heterogeneous domains. Brittleness lives wherever a hyperparameter's right value depends on the arbitrary scale or shape of rewards, returns, or observations, so I went knob by knob and made each objective scale-invariant — symlog and symexp-twohot for predictions whose targets span orders of magnitude, KL balancing with free bits for a representation that stays informative without a per-domain regularizer, $1\%$ unimix to keep categoricals from spiking, a percentile-range return normalization with a unit floor so a single entropy scale explores when rewards are sparse and exploits when they are dense regardless of reward rescaling, a Reinforce actor that is one update path for discrete and continuous, an EMA-regularized distributional critic that bootstraps stably, and adaptive gradient clipping with LaProp so the optimizer is decoupled from loss scale. The inherited world model dreams the futures, the actor-critic learns inside the dream, and the robustness layer is what lets the same numbers learn across all of it.
