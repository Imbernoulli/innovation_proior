Let me start from the thing that has bothered me for years, because it is genuinely strange and I want to stare at it until it gives something up. The simplex method solves linear programs by walking from vertex to vertex of the feasible polyhedron, each step improving the objective, and in practice it does this in a number of steps that looks linear in the size of the program. Everyone who runs it sees this. And yet I can write down, in a few lines, a family of linear programs on which it takes 2ᵈ steps — Klee and Minty's squashed cube, a combinatorial d-cube whose facets are tilted just enough that Dantzig's largest-coefficient rule snakes through all of its corners before reaching the top. So the worst case is exponential and the everyday case is fast, and the gap between them has gone unexplained for half a century. I don't want to add another partial result on top of the pile. I want to understand *why the two analyses we have both miss*.

So let me interrogate the two analyses directly. Worst-case analysis says: report the maximum over all inputs of the running time. On the Klee–Minty cube that maximum is exponential, and the verdict is "intractable." But look at what that cube actually is. It is a cube whose facets have been tilted by precise amounts; the hardness depends on those amounts being exactly right. If I nudge a facet a hair, the snake-path collapses. The bad input is not a region — it is a thin, brittle sliver in the space of all linear programs. Worst-case analysis reports it because the *max* operator is blind to whether the maximizing input is a fat region of the input space or a single contrived point. It cannot tell "hard everywhere near here" from "hard only at this measure-zero filament." That is the defect: the max sees the filament and reports catastrophe.

Now average-case. Borgwardt did something beautiful: draw the constraint vectors from a spherically symmetric distribution — Gaussians centered at the origin — and prove the expected number of steps of the shadow-vertex rule is polynomial. Smale and Megiddo did the analogous thing for the self-dual parametric method. These are real theorems and they gave the first inkling of why simplex is fast. But I keep coming back to Edelman's warning, which I think is exactly right: it is a mistake to link "a random matrix" with "a typical matrix." A spherically symmetric random input has enormous special structure with probability essentially one — the rotational symmetry itself is doing the work. The bound is a statement about *that distribution*, and a real linear program from a refinery scheduling problem or a network flow is nowhere near spherically symmetric. So average-case analysis answers the wrong question: it tells me about the behavior on a cloud of idealized random inputs, not about the behavior near the particular, structured, real input I care about.

And here is the sharp version of the objection, because I want to be sure average-case can't be patched. Suppose I had an average-case analysis so strong it bounded every moment of the running-time distribution. Would that tell me the algorithm is fast near a fixed real input ā? No — because the neighborhood of ā that I care about, a little ball of radius proportional to the noise in my data, has exponentially small probability under the global random distribution. A statement about the average, even about all the moments of the average, integrates over a space in which my neighborhood is invisible. Higher moments don't rescue me. The two analyses fail for opposite reasons: worst-case is dominated by an isolated brittle adversary, average-case is dominated by random-looking inputs that drown out the real ones.

So what do I actually want? I want to keep the adversary — I don't want to assume the input is random, because real inputs aren't — but I want to forbid the adversary from being *infinitely precise*. The Klee–Minty cube is only hard because it can be specified to infinite precision. Real data can't. Every real linear program comes from measurements, from finite-precision arithmetic, from rounding — there is always a little noise smeared over the low-order digits. The adversary may pick the nastiest center it likes, but then nature jiggles it slightly before I see it. If even a tiny jiggle destroys the hardness, then the hardness was never real.

Let me try to make that into a complexity measure. Worst-case is a max over inputs of the running time. Average-case is an expectation over random inputs. What I want seems to be *both, nested in the right order*: the adversary maxes over inputs, and then for each chosen input I take an expectation over a small random perturbation of it. So for an algorithm A with complexity C on input x, write down

  smoothed complexity (n, σ) = max over x of size n of  E_g [ C_A( x + σ‖x‖ g ) ],

where g is a vector of independent standard Gaussians, so x + σ‖x‖g is x with a Gaussian perturbation whose standard deviation is σ times the magnitude of x. The max keeps the adversary; the expectation softens it; σ controls how much. I'll measure complexity as a function of both the size n and the perturbation magnitude σ.

Before I trust this, I want to actually watch it do what I claimed — that it interpolates between the two analyses — instead of just asserting it from the formula. Let me build the smallest caricature that has the feature I care about: a "hard sliver" cost. Put a one-dimensional input x and let the cost be C(u) = 1 if |u − c| < δ and 0 otherwise, a thin band of half-width δ at a point c — this is my toy Klee–Minty filament. The adversary, maximizing over x, will place x exactly at c, on the band. With noise of standard deviation σ‖x‖ = σ|c|, the smoothed cost at that adversarial center is E_g[C(c + σ|c|·g)] = P[|σ|c|g| < δ] = P[|g| < δ/(σ|c|)] = erf( δ / (σ|c|√2) ). That closed form is already telling, but let me confirm it numerically and read off the two ends. With c = 1 and δ = 0.02:

  σ = 0       cost = 1.00000   (adversary sits on the band, no noise)
  σ = 1e-4    E_g = 1.00000
  σ = 1e-2    E_g = 0.95434    (erf form: 0.95450)
  σ = 1e-1    E_g = 0.15710    (erf form: 0.15852)
  σ = 1       E_g = 0.01627    (erf form: 0.01596)
  σ = 10      E_g = 0.00158    (erf form: 0.00160)
  σ = 1000    E_g = 0.00001    (erf form: 0.00002)

The Monte-Carlo column tracks erf(δ/(σ|c|√2)) the whole way down, so I trust the closed form. And now the interpolation is not a hope, it's the limit of an actual function: as σ → 0, δ/(σ|c|√2) → ∞ and erf → 1 — the smoothed cost recovers the worst case, the adversary's band is seen at full strength. As σ → ∞, the argument → 0 and erf → 0 — the band washes out exactly as it does under a wide-Gaussian average. So this single quantity really is a dial that slides continuously from the worst case at one end to the average case at the other, and the brittle band's contribution decays smoothly in between. The interesting region is σ small but nonzero — a perturbation that lives in the low-order digits, the size of real measurement noise. That's the regime that should correspond to "real inputs," and in the toy it is exactly the regime where erf(δ/(σ|c|√2)) has already fallen well below 1 for any band thinner than the noise.

Why Gaussian and not some other noise? Several reasons converge. It's the natural continuous noise model. It's rotationally invariant, which will matter because the geometry I'll be analyzing doesn't prefer a direction. Gaussians add — the sum of two independent Gaussians is Gaussian with summed covariance — which means I'll be able to split one perturbation into two and analyze them in stages. And a Gaussian integrated over a half-space reduces to a one-dimensional integral, which I'll lean on constantly. And why scale the standard deviation by ‖x‖ — by σ·max_i‖(y_i, a_i)‖ for a linear program? Because otherwise the measure isn't scale-free: doubling all the data should be the same problem, and tying the noise to the data's magnitude makes it so. (In the toy above this is exactly why I wrote the noise as σ|c|, not σ — it's what makes the erf argument depend only on the ratio δ/|c|, i.e. on the band's *relative* width.)

Now, should I demand the smoothed complexity be polynomial in 1/σ, or in log(1/σ)? My instinct is the weaker-looking poly(1/σ), and let me check that instinct rather than just assert it. Suppose I had an algorithm with smoothed complexity polynomial in d, n, and log(1/σ). Take σ ~ 2^(−L) where L is the bit-length of the input; then log(1/σ) = log(2^L) = L exactly, and the bound becomes polynomial in d, n, L — that's a worst-case polynomial bound in the bit complexity, essentially Khachiyan-strength. So "poly in log(1/σ)" is so strong it almost coincides with "poly in the worst case," and it's not the phenomenon I'm chasing — if I could prove it I would have re-proved that LP is in P the hard way, with the perturbation doing no real work. The honest middle ground is poly(1/σ): call an algorithm to have *polynomial smoothed complexity* if its smoothed complexity is polynomial in n and 1/σ. That's the target. Now to find a pivot rule and prove the simplex method hits it.

The first question is which pivot rule. I need to compute, or bound, the expected number of pivot steps under a Gaussian perturbation of the constraints. For most pivot rules the only handle I have on "which vertex do we visit at step i" is to iterate the rule — vertex i is whatever the rule picks from vertex i−1. That iteration couples the steps together, and once the steps are coupled, the random perturbation entangles everything and I can't get a clean probabilistic statement: the i-th step's distribution depends on all the earlier random choices. What I'd need is a pivot rule whose sequence of visited vertices has a *closed-form geometric description*, independent of running the algorithm — something I can take an expectation of directly. Scanning the rules I know, the one with that property is shadow-vertex, and it's worth being precise about why, because the property is the whole reason I'm choosing it.

In two dimensions the simplex method is trivial: the feasible region is a polygon and the method just walks along its boundary. Shadow-vertex lifts that triviality to high dimensions by *projecting down to two dimensions*. Take the objective z. Take a second objective t that happens to be maximized at a vertex x₀ I already know. Project the whole feasible polyhedron orthogonally onto the plane spanned by t and z; call the projection the *shadow*. The shadow is a convex polygon, and three facts make it useful: every vertex of the polygon is the image of a vertex of the polyhedron; every edge of the polygon is the image of an actual edge between two adjacent vertices of the polyhedron; and both the start vertex x₀ and the true z-optimizer project to vertices of the polygon. So if I rotate the objective continuously from t to z — set q_λ = (1−λ)t + λz and slide λ from 0 to 1, tracking the vertex that optimizes q_λ — I should be walking around the boundary of the shadow polygon, from the image of x₀ to the image of the z-optimum. Each step moves to a neighboring vertex, so the number of pivots is at most the number of vertices of the shadow polygon.

I want to be sure I actually believe that "walk along the shadow boundary" picture before I build an entire analysis on it, so let me run it once on a concrete polytope and look at what happens, rather than take it on faith. Take n = 8 random constraints aᵢᵀx ≤ 1 in d = 3, enumerate the feasible vertices (there are 9 of them), pick two objectives z = (1, 0.3, −0.2) and t = (−0.4, 1, 0.5), and project the vertices onto the plane span(t, z). The convex hull of the projected points — the shadow polygon — comes out with 7 edges. Now I rotate q from t toward z in fine steps and record each time the optimizing vertex changes; the path of distinct vertices it visits has length 2. Two things I wanted to check actually hold on this instance: every vertex the rotation visits projects onto the boundary of that 7-edge shadow polygon (I verified each visited vertex coincides with a hull vertex of the projection), and the pivot count 2 is at most the shadow edge count 7. So the reduction is real and not just suggestive: the running time of this rule is controlled by a single geometric quantity — **how many edges does this two-dimensional shadow have?**

That is the move that makes the perturbation analysis tractable, and the small example is also why I now believe it: the set of vertices I will visit is described not by an iteration but by a projection — the shadow of the polytope onto a fixed plane — and a projection is something I can take an expectation of. So the central object I must bound is the expected size of the shadow of a Gaussian-perturbed polytope onto a fixed plane. If that expectation is polynomial in n, d, 1/σ, I'm in business, modulo wiring it into a real algorithm.

Let me set up the shadow bound cleanly. I'll work in the polar picture, which Borgwardt taught and which is where the probability is natural. Assume for now the right-hand sides y_i are positive and, by rescaling, that the constraint vectors a_i are centered at points of norm at most 1, perturbed to a_i ~ N(ā_i, σ²I). For an objective q, the optimal basis is a d-subset I such that the simplex conv(a_i : i∈I) (after dividing by y_i) is a facet of the convex hull of the origin together with all the a_i/y_i, and q lies in the cone spanned by {a_i : i∈I}. The shadow, as q ranges over the plane span(t,z), is the collection of these optimal simplices — the facets that the rotating ray q sweeps across. So:

  shadow size = number of distinct optimal bases I as q rotates once around the plane.

I want E[shadow size] ≤ some polynomial D(n,d,σ). Let me first reduce to the hard regime. If σ is large — bigger than 1/(3√(d ln n)) — I can simply scale all the data down until σ equals 1/(3√(d ln n)); scaling can only shrink the norms of the centers, so it doesn't violate "centers of norm ≤ 1," and the shadow is scale-invariant. So assume σ ≤ 1/(3√(d ln n)) from here on. Good, that buys me a regime where Gaussian tail bounds are comfortable.

Now, counting "distinct optimal bases as q rotates" is a discrete count, and I want to turn it into an integral of probabilities so I can use linearity of expectation. Assume without loss of generality z ⊥ t, and parametrize the plane by angle: q_θ = z sin θ + t cos θ. Discretize the circle into m equal angles and count how many adjacent pairs (θ_i, θ_{i+1}) have *different* optimal bases; as m → ∞ this count should converge to the number of edges of the shadow. I should make sure this limit is honest rather than assume it. The only way the discretized count could fail to converge to the true edge count is if, with positive probability, there is a basis that is optimal on an interval of θ of length zero — a basis that flashes optimal at a single angle and vanishes. Such a configuration requires two basis-change angles to coincide exactly, which pins the point coordinates to a relation of codimension one; under the product of nondegenerate Gaussian densities a codimension-one set has measure zero, so this happens with probability zero and the limit is clean. Let E_i be the indicator that the optimal basis at angle 2πi/m differs from the one at the next angle. Then shadow size = Σᵢ E_i, and

  E[shadow size] = lim_{m→∞} Σᵢ E[E_i].

If the optimal basis changes between two adjacent angles, then at angle 2πi/m the ray q is *right at the edge* of the cone of its current optimal simplex — it's about to fall out of one cone into the next. In other words, the angle from q to the boundary of its optimal simplex is at most the angular step 2π/m. Define ang_q(a₁,…,aₙ) to be the angle from q to the boundary of the simplex of the current optimal basis. Then E_i = 1 implies ang_{q_θ} ≤ 2π/m. So

  E[shadow size] ≤ lim_{m→∞} Σᵢ P[ ang_{q_{2πi/m}} < 2π/m ].

And now I can see the shape of what I need. If I can prove that P[ ang_q < ε ] grows only *linearly* in ε — say P[ang_q < ε] ≤ K·ε for a constant K = K(n,d,σ) — then each term is ≤ K·(2π/m), there are m terms, the m's cancel, and the whole sum is ≤ 2π·K, independent of the discretization. So the entire shadow bound reduces to one inequality: the probability that q is within angle ε of the boundary of its optimal simplex is at most K·ε, with K polynomial in n, d, 1/σ. The linearity is the load-bearing requirement — a superlinear bound like K·ε^{1/2} would leave a residual m^{1/2} that blows up as the discretization refines, so it has to be exactly linear in ε for the m's to cancel. That tells me precisely what to aim for: near-boundary events rare in proportion to how near, which is exactly the non-degeneracy a perturbation should buy.

Before I chase K, let me restrict to a high-probability good event so the geometry is bounded. Let P be the event that every ‖a_i‖ ≤ 2. I want to confirm this is actually high-probability in my regime and not just plausibly so. Each a_i is centered at norm ≤ 1, so ‖a_i‖ > 2 forces ‖a_i − ā_i‖ > 1, i.e. ‖a_i − ā_i‖²/σ² > 1/σ², and ‖a_i − ā_i‖²/σ² is a χ²_d variable. With σ² = 1/(9 d ln n) the threshold is 1/σ² = 9 d ln n = 9 ln n per degree of freedom, far out in the tail. The Chernoff bound P[χ²_d > t] ≤ (t/d · e^{1−t/d})^{d/2} with t/d = 9 ln n gives, for instance at n = 100, d = 5: a log-probability around −91.8, versus the target log of n^{−2.9d} = −66.8 — comfortably below. I checked this across n ∈ {10,100,1000}, d ∈ {3,5,10} and the Chernoff exponent beats −2.9d·ln n with margin in every case. So P[‖a_i‖ > 2] ≤ n^(−2.9d), hence P[not P] ≤ n^(−2.9d+1), which is below binom(n,d)^(−1). Outside P the shadow can have at most binom(n,d) edges anyway, so the contribution of the bad event to the expectation is at most 1. I'll carry the "+1" and work inside P, where all the points live in a ball of radius 2 and distances between basis points are at most 4.

Now to bound P[ang_q < ε]. The angle being small means q is nearly on the boundary of its optimal simplex; the boundary of a d-simplex is the union of its d facets, so by a union bound over which facet q is near, and a union bound over which basis I is optimal, it's enough to bound, for a fixed basis I and a fixed dropped vertex j: the probability that I is the optimal basis *and* the angle from q to the facet simp(A_{I−j}) is less than ε. There are binom(n,d) bases and d facets each, so naïvely I'd pay a factor of binom(n,d)·d out front — and binom(n,d) is far too large to absorb. But here's the saving observation: only one basis is optimal at a time, so the events "I is optimal" are disjoint and Σ_I P[I optimal] ≤ 1. So after Bayes, the real quantity is the conditional probability:

  P[ angle from q to facet simp(A_{I−j}) < ε  |  I is the optimal basis ],

times a sum-over-I of P[I optimal] which is ≤ 1, times the union factor d. So the whole thing is at most d·(max over I,j of that conditional probability) — the dangerous binom(n,d) has cancelled against the fact that optimality is exclusive, and I'm left with only the d facets to pay for. So now: condition on I = {1,…,d} being the optimal basis, and bound the probability that q is within angle ε of the facet through a₂,…,a_d.

The coordinates of the d basis points are the wrong coordinates for this event. Instead of describing a₁,…,a_d directly, describe them by (ω, s, b₁,…,b_d): the unit normal ω and offset of the plane the points lie on, and the in-plane positions b_i after rotating the plane normal to q. This is the Blaschke change of variables from integral geometry. The Jacobian of this map is (d−1)!·⟨ω,q⟩·vol(simp(b₁,…,b_d)), and after I also condition on the other n−d constraints lying on the right side of this plane, the conditional density is proportional to that Jacobian times the d basis-point Gaussian densities and the n−d half-space probabilities. Two features of this density will turn out to be exactly what I need. The vol(simp) factor *rewards* fat simplices and penalizes slivers — it makes a degenerate, near-flat base less likely, which is good because near-flat is precisely the bad case. And the ⟨ω,q⟩ factor is the cosine of the angle between the facet's normal and q; it will let me separate the geometry into two independent-feeling pieces.

I want the angle from q to the facet simp(a₂,…,a_d) to be at least something I can control. Geometrically, the angle from q to the facet decomposes as: how far the facet's affine hull is from passing through the relevant point (a distance), times how aligned the facet's normal is with q (the ⟨ω,q⟩ cosine), divided by a bounded constant. Concretely, working it out, the angle is at least dist(0, aff(b₂,…,b_d))·⟨ω,q⟩ / (2 + 4√2). The "2 + 4√2" is just the bound on the relevant lengths inside the good event P. So:

  ang(q, facet) small  ⟹  dist(0, aff(b₂,…,b_d)) · ⟨ω,q⟩ is small.

These are two genuinely different geometric failures. The first says the facet's affine hull nearly passes through the origin — the simplex is positioned degenerately. The second says the facet's normal is nearly perpendicular to q — the facet is seen almost edge-on from q. I should not union-bound over arbitrary thresholds, because there is no preferred threshold where "distance failure" stops and "incidence failure" starts. The clean way is to keep the product and use the scale of one factor to decide how small the other must be. Let me state the lemma I want and then *check that it's true*, because the exact form of the hypotheses turns out to matter a lot. Claim: if F is nonnegative with P[F ≤ ε] ≤ αε, and G is nonnegative with max over x of P[G ≤ ε] ≤ (βε)², then P[F·G ≤ ε] ≤ 4αβε. The proof is a dyadic decomposition: either F ≤ 2βε, or F lies in a band F ∈ (2ⁱβε, 2^{i+1}βε] with i ≥ 1; in band i, FG ≤ ε forces G ≤ ε/F ≤ 2^{−i}/β, whose probability is ≤ (β·2^{−i}/β)² = 2^{−2i}; the event F ≤ 2^{i+1}βε has probability ≤ 2^{i+1}αβε; so band i contributes at most 2^{i+1}αβε · 2^{−2i}, and the base case contributes 2αβε. The total coefficient (in units of αβε) is 2 + Σ_{i≥1} 2^{i+1}·2^{−2i}. I want to actually sum that series rather than wave at "it converges": Σ_{i≥1} 2^{i+1}·2^{−2i} = Σ_{i≥1} 2^{1−i} = 2, so the total is 2 + 2 = 4 exactly — the lemma's constant 4 is tight to my decomposition, not a fudge. And I should confirm the square on G is really necessary, because that dictates what I have to prove about the angle factor. If G only had a linear bound P[G ≤ ε] ≤ βε, then in band i the contribution would be 2^{i+1}αβε · 2^{−i} = 2αβε — a *constant* per band, and Σ_{i≥1} of a constant diverges. So a linear bound on G genuinely does not sum; I need the quadratic one. That settles the division of labor: a *linear* bound on the distance factor and a *quadratic* bound on the angle factor. Let me get each.

Take the distance factor first, P[ dist(0, aff(b₂,…,b_d)) < ε ], which I want bounded by (const · d²/σ⁴)·ε. The perturbed simplex is unlikely to sit so that its facet's affine hull skims the origin, and the route is two steps. First, the simplex is unlikely to be a *sliver* — its height is unlikely to be tiny. Concretely, P[dist(b₁, aff(b₂,…,b_d)) < ε], the probability that the apex b₁ is within ε of the affine hull of the other vertices, is at most (3 e^{2/3} d ε/σ²)³, and hence at most the square of the same scale for the ε range I need. Why cubic, and why does it hold? Fix the shape of the base facet b₂,…,b_d and look at b₁; its distance to the base's affine hull is a coordinate that, under the Gaussian, has a density that's bounded near zero, and crucially the vol(simp) weight in the Blaschke density multiplies in an extra factor that vanishes as the simplex flattens — that's where the *cube* in the exponent comes from, three powers of the small distance reinforcing each other. So a perturbed simplex is, with high probability, genuinely fat. Second, condition on the base facet and compare the origin's height above it to b₁'s height above it. I prove that the ratio dist(0, aff(b₂,…,b_d)) / dist(b₁, aff(b₂,…,b_d)) is below ε with probability at most 75dε/σ². The proof is a contraction argument: shrink the simplex toward b₁ by a factor (1−ε), compare the Gaussian density before and after the shrink, and use the density-smoothness bound to show the thin ratio slice has only O(dε/σ²) of the mass. Combining the squared height bound and this linear ratio bound with the same product lemma gives

  P[ dist(0, aff(b₂,…,b_d)) < ε ] ≤ 900 e^{2/3} d² ε / σ⁴.

Linear in ε, polynomial in d and 1/σ. That's my F, and the σ⁴ here is the first piece of bookkeeping I'll want to track.

Now the angle factor, P[ ⟨ω,q⟩ < ε ], which I want bounded by (const · n/σ²)²·ε² — quadratic in ε, which is exactly what the product lemma demanded. The quantity ⟨ω,q⟩ is the cosine of the angle between the facet normal and q; I'm asking that the facet is unlikely to be seen nearly edge-on. Change variables from ω to its "latitude" c = ⟨ω,q⟩ and "longitude" ψ on the sphere of directions at fixed latitude; the Jacobian is (1−c²)^{(d−3)/2}. The density of c is then c·(1−c²)^{(d−3)/2} times two Gaussian-derived factors: ν₃, the product over the d basis points of their Gaussian densities at the positions forced by (ω,s,b), and ν₂, the product over the n−d *non-basis* constraints of the probability that each lies on the correct side of the facet's plane (these are the constraints that have to be satisfied for I to actually be the optimal basis). The leading c factor already vanishes linearly at c=0, which seeds an extra power — this is half of where the square will come from. Over the small interval c ∈ [0, c₀] with c₀ = σ²/(240n), all the other factors vary by at most a factor of 2: as c moves through this tiny range, ω rotates through an angle ≤ 1.001 c₀, the basis points move a distance ≤ 4√2·(that angle), and by Gaussian smoothness ν₃ changes by at most e^{−O(d c₀/σ²)} which is within a constant; and ν₂ changes by at most a factor 1 − O(n c₀/σ²) — *here is where the n appears*, because there are n−d non-basis half-space probabilities and each can shift as the cutting plane tilts, so the per-constraint Gaussian-tail ratio gets multiplied n times, and the choice c₀ ∝ σ²/n is exactly calibrated to keep that product bounded. With the density nearly flat on [0, c₀] and the leading c-factor giving a linear weight in c, integrating up to ε gives the *quadratic* P[c < ε] ≤ 2(ε/c₀)² = 2(240 n ε/σ²)² ≤ (340 n ε/σ²)². So the quadratic isn't an accident: one power of ε from the integration range and one from the leading c in the density, and the n²/σ⁴ inside is exactly the price of the n side constraints, squared. That's my G.

Now let me combine and, importantly, track the σ powers to make sure they land where I expect rather than asserting the final exponent. The angle-to-facet event has the product dist(0, aff(b₂,…,b_d))·⟨ω,q⟩ below (2+4√2)ε. Apply the product lemma with F's rate α ~ d²/σ⁴ (linear) and G's rate β ~ n/σ² (the quadratic bound (βε)²). The lemma multiplies them: the conditional angle probability is ≤ 4αβ·ε ~ (d²/σ⁴)·(n/σ²)·ε. Reading off the σ exponent: 4 from the distance factor and 2 from the incidence factor's β, summing to 6 — so σ⁶ = σ⁴·σ², and crucially the *square* on G in the lemma is what converts its (n/σ²)² rate back down to a single n/σ² in the linear output, which is why the incidence side contributes only σ² and not σ⁴. So the conditional probability that q is within angle ε of this particular facet is an absolute constant times (n d²/σ⁶)·ε. The n is the side constraints, the d² the distance bound, the σ⁶ the distance-price times the incidence-price. Then the union over the d facets multiplies by d (d² → d³), and the discretization factor 2π and the "+1" from the bad event assemble into the final statement:

  E[ shadow size ] ≤ 58,888,678 · n d³ / min(σ, 1/(3√(d ln n)))⁶.

Let me read off what this says, because it is the whole point. The expected number of edges of the two-dimensional shadow of a Gaussian-perturbed polytope is **O(n d³ / σ⁶)** — polynomial in the number of constraints n, the dimension d, and 1/σ. The exponent on 1/σ is six, and I traced it explicitly through the combination: four from the distance factor (the perturbed simplex's fatness, which the cubic height bound and the vol-weight produce), two from the angle factor (the facet-normal not being perpendicular to q). The dependence on n is linear — one factor of n, straight from the n side constraints in the angle bound, the square on the incidence rate having been cashed back to linear by the product lemma. The dependence on d is cubic, mostly from unioning over the d facets and the d-dependence of the Gaussian geometry. This is the geometric foundation: under perturbation, the shadow is small in expectation, because near-boundary configurations are rare in proportion to how near, and the perturbation makes degenerate, ill-conditioned bases improbable. The brittle worst-case cubes simply can't survive in this expectation — they were the measure-zero ill-conditioned filament, the same kind of thin band whose smoothed contribution I watched decay like erf(δ/(σ√2)) in the toy, and the perturbation pushes off it.

But I'm not done, and I have to be honest about the gap, because the shadow bound proved above fixed the plane span(t,z) *in advance*, unrelated to the data. In an actual algorithm I can't do that. The plane onto which I project is determined by my starting objective t, and t in turn depends on the polytope — I have to choose t so that I know a starting vertex. The plane is correlated with the polytope, and that correlation could in principle conspire with the perturbation to make the shadow large. So I can't just invoke the shadow bound as a black box; I have to handle the correlation. And there are three concrete obstacles before the shadow-vertex method even runs: I need to *know* a feasible starting vertex; the program might be infeasible; and the polar analysis wants y_i > 0. A two-phase construction resolves all three.

Let me build it. First, manufacture an easy program LP′ whose feasible start I know. Pick a d-set I to be the start basis. To make sure the geometry of this start is not degenerate — that the basis matrix A_I isn't nearly singular, which would blow up the numbers I'm about to set — don't pick I once; pick a collection of 3nd ln n random d-sets and keep the one whose A_I has the largest smallest singular value s_min(A_I). I'll justify in a moment why this works. Set the right-hand sides of LP′ to M = a power of two just above max_i‖(y_i,a_i)‖ on the constraints in I, and to √d M²/(4κ) on the rest, where κ is a power of two just below s_min(A_I). Then A_I is a feasible basis of LP′ and it optimizes any objective of the form A_I α with α > 0 — so I have a known starting vertex and a known starting objective t₀ = A_I α, with α drawn at random from the simplex of vectors summing to 1 with each coordinate ≥ 1/d². Solve LP′ with the polar shadow-vertex method from (I, t₀).

Then interpolate from LP′ to the real LP. Introduce one extra variable x₀ ∈ [−1,1] and constraints ⟨a_i, x⟩ ≤ ((1+x₀)/2)y_i + ((1−x₀)/2)y′_i, so x₀ = −1 gives LP′ and x₀ = +1 gives LP. Maximizing x₀ drives the solution from LP′ toward LP. Solve this LP⁺ with the shadow-vertex method, starting from the LP′ solution; if at the end x₀ < 1 the original LP was infeasible, otherwise the x-part is the optimum of LP. The number of steps in this second phase is bounded by the shadow size of a related polytope, so the shadow bound applies — once I deal with the correlation.

The correlation between the projection plane and the polytope is the real subtlety, and the way through uses the one structural property of Gaussians I set aside earlier: they add. Write the total Gaussian perturbation as the composition of a first perturbation and a second, smaller one — a Gaussian of the full variance is the sum of two independent Gaussians, so this split is exact and loses nothing. Choose my projection plane based only on the data after the *first* perturbation. Then show that the plane I picked is well-conditioned with high probability after that first perturbation, so that the *second* perturbation — being small — is very unlikely to move the plane appreciably. Then the shadow I actually traverse is, up to a negligible change, the shadow onto a plane that is fixed *relative to the second perturbation*. And to that fixed-relative-to-second-perturbation plane, the shadow bound applies as a black box, because the second perturbation is independent of the plane. That's the mechanism: condition on the first perturbation to fix the plane, let the second perturbation be the randomness the shadow bound consumes.

Two technical loose ends, and I want to actually close them rather than wave. First, in LP⁺ the points a⁺_i/y⁺_i I feed to the polar shadow bound are not Gaussian — they're ratios. But in every small region the distribution can be approximated by a Gaussian; I make this precise with a lemma comparing two densities ν, μ that agree up to a factor c on a set of measure 1−δ, concluding E_ν[X] ≤ c·E_μ[X] + kδ for bounded X. So I locally swap the non-Gaussian distribution for a Gaussian, pay a small error, and apply the shadow bound. The extensions of the shadow bound I need — to centers of arbitrary norm (rescale by the max norm, replacing σ with σ/max norm), to general covariance matrices with eigenvalues between σ² and 1/(9d ln n) (split the Gaussian again via additivity), and to general positive y (rescale a_i by y_i) — all follow by these rescaling-and-splitting moves, each costing at most a constant or a "+1."

Second, the s_min(A_I) issue. If the chosen start basis has a tiny smallest singular value, the right-hand sides M and √d M²/(4κ) blow up and the whole construction degrades. This is why I picked the best of 3nd ln n random d-sets, and it's the one genuinely delicate probabilistic statement, so I want to be careful about why a batch helps. For a single Gaussian-perturbed d×d minor, the probability that s_min ≤ ε is O(√d·ε) — minors are well-conditioned, so any *one* random minor is usually fine. But I can't just take one minor and union-bound, because the deviations compound; what I want is that *not all* of a batch of random minors are simultaneously ill-conditioned, and here distinct minors share columns, so the events aren't independent and I can't multiply probabilities. My first instinct — treat the 3nd ln n minors as independent and raise the single-minor failure probability to that power — is exactly the move the shared columns forbid, so I have to argue about the population of all minors instead. What I can prove is: the probability that as many as (1 − 1/n)·binom(n,d) of *all* the d-minors of the perturbed matrix have s_min below the threshold κ₀ = σ·min(1,σ)/(12 d² n⁷ √ln n) is at most n^(−d) + n^(−n+d−1) + n^(−2.9d+1). The proof relates the count of ill-conditioned d-minors to a count over (d−1)-subsets, bounding how much the overlap between minors can be, and shows the bad configuration is rare despite the dependence. Outside that bad event, at least a 1/n fraction of all d-minors are good. *Now* a batch helps, and independence enters legitimately only here, in the random *choice* of which minors to sample: a batch of 3nd ln n random d-sets then misses every good minor with probability at most (1 − 1/n)^{3nd ln n} ≤ e^{−3d ln n} = n^(−3d). So P[s_min(A_{I(A)}) ≤ κ₀] ≤ 0.417·binom(n,d)^(−1): the start basis I keep is well-conditioned with overwhelming probability, the M and κ stay controlled, and as a bonus M and κ — being set to powers of two — take only polynomially many distinct values, which I use to keep the dependencies between the random variables from proliferating.

Putting the phases together: the expected number of pivots to solve LP′ is bounded by the shadow bound applied after the plane-fixing argument; the expected number to solve LP⁺ likewise; the random restarts and the α-randomization wash out into expectations; the bad events each contribute at most a constant. The conclusion is that the two-phase shadow-vertex method has

  E[ number of simplex steps ] ≤ min( P(d, n, 1/min(σ, σ₀)),  binom(n,d) + binom(n,d+1) + 2 )

for a fixed polynomial P and constant σ₀ — polynomial in d, n, and 1/σ, with the trivial worst-case cap as the other branch of the min. The simplex method has polynomial smoothed complexity. I'll be candid that the polynomial P I get is enormous — when I chase every constant it is something like d to the fifty-something times n to the eighty-something times σ to a large negative power — because at a dozen places I made loose worst-case assumptions that can't all bind at once, and I made no attempt to optimize exponents. The honest carrier of the intuition is not that monstrous polynomial; it is the clean shadow bound O(n d³/σ⁶), the one I traced exponent-by-exponent. That is the result that actually explains the phenomenon.

Let me say the causal chain in one breath, because that is what I want to remember. The simplex method is exponential in the worst case yet linear in practice; worst-case analysis is fooled because its max latches onto the Klee–Minty filament, a brittle measure-zero set; average-case analysis is fooled because its expectation drowns the real input under rotationally symmetric noise; so I define smoothed complexity as max-over-inputs of expected-running-time-under-a-small-Gaussian-perturbation — a dial I checked on a hard-sliver toy slides like erf(δ/(σ√2)) from worst-case at σ→0 to average-case at σ→∞, polynomial in 1/σ being the right demand because poly in log(1/σ) collapses back to bit-worst-case; I analyze the shadow-vertex rule because, alone among pivot rules I tried, its visited vertices are a projection — a shadow onto span(t,z) that on a worked 3-D instance the rotating objective really did walk along — not an iteration; the expected shadow size reduces, through a discretization whose limit I argued is clean, to the probability that the rotating objective is within angle ε of its optimal simplex's boundary, which has to be linear in ε for the discretization to cancel; that probability splits, via the Blaschke change of variables, into a distance factor (the facet's hull near the origin, linear with a d²/σ⁴ rate because perturbed simplices are fat) and an angle factor (the facet seen edge-on, quadratic with an n/σ² rate from the n side constraints), and a dyadic combination lemma — whose constant I summed to exactly 4, and whose square-on-G I confirmed is what stops the series diverging — multiplies a linear and a quadratic bound into a linear one, yielding E[shadow size] = O(n d³/σ⁶) with σ⁶ = σ⁴·σ² traced to the two factors; finally a two-phase construction with a well-conditioned random start (justified by the many-good-choices concentration, where independence enters only in the sampling) and a perturbation split into a plane-fixing first part and a shadow-bound-consuming second part turns the geometric bound into a polynomial bound on the algorithm's expected pivots — and the brittle exponential instances, being exactly the ill-conditioned configurations the perturbation makes improbable, evaporate under the expectation.

The final form I keep is this pair: the definition

  Smoothed complexity:  max_{x ∈ Xₙ}  E_g [ C_A( x + σ‖x‖ g ) ],   poly in n, 1/σ ⟹ "polynomial smoothed complexity,"

and the theorem that the two-phase shadow-vertex simplex method achieves it for linear programming, resting on the shadow-size bound E[|shadow|] ≤ 58,888,678 · n d³ / min(σ, 1/(3√(d ln n)))⁶.
