The goal is to learn, from raw images alone, a latent representation in which each coordinate captures a single interpretable generative factor such as position, scale, rotation, or lighting, while staying invariant to all the others. Real images are produced by a small set of conditionally independent factors of variation, so a representation that recovers them in separate axis-aligned coordinates would generalize across tasks, support zero-shot recombination of known factors, and enable transfer. The difficulty is that the learner receives no labels telling it how many factors exist or what they are, and any practical method must also provide a way to quantify disentanglement so that different models and hyperparameters can be compared.

Several existing approaches fall short for different reasons. PCA and ICA find uncorrelated or statistically independent components, but independence is not the same as interpretability: the recovered axes can be arbitrary rotations of the true generative factors, so the latents may be independent yet still entangled with respect to the factors we care about. InfoGAN discovers interpretable directions by maximizing mutual information between a subset of noise variables and the generated image, but it is built on a GAN, which brings training instability and reduced sample diversity, and it lacks a proper inference network that can encode arbitrary inputs. DC-IGN achieves clean disentanglement, but only by using semi-supervised batches in which exactly one factor is varied at a time, which requires knowing the factors in advance and collecting structured supervision. The standard variational autoencoder has the right ingredients: it is unsupervised, scalable, stable, and comes with an inference network by construction. However, with the default equal weighting of its reconstruction and KL terms, it learns entangled representations in which multiple factors are smeared across many latent coordinates. The question is therefore how to modify the VAE so that it applies enough pressure for individual latents to specialize to single independent factors.

The method is called β-VAE. It keeps the standard VAE architecture and inference network but reweights the objective by a single scalar β placed on the KL divergence term. The VAE objective is the evidence lower bound

E_{q_φ(z|x)}[log p_θ(x|z)] − D_KL(q_φ(z|x) ‖ p(z)).

When the prior p(z) is the isotropic unit Gaussian N(0, I), the KL term does two things at once: it limits the information capacity of the latent channel, and because the prior is factorised, it rewards independence across the latent coordinates. Both properties are exactly what disentanglement needs. The problem is that, with β=1, the reconstruction term is usually strong enough to defeat this pressure and keep the representation entangled. β-VAE formalizes the learning goal as maximizing reconstruction quality subject to a budget on how far the posterior may deviate from the factorised prior. Under the KKT conditions this constrained problem becomes the unconstrained objective

E_{q_φ(z|x)}[log p_θ(x|z)] − β · D_KL(q_φ(z|x) ‖ p(z)),

where β is the Lagrange multiplier on the capacity constraint. Setting β=1 recovers the ordinary VAE. Choosing β>1 tightens the bottleneck and strengthens the pressure toward a capacity-limited, factorised posterior. The model is then squeezed between the reconstruction term, which wants to explain every detail of the image, and the KL term, which wants the posterior to stay close to an independent, low-capacity prior. To satisfy both, the encoder must find the most efficient representation of the data, and because the data are generated by conditionally independent factors, the efficient code is a disentangled one: each latent coordinate captures one factor.

There is a tradeoff. Raising β too far forces the model to drop information that is expensive to encode, which typically appears first as blurrier reconstructions and eventually as the loss of factors that produce only small pixel changes. β is therefore a dial to be tuned rather than maximized, and its appropriate value depends on the dataset and architecture. To compare β across settings, the meaningful quantity is the normalized β_norm = β·M/N, where M is the latent dimension and N is the number of pixels. β itself cannot be learned from the data because it represents the exchange rate between reconstruction quality and latent capacity at the optimum of the constrained problem; it must be set from outside using the disentanglement metric or visual inspection of latent traversals.

Because independence alone is not sufficient, β-VAE is paired with a disentanglement metric that checks both independence and interpretability. The metric works by constructing pairs of images that share the value of one randomly chosen generative factor while differing randomly in all others. Each pair is passed through the encoder, and the absolute difference between their inferred latent means is averaged over a batch. In a disentangled representation, the coordinate corresponding to the shared factor will have low average difference, while the other coordinates will vary. A low-capacity linear classifier is then trained to predict which factor was held fixed from this averaged difference vector, and its accuracy is reported as the disentanglement score. The classifier is deliberately weak so that it cannot itself untangle a tangled representation; a high score therefore means the representation has already placed the relevant information in an axis-aligned, interpretable coordinate.

import torch
from torch import nn
import torch.nn.functional as F

def reparametrize(mu, logvar):
    std = logvar.div(2).exp()
    eps = torch.randn_like(std)
    return mu + std * eps

class BetaVAE(nn.Module):
    def __init__(self, z_dim=10, nc=3):
        super().__init__()
        self.z_dim = z_dim
        self.encoder = nn.Sequential(
            nn.Conv2d(nc, 32, 4, 2, 1), nn.ReLU(True),
            nn.Conv2d(32, 32, 4, 2, 1), nn.ReLU(True),
            nn.Conv2d(32, 64, 4, 2, 1), nn.ReLU(True),
            nn.Conv2d(64, 64, 4, 2, 1), nn.ReLU(True),
            nn.Conv2d(64, 256, 4, 1),  nn.ReLU(True),
            nn.Flatten(),
            nn.Linear(256, z_dim * 2),
        )
        self.decoder = nn.Sequential(
            nn.Linear(z_dim, 256), nn.Unflatten(1, (256, 1, 1)), nn.ReLU(True),
            nn.ConvTranspose2d(256, 64, 4), nn.ReLU(True),
            nn.ConvTranspose2d(64, 64, 4, 2, 1), nn.ReLU(True),
            nn.ConvTranspose2d(64, 32, 4, 2, 1), nn.ReLU(True),
            nn.ConvTranspose2d(32, 32, 4, 2, 1), nn.ReLU(True),
            nn.ConvTranspose2d(32, nc, 4, 2, 1),
        )

    def forward(self, x):
        dist = self.encoder(x)
        mu, logvar = dist[:, :self.z_dim], dist[:, self.z_dim:]
        z = reparametrize(mu, logvar)
        return self.decoder(z), mu, logvar

def reconstruction_loss(x, x_recon, distribution="bernoulli"):
    B = x.size(0)
    if distribution == "bernoulli":
        return F.binary_cross_entropy_with_logits(x_recon, x, reduction="sum") / B
    return F.mse_loss(x_recon, x, reduction="sum") / B

def kl_divergence(mu, logvar):
    klds = -0.5 * (1 + logvar - mu.pow(2) - logvar.exp())
    return klds.sum(1).mean(0)
