# Context

## Research question

Deep neural networks had just been shown to give reinforcement learning agents rich enough representations to act directly from raw pixels. But the recipe that made this work rested on a belief that had hardened into prevailing wisdom: combining a deep function approximator with *simple online* RL updates is fundamentally unstable, and the only known cure is an experience replay buffer. That cure carries three burdens. It forces the learning algorithm to be **off-policy**, because the buffer contains data generated by older policies and only an off-policy update can correctly learn from data not produced by the current policy — which locks out the entire on-policy family (Sarsa, actor-critic, plain policy gradient). It costs **memory and per-step compute**, because a million transitions must be stored and a minibatch replayed at every real interaction. And it pushes practitioners toward **specialized or massively distributed hardware** — a GPU, or a cluster of a hundred-plus machines each with its own replay memory and a central parameter server.

The precise problem: find a way to decorrelate and stabilize the stream of updates fed to a deep RL agent **without a replay buffer**, so that (1) the natural on-policy methods become usable with deep nets, (2) the memory and per-step replay cost disappears, and (3) the whole thing runs on a single commodity multi-core machine rather than a GPU or a cluster. A solution would have to reproduce, by some other mechanism, exactly the two things replay was doing: breaking the temporal correlation between consecutive updates, and averaging the data distribution into something stationary enough for stochastic gradient descent to behave.

## Background

**Why online deep RL was thought unstable.** An agent following a trajectory sees states that change slowly: s_t and s_{t+1} are nearly identical, so the gradients they produce point in nearly the same direction for long stretches. Stochastic gradient descent presumes roughly independent samples; a long run of correlated gradients makes the optimizer drift, overfit to the current region of state space, and behave with high variance. The problem compounds when the target is bootstrapped: a temporal-difference target r + γ max_{a'} Q(s', a'; θ) depends on the *same* parameters being updated, and the data-generating policy itself shifts as θ shifts. So the learner chases a moving target over a moving data distribution. The combination of function approximation, bootstrapping, and off-policy updates is the classic recipe for divergence.

**What replay does about it.** Storing transitions (s, a, r, s') in a large buffer and sampling minibatches uniformly at random accomplishes three things at once: it mixes transitions from many different time steps so consecutive gradients are no longer correlated; it averages over many past policies so the effective data distribution is more stationary; and it reuses each transition many times, improving data efficiency. A separate, slowly-updated **target network** θ⁻, frozen for a fixed number of steps, further stabilizes the bootstrap by holding the target still while the online network moves.

**The structural cost.** Uniform replay is only correct for off-policy algorithms: the stored data was generated by stale policies, so the update rule must be valid regardless of which policy produced the data. Q-learning qualifies — it estimates the optimal action-value function irrespective of the behavior policy. On-policy methods do not: a policy-gradient or Sarsa update is only an unbiased signal for data drawn from the *current* policy, so it cannot be fed stale buffer samples without becoming wrong. This is why the replay-based successes were all value-based and off-policy, and why the policy-gradient / actor-critic family — arguably the most natural way to optimize a parameterized policy — sat outside the "stable deep RL" club.

**Observations about existing parallel/asynchronous RL.** Asynchronous and distributed RL already existed as a way to *scale*. A massively distributed architecture ran ~100 actor processes each with its own environment copy and replay memory plus ~30 parameter-server instances (130 machines total), shipping DQN-loss gradients asynchronously to a central model and pulling updated parameters back at intervals; it reached DQN-level scores far faster on many games. Earlier work parallelized batch RL with linear function approximation purely to speed up matrix operations, and a parallel Sarsa used peer-to-peer sharing of significantly-changed weights between separate learners. Convergence theory for the asynchronous case was known: Q-learning still converges when some information is outdated, provided stale information is eventually discarded and a few technical conditions hold. And lock-free parallel stochastic gradient descent on shared memory — multiple threads writing the shared parameter vector with no locks — was shown to converge at near-optimal rates when updates are sparse or steps are small, because the resulting races act like bounded noise rather than corruption. Evolutionary methods, trivially parallel by distributing fitness evaluations, had also been applied to vision-based control. The common thread: parallelism had been used to *go faster*, but not as the mechanism that *stabilizes* an online deep RL learner in place of replay.

**Supporting concepts for the problem.** The discounted return R_t = Σ_{k≥0} γ^k r_{t+k}. The action-value Q^π(s,a) = E[R_t | s_t=s, a_t=a] and state-value V^π(s) = E[R_t | s_t=s]. Bootstrapped n-step returns r_t + γ r_{t+1} + … + γ^{n-1} r_{t+n-1} + γ^n (value of s_{t+n}), which let a single observed reward update n preceding states in one shot instead of propagating it one slow step at a time. The score-function / policy-gradient identity ∇_θ E[R_t] = E[∇_θ log π(a_t|s_t;θ) R_t], with the variance-reduction trick of subtracting a state-only baseline b(s_t) that leaves the gradient unbiased; using a learned value function as that baseline turns the score multiplier into an estimate of the advantage Q^π(s,a) − V^π(s) — the actor-critic view, policy as actor, value as critic. Adaptive per-parameter step sizes via RMSProp- or Adam-style moving averages. And policy entropy H(π) as a quantity one can add to an objective to keep a softmax policy from saturating.

## Baselines

**Q-learning with a deep network and experience replay (DQN).** Represent the action-value function with a CNN, Q(s,a;θ). Learn by minimizing the squared TD error against a target network: y = r for a terminal s' and y = r + γ max_{a'} Q(s',a';θ⁻) otherwise, with loss E[(y − Q(s,a;θ))²], transitions drawn uniformly from a large replay buffer, and θ⁻ a periodically-copied frozen target. Core idea: replay decorrelates and stationarizes; the target network steadies the bootstrap. Gap it leaves: it is intrinsically off-policy and replay-bound, so on-policy methods cannot be plugged in; it needs a large memory and extra compute per step; and reaching strong results practically requires a GPU.

**Massively distributed asynchronous DQN (Gorila).** Replicate the DQN learner across many machines: each process has an actor in its own environment copy, its own replay memory, and a learner that samples from that memory and computes DQN-loss gradients; gradients go asynchronously to a central parameter server that updates a master copy, which is periodically pushed back to the actors. Core idea: scale DQN across ~130 machines for large speedups. Gap: it still keeps replay (hence still off-policy and memory-heavy) and depends on a substantial distributed cluster with parameter-server communication overhead.

**One-step Q-learning (value-based, off-policy).** Update Q(s,a;θ) toward r for a terminal next state and toward r + γ max_{a'} Q(s',a';θ) otherwise. Core idea: directly approximate Q*. Gap: a reward only directly affects the single state-action pair that led to it; other pairs are touched only indirectly through bootstrapping, so credit propagates slowly — many updates are needed to move a reward back to the actions responsible for it.

**One-step Sarsa (value-based, on-policy).** Same as one-step Q-learning but, when the next state is non-terminal, the target is r + γ Q(s',a';θ) where a' is the action actually taken next, so it evaluates the behavior policy rather than the greedy one; at a terminal next state the target is just r. Same slow one-step credit-assignment gap, and being on-policy it had no clean way to coexist with a replay buffer.

**n-step / multi-step Q-learning.** Update toward an n-step return r_t + γ r_{t+1} + … + γ^{n-1} r_{t+n-1} + γ^n max_a Q(s_{t+n},a), with the bootstrap term replaced by 0 when the rollout ends in a terminal state. Core idea: a single reward directly affects n preceding state-action pairs, so credit propagates much faster, and bootstrapping after several real rewards trades the high bias of one-step targets against the high variance of full Monte-Carlo returns. Multi-step methods were classically implemented in the *backward view* via eligibility traces. Gap left open: how to use them stably with a deep, momentum-trained, possibly recurrent network — and, like the other on-policy variants, how to stabilize them at all without replay.

**REINFORCE and actor-critic (policy-based, on-policy).** Parameterize the policy π(a|s;θ) directly and ascend E[R_t] via ∇_θ log π(a_t|s_t;θ) R_t; subtract a learned value baseline V(s_t) to get the lower-variance ∇_θ log π(a_t|s_t;θ)(R_t − V(s_t)), an advantage actor-critic update. Core idea: optimize the policy itself, with the critic supplying a variance-reducing baseline. Gap: high-variance gradients, and — decisively for the deep setting — being strictly on-policy, it was excluded from the replay-based stabilization that everything deep relied on, so it had no demonstrated stable deep-network recipe.

## Evaluation settings

The natural yardstick is the Arcade Learning Environment (Atari 2600) over the standard suite of games from raw frames, with the established DQN preprocessing (grayscale, downscaling to 84×84, stacking the last few frames) and an action repeat of 4. The standard network for pixel control is the DQN CNN — a convolutional stack followed by a fully connected layer; for a policy-and-value agent the body is shared with a softmax policy head and a single linear value head. Beyond discrete-action Atari, the relevant testbeds are continuous-control physics tasks in a multi-joint dynamics simulator (cart-pole, pendulum, reacher, cheetah, hopper, walker, gripper, and similar, from low-dimensional state and from pixels), the TORCS 3D car-racing simulator, and a first-person 3D maze navigation task from visual input. Metric: average episode score / return, reported against training time in wall-clock hours and against data consumed in environment frames. The experimental protocol of interest is robustness — sweeping many random learning rates (sampled log-uniformly) and random initializations and looking at the distribution of final scores — and scaling behavior as the number of parallel actor-learners grows. Standard discounting γ = 0.99.
