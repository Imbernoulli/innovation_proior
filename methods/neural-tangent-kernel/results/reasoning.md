I first need to stop asking for a good coordinate system in parameter space. The weights are the wrong coordinates for the loss: many different parameters can represent the same function, and even a convex cost on predictions becomes non-convex after composition with the network map. The object I really care about is the current function `f_theta`, not the particular point `theta` that realizes it.

If I perturb the parameters a little, the function changes through the differential of the realization map. That means every parameter direction gives me a feature function, `partial_{theta_p} f_theta`. I do not have to decide yet whether this is useful globally. Locally, these tangent features are exactly what parameter gradient descent can use. When I write the chain rule for the time derivative of `f_theta(x)`, the parameter gradients pair with each other, and the sum over parameters forms a Gram kernel: `Theta(x,x';theta) = sum_p partial_{theta_p} f_theta(x) partial_{theta_p} f_theta(x')`. So the prediction function is already following a kernel gradient flow, except the kernel is attached to the current parameters and can keep changing.

That is the first wall. A moving kernel is not much simpler than the original nonlinear training problem. I need a reason for this tangent Gram matrix to stabilize. The old infinite-width Gaussian-process story gives me only the distribution of the function at initialization. It says that preactivations become jointly Gaussian and that a covariance recursion exists. It does not tell me why the Jacobian features should remain aligned after gradient descent starts.

The scaling is the clue. If I put the factor `1/sqrt(n_l)` inside each affine map while initializing the weights as standard Gaussians, the forward signal has a nontrivial limit, but the derivative of the output with respect to any one connection also becomes small. One weight cannot change a hidden representation much when a layer is very wide. At first that sounds like no learning can happen in the lower layers, which would make the result trivial and disappointing. But the network has many such tiny directions. A single hidden preactivation moves by order `1/sqrt(n)`, while the sum of `n` tangent-feature contributions can still be order one.

I check this against the induction in the proof. At initialization, the last layer lets me split the tangent kernel into two parts. The direct last-layer contribution averages products of activations, so it converges to the next covariance `Sigma`. The lower-layer contribution backpropagates through `sigma'` and the last-layer weights, so it carries the previous tangent kernel multiplied by the derivative covariance `dotSigma`. That gives the recursion `Theta_{l+1} = Theta_l dotSigma_{l+1} + Sigma_{l+1}`. This is not just the GP covariance again; it remembers how learning in lower layers changes the output.

Now I need the same object not to drift during training. The proof's bounded-training-direction assumption gives control over the integral of the force applied to the network. With the `1/sqrt(n)` derivative factors, each hidden preactivation and each suitably normalized weight matrix moves by a vanishing amount on finite time intervals. A Gronwall bound keeps this small motion from accumulating into a large representation change. Then every summand in the tangent kernel changes only a little, and the average over many neurons keeps converging to the same deterministic limit. The individual features are almost frozen, but their collective tangent space still drives a real change in the output.

This resolves the apparent contradiction. The network is not learning by moving far through representation space in this limit. It is learning by taking an order-one step inside the tangent space present at initialization. The proof gives the mechanism directly: the width scaling makes the tangent kernel finite, deterministic, and stable.

Once the kernel is fixed, the hard part collapses. Gradient descent on the parameters becomes kernel gradient descent on the function. For squared loss, the function follows a linear differential equation, so the error decomposes along the principal components of the kernel operator on the data. Large eigenvalue directions decay first; small eigenvalue directions decay slowly. The limiting predictor at convergence has the mean of ridgeless kernel regression with this tangent kernel, with a Gaussian residual term inherited from initialization that vanishes on the training points.

I also see why this is more than applying kernels to neural networks. Earlier work could assign a kernel to an infinitely wide random architecture or use random features as an approximation to a chosen kernel. Here the kernel is the Jacobian Gram matrix of the trainable network itself. The proof does not merely say that a neural network resembles some kernel machine before learning. It says that, in the infinite-width and scaled-gradient regime, the actual training trajectory is generated by an explicit limiting tangent kernel.

The positive-definiteness result supplies the final convergence bridge. If the limiting tangent kernel is positive definite on the data, kernel gradient descent on a convex functional cannot stall away from a global minimum. On the sphere, non-polynomial activations give enough Hermite coefficients, and the Schoenberg-type criterion turns that into strict positive-definiteness. So the method gives not only a descriptive limit but also a reason wide networks can converge under gradient descent in this regime.
