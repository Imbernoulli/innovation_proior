Let me stare at the one part of the Transformer block I keep treating as fixed furniture: the feed-forward sublayer. Between attention layers I run each position's hidden vector x through `f(xW1)W2` — project up to a wide hidden dimension, apply a single pointwise nonlinearity, project back down. That is the whole thing. And it is not a small corner of the model: with the usual 4× expansion the two matrices W1 and W2 hold something like two-thirds of the non-embedding parameters and a big share of the FLOPs. So if I want to spend a fixed budget better, this is the obvious place to push, because it is where most of the budget already sits. Attention mixes across positions; this sublayer is the entire per-position transformation, the only place a token's representation gets reshaped on its own. And the only nonlinearity in it is that one f. Everything the layer expresses about a token, it expresses by taking *one* linear view xW1 and bending it pointwise.

What is f today? Originally ReLU, `max(0, xW1)`. The way I want to read ReLU now is as a hard, sign-based gate: `ReLU(z) = z · 1[z>0]` — multiply z by a 0/1 mask that depends only on whether z is positive. Crisp, cheap, gradient of exactly 1 on the positive side, which is why it flows so well and became the default. But the all-or-nothing decision throws away magnitude and kills the entire negative half. The smoother replacements on hand soften that gate. GELU is `z·Φ(z)`, Φ the standard normal CDF — derived as the expectation of a stochastic 0/1 mask `m ~ Bernoulli(Φ(z))`, whose mean transform is `Φ(z)·z + (1−Φ(z))·0 = zΦ(z)`; the hard ReLU corner becomes a smooth slope continuous through the origin. And Swish is `z·σ(βz)`, the same value-times-gate shape but with a logistic gate; at β=1 it is `z·σ(z)`, which is exactly the sigmoid-CDF cousin of GELU — the same construction that, with the logistic CDF in place of the Gaussian, people also call the SiLU.

So pause on that pattern, because it is staring at me. Every activation I am choosing between for this layer has the *same internal anatomy*. ReLU is `z · 1[z>0]`. GELU is `z · Φ(z)`. Swish is `z · σ(βz)`. Each one is the raw preactivation z multiplied by some gate that is a function of z. The activation is *already* a gating operation — it takes a value (z itself) and multiplies it by a gate (a function of z) deciding how much of the value to let through. And this is not my coincidence: the automated activation search that turned up Swish reported exactly this as its central finding. When it searched over compositions of primitives for good scalar activations, the winners overwhelmingly had the form `b(x, g(x))` — the *raw* preactivation x reused, recombined with a gate g(x) of itself. Even ReLU fits, with `b = max` and `g(x) = 0`. The structure that keeps winning, across an exhaustive-plus-RL search, is value-recombined-with-gate-of-itself.

Now here is what bothers me once I see the layer that way. In `f(xW1)W2`, the value and the gate come from the *same* projection. GELU's value is z = xW1 and its gate is Φ(z) = Φ(xW1); both are functions of the single linear map xW1. Swish's value is xW1 and its gate is σ(xW1) — same projection again. There is exactly one learned linear view of x feeding this whole sublayer, and the gate is forced to be a fixed function of the very thing it gates. The layer never gets to ask one question of x to decide *how much* and a different question of x to decide *what*. The gating is real, but it cannot be independent, because there is only one projection to build it from.

What would it take to make the gate its own thing? I want the gate to be a separately-learned linear function of x, not a fixed function of the value. So instead of one up-projection xW1, give the layer two: a value projection xV and a gate projection xW. Form the hidden vector as their elementwise product, with a nonlinearity sitting on the gate path,

  hidden = f(xW) ⊗ (xV),

and project back down with W2. Now the layer takes two independent linear views of the same input — one decides, through f, how much each hidden unit fires; the other carries the content to be modulated. Amount and content are no longer tied to one projection.

I should check this is not a gadget I am inventing from nothing, because I have seen this exact shape before, just not inside a Transformer FFN. Dauphin, Fan, Auli and Grangier's gated convolutional language model is built on precisely this unit: `(X∗W + b) ⊗ σ(X∗V + c)` — a content projection times a sigmoid gate, one elementwise product of two linear maps of the input. They called it a gated linear unit, and the point of the architecture was to put gating on the path so the network can select which features matter for predicting the next word, without an LSTM's recurrence. So the move I am reaching for — split the FFN's single projection into a gate and a value and multiply — is the gated linear unit, lifted out of the convolutional sequence model and dropped into the position-wise FFN in place of `f(xW1)`.

And there is an older thread saying even the un-gated version of this is meaningful. Drop the activation entirely and take `(xW) ⊗ (xV)`: that is a pure bilinear interaction, a product of two learned linear maps of the same input, exactly the kind of multiplicative coupling Mnih and Hinton used in their log-bilinear language model to combine distributed word representations. A product of two linear views of x can represent pairwise, multiplicative feature interactions that no single-projection-then-pointwise map can — each hidden unit becomes a degree-two function of x rather than a degree-one preactivation bent by a fixed curve. So even before adding any nonlinearity, going from one projection to a product of two is a genuine increase in what the layer expresses per unit, not a rearrangement.

But should I trust *multiplication* as the recombination, or is there something better? The activation search gives me a sharp warning I want to take seriously: functions that used *division* performed poorly, because the output blows up when the denominator approaches zero — division only behaved when its denominator was bounded away from zero. A learned linear gate xV is not bounded away from zero; it crosses zero all the time. So coupling two linear views by dividing one by the other would be a numerical trap. Multiplication has no such pathology: the product of two finite linear maps is always finite. That is a quiet but real reason the elementwise *product* is the right way to make this layer data-dependent on a second projection — not just because it is the GLU shape, but because the alternative coupling the search rejected is exactly the one that detonates.

Why should I believe the *gated* version trains well and not merely expresses more? Here the gated-conv work gives me the argument I actually trust, and it is about gradients, which is what I always worry about once I stack many of these layers. Suppose I had used the LSTM-style gate, content through a tanh and gate through a sigmoid — the gated tanh unit `tanh(X) ⊗ σ(X)`. Differentiate it:

  ∇[tanh(X) ⊗ σ(X)] = tanh'(X)∇X ⊗ σ(X) + σ'(X)∇X ⊗ tanh(X).

Look at the two paths: the upstream gradient ∇X is multiplied, on both terms, by a saturating activation *derivative* — tanh', in [0,1], or σ', in [0, 1/4]. There is no derivative-free content path through this unit; once either nonlinearity saturates, the gradient has to pass through a small derivative factor, and stacking a dozen of these attenuates the signal a dozen times over. Now contrast the linear-gated unit `X ⊗ σ(X)`, content X carried *linearly*, only the gate through σ:

  ∇[X ⊗ σ(X)] = ∇X ⊗ σ(X) + X ⊗ σ'(X)∇X.

The first term is ∇X ⊗ σ(X): the upstream gradient multiplied only by the gate *value* σ(X) ∈ (0,1), not by any activation derivative. For the units the gate has opened — σ(X) near 1 — the gradient passes essentially undiminished, a clean almost-linear highway back through the layer, a multiplicative skip connection. That is the structural reason to keep the *content* path linear and put the nonlinearity only on the *gate*: it buys the expressivity of two-view gating without paying the per-layer gradient-downscaling tax the both-paths-nonlinear unit pays. So the design is not "two projections, nonlinearity anywhere" — it is specifically value carried linearly, gate carried through f, product taken: `f(xW) ⊗ (xV)`, with V's path linear.

Now the question that actually decides which variant I land on: which f for the gate? The gated-conv unit used a plain sigmoid σ. That works, but let me think about what σ as a gate can and cannot do. σ(z) lives strictly in (0,1). As a gate it can only ever *attenuate* — scale the content down toward zero — never pass it at more than unit strength and never flip it. The whole modulation is a soft on/off in [0,1]. That feels like leaving capability on the table, because the content path is already linear and unbounded; a gate that can only shrink it can implement "ignore this feature here" but not "amplify this feature here" or "invert it." I would like a gate that is a richer modulation than pure squashing.

Where do I have a gate function that is not bounded into (0,1)? The activations I was already considering at this layer. Reading them back through their own anatomy `z·gate(z)`: ReLU's gate factor is the hard 1[z>0]; GELU's is the smooth Φ(z); Swish's is z·σ(βz) itself, which as a *function value* (not just the σ inside it) is unbounded above and dips slightly negative. Let me think hard about Swish as the gate specifically, because it has properties the bare sigmoid lacks that line up exactly with what I want from a gate. Swish₁(z) = z·σ(z): it is smooth, it is unbounded above (it grows like z for large positive z, so as a gate it can pass content through at *greater than unit gain* when the gate preactivation is large and positive — the amplification σ cannot do), and it is non-monotonic, with a small bump dipping below zero for moderately negative z, so as a gate it can softly *suppress and even sign-flip* the content on that negative bump rather than only squashing it toward zero. That is strictly richer modulation than σ's (0,1) on/off: amplify, pass, soften, flip. And it is the gate that comes from the activation family the search kept rediscovering, where the winners are value-recombined-with-a-gate-of-themselves — except now I am putting that gate on its *own* projection rather than tying it to the value's.

I want to be honest with myself about one thing that could have argued *against* Swish: ReLU's famous virtue is its exact-1 derivative on the positive side, so gradients flow without downscaling. Doesn't a smooth gate like Swish give that up? Look at Swish's derivative,

  Swish_β'(z) = σ(βz) + βz·σ(βz)(1 − σ(βz)) = β·Swish_β(z) + σ(βz)(1 − β·Swish_β(z)),

which at β=1 has magnitude below 1 for inputs below roughly 1.25 — including the whole negative side and the central positive region — so yes, the gate path itself does not have ReLU's clean unit derivative everywhere. But that worry evaporates in *this* construction, and that is the whole point of keeping the content path linear. The gradient highway I just derived does not run through the gate's derivative at all; it runs through the gate *value* multiplying the linear content's gradient. So I get the clean flow from the linear value path regardless of what the gate's derivative does, and I am free to choose the gate purely for the richness of its modulation — which is exactly where Swish, unbounded above and non-monotonic, beats a sigmoid. The very property that made me hesitate about Swish as a standalone activation — its sub-unit derivative — stops mattering once the nonlinearity is confined to the gate and the value is carried linearly.

So the gate function is Swish₁ = z·σ(z), and the layer is `(Swish₁(xW) ⊗ xV) W2`. The same template instantiates the whole family — put σ on the gate and it is the original GLU, ReLU and it is a hard-gated variant, GELU and it is the Gaussian-CDF gate, nothing-at-all and it is the bilinear product — but Swish₁ on the gate is the one I have argued myself into: a gate that can amplify and sign-flip, from the activation family the search says wins, with the gradient-flow concern handled by the linear value path rather than by the gate's own derivative. Three weight matrices now: the gate projection W, the value projection V, the down-projection W2 — where the old FFN had two.

And that third matrix is a problem I have to confront immediately, because the entire premise of the comparison is matched budget. If I just bolt a third matrix on, of course it might do better — it has more parameters and does more compute, and any quality gain would be confounded with simply spending more. I have to hold parameters and FLOPs fixed against the baseline FFN, which means the extra projection must be paid for by shrinking the hidden width.

Let me do the bookkeeping. Write d for d_model and let the baseline FFN use hidden width d_ff. The baseline has W1 of shape d×d_ff and W2 of shape d_ff×d, so its parameter count is 2·d·d_ff (ignore biases, which the bias-free FFN convention drops anyway). My gated layer has W and V each of shape d×d_ff′ and W2 of shape d_ff′×d, for 3·d·d_ff′ parameters, where d_ff′ is the width I am allowed. Match them:

  3·d·d_ff′ = 2·d·d_ff   ⇒   d_ff′ = (2/3)·d_ff.

So the gated FFN must reduce its hidden dimension to two-thirds of the baseline's. The FLOP count scales the same way — each of the three matmuls is d×d_ff′ work per token, three of them is 3·d·d_ff′ = 2·d·d_ff, the same as the baseline's two d×d_ff matmuls — so matching parameters matches compute here too. Concretely, if the baseline uses the standard 4× expansion, d_ff = 4d, then

  d_ff′ = (2/3)·4d = (8/3)·d ≈ 2.667·d.

That is the number: where the plain FFN went up to 4d, this gated FFN goes up to (8/3)d, and then it has three matrices instead of two, and the two layouts cost exactly the same before any hardware rounding. Sanity-check the parameter counts directly: baseline 2·d·4d = 8d²; gated 3·d·(8/3)d = 8d². Equal, good. At a width where d_ff = 3072 that gives d_ff′ = 2048 exactly. If a host implementation rounds the hidden size to a tile multiple, that rounding is an explicit engineering nudge and the budget delta should be stated; the formula itself is the exact 2/3 width. I omit biases on all three matrices, both to keep the parameter accounting clean and because the FFN baseline I am matching already drops them.

Let me write the forward pass concretely to make sure the shapes click. Input x is (batch, length, d). The gate projection takes x to (batch, length, d_ff′) and I apply Swish₁ to it. The value projection takes x to (batch, length, d_ff′) with no activation. Multiply elementwise — same shape, (batch, length, d_ff′). The reference feed-forward layer applies dropout to that hidden product when dropout is enabled, then the down-projection takes it back to (batch, length, d). Same `(batch, length, d)` in, same out: a true drop-in for the FFN, touching nothing in attention, normalization, the data, or the optimizer. Swish₁ is just `x·σ(x)`, which is the SiLU primitive every framework already ships, so the gate is a one-call function with no new parameters.

Do I have an honest mechanistic *proof* that this lowers loss? No — and I should not invent one. What I have are converging design-time reasons to expect it to help at matched budget. It forms each hidden unit from a product of two independent linear views of x, so the layer can express multiplicative feature interactions a single-projection FFN cannot, and multiplication (unlike division, which the search showed blows up) is the safe way to do that coupling. It keeps the content path linear so the gradient gets a clean, almost-undiminished highway back through every layer, the way the linear-gated unit does and the both-paths-nonlinear unit does not. And it puts Swish₁ on the gate — a gate from the value-times-gate-of-itself family the search kept rediscovering, but now richer than a sigmoid because it can amplify and softly sign-flip rather than only attenuate, with its sub-unit derivative made irrelevant by the linear value path. Whether those add up to a real improvement is something I would validate on held-out language-modeling perplexity against the strictly matched-budget baseline FFN. The reasoning gets me to the architecture; the architecture is simple, exactly budget-matched before any declared implementation rounding, and changes nothing else — which is exactly what makes it worth trying.

So here is the layer I would actually write, filling the open slot in the FFN sublayer — the gated form at two-thirds width, Swish₁ on the gate, content carried linearly, projected back down:

```python
import torch
import torch.nn as nn
import torch.nn.functional as F


class MLP(nn.Module):
    """Gated feed-forward sublayer: Swish_1(xW) ⊗ (xV), projected back down.

    Two independent up-projections of x — a Swish-gated path and a linear value
    path — are multiplied elementwise, then projected down. The hidden width is
    set to 8/3 * d so the three matrices cost the same parameters and FLOPs as the
    baseline two-matrix FFN at 4x expansion. (B, T, n_embd) in, same shape out.
    """

    def __init__(self, config):
        super().__init__()
        d = config.n_embd
        base_hidden = getattr(config, "ffn_hidden", 4 * d)
        if (2 * base_hidden) % 3 != 0:
            raise ValueError("Exact budget match requires 2 * ffn_hidden divisible by 3.")
        hidden_dim = (2 * base_hidden) // 3
        self.w1 = nn.Linear(d, hidden_dim, bias=False)      # gate projection W
        self.w3 = nn.Linear(d, hidden_dim, bias=False)      # value projection V
        self.c_proj = nn.Linear(hidden_dim, d, bias=False)  # down-projection W2
        self.dropout = nn.Dropout(config.dropout)

    def forward(self, x):                                  # x: (B, T, n_embd)
        # Swish_1 == SiLU == x*sigmoid(x) on the gate, value path xW3 kept linear,
        # so the value's gradient gets a clean highway (gate value, not derivative).
        h = F.silu(self.w1(x)) * self.w3(x)
        return self.c_proj(self.dropout(h))
```

To recap the causal chain I have built: the FFN holds most of the model's budget yet forms each hidden unit from a single linear view of x bent by one fixed pointwise function — and every such function, ReLU, GELU, Swish, is itself already value-times-gate-of-itself, with the value and gate tied to the same projection. Untying them — one projection for the gate, one for the value, multiplied elementwise — is the gated linear unit, recognized from gated convolutional language models and, in its un-activated bilinear form, from log-bilinear models; it lets each unit express a multiplicative interaction of two independent linear views of x, and multiplication is the coupling that does not detonate the way the search showed division does. Keeping the value path linear and putting the nonlinearity only on the gate gives the gradient a clean multiplicative-skip highway through every layer, unlike the both-paths-nonlinear unit whose gradient is downscaled by an activation derivative on every path. Putting Swish₁ on that gate — the SiLU `x·σ(x)`, from the family the activation search kept rediscovering — gives a richer-than-sigmoid modulation that can amplify and softly sign-flip rather than only attenuate, and the sub-unit derivative that made Swish unremarkable as a standalone activation no longer matters, because the linear value path, not the gate's derivative, carries the gradient. Because three matrices replace two, I shrink the hidden width to (2/3)·d_ff = (8/3)·d and omit biases so parameters and FLOPs match the baseline FFN exactly before any explicitly declared implementation rounding — so the only thing that changed is the *form* of the per-position transformation, which is exactly the variable I wanted to isolate.
