Let me start from the thing that actually hurts. I have a Transformer and I want to run it on long sequences вҖ” protein chains of eight thousand residues, images flattened into twelve thousand pixels, whole books. And it falls over, not because the model is bad but because of one operation. Attention takes queries, keys, and values `Q, K, V вҲҲ R^{LГ—d}`, forms the score matrix `A = exp(QKбөҖ/вҲҡd)`, normalizes each row by `D = diag(A 1_L)`, and returns `DвҒ»В№ A V`. The cost is entirely in `A`: it is `L Г— L`, so I spend `O(LВІd)` time to build it and `O(LВІ)` memory to hold it. For `L` in the thousands that quadratic term is the whole story. Double the sequence, quadruple the bill. I want this linear in `L`.

So where does the `LВІ` actually come from? Not from anything fundamental about what attention *means* вҖ” it comes from the *order of operations*. I build the full matrix `A` and only then multiply by `V`. If I never had to instantiate `A`, the `LВІ` object would never exist. Let me stare at one row of the unnormalized output, `(A V)_i = ОЈ_j A(i,j) v_j = ОЈ_j exp(q_iбөҖk_j) v_j`. The reason I can't factor anything out is that `exp(q_iбөҖk_j)` is a function of `q_i` and `k_j` *jointly* вҖ” it's not a product of something-in-`i` times something-in-`j`. If instead the score were a plain dot product of per-token vectors, `A(i,j) = ПҶ(q_i)бөҖПҶ(k_j)` for some map `ПҶ : R^d вҶ’ R^r`, then watch what happens:

    (A V)_i = ОЈ_j (ПҶ(q_i)бөҖПҶ(k_j)) v_j = ПҶ(q_i)бөҖ (ОЈ_j ПҶ(k_j) v_jбөҖ).

The inner sum `ОЈ_j ПҶ(k_j) v_jбөҖ` is an `r Г— d` matrix that *does not depend on `i`* вҖ” I compute it once, in `O(Lrd)`, and then every query just multiplies into it, another `O(Lrd)`. The `L Г— L` matrix is gone; I never form it. The same trick handles the normalizer: `D(i) = ОЈ_j A(i,j) = ПҶ(q_i)бөҖ(ОЈ_j ПҶ(k_j))`. In matrix form, if `Q'` and `K'` stack the feature rows `ПҶ(q_i)бөҖ` and `ПҶ(k_j)бөҖ`, the whole thing is

    AttМӮ = DМӮвҒ»В№ (Q'((K')бөҖ V)),    DМӮ = diag(Q'((K')бөҖ 1_L)),

and the parentheses are the entire point: `(K')бөҖV` first (an `r Г— d` matrix), then `Q'` times that. Time `O(Lrd)`, memory `O(Lr + Ld + rd)`. Linear in `L`. So the question is not "how do I speed up attention" вҖ” it's "can I write `exp(q_iбөҖk_j)` as a dot product `ПҶ(q_i)бөҖПҶ(k_j)`?" If I can, I'm done with the complexity problem in one line.

Can I? `exp(xбөҖy)` is not literally a finite dot product вҖ” it's a positive-definite kernel, and a kernel is an inner product in some feature space, but that space is infinite-dimensional. The Gaussian kernel `exp(-вҖ–x-yвҖ–ВІ/2)` has the same issue. But there's a standard escape hatch: don't represent the kernel exactly, represent it *in expectation*. Rahimi and Recht's random features: a shift-invariant kernel is the Fourier transform of a probability density, so `K(x,y) = E_{ПүвҲјp}[О¶_Пү(x) \overline{О¶_Пү(y)}]`. Draw `m` random frequencies `Пү_1,вҖҰ,Пү_m`, form a finite feature map, and the dot product of those maps is an *unbiased* estimate of the kernel with variance falling like `1/m`. That's exactly the form I want вҖ” a dot product `ПҶ(x)бөҖПҶ(y)` whose *mean* is the kernel. If I plug a random-feature `ПҶ` into the reassociation above, I get an unbiased estimate of the unnormalized kernel matrix entries and a cheap normalized attention ratio. I should be careful: the ratio `DМӮвҒ»В№ГӮV` is not, in general, an unbiased estimator of `DвҒ»В№AV`; the clean unbiasedness I get is for the kernel/attention matrix entries before row renormalization, and the normalized output I have to control through approximation quality and stability instead.

Let me make the kernel connection concrete first. My score, dropping the `вҲҡd` by folding it into `q,k`, is the softmax kernel `SM(x,y) = exp(xбөҖy)`. Pull the norms out:

    exp(xбөҖy) = exp(-вҖ–xвҖ–ВІ/2) В· exp(вҖ–x+yвҖ–ВІ/2) В· exp(-вҖ–yвҖ–ВІ/2),

since `вҖ–x+yвҖ–ВІ/2 = (вҖ–xвҖ–ВІ + 2xбөҖy + вҖ–yвҖ–ВІ)/2`, so the middle factor carries `exp(xбөҖy)` times `exp(вҖ–xвҖ–ВІ/2)exp(вҖ–yвҖ–ВІ/2)`, which the outer two factors cancel. Equivalently `SM(x,y) = exp(вҖ–xвҖ–ВІ/2) K_gauss(x,y) exp(вҖ–yвҖ–ВІ/2)` where `K_gauss` is the Gaussian kernel. So the softmax kernel is the Gaussian kernel reweighted by per-vector factors, and the Gaussian kernel is shift-invariant вҖ” random features apply directly.

The textbook random feature for the Gaussian kernel is trigonometric. The general template is

    ПҶ(x) = (h(x)/вҲҡm)(f_1(Пү_1бөҖx),вҖҰ,f_1(Пү_mбөҖx),вҖҰ,f_l(Пү_1бөҖx),вҖҰ,f_l(Пү_mбөҖx)),

with `Пү_i вҲј N(0,I_d)`. Taking `l=2`, `f_1=sin`, `f_2=cos`, `h=1` gives the Gaussian kernel because `E_Пү[cos(ПүбөҖ(x-y))] = exp(-вҖ–x-yвҖ–ВІ/2)` (the real part of the characteristic function of a standard Gaussian). To get the softmax kernel I just absorb the norm reweighting into `h`: set `h(x) = exp(вҖ–xвҖ–ВІ/2)`, keep `sin/cos`, and I have an unbiased estimator of `SM(x,y)`. Call it `SMМӮ_trig`. Drop it into the reassociation and I have a linear-time estimator whose score matrix entries are correct in expectation, before the row-normalized ratio is formed.

Let me actually try to run this in my head, because something feels off. The output row is `DМӮвҒ»В№ (A V)_i`, and `DМӮ(i) = ПҶ(q_i)бөҖ(ОЈ_j ПҶ(k_j))`. The exact `D(i) = ОЈ_j exp(q_iбөҖk_j)` is a sum of positive numbers вҖ” it's a positive normalizer, and `DвҒ»В№` makes each attention row a probability distribution over the value vectors. That positivity is not incidental; it's what "attention" *is*: a convex combination of values. Now my estimate `ПҶ(q_i)бөҖПҶ(k_j)` uses `sin` and `cos`, which take values in `[-1,1]`. Individual estimated scores can be **negative**. And `DМӮ(i)`, a sum of these, can be negative or near zero. Then `DМӮвҒ»В№` flips sign or explodes, and the "convex combination" is garbage. I'd expect this to show up as wildly unstable training вҖ” losses diverging, `NaN`s вҖ” for a feature map that doesn't respect the non-negativity of the scores.

And it's worse than just "sometimes negative." Where does it hurt most? The attention matrix has many small entries вҖ” most token pairs are low-relevance, `SM(x,y)` close to zero. An estimator with high *relative* variance near zero will turn those small positive numbers into noisy values straddling zero. Let me quantify the variance to see how bad. For the trigonometric estimator, with `z = x+y`, `О” = x-y`, and using that the variance of an average of `m` i.i.d. terms is `1/m` times the single-sample variance,

    MSE(SMМӮ_trig) = (1/m) exp(вҖ–xвҖ–ВІ + вҖ–yвҖ–ВІ) В· Var(cos(ПүбөҖО”)).

I need `Var(cos(ПүбөҖО”))` for `Пү вҲј N(0,I)`. Using `cosВІ= (1+cos2)/2` and `E[cos(ПүбөҖО”)] = exp(-вҖ–О”вҖ–ВІ/2)`: `E[cosВІ(ПүбөҖО”)] = ВҪ(1 + E[cos(ПүбөҖ(2О”))]) = ВҪ(1 + exp(-2вҖ–О”вҖ–ВІ))`, so `Var = ВҪ(1+exp(-2вҖ–О”вҖ–ВІ)) - exp(-вҖ–О”вҖ–ВІ) = ВҪ(1 - exp(-вҖ–О”вҖ–ВІ))ВІ`. Therefore

    MSE(SMМӮ_trig) = (1/2m) exp(вҖ–xвҖ–ВІ+вҖ–yвҖ–ВІ)(1 - exp(-вҖ–О”вҖ–ВІ))ВІ = (1/2m) exp(вҖ–zвҖ–ВІ) SM(x,y)вҒ»ВІ (1 - exp(-вҖ–О”вҖ–ВІ))ВІ,

where the last step uses `exp(вҖ–xвҖ–ВІ+вҖ–yвҖ–ВІ) = exp(вҖ–zвҖ–ВІ)В·exp(-2xбөҖy) = exp(вҖ–zвҖ–ВІ) SM(x,y)вҒ»ВІ`. There it is: the `SMвҒ»ВІ` factor out front. I've manipulated enough exponentials that I don't fully trust the algebra, so before I read anything into it let me check the formula against the thing it's supposed to predict. Take `d=2`, `x=(0.5,-0.3)`, `y=(0.2,0.4)`, `m=1`, and Monte-Carlo a single-sample trig estimate `h(x)h(y)cos(ПүбөҖО”)` over a few million draws of `ПүвҲјN(0,I)`. The sample mean lands at `0.9805` against `exp(xбөҖy)=0.9802` вҖ” unbiased, good вҖ” and the sample MSE is `0.1663`. The closed form above gives `0.1662`. They agree, so the `SMвҒ»ВІ` is really there and not an artifact of a dropped factor.

Now the consequence. `SMвҒ»ВІ` blows up as `SMвҶ’0`, and the small entries вҖ” the vast majority of token pairs, low-relevance, `SM` near zero вҖ” are exactly where the matrix lives. Let me watch it actually diverge: hold `вҖ–zвҖ–` moderate and push `xбөҖy` negative by taking `x=(a,0.4)`, `y=(-a,0.4)`, so `SM=exp(0.16-aВІ)` shrinks as `a` grows.

    a=1.0:  SM=4.32e-1   MSE_trig=4.90e+0
    a=1.5:  SM=1.24e-1   MSE_trig=6.20e+1
    a=2.0:  SM=2.15e-2   MSE_trig=2.05e+3
    a=2.5:  SM=2.27e-3   MSE_trig=1.85e+5

The MSE grows by roughly two orders of magnitude every time `SM` drops one вҖ” it is *least* accurate precisely on the entries that dominate the matrix, and those errors get amplified again by `DМӮвҒ»В№`. So trigonometric features aren't just occasionally negative вҖ” they have exploding relative error in the regime that dominates. This is the wall. The reassociation trick is correct and the cost is linear, but `sin/cos` features make the thing untrainable.

I need a feature map that is (a) unbiased for the softmax kernel, (b) **non-negative**, so `DМӮ` stays a positive normalizer, and (c) low-variance *as the kernel goes to zero*, the opposite of trig. Non-negativity says: no `sin`, no `cos`. I want something like `exp`, which is always positive. Can I write `exp(xбөҖy)` as an expectation of a product of `exp`s? Let me go back to completing the square, but this time aim for the exponential form directly.

I have `exp(xбөҖy) = exp(-вҖ–xвҖ–ВІ/2)В·exp(вҖ–x+yвҖ–ВІ/2)В·exp(-вҖ–yвҖ–ВІ/2)`. The awkward factor is `exp(вҖ–x+yвҖ–ВІ/2)`. I want to turn it into an expectation over a Gaussian `Пү`. The trick is the Gaussian normalization identity: for any fixed `c вҲҲ R^d`,

    (2ПҖ)^{-d/2} вҲ« exp(-вҖ–Пү - cвҖ–ВІ/2) dПү = 1,

because it's just a shifted unit Gaussian integrating to one. So `exp(вҖ–x+yвҖ–ВІ/2)` equals itself times that integral, with `c = x+y`:

    exp(вҖ–x+yвҖ–ВІ/2) = (2ПҖ)^{-d/2} вҲ« exp(-вҖ–Пү-(x+y)вҖ–ВІ/2) В· exp(вҖ–x+yвҖ–ВІ/2) dПү.

Expand the exponent: `-вҖ–Пү-(x+y)вҖ–ВІ/2 + вҖ–x+yвҖ–ВІ/2 = -вҖ–ПүвҖ–ВІ/2 + ПүбөҖ(x+y) - вҖ–x+yвҖ–ВІ/2 + вҖ–x+yвҖ–ВІ/2 = -вҖ–ПүвҖ–ВІ/2 + ПүбөҖ(x+y)`. The `вҖ–x+yвҖ–ВІ` terms cancel exactly. So

    exp(вҖ–x+yвҖ–ВІ/2) = (2ПҖ)^{-d/2} вҲ« exp(-вҖ–ПүвҖ–ВІ/2) exp(ПүбөҖx) exp(ПүбөҖy) dПү = E_{ПүвҲјN(0,I)}[exp(ПүбөҖx) exp(ПүбөҖy)].

That's clean. Substitute back:

    SM(x,y) = exp(xбөҖy) = E_{ПүвҲјN(0,I)}[ exp(ПүбөҖx - вҖ–xвҖ–ВІ/2) В· exp(ПүбөҖy - вҖ–yвҖ–ВІ/2) ].

This is exactly the unbiased-dot-product form I wanted, and crucially the two factors are *separable* вҖ” one depends only on `x`, the other only on `y` вҖ” and each is an exponential, hence strictly positive. So the feature map is

    ПҶвҒә(u) = (exp(-вҖ–uвҖ–ВІ/2)/вҲҡm)(exp(Пү_1бөҖu), вҖҰ, exp(Пү_mбөҖu)),    Пү_i вҲј N(0,I),

and `ПҶвҒә(x)бөҖПҶвҒә(y) = (1/m)ОЈ_i exp(Пү_iбөҖx - вҖ–xвҖ–ВІ/2)exp(Пү_iбөҖy - вҖ–yвҖ–ВІ/2)` is an unbiased, non-negative estimator of `SM(x,y)`. Call it `SMМӮвҒә`. Every entry it produces is positive, so `DМӮ` is a genuine positive normalizer and `DМӮвҒ»В№` never blows up. Property (a) and (b) are satisfied by construction. Now I need to check (c), the variance near zero вҖ” the place trig features died.

Same computation as before. `MSE(SMМӮвҒә) = (1/m) exp(-(вҖ–xвҖ–ВІ+вҖ–yвҖ–ВІ)) Var(exp(ПүбөҖz))`, `z = x+y`, because the feature is `exp(-вҖ–xвҖ–ВІ/2)exp(ПүбөҖx)` and likewise for `y`, so the product's randomness is `exp(ПүбөҖz)` scaled by `exp(-(вҖ–xвҖ–ВІ+вҖ–yвҖ–ВІ)/2)`, squared in the variance. I need `Var(exp(ПүбөҖz)) = E[exp(2ПүбөҖz)] - E[exp(ПүбөҖz)]ВІ`. The moment generating function of `ПүбөҖz вҲј N(0,вҖ–zвҖ–ВІ)` is `E[exp(t ПүбөҖz)] = exp(tВІвҖ–zвҖ–ВІ/2)`, so `E[exp(ПүбөҖz)] = exp(вҖ–zвҖ–ВІ/2)` and `E[exp(2ПүбөҖz)] = exp(2вҖ–zвҖ–ВІ)`. Thus `Var = exp(2вҖ–zвҖ–ВІ) - exp(вҖ–zвҖ–ВІ) = exp(вҖ–zвҖ–ВІ)(exp(вҖ–zвҖ–ВІ)-1)`. Putting it together,

    MSE(SMМӮвҒә) = (1/m) exp(-(вҖ–xвҖ–ВІ+вҖ–yвҖ–ВІ)) exp(вҖ–zвҖ–ВІ)(exp(вҖ–zвҖ–ВІ) - 1).

Now `exp(-(вҖ–xвҖ–ВІ+вҖ–yвҖ–ВІ))exp(вҖ–zвҖ–ВІ) = exp(2xбөҖy) = SM(x,y)ВІ`, so

    MSE(SMМӮвҒә) = (1/m) exp(вҖ–zвҖ–ВІ) SM(x,y)ВІ (1 - exp(-вҖ–zвҖ–ВІ)).

Let me sanity-check this one the same way before I trust it: same `d=2`, `x=(0.5,-0.3)`, `y=(0.2,0.4)`, `m=1`. Monte-Carlo of `exp(ПүбөҖx-вҖ–xвҖ–ВІ/2)exp(ПүбөҖy-вҖ–yвҖ–ВІ/2)` gives mean `0.9797` (unbiased against `0.9802`) and MSE `0.6229`; the closed form gives `0.6233`. Agreement again, and worth noting: at *this* point, where `SMвүҲ0.98` is large, the positive estimator's MSE (`0.62`) is actually *worse* than the trig estimator's (`0.17`). So it's not uniformly better вҖ” the trade is regime-dependent, and I have to look at where it matters.

Where it matters is `SMвҶ’0`. Trig had `SMвҒ»ВІ` out front; this has `SMВІ`. Run the same shrinking-`SM` sweep `x=(a,0.4)`, `y=(-a,0.4)` through the positive formula and put it next to the trig column from before:

    a=1.0:  SM=4.32e-1   MSE_pos=1.67e-1    MSE_trig=4.90e+0
    a=1.5:  SM=1.24e-1   MSE_pos=1.37e-2    MSE_trig=6.20e+1
    a=2.0:  SM=2.15e-2   MSE_pos=4.14e-4    MSE_trig=2.05e+3
    a=2.5:  SM=2.27e-3   MSE_pos=4.60e-6    MSE_trig=1.85e+5

They move in opposite directions: as `SM` falls, `MSE_pos` falls *with* it while `MSE_trig` climbs. The exponential estimator is most accurate exactly where the trigonometric one is least accurate вҖ” on the small entries that dominate the matrix and that the normalizer amplifies. That's the trade I care about. Positive features aren't a cosmetic fix for the sign вҖ” in the regime that was killing training they cut the variance by orders of magnitude, and they pay for it only on the large entries, where there's accuracy to spare.

Can I push the variance down further for free? The estimator uses `exp(ПүбөҖz)`. Since `Пү` and `-Пү` have the same distribution (Gaussian is symmetric), I could symmetrize: use both `exp(ПүбөҖz)` and `exp(-ПүбөҖz)`, i.e. set `l=2` with `f_1=exp(u)`, `f_2=exp(-u)`, and `h(u) = exp(-вҖ–uвҖ–ВІ/2)/вҲҡ2`. The estimator becomes proportional to `cosh(ПүбөҖz)`, still positive, still unbiased (the cross terms in expectation give the same `SM`). Its MSE: averaging `ВҪ(e^{ПүбөҖz} + e^{-ПүбөҖz})` and using that the two have equal variance and a *negative* covariance, the variance of the average should drop below the variance of either alone. Working it out, `MSE(SMМӮ^{hyp+}) = ВҪ(1 - exp(-вҖ–zвҖ–ВІ)) MSE(SMМӮвҒә)`. The factor is `ВҪ(1 - exp(-вҖ–zвҖ–ВІ))`, which is at most `ВҪ` вҖ” so the symmetrized estimator's MSE is at most half the plain one's, no matter what `x,y` are. Let me look at the factor across a few `вҖ–zвҖ–ВІ`: at `вҖ–zвҖ–ВІ=0.41` it's `0.168`, at `2.65` it's `0.465`, at `9.25` it's `0.500`. It saturates at `ВҪ` for large `вҖ–zвҖ–` and is smaller for small `вҖ–zвҖ–` вҖ” never above `ВҪ`. So `SMМӮ^{hyp+}` is always at least as good as `SMМӮвҒә`, and by more than a rounding margin: at the same compute cost вҖ” two exponentials per `Пү`, matching what `2m` fresh projections would cost in the plain estimator вҖ” its MSE comes out `(1-exp(-вҖ–zвҖ–ВІ))` times smaller than doubling `m` in the plain map, strictly better for any finite `z`. That comparison, though, is against fresh *independent* directions; I haven't worked out how symmetrizing `m` directions stacks up against spending the same budget on `m` more *orthogonal* directions instead, which is its own separately-quantified lever. I'll leave that trade unresolved and ship the plain positive map with orthogonal sampling as the default вҖ” the lever I've actually quantified вҖ” keeping symmetrization as a documented option rather than folding it in.

Now the second lever. I'm drawing `m` independent Gaussian directions `Пү_i`. Independence is wasteful вҖ” independent random directions in high dimensions cluster and waste samples. If I instead force `Пү_1,вҖҰ,Пү_m` to be *exactly orthogonal*, I cover the sphere more evenly. And I can do this without breaking unbiasedness: for an isotropic distribution like `N(0,I)`, I can sample a Gaussian block and run Gram-Schmidt, which keeps each `Пү_i`'s marginal distribution the same (still `N(0,I)` directionally with chi-distributed length) while making rows pairwise orthogonal inside a block. The marginals are unchanged, so each individual feature is still unbiased, so the average is still unbiased. A single orthogonal block has at most `d` rows; if I want more than `d` features, I stack independent `d Г— d` orthogonal blocks and a final partial block. Does orthogonality actually reduce variance, and by how much?

Let me set up the general object, because the same argument will cover softmax, its symmetrized version, and the Gaussian kernel at once. All of these have the form

    F(z) = E_{ПүвҲјО©}[g(ПүбөҖz)],

where `О©` is isotropic and `g` is an entire function with **non-negative** power-series coefficients вҖ” for softmax, `О© = N(0,I)`, `g = exp`, `g(u) = ОЈ_s a_s u^s` with `a_s = 1/s! вүҘ 0`, and `SM(x,y) = exp(-(вҖ–xвҖ–ВІ+вҖ–yвҖ–ВІ)/2) F(x+y)`. The non-negativity of the `a_s` is going to matter; it's the algebraic fingerprint of positivity. Call such an `F` a well-behaved (entire, non-negative-coefficient) kernel function. The independent estimator is `FМӮ^iid = (1/m)ОЈ_i g(Пү_i^{iid бөҖ}z)`, the orthogonal one `FМӮ^ort = (1/m)ОЈ_i g(Пү_i^{ort бөҖ}z)`.

Both are unbiased, so MSE equals variance, and the difference is entirely in the cross terms:

    MSE(FМӮ^iid) - MSE(FМӮ^ort) = (1 - 1/m)(E[X_1^{iid}X_2^{iid}] - E[X_1^{ort}X_2^{ort}]),

where `X_i = g(Пү_iбөҖz)`, because the diagonal terms `E[X_iВІ]` are equal (same marginals) and there are `m(m-1)` ordered cross pairs, each scaled by `1/mВІ`, giving the `(1-1/m)` prefactor. So I need the sign and size of `E[X_1^{iid}X_2^{iid}] - E[X_1^{ort}X_2^{ort}]`. Expand each `X_i` in the power series: `X_i = ОЈ_s a_s (Пү_iбөҖz)^s`. The cross expectation is a sum over monomials `E[(Пү_1бөҖz)^{d_1}(Пү_2бөҖz)^{d_2}]` with non-negative coefficients `a_{d_1} a_{d_2}`. So it suffices to compare these mixed moments under independence versus orthogonality.

Here's the geometric fact that does the work. Write `Пү_i = вҖ–Пү_iвҖ– В· Г»_i`, where `Г»_i` is the direction. The lengths `вҖ–Пү_iвҖ–` are drawn the same way in both schemes (orthogonalization only touches directions), and they're independent of the directions. For the directions: a set of `m` orthonormal directions is a random rotation of the canonical basis `e_1,вҖҰ,e_m`. Rotating the basis and then projecting `z` onto it is the same in distribution as fixing the basis and randomly rotating `z` вҖ” and a randomly rotated `z` is distributed as `вҖ–zвҖ– В· g/вҖ–gвҖ–` for a fresh Gaussian `g`. So

    E[(Пү_1^{ort бөҖ}z)^{d_1}вӢҜ(Пү_m^{ort бөҖ}z)^{d_m}] = (вҲҸ_i E[вҖ–Пү_iвҖ–^{d_i}]) вҖ–zвҖ–^{ОЈd_i} E[ g_1^{d_1}вӢҜg_m^{d_m} / вҖ–gвҖ–^{ОЈd_i} ],

while for the independent case the directions are independent, so the last expectation *factorizes* into `вҲҸ_i E[g_i^{d_i}/вҖ–gвҖ–^{d_i}]`. The difference between the joint moment and the product of marginal moments is exactly an orthogonality effect. Using a normalization lemma вҖ” `E[g_1^{k_1}вӢҜg_s^{k_s}/вҖ–gвҖ–^{k_1+вҖҰ+k_s}] = вҲҸ_i E[g_i^{k_i}] / E[вҖ–gвҖ–^{k_1+вҖҰ+k_s}]` (proved by introducing an independent Gaussian length and using that `g/вҖ–gвҖ–В·вҖ–gМғвҖ– вҲј g`) вҖ” both sides reduce to ratios of `ПҮ`-distribution moments, and the orthogonal-minus-independent comparison comes down to a single ratio

    П„(d_1,вҖҰ,d_m) = вҲҸ_i Ој_d(d_i) / Ој_d(ОЈ_i d_i),    Ој_d(j) = j-th moment of the ПҮ_d distribution.

Two facts pin it down. First, any monomial with an *odd* `d_i` contributes zero (odd Gaussian moments vanish), and any monomial supported on a *single* index contributes zero (then it's the same in both schemes). So only monomials with at least two even, positive exponents survive. Second, for those, with `Ој_d(j) = 2^{j/2} О“((d+j)/2)/О“(d/2)`, the lowest nontrivial monomial is `d_i = d_j = 2`, all others zero, where `П„ = Ој_d(2)ВІ / Ој_d(4)`. Let me just evaluate that and compare it to the `d/(d+2)` I'm claiming it equals, across a range of dimensions:

    d=2:   Ој_d(2)ВІ/Ој_d(4) = 0.5000   d/(d+2) = 0.5000   1-П„ = 0.5000
    d=4:                    0.6667             0.6667         0.3333
    d=8:                    0.8000             0.8000         0.2000
    d=16:                   0.8889             0.8889         0.1111
    d=64:                   0.9697             0.9697         0.0303

They match exactly at every `d`, so `П„(2,2) = d/(d+2)`, and a check of the higher monomials confirms this is the largest survivor, so `П„ вүӨ d/(d+2)` throughout, meaning `1 - П„ вүҘ 2/(d+2) > 0`. Because all the coefficients `a_s a_t` are non-negative (this is where positivity is load-bearing вҖ” if coefficients could be negative the surviving terms could cancel or flip sign), every surviving term in the difference is `вүҘ 0`, so the orthogonal estimator has *smaller or equal* second moment, hence smaller MSE. Keeping the leading `(d_i=d_j=2)` term as a lower bound gives the explicit gap:

    MSE(FМӮ^ort) вүӨ MSE(FМӮ^iid) - (1 - 1/m)(2/(d+2))(F(z) - a_0)ВІ,

where `a_0 = g(0)` is the constant term (for `exp`, `a_0 = 1`). Translating back to the softmax kernel via `F(z) = SM(x,y)exp((вҖ–xвҖ–ВІ+вҖ–yвҖ–ВІ)/2)` and the `exp(-(вҖ–xвҖ–ВІ+вҖ–yвҖ–ВІ))` prefactor, the gap is `(2(m-1)/(m(d+2)))(SM(x,y) - exp(-(вҖ–xвҖ–ВІ+вҖ–yвҖ–ВІ)/2))ВІ`. This derivation leaned on several distributional facts вҖ” lengths independent of directions, an orthonormal frame as a random rotation of the basis, the normalization lemma вҖ” and any one of them could be subtly wrong, so I want to see the conclusion directly rather than just trust the chain. Take `d=4`, `m=2`, a random `x,y` of modest norm, and Monte-Carlo both estimators: for iid I draw `m` independent `N(0,I)` rows; for orthogonal I draw a Gaussian block, orthonormalize it into a *sign-corrected* (Haar) frame so the marginals stay isotropic, and rescale each row by an independent `ПҮ_d` length. With three million trials I get the orthogonal estimator's mean at `0.686` against `SM=0.686` вҖ” still unbiased, as required вҖ” and `MSE_iid = 6.18e-2` versus `MSE_ort = 6.08e-2`. The orthogonal one is genuinely lower; the gap has the predicted sign. (It also flagged a trap: my first attempt used the raw `numpy` QR factor, whose sign convention is *not* sign-uniform, and that biased the estimator вҖ” the marginals must be exactly preserved or unbiasedness breaks, which is precisely the condition the Gram-Schmidt-on-isotropic argument needs.) And note: the gap holds for **every** `d > 0`, not just asymptotically вҖ” orthogonality helps even in low dimension, which earlier orthogonal-feature analyses couldn't claim. The same power-series machinery, applied to the moment generating function rather than the second moment, gives exponentially small tail bounds for the regularized-softmax estimator that subtract an explicit positive correction from the iid bound.

So two ingredients have survived their checks: positive (`R+`) features, which keep the normalizer positive and drive the variance toward zero on exactly the small entries that dominate; and orthogonal (`O`) sampling of the random directions, which cuts the variance further by a gap I confirmed both analytically and by Monte Carlo. Stacking them into the reassociation gives a stable, linear-time approximation whose softmax-kernel entries are unbiased and whose normalized attention output avoids the negative-normalizer failure mode.

But an MSE bound, even an exponential tail bound, is a promise about one entry `(i,j)`. What I actually need is every one of the `LВІ` entries close to the truth *at once*, and I need the feature count `m` to pay for that guarantee without growing with `L` вҖ” otherwise "linear in `L`" quietly stops being true the moment I ask for a real correctness guarantee instead of an average-case one. The naive fix, union-bounding the per-entry failure probability over all `LВІ` pairs, is exactly what breaks this: driving `LВІ` failure probabilities down forces the per-entry probability down like `1/LВІ`, and with an exponential tail bound that costs `m ~ log L` вҖ” a feature budget that grows, however slowly, with the thing I'm trying to be independent of. The way out is that queries and keys don't range over all of `R^d`; after the projections that produce them their norms are bounded by some radius `R` that has nothing to do with `L`. So instead of union-bounding over the specific `LВІ` pairs that happen to occur, I can cover that fixed ball once with a finite `Оө`-net whose size depends only on `d` and `R`, apply the tail bound at each net point, and extend to the rest of the ball using that the kernel and the feature map vary smoothly with their argument. That covering-based extension of orthogonal-feature tail bounds is worked out in recent random-feature theory; running it through gives `m = Оҳ((d/ОҙВІ) log(4d^{3/4}R/Оҙ))` features for uniform entrywise error `Оҙ`, with `L` never appearing вҖ” the radius `R` costs only a log, and doubling the sequence doubles the work but never the feature budget.

One more variant worth deriving, because the unbounded `exp` makes me slightly nervous about numerical range. What if I sample `Пү` not from `N(0,I)` but uniformly from the sphere of radius `вҲҡd` вҖ” i.e. replace `Пү` by `вҲҡd В· Пү/вҖ–ПүвҖ–`? Call the resulting kernel `SMREG`, the regularized softmax kernel. It's the same construction with a different isotropic `О©`, so all the orthogonality results carry over. How close is `SMREG` to `SM`? Expand `SM` and `SMREG` in the Taylor series of the exponential. For `SM`, `F(z) = ОЈ_k (1/(2k)!)вҖ–zвҖ–^{2k} d^k E[(ПүМӮбөҖe_1)^{2k}]` after using isotropy to kill odd terms and writing `Пү = вҖ–ПүвҖ–ПүМӮ`. The angular moment `A(2k,d) = E[(ПүМӮбөҖe_1)^{2k}] = (2k-1)!! / [(d+2k-2)(d+2k-4)вӢҜd]`, which I can get by integrating `sin^{d-2}Оё` against `cos^{2k}Оё` and a partial-integration recursion `F(k,d) = (k-1)/(d+1) F(k-2,d+2)`. The ratio of the `k`-th terms is `f(k,d) = d^k/[(d+2k-2)вӢҜd]`. Let me tabulate it at `d=16` to see whether it really stays `вүӨ 1`: `f(0..6,16) = 1, 1, 0.889, 0.711, 0.517, 0.345, 0.212`. Every term is at most `1`, with equality only at `k=0,1`, and the later terms decay вҖ” so the regularized series is dominated by the softmax series term by term, i.e. `SMREG вүӨ SM`. A direct Monte-Carlo of the radius-`вҲҡd` estimator at `d=16`, for one random `x,y` pair, gives `SMREG/SM вүҲ 0.906` вҖ” consistent with the term-by-term dominance just shown, modestly below `SM`, and the gap should tighten as `d` grows since every ratio `f(k,d)` in the tail approaches `1`. Bounding the tail with a Poisson concentration argument (the partial sums of `w^k/k!` are a Poisson CDF, `w = вҖ–zвҖ–ВІ/2`) gives the asymptotic statement

    SMREG(x,y)/SM(x,y) вүҘ 1 - 2/d^{1/3} + o(1/d^{1/3}),

so for the dimensions I actually use, `SMREG` stays a bounded, clean lower bound that tracks `SM` вҖ” a safe drop-in proxy when I'd rather not carry the unbounded `exp`.

Now the causal case, because half of what I care about is autoregressive. There, `A` is masked to its lower triangle: token `i` attends only to `j вүӨ i`. The reassociation I love relies on summing `ОЈ_j ПҶ(k_j)v_jбөҖ` over *all* `j`; with masking I instead need, for each `i`, the *prefix* sum over `j вүӨ i`. So define the running outer-product accumulator `G_i = ОЈ_{jвүӨi} ПҶ(k_j) (v_j, 1)бөҖ`, an `r Г— (d+1)` matrix (I append a `1` so the same machinery computes the normalizer `DМӮ` alongside `AV`). Then the `i`-th output row is `ПҶ(q_i)бөҖ G_i`. Computing `G_i` from `G_{i-1}` adds one `r Г— d` outer product, so the whole pass is `O(Lrd)` work, `O(rd)` streaming state if I scan sequentially, and `O(log L)` parallel depth if I materialize a prefix-sum tensor.

Before I believe "prefix sum = masked attention," let me trace it on a tiny case, `L=3`, `r=2`, `d=2`, with random non-negative feature rows `ПҶ(q_i)`, `ПҶ(k_j)` and random `V`. The honest reference is masked attention computed the slow way: form the `3Г—3` score estimate `ГӮ = Q'(K')бөҖ`, zero its strict upper triangle, row-normalize, multiply by `V`. The fast way is the prefix scan: `kv_prefix = cumsum_j ПҶ(k_j)v_jбөҖ`, `k_prefix = cumsum_j ПҶ(k_j)`, then row `i` is `ПҶ(q_i)бөҖ kv_prefix_i` over `ПҶ(q_i)бөҖ k_prefix_i`. Running both, the reference output rows come out

    [-0.281, -0.668]
    [-0.806, -0.480]
    [-0.593, -0.486]

and the prefix-scan output matches them to `1.1e-16` вҖ” floating-point identical. So the running sum really does reproduce lower-triangular masking, and I never form the `L Г— L` mask: causality is just "use the running sum so far." Linear in `L`, and exactly the same features.

A last generalization falls out for free. Nothing in the reassociation needed the feature map to come from the softmax kernel specifically вҖ” it only needed `ПҶ(x) вүҘ 0` so the normalizer stays positive, and some `ПҶ` that produces a sensible similarity. So I can take `ПҶ(x) = f(x) + Оө` directly, or optionally `ПҶ(x)=f(Wx)+Оө` with random features, for any non-negative `f` and treat the choice of `f` as a hyperparameter вҖ” a *generalized* kernelizable attention. I'll make `f = ReLU` the default: deterministic features applied straight to `x` (no projection unless I ask for one), no softmax norm correction, and a small additive `Оө = 10^{-3}` for numerical safety. The softmax estimator is then one option (`f = exp` with the norm-correction and redrawn orthogonal projections) and ReLU another, both at linear cost.

Coding it up: softmax attention needs nonnegative random features with the `d^{-1/4}` normalization derived above, an orthogonal Gaussian projection built by drawing Gram-Schmidt blocks and rescaling each row by an independent chi-length (exactly the construction that kept the marginals unbiased), a per-query/per-key max subtraction so the unbounded `exp` stays in range on real hardware, and a denominator stabilizer (`1e-6` вҖ” a negligible nudge for any real attention score, it only perturbs the rare `t` that lands within `1e-6` of zero) so `tвүҲ0` can't produce a divide-by-zero; generalized attention swaps in deterministic ReLU features with `Оө = 10^{-3}` вҖ” a bigger floor than the softmax path needs, because `ReLU`, unlike `exp`, actually reaches zero вҖ” and skips the redraw. For the feature count, the theory above only pins down a growth rate, `m = Оҳ((d/ОҙВІ) log(4d^{3/4}R/Оҙ))`; for the per-head dimensions and the accuracy I need in practice that lands in the low hundreds, so `nb_features=256` is the practical value in that regime rather than a separately derived constant. Both feed the same two contractions вҖ” global sum for the bidirectional case, prefix scan for the causal one вҖ” that the reassociation above already derived.
