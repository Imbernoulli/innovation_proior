Let me start from the thing that actually bothers me. I can minimize a function subject to equality constraints — Lagrange handed that down two centuries ago. min f(x) with h_j(x)=0 for all j: at the optimum, ∇f(x̄) + Σ_j ν_j ∇h_j(x̄) = 0. I can even see *why* it has to be true. Walk along any feasible direction d — one that keeps every h_j = 0 to first order, so ∇h_j(x̄)ᵀd = 0. If ∇f(x̄)ᵀd were nonzero, I'd move in whichever of ±d decreases f and improve, contradicting optimality. So ∇f(x̄)ᵀd = 0 for every d orthogonal to all the ∇h_j, which means ∇f(x̄) lies in the span of the ∇h_j. That's the multiplier rule: −∇f is a linear combination of constraint gradients, the coefficients are the ν_j, and they're unrestricted in sign because the constraint is two-sided — both +d and −d are feasible, so the argument is symmetric.

Now I want inequalities. min f(x) subject to g_i(x) ≤ 0. And the symmetric argument breaks the moment I write it down. Stand at an optimum x̄. The constraints split into two kinds: the ones that are *active*, g_i(x̄) = 0 — I'm pressed right up against that wall — and the ones that are *slack*, g_i(x̄) < 0 — that wall is far away. For a slack constraint, a small step in any direction keeps me feasible with respect to it; it does nothing, locally it might as well not exist. For an active constraint, only *some* directions stay feasible: I can move into the region (g_i decreasing) freely, but I cannot move out (g_i increasing) without violating it. So the feasible directions at x̄ no longer form a subspace. They form a *cone*. And on a cone the symmetric trick dies: I can't say "if ∇fᵀd ≠ 0 then −d improves," because −d might be infeasible.

So what *can* I say? Optimality means: there is no feasible descent direction. No direction d that simultaneously (a) decreases f, ∇f(x̄)ᵀd < 0, and (b) keeps every active constraint from being violated, ∇g_i(x̄)ᵀd ≤ 0 for each active i. (Slack constraints impose no first-order restriction, so they drop out — already that's telling me the slack ones won't appear in the final condition.) Rewrite (a) as (−∇f)ᵀd > 0. So optimality says: there is no d with (−∇f)ᵀd > 0 and ∇g_iᵀd ≤ 0 for all active i. Equivalently, flipping the active-constraint inequalities to (−∇g_i)ᵀd ≥ 0: for every d in the cone {d : (−∇g_i)ᵀd ≥ 0, i active}, I must have (−∇f)ᵀd ≤ 0.

Stare at that. "(−∇f)ᵀd ≤ 0 for every d in the cone cut out by the (−∇g_i)." That is *exactly* the shape of a statement Farkas settled. His lemma — really a fact about homogeneous linear inequalities, the kind Minkowski indicated and Weyl cleaned up — says: bᵀd ≤ 0 holds for *all* d satisfying a system a_iᵀd ≥ 0 if and only if b is a nonnegative combination of the *negative* rows, b = −Σ_i λ_i a_i with λ_i ≥ 0. Apply it with b = −∇f and a_i = −∇g_i over the active i:

−∇f(x̄) = Σ_{i active} λ_i ∇g_i(x̄),  λ_i ≥ 0,

that is, ∇f(x̄) + Σ_{i active} λ_i ∇g_i(x̄) = 0 with λ_i ≥ 0. There it is — and look what fell out by *itself*. The multipliers are **nonnegative**, λ_i ≥ 0, where the equality multipliers ν_j were free. That asymmetry isn't an axiom I'm imposing; it's the cone being one-sided. Farkas gives me a *nonnegative* combination because the directions are one-sided, full stop. And the slack constraints never entered, because they put no constraint on d. If I now want to write a single clean condition over *all* i, not just the active ones, I can include the slack constraints with multiplier zero: set λ_i = 0 whenever g_i(x̄) < 0. Then

∇f(x̄) + Σ_i λ_i ∇g_i(x̄) = 0,  λ_i ≥ 0,  and λ_i = 0 for slack i.

That last clause — "λ_i = 0 unless g_i is active" — I can state more slickly. Active means g_i(x̄)=0, slack means g_i(x̄)<0 (and then λ_i=0). In both cases the product λ_i g_i(x̄) = 0. So the clause is exactly **λ_i g_i(x̄) = 0 for every i**: complementary slackness. Either the constraint is tight or its multiplier is off. One of the two slacks — the constraint's, or the multiplier's — is always zero. I didn't postulate that either; it's just bookkeeping for "far-away walls exert no force."

Throwing the equality constraints h_j back in changes nothing about this argument — they contribute their two-sided span, multipliers ν_j free — so the full first-order conditions are

  ∇f(x̄) + Σ_i λ_i ∇g_i(x̄) + Σ_j ν_j ∇h_j(x̄) = 0   (stationarity)
  g_i(x̄) ≤ 0,  h_j(x̄) = 0                            (primal feasibility)
  λ_i ≥ 0                                             (dual feasibility / sign)
  λ_i g_i(x̄) = 0                                      (complementary slackness)

and I notice the stationarity equation is precisely "the gradient of the function L(x,λ,ν) = f(x) + Σ_i λ_i g_i(x) + Σ_j ν_j h_j(x) vanishes in x." So a single scalar object L — fold the constraints into the objective with these multiplier weights — has a critical point in x at the optimum. That object is going to be the protagonist; let me call it the Lagrangian and come back to *why* this particular combination, because right now I've only shown the conditions are necessary, and I quietly used a step that can fail.

Where can it fail? I passed from "no feasible descent direction" to "−∇f is in the cone of active gradients" through Farkas. Farkas is about the *linear* (first-order) cone {d : (−∇g_i)ᵀd ≥ 0}. But the *real* feasible directions are the ones tangent to actual feasible arcs of the curved constraint set, and a priori the linearized cone could be bigger than the true tangent cone — there could be a d with (−∇g_i)ᵀd ≥ 0 that is *not* tangent to any feasible arc, a direction the linearization says is allowed but the curvature forbids. If that happens, "no feasible descent arc" no longer forces "(−∇f)ᵀd ≤ 0 on the whole linear cone," and the multipliers needn't exist.

Let me actually build the pathology so I trust it. Take the region (1 − x₁)³ − x₂ ≥ 0 with x₁ ≥ 0, x₂ ≥ 0, and sit at the point (1, 0). The curved inequality in standard form is g = x₂ − (1−x₁)³ ≤ 0; its gradient at (1,0) is ∇g = (3(1−x₁)², 1) = (0, 1). The wall x₂ ≥ 0 is also active, written as −x₂ ≤ 0 with gradient (0, −1), while x₁ ≥ 0 is slack. The direction d = (1, 0) satisfies ∇gᵀd = 0 and ∇(−x₂)ᵀd = 0, so the linearization calls it admissible. But no feasible arc has that derivative. If x(t) is feasible with x(0)=(1,0) and x'(0)=d, then x₁(t)=1+t+o(t)>1 for small t>0, so (1−x₁(t))³<0; feasibility also requires 0≤x₂(t)≤(1−x₁(t))³, which cannot hold. So take f(x) = −x₁ and look at (1,0): every feasible point has x₁≤1, so (1,0) really minimizes f, while the linear cone still contains the "descent" direction d=(1,0). The Farkas step, fed that spurious direction, produces no multipliers. The conditions are simply false here: active gradients have zero first component, so there are no nonnegative multipliers with ∇f(1,0) + λ∇g(1,0) + η∇(−x₂)(1,0) = 0, since ∇f = (−1,0). The cusp has broken stationarity.

So I need a hypothesis that rules this out — a **constraint qualification**: at the boundary, every direction the linearization admits must actually be tangent to a feasible arc. Precisely: for each x̄ on the boundary and each d satisfying ∇g_i(x̄)ᵀd ≤ 0 over the active inequalities and ∇h_j(x̄)ᵀd = 0 over the equalities, there is a differentiable feasible arc issuing from x̄ in the direction d. With that, the linear cone equals the tangent cone, "no feasible descent" really does mean "(−∇f)ᵀd ≤ 0 on the linear cone," and Farkas applies. The cusp is exactly what it excludes. (I'll keep this in the back of my mind — there ought to be a cleaner, more checkable version of "qualified" once I have the convex picture; a strict-interior condition feels like it should be enough.)

Good — necessary conditions, with the qualification, are nailed. But I haven't justified the *form* of L, and I haven't gotten what I actually came for, which is duality. Let me reset and ask the question that's really driving me: the linear program max cᵀx s.t. Ax ≤ b, x ≥ 0 has a dual, min bᵀu s.t. Aᵀu ≥ c, u ≥ 0, with the *same* optimal value, and the bridge is a saddle point of the bilinear φ(x,u) = cᵀx + uᵀ(b − Ax). I want that for nonlinear problems. What is the dual of a *nonlinear* program, and when do the two optima meet?

The exact unconstrained version of the problem uses brutal penalties: minimize f(x) + Σ_i I₋(g_i(x)) + Σ_j I₀(h_j(x)), where I₋(u) = 0 if u ≤ 0 and +∞ if u > 0 — an infinitely hard wall against violating an inequality — and I₀(u) = 0 if u = 0, +∞ otherwise — an infinitely hard wall against missing an equality. This is exactly equal to my constrained problem: any infeasible x pays ∞, every feasible x pays just f(x). But those walls are impossible to compute against.

So *soften* them. Replace the wall I₋(u) by a line λ_i u, and I₀(u) by a line ν_j u. When is a line an *under*-estimator of the wall — sitting at or below it everywhere — so that softening can only lower the objective? For the inequality wall I₋: I need λ_i u ≤ I₋(u) for all u. When u ≤ 0, I₋(u)=0, so I need λ_i u ≤ 0 for u ≤ 0 — that forces **λ_i ≥ 0**. When u > 0, I₋(u)=∞, no constraint. So a line under the inequality wall must have nonnegative slope. *There* is the sign condition again, and this time I see it as pure geometry: a downward-sloping line would poke up above the wall on the feasible side u ≤ 0. For the equality wall I₀: I need ν_j u ≤ I₀(u); at u=0 both are 0, off u=0 the wall is ∞, so *any* slope ν_j works — ν_j is **free**. The asymmetry between λ ≥ 0 and ν free is the same fact a third time: one-sided wall forces a sign, two-sided wall doesn't.

Make those substitutions and the softened objective is f(x) + Σ_i λ_i g_i(x) + Σ_j ν_j h_j(x) — that is L(x,λ,ν), the Lagrangian, now *derived* as the soft under-estimate of the penalized problem, not pulled from a hat. And because each replaced piece is an under-estimator (λ_i g_i ≤ I₋(g_i), ν_j h_j ≤ I₀(h_j) when λ ⪰ 0), the softened minimum can only be ≤ the true minimum. Let me make that exact, because it's the whole game. Define the **dual function**

  d(λ,ν) = inf_x L(x,λ,ν) = inf_x [ f(x) + Σ_i λ_i g_i(x) + Σ_j ν_j h_j(x) ].

Claim: for any λ ⪰ 0 and any ν, d(λ,ν) ≤ p⋆, where p⋆ is the true optimal value. Proof, and watch where the sign is used: take any feasible x̃ — so g_i(x̃) ≤ 0 and h_j(x̃) = 0. Then Σ_i λ_i g_i(x̃) ≤ 0, because each λ_i ≥ 0 and each g_i(x̃) ≤ 0 — a product of a nonnegative and a nonpositive number is nonpositive — and Σ_j ν_j h_j(x̃) = 0 because every h_j(x̃) = 0 regardless of ν_j. Hence

  L(x̃,λ,ν) = f(x̃) + (≤ 0) + (= 0) ≤ f(x̃).

And d(λ,ν) = inf_x L(x,λ,ν) ≤ L(x̃,λ,ν) ≤ f(x̃). This holds for *every* feasible x̃, so it holds for the best one: d(λ,ν) ≤ inf_{x̃ feasible} f(x̃) = p⋆. Done. That's **weak duality**: every choice of multipliers (with λ ⪰ 0) gives a *lower bound* on the optimum. And I see precisely why λ ⪰ 0 is non-negotiable — it is the exact condition that makes the inequality penalty nonpositive on the feasible set; drop it and λ_i g_i(x̃) could be positive and the bound collapses. The equality multipliers can be anything because their term is identically zero on the feasible set.

This is already worth something on its own. The lower bound d(λ,ν) is a function of multipliers I get to choose, so I should choose them to make the bound as tight as possible — push it up toward p⋆. That's a new optimization problem, the **dual problem**:

  maximize d(λ,ν) subject to λ ⪰ 0,

with optimal value d⋆. By weak duality, d⋆ ≤ p⋆ always. Another useful thing comes free: d(λ,ν) = inf_x L(x,λ,ν) is, for each x, an *affine* function of (λ,ν) (L depends linearly on the multipliers), and a pointwise infimum of affine functions is **concave**. So d is concave, the constraint λ ⪰ 0 is convex, and the dual problem is a *concave maximization* — a convex problem — **no matter how ugly the primal is**. Even a hideous nonconvex, even combinatorial, primal has a convex dual whose optimum is a certified lower bound on it. (Picture min xᵀWx over x_i ∈ {±1}, i.e. x_i² = 1 — 2ⁿ feasible points, hopeless to enumerate. Lagrangian xᵀ(W + diag(ν))x − 1ᵀν; the inf over x of a quadratic form is −∞ unless W + diag(ν) ⪰ 0, in which case it's −1ᵀν. So the dual is a clean semidefinite problem maximize −1ᵀν s.t. W + diag(ν) ⪰ 0, and any feasible ν — e.g. ν = −λ_min(W)·1 — already lower-bounds the combinatorial optimum by nλ_min(W). The bound dropped out of a problem I can't even solve.)

Now the real question. d⋆ ≤ p⋆ — but is it ever *equal*? The gap p⋆ − d⋆ is the "duality gap." When is it zero — **strong duality** — so that the best lower bound is actually tight and the dual hands me the primal value exactly? In general it is *not* zero; I shouldn't expect it for arbitrary nonconvex problems. But the LP case had equality, and LP is convex, so let me look for what convexity buys.

Go back to the under-estimation picture and make it geometric, because "the gap is zero" should be a statement about a picture. Encode the problem in the space of *achievable constraint-and-objective values*. For a problem with (to keep the picture clean) inequality constraints only, consider the set of pairs (u, t) such that some x achieves objective ≤ t while landing the constraint values at ≤ u:

  𝒜 = { (u, t) : ∃ x, g_i(x) ≤ u_i for all i, f(x) ≤ t }.

(Include the "or worse" slack — larger t, larger u still in 𝒜 — so it's an epigraph-like set; that's what makes it usable.) Two facts about 𝒜. First, p⋆ = inf{ t : (0, t) ∈ 𝒜 }: feasibility is u ≤ 0, and (0,t) ∈ 𝒜 means some feasible-or-better x attains objective ≤ t. Second, the dual function is the support of 𝒜 from below: for λ ⪰ 0,

  d(λ) = inf_x [ f(x) + Σ_i λ_i g_i(x) ] = inf{ λᵀu + t : (u,t) ∈ 𝒜 },

because minimizing λᵀu + t over 𝒜 with λ ⪰ 0 drives u down to the constraint values and t down to f(x). So the hyperplane { (u,t) : λᵀu + t = d(λ) } is a *non-vertical supporting hyperplane* of 𝒜 — non-vertical because the t-coefficient is 1, not 0 — and it meets the t-axis (u = 0) at height d(λ). Weak duality is now just a picture: the point (0, p⋆) sits on the lower boundary of 𝒜 above the u = 0 axis, and any supporting hyperplane with slope −λ crosses u = 0 at d(λ) ≤ p⋆. The gap p⋆ − d(λ) is the vertical drop from (0,p⋆) down to where that supporting line hits the axis.

So **the gap is zero exactly when there is a supporting hyperplane of 𝒜 that touches the boundary point (0, p⋆) and is non-vertical** — a line of slope −λ⋆ through (0,p⋆) with all of 𝒜 above it. If the only supporting lines at (0,p⋆) are *vertical* (slope ∞, λ = ∞, useless), the dual can't reach p⋆ and the gap is positive. Two things have to go right: (i) a supporting hyperplane at (0,p⋆) has to *exist*, and (ii) it has to be *non-vertical*.

Existence of a separating/supporting hyperplane is a convexity theorem — disjoint convex sets can be separated by a hyperplane. So I want 𝒜 convex. Is it? 𝒜 is convex when f and the g_i are convex (and the equality constraints affine): take (u,t),(u',t') ∈ 𝒜 witnessed by x, x', and θ ∈ [0,1]; convexity gives g_i(θx+(1−θ)x') ≤ θg_i(x)+(1−θ)g_i(x') ≤ θu_i+(1−θ)u'_i and likewise f(θx+(1−θ)x') ≤ θt+(1−θ)t', so the convex combination (θu+(1−θ)u', θt+(1−θ)t') ∈ 𝒜. Convexity of the data makes the *value set* convex. (Without it 𝒜 can be a non-convex blob, (0,p⋆) can sit in a dent, and no supporting line touches it there — that's the positive-gap case, and it's why I can't expect strong duality in general.)

Now the clean argument. Assume f, g_i convex, equality constraints Ax = b with redundant equality rows removed so A has full row rank, and p⋆ finite. Build the convex value set, but to separate I want a *second* convex set to push against. Let

  𝒜 = { (u, v, t) : ∃ x ∈ D, g_i(x) ≤ u_i, Ax − b = v, f(x) ≤ t }   (convex),
  ℬ = { (0, 0, s) : s < p⋆ }   (the open vertical ray below the optimum).

These don't meet: a point in both would have u = 0, v = 0, t = s < p⋆ with some x feasible (g_i(x) ≤ 0, Ax = b) and f(x) ≤ t < p⋆ — an x beating the optimum, impossible. Two nonempty disjoint convex sets, so by the separating-hyperplane theorem there is (λ̃, ν̃, μ) ≠ 0 and a scalar α with

  λ̃ᵀu + ν̃ᵀv + μt ≥ α  for all (u,v,t) ∈ 𝒜,   (★)
  λ̃ᵀu + ν̃ᵀv + μt ≤ α  for all (u,v,t) ∈ ℬ.   (★★)

Read off the signs from (★). 𝒜 is "upward/rightward closed": I can make any u_i as large as I like (relax the constraint) and t as large as I like (accept a worse objective) and stay in 𝒜. If any λ̃_i were negative, sending that u_i → +∞ would send the left side of (★) → −∞, breaking the bound — so **λ̃ ⪰ 0**. Likewise if μ were negative, sending t → +∞ breaks it — so **μ ≥ 0**. From (★★), on ℬ we have u = 0, v = 0, t = s < p⋆, so μs ≤ α for all s < p⋆, hence μp⋆ ≤ α. Combine with (★) evaluated at any x ∈ D (where (g(x), Ax−b, f(x)) ∈ 𝒜): for all x ∈ D,

  Σ_i λ̃_i g_i(x) + ν̃ᵀ(Ax − b) + μ f(x) ≥ α ≥ μ p⋆.   (◇)

Everything turns on μ. Suppose **μ > 0** — the non-vertical case. Divide (◇) by μ and set λ = λ̃/μ ⪰ 0, ν = ν̃/μ:

  f(x) + Σ_i λ_i g_i(x) + νᵀ(Ax − b) = L(x, λ, ν) ≥ p⋆  for all x ∈ D.

Minimize the left over x: d(λ,ν) = inf_x L(x,λ,ν) ≥ p⋆. But weak duality already gives d(λ,ν) ≤ p⋆. So d(λ,ν) = p⋆ — the duality gap is zero, **strong duality holds**, and the dual optimum is *attained* at this (λ,ν). The whole thing came from one separating hyperplane, divided by its (positive) vertical slope.

The danger is **μ = 0** — a vertical separator, the useless case. I have to kill it, and this is where I finally pin down the qualification. If μ = 0, (◇) becomes Σ_i λ̃_i g_i(x) + ν̃ᵀ(Ax − b) ≥ 0 for all x ∈ D. Now suppose there is a *strictly* feasible point — an x̃ ∈ relint D with g_i(x̃) < 0 for every i and Ax̃ = b. Plug it in: Σ_i λ̃_i g_i(x̃) ≥ 0. But every g_i(x̃) < 0 and every λ̃_i ≥ 0, so each term λ̃_i g_i(x̃) ≤ 0, and their sum is ≥ 0 only if every term is 0 — which, with g_i(x̃) < 0, forces **λ̃ = 0**. Then (λ̃, ν̃, μ) = (0, ν̃, 0) with ν̃ ≠ 0 (the triple is nonzero). The inequality reduces to ν̃ᵀ(Ax − b) ≥ 0 for all x ∈ D. Since x̃ is in the relative interior of D and ν̃ᵀ(Ax̃ − b) = 0, that linear function cannot have a one-sided minimum at x̃ unless its gradient vanishes along the affine hull of D; in the full-dimensional case this says Aᵀν̃ = 0, and in the relative-interior case it is the same statement after restricting A to the affine hull. With redundant equality rows removed on that affine hull, full row rank rules out a nonzero ν̃ in that nullspace. So μ = 0 is impossible. Therefore μ > 0 and strong duality holds.

That strict-feasibility hypothesis is the clean, checkable constraint qualification I was hoping for: **Slater's condition** — there exists a point strictly inside all the inequality constraints (g_i(x̃) < 0) and satisfying the equalities. Geometrically it is doing exactly one job: it places a witness point of 𝒜 strictly to the *interior* side of the value space, so any separating hyperplane has to *tilt* — it cannot be vertical, because a vertical wall through (0,0,p⋆) would cut through the strictly-feasible witness sitting at negative u. Slater forbids the vertical separator, which forces μ > 0, which is strong duality. And it's so much easier to check than "every linearized direction is tangent to a feasible arc" — for convex problems I just need one strictly interior point. (All nonlinear inequality walls need strict slack here; when some inequality constraints are affine, their walls are flat and cannot curve away, so the standard refinement allows those affine inequalities to hold with equality. For an all-linear problem it degenerates to mere feasibility, which is why LP had strong duality with no fuss.)

Now I can close the loop on the KKT conditions and see them not as something I read off the cone, but as the *fingerprint of strong duality*. Suppose strong duality holds and let x⋆ be primal optimal, (λ⋆, ν⋆) dual optimal. Then

  f(x⋆) = d(λ⋆,ν⋆) = inf_x [ f(x) + Σ_i λ⋆_i g_i(x) + Σ_j ν⋆_j h_j(x) ]
        ≤ f(x⋆) + Σ_i λ⋆_i g_i(x⋆) + Σ_j ν⋆_j h_j(x⋆)
        ≤ f(x⋆).

The first line is zero gap; the second is the inf being ≤ the value at x = x⋆; the last inequality is because λ⋆_i ≥ 0, g_i(x⋆) ≤ 0 (so that sum is ≤ 0) and h_j(x⋆) = 0 (that sum is 0). The chain begins and ends at f(x⋆), so every inequality in it is an *equality*. Two consequences, both forced. From the second line being an equality: x⋆ **minimizes L(·,λ⋆,ν⋆) over x** — so its gradient vanishes there, ∇f(x⋆) + Σ_i λ⋆_i ∇g_i(x⋆) + Σ_j ν⋆_j ∇h_j(x⋆) = 0, stationarity. From the last inequality being an equality: Σ_i λ⋆_i g_i(x⋆) = 0; but it's a sum of terms each ≤ 0, and a sum of nonpositive terms is zero only if **every** term is zero, so λ⋆_i g_i(x⋆) = 0 for each i — complementary slackness, exactly as the cone argument gave me, but now as a corollary of the gap vanishing. Together with primal feasibility and λ⋆ ⪰ 0 these are the four KKT conditions, and I've now derived them twice from opposite directions: Farkas on the tangent cone (geometry of "no improving direction") and the duality chain (algebra of "zero gap"). They are the same conditions.

And the conditions run *both* ways when the problem is convex. Suppose f convex, g_i convex, h_j affine, and a point (x̃, λ̃, ν̃) satisfies all four KKT conditions. Then: primal feasibility holds by the first two. Because λ̃ ⪰ 0 and the g_i are convex, L(·, λ̃, ν̃) = f + Σ λ̃_i g_i + Σ ν̃_j h_j is a convex function of x (nonnegative combination of convex functions plus an affine piece). Stationarity says its gradient vanishes at x̃; for a convex function that means x̃ is a *global* minimizer of L(·, λ̃, ν̃). Therefore

  d(λ̃,ν̃) = inf_x L(x, λ̃, ν̃) = L(x̃, λ̃, ν̃) = f(x̃) + Σ_i λ̃_i g_i(x̃) + Σ_j ν̃_j h_j(x̃) = f(x̃),

where the last step uses h_j(x̃) = 0 and complementary slackness λ̃_i g_i(x̃) = 0 to kill both sums. So the dual value at (λ̃,ν̃) equals the primal value at x̃; by weak duality d ≤ p⋆ ≤ f(x̃) = d, everything is squeezed equal, and x̃ is primal optimal, (λ̃,ν̃) dual optimal, gap zero. KKT is *sufficient* for convex problems. And if such a convex problem also satisfies Slater, the gap is zero and the dual optimum is attained, so an optimal x always *has* KKT multipliers — making KKT **necessary and sufficient**: x is optimal iff there exist (λ,ν) closing the four conditions with it.

Let me also record the symmetric face of all this, because it's why the saddle point kept whispering. Bundle the hard penalties into a sup: for fixed x, sup_{λ ⪰ 0,ν} L(x,λ,ν) equals f(x) when x is feasible (every g_i(x) ≤ 0 and every h_j(x)=0, so the best λ is 0 and ν does nothing), and +∞ when x violates an inequality or equality (push the offending λ_i → ∞, or choose ν_j with the sign of h_j(x)). So inf_x sup_{λ ⪰ 0,ν} L(x,λ,ν) = inf over feasible x of f(x) = p⋆. And by definition sup_{λ ⪰ 0,ν} inf_x L(x,λ,ν) = d⋆. Weak duality is then the utterly general max-min inequality sup inf ≤ inf sup — true for any function, no convexity needed. Strong duality is the statement that I may **swap** the order, sup_{λ⪰0,ν} inf_x L = inf_x sup_{λ⪰0,ν} L, and that the common value is attained at a **saddle point** (x⋆, λ⋆, ν⋆): L(x⋆, λ,ν) ≤ L(x⋆, λ⋆,ν⋆) ≤ L(x, λ⋆,ν⋆) for all x, all λ ⪰ 0, and all ν — x⋆ minimizes L in x, while (λ⋆,ν⋆) maximizes it over the multiplier side. That is precisely the saddle picture LP lived in, now general: the Lagrangian's saddle point simultaneously certifies the primal optimum and the dual optimum, the two players (one choosing x to push L down, one choosing multipliers to push it up) end at the same payoff, and there is no advantage to moving second. The bilinear LP saddle was just this with f and the g_i linear; everything I did was find the right way to read it off the convex value set.

So the landing is this. Attach to min f(x) s.t. g_i(x) ≤ 0, h_j(x) = 0 the Lagrangian L(x,λ,ν) = f(x) + Σ_i λ_i g_i(x) + Σ_j ν_j h_j(x) — the soft, linear under-estimate of the infinitely-hard constraint walls, which forces λ ⪰ 0 (one-sided wall) and leaves ν free (two-sided wall). Its infimum over x, the dual function d(λ,ν) = inf_x L, is concave and is a *lower bound* on the optimal value for every λ ⪰ 0 (weak duality), because the penalty is nonpositive on the feasible set — so the dual problem max_{λ ⪰ 0} d is a convex problem certifying the primal from below no matter how nasty the primal is. The gap p⋆ − d⋆ is the vertical drop from (0,p⋆) to the supporting line of the value set 𝒜; convexity of the data makes 𝒜 convex so a supporting line exists, and a Slater strictly-feasible point forces that line to be non-vertical, which makes the gap **zero** (strong duality, via separating 𝒜 from the ray ℬ below the optimum). When the gap is zero, the chain f(x⋆) = d(λ⋆,ν⋆) ≤ L(x⋆,λ⋆,ν⋆) ≤ f(x⋆) collapses and spits out the **KKT conditions** — stationarity ∇f + Σλ_i∇g_i + Σν_j∇h_j = 0, primal feasibility, dual feasibility λ ⪰ 0, and complementary slackness λ_i g_i = 0 — which are necessary whenever strong duality holds and, for convex data, also sufficient, hence (with Slater) a complete characterization of the optimum. One scalar object, the Lagrangian, ties together the first-order conditions, the lower-bounding dual, and the saddle point, and the single thread running through every piece — the sign λ ⪰ 0, the dropping of slack constraints, the tightness of the bound — is the asymmetry between a one-sided inequality wall and a two-sided equality wall.
