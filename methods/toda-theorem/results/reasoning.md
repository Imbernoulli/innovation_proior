I start from the mismatch between the objects. The polynomial hierarchy is built from alternation: I see an existential choice, then a universal choice, then another existential choice, with a polynomial-time predicate at the bottom. A #P function does not look like that. It counts accepting branches of one nondeterministic computation. If I only say that counting is powerful, I have not explained why this nested logical form should collapse into one deterministic polynomial-time computation with a counting oracle.

I first separate the two kinds of counting that matter. Valiant gives me exact counting as a function resource: #P counts witnesses and the permanent shows that this resource is complete even for a natural combinatorial quantity. Gill's PP gives me threshold counting as a decision resource: a machine accepts when a majority of its computation paths accept. These are related, because exact counts can answer threshold questions, and threshold machines are already thinking arithmetically about computation paths. But this still leaves the hierarchy problem untouched, because alternating quantifiers are not just majority tests.

The non-obvious move is to stop attacking alternation directly. I look for a way to put the lower-level existential problem into a randomized form whose witness test is parity. Valiant-Vazirani supplies exactly that shape. A satisfiable formula can be randomly filtered so that, with noticeable probability, one of the produced formulas has a unique satisfying assignment; an unsatisfiable formula stays unsatisfiable. Unique satisfiability is visible to parity, because one witness is odd and zero witnesses is even. This gives the Toda-observed consequence that NP sits inside randomized polynomial time with an oplusP oracle.

Now I relativize instead of repeating the argument layer by layer by hand. Fortnow's exposition makes this clean: the Valiant-Vazirani consequence relativizes, Papadimitriou-Zachos gives the closure of oplusP under its own oracle, and Zachos's theorem says that if NP is inside BPP, then the whole relativized polynomial hierarchy follows it into BPP. Putting these together yields PH inside BPP^oplusP. The mindset has shifted: I do not simulate each alternating block with a direct count. I use random isolation to turn existential power into parity power, and I use closure and relativization to make the whole hierarchy follow.

That only gives randomized parity access, so I still need to explain why a deterministic machine with a #P oracle can finish the job. Fortnow's second lemma is the arithmetic core. A language in PP^oplusP can be described by a GapP function f(x,y) whose parity decides an oplusP predicate, while the outer PP machine compares how many y values give odd parity against how many give even parity. I need a #P-capable computation to recover that comparison without querying parity separately on exponentially many y values.

The polynomial g(m) = 3m^2 - 2m^3 is the amplifier. If m is even, iterating g drives it to 0 modulo rapidly growing powers of 2; if m is odd, iterating g drives it to 1 modulo those powers. I apply enough iterations to f(x,y), producing a GapP value h(x,y) that preserves only the parity information but makes it robust modulo 2^(q+1), where q bounds the number of y bits. Then I sum h(x,y) over all y. Closure of GapP under exponential-size sums keeps this in GapP, and the residue of the sum modulo 2^(q+1) is exactly the number of y values for which f(x,y) is odd. Comparing that count with its complement decides the PP question.

This is the proof's distinctive proof mindset. The counting oracle is not used as a blunt instrument that somehow knows the hierarchy is hard. It is used after the hierarchy has been reshaped: random isolation turns existential structure into parity; relativized closure carries that through the hierarchy; threshold structure is expressed as a majority over parity outcomes; and a modular GapP amplification turns those parity outcomes into an exact count. Approximate-counting context matters because it shows that randomized sampling and oracle access can make counts usable as structural information, but the final collapse here is sharper: the proof converts randomized and threshold structure into exact arithmetic that a #P oracle can evaluate.
