The first thing I need is not another way to memorize the sample. I need a way to make the learned function answer for the regions empirical risk leaves silent. If the loss only checks the training points, then a high-capacity model can be perfect on those points and still have arbitrary shape nearby. Random labels expose the memorization side of that failure, and adversarial perturbations expose the local-smoothness side.

So I start from vicinal risk rather than from a new weight penalty. The old idea is to replace each empirical point mass by a distribution around the point, then train on virtual samples from that neighborhood. That helps when I know a label-preserving move: translate an image a little, add harmless noise, move along a tangent direction. But that story keeps each virtual example tied to one original label. It does not say what should happen as one training example turns into another.

Interpolation is already nearby, so the line segment itself cannot be the whole discovery. A same-class interpolation method can thicken a minority class region, and feature-space interpolation can do something similar after an encoder. Those choices avoid the hard case by staying inside one class or a chosen local neighborhood. If I move between two arbitrary examples, especially two different classes, a fixed hard label is no longer an honest target.

My first tempting shortcut is to average the inputs and keep one endpoint's label. That creates more training locations, but the supervision has a cliff. A point that is just over halfway to one endpoint gets one target, while a point just on the other side gets the other. It also makes a 55-45 mixture and a 95-5 mixture look identical to the loss if both are assigned the closer endpoint's class. That cannot express the smooth behavior I want.

My second shortcut is ordinary label smoothing. It reduces overconfidence, which is useful, but it is blind to the direction of the synthetic input. It assigns the same small probability mass no matter which example I moved toward. I need the target to know both the partner example and the amount of movement.

The useful prior is then forced into a simple shape: along the line from one training example to another, the desired prediction should move linearly from one target to the other. I take two examples `(x_i, y_i)` and `(x_j, y_j)`, draw `lambda` in `[0, 1]`, and form `x_tilde = lambda x_i + (1 - lambda) x_j` together with `y_tilde = lambda y_i + (1 - lambda) y_j`. The same coefficient controls the input and the target. That coupling is the essential move.

I have to keep the claim modest. I am not claiming that an averaged image is a natural photograph. The classifier is already a function on the ambient input vector space, so it already has some output on that averaged vector. The question is what target should constrain that output. The averaged input gives me a place to ask; the averaged target gives the answer that makes the constraint meaningful.

Now this becomes a two-example vicinal distribution. I draw one example, draw a partner example, draw `lambda ~ Beta(alpha, alpha)`, and train on the paired feature-target convex combination. As `alpha` tends toward zero, the beta distribution concentrates near the endpoints and the virtual samples approach ordinary empirical examples. Larger positive values put more mass away from the endpoints, so the training rule asks more questions between examples.

The regularization argument only works if I state it carefully. I define the averaged predictor `tilde f(x) = E_{x'', lambda} hat f(lambda x + (1 - lambda) x'')` and measure a Lipschitz constant only over real training inputs. If `hat f` fits the virtual examples, then at paired virtual locations I can replace the learned output by the mixed target: `hat f(lambda x' + (1 - lambda)x'') = lambda f(x') + (1 - lambda)f(x'')` and similarly for `x`.

When I subtract those two expressions, the shared `(1 - lambda)f(x'')` term cancels. What remains is `lambda(f(x') - f(x))`. Since `lambda` is nonnegative, the norm of the expectation is bounded by `E[lambda]` times the corresponding training-data Lipschitz numerator for `f`, giving `hat Lip(tilde f) <= E[lambda] hat Lip(f)`. For the symmetric beta distribution this expectation is `1/2` whenever `alpha > 0`, so this proof is evidence for the cancellation produced by paired targets, not an alpha-dependent constant explaining the whole method.

That cancellation also tells me why the alternatives fail algebraically. If I add Gaussian noise, I do not know the target value at the perturbed point. If I average inputs but keep one hard label, the output is not the convex combination needed for the cancellation. If I use fixed label smoothing, the target softness is detached from the actual partner and coefficient.

The implementation should preserve exactly that coupling without extra machinery. In a minibatch, I sample one scalar `lambda`, shuffle the batch to choose partners, form `mixed_x = lambda x + (1 - lambda) x_shuffled`, and compute `lambda CE(pred, y) + (1 - lambda) CE(pred, y_shuffled)`. Cross-entropy is linear in the target distribution, so this weighted pair of hard-label losses is the same objective as cross-entropy against the dense target `lambda y + (1 - lambda)y_shuffled`.

The design choices now fit together. Random partners cover more of the span of high-dimensional training data than a nearest-neighbor rule and avoid choosing a metric. Raw input interpolation constrains the classifier where it is actually evaluated, even when the virtual input is not perceptually natural. Two-example mixtures keep the procedure cheap, while adding more examples with Dirichlet weights increases cost without changing the core prior. If `alpha <= 0` in code, setting `lambda = 1` turns the routine back into ordinary minibatch training.

There is still a limitation inside the same prior. If an interpolated point lands on the true data manifold with a soft target that conflicts with the real label there, the constraint can cause underfitting. That does not invalidate the construction; it names the assumption. I am imposing local linear behavior between sampled examples unless the data geometry makes that bias wrong.

So the discovery is a coupled vicinal rule: supervise both the synthetic input position and the synthetic target with the same interpolation coefficient. The method is not input averaging alone, and it is not target smoothing alone. It is the demand that predictions vary linearly along data-dependent chords because the target attached to each chord varies linearly too.
