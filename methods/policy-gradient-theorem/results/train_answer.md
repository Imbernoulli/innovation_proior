The canonical method I am presenting is the Policy Gradient Theorem. It gives the correct direction for improving a parameterized stochastic policy when the agent only has trajectories generated by that same policy, without requiring a model of the environment's transition probabilities. The core difficulty it solves is that the long-run performance objective depends on the policy parameters in two places: directly through the action probabilities and indirectly through the distribution of states that the policy visits. The theorem shows that the indirect dependence cancels out, leaving a gradient expression that only differentiates the policy probabilities.

Start with a Markov decision process in which the agent observes a state, samples an action from a differentiable policy pi_theta(a|s), receives a reward, and transitions to a next state. In the continuing average-reward formulation, the objective rho(theta) is the expected reward per step under the stationary state distribution induced by the policy. Writing this objective naively as a sum over states weighted by the stationary distribution would force us to differentiate that stationary distribution with respect to theta, which is not available without a model of the environment dynamics. The same problem appears in the discounted start-state formulation, where the objective is the value of a fixed start state and the future-state distribution is weighted by discounting.

The trick is to differentiate the Bellman equation for the value function rather than the one-step reward expression. In the average-reward case, the differential action value Q^pi(s,a) satisfies Q^pi(s,a) = R(s,a) - rho(theta) + sum_{s'} P(s'|s,a) V^pi(s'). Differentiating V^pi(s) with respect to theta and substituting the derivative of the Bellman equation gives an identity that contains the unwanted derivative dV^pi(s')/dtheta transported one step forward. When the entire identity is averaged over the stationary distribution, the transported term becomes exactly the same as the original dV^pi(s)/dtheta term, because stationarity means the distribution after one transition is the same as the distribution before it. The two terms cancel, leaving only the derivative of the policy probabilities.

The resulting expression is d rho(theta) / d theta = sum_s d^pi(s) sum_a [d pi_theta(a|s) / d theta] Q^pi(s,a), where d^pi is the stationary state distribution. In the discounted start-state case, the same unrolling argument produces an analogous identity, except that d^pi(s) becomes the unnormalized discounted occupancy sum_{t>=0} gamma^t Pr(S_t = s | S_0 = s0, pi). The crucial point is identical: the derivative of the state weighting does not appear.

To make the gradient sampleable, rewrite the action sum using the identity d pi_theta(a|s) / d theta = pi_theta(a|s) grad_theta log pi_theta(a|s). This converts the exact gradient into an expectation over actions drawn from the policy: d rho(theta) / d theta = sum_s d^pi(s) E_{a ~ pi_theta(.|s)} [ grad_theta log pi_theta(a|s) Q^pi(s,a) ]. Because states are sampled by following the policy and actions are sampled from the policy, the entire expression can be estimated from on-policy trajectories. In the discounted case, each step along a trajectory contributes with an additional gamma^t weight. In the average-reward case, one uses a differential return around the long-run average reward after the chain has mixed.

The theorem also justifies baselines. If we replace Q^pi(s,a) by Q^pi(s,a) - b(s) for any function b that depends only on the state, the expected gradient is unchanged. The reason is that sum_a [d pi_theta(a|s) / d theta] b(s) = b(s) d/dtheta sum_a pi_theta(a|s) = b(s) d/dtheta 1 = 0. This is the same zero-score identity that underlies REINFORCE: the expected log-policy gradient is zero because action probabilities sum to one. Subtracting an estimated state value is therefore a variance-reduction device, not an approximation, and the natural choice is b(s) ~= V^pi(s), turning Q into an advantage.

A second consequence is the compatible critic condition. If we want to replace the true action value by a learned critic f_w(s,a), simply plugging in an arbitrary approximator would bias the gradient. The theorem shows that the bias disappears if the critic is trained to a least-squares fixed point and its features match the policy score: d f_w(s,a) / d w = grad_theta log pi_theta(a|s). Under this compatibility condition, the critic's approximation error is orthogonal to the only direction used by the policy update, so the actor-critic update becomes exact gradient ascent when the critic is at its fixed point. For a softmax policy with linear preferences theta^T phi(s,a), the score is phi(s,a) - sum_b pi_theta(b|s) phi(s,b), so the compatible critic is linear in the centered features and naturally estimates an advantage-like signal.

Putting the pieces together, the Policy Gradient Theorem is the foundation of modern policy-gradient and actor-critic methods. It explains why we can safely differentiate only the policy log probabilities, why state baselines do not bias the update, and why a critic trained with the right features can replace the true action value. Below is a small, self-contained Python verification on a tiny tabular MDP. It builds a softmax policy, computes the exact policy gradient using the occupancy form of the theorem, and compares it with a sample-based Monte Carlo gradient obtained from trajectories. The two estimates agree up to Monte Carlo noise, which confirms the identity in a concrete setting.

```python
import numpy as np

np.random.seed(0)

# Tiny MDP: two states, two actions.
nS, nA = 2, 2
# Transition kernel P[s,a,s'] is deterministic for simplicity.
P = np.zeros((nS, nA, nS))
P[0, 0, 0] = 1.0
P[0, 1, 1] = 1.0
P[1, 0, 1] = 1.0
P[1, 1, 0] = 1.0
# Expected rewards R[s,a].
R = np.array([[1.0, 0.0],
              [0.0, 2.0]])
gamma = 0.95
start_state = 0

# Softmax tabular policy parameterized by theta[s,a].
def policy(theta):
    e = np.exp(theta - theta.max(axis=1, keepdims=True))
    return e / e.sum(axis=1, keepdims=True)

def score(theta, s, a):
    pi = policy(theta)
    return np.eye(nA)[a] - pi[s]

def value_iteration(theta, tol=1e-12):
    pi = policy(theta)
    V = np.zeros(nS)
    while True:
        Q = R + gamma * (P @ V)
        V_new = (pi * Q).sum(axis=1)
        if np.max(np.abs(V_new - V)) < tol:
            break
        V = V_new
    Q = R + gamma * (P @ V)
    return V, Q

def discounted_occupancy(theta):
    pi = policy(theta)
    # M[s',s] = Pr(S_{t+1}=s' | S_t=s)
    M = (pi[:, :, None] * P).sum(axis=1).T
    d = np.zeros(nS)
    d[start_state] = 1.0
    occ = np.zeros(nS)
    power = 1.0
    for _ in range(2000):
        occ += power * d
        d = M @ d
        power *= gamma
        if power < 1e-14:
            break
    return occ

def exact_gradient(theta):
    pi = policy(theta)
    _, Q = value_iteration(theta)
    occ = discounted_occupancy(theta)
    grad = np.zeros_like(theta)
    for s in range(nS):
        for a in range(nA):
            grad[s] += occ[s] * pi[s, a] * score(theta, s, a) * Q[s, a]
    return grad

def sample_gradient(theta, n_traj=10000, horizon=200):
    pi = policy(theta)
    _, Q_true = value_iteration(theta)
    accum = np.zeros_like(theta)
    for _ in range(n_traj):
        s = start_state
        traj_log_grad = []
        rewards = []
        states = []
        actions = []
        for t in range(horizon):
            a = np.random.choice(nA, p=pi[s])
            states.append(s)
            actions.append(a)
            traj_log_grad.append(score(theta, s, a))
            rewards.append(R[s, a])
            s = np.random.choice(nS, p=P[s, a])
        # Compute discounted returns and accumulate gradient.
        G = 0.0
        for t in range(horizon - 1, -1, -1):
            G = rewards[t] + gamma * G
            accum[states[t]] += traj_log_grad[t] * G
    return accum / n_traj

theta = np.random.randn(nS, nA)
grad_exact = exact_gradient(theta)
grad_sample = sample_gradient(theta, n_traj=20000, horizon=150)

print("Exact policy gradient (occupancy form):")
print(grad_exact)
print("\nSample-based Monte Carlo policy gradient:")
print(grad_sample)
print("\nRelative error (Frobenius norm):", np.linalg.norm(grad_exact - grad_sample) / (np.linalg.norm(grad_exact) + 1e-8))
```
