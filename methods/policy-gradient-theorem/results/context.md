## Research Question

An agent interacts with a Markov decision process by observing a state, choosing an action, receiving a scalar reward, and moving to a new state according to environment dynamics it does not know. The goal is to improve long-run performance: average reward per step in continuing tasks, or discounted return from a designated start state in episodic tasks.

The agent cannot keep a separate table entry for every state-action pair in realistic problems. It needs function approximation: a differentiable stochastic controller with shared parameters, and possibly a second approximator that predicts values from the same sampled experience. The open question is how to update the controller parameters so that the expected return improves, while using only trajectories generated by the current controller.

## Research Context

The dominant control recipe estimates a value function and then acts greedily with respect to that estimate. A greedy rule is naturally deterministic, while aliased or partially observed settings can require randomized behavior.

An alternative is to parameterize the stochastic controller directly. The performance objective then depends on the parameters through both action probabilities and the distribution of states the controller visits.

## Background Objects

The standard objects are already available. The environment has transition probabilities `P(s'|s,a)` and expected rewards `R(s,a)`. A policy gives probabilities `pi_theta(a|s)`. The state value `V(s)` and action value `Q(s,a)` satisfy Bellman consistency equations. In the discounted case, an action value is immediate reward plus a discounted successor value. In the average-reward case, it is a differential return around the long-run average.

There are also two useful ideas in the surrounding literature. Score-style stochastic updates can move parameters in the direction of higher expected reinforcement without enumerating all randomness, and subtracting a comparison signal can reduce variance when the comparison is independent of the sampled output. Separately, actor-critic architectures suggest using a learned value estimate to make policy updates less noisy.

## Evaluation Setting

The relevant setting is on-policy sampled interaction with no model of the environment. The agent repeatedly collects states, actions, and rewards generated by its own current stochastic controller. Performance is judged by average reward or discounted return, and by whether the parameter sequence behaves like ascent on a well-defined objective.

The comparison class includes greedy value-function control, raw return-weighted stochastic updates, and actor-critic variants that use a learned value estimate.
