## Research Question

An agent interacts with a Markov decision process by observing a state, choosing an action, receiving a scalar reward, and moving to a new state according to environment dynamics it does not know. The goal is to improve long-run performance: average reward per step in continuing tasks, or discounted return from a designated start state in episodic tasks.

The agent cannot keep a separate table entry for every state-action pair in realistic problems. It needs function approximation: a differentiable stochastic controller with shared parameters, and possibly a second approximator that predicts values from the same sampled experience. The open question is how to update the controller parameters so that the expected return improves, while using only trajectories generated by the current controller.

## Existing Pressure Points

The dominant control recipe estimates a value function and then acts greedily with respect to that estimate. This creates two problems before any new method is introduced. First, a greedy rule is naturally deterministic, while aliased or partially observed settings can require randomized behavior. Second, the map from value estimates to greedy actions is discontinuous: a tiny change in one estimate can flip the chosen action.

That discontinuity is exactly the kind of object that resists convergence arguments under function approximation. Empirical and theoretical examples already show value-based control methods cycling, diverging, or failing to settle on any policy even in simple problems with simple approximators. A smoother alternative is to parameterize the stochastic controller directly, but then the performance objective depends on the parameters through both action probabilities and the distribution of states the controller visits.

## Background Objects

The standard objects are already available. The environment has transition probabilities `P(s'|s,a)` and expected rewards `R(s,a)`. A policy gives probabilities `pi_theta(a|s)`. The state value `V(s)` and action value `Q(s,a)` satisfy Bellman consistency equations. In the discounted case, an action value is immediate reward plus a discounted successor value. In the average-reward case, it is a differential return around the long-run average.

There are also two useful ideas in the surrounding literature. Score-style stochastic updates can move parameters in the direction of higher expected reinforcement without enumerating all randomness, and subtracting a comparison signal can reduce variance when the comparison is independent of the sampled output. Separately, actor-critic architectures suggest using a learned value estimate to make policy updates less noisy, but before the missing update rule is identified there is no general guarantee that the actor is climbing the actual performance objective.

## Evaluation Setting

The relevant setting is on-policy sampled interaction with no model of the environment. The agent repeatedly collects states, actions, and rewards generated by its own current stochastic controller. Performance is judged by average reward or discounted return, and by whether the parameter sequence behaves like ascent on a well-defined objective.

The comparison class includes greedy value-function control, raw return-weighted stochastic updates, and actor-critic variants that use a learned value estimate. A satisfactory rule must explain both what state weighting is being sampled and what role, if any, the learned value estimate is allowed to play without biasing the controller update.
