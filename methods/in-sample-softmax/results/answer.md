# In-Sample Softmax / In-Sample Actor-Critic

## Problem

Offline RL learns from a fixed dataset `D = {(s,a,r,s')}` generated by an
unknown behavior policy `pi_D`. Standard Bellman targets can bootstrap from
unsupported actions:

```text
r(s,a) + gamma max_{a'} q(s',a').
```

If an unsupported action has an extrapolated high value, the target overestimates
and the error propagates. The goal is to improve beyond a possibly suboptimal
behavior policy while keeping the backup and extracted actor tied to the data's
action support.

## Core Operator

Replace the hard support-constrained max with a support-constrained log-sum-exp:

```text
F_{beta,tau}(q) = tau log sum_{a: beta(a)>0} exp(q(a)/tau).
```

The in-sample soft Bellman optimality operator is

```text
(T^*_beta q)(s,a)
  = r(s,a) + gamma E_{s'|s,a}[F_{beta(.|s'),tau}(q(s',.))].
```

As `tau -> 0`, `F_{beta,tau}(q)` approaches
`max_{a: beta(a)>0} q(a)`, so the fixed point approaches the hard
support-constrained optimum.

## Sampling Identity

For `beta = pi_D`,

```text
sum_{a: pi_D(a|s)>0} exp(q(s,a)/tau)
  = E_{a~pi_D(.|s)}
      [exp(q(s,a)/tau - log pi_D(a|s))].
```

The expectation uses dataset actions only. An estimated behavior density
`pi_omega` is needed only to score observed actions through `-log pi_omega(a|s)`;
it is not used to generate candidate actions whose support must be trusted.

## Greedy Policy

The maximum-entropy identity on the restricted simplex is

```text
max_{pi <= beta} pi . q + tau H(pi) = F_{beta,tau}(q),
```

with greedy policy

```text
f_{beta,tau}(q)(a)
  = beta(a) exp(q(a)/tau - log beta(a))
    / sum_{b: beta(b)>0} exp(q(b)/tau).
```

Thus

```text
f_{beta,tau}(q)(a) proportional exp(q(a)/tau)  if beta(a)>0,
f_{beta,tau}(q)(a) = 0                         if beta(a)=0.
```

The `-log beta` term cancels the behavior-probability multiplier. This differs
from behavior-regularized improvement, `pi'(a) proportional beta(a) exp(q(a)/tau)`,
which inherits the behavior policy's action-frequency skew.

## Convergence

`F_{beta,tau}` is a sup-norm non-expansion. If `pi1` maximizes
`F_{beta,tau}(q1)`, then

```text
F(q1) - F(q2)
  <= pi1 . q1 + tau H(pi1) - (pi1 . q2 + tau H(pi1))
   = pi1 . (q1 - q2)
  <= max_{a: beta(a)>0} (q1(a)-q2(a))
  <= ||q1-q2||_infty.
```

Swapping `q1,q2` gives `|F(q1)-F(q2)| <= ||q1-q2||_infty`. Therefore

```text
||T^*_beta q1 - T^*_beta q2||_infty <= gamma ||q1-q2||_infty,
```

so for `gamma < 1` the fixed point is unique and value iteration converges.

Soft policy improvement also stays supported. For `pi <= beta`, define

```text
pi'(a|s) proportional beta(a|s)
  exp(q_tilde^pi(s,a)/tau - log beta(a|s)).
```

The restricted maximum-entropy identity gives

```text
E_{pi'}[q_tilde^pi - tau log pi']
  >= E_pi[q_tilde^pi - tau log pi]
```

at every state. Hence `q_tilde^pi = T^pi q_tilde^pi <= T^{pi'} q_tilde^pi`.
Monotonicity and contraction of `T^{pi'}` yield
`q_tilde^pi <= q_tilde^{pi'}`. Repeating evaluation and greedification converges
to the in-sample softmax optimal policy.

## InAC Losses

InAC uses an SAC-style actor-critic with a behavior model:

```text
pi_psi      stochastic actor
q_theta     double critic, using min(q1,q2)
v_phi       scalar soft value / normalizer estimate
pi_omega    behavior density fit by maximum likelihood
```

Behavior model:

```text
L_behavior(omega) = -E_{(s,a)~D}[log pi_omega(a|s)].
```

Actor as forward-KL projection onto the supported soft greedy target:

```text
L_actor(psi) =
  -E_{(s,a)~D}[
      exp((q_theta(s,a) - v_phi(s))/tau - log pi_omega(a|s))
      log pi_psi(a|s)
    ].
```

The forward KL is essential because it rewrites the gradient as an expectation
over dataset actions. The value `v_phi(s)` stands in for the normalizer
`Z(s) = tau log E_{a~pi_D}[exp(q_theta(s,a)/tau - log pi_omega(a|s))]`, which is
the soft value of the supported greedy policy.

Value target:

```text
L_value(phi) =
  E_{s~D, a~pi_psi(.|s)}
    [0.5 (v_phi(s) - (min_i q_theta_i(s,a) - tau log pi_psi(a|s)))^2].
```

Critic target, as a clean objective:

```text
L_critic(theta) =
  E_{(s,a,r,s')~D}[0.5 (r + gamma v_phi(s') - q_theta(s,a))^2].
```

The canonical training loop computes that next-state soft value directly in the
SAC style, using a sampled next action and the target critic:

```text
r + gamma (1-done) (min_i q_target_i(s',a') - tau log pi_psi(a'|s')),
a' ~ pi_psi(.|s').
```

Both forms avoid a free hard max over next-state actions. The actor projection is
what pulls `pi_psi` toward the dataset-supported greedy policy.

## Code

```python
import torch

class InSampleAC:
    def __init__(self, pi, q, q_target, v, beh, tau, gamma=0.99,
                 polyak=0.995, eps=1e-8, exp_clip=10000.0):
        self.pi = pi
        self.q = q
        self.q_t = q_target
        self.v = v
        self.beh = beh
        self.tau = tau
        self.gamma = gamma
        self.polyak = polyak
        self.eps = eps
        self.exp_clip = exp_clip

    def loss_behavior(self, s, a):
        return -self.beh.get_logprob(s, a).mean()

    def loss_value(self, s):
        v_phi = self.v(s).squeeze(-1)
        with torch.no_grad():
            a_pi, logp_pi = self.pi(s)
            target = self.q_t.min(s, a_pi) - self.tau * logp_pi
        return (0.5 * (v_phi - target) ** 2).mean()

    def loss_critic(self, s, a, r, s2, done):
        with torch.no_grad():
            a2, logp2 = self.pi(s2)
            soft_v2 = self.q_t.min(s2, a2) - self.tau * logp2
            target = r + self.gamma * (1.0 - done) * soft_v2
        q1, q2 = self.q(s, a)
        loss1 = (0.5 * (target - q1.squeeze(-1)) ** 2).mean()
        loss2 = (0.5 * (target - q2.squeeze(-1)) ** 2).mean()
        return 0.5 * (loss1 + loss2)

    def loss_actor(self, s, a):
        logp = self.pi.get_logprob(s, a)
        with torch.no_grad():
            min_q = self.q.min(s, a)
            value = self.v(s).squeeze(-1)
            beh_logp = self.beh.get_logprob(s, a)
            weight = torch.exp((min_q - value) / self.tau - beh_logp)
            weight = torch.clip(weight, self.eps, self.exp_clip)
        return -(weight * logp).mean()

    def sync_target(self):
        with torch.no_grad():
            for p_t, p in zip(self.q_t.parameters(), self.q.parameters()):
                p_t.data.mul_(self.polyak)
                p_t.data.add_((1.0 - self.polyak) * p.data)
```

The released implementation uses two 256-unit hidden layers, Adam with learning
rate `3e-4`, batch size `256`, Polyak factor `0.995`, a double critic, a scalar
value network, a behavior policy of the same class as the actor, and clipped
actor weights with upper threshold `10000`.
