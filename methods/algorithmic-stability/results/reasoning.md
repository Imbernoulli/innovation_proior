I start from the failure mode of empirical risk. The predictor is chosen by looking at the sample, so the same sample cannot be treated as an independent test set. If I try to solve this by taking a supremum over the whole hypothesis class, I get a valid route, but I also throw away information about how the algorithm actually selects one predictor. A regularized algorithm, for example, may search an enormous space while still being hard to move.

So I focus on the learning map itself. I imagine changing one coordinate of `S` and asking whether the losses of the two resulting predictors are almost the same on every possible test point. If the answer is yes, then no individual observation has much leverage over the final predictor. This feels like the quantity that should matter for overfitting, because the empirical loss becomes too optimistic precisely when the learner can exploit idiosyncrasies of its own sample.

The first tempting definition is to compare the two output hypotheses directly, but that is too tied to the representation of the hypothesis space. I only need the output through the loss. I therefore measure the worst-case loss change after deleting one point: `sup_z |ell(A(S), z) - ell(A(S \ i), z)|`. If this is at most `beta`, I can compare the full-sample predictor with the leave-one-out predictor on any example. Replacement is not a separate conceptual obstacle, because replacing one point can be decomposed through the common sample with that point deleted. That costs at most a factor of two.

Now I need to see why this sensitivity statement controls the bias of empirical risk. Pick a training point `z_i`. The dangerous term is `ell(A(S), z_i)`, since the model was trained on `z_i`. But `A(S \ i)` was not trained on `z_i`, so by exchangeability the loss of `A(S \ i)` on `z_i` behaves like a fresh test loss for that leave-one-out model. Stability lets me replace `A(S)` by `A(S \ i)` while paying only `beta`. Averaging over `i` turns this into a small expected gap between true risk and the empirical or leave-one-out estimate.

Expectation is not enough. I need the gap to concentrate for a random sample. I view the gap as a function of the independent coordinates of `S`. If one coordinate changes, the true-risk part changes only because the learned predictor changes, and the stability bound controls that. The empirical average changes for the same reason on almost all summands, plus the changed summand itself, which is controlled by the bounded loss assumption. This is exactly the bounded-differences setting. Applying McDiarmid's inequality gives a high-probability bound around the small expectation.

This produces the method: prove a uniform loss-sensitivity bound for the algorithm, then plug it into the concentration theorem. The shape of the final bound makes sense. There is a direct stability term, because the expectation is biased by sample reuse, and there is a sampling-fluctuation term whose bounded-difference constants are `4 beta + M/m`, equivalently displayed after McDiarmid as `(4 m beta + M) sqrt(log(1/delta)/(2m))`. If `beta` scales like `1/m`, both parts vanish with growing sample size.

I then ask which algorithms can actually satisfy such a strong condition. Unregularized empirical-risk minimization can jump between nearly tied minimizers, so a single point may flip the selected hypothesis. A regularized objective is different. Removing one loss term perturbs the average loss by one part in `m`, and the penalty supplies curvature or a norm barrier. In an RKHS, bounded kernels and an admissible loss let me turn a movement bound in function norm into a uniform loss-change bound, giving `beta` on the order of `1 / (lambda m)`.

That also prevents me from overstating the result. A predictor can be stable and still bad if the regularizer forces it toward a poor function. The certificate controls overfitting, not approximation error. The final picture is a tradeoff: larger `lambda` makes the algorithm less sensitive to one example, while smaller `lambda` can fit the sample more closely but weakens the sensitivity certificate.

The insight is therefore to replace a global question about all functions in a class with a local question about the actual algorithm. Generalization follows when the learning rule cannot be moved much by one observation, because then evaluating on the training sample is almost like evaluating on points that were not responsible for the fitted predictor.
