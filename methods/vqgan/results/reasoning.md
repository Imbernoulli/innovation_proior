OK, let me think this through from the pressure point rather than from an architecture name. I want high-resolution images that are globally coherent. Local texture alone is not enough; a model can make every small patch look plausible and still fail the large-scale arrangement. So I need a mechanism that can relate distant parts of the image while still being affordable at image scale.

The obvious global mechanism is self-attention. A transformer forms `softmax(QK^T / sqrt(d_k))V`, so every position can directly compare itself to every other position. For autoregressive generation I mask future positions, then predict `p(x_i | x_{<i})`. That is exactly the global dependency structure I want. But if the tokens are pixels, the sequence length is `H W`, and the attention matrix is `(H W) x (H W)`. Doubling the side length quadruples the tokens and roughly sixteen-tuples the pairwise attention work. Pixel-space transformers run into the wall long before megapixel images.

Restricting attention locally weakens the reason I wanted the transformer. Sparse or axial patterns help but still leave sequence length as the central problem. So the route is not to make a pixel transformer barely cheaper; the route is to stop giving pixels to the transformer.

Images have local redundancy, and convolutional networks are good at exploiting it. They are cheap over pixels because kernels are local and shared. Their weakness is the global arrangement; their strength is local processing. That suggests the split: use a convolutional encoder/decoder to turn an image into a much shorter spatial grid, then use a transformer to model the composition of the grid entries. The transformer pays quadratic cost on the short grid, not on the pixels.

For the transformer prior, I want the grid entries to be discrete tokens. A finite vocabulary gives a categorical next-token distribution and a normal cross-entropy objective. So I start with vector quantization. The encoder maps `x` to `z = E(x) in R^{h x w x n_z}`. A learned codebook `Z = {z_k}_{k=1}^K` stores prototype vectors. At each spatial location I choose the nearest prototype, giving `z_q = q(z)`, and the decoder reconstructs `x_hat = G(z_q)`. The same quantized grid can be read as indices `s_ij = k` whenever `(z_q)_ij = z_k`.

The nearest-neighbor assignment is piecewise constant, so the derivative through the argmin is zero almost everywhere. I use the straight-through estimator: forward pass uses the hard quantized vector, backward pass treats quantization as the identity. In code this is `z_q = z + sg[z_q - z]`; the bracket is frozen, so the gradient with respect to `z` is one while the forward value remains the selected code.

The straight-through copy sends the reconstruction gradient around the codebook, so the codebook needs its own term. The VQ loss has a codebook pull `||sg[E(x)] - z_q||^2`, which updates the selected prototype toward the encoder output, and a commitment pull `||sg[z_q] - E(x)||^2`, which updates the encoder output toward the selected prototype. The original VQ-VAE writes a beta on the commitment term. Here I do not need a separate tuned commitment weight in the final objective, so I use unit codebook and commitment terms.

At this point I have a two-stage plan, but the first stage can still fail. The transformer needs the sequence to be short, so the encoder must compress hard. If a `256 x 256` image becomes a `16 x 16` latent grid, each token must summarize a fairly large image region. A pixel `L2` reconstruction loss asks for the conditional mean under residual uncertainty, and the conditional mean of plausible high-frequency textures is blur. The transformer could learn the code distribution perfectly and still decode to blurry images if the codebook was trained that way.

So the first-stage objective has to stop rewarding pixel averages as the main notion of fidelity. A perceptual feature loss is the first fix: compare `x` and `x_hat` in a fixed deep feature space such as LPIPS/VGG, with a small pixel `L1` term for anchoring. This makes blur costly in a way raw pixel error often does not. But perceptual distance alone can still leave local texture slightly soft, so I add an adversarial signal.

The discriminator should be patch-based. The missing detail under heavy compression is mostly local texture and high-frequency structure. A fully convolutional patch discriminator gives a grid of real/fake logits, one per receptive field, rather than one scalar for the whole image. That gives dense local feedback, works at arbitrary image size, and fits the division of labor: the first stage should make local parts perceptually convincing, while the transformer later models how those parts compose globally.

The adversarial game is `L_GAN = log D(x) + log(1 - D(x_hat))`, with the discriminator maximizing and the autoencoder side minimizing through the generated image. In implementation I should use the steadier hinge surrogate: the discriminator minimizes `0.5 * (E relu(1 - D(x)) + E relu(1 + D(x_hat)))`, and the generator minimizes `-E D(x_hat)`.

Now the coefficient on the adversarial term is not a harmless detail. The reconstruction loss and GAN loss have different scales, and those scales move during training. A fixed `lambda` can be too weak early or too strong later. I want the adversarial gradient arriving at the decoder to be comparable to the reconstruction gradient, so I measure both at the decoder's last layer `G_L`, where they meet. The adaptive weight is `lambda = ||grad_{G_L} L_rec|| / (||grad_{G_L} L_GAN|| + delta)`. I set `delta = 10^-6` in the mathematical objective; in a practical implementation I can use a larger denominator guard such as `1e-4`, clamp the ratio to `[0, 1e4]`, detach it, and multiply by a base discriminator weight.

Even with the adaptive ratio, I do not want the discriminator active against a random decoder and random codebook. I gate the discriminator weight to zero at the beginning, train the encoder, decoder, and codebook as a perceptual reconstructor first, then turn the adversarial term on. A warm-up of at least one epoch is the practical rule.

The architecture should keep the same division of labor. The encoder and decoder are convolutional ResNet-style stacks with downsampling and upsampling. The compression factor is `f = 2^m`, so the latent grid is `H/f x W/f`. A single self-attention block at the lowest resolution is affordable and lets the autoencoder aggregate global context before choosing codes or decoding them. One `1 x 1` convolution maps encoder channels to the codebook dimension before quantization, and another maps code vectors back to decoder channels after quantization.

With the first stage trained, I freeze it and train the second stage. Each image becomes an index sequence `s`. The factorization is `p(s) = prod_i p(s_i | s_{<i})`, and the loss is cross-entropy, `E[-log p(s)]`. The transformer is a GPT-style decoder: token embedding, learned positional embedding, masked self-attention blocks, MLPs, final layer norm, and a linear head over the codebook.

I still have to choose an order for the 2D code grid. There is no linguistic left-to-right order for images. I could try spiral, Morton/Z-curve, coarse-to-fine subsampling, or alternating rows. But sliding-window generation later wants a row-major scan, and row-major gives a stable "above and left are already known" context. The ordering is not mathematically invariant for next-token prediction, so I should treat row-major as a design choice, not a free convention. The empirical ablation supports it: row-major is the best of the tested orders.

Conditioning fits the same decoder-only interface. If the condition is a class label, it can be a one-token prefix. If there is no condition, I use a learned or fixed start token prefix. If the condition is spatial, I convert it to condition tokens, either with a condition-stage quantizer or a deterministic token builder, then prepend those condition tokens to the image-code sequence. The transformer sees `[condition, previous image codes]`; the loss is only on the image-code targets.

The off-by-one alignment matters. If the condition prefix length is `c_len`, I feed all but the last concatenated token. The output at position `c_len - 1` has seen the whole condition prefix and predicts the first image code. Therefore the image logits start at `c_len - 1`, and the targets are exactly the image indices. This case works for a one-token class or SOS prefix and for longer spatial prefixes.

Sampling is then simple for normal-size latent grids: start from the condition prefix, repeatedly predict the next token, divide logits by temperature, optionally truncate to top-k or top-p, sample or take argmax, append the token, and finally map indices back to codebook vectors and decode with the frozen decoder. The common setting is temperature `1.0` and top-k around `100`, with larger top-k sometimes used for larger or class-conditional codebooks.

Megapixel generation needs one more constraint. Even after compression, a very large latent grid can exceed the transformer's block size. I cannot keep increasing the compression factor forever because reconstruction quality collapses past a dataset-dependent point. So I train on latent crops that fit the transformer and sample large grids with a sliding attention window. For each raster position, I take a fixed `16 x 16` latent window around the current location, include the already generated codes above and to the left, prepend the matching condition patch when present, predict the current local position, and write that sampled code into the full grid. If the data are aligned and unconditional, coordinate conditioning is needed so local windows know where they sit on the canvas.

So the whole method is a clean two-stage compromise. The convolutional quantized autoencoder learns a perceptually rich, high-compression vocabulary of image parts using LPIPS, a small pixel term, a patch discriminator, warm-up, and an adaptive adversarial weight. The autoregressive transformer models the global composition of those parts as a short code sequence, with condition tokens as a prefix and row-major order for standard and sliding-window sampling. The transformer never models pixels; the decoder never has to learn the global prior.
