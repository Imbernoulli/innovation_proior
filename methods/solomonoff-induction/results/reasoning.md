I start with the ordinary Bayesian picture because it explains both the strength and the problem Solomonoff wanted to remove. Bayesian prediction says that I should list hypotheses, give them prior probabilities, update by likelihood, and predict by posterior averaging. That is a coherent calculus once the hypothesis space is fixed. But the fixed hypothesis space is doing enormous work. If I choose polynomials, hidden Markov models, neural networks, or a finite set of experts, then everything outside that choice has probability zero. No amount of data can recover a true explanation that I excluded at the beginning.

The natural way to make the hypothesis space less arbitrary is not to ask for a cleverer hand-built family, but to ask what the largest mathematically controlled family could be. Computability gives that boundary. Any effective explanation of a sequence can be represented by a program for a universal machine. A law of nature, a simulation, a stochastic sampler with computable probabilities, or a compressed rule can all be encoded as programs. So the universal hypothesis class is not "all imaginable explanations" in an unrestricted metaphysical sense; it is all computable explanations, which is exactly the range that can be handled by the theory of algorithms.

Once explanations are programs, Occam's razor becomes a coding rule rather than a slogan. A shorter program gets weight `2^{-|p|}`. That choice is not an arbitrary penalty tacked onto a loss function; it is the natural probability mass of prefix-free code space. One extra bit of description costs a factor of two. A simple law is favored because it occupies a shorter description, while a complicated special-purpose explanation remains possible but receives less prior mass.

The next important step is that Solomonoff induction does not simply select the shortest program consistent with the data. If the observed prefix is `x`, every program that outputs a sequence beginning with `x` contributes to the mass `M(x)`. This summation matters. A shortest-program rule would be a hard model-selection principle; Solomonoff's rule is a Bayesian mixture. It keeps all computable explanations alive and lets their total posterior weight determine the prediction.

The universal semimeasure makes this precise. For a universal prefix machine `U`, define `M_U(x)` as the sum of `2^{-|p|}` over all programs `p` whose output starts with `x`. If I want to predict a continuation `y` after observing `x`, I compare `M_U(xy)` with `M_U(x)`. That is the Bayesian update in algorithmic form: prior mass is assigned to programs before the data, observations filter the programs by consistency or likelihood, and future predictions are averaged over the remaining mass.

This is the central conceptual shift. In ordinary modeling I first decide the form of the possible explanations and then do induction inside that form. In Solomonoff induction the form is supplied by computability itself. The prior does not say "consider these ten models" or "consider this parametric family." It says that any computable process capable of producing the data is part of the mixture, with its influence discounted by the length of the program that describes it.

That explains why the theory unifies three ideas that usually live separately. It is Bayesian because prediction is posterior weighting. It is Occamian because shorter descriptions get exponentially larger prior mass. It is computational because the hypotheses are programs and the universality claim is tied to universal Turing machines and algorithmic information theory. The insight is not merely "simpler is better"; it is that simplicity can be measured by program length inside a universal Bayesian prior.

There is also an unavoidable cost. Exact Solomonoff induction is incomputable. To know `M_U(x)` exactly, I would need to know which programs eventually output strings beginning with `x`, and that cannot be decided in general. This does not make the theory meaningless, but it changes its role. It is not a recipe for an immediately runnable predictor. It is an ideal standard against which computable approximations, compression methods, model selection rules, and universal prediction schemes can be compared.

The machine-dependence issue is real but bounded in the usual algorithmic-information sense. A different universal machine changes program lengths by a compiler constant, so prior weights change by multiplicative constants rather than by a data-dependent refitting of the language. The reference machine must still be chosen before observing the data. That requirement is not a detail; it is what prevents the universal prior from becoming a disguised way to encode the answer after the fact.

So the final lesson is that Solomonoff induction turns the problem of induction from "which models should I manually admit?" into "how should I weight all computable explanations?" The answer is: use a universal machine, assign each program length-based prior weight, sum over all programs compatible with the observations, and condition that universal mixture for prediction. This is why it is best understood as an ideal theory of induction rather than just another model-selection criterion.
