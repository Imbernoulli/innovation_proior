I start with the ordinary Bayesian picture because it explains both the strength and the problem I am trying to remove. Bayesian prediction says that I should list hypotheses, give them prior probabilities, update by likelihood, and predict by posterior averaging. That is a coherent calculus once the hypothesis space is fixed. But the fixed hypothesis space is doing enormous work. If I choose polynomials, hidden Markov models, neural networks, or a finite set of experts, then everything outside that choice has probability zero. No amount of data can recover a true explanation that I excluded at the beginning. So the part I distrust is not the updating rule; it is the act of writing down the hypothesis list by hand.

The natural way to make the hypothesis space less arbitrary is not to ask for a cleverer hand-built family, but to ask what the largest mathematically controlled family could be. Computability gives that boundary. Any effective explanation of a sequence can be represented by a program for a universal machine. A law of nature, a simulation, a stochastic sampler with computable probabilities, or a compressed rule can all be encoded as programs. So the candidate hypothesis class is not "all imaginable explanations" in an unrestricted metaphysical sense; it is all computable explanations, which is exactly the range the theory of algorithms can handle. That at least replaces an arbitrary choice with a single principled one.

But "put a prior over all programs" is not yet a prior. I need actual numbers `w(p)` summing to something finite, and I want shorter programs to count for more, because that is the only honest way to import Occam's razor. The obvious guess is `w(p) = 2^{-|p|}`. Before I lean on it I should check that it can even be a prior, because if the weights summed to infinity the whole construction would be empty.

Here I have to be careful, and this is where I nearly went wrong. If I let `p` range over *all* binary strings, then the strings of length `n` number `2^n`, each contributing `2^{-n}`, so every length contributes `1` and the total diverges. So the raw guess does not sum. The fix is that the programs must form a prefix-free set: no valid program is a prefix of another, which is what a self-delimiting (prefix) machine enforces, since the machine reads code until it decides to halt and start executing. Let me verify on a concrete prefix-free set that this actually rescues summability. Take codewords `{0, 10, 110, 111}`. No one is a prefix of another, and the weights are `2^{-1} + 2^{-2} + 2^{-3} + 2^{-3} = 1/2 + 1/4 + 1/8 + 1/8 = 1`. So on a prefix-free set the masses sum to exactly `1`, not to infinity. If I instead allow a non-prefix-free multiset like `{0, 0, 0}` I get `3 * 1/2 = 3/2 > 1`, which is the divergence reappearing. So the prefix condition is not decoration; it is precisely the structural fact (Kraft's inequality) that makes `2^{-|p|}` a legitimate probability assignment over programs. Good — the weighting scheme survives the first sanity check.

Now the design question that decides everything: given the data, do I keep the single best program, or do I keep all of them? The single-best route is tempting because it is concrete and it is just Kolmogorov complexity — assign `x` the mass `2^{-K(x)}` where `K(x)` is the length of the shortest program that outputs `x`. It is clean and it directly says "simplest explanation wins." So before committing to a full sum, I want to see whether the full mixture is actually different from `2^{-K}` or just a constant multiple of it that I could ignore.

Let me build the smallest example that can tell them apart. Take a toy machine with three programs and the infinite stream each emits: `p0` of length 1 emits `000…`, `p1` of length 2 emits `111…`, `p2` of length 3 emits the alternating `0101…`. Define `M(x)` as the sum of `2^{-|p|}` over programs whose output starts with `x`. Suppose I observe the single bit `x = 0`. The programs consistent with `0` are `p0` (weight `1/2`) and `p2` (weight `1/8`), so `M(0) = 1/2 + 1/8 = 5/8`. The shortest-program mass is just `2^{-K(0)} = 1/2`, since the shortest consistent program is `p0` of length 1. The ratio is `(5/8)/(1/2) = 1.25` — bounded, as the coding theorem promises in general, so on the *mass* the two objects really are within a constant.

If that were the whole story I would take the shortest program and save myself an infinite sum. But I care about prediction, so I have to compare what each rule does for the *next bit*, not just the mass of `x`. Conditioning the mixture: `M(00) = 1/2` (only `p0` produces `00`) and `M(01) = 1/8` (only `p2` produces `01`), so `P(next=0 | 0) = (1/2)/(5/8) = 0.8` and `P(next=1 | 0) = (1/8)/(5/8) = 0.2`. The shortest-program rule, by contrast, commits entirely to `p0` and predicts `P(next=1 | 0) = 0`. So the two rules agree up to a constant on the mass but give genuinely different predictions: `0.2` versus `0` for the next bit. The single-best rule throws away a live computable explanation and turns a soft posterior into a categorical one. That is the concrete reason to reject it — not taste, but a worked case where it assigns probability zero to a continuation the evidence has not excluded. The mixture keeps the alternative alive with whatever weight its description length earns.

The same example shows that the sum is not just hedging — it sharpens correctly when the data warrants. Observe one more bit, `x = 01`. Now `p0` (output `00…`) is inconsistent and drops out; only `p2` survives, with `M(01) = 1/8`. Then `M(010) = 1/8` and `M(011) = 0`, so `P(next=0 | 01) = 1.0` and `P(next=1 | 01) = 0.0`. The mixture has filtered the hypotheses by consistency and concentrated on the one surviving program, exactly as Bayesian updating should: prior mass assigned by length, observations remove the incompatible programs, prediction averages over what remains. So the summation is the Bayesian-mixture machinery, and conditioning `M(xy)` against `M(x)` is the update rule in algorithmic form.

So the object I want is the universal semimeasure: for a universal prefix machine `U`, `M_U(x) = sum_{p: U(p) starts with x} 2^{-|p|}`, with prediction of a continuation `y` after observing `x` given by comparing `M_U(xy)` to `M_U(x)`. Reading it back against the three ingredients I started from: it is Bayesian because prediction is posterior weighting over the surviving programs; it is Occamian because the prefix-free `2^{-|p|}` weighting (which I checked sums correctly) gives exponentially more mass to shorter descriptions; and it is computational because the hypotheses are programs for a universal machine. Simplicity is no longer a slogan but the measured quantity "program length," sitting inside an honest prior.

There is a cost I should not paper over. Computing `M_U(x)` exactly would require knowing which programs eventually output strings beginning with `x`, and that is undecidable in general — the halting problem is hiding inside the summation. My toy machine dodged this only because I fixed three programs with known outputs; the real `M_U` ranges over all of them. So this is not an immediately runnable predictor. Its role is as an ideal standard against which computable approximations, compression methods, model-selection rules, and universal prediction schemes can be measured.

One more thing I should pin down rather than assert: how much does the choice of universal machine `U` matter, since I had to fix one? For another universal prefix machine `V`, there is a fixed compiler `q` (a program for `U` that simulates `V`) of some constant length `c = |q|`. Any `V`-program `p` becomes a `U`-program `q p` of length `|p| + c`, so `M_U(x) >= 2^{-c} M_V(x)`, and symmetrically with `U` and `V` swapped. The two universal semimeasures therefore differ only by multiplicative constants `2^{-c}` that do not depend on the data — exactly the kind of constant I already saw between the mixture and `2^{-K}` in the toy example, now playing the role of language invariance. So the machine dependence is real but bounded: it cannot be refit after seeing the data, which is precisely why `U` must be fixed in advance. If I were allowed to choose `U` after the fact I could smuggle the answer into the language, and the constant would no longer be a constant.

So the lesson is that induction can be reframed from "which models should I manually admit?" into "how should I weight all computable explanations?" The answer the construction forces on me: fix a universal machine before the data, assign each program the prior weight `2^{-|p|}` (which is summable exactly because the programs are prefix-free), sum over all programs compatible with the observations, and condition that universal mixture for prediction. The worked example is what convinced me the sum, not the single shortest program, is the right object, and it is why this is best understood as an ideal theory of induction rather than just another model-selection criterion.
