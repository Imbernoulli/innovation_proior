I want the seller's best auction, but I cannot start by picking a familiar format. A second-price auction is truthful because the allocation rule is monotone and the payment is the bid at which the winner would just stop winning. A posted price is revenue-aware because it sometimes refuses to sell. Those are useful signs, but if I simply bolt a reserve onto a second-price auction I am guessing the answer before I know the constraint that makes it optimal. The real object is a direct mechanism: each bidder reports a value, the rule chooses who gets the item, and the rule charges payments. The seller wants expected payments. The bidders care about value times allocation minus payment. Truthfulness and participation are the walls.

Fix one bidder and freeze everything else. In the ex post version I literally hold the other bids fixed; in the Bayesian version I take expectations over them and look at the bidder's interim allocation. Either way the bidder sees a one-dimensional allocation curve q(z), the chance of service if it reports z, and an expected payment P(z). If the bidder's true value is v, truthful utility is U(v)=v q(v)-P(v). Incentive compatibility says that reporting v must beat reporting z, so U(v) >= v q(z)-P(z). It also says that a type z must not want to report v, so U(z) >= z q(v)-P(v).

Let v>z and rewrite the two inequalities in terms of U(v)-U(z). From the first, U(v)-U(z) >= (v-z)q(z). From the second, U(v)-U(z) <= (v-z)q(v). These two inequalities can both hold for all v>z only if q(v) >= q(z). So monotonicity is not a decorative property of nice auctions; it is forced by IC. That already collapses a large part of the design problem. I do not need to search over arbitrary allocation rules. I only get monotone allocation curves.

Now suppose q is monotone. The same inequalities say that the slope of U, wherever the derivative exists, is trapped between q just below and q just above. In the smooth case U'(v)=q(v). In the general monotone case the integral version is the safe one: U(v)=U(0)+int_0^v q(t) dt. Then the payment is no longer free:

P(v)=v q(v)-U(v)=v q(v)-U(0)-int_0^v q(t) dt.

This is the payment identity. If q jumps from 0 to 1 at a threshold r, the formula charges r to every winning type. If q has several jumps, the payment is the sum of threshold times jump size. Payments are the area above the allocation curve, and utility is the area under it. That is the envelope theorem doing the work. The transfer side has disappeared except for U(0).

Individual rationality tells me what to do with U(0). Since U'(v)=q(v) >= 0, the lowest type has the smallest utility. If I set U(0) >= 0 then all higher types are interim IR. But expected revenue falls one-for-one with U(0), so the seller sets U(0)=0. Now every implementable allocation rule has a unique revenue-maximizing payment rule. The whole problem is allocation, plus monotonicity.

Take bidder i with value V drawn from F with density f. Expected payment is E[P(V)] = E[V q(V)] - E[int_0^V q(t) dt], with U(0)=0. The second term is the part I need to unfold. Swap the order of integration:

E[int_0^V q(t) dt] = int_0^infty q(t) Pr[V>=t] dt = int_0^infty q(t)(1-F(t)) dt.

The first term is int v q(v) f(v) dv. Put them under the same integral:

E[P(V)] = int q(v)[v f(v) - (1-F(v))] dv
        = E[q(V)(V - (1-F(V))/f(V))].

There it is: the bidder's contribution to expected revenue is not its value times allocation probability, but its allocation probability times phi(v)=v-(1-F(v))/f(v). The deduction did not recombine known auction formats. It came from IC reducing payments to an envelope identity, then integration by parts turning expected payments into expected virtual surplus.

For several independent bidders, expected revenue sums across bidders, so it becomes E[sum_i x_i(v) phi_i(v_i)], assuming the lowest utilities are set to zero. Feasibility still says I can allocate the item to at most one bidder. If the allocation rule I get by maximizing sum_i x_i phi_i(v_i) is monotone in each bidder's value, then I am done: choose the feasible allocation with the largest nonnegative virtual value, and recover payments from the identity. Negative virtual value means serving that type reduces expected revenue, so the item should be withheld when every feasible winner has negative virtual value.

Regularity is exactly the condition that makes this pointwise rule compatible with incentives. If phi_i(v) is nondecreasing in v, then raising a bid raises that bidder's virtual score. For a single item, assigning the object to the bidder with the highest nonnegative virtual value is monotone: a bidder who wins cannot lose by increasing its report, because its score only rises while others' scores stay fixed. The payments are critical values in real-value space. A bidder pays the smallest report at which its virtual value would still beat the relevant competing virtual value and stay nonnegative.

For identical regular bidders, the virtual transformation preserves the bid ranking, so the highest bid wins when its virtual value is at least zero. The cutoff r satisfies phi(r)=0, or r=(1-F(r))/f(r). That is the monopoly reserve. The mechanism is a second-price auction with reserve r, but that format appears only after the derivation. The reserve is not a patch added to Vickrey; it is the point where the bidder's marginal contribution to expected revenue changes sign.

If bidders have different regular distributions, ranking by raw bids can be wrong. A lower value from a thinner upper tail can have a higher virtual value than a higher value from a distribution where high reports require more information rent. The allocation rule ranks virtual values, not bids. The critical payment also lives in the original value scale: convert the virtual threshold back through phi_i, and charge the corresponding value threshold.

The remaining wall is irregularity. If phi falls somewhere, then pointwise virtual-surplus maximization can tell a bidder that increasing its value makes it less likely to win. No payment rule can rescue that. I cannot ignore monotonicity, because monotonicity was the IC constraint in disguise. I need to change the objective only where it asks for a nonmonotone allocation.

Quantile space makes the repair visible. Write q=1-F(v), so smaller q means a stronger type, and let v(q) be the inverse demand curve. A posted price serving the top q fraction earns R(q)=q v(q). The derivative of R with respect to q is the marginal revenue of serving an additional quantile, which is the virtual value in quantile coordinates. If R is concave, marginal revenue is nonincreasing as I move to weaker quantiles, which corresponds to virtual value increasing with v. If R is not concave, the marginal revenue slopes move the wrong way and the allocation that follows them is nonmonotone.

The least invasive fix is to replace R by its concave envelope. On intervals where the original revenue curve lies below a chord, the envelope has a constant slope. Translating back, all types in that interval receive the same ironed virtual value. Maximizing ironed virtual surplus can therefore bunch those types: they are treated as tied for allocation purposes, with randomization or tie-breaking chosen so the resulting allocation probability is monotone across the interval. The seller gives up the impossible attempt to distinguish types in a region where doing so would violate IC, but keeps the best concave revenue frontier available under monotonicity.

So the final design is clear. Convert IC into monotone allocation plus the payment identity. Set the lowest utility to zero for IR and revenue. Rewrite expected payments as expected virtual surplus using phi(v)=v-(1-F(v))/f(v). If distributions are regular, maximize virtual surplus pointwise and charge threshold payments. If they are irregular, iron the revenue curve, maximize ironed virtual surplus pointwise, bunch types on ironed intervals, and again charge the payments forced by the allocation identity. Optimal auction design is not a recombination of second price, reserves, and posted prices. Those formats are shadows cast by the same reduction: after incentives are absorbed into monotonicity and the envelope formula, revenue maximization becomes virtual-surplus maximization.
