Let me start with the representation problem rather than with image generation. A supervised convolutional net learns a useful visual hierarchy because the label task keeps forcing its filters to become predictive, but labels are the scarce part. If I want to learn from the much larger pile of unlabeled images, I need a pressure that still forces a hierarchy of image features to appear. A reconstruction loss is the obvious pressure, but it has the wrong failure mode: when several outputs are plausible, a pixel loss rewards the average, and the average is blur. I do not want the model's main signal to say "reconstruct this exact pixel grid" if the representation I want is semantic and transferable.

The adversarial game gives me a better pressure. I can let a generator map a simple latent variable `z` to an image and let a discriminator decide whether an image is real or generated. The value function is `min_G max_D E_x log D(x) + E_z log(1 - D(G(z)))`. For the discriminator this is ordinary binary cross-entropy: real images get label `1`, generated images get label `0`. For the generator I should use the non-saturating version, so generated images get label `1` and the generator minimizes `-log D(G(z))`. The sign matters: the discriminator is maximizing the displayed value, but the code will minimize binary-cross-entropy losses; the generator update is not minimizing `log(1 - D(G(z)))`, it is maximizing `log D(G(z))`.

Now I have to face the real obstacle. A shallow adversarial model can be written down easily, but a deep convolutional version tends to fall apart. The generator can emit nonsense, or worse, it can collapse many latent points to one image that happens to exploit the current discriminator. The objective alone is not enough. The useful work is to make the convolutional architecture and training settings gentle enough for this moving-target game.

The first place I look is spatial resampling. A discriminator needs to go from an image to a compact decision, so it has to reduce spatial size. A generator needs to go in the opposite direction, so it has to increase spatial size. Fixed max-pooling in the discriminator and fixed resize or unpooling in the generator are hand-chosen operators sitting inside an already brittle game. The all-convolutional idea gives me a cleaner option: replace fixed pooling with learned strided convolutions. Then use the transposed, or fractionally-strided, convolution in the generator. The discriminator learns its own downsampling, and the generator learns its own upsampling. That keeps spatial resampling inside the learned model instead of baking in a fixed image operation.

The second place is the dense head. A large fully connected stack on top of convolutional features adds many parameters and can make the model memorize a brittle mapping rather than stay convolutional. Global average pooling is the extreme fix and it is attractive because it is stable, but it can slow convergence too much. I want the middle ground: no fully connected hidden layers. The latent input may be projected into a small spatial tensor, because the generator has to start from a vector somehow, and the discriminator may flatten its final spatial tensor into one sigmoid output. Between those boundary operations, I keep the networks convolutional.

The third place is normalization. Poor initialization is deadly in a deep generator because bad early activations can stop useful gradients before the model begins to learn. Batch normalization directly attacks that conditioning problem, and empirically it is critical for keeping the generator from collapsing all samples to one point. But applying it everywhere creates a different instability. I need the boundary layers to keep their raw distributional jobs. The generator output layer has to produce pixels in the data range, so batch-normalizing that output would force batch statistics at the exact place where the model should match image statistics. The discriminator input layer has to see raw real and generated pixels, so normalizing that input can remove or distort useful evidence before the discriminator has learned anything. The rule becomes batch normalization in both networks, except the generator output and discriminator input.

The nonlinearities follow the same boundary logic. Inside the generator I want ReLU: it is simple, standard, and gives clean positive-side gradients through a deep stack. At the output, I want a bounded activation because the images will be scaled to `[-1, 1]`; `tanh` lands exactly in that range and lets the model learn saturated color values quickly. In the discriminator, plain ReLU is a risk because dead units also deaden the gradient passed back to the generator. Leaky ReLU with slope `0.2` preserves a small negative-side derivative, so the discriminator can stay informative even when many activations are below zero.

The optimizer has to respect the fact that the loss is moving. Adam is a reasonable choice, but its default learning rate `0.001` is too aggressive for this game, so I lower it to `0.0002`. The first moment coefficient is more subtle. With `beta1 = 0.9`, each update carries a long memory of gradients computed against an opponent that has already changed. That stale momentum shows up as oscillation. Reducing `beta1` to `0.5` shortens the memory and tracks the current opponent more closely. I keep the usual `beta2 = 0.999`.

The initialization has to keep the first updates small. I initialize convolution weights from a zero-mean normal with standard deviation `0.02`. For batch-normalization layers, the natural stable start is scale near `1.0` with the same small standard deviation and shift exactly zero. I can train with minibatches of `128`, but I should keep batch size separate from architecture: the batch determines stochastic-gradient statistics, not the shape of either network.

The latent prior also needs to be stated carefully. I want a simple independent 100-dimensional source, not a structured code that smuggles labels or image geometry into the model. A uniform prior is a clean choice for this setup, and a standard normal prior would leave the architecture unchanged. The important invariant is that the generator learns the map from a simple 100-dimensional noise source to the image manifold; the exact simple prior is less load-bearing than the convolutional topology, normalization placement, and optimizer settings.

Now the geometry has to close exactly. For a transposed convolution with dilation one and no output padding, the spatial output is `(i - 1) * s - 2p + k`. Starting from a `1x1` latent tensor, a kernel `4`, stride `1`, padding `0` gives `4`. Then kernel `4`, stride `2`, padding `1` gives `(4 - 1) * 2 - 2 + 4 = 8`, and repeating gives `16`, `32`, and `64`. The generator therefore goes `100x1x1 -> 8f x 4 x 4 -> 4f x 8 x 8 -> 2f x 16 x 16 -> f x 32 x 32 -> 3 x 64 x 64`, with batchnorm and ReLU after every generated feature map except the final image, where it uses `tanh`.

The discriminator mirrors this arithmetic. For an ordinary convolution, the output is `floor((i + 2p - k) / s) + 1`. With kernel `4`, stride `2`, padding `1`, `64` becomes `32`, then `16`, then `8`, then `4`. A final kernel `4`, stride `1`, padding `0` maps `4` to `1`. So the discriminator goes `3 x 64 x 64 -> f x 32 x 32 -> 2f x 16 x 16 -> 4f x 8 x 8 -> 8f x 4 x 4 -> 1`, with leaky ReLU after every convolution and batchnorm after every convolution except the first input layer. The final sigmoid makes the output a probability for the binary-cross-entropy objective.

Putting these choices together gives a single stable convolutional adversarial pair. The key is not one isolated trick; it is removing the obvious instability sources one by one. Fixed spatial resampling becomes learned strided or transposed convolution. Dense hidden layers disappear. Batch normalization is used where it improves conditioning and withheld where it would corrupt boundary statistics. The generator uses ReLU internally and `tanh` at the pixel boundary. The discriminator uses leaky ReLU so it keeps sending gradients back. Adam is slowed down and given shorter momentum. With that recipe, the discriminator can be reused as a convolutional feature extractor, and the generator's latent space is worth inspecting rather than being only a source of samples.
