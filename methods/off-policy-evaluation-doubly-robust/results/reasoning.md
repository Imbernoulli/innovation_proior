The central move is not to average a model estimate with an importance-sampling estimate. I need the model to enter as a baseline whose target-policy mean I can compute, and I need the logged reward to enter only through a weighted residual around that baseline. Once I see that, the double protection is no longer mysterious: the model error and the propensity error multiply instead of appearing as separate first-order biases.

I start with the one-step off-policy problem because the scientific mechanism is cleanest there. The direct method predicts `q(x,a)` and averages the target-policy prediction. It is stable, but if the prediction is wrong on the target action, the estimate inherits that model error directly. Importance sampling does the opposite. It keeps the observed reward and corrects the action distribution with `pi(a|x) / p(a|x)`, so it can be unbiased when the behavior propensity is right. But it reweights the raw reward, and small behavior probabilities create large random contributions.

The useful question is therefore not "which estimator should I trust?" It is "what is the smallest random quantity I can safely reweight?" If the model is even moderately good, the residual `r - q_hat(x,a)` is smaller than the reward itself. So I want to reweight that residual, not the raw reward. But the residual alone has the wrong mean, so I add back the model's target-policy mean `q_hat(x,pi) = sum_a pi(a|x) q_hat(x,a)`. The one-step contribution becomes `q_hat(x,pi) + iw * (r - q_hat(x,a))`, with `iw = pi(a|x) / p_hat(a|x)`.

Now I check the bias algebra. Let the true reward mean be `q(x,a)`, the model error be `Delta(a,x) = q_hat(x,a) - q(x,a)`, and the propensity error be `delta(a,x) = 1 - p(a|x) / p_hat(a|x)`. I condition on `x` and assume the nuisance estimates are fixed independently of the evaluation sample. The expected residual term replaces `r` by `q`, and the action expectation of the weighted factual model prediction is `(p / p_hat) q_hat` for the logged action distribution. After cancellation, the expected contribution is the target value plus `sum_a pi(a|x) Delta(a,x) delta(a,x)`. For a deterministic target policy this is simply `V^pi + E[Delta delta]`.

That is the decisive identity. The direct method has first-order bias in `Delta`. Importance sampling has first-order bias in the propensity error. This estimator has bias in the product `Delta delta`. If the reward model is right, `Delta` is zero. If the propensity model is right, `delta` is zero. Either way, the product vanishes. This is why one correct nuisance model is enough.

The same construction explains the variance gain. Importance sampling multiplies the high-variance weight by the whole reward. Here the weight multiplies only `r - q_hat(x,a)`. Dudik, Langford, and Li's variance expression makes that precise: the dangerous factor containing `(1-p)/p` is multiplied by `Delta^2`, the squared model error. In plain IPS, the comparable penalty is multiplied by the squared reward or target-action value. So a useful model reduces variance even if it is not perfect, because it shrinks the object being reweighted.

This is the orthogonality insight in off-policy form. In the missing-data and causal-inference literature, augmented inverse-probability estimators are built from estimating functions whose first-order sensitivity to each nuisance vanishes at the truth. Here the baseline plus weighted residual creates the same structure for policy value. The target-policy model mean and the weighted factual model prediction have the same expectation when the propensity is correct, so the model term is a control variate. The correction does not blend two answers; it subtracts a correlated quantity from the noisy IS term and adds back its known target-policy mean.

I then lift the idea to a finite-horizon MDP by rewriting step-wise importance sampling recursively. Step-wise IS can be written as `V_step^{H+1-t} = rho_t (r_t + gamma V_step^{H-t})`. At time `t`, this is a one-step bandit: the context is `s_t`, the action is `a_t`, and the stochastic outcome is the immediate reward plus the discounted future estimate. The conditional mean of that outcome is `Q(s_t,a_t)`. So I can apply the one-step residual correction at every time step.

With a fitted action-value model `Q_hat` and `V_hat(s) = sum_a pi_1(a|s) Q_hat(s,a)`, the recursion becomes `V_DR^0 = 0` and `V_DR^{H+1-t} = V_hat(s_t) + rho_t (r_t + gamma V_DR^{H-t} - Q_hat(s_t,a_t))`. The model baseline is the target-policy mean of `Q_hat`; the weighted term corrects the factual residual. If `Q_hat` is identically zero, this collapses back to step-wise importance sampling. If `Q_hat` is accurate, the residual is small and the variance falls.

The unbiasedness proof is the same cancellation repeated by induction. Conditional on `s_t`, `E_{a_t~pi_0}[rho_t Q_hat(s_t,a_t)] = V_hat(s_t)`. Therefore the model term has zero net effect on the expectation, while the weighted reward-plus-future term transports the behavior-policy sample to the target-policy distribution. The future term is unbiased by the induction hypothesis, so the whole recursion is unbiased.

The variance recursion shows exactly what the model can remove. If `Delta(s,a) = Q_hat(s,a) - Q(s,a)`, Jiang and Li decompose the variance into a transition term, an action-stochasticity term `E[Var(rho_t Delta(s_t,a_t) | s_t)]`, a reward-noise term, and a discounted future-variance term. Only the action term depends on `Delta`. A better value model shrinks that term; it does not erase transition noise or reward noise.

This limitation matters because it prevents me from overclaiming. Even a perfect `Q_hat` leaves variance from stochastic transitions and rewards. Jiang and Li's lower-bound argument says this leftover variance is intrinsic in tree MDPs: the perfect-model estimator reaches the Cramer-Rao lower bound for unbiased off-policy estimators. The method is therefore optimal in the sense that a perfect value model removes the reducible action-mismatch variance and leaves only the information-theoretic floor.

There is one practical condition I cannot skip. The nuisance estimates must be independent of the evaluation fold. The identities `E[rho q_hat] = q_hat(pi)` and `E[rho Q_hat] = V_hat` treat the model as fixed while taking expectations over the logged action. If I fit the model on the same sample I evaluate, this conditional cancellation can fail. Sample splitting or cross-fitting restores the needed independence while still using all data across folds.

So the answer is a residual-corrected importance estimator. It uses direct modeling to create a baseline and importance weighting to correct only what the baseline missed. The product-bias identity gives the one-correct-nuisance property, and the control-variate view explains why the variance penalty is tied to model error rather than reward magnitude. In sequential RL, the same insight appears recursively because each time step can be viewed as a bandit whose outcome is reward plus discounted future return.
