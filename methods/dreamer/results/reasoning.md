I start from the bottleneck: I want an agent that controls from pixels, and every real environment step is expensive. A model-free policy gradient pays twice. It has to collect real transitions for each update, and the score-function gradient uses a scalar return as the multiplier on each action score. That scalar contains later action noise and environment noise, so the variance grows with horizon. A value baseline can subtract a state-dependent constant, but it still leaves me estimating "return times score" instead of asking how the return changes when the action changes.

The obvious way to reuse data is to learn a model. If I can predict future states and rewards from a current state and an action, then I can improve behavior inside that model instead of spending real interactions for every trial. But that only solves half the problem unless I also use the differentiability of the model. If the predicted return is a differentiable computation of the actions, I can backpropagate through the transition and take a pathwise gradient. That gradient uses the local sensitivities of reward, value, and dynamics, not only a sampled scalar return.

Pixels are the wrong state space for this. I do not need to render every imagined frame in order to choose an action. I need a compact state that preserves information useful for predicting reward and future observations. So I build a latent state-space model. On real data I infer a posterior state from the previous state, previous action, and current observation. In imagination I use a prior transition that sees the previous state and action but not the next observation. The prior is the model I dream with, so the posterior and prior must be trained to agree.

The variational objective makes this precise. The positive terms are observation and reward prediction, $\log q(o_t\mid s_t)$ and $\log q(r_t\mid s_t)$. The regularizer is subtracted and has direction

$$\mathrm{KL}\!\left(p(s_t\mid s_{t-1},a_{t-1},o_t)\,\|\,q(s_t\mid s_{t-1},a_{t-1})\right).$$

That direction matters. The posterior is what I can infer when the real observation is available; the prior is what I must use when I imagine. Penalizing posterior-to-prior KL forces the transition prior to land where the observation-conditioned representation would have landed. In loss form I minimize a positive KL term minus the log likelihood terms, with a free-nats floor so the KL cannot dominate early.

I also need the latent dynamics to remember. A fully stochastic latent Markov chain has to carry information through a sampled bottleneck at every step. A purely deterministic recurrent model carries memory but has no stochastic state for uncertainty and no useful posterior-prior KL. The recurrent state-space split resolves this. A deterministic recurrent state $h_t=f(h_{t-1},s_{t-1},a_{t-1})$ carries memory through a GRU. A stochastic state $s_t$ is sampled from a prior $q(s_t\mid h_t)$ or from a posterior $p(s_t\mid h_t,o_t)$. The feature used downstream is the concatenation of stochastic and deterministic parts. The samples are diagonal Gaussians and are reparameterized, so gradients can pass through sampled imagined states.

There is an alternative representation objective that avoids decoding pixels. Instead of predicting $o_t$ from $s_t$, I can predict $s_t$ from $o_t$ and use a contrastive term

$$\log q(s_t\mid o_t)-\log\sum_{o'} q(s_t\mid o').$$

The first term makes the state predictable from its own image; the negative log-sum over other observations prevents collapse by making the state less predictable from every image in the batch. This changes how the representation is learned, but it does not change the behavior-learning problem once I have latent states, rewards, and transitions.

Now I need a behavior. Online planning would search over action sequences at every environment step, score them by predicted reward over a finite horizon, execute the first action, and repeat. That reuses the model but pays a large action-time cost and remains shortsighted: reward after the planning horizon is invisible. I want a policy network for cheap acting and a value model for the tail beyond the finite imagination horizon.

I start imagined rollouts from posterior states of replayed real sequences. These roots are filtered using actual observations, so the rollout begins near the data manifold. From each root I roll the prior forward in latent space. At each imagined step I sample an action from a tanh-transformed Gaussian policy, sample the next latent state with the reparameterized transition, and predict reward and value from the latent feature. No image decoding is needed for behavior learning.

The action distribution must be reparameterized for the same reason as the latent state. I write a continuous action as

$$a_\tau=\tanh(\mu_\phi(s_\tau)+\sigma_\phi(s_\tau)\epsilon),\qquad \epsilon\sim\mathcal N(0,I).$$

Then a sampled action is a differentiable function of the actor outputs. For discrete control I cannot use that Gaussian path, so I use a straight-through estimator for the categorical action sample.

For each imagined state I need a target value. A one-step bootstrap has low variance but inherits value bias. A long rollout has lower bootstrap bias but accumulates model error and stochastic variance. The finite-horizon $k$-step target is

$$\mathrm{V}_{\mathrm N}^k(s_\tau)=\mathbb E\!\left[\sum_{n=\tau}^{h-1}\gamma^{n-\tau}r_n+\gamma^{h-\tau}v_\psi(s_h)\right],\qquad h=\min(\tau+k,t+H).$$

I want a single dial that averages these $k$-step targets so I can sit between the one-step bootstrap and the full rollout. The exponentially weighted average is the natural choice:

$$\mathrm{V}_\lambda(s_\tau)=(1-\lambda)\sum_{k=1}^{H-1}\lambda^{k-1}\mathrm{V}_{\mathrm N}^k(s_\tau)+\lambda^{H-1}\mathrm{V}_{\mathrm N}^H(s_\tau).$$

For this to be a proper average I need the coefficients to be a probability distribution over $k$. Adding them up for a horizon of, say, $H=6$: the first $H-1$ weights are $(1-\lambda)\lambda^{k-1}$ for $k=1\ldots5$, and the last is $\lambda^{5}$. The geometric part is $(1-\lambda)(1+\lambda+\lambda^2+\lambda^3+\lambda^4)=(1-\lambda)\frac{1-\lambda^5}{1-\lambda}=1-\lambda^5$, and adding the trailing $\lambda^5$ gives exactly $1$. So the trailing term is precisely the leftover mass that the geometric sum left short, and $\mathrm{V}_\lambda$ is a genuine weighted average of the $k$-step targets. For intermediate $\lambda$ it trades bootstrap bias against rollout variance and model error.

The forward sum is awkward to compute; I would rather have a backward recursion. I claim the same quantity is

$$G_\tau=r_\tau+\gamma\left((1-\lambda)v_\psi(s_{\tau+1})+\lambda G_{\tau+1}\right),\qquad G_{t+H}=v_\psi(s_{t+H}),$$

but I should not take that on faith, because the algebra that collapses a weighted sum of $k$-step returns into a one-line recursion is exactly the kind of thing that hides a factor-of-$\gamma$ or off-by-one error. So I evaluate both forms on a small random instance — five rewards, six values (the last being the tail bootstrap), $\gamma=0.99$ — and compare. At $\lambda=0.3$ the backward recursion returns $2.671984$ and the forward weighted sum returns $2.671984$; at $\lambda=0.95$ both give $6.495500$. They agree across $\lambda\in\{0,0.3,0.95,1\}$, so the recursion does compute the same return as the explicit average.

The edge cases confirm the direction. Setting $\lambda=0$ in the recursion kills the $\lambda G_{\tau+1}$ term and leaves $G_\tau=r_\tau+\gamma v_\psi(s_{\tau+1})$ — a one-step bootstrap. On the same random instance the recursion at $\lambda=0$ gives $G_0=2.70464$, matching $r_0+\gamma v_\psi(s_1)=2.70464$ computed directly. Setting $\lambda=1$ kills the $(1-\lambda)v_\psi(s_{\tau+1})$ shortcut and leaves $G_\tau=r_\tau+\gamma G_{\tau+1}$, which unrolls to the discounted Monte Carlo sum plus the final value bootstrap; the recursion gives $7.224758$, matching $\sum_{n}\gamma^n r_n+\gamma^5 v_\psi(s_5)=7.224758$ computed directly. The two limits I care about — pure bootstrap and pure rollout — both come out right.

If episodes can end early, I replace the constant $\gamma$ at each step with a predicted continuation-discount term, so the recursion becomes `reward + pcont * ((1-lambda) * next_value + lambda * next_return)`.

Written as vectorized code rather than a pencil recursion, the natural form looks different: precompute `inputs = reward + pcont * next_values * (1 - lambda)` once, then scan `agg = inputs + pcont * lambda * agg` in reverse from the bootstrap. That factoring is not obviously the same as `reward + pcont * ((1-lambda)*next_value + lambda*next_return)`, so I check it on a fresh random instance: the two forms agree to floating point at $\lambda\in\{0,0.95,1\}$ (for instance `returns[0]` of $1.553451$ at $\lambda=0.95$ from both). The rearrangement is just distributing $\gamma\lambda$ across the two terms — same recursion, vectorized.

The value update is then a semi-gradient regression. I fit $v_\psi(s_\tau)$ to the computed $\mathrm{V}_\lambda(s_\tau)$ and stop the gradient through the target. Without that stop-gradient, the value network could reduce the loss by moving both prediction and target, because the target itself contains value estimates — the loss would shrink without the prediction getting any closer to a fixed thing to predict. I make the value head a Normal distribution with fixed variance, so minimizing negative log probability of the stopped target is the same squared-error regression up to constants.

Now the actor update. I maximize the imagined lambda-return estimates. The question is how to differentiate that objective with respect to the policy. REINFORCE on imagined actions is available, but it would throw away exactly the information I built the model to expose: the return depends on predicted rewards and values; those depend on imagined states; those states depend on previous actions through the transition; and the actions depend on actor parameters through the reparameterized policy. Each of these maps is a differentiable network or a reparameterized sample, so the whole chain admits a gradient — the actor gradient can be the analytic pathwise gradient through the learned model rather than a score-function estimate.

This also explains why I only need a state value instead of an action value. In DDPG or SAC, the environment is not differentiable, so the critic must be an action value $Q(s,a)$ to expose how changing $a$ changes the future. Here the learned transition supplies that action-to-future derivative directly. The value tells me how good a state is, and the transition tells me how the action moves me into future states. SVG has the same value-gradient instinct, but its practical variants use short model expansions; here I use compact latent rollouts over the imagination horizon and bootstrap the rest with the value model.

There is one implementation detail I need to keep exact. When I ask the actor for an action during imagination, I stop the gradient on the state feature passed as the actor input. The action must still depend on actor parameters, and the next state must still depend on the action. What I remove is the extra path where the actor changes an earlier action, that changes the current state, and then the current state changes the actor's input at the same step. The intended credit assignment is through future rewards and values after actions are applied, not through differentiating the policy's conditioning state as if that were another actor output.

The world model stays fixed during behavior learning. Gradients for the actor and value losses may pass through the model computations to reach the actor, but the optimizer for those losses updates only actor or value parameters. The representation is trained by its own objective on real sequences. This separation keeps the latent model anchored to observation and reward prediction rather than letting behavior gradients reshape the state space.

The full algorithm is three loops: fit the latent dynamics on replayed sequences (reconstruction or contrastive prediction, reward prediction, posterior-prior KL); train actor and value inside imagined latent rollouts from posterior roots, using lambda returns and analytic gradients through the prior transition; act in the real environment with the learned actor plus exploration noise and append the episode to replay. It is not merely "plan in a model" — it is actor-critic learning inside a compact learned MDP, where multi-step pathwise gradients and value bootstrapping do the work that a finite planning horizon alone could not.
