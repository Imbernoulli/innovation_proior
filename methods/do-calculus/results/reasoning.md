I start with the old adjustment picture and immediately run into its limits. If I only say that I should control for confounders, I have no general way to say which variables count, whether a descendant is useful, or whether conditioning on a collider quietly creates the very association I am trying to remove. The trouble is that ordinary conditional probability writes two different acts the same way: seeing that a variable has a value, and forcing that variable to have the value. I want a notation that keeps them apart, and a set of rules for moving between them.

So I treat the causal diagram as a compact description of mechanisms. Each node has a local equation, and the arrows into the node name the other variables whose values its mechanism reads. If I intervene on `X`, I do not merely condition on `X=x`; I replace the equation for `X` by the constant `x`. In the graph that means deleting the incoming arrows into `X` while leaving the other mechanisms untouched. I will write that mutilated graph `G_{\bar X}`. Whether this graph surgery actually buys me the right answer is something I should check on a concrete model before I trust it, not assume.

Let me set up the smallest case where adjustment is supposed to be needed: a confounder `Z` that points into both `X` and `Y`, with `X -> Y` as well. I give it numbers. Take `P(Z=1)=0.4`, let `X` depend on `Z` (`P(X=1|Z=0)=0.3`, `P(X=1|Z=1)=0.8`), and let `Y` depend on both. The interventional distribution, by definition, is the truncated factorization: cut `Z -> X`, set `X=x`, and let the rest run, giving `P(y|do(x)) = sum_z P(z) P(y|x,z)`. I compute that directly from the numbers and get `P(Y=1|do(X=0)) = 0.2200` and `P(Y=1|do(X=1)) = 0.6400`. For contrast I also compute the naive observational `P(Y=1|X=x)`, which comes out `0.1480` and `0.6640`. So the two genuinely disagree — the confounding is real, conditioning on `X` alone is biased, and any candidate method has to land on `0.2200` / `0.6400`, not on the observational numbers.

Now I want the surgery-plus-d-separation idea to reproduce those target numbers symbolically, so I trace it. I expand the target on `Z`:

`P(y|do(x)) = sum_z P(y|do(x), z) P(z|do(x))`.

This has two pieces, each still carrying a `do(x)`, and I need a licensed move for each. For `P(z|do(x))`: `Z` is a pre-treatment confounder, an ancestor of `X`, so intervening on `X` should not touch its distribution. The move that removes an action is the one whose separation test runs in the graph where I have also deleted arrows into `X`. In `G_{\bar X}` there are no arrows into `X` at all, so `X` is a source there and `Z` (which sits upstream) is d-separated from `X` — the test `(Z \perp X)` holds. That licenses `P(z|do(x)) = P(z)`. For `P(y|do(x), z)`: I want to trade the action on `X` for an observation of `X`. The relevant test is in `G_{\underline X}`, the graph with arrows *out of* `X` deleted. Deleting `X`'s outgoing arrow kills the direct `X -> Y` path; the only remaining `X`–`Y` connection runs through the back door `X <- Z -> Y`, and that is blocked once I condition on `Z`. So `(Y \perp X \mid Z)` holds in `G_{\underline X}`, which licenses `P(y|do(x), z) = P(y|x, z)`. Putting the two licensed rewrites back:

`P(y|do(x)) = sum_z P(y|x, z) P(z)`.

That is the classical back-door adjustment formula, and crucially I did not assert it — I evaluated its right-hand side on the same numbers and got `0.2200` and `0.6400`, matching the truncated-factorization truth exactly and differing from the biased observational values. So the surgery-and-separation discipline reproduces the answer on at least this case, and it does so by two distinct kinds of move (remove an action; exchange an action for an observation) tested in two *different* mutilated graphs. That difference is a hint about the shape of the final object: it is not one criterion but several rewrite permissions, each tied to its own graph.

Before I generalize, I try to collapse it back to a single adjustment criterion — "find a covariate set that blocks the back doors and condition on it" — because that is simpler and I would prefer it if it worked. It fails on a structure I can write down: hidden confounding between `X` and `Y`. Put an unobserved `U` with `U -> X` and `U -> Y`, and a mediator `Z` on the only directed path, `X -> Z -> Y`. Now there is no observed pre-treatment covariate that blocks the `X <- U -> Y` back door, because `U` is unobserved and nothing else lies on that path. A single admissible adjustment set simply does not exist; the one-criterion idea is not just awkward here, it is impossible. So I cannot stop at back-door. I need moves that can decompose the target into pieces, where confounding that defeats one piece need not defeat another.

I check whether decomposition can rescue the same hidden-`U` graph, again with numbers (`P(U=1)=0.5`, `U` driving `X`, `Z` depending only on `X`, `Y` depending on `Z` and `U`). The directed effect runs entirely through `Z`, and two facts hold by inspection of this graph: there is no unblocked back-door path from `X` to `Z` (any such path would go `X <- U -> Y <- Z`, blocked at the collider `Y`), and every back-door path from `Z` to `Y` is blocked by conditioning on `X` (the path `Z <- X <- U -> Y` is blocked at `X`). Those two clean sub-structures are exactly what the moves can exploit. I split the effect of setting `X` into the effect of `X` on the mediator and the effect of the mediator on `Y`: replace the interventional distribution of `Z` by its observed conditional `P(z|x)` (clean because `X -> Z` has no open back door), and express the effect of *setting* `Z` by averaging the outcome over the natural distribution of treatment, `sum_{x'} P(y|z,x') P(x')` (clean because conditioning on `X` blocks `Z`'s back doors). Chaining them gives

`P(y|do(x)) = sum_z P(z|x) sum_{x'} P(y|z,x') P(x')`.

I do not want to take that on faith either, since it was assembled from two separate arguments. So I compute its right-hand side from the model — using only observable quantities, with `U` integrated out as it would be in real data — and compare against the surgery truth `sum_u P(u) sum_z P(z|x) P(y|z,u)`. The formula returns `0.46000` and `0.58000`; the truth returns `0.46000` and `0.58000`. They agree to all digits, even though `X` and `Y` share an unobserved cause that no adjustment set could remove. The naive observational `P(Y=1|X=x)` here is `0.34333` and `0.67545`, biased and, interestingly, biased in *opposite* directions at the two treatment levels — so the front-door correction is doing real, non-monotone work, not just shrinking a number. The decomposition recovered an interventional quantity that the single-criterion approach provably could not reach.

Now the structure of the general object is forced. I need an algebra over mixed expressions that contain both observations and actions, with three kinds of move, each a rewrite permission gated by a d-separation test in a specific mutilated graph. One move deletes an observation when, after the current actions are represented by deleting arrows into the acted-on variables, the outcome is d-separated from that observed variable given the acted-on and already-conditioned variables (test in `G_{\bar X}`). A second move exchanges an action for an observation when the same separation holds after I additionally delete the *outgoing* arrows from the variable being exchanged (test in `G_{\bar X, \underline Z}`) — this is the very move I used to turn `do(x)` into `x` in the back-door case. A third move deletes an action under the more careful test: after deleting arrows into the already-acted-on variables, I delete arrows into only those newly-acted-on variables that are *not* ancestors of the conditioning variables in `G_{\bar X}`, and then check the separation (test in `G_{\bar X, \overline{Z(W)}}`). The restriction to non-ancestors is not decoration; it is what kept `P(z|do(x)) = P(z)` honest above, where `Z` was an ancestor of nothing being conditioned on and the deletion was therefore allowed.

This reframes the whole task. Identification stops being "find the right controls" and becomes "reduce the intervention expression": apply ordinary probability algebra between moves, and use the three graph-indexed rewrites to peel off `do(...)` symbols one at a time. The two worked cases are the two characteristic shapes of that reduction — back-door is a short reduction with one expansion and two moves; front-door is a longer reduction that decomposes through a mediator and survives hidden confounding.

The same machinery also tells me when to stop trying. If the graph has a structure such as a direct `X -> Y` link wrapped in latent confounding positioned so that no move's separation test ever fires, then every attempted reduction preserves a `do` marker that cannot be eliminated. I read that not as a failing of some particular regression but as a real statement about the model: two structural models can then agree on every observable probability while disagreeing on the interventional one, so no function of the observational distribution could possibly return the effect. The rewrite system is thus a derivation procedure and, when it gets stuck, a non-identifiability witness.

What I have arrived at is a calculus: each step is local, syntactic, and justified by a graphical separation test, and the syntax is anchored in structural interventions — actions delete mechanisms, observations restrict distributions, and the mutilated graph attached to each rule decides which distinction is in force at that point. A derivation either rewrites away every intervention symbol, yielding a formula in observational quantities that I have checked reproduces the true interventional distribution, or it stalls and thereby exhibits why the observational distribution is insufficient. Whether these three rules suffice for *every* identifiable effect is not something my two examples establish; I have only shown they handle the back-door and front-door cases correctly, and I would want to test the system against harder graphs — or a completeness argument — before believing it is exhaustive.
