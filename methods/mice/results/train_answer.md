We are handed a numeric table with $n$ rows and $p$ variables in which $\texttt{NaN}$s are scattered across several columns at once — different rows missing in different columns — and we want a completed table that downstream analysis can use as if it had been observed. Two things must hold for the fill to be honest. It has to borrow strength across correlated columns: a row with a high BMI and a missing cholesterol should get a cholesterol informed by that BMI, not the column's global average. And it must not pretend to know more than it does: a single deterministic guess plugged in as if observed shrinks the column's variance and pulls its correlations toward zero. Mean-filling fails both at once — it ignores every other variable, and by inserting the identical constant into every hole it adds zero scatter. The obvious repair is to regress each incomplete column $Y_j$ on the others $Y_{-j}$ over the rows where $Y_j$ was observed, which is licensed by Rubin's ignorability result: under missing-at-random (and MCAR is its strict special case), the conditional distribution $P(Y_j^{\text{mis}} \mid Y_j^{\text{obs}}, Y_{-j})$ is exactly the right object to draw from, and that conditional is what a regression of $Y_j$ on $Y_{-j}$ estimates. But the moment we write it down we hit the wall that makes multivariate missingness hard: the predictors $Y_{-j}$ are themselves incomplete, and the dependence is circular — $Y_1$ would help fill $Y_2$ and $Y_2$ would equally help fill $Y_1$ — so there is no complete column to regress on. The textbook escape, positing one joint distribution over all $p$ variables (Schafer's multivariate normal with data-augmentation MCMC), is hostage to the existence of a tractable joint: real tables mix binary, ordinal, and skewed-bounded continuous columns with nonlinear and constrained relations, and a multivariate normal forced onto them hands back continuous fills for yes/no variables and negative fills for strictly positive ones. The joint is the bottleneck, and for this data we cannot write it down.

The resolution I propose is MICE — Multivariate Imputation by Chained Equations, also called Fully Conditional Specification. The key move is to never form the joint at all. For any single column, given the others, a sensible model is easy to choose: linear regression for a continuous variable, logistic for a binary one, ordered-logit for an ordinal one. So we specify $P(Y_j \mid Y_{-j}, \theta_j)$ one column at a time, each with its own functional form and its own little parameter $\theta_j$, and sidestep the entire bottleneck. The honest price is that a set of conditionals chosen independently need not be the conditionals of any single joint — they can be mutually incompatible — but the theoretical blemish stays mild in practice and we contain it with the visit order, below. To beat the circularity we break the deadlock with a rough start and then refine: fill every column's holes crudely (the cheapest sensible choice is each column's observed mean, which the first sweep overwrites anyway), so that every variable is now crudely complete and the single-column regression can actually run. Then we sweep: take $Y_j$, discard its current fill, regress it on the *current* values of all the other columns over the rows where $Y_j$ was truly observed, re-impute its holes, and move to $Y_{j+1}$, which immediately sees the updated $Y_j$ among its predictors. One pass through all $p$ columns is a sweep; we repeat. Nobody is ever frozen as a fixed complete predictor — everybody is conditionally re-estimated given everybody else's latest state — and that chained structure is what dissolves the circular dependence.

What makes this principled rather than a hack is recognizing the abstract move: at each column we are sampling the block $Y_j$ from its full conditional and cycling through $j = 1, \dots, p$ repeatedly — a Gibbs sampler. If the per-column conditionals were the genuine conditionals of some joint $P(Y \mid \theta)$, the sweep draws the completed tables from the correct posterior predictive of the missing data. That reading carries two consequences. First, it dictates what a *proper* step must do — draw the parameters $\theta_j$ from their posterior, then draw $Y_j$'s missing entries given those drawn parameters, two draws, not a plug-in. For a continuous column with observed response $y$ ($n_1$ rows) and predictor matrix $X$ ($q$ columns) the proper Bayesian linear-regression draw is
$$S = X_{\text{obs}}^{\top} X_{\text{obs}}, \qquad V = (S + \kappa\,\mathrm{diag}(S))^{-1}, \qquad \hat\beta = V X_{\text{obs}}^{\top} y_{\text{obs}},$$
with a small ridge $\kappa = 0.0001$ added inside $V$ purely to keep the inverse from exploding on collinear, small-$n$, or empty-cell columns; because the same $V$ appears in $\hat\beta$ and in the coefficient covariance, the point estimate and the coefficient uncertainty stay matched to the same stabilized regression. Then draw the residual variance from its scaled inverse-$\chi^2$ posterior, $g \sim \chi^2_{n_1 - q}$ and $\dot\sigma^2 = (y_{\text{obs}} - X_{\text{obs}}\hat\beta)^{\top}(y_{\text{obs}} - X_{\text{obs}}\hat\beta)/g$; draw the coefficients $\dot\beta = \hat\beta + \dot\sigma\,\mathrm{chol}(V)\,z_1$ with $z_1 \sim N(0, I_q)$; and form the imputations on the missing rows as
$$y_{\text{imp}} = X_{\text{mis}}\dot\beta + \dot\sigma\,z_2, \qquad z_2 \sim N(0, I_{n_0}).$$
Each of the three random pieces is load-bearing for valid multiple imputation. The $\chi^2$ draw says the noise level is unknown; the normal $\dot\beta$ says the coefficients are unknown — parameter uncertainty; the additive $\dot\sigma\,z_2$ says even with the true model the value scatters about its conditional mean. They are exactly what makes Rubin's between-imputation variance come out right: producing $m$ completed tables and pooling a scalar estimand $Q$ via $\bar Q = \frac1m\sum_h \hat Q_h$, $\bar U = \frac1m\sum_h U_h$, $B = \frac{1}{m-1}\sum_h(\hat Q_h - \bar Q)^2$, $T = \bar U + (1 + \frac1m)B$, the term $B$ is the extra variance from having missing data and is nonzero only because the draws genuinely differ across completions. Plug in $\hat\beta$ with no noise and every completion is identical, $B = 0$, intervals collapse to $\bar U$; add the residual but freeze the coefficients and you omit the parameter draw, which is precisely the variant whose interval coverage comes out near $0.908$ instead of $0.95$. A robust sibling, predictive mean matching, uses the same regression only to *match*: compute $\eta_{\text{obs}} = X_{\text{obs}}\hat\beta$ for observed cases and $\eta_{\text{mis}} = X_{\text{mis}}\dot\beta$ for missing ones, form a donor pool of the $d \approx 5$ observed cases with $\eta_{\text{obs}}$ closest to each $\eta_{\text{mis}}$, pick one at random, and donate its actually-observed $y$ — so every fill is a value that genuinely occurred, automatically respecting bounds, skew, and discreteness, at the cost of never extrapolating beyond the observed range. The second consequence of the Gibbs reading is that the chain mixes fast: when we re-impute $Y_j$ we discard its own prior fill and regenerate it from scratch, so iteration $t-1$ influences iteration $t$ only *indirectly*, through how it changed the other columns that feed back as predictors. There is no direct autocorrelation in a column's own imputed values, so the dependence on the crude initialization washes out in a handful of sweeps — five to twenty, not hundreds — with no long burn-in, which is what makes an MCMC over the missing data cheap enough to be practical.

The incompatibility worry is real but containable. Two linear conditionals are jointly compatible, but mixing a linear and an ordinal model may leave no joint with both as conditionals, so a compatible Gibbs sampler is order-invariant while an incompatible one is not. We therefore visit the columns in ascending order of fraction-missing: the least-incomplete columns are the ones estimable most reliably from real observed data, so imputing them first means the heavily-missing columns find their predictors already as good as they will get, and the early sweeps are informative rather than wasted.

One design decision is dictated by what this benchmark actually scores. The objective here is not calibrated inference with pooled standard errors but a single completed table judged on squared-error reconstruction against held-out truth and on a downstream model's accuracy. Under squared-error loss the single fill that minimizes expected loss is the *conditional mean* of the missing value given the observed — and a proper multiple-imputation draw deliberately adds scatter around that mean, which is exactly what makes $B$ right for inference but *increases* squared error against a fixed truth. So for this objective I switch the randomness off and ship the point version: the chained round-robin stays, since that is what handles the multivariate circularity and borrows cross-variable strength, but each step becomes a deterministic conditional-mean regression rather than a stochastic posterior draw. The per-column model is a self-regularizing Bayesian linear regression that sets its own ridge by evidence maximization (MacKay's framework, sklearn's BayesianRidge), so I never hand-pick a penalty across heterogeneous columns. On centered data with a fitted intercept, the posterior-mean coefficients solve $\beta = (\frac{\lambda}{\alpha} I + X^{\top}X)^{-1} X^{\top}y$, so the ridge strength is the ratio $\lambda/\alpha$, and the precisions are iterated from the data: with eigenvalues $d_i$ of $X^{\top}X$, the effective number of parameters $\gamma = \sum_i \frac{\alpha d_i}{\lambda + \alpha d_i}$ feeds $\lambda = (\gamma + 2\lambda_1)/(\sum \beta^2 + 2\lambda_2)$ and $\alpha = (n - \gamma + 2\alpha_1)/(\text{SSE} + 2\alpha_2)$, with all four Gamma-hyperprior constants defaulting to $10^{-6}$ — essentially $\gamma/\sum\beta^2$ and $(n-\gamma)/\text{SSE}$, the collinear or small-$n$ column shrunk more, the well-determined one less, with no manual tuning, while the intercept stays an ordinary centered-data offset rather than a random coefficient. Concretely the loop initializes every column to its observed mean, visits columns least-missing-first, and for a capped number of rounds fits the Bayesian ridge on each column's observed rows, predicts its missing rows, and writes the prediction back so the next column sees it immediately; after each sweep it checks the infinity-norm change in the imputed entries relative to the data scale and stops early when that drops below a small $\texttt{tol}$, since the fixpoint is effectively reached and chasing exact convergence rarely improves the downstream score. The round cap leans on the fast-mixing argument, and otherwise the round-robin, the fit-on-observed / predict-on-missing per column, the ascending order, the mean initialization, and the tolerance-based early stop are the standard iterative-imputer behaviors I wrap so as not to re-implement that bookkeeping by hand.

```python
import numpy as np
from sklearn.base import BaseEstimator, TransformerMixin


class CustomImputer(BaseEstimator, TransformerMixin):
    """MICE / chained-equations imputation (point variant).

    Each incomplete column is regressed on all the others in a round-robin,
    iterated to a fixpoint. The per-column engine is a self-regularizing
    Bayesian linear regression (evidence-based ridge), and we impute the
    conditional mean -- the squared-error-optimal point fill.
    """

    def __init__(self, random_state=42, max_iter=30):
        self.random_state = random_state
        self.max_iter = max_iter

    def _build(self):
        from sklearn.experimental import enable_iterative_imputer  # noqa: F401
        from sklearn.impute import IterativeImputer
        from sklearn.linear_model import BayesianRidge
        return IterativeImputer(
            estimator=BayesianRidge(),       # self-tuning ridge per column
            sample_posterior=False,           # point fills: posterior-mean prediction
            max_iter=self.max_iter,          # round-robin sweep cap (early-stops sooner)
            random_state=self.random_state,
            imputation_order="ascending",    # least-missing columns first
            initial_strategy="mean",         # complete starting table = column means
            tol=1e-3,                        # stop when fills stop moving (scaled inf-norm)
        )

    def fit(self, X, y=None):
        self._imputer = self._build()
        self._imputer.fit(X)
        return self

    def transform(self, X):
        return self._imputer.transform(X)

    def fit_transform(self, X, y=None):
        self._imputer = self._build()
        return self._imputer.fit_transform(X)
```
