I start with the plain linear fit because it shows both failures clearly. I have training pairs `(x_t, y_t)` and I can minimize `sum_t (y_t - w^T x_t)^2`, giving the formal solution `w = (X^T X)^-1 X^T y`. This is cheap when the design is well-conditioned, but it is the wrong shape for a curved response, and it is numerically fragile. If `X = UDV^T`, the small singular values in `D` make coefficient variance blow up along the corresponding directions; if a singular value is zero, the inverse does not exist. So I need more flexibility and more stability at the same time.

The stability part already has a clean answer. I add a positive penalty on the coefficient norm and minimize `a ||w||^2 + sum_t (y_t - w^T x_t)^2`. The normal equation becomes `(X^T X + aI)w = X^T y`. Since `X^T X` is positive semidefinite and `aI` is positive definite for `a > 0`, the shifted system is invertible. In singular-value coordinates, the unstable directions are shrunk by factors like `d_j^2/(d_j^2 + a)`. That solves the conditioning problem, but it has not made the function nonlinear in the original input.

The direct nonlinear move is to replace `x` by a feature vector `phi(x)` and run the same ridge calculation there: `w = (Phi^T Phi + aI)^-1 Phi^T y`, prediction `w^T phi(x)`. This is exactly where the computation hits the wall. The matrix being inverted is feature-dimension by feature-dimension. A polynomial map can already have far more coordinates than examples, and a radial-basis feature map is naturally infinite-dimensional. The fit I want is the ridge fit in that feature space, but I cannot afford to write the feature-space normal equation in primal form.

So I look for an algebraic way to move the inverse from feature dimension to example dimension. For compatible rectangular matrices `P` and `Q`, `(PQ + aI)P = P(QP + aI)`, since both sides equal `PQP + aP`. With the shifted matrices invertible, multiplying on the left by `(PQ + aI)^-1` and on the right by `(QP + aI)^-1` gives `(PQ + aI)^-1 P = P(QP + aI)^-1`. Setting `P = Phi^T` and `Q = Phi` gives `(Phi^T Phi + aI)^-1 Phi^T = Phi^T(Phi Phi^T + aI)^-1`. So the feature-space ridge solution should also be writable as `w = Phi^T(Phi Phi^T + aI)^-1 y`.

I do not fully trust an identity I rearranged from a one-line manipulation, so I want to see it hold on actual numbers before I build anything on it. Take three examples with a feature map that has more coordinates than examples, so the two sides really are different-sized solves:

```
Phi = [[1, 2, 0, 1],
       [0, 1, 1, 2],
       [2, 0, 1, 1]]   # T=3 rows, m=4 features
y   = [1, -1, 2],   a = 2.
```

The primal solve `w = (Phi^T Phi + aI)^-1 Phi^T y` is a 4x4 inverse and returns `w = [1.1164, 0.0418, -0.1015, -0.3403]` (with `a = 0.5`; I will switch `a` deliberately below). The push-through form `w = Phi^T (Phi Phi^T + aI)^-1 y` is a 3x3 inverse and returns the same vector to the last digit, `max|difference| ~ 4e-16`. So the identity is not just formal; the feature-dimension solve and the example-dimension solve land on the same weight vector. The size of the inverted matrix has dropped from `m` to `T`.

That changes what I have to compute. `Phi Phi^T` is a `T x T` matrix, and entry `(s,t)` is just `phi(x_s)^T phi(x_t)`. If I define `c = (Phi Phi^T + aI)^-1 y`, then `w = Phi^T c = sum_t c_t phi(x_t)`. The learned weight vector lies in the span of the training feature vectors. That also makes geometric sense: any component of `w` orthogonal to every `phi(x_t)` changes no training prediction but increases `||w||^2`, so the penalty removes it.

A new prediction uses the same span: `w^T phi(x) = sum_t c_t phi(x_t)^T phi(x)`. The feature map appears only through inner products between examples. So I can replace each inner product by a kernel value `K(u,v) = phi(u)^T phi(v)`, provided `K` is a valid positive-semidefinite kernel. The training Gram matrix is `K_{s,t} = K(x_s, x_t)`, the test vector is `k(x)_t = K(x_t, x)`, and the coefficients satisfy `(K + aI)c = y`. The prediction is `f(x) = k(x)^T c = sum_t c_t K(x_t, x)`. The feature dimension has disappeared from the computation. On the same numbers, evaluating at a held-out point with feature vector `[1,1,1,0]`: the primal `w^T phi(x)` gives `1.05672` and the kernel form `k(x)^T c` gives `1.05672`. They agree, which is the concrete statement that I never needed the feature coordinates.

I want to re-derive the same coefficients a second way through an explicit constrained problem, because the inner-product substitution is easy and the place I expect to slip is a stray factor of two in the constants. I rewrite the regularized problem with explicit residuals: minimize `a ||w||^2 + sum_t xi_t^2` subject to `y_t - w^T phi(x_t) = xi_t`. With multipliers `alpha_t`, the Lagrangian is `a ||w||^2 + sum_t xi_t^2 + sum_t alpha_t(y_t - w^T phi(x_t) - xi_t)`. Differentiating in `w` gives `2a w - sum_t alpha_t phi(x_t) = 0`, so `w = (1/(2a)) sum_t alpha_t phi(x_t)`. Differentiating in `xi_t` gives `2 xi_t - alpha_t = 0`, so `xi_t = alpha_t/2`.

Substituting those stationary values leaves a dual objective whose derivative in `alpha` is `-(1/(2a))K alpha - (1/2)alpha + y`. Setting it to zero gives `(K + aI)alpha = 2a y`, hence `alpha = 2a(K + aI)^-1 y`. These `alpha_t` are the Lagrange multipliers, not the coefficients I would store for prediction; the stationarity condition says `w = (1/(2a)) sum_t alpha_t phi(x_t)`, so the prediction is `w^T phi(x) = (1/(2a)) k(x)^T alpha`. Pulling the `1/(2a)` inside, `c = alpha/(2a) = (K + aI)^-1 y`, which is the same `c` as before.

I want this constant to be a real check, not a coincidence, so I pick `a = 2` precisely so that `2a = 4` and `alpha` and `c` cannot accidentally coincide. On the example, `alpha = [0.6923, -1.3077, 1.2308]` while the directly solved `c = [0.1731, -0.3269, 0.3077]`, and indeed `alpha/4 = [0.1731, -0.3269, 0.3077] = c` to all printed digits. The prediction computed as `(1/(2a)) k(x)^T alpha` equals the prediction computed as `k(x)^T c`, both `0.78846`. So the multiplier convention and the stored-coefficient convention are consistent, and the `2a` genuinely cancels rather than leaking into the predictor. This is the constant I will have to match against whatever library I call: the stored coefficients are `(K + aI)^-1 y`, with no `2a` in front.

I also want the function-space argument, because it explains what happens if the Gram matrix is singular and the inverses above stop being literal. In the Hilbert space generated by the kernel, minimize `sum_t (y_t - f(x_t))^2 + a ||f||^2`. Any candidate function splits into a part in the span of `K(., x_t)` and a part orthogonal to that span. The reproducing property says the orthogonal part changes none of the training values, while Pythagoras says it increases the norm. A minimizer can therefore be written as `f = sum_t c_t K(., x_t)`.

For that representation, training values are `Kc` and the norm is `c^T K c`, so the objective is `||y - Kc||^2 + a c^T K c`. Its gradient is `2K(Kc - y) + 2aKc = 2K((K+aI)c - y)`. If `K` is invertible, this immediately gives `(K+aI)c = y`. If `K` is singular, I should not cancel `K`. Diagonalize `K = U diag(d_i) U^T`, write `c = Uz` and `y = Ur`. Each coordinate contributes `(r_i - d_i z_i)^2 + a d_i z_i^2`. For `d_i > 0`, the minimizer satisfies `(d_i + a)z_i = r_i`. For `d_i = 0`, the coefficient coordinate changes neither the fitted values nor the RKHS norm; it represents the zero function. The shifted system `(K+aI)c = y` selects one valid coefficient representative.

The claim I should not take on faith is that two representatives differing in a nullspace direction give the same predictions, not just the same training fit. Let me force it: duplicate one example in feature space (set row 3 of `Phi` equal to row 1), which makes `K` rank-2 with eigenvalues about `[0, 2.60, 15.40]`. Solving `(K + aI)c = y` with `a = 0.5` gives `c = [-0.067, -0.4406, 0.533]`. Now take the eigenvector `v` for the near-zero eigenvalue, confirm `Kv ~ 0`, and form `c2 = c + 3.7 v`. The fitted values `Kc` and `Kc2` agree, as expected. The prediction at the test point: `k(x)^T c = 0.51675` and `k(x)^T c2 = 0.51675`. They match, because the test cross-kernel `k(x) = Phi phi(x)` lies in the row space of `Phi`, which is exactly the range of `K`, so it is orthogonal to the nullspace direction I added. So the non-uniqueness of `c` under a singular `K` does not leak into predictions, and `a > 0` quietly picks a representative anyway.

Now I can see why squared loss is what makes all of this close in form. The derivative of the squared residual is linear in the fitted values, so after the representer step I get one linear system. If I used an epsilon-insensitive support-vector loss instead, I would gain sparsity because only points outside the epsilon tube need active coefficients, but I would trade the closed form for a quadratic program. For a medium-sized regression problem, a dense coefficient vector and one stable linear solve can be the simpler trade.

The diagonal shift is not a cosmetic numerical patch. A Gram matrix is positive semidefinite, but it can be singular when examples duplicate each other in feature space (exactly the rank-2 case I just built) or merely ill-conditioned when they are close. With `a = 0`, solving `Kc = y` asks for exact interpolation and can amplify noise in small-eigenvalue directions. Adding `aI` lifts every eigenvalue by `a`, making the solve stable for `a > 0` and shrinking the fitted function toward zero. There is also a probabilistic reading I can check. If `w` has a zero-mean Gaussian prior with covariance `(1/(2a))I` and the observation noise variance is `1/2`, then `cov(y) = (1/(2a))(K+aI)` and `cov(y, w^T phi(x)) = (1/(2a))k(x)`. The Gaussian-conditioning posterior mean is `cov(y,f) cov(y)^-1 y`; the `1/(2a)` factors cancel between the two covariances, leaving `k(x)^T(K+aI)^-1 y`. Computing it on the `a = 2` example returns `0.78846` — the same prediction as the ridge and dual derivations. So the diagonal constant `a` is the noise-to-prior-variance ratio under that normalization, and the Kriging/Gaussian-process posterior mean is the same estimator wearing different vocabulary, not merely an analogy.

For the actual nonlinear similarity, the radial-basis kernel is the clean default when I want locality and few knobs: `K(x,y) = exp(-gamma ||x-y||^2)`. It is symmetric and positive semidefinite for `gamma > 0`, it corresponds to an infinite-dimensional feature expansion, and `gamma` controls bandwidth. Large `gamma` makes the bumps narrow and the fit local; small `gamma` makes similarities broad and the fit smoother. Because Euclidean distance is scale-sensitive, the numeric inputs have to be put on comparable scales before this kernel is meaningful. If the descriptors span orders of magnitude, a log-transformed copy can make multiplicative structure visible, while standardized raw features preserve ordinary linear trends. Categorical descriptors have to become numeric, for example by one-hot encoding, before they can enter the distance.

The scaffold implementation should therefore separate preprocessing from the core algorithm. Fit the preprocessing only on training inputs, transform training rows to a numeric feature matrix, set the RBF bandwidth after the final feature dimension is known, and call the canonical kernel-regression estimator. That estimator forms the Gram matrix, adds the regularization constant to its diagonal, solves for `dual_coef_ = (K + aI)^-1 y`, stores the training rows, and predicts by a cross-kernel matrix times `dual_coef_`. That stored convention is the one I matched by hand above — coefficients `(K + aI)^-1 y` with no `2a` prefactor — so the library's `dual_coef_` and my derivation's `c` are the same object. The method is not a hand-built optimizer; it is the regularized kernel linear system, and every link in the chain from primal ridge to dual coefficients to GP posterior mean has been checked to produce the same numbers on the same small problem.
