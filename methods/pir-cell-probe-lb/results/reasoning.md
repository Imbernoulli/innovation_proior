Let me start from the thing that actually bothers me. In the cell-probe model — $w$-bit cells, addresses in $[2^w]$, only probes counted, all computation free — the highest lower bound anyone has for *any* explicit data-structure problem, static or dynamic, at the natural cell size $w=\Theta(\lg n)$, is $\Omega(\lg n)$. That is an embarrassingly low ceiling. We have upper bounds of $\lg^2 n$ all over the place; for dynamic weighted orthogonal range counting a range tree spends $O(\lg^2 n)$ per update and per query. Is that second logarithm forced? Nobody can show more than the first one is. I want to break the $\lg n$ barrier for a natural problem, and weighted 2-D range counting — insert points each with a $\Theta(\lg n)$-bit weight, query a corner $q=(x,y)$ and return the sum of weights of points dominated by $q$ — is the right target, because it has just enough structure to be hard but is completely natural.

So what do I have to work with. The one technique that actually decomposes a *dynamic* problem into a logarithmic structure is the chronogram of Fredman and Saks. Let me re-derive it so I feel exactly where its power comes from and where it runs out. I do $n$ random updates, and I cut them into epochs $i=1,\dots,\lg_\beta n$, where epoch $i$ has $\beta^i$ updates, performed in time from the largest epoch down to the smallest, and then I ask one query at the very end. Now I take the final memory and I *associate* each cell to the last epoch that wrote into it. The crucial bookkeeping fact: each cell is associated to exactly one epoch, so the sets $S_i$ of cells associated to epoch $i$ are disjoint, and the total number of cells the query reads is $\sum_i t_i$ where $t_i$ is the number of probes it makes into $S_i$. That disjointness is the whole reason this gives an additive bound.

Why must the query touch each epoch? Two observations. The updates of epochs *later* than $i$ — smaller epochs, performed after epoch $i$ but before the query — write only $O(\beta^{i-1} t_u)$ cells in total, a geometric sum dominated by epoch $i-1$. If I pick $\beta$ comfortably larger than $w t_u$, then $\beta^{i-1} t_u = o(\beta^i)$, and those few cells simply cannot encode the $\beta^i$ random updates of epoch $i$ — there is not enough information in them. And the updates of epochs *earlier* than $i$ knew nothing about epoch $i$, because epoch $i$ hadn't happened when they were written. So if the answer to the query depends on epoch $i$'s updates, the query algorithm has to read a cell in $S_i$. That forces one probe per epoch, and summing over the $\lg_\beta n$ epochs gives $t_q = \Omega(\lg_\beta n) = \Omega(\lg n/\lg(w t_u))$.

And there is the wall, staring at me. The chronogram extracts *one* cell per epoch. One. That is precisely a single logarithm. To break the barrier I need to force not one but $\Omega(\lg_\beta n)$ probes into *each* epoch — then the sum is $\Omega(\lg_\beta^2 n) = \Omega((\lg n/\lg(w t_u))^2)$, which at $w=\Theta(\lg n)$ and poly-log $t_u$ is $\Omega((\lg n/\lg\lg n)^2)$, the quadratic I'm after. So the real question is: how do I argue that *many* cells per epoch are needed, not just one?

There is a route that does this, due to Pătraşcu. Let me trace it, because I think its trouble is going to tell me what to invent. The idea is to drop a *static* lower bound into each epoch. Freeze attention on epoch $i$ and set up a communication game: Alice holds the queries and the updates of the epochs preceding epoch $i$, Bob holds all the updates; together they must compute the answers. Alice simulates the query algorithm, and whenever it asks for a cell associated to epoch $i$, she asks Bob for the contents. The communication is then proportional to the number of probes into $S_i$, so a communication lower bound becomes a per-epoch probe lower bound. The catch is that to run the simulation Alice also needs the cells *not* associated to epoch $i$ without asking Bob. The cells of earlier epochs Bob just sends her wholesale — for $\beta$ large that's affordable. But she also needs to know which probed cells belong to epoch $i$ (those she must ask about) versus which are future-epoch cells she can recompute. So Bob sends a Bloom filter of the addresses of the epoch-$i$ cells, at false-positive rate $1/\lg^c n$, which costs $O(\lg\lg n)$ bits per epoch-$i$ cell.

Now I see exactly where this strains. Bob's Bloom filter has to cost fewer bits than it takes to describe his own input — epoch $i$'s updates — or there's no contradiction. But epoch $i$ can have as many as $\beta^i t_u$ associated cells, so the Bloom filter costs on the order of $\beta^i t_u \lg\lg n$ bits. For that to be dominated by the information in epoch $i$, each update has to carry a *lot* of bits. That is why this argument only works when the weights are $\lg^{2+\varepsilon}n$-bit integers, and since a cell has to be able to hold a weight, only when $w=\lg^{2+\varepsilon}n$. At $w=\Theta(\lg n)$ with $\Theta(\lg n)$-bit weights — the natural regime, the one I care about — it gives nothing. The overhead is per cell, and there are too many cells. Pătraşcu himself flags this and poses removing it as an open problem.

So the obstruction is sharp and I can name it: *the decoder has to be told which of the epoch's cells matter, and that costs bits proportional to the number of those cells.* If I could avoid pointing at all of them — if I only had to commit to a tiny handful of epoch-$i$ cells and still extract a lot of information — the per-cell tax would vanish. Is there any tool that produces a *small* set of cells carrying a lot of answers?

There is, and it has only ever been used statically: cell sampling, from the near-neighbor lower bounds of Panigrahy, Talwar, and Wieder. Let me recall the shape of it. You have a $t$-probe static structure with $m$ cells. You sample a random subset of about $m/\Phi^{1/t}$ cells. Then a $1/\Phi$ fraction of all queries are answered while probing *only* sampled cells — say such a query is *resolved* by the sample. And the argument is a pure counting one: a resolved query's answer is a function of the small sample, so across all resolved queries the answers are determined by the sample's bits; if the sample is too small relative to the information those answers carry, no correct structure can exist. The numbers come out of a union bound over the possible contents of the sample. That is exactly the shape I want — a *small* cell set that resolves *many* queries — and it sidesteps the "name every cell" problem because I never enumerate all the epoch's cells, only the sample.

Let me try to drop cell sampling into one epoch of the chronogram and see what survives. Fix epoch $i$. Suppose, for contradiction, the query algorithm probes only $o(\lg_\beta n)$ cells of $S_i$ on average over queries. I want to find a small subset $C_i \subseteq S_i$ and a large set $Q$ of queries each resolved by $C_i$ — meaning each $q\in Q$ probes no cell of $S_i\setminus C_i$. If I have that, I'll run an encoding argument: from the claimed cheap data structure I'll encode epoch $i$'s updates in fewer bits than their entropy, which is impossible.

Let me first nail the sampling step quantitatively, in the cleanest possible setting before I deal with the geometry of range counting. Say I pick a uniform random subset $C_i'$ of exactly $\beta^{i-1}$ cells from $S_i$. Let $Q'$ be the queries with $t_i(U,q)\le \tfrac14\lg_\beta n$ probes into $S_i$ — by Markov, since the average is $o(\lg_\beta n)$, this $Q'$ is a constant fraction of all queries. Take a $q\in Q'$ probing some $a \le \tfrac14\lg_\beta n$ cells of $S_i$. The probability that all $a$ of those cells land in my random $C_i'$ is

$$\frac{\binom{|S_i|-a}{\beta^{i-1}-a}}{\binom{|S_i|}{\beta^{i-1}}} = \frac{\beta^{i-1}(\beta^{i-1}-1)\cdots(\beta^{i-1}-a+1)}{|S_i|(|S_i|-1)\cdots(|S_i|-a+1)}.$$

Now $|S_i| \le \beta^i t_u$ — at most $t_u$ cells per update, $\beta^i$ updates. So each factor is at least $(\beta^{i-1}-a)/(\beta^i t_u)$, and with $a\le \tfrac14\lg_\beta n$ this is at least $\big(\tfrac{1}{2\beta t_u}\big)^{\frac14\lg_\beta n}$ once $\beta^{i-1}\gg \lg_\beta n$. Taking logs: that's $2^{-\frac14\lg_\beta n\,\lg(2\beta t_u)}$. If $\beta = (w t_u)^2$ then $\lg(2\beta t_u) \le 2\lg\beta$, so the exponent is at least $-\tfrac12\lg_\beta n\,\lg\beta = -\tfrac12\lg n$, i.e. the probability is at least $n^{-1/2}$. So the expected number of queries in $Q'$ resolved by my random $\beta^{i-1}$-cell sample is $\Omega(n^2)\cdot n^{-1/2} = \Omega(n^{3/2})$, and therefore *some* choice of $C_i$ of size $\beta^{i-1}$ resolves $\Omega(n)$ queries. There it is: a small set of cells, only $\beta^{i-1}$ of them, resolving a huge set of queries. The per-cell tax is gone — I encode $\beta^{i-1}$ cells, not all $\beta^i t_u$ of them.

But now the dynamic setting bites me where the static one didn't. A resolved query in a *static* structure reads nothing outside the sample, full stop — so a decoder can blindly re-simulate every query and just keep the ones that never leave the sample. In my dynamic structure, a query resolved by $C_i$ reads no *epoch-$i$* cell outside $C_i$, but it still reads cells of other epochs. Future epochs (smaller, later) I'm fine with — there are few of them and I can recompute them. Earlier epochs I can have the encoder ship outright. But here's the thing that kills the free re-simulation: if I just try every query, a query that is *not* resolved by $C_i$ will, when I simulate it, ask for some epoch-$i$ cell outside $C_i$ that I don't have — and I won't be able to tell that this happened, because I'd silently feed it whatever stale contents I do have and get a wrong answer with no flag. So unlike the static case, I cannot recover the set of good queries for free. **The encoder must explicitly tell the decoder which queries to simulate.** That is an extra cost the static argument never paid.

This is a real constraint, and I should respect it rather than fight it: writing down a query costs bits. If a query is described by an index into a family of $|V|$ queries, that's $\lg|V|$ bits per query I commit to. So for the whole scheme to net me something, *answering a query must yield more bits than it costs to name it.* That is the design pressure that will shape the hard instance: I want the output of a query — a weighted sum — to carry more entropy than $\lg|V|$. With $\Theta(\lg n)$-bit weights I have room: if a query is named with $2\lg n$ bits but its answer reveals a $4\lg n$-bit quantity, I'm ahead. Good — let me hold that inequality and build the instance to satisfy it. (And I should note the flip side now, because it's the boundary of the whole technique: if a query's output were a single bit, naming the query would cost more than the answer reveals, and this approach would collapse. The technique is going to be intrinsically about $\Omega(\lg n)$-bit-output problems.)

Before I tangle with two-dimensional geometry, let me build an *artificial* range-counting problem with exactly the independence I need, so I can run the whole encode/decode end to end and see it work. What property do I want from the queries? When I learn the answers to a set of queries, I want them to pin down the epoch's weights — and not just a few weights, but as many as the epoch is large. The clean way to say "the answers determine the weights" is: the answer to query $q$ is a linear functional of the weight vector, and the functionals for my chosen queries are *linearly independent* over some field, so I can solve for the weights. And I want this independence to survive even when I look only at the coordinates touched by a large epoch — the *last* $k$ coordinates of the weight vector.

So I'll ask for a family $V$ of $n^2$ $\{0,1\}$-vectors in $[\Delta]^n$, with the property that for every $\sqrt n\le k\le n$, *any* subset of up to $k/(22\lg k)$ of them are linearly independent when I restrict to the last $k$ coordinates. Here $\Delta$ is a prime with $n^4/2\le\Delta\le n^4$, so $[\Delta]$ is a finite field — I need a field, because I'm going to invert a matrix to recover the weights, and over the integers I'd be stuck. Each vector $v_j$ is a query: the answer is $\sum_i v_{j,i}\,\delta_i$, the sum of the weights at the coordinates where $v_j$ is $1$. An update sets a weight; a query is named by $j\in[n^2]$, costing $2\lg n$ bits, while a weight is in $[\Delta]$, carrying $\lg\Delta\le 4\lg n$ bits. So a query that lets me read off a weight nets me bits — exactly the inequality I wanted.

Does such a $V$ exist? Let me build it greedily and check with a union bound. Start with $V$ empty and keep adding $\{0,1\}$-vectors as long as the property holds. Take a uniform random $v\in\{0,1\}^n$. Fix $\sqrt n\le k\le n$ and a set $V'$ of up to $k/(22\lg k)$ vectors already in $V$. The probability that $v$, restricted to the last $k$ coordinates, lies in the span of $V'$ over $[\Delta]$ is at most $\Delta^{|V'|}/2^k$ — the span has at most $\Delta^{|V'|}$ points and $v$'s last $k$ bits are uniform in a set of size $2^k$. Now $\Delta^{|V'|} = 2^{|V'|\lg\Delta} \le 2^{(k/22\lg k)\cdot 4\lg n}$, and since $k\ge\sqrt n$ we have $\lg n \le 2\lg k$, so the exponent is at most $(k/22\lg k)\cdot 8\lg k = 8k/22 < k/2$. Thus the probability is at most $2^{k/2}/2^k = 2^{-k/2}$. How many such sets $V'$ are there? At most $\sum_{a\le k/22\lg k}\binom{n^2}{a}$, which is below $2^{k/3}$ (the binomials are tiny because $a$ is so much smaller than $n^2$; with $n^2\le k^8$ in the range that matters, $\binom{k^8}{k/22\lg k}\le (22e k^7\lg n)^{k/22\lg k+1} < 2^{k/3}$). Union bound: the chance $v$ is in the span of *some* bad set on the last $k$ coordinates is at most $2^{k/3-k/2} = 2^{-k/6}$, and a further union over the $\le n$ values of $k$ still leaves room. So a good $v$ exists to add, and I can grow $V$ all the way to $n^2$. The constant $22$ is just slack to make these inequalities go through; nothing magical.

Now the chronogram on this artificial problem. Hard distribution: for $i=0,\dots,n-1$ set the weight of point $p_i$ to a uniform random $\delta_i\in[\Delta]$, then ask a uniform random query $v\in V$. Epochs of geometrically decreasing size from the end: epoch $1$ is the last $\beta$ updates, epoch $i$ the $\beta^i$ updates preceding epoch $i-1$, with $\beta=(w t_u)^2$. Let $\mathbf U_i$ be epoch $i$'s updates, $\mathbf U$ all of them. Associate cells to last-writing epoch, $S_i(\mathbf U)$ the associated set; $t_i(\mathbf U,v)$ the probes into $S_i$ when answering $v$; $t_i(\mathbf U)$ their average over $v\in V$.

The claim I want is: with $\beta=(wt_u)^2$, $\mathbb E[t_i(\mathbf U,\mathbf v)] = \Omega(\lg_\beta n)$ for every epoch in $\tfrac23\lg_\beta n \le i < \lg_\beta n$. Granting that, since the $S_i$ are disjoint the total probes are $\sum_i t_i(\mathbf U,\mathbf v)$, and linearity of expectation over the $\Omega(\lg_\beta n)$ large epochs gives $\Omega(\lg_\beta n\cdot\lg_\beta n)=\Omega((\lg n/\lg(wt_u))^2)$. So the entire game is in proving the per-epoch claim.

Suppose it fails: there's an epoch $i^*$ in that range with $\mathbb E[t_{i^*}(\mathbf U,\mathbf v)]=o(\lg_\beta n)$. I'll encode $\mathbf U_{i^*}$ in fewer than $H(\mathbf U_{i^*})$ bits in expectation, conditioned on the later epochs $\mathbf U_{\lg_\beta n}\cdots\mathbf U_{i^*+1}$. First, the entropy: $\mathbf U_{i^*}$ is independent of the later epochs (they were chosen earlier in time, before these random weights), so $H(\mathbf U_{i^*}\mid \text{later}) = H(\mathbf U_{i^*}) = \beta^{i^*}\lg\Delta$ — $\beta^{i^*}$ fixed points each given a uniform weight in $[\Delta]$.

The encoder, knowing all of $\mathbf U$, runs the structure, simulates the query algorithm on every $v\in V$, and computes $t_{i^*}(\mathbf U)$. It first writes a flag bit. If $t_{i^*}(\mathbf U) > 2\,\mathbb E[t_{i^*}(\mathbf U,\mathbf v)]$ — a bad draw — it writes a $1$ and just spells out all of $\mathbf U_{i^*}$ in $\lceil \beta^{i^*}\lg\Delta\rceil = H(\mathbf U_{i^*})+O(1)$ bits, reading the weights as one big integer in $[\Delta^{\beta^{i^*}}]$. No saving on this branch, but I'll show it's the rare branch.

If $t_{i^*}(\mathbf U)\le 2\,\mathbb E[t_{i^*}(\mathbf U,\mathbf v)] = o(\lg_\beta n)$ — the good branch — it writes a $0$ and now exploits the cell sample. By the sampling argument above there's a $C_{i^*}\subseteq S_{i^*}$ with $|C_{i^*}|=O(\beta^{i^*-1})$ resolving a set $Q$ of $\Omega(n)$ queries (no resolved query touches $S_{i^*}\setminus C_{i^*}$); the encoder finds them by brute-force search over all candidate sets, since with the full structure in hand it can verify the resolving property. It writes $C_{i^*}$ — addresses and contents — for $O(w)+2|C_{i^*}|w$ bits. Then it picks $\beta^{i^*}/(22\lg\beta^{i^*})$ queries out of $Q$ — call this $V'$ — and writes their indices, costing $(\beta^{i^*}/22\lg\beta^{i^*})\lg|V| \le (\beta^{i^*}/22\lg(n^{2/3}))\cdot 2\lg n = \tfrac{3}{22}\beta^{i^*}$ bits (using $i^*\ge\tfrac23\lg_\beta n$ so $\beta^{i^*}\ge n^{2/3}$). The number of queries I keep is set exactly to the independence threshold of the family on the last $k_{i^*}$ coordinates, where $k_{i^*}=\sum_{j\le i^*}\beta^j$ is the total length of epochs $\le i^*$ — so $|V'|=\beta^{i^*}/22\lg\beta^{i^*}$ vectors are linearly independent there.

Now the linear-algebra payload, which is the actual transfer of information. The answers to the queries in $V'$ involve all the weights, but I only want to commit to the part determined by epochs $\le i^*$, i.e. the last $k_{i^*}$ weights, gathered into a vector $\mathbf u\in[\Delta]^{k_{i^*}}$. The encoder builds a full basis of $[\Delta]^{k_{i^*}}$: start with $V'$ restricted to the last $k_{i^*}$ coordinates (these are independent, dimension $|V'|$), then walk through $[\Delta]^{k_{i^*}}$ in a fixed order adding any vector not yet in the span until the span is everything. For each *added* vector $x$ it writes $\langle \mathbf u, x\rangle \bmod\Delta$. The $V'$ part of the basis costs nothing to transmit — those inner products are exactly the (modular) query answers the decoder will reconstruct on its own — so the payload is only for the $k_{i^*}-|V'|$ completion vectors: $\lceil(k_{i^*}-\beta^{i^*}/22\lg\beta^{i^*})\lg\Delta\rceil$ bits. Finally the encoder ships all earlier-epoch cell sets $S_{i^*-1}(\mathbf U),\dots,S_1(\mathbf U)$ with addresses and contents, $\sum_{j<i^*}(2|S_j(\mathbf U)|w+O(w))$ bits, and sends.

Can the decoder rebuild $\mathbf U_{i^*}$? It knows the later epochs (we conditioned on them) and the message. If the flag is $1$, done immediately. If $0$: first it re-executes the later epochs $\mathbf U_{\lg_\beta n}\cdots\mathbf U_{i^*+1}$ on the structure to get the cells *as they were just after those epochs* — call them $S_j^{i^*+1}(\mathbf U)$ for $j>i^*$. It recovers $V'$, $C_{i^*}$, and $S_{i^*-1},\dots,S_1$ from the message. Now for each $v\in V'$ it simulates the query algorithm and, for each cell $c$ requested, reconstructs $c$'s contents *as if all updates had run*: if $c$ is in one of $S_{i^*-1},\dots,S_1$ it has them from the message; else if $c\in C_{i^*}$ it has them from the message; else — and this is the payoff of resolving — $c$ is *not* in $S_{i^*}$ at all (a resolved query touches no $S_{i^*}\setminus C_{i^*}$ cell), so $c$ was last written in some later epoch and its contents are exactly what the decoder already computed by re-executing the later epochs. So the decoder computes the true answer $\sum_{m=0}^{n-1} v_m\,\delta_m$ for every $v\in V'$.

But I want the part of that sum coming from epochs $\le i^*$ only — i.e. $\sum_{m=0}^{k_{i^*}-1} v_{n-k_{i^*}+m}\,\mathbf u_m$. The decoder gets it by subtracting off the later-epoch contribution $\sum_{m=0}^{n-k_{i^*}-1} v_m\,\delta_m$, which it can compute because those $\delta_m$ are given (they're in the conditioned-on later epochs). Reduce mod $\Delta$ and that's $\langle \mathbf u, x\rangle\bmod\Delta$ for $x = v$ restricted to the last $k_{i^*}$ coordinates. So for every basis vector — whether it came from $V'$ (answer reconstructed) or was a completion vector (inner product read from the message) — the decoder holds $\langle \mathbf u, x\rangle\bmod\Delta$. Assemble the basis as the rows of a $k_{i^*}\times k_{i^*}$ matrix $A$ and the inner products into a vector $z$; since $A$ has full rank over the field $[\Delta]$, the system $A\otimes\mathbf y = z$ (matrix–vector product mod $\Delta$) has a unique solution, and $\mathbf u$ is a solution, so solving recovers $\mathbf u$ — hence $\mathbf U_{i^*}$ (and indeed $\mathbf U_{i^*}\cdots\mathbf U_1$). The decoding works.

Now the size, the moment of truth. On the good branch the message is

$$2|C_{i^*}|w + \tfrac{3}{22}\beta^{i^*} + (k_{i^*}-\beta^{i^*}/22\lg\beta^{i^*})\lg\Delta + \sum_{j<i^*}2|S_j(\mathbf U)|w + O(w\lg_\beta n).$$

Use $H(\mathbf U_{i^*})=\beta^{i^*}\lg\Delta = k_{i^*}\lg\Delta - \sum_{j<i^*}\beta^j\lg\Delta$ to rewrite $k_{i^*}\lg\Delta = H(\mathbf U_{i^*})+\sum_{j<i^*}\beta^j\lg\Delta$. Also $|C_{i^*}|w = O(\beta^{i^*-1}w)$, $\lg\Delta=O(w)$, and $|S_j(\mathbf U)|\le\beta^j t_u$. The message is at most

$$H(\mathbf U_{i^*}) + \tfrac{3}{22}\beta^{i^*} - \frac{\beta^{i^*}}{22\lg\beta^{i^*}}\lg\Delta + O\!\Big(\sum_{j<i^*}\beta^j w t_u\Big).$$

The error term: $\sum_{j<i^*}\beta^j w t_u = O(\beta^{i^*-1}w t_u)$, and since $\beta=(wt_u)^2$ this is $O(\beta^{i^*}/(wt_u)) = o(\beta^{i^*})$. And $\lg\Delta\ge\lg(n^4/2)\ge 4\lg(\beta^{i^*})-1$ because $\beta^{i^*}\le n$, so $\frac{\beta^{i^*}}{22\lg\beta^{i^*}}\lg\Delta \ge \frac{4}{22}\beta^{i^*}-o(\beta^{i^*})$. Therefore the good-branch message is at most

$$H(\mathbf U_{i^*}) + \tfrac{3}{22}\beta^{i^*} - \tfrac{4}{22}\beta^{i^*} + o(\beta^{i^*}) = H(\mathbf U_{i^*}) - \Omega(\beta^{i^*}).$$

The completion payload — paying $\lg\Delta\ge 4\lg\beta^{i^*}$ bits per query I *don't* have to send because I sent the query index ($\tfrac{3}{22}\beta^{i^*}$ total) instead — buys back strictly more than it cost. That's the engine: each named query lets me drop a full $\lg\Delta$-bit weight's worth from the basis completion, and $\lg\Delta$ beats the per-query naming cost because the output is wider than the query description. This is exactly the inequality I set up the instance to satisfy.

Finish with the rare branch. Since $\mathbf v$ is uniform, $\mathbb E[t_{i^*}(\mathbf U)] = \mathbb E[t_{i^*}(\mathbf U,\mathbf v)]$, so Markov gives $\Pr[t_{i^*}(\mathbf U)>2\mathbb E[t_{i^*}(\mathbf U,\mathbf v)]] < \tfrac12$. The expected message length is then below

$$O(1) + \tfrac12 H(\mathbf U_{i^*}) + \tfrac12\big(H(\mathbf U_{i^*})-\Omega(\beta^{i^*})\big) < H(\mathbf U_{i^*}),$$

contradicting that I cannot encode $\mathbf U_{i^*}$ below its entropy. So the per-epoch claim holds, and the artificial problem has the $\Omega((\lg n/\lg(wt_u))^2)$ bound. The whole machine ran. Now I have to do it for honest weighted orthogonal range counting, where I don't get to *design* the queries to be independent — they're forced to be dominance queries $q=(x,y)$, and the points are forced to be a 2-D set. The independence has to come from geometry.

For the real problem the hard distribution again uses epochs of geometrically decreasing size, $\beta^i$ updates in epoch $i$, all run before one uniform random query in $[n]\times[n]$. The question is which points to insert in each epoch. I want the per-epoch point set to be so *uniform* that a spread-out set of dominance queries automatically has independent answer-functionals. The bit-reversal permutation does this in Pătraşcu's static proof, but I find I can argue it more geometrically with the **Fibonacci lattice** $F_m=\{(j, jf_{k-1}\bmod m): j=0,\dots,m-1\}$, $m=f_k$. Its defining virtue (Fiat–Shamir, via Matoušek's discrepancy theory): scaled to $[0,n]^2$, *every* axis-aligned rectangle of area $\alpha n^2/\beta^i$ holds between $\lfloor\alpha/a_1\rfloor$ and $\lceil\alpha/a_2\rceil$ points, $a_1\approx1.9$, $a_2\approx0.45$. So if a rectangle has area a few times $n^2/\beta^i$, it is *guaranteed* to contain a lattice point, and not too many. So epoch $i$ inserts the scaled $F_{\beta^i}$, each point a uniform weight in $[\Delta]$, $\Delta$ the largest prime below $n^4$. (I assume $\beta^i$ is a Fibonacci number dividing $n$; this is cosmetic and removable by fiddling constants.)

Let me set the algebra up to mirror the artificial case. For a query $q=(x,y)$ and epoch $i$ define the *incidence vector* $\chi_i(q)\in\{0,1\}^{\beta^i}$: its $j$-th coordinate is $1$ iff the $j$-th point inserted in epoch $i$ is dominated by $q$, i.e. $jn/\beta^i\le x$ and $(jf_{k_i-1}\bmod\beta^i)n/\beta^i\le y$. Let $\mathbf u_i\in[\Delta]^{\beta^i}$ collect epoch $i$'s weights. Then the answer to $q$ after all updates is exactly $\sum_{i=1}^{\lg_\beta n}\langle\chi_i(q),\mathbf u_i\rangle$ — a signed (well, weighted) sum over all epochs of how the query sees each epoch's points. This is the information-transfer structure: the query is a single linear combination spanning every epoch, and the decoder will *difference out* the epochs it knows to expose the one it wants. To extract $\langle\chi_{i^*}(q),\mathbf u_{i^*}\rangle$ it subtracts $\sum_{i\ne i^*}\langle\chi_i(q),\mathbf u_i\rangle$: the later epochs come from re-execution and the conditioning, the earlier epochs from the encoder's shipped cells and weights. Same skeleton as before.

So everything reduces, as in the artificial case, to one static structural lemma: if epoch $i$ ($i\ge\tfrac{15}{16}\lg_\beta n$) is probed cheaply on average, $t_i(U)=o(\lg_\beta n)$, then there exist a small cell set $C_i\subseteq S_i$ with $|C_i|=O(\beta^{i-1}w)$ and a query set $Q\subseteq[n]^2$ with $|Q|=\Omega(\beta^{i-3/4})$ such that (a) the incidence vectors $\chi_i(Q)$ are linearly independent over $[\Delta]$, and (b) every $q\in Q$ probes no cell of $S_i\setminus C_i$. With that, the encode/decode and the size analysis go through almost verbatim — I'll check the arithmetic at the end. The hard part, the genuinely new geometry, is producing $Q$: many queries, resolved by a tiny cell set, with independent incidence vectors. Let me build it in two moves.

First move: turn "many spread-out queries" into "independent incidence vectors." When are dominance queries' incidence vectors independent? If I can *order* the queries $q_1,\dots,q_{|Q|}$ so that each $q_j$ dominates some lattice point that *none* of $q_1,\dots,q_{j-1}$ dominates, then each $\chi_i(q_j)$ has a coordinate equal to $1$ where all earlier ones are $0$ — a staircase pattern — and that forces linear independence. To get such private points I'll use a grid and a crossing-out trick. Take a grid $G$ of width $\mu$ and height $n^2/(\beta^i\mu)$ (so each grid cell has area $n^2/\beta^i$, the unit at which the Fibonacci lemma guarantees a point), with $n/\beta^i\le\mu\le n$. Let the *hitting number* $h$ be the number of distinct grid cells hit by my query set $Q'$. I claim I can extract from $Q'$ an independent subset $Q$ of size $\Omega(h - 6n/\mu - 6\mu\beta^i/n)$.

Construct it: keep one query per hit cell. Then cross out the bottom two rows and the leftmost two columns entirely (delete all queries hitting those). Among the remaining columns, cross out whichever of the even/odd columns holds fewer surviving queries; repeat once more on the columns; then do the same twice for the rows. Now order the survivors by column index, then by row index, giving $q_1,\dots,q_{|Q|}$. I claim each $q_j$ has a private dominated lattice point. Associate to $q_j$ the two-by-two block of crossed-out cells immediately to its bottom-left. Those four cells have total area $4n^2/\beta^i$ and (being below-left of a surviving cell, hence inside $[0,n-n/\beta^i]^2$) the Fibonacci area lemma guarantees at least one lattice point inside — which $q_j$ dominates. Does any earlier $q_k$, $k<j$, dominate that point? Earlier in the order means either a column index at least three smaller (I crossed out the two columns just left of $q_j$'s, so consecutive survivors differ by $\ge3$ columns) or the same column but a row index at least three smaller (same crossing on rows). Either way $q_k$ lies too far down-left to reach into $q_j$'s bottom-left block. So the point is private to $q_j$, and the incidence vectors are independent.

Size accounting: I started with $|Q|=h$, one per hit cell. The bottom two rows have area $2\cdot\mu\cdot(n^2/\beta^i\mu)\cdot(n/\mu) = 2n^3/(\beta^i\mu)$... let me just count points by area: bottom two rows together have area $2n^3/(\beta^i\mu)$, so by the Fibonacci lemma at most $6n/\mu$ lattice points, hence at most that many hit cells lost; the left two columns have area $2n\mu$, at most $6\mu\beta^i/n$ points lost. That leaves $\ge h-6n/\mu-6\mu\beta^i/n$. The even/odd crossings each keep the larger half, so they cost at most a factor $16$ total. So $|Q|=\Omega(h-6n/\mu-6\mu\beta^i/n)$, as claimed. The geometry has converted spread into algebraic independence.

Second move: produce a small cell set $C_i$ whose resolved queries hit *many* grid cells. I'll define a family of grids $G_2,\dots,G_{2i-2}$ where $G_j$ has width $n/\beta^{i-j/2}$ and height $n/\beta^{j/2}$ — same cell area $n^2/\beta^i$, but the aspect ratio sweeps from tall-thin to short-wide as $j$ grows. The reason for a *family* rather than one grid: whatever shape the resolved query set takes, one of these aspect ratios will catch its spread as a large hitting number. The lemma I want: if $t_i(U)=o(\lg_\beta n)$, there's a $C_i$, $|C_i|=O(\beta^{i-1}w)$, and an index $j\in\{2,\dots,2i-2\}$ with the resolved set $Q_{C_i}$ having hitting number $\Omega(\beta^{i-3/4})$ on $G_j$.

I prove this by *another* encoding argument, but now I encode a random *query set*, not an update sequence. Draw $\mathbf Q$ by picking one uniform query from each of the $\beta^{i-1}$ vertical slabs $[hn/\beta^{i-1},(h+1)n/\beta^{i-1})\times[0,n)$, $h\in[\beta^{i-1}]$. Its entropy is $H(\mathbf Q)=\beta^{i-1}\lg(n^2/\beta^{i-1})$. Suppose, for contradiction, that for *every* small $C$ (size $O(\beta^{i-1}w)$) and every $j$, the hitting number of $Q_C$ on $G_j$ is $o(\beta^{i-3/4})$. I'll encode $\mathbf Q$ below its entropy.

I need $\mathbf Q$ to be *well-separated*: no two of its queries inside a common rectangle of area $n^2/\beta^{i-1/2}$ (call a query well-separated if it's separated from all others; the set well-separated if at least half its queries are). Why this holds with good probability: fix the slab-$h$ query $\mathbf q_h$. The chance it lands within $n/\beta^{i-3/4}$ of a slab boundary is $(2n/\beta^{i-3/4})/(n/\beta^{i-1})=2/\beta^{1/4}$. Otherwise, for another query $\mathbf q_k$, the $x$-gap is at least $(|k-h|-1)n/\beta^{i-1}+n/\beta^{i-3/4}$, so the two can share an area-$n^2/\beta^{i-1/2}$ rectangle only if their $y$-coordinates differ by at most $n/((|k-h|-1)\beta^{1/2}+\beta^{1/4})$, an event of probability at most $2/((|k-h|-1)\beta^{1/2}+\beta^{1/4})$. Summing over $k$:

$$\frac{2}{\beta^{1/4}} + \sum_{k\ne h}\frac{2}{(|k-h|-1)\beta^{1/2}+\beta^{1/4}} = O\!\Big(\frac{1}{\beta^{1/4}}+\frac{\lg n}{\beta^{1/2}}\Big),$$

which is $o(1)$ since $\beta=(wt_u)^9 = \omega(\lg^2 n)$. (This is exactly why I want $\beta$ a *large* power of $wt_u$ here — the well-separation slack needs $\beta=\omega(\lg^2 n)$, more than the artificial case demanded; the exponent $9$ is just a convenient value that makes every slack term vanish, nothing tighter is intended.) By linearity and Markov, $\mathbf Q$ is well-separated with probability $\ge 3/4$; combined with $\mathbb E|S_i(\mathbf Q,U)|=\beta^{i-1}t_i(U)$ (a uniform query from $\mathbf Q$ is uniform in $[n]^2$), Markov and a union bound give: with probability $\ge1/2$, $\mathbf Q$ is well-separated *and* $|S_i(\mathbf Q,U)|\le 4\beta^{i-1}t_i(U)$, where $S_i(\mathbf Q,U)$ is the set of epoch-$i$ cells probed across all of $\mathbf Q$.

The encoding of a good $\mathbf Q$: write the not-well-separated queries (at most half) explicitly, $O(w)+O(\beta^{i-1})+|\mathbf Q'|\lg(n^2/\beta^{i-1})$ bits. Then write the cell set $S_i(\mathbf Q,U)$ — *addresses only, not contents* — costing $\lg\binom{|S_i(U)|}{|S_i(\mathbf Q,U)|}$. Bound it: $|S_i(U)|\le\beta^i t_u$ and $|S_i(\mathbf Q,U)|\le 4\beta^{i-1}t_i(U)$, so this is $O(\beta^{i-1}t_i(U)\lg(\beta t_u)) = o(\beta^{i-1}\lg(n^2/\beta^{i-1})) = o(H(\mathbf Q))$, using $t_i(U)=o(\lg_\beta n)$. The point of sending only addresses: the decoder, who has $U$, already knows the contents — it just needs to know *which* cells form the sample so it can identify which queries are resolved by it. Next encode the $x$-coordinates of the well-separated queries as offsets within their slabs, and their $y$-offsets within the $G_{2i-2}$ cell they hit, $(|\mathbf Q|-|\mathbf Q'|)\lg(n^2/\beta^{2i-2})+O(1)$ bits. Finally, the crux: the encoder simulates *all* queries in $[n]^2$ to find $Q_{S_i(\mathbf Q,U)}$, the set resolved by the sample (note $\mathbf Q\subseteq Q_{S_i(\mathbf Q,U)}$), and for each grid $G_j$ writes which of the *hit cells of $Q_{S_i(\mathbf Q,U)}$* are also hit by the well-separated part of $\mathbf Q$ — $\sum_{j=2}^{2i-2}\lg\binom{|G_j^{Q_{S_i(\mathbf Q,U)}}|}{|G_j^{\mathbf Q}|}$ bits.

Here the contradictory assumption pays off. Since $|S_i(\mathbf Q,U)|=o(\beta^{i-1}\lg_\beta n)=o(\beta^{i-1}w)$, the assumed bad hitting number says $|G_j^{Q_{S_i(\mathbf Q,U)}}|=o(\beta^{i-3/4})$ for every $j$. So each $\lg\binom{\cdot}{\cdot}$ is small, and summing over the $\le 2i$ grids:

$$(|\mathbf Q|-|\mathbf Q'|)\lg\!\big(\beta^{i-3/4}e/(|\mathbf Q|-|\mathbf Q'|)\big)(2i-3) \le (|\mathbf Q|-|\mathbf Q'|)\tfrac14\lg\beta\cdot 2\lg_\beta n + O(\beta^{i-1}\lg_\beta n) \le (|\mathbf Q|-|\mathbf Q'|)\tfrac12\lg n + o(H(\mathbf Q)).$$

Can the decoder rebuild $\mathbf Q$? It has $U$, re-executes it, recovers $\mathbf Q'$, recovers the *addresses* of $S_i(\mathbf Q,U)$ (contents it knows), and so simulates every query in $[n]^2$ to reconstruct $Q_{S_i(\mathbf Q,U)}$ and each $G_j^{Q_{S_i(\mathbf Q,U)}}$. From those plus the written subsets it recovers each $G_j^{\mathbf Q}$. In $G_2$ there's at most one query per column, so $G_2^{\mathbf Q}$ pins down each well-separated query's $G_2$ cell. Then it climbs the aspect-ratio ladder: the rectangle enclosing the $\beta^{1/2}$ cells of $G_{j+1}$ meeting one cell of $G_j$ has area $n^2/\beta^{i-1/2}$, and well-separation forbids two $\mathbf Q$-queries in such a rectangle, so $G_{j+1}^{\mathbf Q}$ has at most one cell per such group — letting the decoder refine each query's location from its $G_j$ cell to its $G_{j+1}$ cell, all the way to $G_{2i-2}$. With the $G_{2i-2}$ cell plus the stored $x$-offset and $y$-offset, every well-separated query is recovered, and $\mathbf Q$ with it.

The size of the good encoding:

$$|\mathbf Q'|\lg(n^2/\beta^{i-1}) + (|\mathbf Q|-|\mathbf Q'|)\big(\lg(n^2/\beta^{2i-2})+\tfrac12\lg n\big) + o(H(\mathbf Q)),$$

which regroups as $|\mathbf Q|\lg(n^{2+1/2}/\beta^{2i-2}) + |\mathbf Q'|\lg(\beta^{i-1}/n^{1/2}) + o(H(\mathbf Q))$. Because $i\ge\tfrac{15}{16}\lg_\beta n$, we have $\lg(n^{5/2}/\beta^{2i-2}) \le \lg(n^{5/8}\beta^2)$, so this is at most $|\mathbf Q|\lg(n^{5/8}\beta^2)+|\mathbf Q'|\lg(n^{1/2})+o(H(\mathbf Q))$, and with $|\mathbf Q'|\le\tfrac12|\mathbf Q|$ at most $|\mathbf Q|\lg(n^{7/8}\beta^2)+o(H(\mathbf Q))$. But $H(\mathbf Q)=|\mathbf Q|\lg(n^2/\beta^i)\ge|\mathbf Q|\lg n$, so the encoding uses below $\tfrac{15}{16}H(\mathbf Q)$ bits on the good event. Folding in the $\ge1/2$ probability of the good event, the expected length is $O(1)+\Pr[\text{good}](1-\Omega(1))H(\mathbf Q)+(1-\Pr[\text{good}])H(\mathbf Q) < H(\mathbf Q)$ — impossible. So the bad-hitting-number assumption fails, and a $C_i$ with hitting number $\Omega(\beta^{i-3/4})$ on some $G_j$ exists.

Now the two moves compose. With $t_i(U)=o(\lg_\beta n)$, the hitting lemma gives $C_i$, $|C_i|=O(\beta^{i-1}w)$, and a grid $G_j$ on which $Q_{C_i}$ hits $\Omega(\beta^{i-3/4})$ cells. That $G_j$ has the width $\mu=n/\beta^{i-j/2}$ required by the spread lemma, so I extract $Q\subseteq Q_{C_i}$ with $|Q|=\Omega(\beta^{i-3/4}-12\beta^{i-1})=\Omega(\beta^{i-3/4})$ and $\chi_i(Q)$ linearly independent. (The loss terms $6n/\mu$ and $6\mu\beta^i/n$ are both $\le 6\beta^{i-1}$ for grids in the family, negligible against $\beta^{i-3/4}$.) That is exactly the static structural lemma: small resolving cell set, large independent query set. Property (b) — queries in $Q$ probe no $S_i\setminus C_i$ — holds because $Q\subseteq Q_{C_i}$.

Finally the per-epoch encoding for the real problem, mirroring the artificial one with $\chi_{i^*}$ in place of the abstract query vectors. On the good branch the encoder writes $C_{i^*}$ (addresses + contents, $2|C_{i^*}|w$ bits), the query set $Q(\mathbf U)$ as a subset of $[n]^2$ ($\lg\binom{n^2}{|Q|}$ bits), then completes $\chi_{i^*}(Q)$ to a basis of $[\Delta]^{\beta^{i^*}}$ and writes the $(\beta^{i^*}-|Q|)$ extra inner products $\langle x,\mathbf u_{i^*}\rangle\bmod\Delta$, then ships $S_{i^*-1},\dots,S_1$ with the earlier weight vectors $\mathbf u_{i^*-1},\dots,\mathbf u_1$. The decoder reconstructs every $q\in Q$'s full answer $\sum_i\langle\chi_i(q),\mathbf u_i\rangle$ exactly as before (resolved $\Rightarrow$ any epoch-$i^*$ cell touched is in $C_{i^*}$; everything else is known), differences out the other epochs to isolate $\langle\chi_{i^*}(q),\mathbf u_{i^*}\rangle$, then with the completion inner products forms the full-rank system $A\otimes\mathbf y=z$ over $[\Delta]$ and solves for $\mathbf u_{i^*}$.

Size: the good-branch message is

$$2|C_{i^*}|w + \lg\binom{n^2}{|Q|} + (\beta^{i^*}-|Q|)\lg\Delta + O(w\lg_\beta n) + \sum_{j<i^*}(2|S_j|w+\beta^j\lg\Delta).$$

Use $\beta^{i^*}\lg\Delta=H(\mathbf U_{i^*})$ and $|C_{i^*}|w=O(\beta^{i^*-1}w)=o(|Q|)$ (since $|Q|=\Omega(\beta^{i^*-3/4})$ dwarfs $\beta^{i^*-1}$). The message is at most

$$H(\mathbf U_{i^*}) - |Q|\lg(\Delta/n^2) + o(|Q|) + \sum_{j<i^*}(2|S_j|w+\beta^j\lg\Delta).$$

The trailing sums are geometric and small: $\sum_{j<i^*}\beta^j\lg\Delta\le 2\beta^{i^*-1}\lg\Delta=o(|Q|\lg\Delta)$, and $\sum_{j<i^*}2|S_j|w\le 4\beta^{i^*-1}wt_u=o(|Q|)$ since $\beta=\omega(wt_u)$. And here is why the *weight* now only needs $\Theta(\lg n)$ bits and not $\lg^{2+\varepsilon}n$: the largest prime below $n^4$ is at least $n^3$ infinitely often, so $\lg(\Delta/n^2)=\Omega(\lg\Delta)$ — naming a query costs $\le 2\lg n$ bits but resolves a weight worth $\Omega(\lg\Delta)=\Omega(\lg n)$ bits, the same output-beats-description inequality, now at the natural weight size. So the message is at most

$$H(\mathbf U_{i^*}) - \Omega(|Q|\lg\Delta) = H(\mathbf U_{i^*}) - \Omega(\beta^{i^*-3/4}\lg\Delta).$$

The rare branch contributes $\Pr[\cdot](H(\mathbf U_{i^*})+O(1))$, with $\Pr[t_{i^*}(\mathbf U)>2\mathbb E[t_{i^*}(\mathbf U,\mathbf q)]]<\tfrac12$ by Markov (uniform query $\Rightarrow \mathbb E[t_{i^*}(\mathbf U)]=\mathbb E[t_{i^*}(\mathbf U,\mathbf q)]$). Expected length $< O(1)+\tfrac12 H(\mathbf U_{i^*})+\tfrac12(H(\mathbf U_{i^*})-\Omega(\beta^{i^*-3/4}\lg\Delta)) < H(\mathbf U_{i^*})$. Contradiction. So $\mathbb E[t_i(\mathbf U,\mathbf q)]=\Omega(\lg_\beta n)$ for every $i\ge\tfrac{15}{16}\lg_\beta n$, and summing the $\Omega(\lg_\beta n)$ large epochs over disjoint $S_i$ yields the main bound.

So the theorem stands: any cell-probe data structure for dynamic weighted orthogonal range counting in 2-D, with $\Theta(\lg n)$-bit weights, has $t_q=\Omega((\lg n/\lg(wt_u))^2)$ — at $w=\Theta(\lg n)$ and poly-logarithmic $t_u$ this is $\Omega((\lg n/\lg\lg n)^2)$, almost a quadratic jump over the old $\Omega(\lg n)$ ceiling, and tight once $t_u\ge\lg^{2+\varepsilon}n$. And it holds for any $\varepsilon\lg n$-bit weights, since a structure for $\varepsilon\lg n$-bit weights handles $O(\lg n)$-bit weights at a constant-factor cost by splitting the weight bits into $O(1)$ chunks and keeping one structure per chunk.

Let me retrace the causal chain so I'm sure nothing is propped up by hindsight. The chronogram decomposes a dynamic problem into disjoint epochs and forces one cell per epoch, capping at $\lg n$. Pushing to many cells per epoch by a static argument requires telling a decoder which epoch cells matter, and a Bloom filter charges per cell, which forces enormous weights. Cell sampling produces a *tiny* set of cells resolving *many* queries, so I commit to a handful, not all — killing the per-cell tax. But dynamic re-simulation reads other epochs' cells and can't silently detect a non-resolved query, so I must name the resolved queries; that costs $\lg(\text{queries})$ bits and forces the instance to make a query's *output* (a $\Theta(\lg n)$-bit weighted sum over a field $[\Delta]$) wider than its description. The answer is a sum over all epochs of $\langle\chi_i(q),\mathbf u_i\rangle$, so I difference out the known epochs to expose epoch $i^*$, and independence of the chosen incidence vectors lets me solve a full-rank linear system over $[\Delta]$ to recover the epoch's weights. The independence comes from geometry: the Fibonacci lattice's uniformity guarantees private dominated points for a spread query set, "spread" is certified by a large hitting number on one of a ladder of grids, and that hitting number is itself secured by an encoding argument against a random query set. Each piece earns its place by removing a specific failure of the piece before it.

```python
# Lower bound for dynamic weighted orthogonal range counting in the cell-probe model.
# This is a PROOF, not an algorithm: the "code" is the structure of the reduction
# and the static lemma, exactly the slots the context scaffold left open.

def hard_distribution(n, w, t_u, Delta):
    beta = (w * t_u) ** 9                          # large enough: beta = omega(lg^2 n)
    epochs = []                                    # epoch i inserts beta**i Fibonacci points
    for i in range(1, ilog(n, beta) + 1):          # geometrically decreasing, largest first
        pts = scaled_fibonacci_lattice(beta ** i, n)   # F_{beta^i} scaled into [n]x[n]
        weights = [uniform([Delta]) for _ in pts]      # uniform weights -> H(U_i)=beta^i lg Delta
        epochs.append(list(zip(pts, weights)))
    query = uniform_point([n], [n])                # one uniform random query at the end
    return epochs, query                           # Delta = largest prime < n**4

def incidence_vector(q, epoch_i_points):           # chi_i(q) in {0,1}^{beta^i}
    return [1 if dominated(p, q) else 0 for p in epoch_i_points]
# answer(q) = sum_i  < chi_i(q), u_i >   over the field [Delta]   (the information-transfer sum)

def static_lemma(S_i, ds, U, i, n, beta, w, Delta):
    # Premise: t_i(U) = o(lg_beta n). Produce (C_i, Q): small resolving cells, many
    # independent incidence vectors. Two moves below.
    C_i, j = hit_many(S_i, ds, U, i, n, beta, w)   # |C_i|=O(beta^{i-1} w); Q_{C_i} hits
                                                   #   Omega(beta^{i-3/4}) cells of grid G_j
    Q_resolved = {q for q in all_queries(n)        # resolved: probe no cell of S_i \ C_i
                  if ds.probes(q, U) & S_i <= C_i}
    Q = spread_to_independent(Q_resolved, grid(j, n, beta, i))  # staircase via 2x2 crossing
    return C_i, Q                                  # chi_i(Q) linearly independent over [Delta]

def hit_many(S_i, ds, U, i, n, beta, w):
    # Encoding argument on a RANDOM query set Q (one per vertical slab): if no small C
    # gave hitting number Omega(beta^{i-3/4}) on any G_j in {G_2..G_{2i-2}}, then a
    # well-separated Q compresses below H(Q) = beta^{i-1} lg(n^2/beta^{i-1}) -- impossible.
    raise Contradiction  # => such (C_i, j) must exist

def spread_to_independent(Q_resolved, G):
    Q = one_query_per_hit_cell(Q_resolved, G)
    cross_out(G, bottom_rows=2, left_cols=2)        # guarantee a private 2x2 block per query
    for _ in range(2): cross_out_sparser(G, axis="col")   # halve dependencies, keep larger half
    for _ in range(2): cross_out_sparser(G, axis="row")
    # order by (column, row): each q_j dominates a Fibonacci point in its bottom-left 2x2
    # block (area 4n^2/beta^i => >=1 lattice point) that no earlier q_k reaches.
    return ordered_survivors(Q, G)                  # |Q| = Omega(h - 6n/mu - 6 mu beta^i/n)

def encode(U, i_star, ds, n, w, t_u, Delta):
    beta = (w * t_u) ** 9
    if t_istar_avg(ds, U, i_star) > 2 * expected_probes(ds, i_star):   # rare branch
        return [1] + spell_out(U[i_star])                              # H(U_{i*}) + O(1) bits
    C, Q = static_lemma(S(ds, U)[i_star], ds, U, i_star, n, beta, w, Delta)
    basis, extra_inner = complete_to_basis(incidence_vectors(Q, U, i_star), U[i_star], Delta)
    return ([0] + write_cells(C) + write_subset(Q, n*n)               # < H(U_{i*}) - Omega(|Q| lg Delta)
            + write(extra_inner) + write_cells_and_weights(S(ds,U)[:i_star]))

def decode(message, future_epochs, ds, Delta):
    if message[0] == 1: return read_spelled_out(message)
    cells_below, C, Q, extra = parse(message)
    A, z = [], []
    for q in Q:                                     # reconstruct each resolved query's answer
        ans = simulate(ds, q, future_epochs, known={**cells_below, **C})   # resolved => no missing S_{i*} cell
        inner = ans - sum_other_epochs(q, future_epochs, cells_below)      # difference out other epochs
        A.append(incidence_vector(q, future_epochs)); z.append(inner % Delta)
    A, z = add_completion_rows(A, z, extra)         # full-rank beta^{i*} x beta^{i*} system
    return solve_linear_system_mod(A, z, Delta)     # unique u_{i*} over the field [Delta]

def lower_bound(n, w, t_u):
    # per-epoch: E[t_i(U,q)] = Omega(lg_beta n) for i >= (15/16) lg_beta n, else encode/decode
    #            beats H(U_i) -- contradiction.
    # disjoint S_i + linearity over Omega(lg_beta n) large epochs:
    return Omega((log(n) / log(w * t_u)) ** 2)      # = Omega((lg n / lg lg n)^2) at w=Theta(lg n)
```
