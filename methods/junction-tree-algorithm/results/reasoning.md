I start with the local conditional tables and I want a posterior marginal, but the marginal is still a global sum. On an ordinary tree I know what to do: cut an edge, summarize one side as a function of the shared variables, and pass that summary across. That works because once the separator is fixed, no other path carries the same dependence back around.

The first tempting move is to keep the original graph and run the same local messages anyway. Before I trust or reject that, I want to watch it fail on something I can compute by hand. Take a four-cycle A-B-C-D-A with pairwise potentials and a field on A, no chord. The closing edge D-A means A sits on both ends of the chain. If I pretend the cycle is the chain A-B-C-D and pass messages inward, then when I summarize the C-D-A end I have to sum out something before I reach A, and the natural thing the tree pass does is fold the D-A factor into the message while marginalizing A away early. I tried exactly that and read off P(A): I get [0.227, 0.773]. The brute-force marginal over all four variables is [0.255, 0.745]. So the single pass is wrong by about 0.028 in each component — not a rounding artifact, a real bias from handling A's dependence twice along the loop. That settles it: the graph has useful factorization, but it does not have the separators that make a tree pass exact, and pushing sums inward double-counts.

So I need to understand where the unavoidable computation actually lives. Variable elimination is exact, so I use it as a probe. When I eliminate a variable, all factors containing it are multiplied, the variable is summed out, and a new factor remains on the union of the neighboring variables. Those new scopes are not accidents; they are the dependencies the original graph forces me to create. On that same four-cycle, eliminating D from the C-D and D-A factors leaves a factor on {A,C} — a pair that was not adjacent before. If I throw away that scope after one query, I am discarding the very structure that made the exact sum possible.

So I stop treating fill-in as a nuisance and record it. For a directed model I first marry parents and drop directions, because the local conditional tables have to become undirected potentials. Then I add fill-in edges until every cycle of length at least four has a chord. On the four-cycle that means adding the chord A-C, which is precisely the {A,C} scope elimination forced on me. The maximal complete sets of the resulting chordal graph are where the intermediate factors can live without pretending the problem is smaller than it is — here {A,B,C} and {A,C,D}.

Now I need to arrange these complete sets so that local agreement implies global consistency. The condition I want is that every variable appears in one connected region of the arrangement. If the cliques holding a variable were split apart, two parts of the computation could carry incompatible beliefs about it. If they are connected, separator agreement along the connecting path ties all copies together. So the object I am building is not the original loopy graph forced to behave like a tree; it is a new tree whose nodes are the cliques, each edge carrying the intersection of its two cliques. For the four-cycle the tree is {A,B,C} — {A,C} — {A,C,D}, and the separator {A,C} genuinely cuts the reconstructed object in two.

I want to see this actually calibrate, on a case small enough to check against brute force. Take a chain A-B-C with binary variables and potentials phi_AB and phi_BC, cliques {A,B} and {B,C}, separator {B}. The unnormalized joint sums to Z = 29. I run the cached separator update: pick {A,B} as root, collect the message from {B,C}, which is sum over c of beta_BC, giving the function [3, 5] over b; multiply it into beta_AB and store it as mu_B. Then distribute back: sum beta_AB over a, giving [9, 20] over b, and update beta_BC by multiplying [9,20]/[3,5] = [3,4] in. Now I check what I actually wanted. The B-marginal read from {A,B} is [9, 20]; the B-marginal read from {B,C} is also [9, 20]; the stored separator is [9, 20]. They agree — that is calibration, computed, not asserted. And the calibrated beta_AB comes out [[6,5],[3,15]], which is exactly the true unnormalized AB-marginal of the joint. Reading single-variable marginals off the calibrated cliques and normalizing gives P(A) = [0.379, 0.621] and P(C) = [0.655, 0.345], matching the brute-force marginals to all digits.

So the recipe is: insert observed values into the factors that mention them and keep the normalizing constant implicit; that handles evidence as a local modification. A clique sends a separator marginal to a neighbor, the receiver multiplies by the new separator information divided by the old stored separator when there is one (the [3,4] step above), and after a valid collect/distribute schedule neighboring cliques agree on every separator. The same two-pass target can be reached through a set-chain representation moving backward from potentials to conditionals and forward to clique marginals; caching the division in separator storage is the HUGIN form.

I want to confirm the loopy case is genuinely repaired, not just the chain. Back on the four-cycle: assign phi_AB, phi_BC and the A-field to clique {A,B,C} and phi_CD, phi_DA to {A,C,D}, then calibrate over {A,C}. The separator {A,C} agrees across both cliques after the two passes, and P(A) read from the calibrated {A,B,C} comes out [0.255, 0.745] — exactly the brute-force marginal that the naive loop-as-tree pass missed by 0.028. So the clique beliefs are not arbitrary local approximations: chordality makes each clique scope large enough to hold the factors exact elimination creates, and the running-intersection connectivity makes separator agreement glue those clique beliefs into one joint distribution. Any single-variable marginal is read from any calibrated clique that contains it.

The hard part has not vanished. The triangulation that fixed the four-cycle added one chord and the largest clique stayed at three variables, so the tables stayed tiny. But if fill-in forces a large clique, its table is exponential in the clique size. So exact inference is feasible exactly when I can find a triangulation whose largest clique stays small, and the cost is governed by the induced width — treewidth plus one. The algorithm turns that condition into a reusable local-computation structure.
