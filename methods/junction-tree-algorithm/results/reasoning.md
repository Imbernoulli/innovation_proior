I start with the local conditional tables and I want a posterior marginal, but the marginal is still a global sum. On an ordinary tree I know what to do: cut an edge, summarize one side as a function of the shared variables, and pass that summary across. That works because once the separator is fixed, no other path carries the same dependence back around.

The first tempting move is to keep the original graph and run the same local messages anyway. That fails as an exact argument. In a loopy graph, a variable can disappear into one message and still be needed through another route, so I cannot justify pushing every sum inward. The graph has useful factorization, but it does not yet have the separators needed for exact dynamic programming.

I then look at variable elimination, because it is exact and it tells me where the real computation lives. When I eliminate a variable, all factors containing it are multiplied, the variable is summed out, and a new factor remains on the union of the neighboring variables. Those new scopes are not accidents; they are the dependencies the original graph forces me to create. If I throw away those scopes after one query, I am losing the structure that made the exact sum possible.

So I stop treating fill-in as a nuisance and make it explicit. For a directed model I first marry parents and ignore directions, because the local conditional tables must become undirected potentials. Then I add fill-in edges until every long cycle has a chord. The maximal complete sets of this chordal graph are precisely the places where the intermediate factors can live without pretending the problem is smaller than it is.

Now I still need a way to arrange these complete sets so that local agreement means global consistency. The condition I need is that every variable appears in one connected region of the arrangement. If the cliques containing a variable were split apart, two different parts of the computation could carry incompatible beliefs about it. If they are connected, then separator agreement along the path ties all copies together.

This is the real turn: I am not trying to make the original loopy graph behave like a tree. I am building a new tree whose nodes are large enough cliques. Each edge carries the intersection of two neighboring cliques. A message over that intersection is now legitimate because the intersection really separates the two sides in the reconstructed object.

With that structure in place, evidence is just a local modification of potentials: I insert observed values into the factors that mention them and keep the normalizing constant implicit. The original transfer view moves backward from potentials to set-chain conditionals and forward to clique marginals. In a cached message-passing form, a clique sends a separator marginal to a neighbor; the receiver multiplies by the new separator information, divided by the old stored separator when there is one. After the valid tree-edge messages have been sent, neighboring cliques agree on their separators.

At that point the clique beliefs are not arbitrary local approximations. They are a consistent family of marginals for the same joint distribution, because chordality gives the decomposable clique structure and the running-intersection property prevents variables from being summed out too early. Any single-variable marginal can be read from a calibrated clique that contains it.

The hard part has not vanished. If the fill-in creates a large clique, the table for that clique is enormous. The useful conclusion is sharper: exact inference is feasible when I can find a triangulation whose largest clique remains small, and the algorithm turns that condition into a reusable local-computation structure.
