I start from the theorem that gives me permission to think locally: a matching is maximum precisely when I cannot find an augmenting alternating path. So I do not need to guess the final matching. I need a search procedure that either exposes one more path between exposed vertices or gives a certificate that no such path remains.

The bipartite case tempts me because it makes the search almost mechanical. I grow alternating trees from exposed roots. An unmatched edge takes me from an outer vertex to an inner vertex, and the matched edge incident to that inner vertex takes me back to a new outer vertex. The parity labels stay coherent because the graph has no odd cycle. If I hit a new exposed vertex, I have an augmenting path; if I cannot grow, I have learned something real about the region I searched.

Then I try the same discipline in a general graph and the first serious failure is an edge between two outer vertices. The two paths from those vertices back toward the root have the same parity, so adding the edge between them makes an odd alternating circuit. This is not the same as reaching an exposed vertex in another tree. It does not directly augment the matching. But it is also too structured to throw away: all around that odd circuit, once I choose one vertex to leave exposed, the rest of the circuit has a unique alternating way to be matched as much as possible.

If I branch on every possible way through the circuit, I am back to the search tree I am trying to avoid. If I forbid the edge or mark the circuit as hopeless, I can lose a path that later enters the circuit from outside and exits through the only parity-compatible side. The circuit is acting like a local ambiguity, not like a global obstruction. I need to make the outside search stop seeing the ambiguity while remembering enough to restore it.

So I treat the odd alternating circuit as a single temporary vertex. The base is the point where the stem from the exposed root meets the circuit. From outside, the whole circuit behaves as though one exposed choice can be moved to whichever boundary vertex an eventual path needs. I shrink the circuit and continue the same alternating search in the reduced graph.

Now I have to check that this is not just a convenient picture. If the reduced graph finds an augmenting path that avoids the temporary vertex, the same path works in the original graph. If it uses the temporary vertex, then the stored odd circuit tells me which internal matching is compatible with the outside incidence. In the tree path view, this is the unique alternating route from the touched circuit vertex back to the base. In the matching view, after the flip in the reduced graph there is at most one matched outside edge incident to the temporary vertex, so I use the unique maximum matching of the odd circuit that leaves that incident circuit vertex exposed. Expanding the temporary vertex with that choice gives a genuine augmenting path or, equivalently, a genuine one-edge augmentation in the original graph.

The converse matters just as much. If an augmenting path exists before shrinking, the shrink must not destroy the fact that some augmenting path exists. If the path stays outside the circuit, nothing changes. If it touches the circuit, the odd circuit can be represented by the temporary vertex, and the alternating conditions survive after contraction. When there is a nonempty stem, I can first flip along the stem without changing the matching size, moving the exposed status to the base, and then reduce to the same argument.

This gives me the actual search rule. I grow an alternating forest. When an edge from an outer vertex reaches an unreached matched vertex, I add the two-edge extension through that vertex and its matched mate. When the edge reaches an inner vertex already in the forest, it gives no new parity information, so I mark it examined and move on. When it reaches an outer vertex, I trace the two parent paths. Different roots give an augmenting path; a common root gives the odd circuit, so I store its base and cyclic order, shrink it, and keep searching. When a later augmenting path passes through a temporary vertex, I expand nested temporary vertices in dependency order and choose the alternating route that lifts the path.

The stopping proof returns to the first theorem. Each successful phase increases the matching by one. If a phase cannot find an augmenting path even after all necessary odd circuits have been represented and searched through, then Berge's criterion says the current matching is maximum. The odd-set dual theorem explains why the certificate has this shape: the structures I shrink during search are the same odd-set obstructions that must appear in the linear description of general matching.
