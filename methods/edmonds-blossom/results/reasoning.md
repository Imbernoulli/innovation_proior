I start from the theorem that gives me permission to think locally: a matching is maximum precisely when I cannot find an augmenting alternating path. So I do not need to guess the final matching. I need a search procedure that either exposes one more path between exposed vertices or gives a certificate that no such path remains.

The bipartite case tempts me because it makes the search almost mechanical. I grow alternating trees from exposed roots. An unmatched edge takes me from an outer vertex to an inner vertex, and the matched edge incident to that inner vertex takes me back to a new outer vertex. The parity labels stay coherent because the graph has no odd cycle. If I hit a new exposed vertex, I have an augmenting path; if I cannot grow, I have learned something real about the region I searched.

Then I try the same discipline in a general graph and the first thing that breaks is an edge between two outer vertices. The two paths from those vertices back toward the root have the same parity, so adding the edge between them makes an odd alternating circuit. This is not the same as reaching an exposed vertex in another tree. It does not directly augment the matching. But it is also too structured to throw away, and I want to understand exactly how much structure it has before I decide what to do with it.

Let me look at the smallest case carefully. Take the cycle `1-2-3-4-5-1`, with `M = {2-3, 4-5}`, so the cycle has two matched edges and vertex `1` exposed. I want to know: for each vertex `v` of this cycle, can I rearrange the matching on the cycle so that exactly `v` is left exposed, and how many ways? I enumerate the matchings of the 5-cycle that match four of the five vertices, grouped by which vertex stays exposed:

```
exposed=1: {2-3, 4-5}
exposed=2: {1-5, 3-4}
exposed=3: {1-2, 4-5}
exposed=4: {1-5, 2-3}
exposed=5: {1-2, 3-4}
```

Two facts fall out of this little table, and both matter. First, every one of the five vertices can be the exposed one — the odd circuit can present any of its vertices to the outside as the free end. Second, for each choice of exposed vertex there is exactly one internal matching that achieves it. So the circuit is genuinely ambiguous about which vertex it offers up, but completely determined once that vertex is fixed.

That tells me what I must not do. I must not commit to one internal matching of the circuit while I am still searching, because I do not yet know which vertex an eventual outside path will want to enter through, and each entry vertex forces a different internal matching. And I must not delete the closing edge or mark the circuit hopeless, because that would throw away exactly the vertices a later path might need. The circuit is a local ambiguity to be deferred, not a global obstruction to be discarded. What I want is to make the outside search stop seeing the ambiguity while remembering enough to restore it once the entry vertex is known.

So I treat the odd alternating circuit as a single temporary vertex. The base is the point where the stem from the exposed root meets the circuit. From outside, the whole circuit behaves as though one exposed choice can be moved to whichever boundary vertex an eventual path needs — which is precisely the freedom the table above showed. I shrink the circuit and continue the same alternating search in the reduced graph.

Now I have to check that this contraction does not change the question I am asking. The claim I need is: `G` has an augmenting path with respect to `M` if and only if the contracted graph has one with respect to the contracted matching. Asserting this is cheap; I want to actually run it once on a case where the path goes through the circuit, because that is the case where the lift does real work.

I enlarge the example so the circuit has a stem and a second exposed vertex on the far side. Vertices `1..8`: the body cycle `1-2-3-4-5-1`, a stem `1-6` then `6-7`, and a pendant `3-8`. Take `M = {2-3, 4-5, 6-7}`, which leaves `1` and `8` exposed, `|M| = 3`. A brute-force search over all edge subsets reports a maximum matching of size `4`, so an augmenting path between `1` and `8` must exist — and I want to recover it through the shrink.

Growing the forest from root `1`: edge `1-2` makes `2` inner and `3` outer; `1-5` makes `5` inner and `4` outer; `1-6` makes `6` inner and `7` outer. Now scanning outer vertex `3`, I see its neighbor `4`, which is also outer with the same root `1`. Their parent paths `3-2-1` and `4-5-1` share base `1`, so together with edge `3-4` they close the odd circuit `1-2-3-4-5-1`. I contract that whole set into a pseudovertex `b`. The reduced graph has edges `b-6`, `6-7`, `b-8`; the reduced matching keeps only `6-7`, since `2-3` and `4-5` are internal to the blossom and vanish. The base `1` was exposed, so `b` is exposed; `8` is still exposed.

In this reduced graph the augmenting path is immediate: `b` and `8` are both exposed and joined by edge `b-8`, so `b-8` is an augmenting path of length one. Augmenting adds `b-8` and gives a matching of size `2` on the reduced graph. Now I lift. The reduced edge `b-8` came from the original edge `3-8`, so the path enters the blossom at vertex `3` and must travel to the base `1` along the circuit. From the table, the internal matching leaving `3` exposed is `{1-2, 4-5}`, and the alternating route from `3` to base `1` consistent with it is `3-2-1` (`3-2` matched, `2-1` unmatched). The lifted path is therefore `8-3-2-1`. I take the symmetric difference of `M` with this path: drop `2-3`, add `8-3` and `1-2`, giving `{4-5, 6-7, 8-3, 1-2}`. I check it: no vertex repeats, so it is a valid matching, and it has size `4` — one more than I started with, and equal to the brute-force maximum. The contraction did not hide the improvement, and the lift reconstructed a real augmenting path in the original graph. This is the one concrete instance I most wanted to see work, and it does.

The two directions of the invariant are exactly what this trace exercised. Forward: a reduced augmenting path that avoids `b` is already a path in `G`; one that uses `b` enters at some boundary vertex, and the stored circuit supplies the unique internal matching leaving that vertex exposed, so the lift is forced and valid — here entering at `3` forced `{1-2, 4-5}` and the route `3-2-1`. Backward: if an augmenting path existed before shrinking, a path staying outside the circuit is untouched, and a path touching the circuit survives contraction because the circuit collapses to `b` while the alternating conditions at the boundary are preserved; when there is a nonempty stem I can first flip along the stem, which keeps `|M|` fixed and moves the exposed status to the base, then reduce to the same argument.

This is what the search rule has to be. I grow an alternating forest. When an edge from an outer vertex reaches an unreached matched vertex, I add the two-edge extension through that vertex and its matched mate. When the edge reaches an inner vertex already in the forest, it gives no new parity information, so I mark it examined and move on. When it reaches an outer vertex, I trace the two parent paths. Different roots give an augmenting path; a common root gives the odd circuit, so I store its base and cyclic order, shrink it, and keep searching. When a later augmenting path passes through a temporary vertex, I expand nested temporary vertices in dependency order and choose the alternating route that lifts the path — the route that, as in the example, is uniquely fixed by the entry vertex.

The stopping argument returns to the first theorem. Each successful phase increases the matching by one, as the size went `3 → 4` above. If a phase cannot find an augmenting path even after every necessary odd circuit has been shrunk and searched through, then Berge's criterion says the current matching is maximum. I would still want to confirm on larger inputs that no augmenting path slips through an unshrunk configuration, but the per-vertex table is what convinces me the shrink loses nothing locally. The odd-set dual theorem then explains why the certificate has this shape: the structures I shrink during search are the same odd-set obstructions that must appear in the linear description of general matching.
