I start from the fact that reachability is too easy for nondeterminism in exactly the wrong way. If I want to prove that a target is reachable, I guess a path and check edges one at a time. If I want to prove that the target is not reachable, the same habit gives me nothing. Guessing one failed path says almost nothing, and guessing an accepting configuration so that I can reject is not a complementing machine, because some other nondeterministic branch may simply guess badly and accept for the wrong reason. The condition is not "there exists a branch that sees no path"; it is "there exists an accepting computation exactly on the no-instances, and no accepting computation on the yes-instances." That is a much sharper requirement.

I look at Savitch's theorem and see a real but wrong kind of help. The midpoint recursion turns reachability into a deterministic question: try every middle configuration, recurse on the left half and right half, reuse the work space at each recursive call. This is beautiful, but it squares the space. It answers a simulation question, not the same-space complement question. If I follow that lens, I am searching for a deterministic universal traversal of the configuration graph. The theorem I need must still use nondeterminism in the original space, so I need a certificate shape that nondeterminism can verify without becoming deterministic search.

The thing I can always certify is positive reachability. A path to a vertex is small enough to stream and check. So I ask whether enough positive certificates can add up to a negative certificate. If I somehow know the number of vertices reachable from the source within `i` steps, call it `r_i`, then a certificate that a particular vertex `v` is not among them can be a complete list of the reachable vertices, in the fixed vertex order, each with its path certificate, with `v` absent. I do not store the list. I scan it in order, reject repeated or out-of-order names, check one path at a time, count how many distinct genuine reachable vertices have been presented, and accept only when the count is exactly `r_i`. Now omission becomes detectable. If `v` is actually reachable, then no ordered list of exactly `r_i` distinct genuine reachable vertices can omit it. If `v` is not reachable, there is a branch that supplies all the reachable vertices and omits `v`.

This is the turn. I do not try to certify nonexistence directly. I certify all the existing things, and I use the count to force the word "all." Nondeterminism remains existential at the local level: every item in the list comes with a guessed path. The count supplies the global completeness condition that a single path cannot supply.

But now I have moved the difficulty into the count. Knowing `r_i` is strong; I have to get it without already knowing the reachable set. I try to compute the counts layer by layer. At layer zero the count is one, only the source. Suppose I have `r_i`. To compute `r_{i+1}`, I scan every candidate vertex `v` and decide whether it is reachable in one more step. If `v` is already reachable within `i` steps, or if some reachable predecessor `u` has an edge to `v`, a path to `u` proves the positive case. If `v` is not reachable within `i+1` steps, I again use the count: enumerate all `r_i` vertices reachable within `i` steps and check that none is `v` and none points to `v`. That complete counted enumeration certifies the negative case for this particular `v`.

So for each `v`, an accepting branch can present either a positive local witness or a complete negative enumeration in the fixed order. I count the vertices certified positive, and when the scan ends the counter is `r_{i+1}`. I keep only the current vertex, a predecessor, the previous enumerated name, a path counter, the old count, the new count, and the layer index. The set itself never appears on the tape.

The soundness rests on the same pressure at every layer. A branch may guess too few reachable vertices while trying to prove a negative, but then its enumeration count is below `r_i` and it rejects. It cannot compensate by adding fake vertices, because every counted vertex must carry a real path certificate, and it cannot compensate by duplicating a real one, because the ordered scan counts each name at most once. Thus the count prevents undercounting, overcounting, and cheating. The only surviving branches are the ones whose local certificates match the true reachable frontier.

Once the induction reaches `n-1` in an `n`-vertex graph, every simple reachable path has already been covered. Now I apply the same nonmembership test to the target. If the target is absent from the complete counted reachable set, I accept non-reachability. If the target is reachable, every complete counted enumeration includes it, and every branch rejects.

The configuration-graph version is the same proof with longer vertex names. A space-`s(n)` machine has configurations of `O(s(n))` bits and at most exponentially many of them, so a counter for configurations still fits in `O(s(n))` space. Counting reachable configurations replaces counting graph vertices, and the induction continues until the count stops changing. When the count of all reachable configurations is known, I can enumerate all reachable configurations in lexicographic order, reject if an accepting one appears, and accept exactly when the complete count is met with no accepting configuration reachable.

This is why the proof does not feel like a recombination of known reachability tricks. It changes the role of nondeterminism. Instead of asking nondeterminism to find the forbidden object or to simulate a universal search, it asks nondeterminism to produce local witnesses under an exact global census. The census is small enough to store, but strong enough to certify that an omitted target is truly absent.
