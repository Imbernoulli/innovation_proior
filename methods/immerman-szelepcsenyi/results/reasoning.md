I start from the fact that reachability is too easy for nondeterminism in exactly the wrong way. If I want to prove that a target is reachable, I guess a path and check edges one at a time. If I want to prove that the target is not reachable, the same habit gives me nothing. Guessing one failed path says almost nothing, and guessing an accepting configuration so that I can reject is not a complementing machine, because some other nondeterministic branch may simply guess badly and accept for the wrong reason. The condition is not "there exists a branch that sees no path"; it is "there exists an accepting computation exactly on the no-instances, and no accepting computation on the yes-instances." That is a much sharper requirement, and it is exactly why everyone, myself included, expects same-space closure to fail here: existential machinery does not obviously certify universal statements.

I have another reason to look hard at this question right now. The log-space alternating hierarchy has just been shown to collapse to its second level, and that result only makes sense against a backdrop of tight relationships between nondeterministic log space and its complement. So the timing is right to attack NL versus co-NL directly, even though I still expect, like everyone I have discussed it with, that the answer is negative — that no same-space nondeterministic machine can decide non-reachability.

I do not attack this by looking for a clever machine first. I attack it the way separations are usually attacked: by trying to rule the positive result out. I already know, from earlier work on how weak a reduction can be while still preserving completeness, that reachability stays complete for nondeterministic log space even under transformations with almost no computational power of their own — transformations where each bit of the graph I hand to the reachability test depends on at most one bit of the original problem, selected by a rule fixed in advance that looks only at positions, never at the actual input values. That fact turns the class-level question into a much more concrete one: same-space closure holds exactly when the statement "there is no path from s to t" can itself be recoded, by a transformation this weak, into the statement "there is a path from s' to t'" in some derived graph. If I can show that no transformation this restrictive can do that recoding, I will have the separation everyone expects. So my first real move is to pin down exactly what such a transformation would have to look like, hoping that nailing down its shape exposes a contradiction.

Before going further down that road I check the tool everyone already has for space-bounded nondeterminism: Savitch's midpoint recursion. It turns reachability into a deterministic question by trying every middle configuration and recursing on the two halves, reusing work space at each call. This is beautiful, but it squares the space. It answers a simulation question, not the same-space complement question, so it tells me nothing about whether my restrictive-transformation attempt should succeed or fail. If anything it reinforces the expectation of failure: the only general method anyone has for handling nondeterministic space deterministically pays a quadratic price, so a same-space certificate for non-reachability looks like it is asking for something Savitch's method was never built to give.

Here is where the attempted refutation turns into a construction. A transformation this weak has almost no room to look at the graph itself — the only thing it is allowed to reference beyond fixed positions is a small amount of numeric information, and the cheapest numeric information I can imagine is a count: how many things, out of everything examined so far, satisfy some property. So when I try to write down precisely what my impossibility argument needs to rule out, I keep landing on the same shape: a scheme that knows, at each point, an exact count of how many reachable vertices there are, and uses that count as the only extra resource beyond position and comparison. I write the requirement down expecting it to be unsatisfiable, and instead I notice it is satisfiable. Call `r_i` the number of vertices reachable from `s` in at most `i` steps. If I already know `r_i`, a nondeterministic machine can certify that a particular vertex `v` is not among them: scan candidate names in a fixed order, and for each one either supply a path from `s` of length at most `i` and count it, or skip it; accept only when the running count reaches exactly `r_i` and `v` was never one of the counted names. I do not store the reachable set. I scan it in order, reject repeated or out-of-order names, check one path at a time, and accept only when the count is exactly `r_i`. Now omission becomes detectable. If `v` is actually reachable, then no ordered list of exactly `r_i` distinct genuine reachable vertices can omit it. If `v` is not reachable, there is a branch that supplies all the reachable vertices and omits `v`. This is the turn. I do not try to certify nonexistence directly. I certify all the existing things, and I use the count to force the word "all." Nondeterminism remains existential at the local level: every item in the list comes with a guessed path. The count supplies the global completeness condition that a single path cannot supply — the very condition my impossibility attempt assumed could not be captured this cheaply.

But now I have moved the difficulty into the count. Knowing `r_i` is strong; I have to get it without already knowing the reachable set. I try to compute the counts layer by layer. At layer zero the count is one, only the source. Suppose I have `r_i`. To compute `r_{i+1}`, I scan every candidate vertex `v` and decide whether it is reachable in one more step. If `v` is already reachable within `i` steps, or if some reachable predecessor `u` has an edge to `v`, a path to `u` proves the positive case. If `v` is not reachable within `i+1` steps, I again use the count: enumerate all `r_i` vertices reachable within `i` steps and check that none is `v` and none points to `v`. That complete counted enumeration certifies the negative case for this particular `v`.

So for each `v`, an accepting branch can present either a positive local witness or a complete negative enumeration in the fixed order. I count the vertices certified positive, and when the scan ends the counter is `r_{i+1}`. I keep only the current vertex, a predecessor, the previous enumerated name, a path counter, the old count, the new count, and the layer index. The set itself never appears on the tape.

The soundness rests on the same pressure at every layer. A branch may guess too few reachable vertices while trying to prove a negative, but then its enumeration count is below `r_i` and it rejects. It cannot compensate by adding fake vertices, because every counted vertex must carry a real path certificate, and it cannot compensate by duplicating a real one, because the ordered scan counts each name at most once. Thus the count prevents undercounting, overcounting, and cheating. The only surviving branches are the ones whose local certificates match the true reachable frontier.

Once the induction reaches `n-1` in an `n`-vertex graph, every simple reachable path has already been covered. Now I apply the same nonmembership test to the target. If the target is absent from the complete counted reachable set, I accept non-reachability. If the target is reachable, every complete counted enumeration includes it, and every branch rejects.

The configuration-graph version is the same proof with longer vertex names. A space-`s(n)` machine has configurations of `O(s(n))` bits and at most exponentially many of them, so a counter for configurations still fits in `O(s(n))` space. Counting reachable configurations replaces counting graph vertices, and the induction continues until the count stops changing. When the count of all reachable configurations is known, I can enumerate all reachable configurations in lexicographic order, reject if an accepting one appears, and accept exactly when the complete count is met with no accepting configuration reachable.

This is why the proof does not feel like a recombination of known reachability tricks, and why it took a wrong attempt to find a right one. I set out to prove that a transformation with almost no computational power could never turn a non-reachability instance into a reachability instance, because that was the separation everyone expected. Pinning down exactly what such a transformation would have to do — its only leverage being fixed positions and a count — is what handed me the transformation itself, working instead of contradictory. The proof changes the role of nondeterminism rather than confirming the failure I went looking for. Instead of asking nondeterminism to find the forbidden object or to simulate a universal search, it asks nondeterminism to produce local witnesses under an exact global census. The census is small enough to store, but strong enough to certify that an omitted target is truly absent.
