# The Fourier Neural Operator (FNO)

## Problem

Learn the **solution operator** of a parametric family of PDEs, `G† : a ↦ u`, mapping an input function `a` (coefficient, forcing, or initial condition) on a domain `D ⊂ R^d` to the solution function `u`. Train once from observed pairs `{a_j, u_j}`; then each new instance is a single forward pass instead of a fresh mesh-bound solve. The operator must be **discretization-invariant**: defined in function space so one set of parameters works at any resolution and transfers between meshes.

## Key idea

Build a learnable map between function spaces by composing pointwise nonlinearities with a **kernel integral operator** — the function-space analogue of a linear layer, and exactly the form the solution operator of a linear PDE takes through its Green's function. A general kernel integral is `O(N²)`. Restrict the kernel to a translation-invariant convolution `κ(x, y) = κ(x − y)` (the constant-coefficient Green's function), and apply the **convolution theorem**: convolution becomes pointwise multiplication in Fourier space. The integral operator collapses to

```
(K v)(x) = F⁻¹( R · F(v) )(x),
```

where `R` is a learned complex multiplier parameterized **directly in Fourier space**. Truncating to a fixed low-mode block gives a finite, resolution-independent parameterization (justified by the spectral decay of the data; the high modes dropped per layer are regenerated by the physical-space nonlinearities between layers). The FFT computes the transforms in `O(N log N)`.

## Architecture

Lift, iterate, project. With lifting `P` and projection `Q` shallow pointwise (channel) maps:

```
v_0(x) = P(a(x))
z_{t+1}(x) = W v_t(x) + (K v_t)(x)
v_{t+1}(x) = σ(z_{t+1}(x)),   t = 0,…,T−2
v_T(x) = z_T(x)               # implemented final block before Q
u(x)   = Q(v_T(x))
```

- `K` is the **Fourier layer**: FFT → keep a fixed low-mode block → multiply each retained mode `k` by a learned complex matrix `R(k) ∈ C^{d_v × d_v}` (channel mixing) → inverse FFT. Per mode, `(R · F v)_{k,l} = Σ_{j=1}^{d_v} R_{k,l,j} (F v)_{k,j}`. The real FFT stores a one-sided spectrum and `irfft` supplies the conjugate half; in 2-D and 3-D the axes not halved by `rfft` still need learned negative-frequency corner tensors.
- `W` is a pointwise-in-space linear map (a 1×1 convolution) — the local/bias branch that carries non-periodic boundary information the periodic Fourier branch cannot.
- `σ` (GELU/ReLU) is applied in **physical space**, where a pointwise nonlinearity regenerates high frequencies; in Fourier space it would amount to a spatial convolution and lose that role.

On a uniform `s_1 × … × s_d = N` grid, `m_j` is a retained-mode count: a full FFT axis keeps slices `0:m_j` and `s_j−m_j:s_j`, while the real-FFT last axis keeps only `0:m_d` and reconstructs the missing half in `irfft`. The stored low-mode block is therefore realized as the **corners** of the spectrum: two corners in 2-D (two weight tensors), four in 3-D. Complexity is `O(|Z_m| d_v²)` for the complex channel mixing plus `O(N log N)` for the FFTs; with fixed width and mode count this is quasi-linear in `N`. Discretization invariance follows because the parameters `R` live in Fourier space with a fixed mode count independent of `N`, enabling zero-shot super-resolution. Typical settings: 4 Fourier layers, `m_j = 12`, `d_v = 32` (2-D); `m_j = 16`, `d_v = 64` (1-D).

## Code

```python
import torch
import torch.nn as nn
import torch.nn.functional as F


class SpectralConv2d(nn.Module):
    """Fourier layer: rfft -> keep low modes -> per-mode complex matmul by R -> irfft."""
    def __init__(self, in_channels, out_channels, modes1, modes2):
        super().__init__()
        self.in_channels = in_channels
        self.out_channels = out_channels
        self.modes1 = modes1            # retained modes on axis 1 (both corners)
        self.modes2 = modes2            # retained modes on axis 2 (rfft half)
        self.scale = 1 / (in_channels * out_channels)
        self.weights1 = nn.Parameter(   # low-positive corner of axis 1
            self.scale * torch.rand(in_channels, out_channels, modes1, modes2, dtype=torch.cfloat))
        self.weights2 = nn.Parameter(   # high (negative) corner of axis 1
            self.scale * torch.rand(in_channels, out_channels, modes1, modes2, dtype=torch.cfloat))

    def compl_mul2d(self, inp, weights):
        # (batch,in,x,y),(in,out,x,y) -> (batch,out,x,y)
        return torch.einsum("bixy,ioxy->boxy", inp, weights)

    def forward(self, x):
        b = x.shape[0]
        x_ft = torch.fft.rfft2(x)                          # F(v): (b, c, s1, s2//2+1)
        out_ft = torch.zeros(b, self.out_channels, x.size(-2), x.size(-1) // 2 + 1,
                             device=x.device, dtype=torch.cfloat)
        out_ft[:, :, :self.modes1, :self.modes2] = self.compl_mul2d(
            x_ft[:, :, :self.modes1, :self.modes2], self.weights1)
        out_ft[:, :, -self.modes1:, :self.modes2] = self.compl_mul2d(
            x_ft[:, :, -self.modes1:, :self.modes2], self.weights2)
        return torch.fft.irfft2(out_ft, s=(x.size(-2), x.size(-1)))


class FNO2d(nn.Module):
    def __init__(self, modes1, modes2, width):
        super().__init__()
        self.padding = 9                       # pad for non-periodic domains
        self.fc0 = nn.Linear(3, width)         # lift P: (a(x,y), x, y) -> width channels
        self.conv0 = SpectralConv2d(width, width, modes1, modes2)
        self.conv1 = SpectralConv2d(width, width, modes1, modes2)
        self.conv2 = SpectralConv2d(width, width, modes1, modes2)
        self.conv3 = SpectralConv2d(width, width, modes1, modes2)
        self.w0 = nn.Conv2d(width, width, 1)   # pointwise-in-space linear W
        self.w1 = nn.Conv2d(width, width, 1)
        self.w2 = nn.Conv2d(width, width, 1)
        self.w3 = nn.Conv2d(width, width, 1)
        self.fc1 = nn.Linear(width, 128)       # projection Q
        self.fc2 = nn.Linear(128, 1)

    def forward(self, x):
        grid = self.get_grid(x.shape, x.device)
        x = torch.cat((x, grid), dim=-1)
        x = self.fc0(x).permute(0, 3, 1, 2)
        x = F.pad(x, [0, self.padding, 0, self.padding])
        for conv, w in [(self.conv0, self.w0), (self.conv1, self.w1),
                        (self.conv2, self.w2), (self.conv3, self.w3)]:
            x = conv(x) + w(x)                 # global Fourier branch + local pointwise branch
            if conv is not self.conv3:
                x = F.gelu(x)                  # nonlinearity in physical space
        x = x[..., :-self.padding, :-self.padding].permute(0, 2, 3, 1)
        x = F.gelu(self.fc1(x))
        return self.fc2(x)

    def get_grid(self, shape, device):
        b, sx, sy = shape[0], shape[1], shape[2]
        gx = torch.linspace(0, 1, sx, device=device).reshape(1, sx, 1, 1).repeat(b, 1, sy, 1)
        gy = torch.linspace(0, 1, sy, device=device).reshape(1, 1, sy, 1).repeat(b, sx, 1, 1)
        return torch.cat((gx, gy), dim=-1)


def relative_l2(pred, target):
    diff = (pred - target).reshape(pred.size(0), -1).norm(dim=1)
    return (diff / target.reshape(target.size(0), -1).norm(dim=1)).mean()


def train(model, train_loader, epochs=500, lr=1e-3):
    opt = torch.optim.Adam(model.parameters(), lr=lr, weight_decay=1e-4)
    sched = torch.optim.lr_scheduler.StepLR(opt, step_size=100, gamma=0.5)
    for _ in range(epochs):
        for a, u in train_loader:
            opt.zero_grad()
            loss = relative_l2(model(a), u)
            loss.backward()
            opt.step()
        sched.step()
```

The 1-D layer uses `rfft`/`irfft` with a single low-mode block `[:modes1]`; the 3-D layer uses `rfftn`/`irfftn` with four corner tensors. The map can be applied per time step with a recurrent structure (FNO-2D) or directly over space-time (FNO-3D).
