I start from the part that feels wasteful rather than from the part that already works. Residual shortcuts let me train networks that would otherwise be impossible, but the empirical exchange rate is poor: once I am near the best thin residual models, a small accuracy gain asks for a very large increase in layers. That is a warning that depth is not being converted into useful representation efficiently.

The identity shortcut explains why. A residual block adds `F(x_l, W_l)` to `x_l`, so the signal has an exact path around the residual weights. That is the optimization gift, but it is also an escape hatch. In backpropagation, the gradient can travel through the shortcut even if the residual branch is weak. So I should not assume that every extra block is learning a new useful transformation. Stochastic depth makes this less speculative: if whole residual blocks can be randomly skipped during training and the network still improves, then many blocks in a very deep residual chain are not individually indispensable.

My first instinct is to change the block, but I need to separate several possibilities. A block can become stronger by adding more convolutions, by changing the mix of `3x3` and `1x1` convolutions, or by increasing the channel count. Larger spatial filters are not the right first experiment because the surrounding evidence from VGG and Inception already favors small filters, and larger filters would blur the question with an older design choice. I keep the filters at `3x3` or `1x1` and ask what the residual block is really buying.

The bottleneck block is also the wrong object for the small-image question I am trying to answer. It exists to make a block cheaper so I can stack more layers. If the thing under suspicion is the habit of spending nearly everything on depth, then a thinning device would bake in the old answer. I keep the same number of feature planes across the residual branch and use pre-activation, because the identity-mapping work has already shown that `BN -> ReLU -> conv` is the better residual-unit order.

Now I test block structure at comparable parameter count. I compare the basic `B(3,3)` block against variants with `1x1` layers such as `B(3,1)`, `B(1,3)`, `B(1,3,1)`, `B(3,1,3)`, and `B(3,1,1)`. This is not the place for a dramatic conclusion: the results are close. `B(3,3)` is the best by a small margin, while `B(3,1)` and `B(3,1,3)` are very near it and cheaper. That tells me the exact internal mix is not the main source of leverage. The simple two-`3x3` block is still the cleanest choice, but not because every alternative collapses.

Then I test whether each block should contain more convolutions. I hold the total number of convolution layers and the parameter count fixed, and vary the deepening factor `l`. This is the point where I have to correct a tempting thought: if a block is the useful unit, maybe making each block deeper is better. The experiment says no. `l=2` wins; `l=1` is too weak, and `l=3` or `l=4` gets worse. The likely reason is that with the total convolution budget fixed, deeper blocks mean fewer residual blocks and therefore fewer shortcuts. The shortcut is not just a passive wire; it is part of the optimization machinery. Removing too many of those addition points makes the network harder to optimize.

So the block settles down to the basic pre-activation two-convolution unit. The remaining axis is channel count. This is the axis the thin residual designs deliberately minimized in order to buy depth, and now I have reasons to distrust that trade. Width costs quadratically in parameters and FLOPs, while more blocks cost linearly, so on paper width looks expensive. But wall-clock training is not paper arithmetic. A thousand small sequential convolutions underuse a GPU; fewer larger convolutions give the hardware more parallel work. I should measure accuracy per training time, not only parameter count.

When I sweep the feature-plane multiplier and depth, the pattern is the answer. At fixed modest depths, increasing the channel multiplier keeps improving the CIFAR results over the tested range. At fixed large multipliers, increasing depth helps at first but then stops helping or hurts. A 40-layer, 4x channel model has about the same parameter scale as the 1001-layer pre-activation baseline and compares favorably on CIFAR, while training far faster. A 28-layer, 10x channel model is much larger, but it trains successfully and gives better accuracy than the thin 1001-layer reference. The old depth-to-width ratio was not sacred; it was just one expensive point in the design space.

This also changes how I think about regularization. Increasing channels creates many more parameters, and batch normalization alone is not a complete answer, especially when the data do not get much augmentation. SVHN is the clean test: no augmentation, just scaling to `[0,1]`, and the training curves can show the model driving training loss very low while test error stalls. That is the regime where I need extra stochastic regularization.

I should not put dropout on the shortcut. The shortcut is the stable path that lets information and gradients move through the net; randomly damaging it fights the reason residual nets train. The dropout has to live inside the residual branch, between the two convolutions, after a ReLU. The clean placement is first BN/ReLU and convolution, then second BN/ReLU, then dropout, then the second convolution. That placement perturbs the residual transformation without corrupting the identity path, and it gives the following block's batch normalization a less stale activation distribution.

The final shape is therefore narrow in concept even though it is wide in channels: keep the simple pre-activation `B(3,3)` residual block, use fewer blocks than the thousand-layer baseline, multiply the feature planes in the three residual groups by `k`, and place dropout only inside the residual branch when regularization is needed. The depth parameter should satisfy `(depth - 4) % 6 == 0`, giving `N = (depth - 4) / 6` blocks per group. The stage widths are `16`, `16k`, `32k`, and `64k`; downsampling occurs at the first blocks of the second and third residual groups; the classifier ends with BN, ReLU, `8x8` average pooling, and a linear layer.

The implementation details need to make the shortcut semantics exact. If the input and output planes match, the addition uses the raw identity input. If the planes change, the shortcut is a `1x1` projection with the block stride applied to the first preactivated tensor, so the shortcut and residual tensors align. Dropout comes after the second BN/ReLU and before the second convolution. Convolutions have no bias. The weight scale should be He/MSR fan-in, with standard deviation `sqrt(2/(kW*kH*nInputPlane))`, so a PyTorch version should use `mode="fan_in"`, not fan-out.
