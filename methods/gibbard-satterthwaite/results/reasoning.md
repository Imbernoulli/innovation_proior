I want a voting rule that is not absurdly narrow. Every alternative should be possible somewhere, so I cannot hide an option by never choosing it. I also do not want a single fixed voter to command the outcome at every profile. And I want truthful reporting to be safe for each voter no matter what the others report. This is not a problem about recombining rankings into a clever aggregate order; the pressure is strategic. With two alternatives this still sounds plausible; majority rule has a clean median-like force in the binary case. With three alternatives, the familiar examples already feel unstable. Plurality asks a voter to think about viability. Borda asks a voter to think about how high to place rivals. Runoff asks a voter to think about who reaches the final comparison. So the question is not whether these named rules fail. The question is whether any deterministic rule can avoid that pressure.

Let me pin down what strategy-proofness buys before trying to prove too much. Suppose the rule chooses `a` at a profile `R`. Now form another profile `R'` by changing rankings in a way that never makes `a` worse relative to any alternative it used to beat. If the outcome changed from `a` to some `b`, then along a one-voter-at-a-time path there is a first voter whose report change changes the outcome. At the old report the outcome is `a`; at the new report it is `b`. In that voter's old ranking, `a` is at least as good as before relative to `b`, and in the new ranking `a` has not been pushed below an alternative it previously beat. If the voter prefers `b` to `a` under the new report, then when his true ranking is the new one he would profit by reporting the old one, or if he prefers `a` to `b` under the old report, then when his true ranking is the old one he would profit by reporting the new one. Strict preferences leave no tie to hide behind. The outcome cannot change. So strategy-proofness forces monotonicity: once `a` wins, raising `a` cannot make `a` lose.

Onto now matters. If everyone ranks `a` above `b`, can the rule select `b`? Suppose it does at some profile `R`. Because the rule is onto, there is another profile `Q` where `a` wins. Build a middle profile `S` where every voter ranks `a` first, `b` second, and all remaining alternatives below them. From `Q` to `S`, the winning alternative `a` only moves upward relative to everything else, so monotonicity keeps the outcome at `a`. From `R` to `S`, the alternative `b` keeps beating every alternative it beat at `R`, because at `R` every voter already has `a` above `b` and in `S` the only alternative above `b` is still `a`; all others are below `b`. Monotonicity would also keep the outcome at `b`. The same profile `S` cannot choose both `a` and `b`. So an onto strategy-proof rule cannot choose a Pareto-dominated alternative. Weak Pareto is not an extra axiom; it has been forced by nonimposition and incentives.

Now I need to find the person whose report actually pins the outcome. The two-voter case is the clean place to stare. Pick two alternatives `a` and `b`, and put voter 1's ranking with `a` above `b` above all remaining alternatives, while voter 2 ranks `b` above `a` above all remaining alternatives. Weak Pareto says the outcome is either `a` or `b`; all lower alternatives are dominated by both `a` and `b`. If the outcome is `a`, try dragging voter 2's report so that `a` falls to the bottom while `b` stays high for him. Weak Pareto still restricts the outcome to `a` or `b`, and strategy-proofness prevents the outcome from becoming `b`: if changing his report from the first ranking to the second made `b` win, then under the first ranking he would prefer the result of the changed report. So `a` still wins. Now monotonicity lets voter 1 move every other part of his ranking around while keeping `a` at the top. The rule must keep choosing `a` whenever voter 1 reports `a` first. For this pair, voter 1 is decisive for `a`.

If the initial outcome in that same construction had been `b`, the symmetric argument would make voter 2 decisive for `b`. Repeating the construction over pairs of alternatives partitions alternatives into those for which voter 1 is decisive and those for which voter 2 is decisive. Both voters cannot each own different alternatives. Suppose voter 1 is decisive for `a` and voter 2 is decisive for `c`. Build a profile where voter 1 puts `c` above `a`, and voter 2 puts `a` above `c`, with a third alternative available to keep the lower alternatives from doing irrelevant work. Voter 1's decisiveness for `a` and voter 2's decisiveness for `c` collide: the rule would have to choose `a` because voter 1 can put `a` first in the relevant monotonic extension, and choose `c` because voter 2 can put `c` first in the corresponding extension. A single-valued rule cannot do both. So in the two-voter world, once someone becomes decisive for one alternative, the same voter must be decisive across alternatives. That voter is a dictator.

The many-voter case should not require a new idea; it should compress voters until the two-voter argument can see a pivotal block. Split the electorate into voter 1 and a block containing all remaining voters who report the same ranking. Define a derived two-agent rule: the first agent reports voter 1's ranking, the second agent reports the common ranking for the whole block, and the original rule is applied to the resulting profile. If the original rule is onto and strategy-proof, this derived rule is onto and strategy-proof. Onto follows because weak Pareto lets a unanimous block force top alternatives in the derived profiles; strategy-proofness follows because a profitable manipulation by either derived agent would be a profitable manipulation by voter 1 or by one member of the identical block along a one-at-a-time path.

The two-agent theorem says the derived rule has a dictator. If voter 1 is the derived dictator, monotonicity extends that control back to the original rule: whenever voter 1 ranks an alternative first, changing the rest of the voters from a common ranking to arbitrary rankings cannot dislodge that top alternative without creating the same forbidden one-voter manipulation along the path. Then voter 1 is the original dictator.

If the block is the derived dictator, fix voter 1's report and look only at the remaining voters. The original rule induces a smaller voting rule on that block. It is strategy-proof because any profitable manipulation inside the smaller rule is one in the original rule. It is onto because the block's derived dictatorship means that for each alternative, when the block unanimously ranks it first, the original rule can select it under the fixed outside report. By induction on the number of voters, the smaller rule has a dictator. Now free the fixed outside report: if that block voter can dictate for every fixed report of voter 1, then monotonicity again carries the conclusion across profiles, and the same voter dictates in the original electorate.

So the escape routes close one by one. Strategy-proofness is not a mild local condition; it freezes the rule under favorable movements of the chosen alternative. Onto is not a harmless richness condition; together with that monotonicity it prevents Pareto-dominated choices. Those two facts create pivotal situations in which one voter must be decisive for an alternative, and the presence of at least three alternatives prevents decisiveness from being split consistently across different voters. The only deterministic onto rules on unrestricted strict rankings that leave no profitable manipulation are dictatorships. Equivalently, any onto non-dictatorial rule with at least three alternatives can be strategically manipulated.
