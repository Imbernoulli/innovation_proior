I start from the communication primitive, because that is where most tempting compressors quietly fail. Dense distributed SGD is expensive, but it has one property I cannot lose: the object being communicated is a fixed-shape tensor, and aggregation is addition. Ring or tree all-reduce can exploit that associativity, so the message does not become more structurally complicated as I add workers. If I send a compressed object whose structure depends on each worker's local gradient, then the sum of those objects may no longer be a compressed object of the same type. Once that happens I am pushed into gather, decode, merge, or re-encode logic, and the nominal compression ratio stops predicting wall-clock speed.

That immediately explains why the familiar coordinate schemes are not enough. Top-k with memory is attractive as an optimizer, because a residual can carry all dropped coordinates forward, but worker A's top coordinates and worker B's top coordinates are not the same set. Their sum has the union support, and that union can grow with the number of workers. Sign methods are tiny, but signs are nonlinear and majority vote is not averaging. Stochastic quantization is easier to analyze because it is unbiased, but quantized codes are still not generally closed under addition in the way a dense tensor is. I need the compressed representation to be built out of operations that commute with summing workers.

The gradient tensors themselves suggest another structure. A fully connected layer gives me a matrix gradient. A convolutional filter gradient can be flattened into a matrix by keeping output channels as rows and folding input and kernel dimensions into columns. Matrices can be low-rank even when their entries are not sparse. If an `m x n` gradient is well approximated by rank `r`, then a factorization `P Q^T` costs `(m+n)r` numbers instead of `mn`. This is the right scale for aggressive compression. But I have to be careful: the best rank-`r` approximation from an SVD is not a linear function of `M`, and it is expensive. If every worker computes its own truncated SVD, averaging those factors does not give the truncated SVD of the averaged gradient. That would repeat the same structural failure in spectral coordinates.

So I should not ask for the exact best low-rank approximation. I should ask for a low-rank approximation whose expensive steps are linear in `M`. Subspace iteration has exactly that shape. Start from a shared query matrix `Q`, compute `P = M Q`, orthogonalize `P`, and project back with `Q = M^T P`. The two matrix multiplications are linear in the local gradient as long as the other factor is shared. This gives me a route to an all-reduceable compressor, but only if I respect the order of operations.

The first multiplication must happen locally with the same query `Q` on every worker: `P_w = M_w Q`. Then I all-reduce the mean of those `P_w`, giving `P = (1/W) sum_w M_w Q`. Only after that can every worker orthogonalize the same aggregated `P`. Then each worker computes `Q_w = M_w^T P` against that common left factor, and a second all-reduce gives `Q = (1/W) sum_w M_w^T P`. The reconstructed update is `P Q^T`. This is the key implementation constraint. A one-shot hook that computes both local `P_w` and local `Q_w` before communication, then all-reduces both factors once, is not the distributed algorithm. It only simulates the method when there is one worker.

Now I have a low-rank compressor that is implementable, but it is still biased. A rank-`r` approximation drops trailing singular directions, and a single subspace-iteration step can miss part of the leading subspace too. I cannot pretend that the reconstructed matrix is an unbiased stochastic gradient. This is where error feedback enters. Before compression I add the worker's residual to the current gradient, in the same units the optimizer will update with. After reconstruction I store the difference between the corrected local matrix and the reconstructed aggregate approximation. In a one-worker run this is exactly `M - P Q^T`; in the distributed version the reference implementation uses the local corrected matrix minus the averaged low-rank update. The important invariant is that the local buffer holds what this step failed to send, so a test should see original corrected input equal approximation plus residual in the one-worker case.

I also need the subspace estimate to be good enough with only one left and one right multiplication. Starting from a fresh random `Q` every step wastes information. The gradient distribution changes gradually under small SGD steps, so the dominant subspace at this step is usually related to the previous one. Reusing the previous `Q` makes the next step continue a power iteration rather than restart it. The source argument is Oja-like stochastic subspace tracking, and the ablations show why warm start matters. I should keep this as an empirical and algorithmic rationale, not as a full theorem for the one-step changing-matrix case.

The orthogonalization details matter because the factors otherwise absorb arbitrary scale. The algorithm orthogonalizes the aggregated `P` before computing `Q = M^T P`. The reference package also orthogonalizes the input factor before each multiplication. That means I should normalize the warm-started `Q` before using it, compute and reduce `P`, orthogonalize the reduced `P`, compute and reduce `Q`, then store that reduced `Q` for the next step. On the next iteration I normalize it again before multiplying. This matches the reference code's stateful behavior and avoids scale drift.

There is a rank interface question. The original algorithm and official implementation expose rank as the main knob: rank one, two, four, and so on. The scaffold gives a `compress_ratio`, so if I must obey that interface I can solve `(m+n)r <= ratio * mn` and cap at `min(m,n)`. But that ratio-derived rank is a translation layer, not the essence of the method. I should expose a fixed `rank` parameter first and use `compress_ratio` only when no fixed rank is supplied. I should also skip matrix compression when it does not actually save enough communication; the official package uses a minimum compression-rate threshold for this reason.

Finally I need to bound the claims. Error feedback gives a corrected-iterate analysis under a contraction assumption; it does not make each actual compressed step equal dense SGD. The convergence appendix is conditional and partly informal for the one-step warm-started approximation, and the reviews explicitly call out the lack of a complete proof for the full practical algorithm. Likewise, the systems win is conditional. When bandwidth is the bottleneck and the encoded matrices are much smaller, the method can reduce wall time. When bandwidth is high, overlap is good, or encode/decode overhead dominates, third-party systems measurements show it can lose to optimized synchronous SGD. The final artifact should therefore say exactly what the method does: it trades dense gradient all-reduce for two fixed-shape low-rank all-reduces, uses warm-started subspace iteration to make those factors useful, and uses error feedback so the omitted part remains in local memory instead of disappearing.
