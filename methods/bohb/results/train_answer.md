We are tuning a machine-learning model whose validation loss depends on a mixed bag of hyperparameters: continuous learning rates, integer layer counts, categorical optimizers, and log-scaled regularizers. Each full evaluation costs hours or days, so the total tuning budget may only cover a handful of complete trainings. We need good configurations immediately and the best configuration eventually, while exploiting parallel workers, scaling to dozens of dimensions, and surviving noisy objectives like deep RL. The existing methods each solve part of this but never all of it at once.

Random search is robust and trivially parallel, yet it never learns: every draw ignores every previous result, so progress per evaluation never improves. Bayesian optimization with a Gaussian process provides guidance, but GP fitting is cubic in the number of observations, scales poorly beyond low-dimensional continuous spaces, needs bespoke kernels for categorical variables, and starts almost randomly while the surrogate is data-starved. That cold start is lethal when every evaluation is a full training run. Tree Parzen Estimators improve scalability by modeling input densities rather than the objective, but they still evaluate every configuration at full fidelity, so they inherit the cold-start cost. On the other side, multi-fidelity methods like Successive Halving and Hyperband use cheap approximations of the objective to triage configurations early, yielding strong anytime performance and staying robust across hyperparameter types. Successive Halving, however, forces an awkward choice between many cheap configurations and a few expensive ones. Hyperband hedges that choice by running a ladder of successive-halving brackets, but it draws every configuration uniformly at random, so its final quality plateaus toward random search as the budget grows.

The way forward is to keep what Hyperband does well and fix what it does poorly. Hyperband is excellent at deciding how much budget to allocate to which configurations, but it is blind when it chooses those configurations. A model-based acquisition is the opposite: poor at scheduling fidelities, excellent at proposing promising points. So we leave Hyperband's bracket schedule completely intact and replace only its random configuration draw with a density-ratio model.

The method is BOHB, Bayesian Optimization and HyperBand. It marries Hyperband's multi-fidelity skeleton with a TPE-style kernel density estimator. The surrogate is not a Gaussian process over loss values but two kernel density estimators over configuration space: one fit to the best observed configurations and one fit to the rest. A configuration is attractive when it looks like the good ones and unlike the bad ones, which means maximizing the ratio of good density to bad density. This ratio is not a heuristic: with a quantile threshold separating good from bad, maximizing good density over bad density is exactly maximizing expected improvement. The derivation is clean because the integral over losses factorizes into an x-independent constant divided by a monotone function of g over l, so argmax l over g is the EI maximizer without ever modeling the loss distribution.

To make this work under Hyperband, observations are separated by budget. For each fidelity we maintain a set of completed trials, and we fit good and bad KDEs on the largest budget that already has enough data. Early in the run this is a small budget, so cheap evaluations bootstrap the search quickly; as more high-fidelity results arrive, the model used for proposals climbs the fidelity ladder and can overturn earlier low-budget verdicts. We need at least d plus one points to fit a d-dimensional KDE, and we require both the good slice and the bad slice to be nondegenerate before trusting the model; if no budget has a usable model yet, we fall back to uniform random sampling. Crashed or non-finite runs are recorded as infinite loss so they are naturally classified as bad configurations and push the densities away from broken regions.

Proposals are generated by sampling candidates from the good density and scoring them with the un-widened good-over-bad ratio. We deliberately avoid fully optimizing the acquisition function: we draw a modest number of candidates and keep the best, which keeps consecutive suggestions diverse and lets parallel workers stay productive. To prevent the high-budget model from stagnating, since it is queried often but updated rarely, we sample candidates from a widened version of the good KDE while still scoring with the original fitted bandwidths. A small minimum bandwidth floor prevents any dimension from collapsing to a non-exploring spike. Finally, a constant fraction of proposals remains uniformly random, preserving Hyperband's convergence guarantee: even if the lower fidelities mislead the model, BOHB cannot starve any region forever and remains at most a constant factor slower than pure random search in the worst case.

```python
import numpy as np
import scipy.stats as sps
import statsmodels.api as sm


class BOHBConfigGenerator:
    """Model-based configuration generator for BOHB.

    Replaces Hyperband's random configuration sampling with a TPE-style
    density-ratio model. For each budget, fit a "good" KDE on the best
    observed configurations and a "bad" KDE on the rest, then propose the
    configuration maximizing l(x)/g(x), which is the expected-improvement
    maximizer.
    """

    def __init__(self, space, top_n_percent=15, num_samples=64,
                 random_fraction=1/3, bandwidth_factor=3, min_bandwidth=1e-3,
                 min_points_in_model=None, seed=42):
        self.space = space
        self.rng = np.random.RandomState(seed)
        self.top_n_percent = top_n_percent
        self.num_samples = num_samples
        self.random_fraction = random_fraction
        self.bw_factor = bandwidth_factor
        self.min_bandwidth = min_bandwidth
        d = space.dim
        self.min_points = (d + 1) if min_points_in_model is None else max(min_points_in_model, d + 1)
        self.kde_vartypes = "".join(
            'u' if p.type == 'categorical' else 'c' for p in space.params
        )
        self.vartypes = np.array([
            len(p.choices) if p.type == 'categorical' else 0
            for p in space.params
        ], dtype=int)
        self.configs = {}      # budget -> list of encoded config vectors
        self.losses = {}       # budget -> list of losses (lower is better)
        self.kde_models = {}   # budget -> {'good': KDE, 'bad': KDE}

    def new_result(self, config, loss, budget):
        # Crashed or non-finite runs count as bad configurations.
        loss = loss if np.isfinite(loss) else np.inf
        self.configs.setdefault(budget, []).append(self.space.to_array(config))
        self.losses.setdefault(budget, []).append(loss)

        # If a larger budget already has a model, do not refit a smaller one.
        if self.kde_models and max(self.kde_models) > budget:
            return
        if len(self.configs[budget]) <= self.min_points - 1:
            return

        X = np.array(self.configs[budget])
        y = np.array(self.losses[budget])
        n_good = max(self.min_points, (self.top_n_percent * X.shape[0]) // 100)
        n_bad = max(self.min_points, ((100 - self.top_n_percent) * X.shape[0]) // 100)
        order = np.argsort(y)
        good = X[order[:n_good]]
        bad = X[order[n_good:n_good + n_bad]]
        if good.shape[0] <= good.shape[1] or bad.shape[0] <= bad.shape[1]:
            return

        good_kde = sm.nonparametric.KDEMultivariate(
            good, var_type=self.kde_vartypes, bw='normal_reference'
        )
        bad_kde = sm.nonparametric.KDEMultivariate(
            bad, var_type=self.kde_vartypes, bw='normal_reference'
        )
        good_kde.bw = np.clip(good_kde.bw, self.min_bandwidth, None)
        bad_kde.bw = np.clip(bad_kde.bw, self.min_bandwidth, None)
        self.kde_models[budget] = {'good': good_kde, 'bad': bad_kde}

    def get_config(self):
        # Random fraction, or fallback when no model exists yet.
        if not self.kde_models or self.rng.rand() < self.random_fraction:
            return self.space.sample_uniform(self.rng)

        budget = max(self.kde_models)
        kde_good = self.kde_models[budget]['good']
        kde_bad = self.kde_models[budget]['bad']
        l = kde_good.pdf
        g = kde_bad.pdf
        ratio = lambda x: max(1e-32, g(x)) / max(l(x), 1e-32)

        best_val, best_vec = np.inf, None
        for _ in range(self.num_samples):
            datum = kde_good.data[self.rng.randint(len(kde_good.data))]
            vec = []
            for value, bw, vt in zip(datum, kde_good.bw, self.vartypes):
                bw = max(bw, self.min_bandwidth)
                if vt == 0:  # continuous: jitter around good datum with widened kernel
                    bw = self.bw_factor * bw
                    a, b_ = (0 - value) / bw, (1 - value) / bw
                    vec.append(sps.truncnorm.rvs(
                        a, b_, loc=value, scale=bw, random_state=self.rng
                    ))
                else:  # categorical: keep or randomly flip
                    if self.rng.rand() < (1 - bw):
                        vec.append(int(value))
                    else:
                        vec.append(self.rng.randint(vt))
            val = ratio(vec)
            if val < best_val:
                best_val, best_vec = val, vec

        if best_vec is None:
            return self.space.sample_uniform(self.rng)
        return self.space.from_array(best_vec)


def hyperband_brackets(min_budget, max_budget, eta=3):
    """Hyperband's geometric ladder of successive-halving brackets.

    Each bracket consumes roughly the same total resource and starts
    configurations at a different initial budget.
    """
    max_sh_iter = int(np.log(max_budget / min_budget) / np.log(eta)) + 1
    budgets = max_budget * eta ** (-np.arange(max_sh_iter - 1, -1, -1))
    brackets = []
    for s in range(max_sh_iter - 1, -1, -1):
        n0 = int(np.floor(max_sh_iter / (s + 1)) * eta ** s)
        ns = [max(int(n0 * eta ** (-i)), 1) for i in range(s + 1)]
        brackets.append({'budgets': budgets[-(s + 1):], 'num_configs': ns})
    return brackets
```

The overall algorithm is then: run Hyperband's bracket schedule unchanged, but whenever a new configuration is needed, call the model generator above instead of drawing uniformly at random. Completed trials are registered with their budget so the per-budget KDEs accumulate evidence across all brackets and iterations. This pairing gives Hyperband's strong early progress through cheap fidelities and TPE's strong final progress through model-guided configuration selection, while remaining linear-time, mixed-space native, embarrassingly parallel, and simple to implement.
