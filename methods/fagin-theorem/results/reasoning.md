I want a description of nondeterministic polynomial time that does not begin with a machine. The machine definition tells me that an accepting computation exists, but the object I care about is often a finite graph, a finite database, or some other finite structure. If the property is really about that structure, then the names used in one string encoding should not be essential.

I start from spectra because they already turn finite existence into logic. A first-order sentence can have finite models of some sizes and not others. That is a clean logical object, but it is also the first wall: a set of sizes is not the same thing as a class of finite structures. If I only ask whether there is a model of size `n`, the input is essentially the number `n`; a search through relation tables on `n` elements is enormous compared with the length of the number. This points toward exponential time, not ordinary polynomial time.

So I shift from sizes to structures. I let the input itself be a finite structure with its relation tables already encoded. Now adding a guessed relation of fixed arity costs only polynomially many bits in the size of the structure. This is the scale on which nondeterministic polynomial time lives. The guessed object in an NP computation can be reinterpreted as an extra finite relation on the input universe.

The easy half becomes almost forced. If a sentence says that there exist auxiliary relations `R1,...,Rk` such that a first-order condition `phi` holds, then a nondeterministic algorithm can guess the finite tables for those relations and evaluate `phi` on the expanded structure. The arities and the formula are fixed, so the guessed tables are polynomial in the input size and the first-order check is polynomial.

The harder half is to make every polynomial-time nondeterministic computation look like that. I first think of simply saying "there is an accepting run," but first-order logic cannot quantify over a sequence of polynomial length as a sequence. The repair is to represent the run as relations over tuples of elements. If the machine runs for `n^k` steps, then `k`-tuples from an `n`-element universe provide enough indices for time and tape positions.

I then let the second-order existential prefix guess the whole computation tableau. For each possible tape-cell content, including state-symbol pairs when the head is present, I use a relation `C_i(s,t)` saying that cell position `s` at time `t` has content `i`. I use another relation for nondeterministic choices when that makes the local transition formula simpler. This guessed tableau is exactly the certificate, but expressed as finite relations rather than as a string witness.

The first-order part now has a concrete job. It checks that the first row encodes the input, that each cell has a consistent content, that every row follows legally from the previous one by looking only at a cell and its neighbors, and that an accepting state appears at the designated final time. These are local constraints, so first-order logic can say them once and quantify over all tuple-indexed positions.

I still need order. A machine tape and time line are ordered, while an arbitrary finite structure is not. I do not want to build order into the input property, because that would make the result depend on a presentation of the structure. The solution is to existentially guess an auxiliary order as part of the witness. It is bookkeeping for tuple indexing, not extra computational power beyond the same second-order existential guess.

This also explains why the nonempty-structure setting is the right one for ordinary NP. With a real input vocabulary, the encoding of a structure has polynomial room for its relations, so a polynomial-time machine can be matched by tuple-indexed first-order constraints. With no input relations, the older spectrum question falls back to cardinalities and the exponential scale reappears.

The resulting theorem is the bridge I was looking for. Nondeterminism becomes existential quantification over finite relations. Polynomial-time verification becomes first-order checking of the enriched structure. The class is no longer only a runtime class; it is exactly the expressive power of adding finite relational witnesses and then reasoning first-order about them.
