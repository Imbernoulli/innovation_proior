## Research question

We have a database `D` of `m` complete cases over `n` discrete variables `x_1, ..., x_n`, each
case an integer-coded joint observation, and we believe the data was generated by some
Bayesian network — a directed acyclic graph `G` whose factorization
`p(x_1, ..., x_n) = ∏_i p(x_i | Π_i)` (each variable conditioned on its parents `Π_i`) encodes
the conditional-independence structure of the domain. The graph is hidden; we want to recover
the network structure that the data most supports from the data alone, using a probabilistic
criterion computable from the sufficient statistics (counts) of the database. Two networks that
assert exactly the same independences are statistically indistinguishable from observational
data, so the recoverable object is the *equivalence class* of the true graph rather than
necessarily a single oriented DAG.

The space being searched is combinatorial. The number of DAGs on `n` nodes grows
super-exponentially — Robinson's recursion gives 3 structures for `n = 2`, 25 for `n = 3`,
about 29,000 for `n = 5`, and roughly `4.2 × 10^18` for `n = 10`.

## Background

**The Bayesian scoring framework.** A Bayesian belief network is a pair `(B_S, B_P)`: a DAG
structure `B_S` and a set of conditional probability distributions `B_P`. From the chain rule
plus the independence assertions of `B_S`, any joint factors as `∏_i p(x_i | Π_i)`. To rank
structures by how well they explain the data, one treats the structure itself as uncertain and
computes its posterior probability,
`p(B_S | D) = c · p(B_S) · p(D | B_S)`, where `c = 1/p(D)` is a normalizing constant common to
all structures. Because `c` is the same for every `B_S`, only the relative size of
`p(B_S, D) = p(B_S) · p(D | B_S)` matters, and that joint is what people compute as a
**score**. The marginal likelihood `p(D | B_S) = ∫ p(D | B_S, Θ) p(Θ | B_S) dΘ` integrates the
unknown table parameters `Θ` out against a prior — it is not a single fitted likelihood but an
average over all parameter settings, which automatically penalizes overly complex structures
(they spread their prior mass thinly and average down).

**Closed form via Dirichlet conjugacy.** A standard set of assumptions makes that integral
collapse to counts. (1) Multinomial sampling: the data is an i.i.d. multinomial sample from
some fixed network. (2) Parameter independence: the parameter vectors `Θ_ij` for different
variables, and for different parent configurations of the same variable, are a priori
independent (only the entries of one conditional distribution, which must sum to one, are
coupled). (3) Complete data: no missing values. (4) Parameter modularity: if `x_i` has the
same parents in two structures, its prior density `ρ(Θ_ij)` is the same in both — the local
prior depends only on the local family. (5) Dirichlet priors: each `Θ_ij` is Dirichlet with
exponents `N'_ijk > 0`. The Dirichlet is conjugate to the multinomial, so after seeing counts
`N_ijk` (the number of cases with `x_i = k` and `Π_i = j`) the posterior is again Dirichlet,
and the marginal likelihood is a product of Gamma-function ratios:

```
p(D, B_S) = p(B_S) · ∏_i ∏_{j=1}^{q_i} {
    [ Γ(N'_ij) / Γ(N'_ij + N_ij) ]
    · ∏_{k=1}^{r_i} [ Γ(N'_ijk + N_ijk) / Γ(N'_ijk) ]
}
```

where `r_i` is the number of states of `x_i`, `q_i = ∏_{p ∈ Π_i} r_p` is the number of parent
configurations, `N_ij = Σ_k N_ijk`, and `N'_ij = Σ_k N'_ijk`. The `N'_ijk` are Dirichlet
pseudocounts (prior "imaginary" observations); `Γ(x+1) = x·Γ(x)` reduces the ratios to
factorials when the exponents are integers. This is a Bayesian-Dirichlet-style score.

**Decomposability.** The score is a *product over nodes*, each factor a function only of
`x_i` and its parents `Π_i`. Equivalently, the log-score is a sum:
`log p(D, B_S) = Σ_i s(x_i | Π_i)`, with `s(x_i | Π_i)` the local family score. Two structures
that differ only in the parents of one node share every term except that node's, so their
scores can be compared by recomputing a single local term.

**Score equivalence and what is identifiable.** Two structures are *isomorphic* (Markov
equivalent) when they assert the same set of conditional independences — for example
`x → y → z`, `x ← y → z`, and `x ← y ← z` all assert only "x ⫫ z | y". A score has the
property of **score equivalence** when isomorphic structures always receive the same score.
This matters because observational data cannot tell isomorphic structures apart: any criterion
based on `p(D | B_S)` *should* score them equally, and if it does, the best one can hope to
recover is the whole equivalence class. An equivalence class is summarized by a Completed
Partially Directed Acyclic Graph (CPDAG): edges that are oriented the same way in every member
are directed, and edges whose orientation varies within the class are left undirected.

**An equivalent-sample-size construction for the pseudocounts.** Specifying a separate
`N'_ijk` for every structure by hand is hopeless. The pseudocounts can instead be tied to two
inputs: a prior probability distribution over the domain (encoding prior belief about the next
case) and a single scalar `N'` interpreted as an *equivalent sample size* — the number of
prior "imaginary" cases the prior is worth. Winkler (1967) showed how a user can be trained to
assess such an `N'`. The constraint that ties them is
`N'_ijk = N' · p(x_i = k, Π_i = j | prior)`. The scalar `N'` is a gain/smoothing control: a
small `N'` lets the data dominate the prior predictive distribution quickly; a large `N'`
keeps the marginal likelihood anchored to it.

**Diagnostic prior-art observations.** A greedy parent-selection procedure (below), given a
correct ordering of the variables and 10,000 cases Monte-Carlo-sampled from the 37-node,
46-arc ALARM medical-monitoring network, reconstructed it up to one missing and one extra arc,
and reproduced essentially the same network from only 3,000 cases — evidence that a
Bayesian-scored greedy search can recover a moderate network from a feasible amount of data.
Robinson's super-exponential count of DAGs, and the established NP-hardness of finding the
top-scoring structure once a node may have more than one parent (even with a score-equivalent
metric), rule out exhaustive search.

## Baselines

These are the prior structure-learning methods a new procedure would be measured against and
react to.

**Exhaustive Bayesian scoring (Cooper & Herskovits 1992).** With uniform structure prior and
unit Dirichlet pseudocounts (`N'_ijk = 1`), the structure-dependent part of the score reduces to
`∏_i ∏_{j=1}^{q_i} { [ (r_i − 1)! / (N_ij + r_i − 1)! ] · ∏_{k=1}^{r_i} N_ijk! }`, computable
from counts alone. One can in principle apply this to every structure and take the maximum.

**K2 — greedy parent selection under a fixed ordering (Cooper & Herskovits 1992).** Exploit
decomposability: since the total score factors over nodes, maximize each node's local family
score independently. Assume a topological ordering of the variables; for each node `x_i`,
start with no parents and repeatedly add, from the predecessors of `x_i` in the ordering, the
single candidate parent whose addition most increases the local score `g(i, Π_i)`, stopping
when no addition helps or a parent-count cap is reached. The ordering guarantees the result is
acyclic for free. With factorials precomputed, K2 runs in low-order polynomial time and
recovered ALARM from sampled data.

**Constraint-based skeleton recovery (the Spirtes–Glymour–Scheines line).** Rather than score
structures, run a battery of conditional-independence tests to decide which arcs to include,
then orient what the tests determine, returning a CPDAG.

**Special-case polynomial structure search (Chickering, Geiger & Heckerman 1995).** When every
node is restricted to at most one parent, the highest-scoring structure can be found exactly in
polynomial time by reducing to a maximum-branching / maximum-weight-spanning-tree problem on
edge weights `w(x_i, x_j) = s(x_i | x_j) − s(x_i | ∅)`. With a score-equivalent metric
the two orientations of an edge carry equal weight, so the problem becomes an undirected
maximum-weight forest (a Chow–Liu-style tree).

## Evaluation settings

The natural yardsticks that already exist for discrete structure learning. Benchmarks are the
well-known discrete Bayesian networks from the bnlearn repository, each with a known
ground-truth DAG, sampled to produce an integer-coded observational database: small
(Cancer, 5 nodes / 4 edges; Child, 20 / 25), medium (Alarm, 37 / 46; Hailfinder, 56 / 66) and
large (Win95pts, 76 / 112), spanning medicine, meteorology and IT and a range of variable
cardinalities and edge densities. Each network is sampled at a fixed observational sample size,
so a method must generalize across scales and cardinality patterns. The output is compared to
the ground-truth network's CPDAG (obtained by collapsing the true DAG to its equivalence
class). The standard metrics are Structural Hamming Distance between estimated and true CPDAG
(total edge insertions/deletions/orientation errors, lower better), adjacency precision/recall
(skeleton recovery), and arrowhead precision/recall (orientation accuracy). The interface is
fixed: a function receiving an integer matrix `X` of shape `(n_samples, n_variables)` and
returning the estimated CPDAG as a `causallearn` `GeneralGraph`.

## Code framework

The procedure plugs into a generic score-based structure-search harness. What already exists:
the data matrix; a decomposable local score over a child node and a candidate parent set,
exposed as `score(child, parents)` returning a number such that lower is better, summing over
nodes to the structure's total score, with a per-node cache so a local family score is computed
once per distinct parent set; a DAG kept as an adjacency matrix with a reachability test for
acyclicity; and the graph datatypes (`GraphNode`, `GeneralGraph`) used to build the output. The
harness stops before specifying the search procedure.

```python
import numpy as np
from causallearn.graph.GeneralGraph import GeneralGraph
from causallearn.graph.GraphNode import GraphNode
from causallearn.score.LocalScoreFunctionClass import LocalScoreClass


def run_causal_discovery(X: np.ndarray) -> GeneralGraph:
    """
    Input:  X of shape (n_samples, n_variables), integer-encoded discrete data
    Output: estimated graph as causallearn.graph.GeneralGraph.GeneralGraph
    """
    N = X.shape[1]
    nodes = [GraphNode(f"X{i + 1}") for i in range(N)]

    # A decomposable, cached local score over (child, parent_set):
    # score_func.score(j, parents) -> number; lower total is better, and the
    # structure's total score is the sum of per-node local scores.
    score_func = LocalScoreClass(data=X, local_score_fun=None, parameters=None)

    # The structure as a directed adjacency matrix, plus an acyclicity test.
    adj = np.zeros((N, N), dtype=int)        # adj[i, j] == 1 means edge i -> j

    def _has_path(src, tgt):
        """Is there a directed path src -> ... -> tgt in adj? (acyclicity guard)"""
        visited, stack = set(), [src]
        while stack:
            node = stack.pop()
            if node == tgt:
                return True
            if node in visited:
                continue
            visited.add(node)
            for c in np.where(adj[node] == 1)[0]:
                if int(c) not in visited:
                    stack.append(int(c))
        return False

    # TODO: the structure-learning procedure.
    pass

    G = GeneralGraph(nodes)
    for i in range(N):
        for j in range(N):
            if adj[i, j] == 1:
                G.add_directed_edge(nodes[i], nodes[j])
    return G
```

The harness supplies the local-score machinery, the cache, the acyclicity test, and the graph
datatypes; the empty slot is the structure-learning procedure.
