I start from the part of exploration that the usual deep-RL agents make too local. If I have a value network and I use `epsilon`-greedy, the randomness is not a belief or a plan; it is a coin flip at the action output. If the coin says explore, I sample a uniform action, and the next state gets a fresh coin. In actor-critic, the entropy bonus keeps the policy distribution broad, but the sampled action is still local stochasticity from the current policy. This can help me avoid a brittle greedy policy, but it does not naturally commit me to a structured sequence of exploratory actions. A useful detour through an Atari game may require several coordinated choices before any payoff appears, and independent action noise is a poor way to find that kind of information.

The second flaw is the scale. `epsilon` and the entropy coefficient are numbers I choose outside the learning problem. They are the same kind of external knob across states that I know well and states that I do not know at all. I would rather the agent controlled its own amount of stochasticity from the same loss that trains the value or policy network. If exploration is still useful, the perturbation should be allowed to remain; if the learned behaviour has become stable, the perturbation should be allowed to shrink. For that to happen the noise scale has to be a parameter that gradients can move, not a schedule I impose.

So I try moving the stochasticity upstream. Instead of corrupting the selected action, I corrupt the function that maps states to values or policies. A fixed perturbation of network parameters induces a different function, and because the perturbation flows through the network, its effect depends on the input state. That gives me the two properties action dithering lacks: the variation is state-dependent, and while a sample is fixed the agent behaves according to one coherent sampled function rather than a new random action rule at every state. I need to be precise about what "while a sample is fixed" means, though. My first instinct is "draw once per episode", but that is not quite the right rule and I do not want to bake an episode boundary into the layer. The cleaner statement is that the sampled parameters are held fixed between optimization steps. In the DQN and dueling instantiations there is one replay update per action step, so in those agents this happens to mean resampling before each action; in A3C the update happens after an `n`-step rollout, so the same sample has to stay fixed for the whole rollout. The rule is tied to the update cadence, not to the episode.

Now I need this perturbation to be trainable, and this is where I have to be careful, because the naive thing â€” make the variance a parameter and try to differentiate through a random draw â€” does not give a usable gradient unless the randomness is pushed outside the parameters. Let the noisy parameter vector be `theta = mu + Sigma * epsilon`, with elementwise multiplication, zero-mean fixed-statistics noise `epsilon`, and learnable `zeta = (mu, Sigma)`. The point of writing it this way is that `epsilon` carries all the randomness and its distribution does not depend on `zeta`. So the objective `Lbar(zeta) = E_epsilon[L(mu + Sigma * epsilon)]` can be differentiated under the expectation: `grad_zeta Lbar(zeta) = E_epsilon[grad_{mu,Sigma} L(mu + Sigma * epsilon)]`, which I approximate in practice with one draw. I want to know concretely what each component of that gradient looks like, because if `Sigma` does not get a sensible signal the whole idea is dead. The derivative with respect to `mu` should just be the ordinary weight derivative, since `theta = mu + Sigma*epsilon` and `d theta / d mu = 1`. The derivative with respect to `Sigma` should be the same local derivative scaled by the noise, since `d theta / d Sigma = epsilon` elementwise. Let me actually run autograd on a tiny `2 x 3` noisy linear layer to check I have not confused myself: with fixed sampled `epsilon` treated as a constant, `y = (mu + sigma*epsilon) x`, and a scalar loss `L = sum(y^2)`. Autograd gives

```text
grad_mu    = [[ 4.5261,  7.6574, -1.8794],
              [-0.3050, -0.5161,  0.1267]]
grad_sigma = [[-3.8774,  8.4278,  2.0132],
              [-0.0374,  0.2923,  0.0473]]
grad_mu * epsilon = [[-3.8774,  8.4278,  2.0132],
                     [-0.0374,  0.2923,  0.0473]]
```

`grad_sigma` equals `grad_mu` multiplied elementwise by the sampled noise, exactly. So `Sigma` receives the local weight gradient weighted by whichever way the sampled noise happened to push that weight, which means backpropagation can learn, per parameter, whether injecting variation there helped or hurt the loss. That is the property I wanted, and now I have seen it rather than assumed it.

For a linear layer with `p` inputs and `q` outputs, the ordinary map is `y = w x + b`, where `w` has shape `q x p` and `b` has shape `q`. I replace it with
`y = (mu^w + sigma^w * epsilon^w) x + mu^b + sigma^b * epsilon^b`.
The learnable tensors `mu^w` and `sigma^w` have shape `q x p`; `mu^b` and `sigma^b` have shape `q`. The independent case is the obvious one: every weight-noise entry and every bias-noise entry is an independent standard Gaussian. That costs `pq + q` Gaussian draws per layer. In the distributed A3C case random-number generation is not the bottleneck, so I can afford it there.

For a single-threaded value agent, though, drawing a full `q x p` noise matrix for every noisy fully connected layer is the part I worry about: it is the same `O(pq)` work as the matrix multiply itself, just to produce noise. So I look for a way to generate the `q x p` weight noise from far fewer draws. The natural move is to make the noise matrix a rank-one outer product: draw `p` input noises and `q` output noises, and set `epsilon^w_{j,i} = g(epsilon^out_j) g(epsilon^in_i)` for some scalar transform `g`, with `epsilon^b_j = g(epsilon^out_j)`. That drops the count from `pq + q` to `p + q`. The question is what `g` should be and whether the resulting noise stays well-behaved.

If I used `g = identity`, a weight entry would be a product of two standard Gaussians; its variance would be `E[Z_out^2] E[Z_in^2] = 1`, which is fine, but products of raw Gaussians have heavy tails and the factors themselves are not bounded, which makes the per-entry scale swing more than I want when I later multiply by `sigma`. The transform `g(x) = sign(x) sqrt(|x|)` is the compromise that is actually used: it shrinks large magnitudes (square-rooting them) while preserving sign and zero-mean structure. I should check it stays order one rather than just assert it. For `Z ~ N(0,1)`, `g` is odd so `E[g(Z)] = 0` by symmetry, and `E[g(Z)^2] = E[|Z|]`, the mean of a half-normal, which is `sqrt(2/pi)`. A factorized weight entry is a product of two independent such factors, so its variance is `(E[g(Z)^2])^2 = (sqrt(2/pi))^2 = 2/pi`. Let me sanity-check these against a Monte-Carlo draw of twenty million samples:

```text
E[g(Z)]        ~ -2.4e-05      (target 0)
E[g(Z)^2]      ~  0.79764      vs sqrt(2/pi) = 0.79788
E[|Z|]         ~  0.79764      (matches E[g(Z)^2], as it must)
mean eps_w     ~ -2.1e-04      (target 0)
var  eps_w     ~  0.63666      vs 2/pi = 0.63662
```

The factors are zero-mean, and the weight-noise variance is `2/pi ~ 0.637`, not one. I should not claim the factorization reproduces unit-variance independent noise â€” it does not â€” but `0.637` is order one, and that is all I need: the noise stays at a sensible scale while the random-number cost drops from `pq + q` to `p + q`. The slightly-below-one variance can be absorbed into the learned `sigma`, which is exactly the point of making `sigma` trainable.

The initialization has two cases and I should keep the constants separate, because they target different variances. For independent noise I want the mean weights to start with per-output variance about `1/p` (a standard fan-in scaling), so I draw them from `U[-sqrt(3/p), sqrt(3/p)]`. A uniform on `[-a, a]` has variance `a^2/3`, and with `a = sqrt(3/p)` that is `(3/p)/3 = 1/p` â€” I checked numerically that `U[-sqrt(3/7), sqrt(3/7)]` gives `0.1429` against `1/7 = 0.1429`, so the constant `3` is doing exactly that job. The scale parameters in the independent case are all initialized to `0.017`; that constant is inherited from earlier noisy supervised-network experiments and is not something I derive here. For factorized noise the mean parameters are initialized from `U[-1/sqrt(p), 1/sqrt(p)]` instead â€” variance `1/(3p)`, three times smaller, which pairs with the factorized noise having already been scaled â€” and I set the scale initialization to `sigma_0 / sqrt(p)` with `sigma_0 = 0.5`. For the code artifact I also have to track the community Rainbow implementation, which I am following for the saved code: it initializes `weight_sigma` with `std_init / sqrt(in_features)` and `bias_sigma` with `std_init / sqrt(out_features)`. That `out_features` in the bias is an implementation convention in the reference code, not a new derivation, and I keep it so the saved artifact matches the reference.

I then wire the sampled functions into the RL losses. For DQN, I remove `epsilon`-greedy and choose the greedy action under the current sampled value function. The replay loss has to use one noise sample for the online network and an independent sample for the target network:
`[r + gamma max_b Q(y,b,epsilon'; zeta^-) - Q(x,a,epsilon; zeta)]^2`.
The independence here is not cosmetic. If I shared the same `epsilon` between the online and target networks, the perturbation in the bootstrapped target would be correlated with the perturbation in the current estimate, and the regression target would move together with the prediction in a way that defeats the point of a fixed target. So I draw them separately. For action collection I draw yet another independent sample and act greedily with respect to it. Within one optimization step the whole replay batch shares the current noise sample, so every example in that step sees the same sampled network â€” the sample is the agent's current "function", and it should be consistent across the batch.

For dueling DQN, the streams and the mean-subtracted advantage combination stay the same. The target is the double-DQN form: an online sampled network selects `b*(y) = argmax_b Q(y,b,epsilon''; zeta)`, and the target sampled network evaluates `Q(y,b*(y),epsilon'; zeta^-)`. This is the same selection/evaluation separation as ordinary double DQN, now with independent noise for online evaluation, target evaluation, and action selection â€” the same independence argument as above, applied to all three roles.

For A3C, I remove the entropy bonus because the stochasticity now comes from sampling the network parameters, and I make the fully connected policy and value heads noisy, using independent Gaussian noise in my main A3C variant. The on-policy detail is the one that actually constrains the design. A3C's `n`-step return
`Qhat_i = sum_{j=i}^{k-1} gamma^{j-i} r_{t+j} + gamma^{k-i} V(x_{t+k}; zeta_V, epsilon_i)`
is only a valid estimate for one policy. If I resampled inside the rollout, the actions `r_{t+j}` would be generated by a sequence of different sampled policies and the bootstrap `V` by yet another, so the return would not correspond to any single policy and the policy-gradient update would be biased. So I force every `epsilon_i` in the rollout to be the same `epsilon`, and the policy-gradient and value-gradient updates are taken with that one sampled network fixed throughout the rollout. This is the same "hold fixed between optimization steps" rule from the start, instantiated for an agent whose step spans a whole rollout rather than a single action.

Putting it together: replace only the fully connected heads with a layer whose means and noise scales are learnable; use independent Gaussian noise where random-number cost is acceptable and factorized Gaussian noise where it is not; delete the separate value-agent `epsilon` schedule and the actor-critic entropy bonus; keep online, target, and acting samples independent in value learning; and keep one sample fixed across an on-policy rollout. What this is not is a Bayesian posterior sampler, and it is not a fixed-variance parameter-noise heuristic â€” the verified gradient `grad_sigma = grad_mu * epsilon` is what separates it from both: it is ordinary deep-RL optimization with a reparameterized, learnable source of parameter noise whose scale the loss itself controls.
