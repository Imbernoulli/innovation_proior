I start from the part of exploration that the usual deep-RL agents make too local. If I have a value network and I use `epsilon`-greedy, the randomness is not a belief or a plan; it is a coin flip at the action output. If the coin says explore, I sample a uniform action, and the next state gets a fresh coin. In actor-critic, the entropy bonus keeps the policy distribution broad, but the sampled action is still local stochasticity from the current policy. This can help me avoid a brittle greedy policy, but it does not naturally commit me to a structured sequence of exploratory actions. A useful detour through an Atari game may require several coordinated choices before any payoff appears, and independent action noise is a poor way to find that kind of information.

The second flaw is the scale. `epsilon` and the entropy coefficient are numbers I choose outside the learning problem. They are the same kind of external knob across states that I know well and states that I do not know at all. I want the agent to control its own amount of stochasticity from the same loss that trains the value or policy network. If exploration is still useful, the perturbation should be allowed to remain; if the learned behaviour has become stable, the perturbation should be allowed to shrink.

So I move the stochasticity upstream. Instead of corrupting the selected action, I corrupt the function that maps states to values or policies. A fixed perturbation of network parameters induces a different function, and because the perturbation flows through the network, its effect depends on the input state. That gives me the two properties action dithering lacks: the variation is state-dependent, and while a sample is fixed the agent behaves according to one coherent sampled function rather than a new random action rule at every state. I need to be precise about the implementation, though. The general rule is not "one draw forever" or even always "one draw per episode"; the rule is to hold the sampled parameters fixed between optimization steps. In the DQN and dueling instantiations, there is one replay update per action step, so the algorithm resamples before each action. In A3C, the update happens after an `n`-step rollout, so the same sample must stay fixed for the whole rollout.

Now I need this perturbation to be trainable. Let the noisy parameter vector be
`theta = mu + Sigma * epsilon`, with elementwise multiplication, zero-mean fixed-statistics noise `epsilon`, and learnable `zeta = (mu, Sigma)`. This is not a posterior distribution and I should not describe it as one. It is a parameterized source of noise whose scale is trained by gradient descent. The objective becomes `Lbar(zeta) = E_epsilon[L(mu + Sigma * epsilon)]`. Because the noise distribution does not depend on `zeta`, the gradient can be estimated by the reparameterized single-sample gradient
`grad_zeta Lbar(zeta) = E_epsilon[grad_{mu,Sigma} L(mu + Sigma * epsilon)]`, approximated in practice with one draw. The derivative with respect to `mu` is the ordinary weight derivative. The derivative with respect to `Sigma` is the same local derivative multiplied by the sampled noise, so backpropagation can learn whether a parameter wants more or less injected variation.

For a linear layer with `p` inputs and `q` outputs, the ordinary map is `y = w x + b`, where `w` has shape `q x p` and `b` has shape `q`. I replace it with
`y = (mu^w + sigma^w * epsilon^w) x + mu^b + sigma^b * epsilon^b`.
The learnable tensors `mu^w` and `sigma^w` have shape `q x p`; `mu^b` and `sigma^b` have shape `q`. The independent case is straightforward: every weight-noise entry and every bias-noise entry is an independent standard Gaussian. That costs `pq + q` Gaussian draws per layer. It is acceptable for the distributed A3C case, where random-number generation is not the bottleneck.

For a single-threaded value agent, drawing a full `q x p` noise matrix for every noisy fully connected layer is too expensive relative to the actual matrix multiply. I factor the noise. I draw `p` independent input noises and `q` independent output noises, transform them with `f(x) = sign(x) sqrt(|x|)`, and set `epsilon^w_{j,i} = f(epsilon^out_j) f(epsilon^in_i)` and `epsilon^b_j = f(epsilon^out_j)`. The weight-noise tensor is an outer product, so the number of Gaussian draws falls from `pq + q` to `p + q`. The transform keeps the factors zero-mean and moderate in scale: for `Z ~ N(0,1)`, `E[f(Z)] = 0` and `E[f(Z)^2] = E[|Z|] = sqrt(2/pi)`, so a factorized weight entry has variance `2/pi`. The variance is not exactly one, and I should not claim it is; the important point is that it remains order one while the random-number cost is much lower.

The initialization has two cases and I should keep the constants separate. For independent noise, the mean parameters are initialized from `U[-sqrt(3/p), sqrt(3/p)]`, whose variance is `1/p`, and all scale parameters are initialized to `0.017`. That constant is inherited from earlier noisy supervised-network experiments and is not something I derive here. For factorized noise, the mean parameters are initialized from `U[-1/sqrt(p), 1/sqrt(p)]`, and I set the scale initialization to `sigma_0 / sqrt(p)` with `sigma_0 = 0.5`. For the code artifact I also have to track the community Rainbow implementation: it initializes `weight_sigma` with `std_init / sqrt(in_features)` and `bias_sigma` with `std_init / sqrt(out_features)`. That bias constant is an implementation convention in the reference code, not a new derivation.

I then wire the sampled functions into the RL losses. For DQN, I remove `epsilon`-greedy and choose the greedy action under the current sampled value function. The replay loss uses one noise sample for the online network and an independent sample for the target network:
`[r + gamma max_b Q(y,b,epsilon'; zeta^-) - Q(x,a,epsilon; zeta)]^2`.
The independence matters because sharing the same random variables between online and target networks would correlate the bootstrapped target with the current estimate; the algorithm explicitly avoids that. For action collection, I draw another independent sample and act greedily with respect to it. The replay batch itself shares one current noise sample, so all examples in that optimization step see the same sampled network.

For dueling DQN, the streams and the mean-subtracted advantage combination stay the same. The target is the double-DQN form: an online sampled network selects `b*(y) = argmax_b Q(y,b,epsilon''; zeta)`, and the target sampled network evaluates `Q(y,b*(y),epsilon'; zeta^-)`. This is the same separation as ordinary double DQN, now with independent noise for online evaluation, target evaluation, and action selection.

For A3C, I remove the entropy bonus because the stochasticity now comes from sampling the network parameters. I make the fully connected policy and value heads noisy, using independent Gaussian noise in my main A3C variant. The on-policy detail is the critical one: if I resample inside a rollout, the actions and the bootstrap value are no longer produced by one fixed policy. So for the rollout return
`Qhat_i = sum_{j=i}^{k-1} gamma^{j-i} r_{t+j} + gamma^{k-i} V(x_{t+k}; zeta_V, epsilon_i)`,
I force every `epsilon_i` in the rollout to be the same `epsilon`. The policy-gradient and value-gradient updates are then taken with that same sampled network fixed throughout the rollout.

This gives me the final shape of the method. Replace only the fully connected heads with a layer whose means and noise scales are learnable; use independent Gaussian noise where random-number cost is acceptable and factorized Gaussian noise where it is not; delete the separate value-agent `epsilon` schedule and the actor-critic entropy bonus; keep online, target, and acting samples independent in value learning; and keep one sample fixed across an on-policy rollout. The result is not a Bayesian posterior sampler, and it is not a fixed-variance parameter-noise heuristic. It is ordinary deep-RL optimization with a reparameterized, learnable source of parameter noise.
