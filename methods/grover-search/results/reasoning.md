Let me strip this down to the barest version of the problem, because everything I know about search wants to assume some structure that isn't here. I have `N` items. Exactly one is special — there's a condition `C` I can test, and `C` is true on a single unknown state `S_ν` and false on every other one. Testing `C` on any given state costs one step. That's the whole problem: find `ν`. No sorting, no hash, no metric, nothing to navigate by. The phone book is in random order.

Classically I'm stuck and I can see exactly why I'm stuck. Each test inspects one item and tells me one bit: this one, or not this one. If it's "not," I've eliminated exactly one of the `N` possibilities and learned nothing about where the target is among the rest. So in the worst case I look at `N−1` of them, on average `N/2`, and randomizing the order doesn't help — a random probe still kills only one candidate, so I need on the order of `N` probes to have a decent chance. It's `Θ(N)` and it feels like a wall made of information theory: there are `N` independent yes/no facts and each query buys me one.

So the only hope is to change what a "query" can do, and that's what a quantum computer offers. Let me hold the model clearly in mind, because the whole game is going to be played inside it. The register holds not a state but a vector of `N` complex *amplitudes*, one per basis state; when I measure, I get state `i` with probability `|amplitude_i|²`. Evolution is by unitary matrices — and unitarity isn't a stylistic choice, it's forced: the probabilities `|amplitude_i|²` have to keep summing to one, and the linear maps that preserve that sum are exactly the unitaries. It's the same shape as a classical probabilistic algorithm — there I'd premultiply a probability vector by a stochastic matrix — except the entries are complex and can carry a sign or phase. That one difference is the entire resource I have that the classical world doesn't. Probabilities are non-negative and can only pile up; amplitudes can cancel. If I'm going to beat `Θ(N)`, the cancellation is where it has to come from. Wrong answers have to interfere destructively while the right one survives.

What can I actually build cheaply? The equal superposition is available. There's a one-bit operation `M = (1/√2)·[[1,1],[1,−1]]` — it takes `|0>` to `(1/√2, 1/√2)`. Run it on each of `n` bits and from the all-zeros start I get every one of the `N=2ⁿ` amplitudes equal to `1/√N`, in only `O(log N)` gates. That's the Walsh–Hadamard transform, `W_{ij}=2^{-n/2}(−1)^{i·j}`. Tempting — I've now "touched" all `N` states at once. But let me not fool myself: if I measure this uniform state, I get a *uniformly random* item. Probability of hitting the target: `1/N`. That's no better than one classical guess. So "examine all states in superposition" is a mirage. The breadth was never the hard part. The hard part is getting the amplitude to *pile up on the one state I want* before I look. Parallelism without concentration is nothing.

So I need to concentrate. What's the only tool that can even refer to the target? The oracle — the coherent test of `C`. I want it to do *something* to the target's amplitude and nothing to the others, and it has to be unitary. The most disruptive thing I can do to a single amplitude without touching any magnitude is flip its sign — rotate its phase by `π`, multiply it by `−1`. That's a selective phase rotation, a diagonal unitary, and it has the lovely property that `|amplitude|²` is unchanged everywhere, so it conserves probability automatically. It also has no classical-probabilistic shadow at all — you can't "negate a probability." Good. So let the oracle do exactly that: leave every amplitude alone except the target's, which it negates.

But now I've hit the obvious wall. I've flipped a sign, and a sign is invisible. All `N` magnitudes are still `1/√N`; measuring still gives me a random item with probability `1/N`. The oracle alone accomplishes nothing observable. I've tagged the target with a minus sign that I can't see. So the real question becomes sharp: I need a *second* operation that converts "exactly one amplitude has the wrong sign" into "exactly one amplitude is large." And it too must be unitary.

Let me look hard at the configuration after the oracle. Every amplitude is `+1/√N` except one, which is `−1/√N`. What operation singles out the odd one? The lone negative entry is the only one below the average. If I compute the mean of all the amplitudes, it's `((N−1)(+1/√N) + (−1/√N))/N`, essentially `+1/√N` for large `N` because one flipped term barely moves an average over `N`. So the target sits at `−1/√N` while the mean sits at about `+1/√N`. The target is roughly `2/√N` *below* the mean, and everyone else is sitting right *at* the mean.

That suggests the operation immediately: reflect every amplitude about the mean. Send `a_i → 2⟨a⟩ − a_i`. Anyone sitting at the average stays put. The one that's `2/√N` below the average lands `2/√N` above it — it goes from `−1/√N` to roughly `+3/√N`. So in one stroke the target's amplitude has tripled in magnitude and the others have barely budged. Let me name this "inversion about the mean" and check it's a legitimate, unitary operation rather than wishful thinking. Write it as a matrix `D`. "Reflect about the mean" means `(Dv)_i = 2⟨v⟩ − v_i = (1/N)·2·Σ_j v_j − v_i`. So `D_{ij} = 2/N − δ_{ij}`: off-diagonal entries `2/N`, diagonal entries `−1 + 2/N`. Cleanly, `D = −I + 2P` where `P_{ij}=1/N` is the projector onto the uniform vector (`P² = P` because averaging twice is averaging once). Is `D` unitary? `D² = (−I+2P)² = I − 4P + 4P² = I − 4P + 4P = I`. And `D` is real symmetric, so `D†=D`, hence `D†D = D² = I`. Yes — unitary. And `(Dv)_i = ⟨v⟩ + (⟨v⟩ − v_i)` literally says each component ends up as far above the mean as it started below it. That's exactly the reflection I wanted, and it's a valid quantum operation.

One worry: is `D` something I can actually *implement* cheaply, or is it a dense `N×N` matrix I have no right to apply? It connects every state to every other state, so naively it's non-local. But watch — `R := −I + 2|0…0><0…0|` is a reflection about the all-zeros basis state: it keeps `|0…0>` fixed and flips the phase of the orthogonal basis states, equivalently a selective flip of `|0…0>` up to a global sign. Conjugating that reflection by the Walsh–Hadamard transform moves the mirror from the all-zeros state to the uniform state: `W` maps `|0…0>` to the uniform vector. Since `W²=I`, `W(−I)W = −I`, and `W(2|0><0|)W = 2(W|0…0>)(W|0…0>)† = 2P`. So `WRW = −I + 2P = D`. The inversion about the mean is `W`, then this one-state reflection, then `W` again — all cheap in the same sense as the primitives I already have.

Now let me actually iterate the pair — oracle then `D` — and see how fast the target grows. After preparing the uniform state, the oracle makes the target `−1/√N` and the rest `+1/√N`; `D` reflects about the mean and the target jumps by about `2/√N`. Do it again: oracle flips the (now larger) target negative, `D` reflects, and it climbs by roughly another `2/√N`. Each full iteration adds something like `2/√N` to the target amplitude, as long as that amplitude is still small. To drive the target amplitude from `1/√N` up to order `1`, I need on the order of `√N` iterations. There it is — `O(√N)` oracle calls instead of `O(N)`. The interference I was promised is real: the wrong answers keep getting reflected back near the mean and cancel into staying small, while the right answer's flipped sign lets it ratchet upward each round.

I trust the count, but "adds about `2/√N` each time" is shaky near the end, because once the target amplitude is no longer tiny the mean shifts and the increments change. I want an exact picture so I can both nail the constant and see where the approximation breaks. Let me find the right coordinates.

I never leave a two-dimensional plane. Group the states into "the marked one" and "everything else." Let `|w>` be the marked state and `|s'>` be the normalized uniform superposition over the `N−1` unmarked states, `|s'> = (1/√(N−1)) Σ_{x≠w} |x>`. These are orthogonal unit vectors. The starting uniform state `|s>` is a combination of just these two: it has amplitude `1/√N` on `|w>` and the rest spread over `|s'>`, so `|s> = (1/√N)|w> + (√(N−1)/√N)|s'>`. Define an angle `θ` by `sin θ = ⟨w|s> = 1/√N`; then `cos θ = √(N−1)/√N`, and `|s> = sin θ |w> + cos θ |s'>`. So the initial state sits at a small angle `θ` up from the `|s'>` axis, where `sin θ = 1/√N`.

Now what are my two operations, geometrically, in this plane? The oracle negates the `|w>` component and leaves everything orthogonal to `|w>` alone — that's `U_w = I − 2|w><w|`, and a map that flips the component along one axis and keeps the perpendicular one is a *reflection*, here a reflection across the `|s'>` axis. What about `D`? In this two-dimensional picture, inversion about the mean is reflection about the uniform direction `|s>`: it's exactly `U_s = 2|s><s| − I = −I + 2P`, which flips the sign of everything perpendicular to `|s>` and keeps the `|s>` component — a reflection across the `|s>` line.

So one full iteration is: reflect across `|s'>`, then reflect across `|s>`. Two reflections. And I know what the composition of two reflections is — it's a rotation, by twice the angle between the two mirror lines. The angle between the `|s'>` axis and the `|s>` line is exactly `θ`. So one iteration is a rotation by `2θ`, and it rotates the state from `|s'>` toward `|w>` — i.e. it tilts the state up, increasing the `|w>` component, exactly the concentration I wanted. Let me confirm the `2θ` isn't hand-waving. In the `(|s'>, |w>)` basis, `U_w = diag(1, −1)` (keep `s'`, flip `w`). And `U_s = 2|s><s| − I` with `|s> = (cos θ, sin θ)` works out to `[[cos 2θ, sin 2θ],[sin 2θ, −cos 2θ]]`. Multiply: `U_s U_w = [[cos 2θ, −sin 2θ],[sin 2θ, cos 2θ]]`. That's precisely the rotation matrix by angle `2θ`, exactly, with no approximation.

That changes everything about reasoning on the endgame, because rotation is rigid and periodic. The initial state is at angle `θ` from `|s'>`. After `r` iterations it's at angle `θ + 2rθ = (2r+1)θ`. The amplitude on `|w>` is the sine of that angle: `sin((2r+1)θ)`, and the success probability when I measure is `sin²((2r+1)θ)`. For small `r` this matches my earlier increment estimate — `sin((2r+1)θ) ≈ (2r+1)θ ≈ (2r+1)/√N` grows by about `2/√N` per step — but now I have the exact law all the way to the top.

When do I stop? I want the `|w>` amplitude as large as possible, i.e. `(2r+1)θ` as close to `π/2` as I can get. Setting `(2r+1)θ = π/2` gives `r = (π − 2θ)/(4θ)`, which isn't an integer in general, so I take `r = ⌊π/(4θ)⌋`. For large `N`, `θ` is tiny and `θ ≈ sin θ = 1/√N`, so `r ≈ π/(4θ) ≈ (π/4)√N`. The constant falls out clean: about `(π/4)√N` iterations. And the success probability at that stopping point — since rounding moves `(2r+1)θ` off `π/2` by at most `θ` — has `cos²((2r+1)θ) ≤ sin²θ = 1/N`, so the *failure* probability is at most `1/N` and I find the marked item with probability at least `1 − 1/N`. (Even the crude bound is fine: there's an integer `M < √(2N)` after which the amplitude exceeds `1/√2`, giving success probability over a half; the geometry just tightens that to `(π/4)√N` with near-certainty.)

The increment picture has hidden the periodicity. The dynamics is a *rotation*, so running it longer is not better. If I keep going past `(2r+1)θ = π/2`, the state rotates *past* `|w>` and starts heading back down toward `|s'>`. The success probability `sin²((2r+1)θ)` is a sinusoid in `r`: it climbs to near `1` at the optimum and then *falls*. Let me feel this with a concrete case. Take `N=16`, so `sin θ = 1/4`, `θ ≈ 0.2527`, and `(π/4)√16 = π ≈ 3.14`, so `r=3`. At `r=3` the amplitude on `|w>` is `sin(7θ) ≈ 0.980`, probability `0.961` — almost certain. But push on: `r=4` gives `sin(9θ) ≈ 0.763`, probability `0.58`; `r=5` gives probability `0.13`; `r=6` drops to `0.02`. I've over-rotated and walked the amplitude right back off the target. So the *precise* number of iterations genuinely matters — this is not an algorithm where you hedge by running extra rounds. You aim for `⌊π/(4θ)⌋` and stop.

Now the question that's been nagging me: is `√N` the best possible, or did I just find *a* quadratic speedup and a cleverer scheme could do `log N`? The rotation gives me a fixed step size of `2θ ≈ 2/√N` per query, and I can't take a bigger step without knowing more about where the target is — but "it feels forced" isn't a proof. Let me see whether any quantum algorithm, however exotic, can avoid `Ω(√N)` queries on this black-box problem.

I should ask how much the algorithm's final state can possibly *depend* on which item is the marked one, given a limited number of queries. Run the algorithm for `T` query-steps against the trivial oracle where *nothing* is marked — `C(S)=0` everywhere. At each step `i` the register is in some superposition `|φ_i>`. For each candidate string `y`, define the "query magnitude" `q_y(|φ_i>)` as the total squared amplitude sitting on configurations that are querying the oracle about `y` at step `i`. Since `|φ_i>` is a unit vector, summing the query magnitude over all `y` at a fixed step gives at most `1`: `Σ_y q_y(|φ_i>) ≤ 1`. Sum that over the `T` steps: `Σ_y Σ_{i<T} q_y(|φ_i>) ≤ T`. There are `N` candidate strings, so by averaging there must be a string — in fact most strings — whose *total* query magnitude across the whole run is at most `T/N`. That string is one the algorithm barely ever asks about.

Now plant the target at exactly such a lightly-queried `y`: switch to the oracle that marks `y` and nothing else. How different can the final state become? Changing the oracle's answer only matters at the steps where amplitude was actually querying `y`. At step `i`, the altered oracle can move only the slice of norm `√q_y(|φ_i>)`, so the error vector introduced at that step has norm at most a constant times `√q_y(|φ_i>)`. The later unitary operations do not change that norm. In the worst case all these error vectors line up, so I have to sum their norms, not just their squared norms:
`Σ_{i<T} √q_y(|φ_i>) ≤ √(T Σ_{i<T} q_y(|φ_i>)) ≤ T/√N`.
Up to the harmless constant from the phase flip, the final state with `y` marked is therefore only `O(T/√N)` away from the all-zero-oracle final state. Equivalently, in the BBBV hybrid theorem, a target whose total query magnitude is `Q` gives perturbation `O(√(TQ))`; with `Q ≤ T/N`, that is `O(T/√N)`. But to reliably *report* that `y` is the marked item rather than behaving as if nothing is marked, the output distributions have to differ by a constant, so the final states must be a constant Euclidean distance apart. That forces `T/√N = Ω(1)`, hence `T = Ω(√N)`. No quantum algorithm can identify the marked item in fewer than `Ω(√N)` queries.

So `√N` is not an artifact of my particular two-reflection scheme; it's the floor for unstructured quantum search. And my algorithm hits it: `(π/4)√N ≈ 0.785√N` queries against a `Ω(√N)` lower bound — optimal up to the constant. That also explains, cleanly, why I never had a shot at `log N` here: `log N` would require structure to lever, the way period-finding levers periodicity for factoring. Strip the structure out and the rotation step can't be larger than `~1/√N` per query, provably.

The finished procedure is short. Prepare the uniform superposition with Walsh–Hadamard on the all-zeros register — equal amplitude `1/√N` everywhere. Then repeat `⌊π/(4θ)⌋ ≈ (π/4)√N` times the pair: apply the oracle, which phase-flips the marked state's amplitude (a selective `π` phase rotation, unitary, leaves all magnitudes untouched); then apply `D = 2|s><s| − I`, inversion about the mean, implemented locally as `W R W` with `R = −I + 2|0…0><0…0|`. Geometrically each pair is a rotation by `2θ` (with `sin θ = 1/√N`) in the plane spanned by the marked state and the unmarked-uniform state, carrying the state from angle `θ` up toward `|w>`; after `r` rounds the marked amplitude is `sin((2r+1)θ)` and the success probability `sin²((2r+1)θ)`. Stop at the round nearest `(2r+1)θ = π/2` — going further over-rotates and the probability falls back. Measure; the marked item appears with probability at least `1 − 1/N`. Total cost `O(√N)` oracle queries, and the matching `Ω(√N)` lower bound says you cannot do better for unstructured search.

The causal chain in one breath: classical search is `Θ(N)` because each query kills one candidate and there's no structure to do better; the quantum register lets amplitudes carry signs that can cancel, so I tag the target by flipping its phase (invisible alone) and then reflect all amplitudes about their mean (which lifts the lone flipped one and barely moves the rest); seen in the two-dimensional marked/unmarked plane those two reflections compose to a fixed rotation by `2θ` with `sin θ = 1/√N`, so `≈(π/4)√N` rotations bring the state onto the target — no more, because over-rotating swings it back — and a hybrid argument bounding how much a few queries can change the final state proves `Ω(√N)` is unavoidable, so the algorithm is optimal up to the constant.

```python
import numpy as np

def grover_search(N, marked, iterations=None):
    """Find `marked` among N items with O(sqrt N) oracle queries."""
    theta = np.arcsin(1.0 / np.sqrt(N))
    n_iter = iterations if iterations is not None else int(np.floor(np.pi / (4.0 * theta)))

    # Uniform superposition: Walsh-Hadamard on |0...0>, amplitude 1/sqrt(N) everywhere.
    state = np.full(N, 1.0 / np.sqrt(N))

    for _ in range(n_iter):
        # Oracle: selective pi-phase rotation of the marked amplitude (sign flip).
        # Unitary; leaves every |amplitude|^2 unchanged -- tags the target invisibly.
        state[marked] *= -1.0

        # Diffusion D = 2|s><s| - I, "inversion about the mean".
        # Reflects each amplitude about the average; the lone flipped one is lifted,
        # the rest stay put. (Implemented physically as W . R . W with R=-I+2|0><0|.)
        mean = state.mean()
        state = 2.0 * mean - state

    # One iteration = rotation by 2*theta, sin(theta)=1/sqrt(N), in span{|w>,|s'>}.
    # After r iters the marked amplitude is sin((2r+1)theta); stop near (2r+1)theta = pi/2.
    probs = np.abs(state) ** 2          # measurement: outcome i w.p. |state[i]|^2
    return probs, n_iter
```
