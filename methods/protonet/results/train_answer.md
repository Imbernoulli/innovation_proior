We are handed a few-shot classification task that punishes complexity directly: an episode names $N$ classes never seen during training, gives $K$ labeled support examples per class — sometimes just one — and demands immediate labels for fresh query images of those same classes, with no retraining. Fine-tuning a deep network on twenty-five images is hopeless; millions of weights against a couple dozen points memorizes the support set and tells us nothing about the queries. Yet a person recognizes a new character from a single glance. The lesson the data scarcity screams is that the answer cannot be "fit more parameters per task" — it has to be an inductive bias strong and simple enough that a handful of points pins it down. Every extra knob is one more thing five images per class cannot constrain, so the right design is biased toward the simplest hypothesis that still separates the classes.

Two ideas from the prior art survive that test and two do not. Episodic training — sampling mini-tasks during training that look exactly like test tasks, $N$ classes, $K$ shots, a query set, and directly optimizing query-set accuracy conditioned on the support set — is clearly correct, because the failure mode of ordinary training is a mismatch between the balanced-minibatch objective optimized and the tiny $N$-way $K$-shot objective judged, and matching the two is itself a powerful regularizer in a data-starved regime. That we keep wholesale. What needs rethinking is the classifier *inside* the episode. Matching Networks embeds every support and query image and classifies a query $\hat x$ by a weighted vote over support labels, $\hat y = \sum_i a(\hat x, x_i)\, y_i$ with $a(\hat x, x_i) = \exp(c(f(\hat x), g(x_i))) / \sum_j \exp(c(f(\hat x), g(x_j)))$, a softmax over cosine similarity. Structurally that is a differentiable weighted nearest-neighbor classifier in embedding space — nonparametric, so nothing to overfit, which is good — but it keeps *every individual support point* and assigns each its own attention weight, so the state it carries to make a prediction grows with the support-set size and there is no notion of a class as a single thing. Its strong variant, full context embeddings, then runs a bidirectional LSTM over the support set, adding parameters twenty-five images cannot fit and imposing an arbitrary sequential ordering on an unordered set. The bias points the wrong way, toward more complexity exactly where the data begs for less.

I propose Prototypical Networks. The move is to ask for the *minimal* summary of a class one could compare a query against: independent of whether a class has one example or fifty, the most natural single object for a cloud of vectors is its center. So summarize each class by the mean of its embedded support points,
$$c_k = \frac{1}{|S_k|} \sum_{(x_i, y_i) \in S_k} f_\phi(x_i),$$
call it the prototype, and classify a query by which prototype it is closest to. This fixes both complaints at once: per-class state is one vector regardless of shot, and a class becomes a concrete entity rather than a bag of points. The nearest-class-mean idea is not new, but prior work used a *linear* embedding with a learned Mahalanobis metric and, when it wanted nonlinearity, bolted on a separate $k$-means partitioning step in input space, decoupled from the metric. Here $f_\phi$ is a deep nonlinear network, the mean is computed *in the learned space*, and the whole thing is trained end-to-end by the episodic objective so $f_\phi$ learns to make the means separable.

To make "nearest prototype" differentiable, a hard argmin has no gradient, so define a distribution over classes for a query $x$ by a softmax over negative distances to prototypes,
$$p_\phi(y = k \mid x) = \frac{\exp(-d(f_\phi(x), c_k))}{\sum_{k'} \exp(-d(f_\phi(x), c_{k'}))},$$
and minimize the negative log-probability of the true class, $J(\phi) = -\log p_\phi(y = k \mid x)$, averaged over the query points of an episode. The numerator's exponent uses the correct class $c_k$ while the normalizer sums $\exp(-d(\cdot, c_{k'}))$ over every $k'$ including $k$; gradients flow through the distances, through the prototypes, into $f_\phi$. It is just cross-entropy on logits that happen to be negative distances.

The choice of $d$ is where the design earns its keep, and it is not free. Having committed the prototype to be the *mean*, the right question is: for which distances is the mean actually the best representative of a class, the point minimizing total distance to the class's points? That is $\arg\min_c \sum_{x \in S_k} d(f(x), c)$, and it has a precise answer. For the entire family of Bregman divergences, $d_\phi(z, z') = \phi(z) - \phi(z') - (z - z')^\top \nabla\phi(z')$ for strictly convex $\phi$, the minimizer of the summed divergence to a set is exactly its arithmetic mean. The derivation is direct: differentiate $g(c) = \sum_x d_\phi(x, c)$; the $\phi(x)$ term is constant in $c$, and for $h(c) = (x - c)^\top \nabla\phi(c)$ the product rule gives $\nabla h(c) = -\nabla\phi(c) + \nabla^2\phi(c)(x - c)$, so $\nabla_c d_\phi(x, c) = \nabla^2\phi(c)(c - x)$. Summing, $\nabla g(c) = \nabla^2\phi(c)\,(|S_k|\, c - \sum_x x)$, and with $\nabla^2\phi$ positive definite the zero is $c = (1/|S_k|)\sum_x x$, the mean. Conversely, among smooth distortions this mean-as-minimizer property *characterizes* the Bregman divergences, so the distance had better be one or else we are averaging points the distance does not think the average represents. Cosine distance, $1 - \langle z, z'\rangle / (\|z\|\|z'\|)$, is not Bregman — it keeps only angle, and its best representative is a normalized direction, not the mean — so averaging embeddings and then scoring by cosine is internally inconsistent. Squared Euclidean, $d(z, z') = \|z - z'\|^2$, *is* Bregman: take $\phi(z) = \|z\|^2$, then $\phi(z) - \phi(z') - (z - z')^\top(2z') = \|z\|^2 - 2z^\top z' + \|z'\|^2 = \|z - z'\|^2$. So $\phi = \|\cdot\|^2$ generates exactly squared Euclidean, the mean is its optimal representative, and we use it — with the concrete prediction that the Euclidean-over-cosine gain should be *largest* for this mean-prototype method, since the averaging step is precisely where cosine is inconsistent.

There is a deeper reading of "squared Euclidean is Bregman" that names the modeling assumption being made. The bijection between regular exponential-family densities and regular Bregman divergences lets any $p_\psi(z \mid \theta) = \exp\{z^\top\theta - \psi(\theta) - g_\psi(z)\}$ be written as $\exp\{-d_\phi(z, \mu(\theta)) - g_\phi(z)\}$. Modeling each class in embedding space as one component of an equally-weighted mixture with component mean $c_k$, the posterior over which component generated $z$ is $\pi_k \exp(-d_\phi(z, c_k)) / \sum_{k'} \pi_{k'}\exp(-d_\phi(z, c_{k'}))$, and with equal $\pi_k = 1/K$ those cancel, leaving exactly the classification softmax. So the classifier *is* posterior inference in an exponential-family mixture with one component per class, and squared Euclidean corresponds to a spherical Gaussian family with common fixed scale — every class a round blob of equal size around its mean. That is a strong, simple assumption, and in this regime "strong and simple" is exactly the bias we want.

A worry: squared Euclidean is *quadratic* in the embedding — is this a weak classifier? Expand the exponent, $-\|f_\phi(x) - c_k\|^2 = -f_\phi(x)^\top f_\phi(x) + 2 c_k^\top f_\phi(x) - c_k^\top c_k$. The first term is identical for every class $k$, so it cancels top and bottom in the softmax, and what survives is $2 c_k^\top f_\phi(x) - c_k^\top c_k$, which is *linear* in $f_\phi(x)$: write it as $w_k^\top f_\phi(x) + b_k$ with $w_k = 2 c_k$ and $b_k = -c_k^\top c_k$. The prototypical classifier is, after the constant drops, a plain linear classifier on the embedding whose weights and biases are read directly off the prototypes — no extra parameters, a linear head *induced* by the means. This is the right amount of expressiveness: the head has nothing to overfit because it is determined by five-image means, and all nonlinearity lives in $f_\phi$, trained across thousands of episodes where it can be constrained. We put the capacity where the data can pay for it.

A sanity check ties this back to its ancestor. In the one-shot case $|S_k| = 1$, the prototype is the single support embedding, so the classifier becomes a soft nearest-neighbor over support points — exactly what Matching Networks reduces to in one-shot. With the same embedding and distance the two methods coincide at $K = 1$; they diverge only at $K > 1$, where we collapse each class to its mean and vote over classes instead of over individual points, which is the whole point. Allowing several prototypes per class is tempting for multimodal classes, but it would require a partitioning of each class's support — $k$-means or its kin — as a *separate* step decoupled from the gradient updates of $f_\phi$, breaking end-to-end training. One prototype per class keeps the method differentiable in one loop, and the deep embedding can warp space so a class is unimodal around its mean. The same logic kills FCE, and the same redundancy argument predicts a per-dimension variance buys nothing: $f_\phi$ is already free to rescale and rotate axes, so it absorbs per-dimension variance itself, and the spherical assumption stays.

Two episode-composition choices follow from reasoning rather than habit. The training *way* need not match the test way; training on $20$- or $30$-way episodes forces $f_\phi$ to keep many more prototypes simultaneously separable, a strictly harder constraint that yields a finer embedding which comfortably separates the few classes at test, so we train with higher way and tune it upward on validation. The *shot* is different: a one-shot prototype is a raw embedding while a five-shot prototype is an average of five, living at different noise levels, so we match the training shot to the test shot to keep prototype noise realistic. Per episode, then, sample $N_C$ classes, split each into support $S_k$ of $N_S$ points and query $Q_k$ of $N_Q$ points, form $c_k = (1/N_S)\sum_{(x_i,y_i)\in S_k} f_\phi(x_i)$ — the normalizer is the number of *support* points, not classes — and for each query $(x, y)$ with true class $k$ accumulate $(1/(N_C N_Q))\,[\,\|f_\phi(x) - c_k\|^2 + \log\sum_{k'}\exp(-\|f_\phi(x) - c_{k'}\|^2)\,]$, which is exactly $-\log p_\phi(y = k \mid x)$; average, backprop, step. The same prototype primitive extends to zero-shot for free: when given class meta-data $v_k$ rather than support images, produce the prototype by a separate learned map $c_k = g_\vartheta(v_k)$ constrained to unit length, leaving the query embedding free, and everything downstream is unchanged.

One implementation detail must stay explicit: the score is the negative *squared* Euclidean distance, not the negative Euclidean norm. Both give the same nearest-prototype argmax for a fixed embedding, but the softmax probabilities, the gradients, and the Bregman / spherical-Gaussian / linear-head equivalences all rely on the square, so the implementation keeps it.

```python
import torch
import torch.nn.functional as F
from torch import Tensor


class PrototypicalNetworks(FewShotClassifier):
    """Each class is summarized by the mean of its embedded support examples (its prototype);
    a query is classified by a softmax over its negative squared Euclidean distance to the
    prototypes. Trained episodically with cross-entropy on the negative squared-distance logits."""

    def __init__(self):
        backbone = make_backbone(use_pooling=True)        # nonlinear embedding f_phi
        super().__init__(backbone=backbone, use_softmax=False)

    def process_support_set(self, support_images: Tensor, support_labels: Tensor):
        support_features = self.compute_features(support_images)
        self._raise_error_if_features_are_multi_dimensional(support_features)
        # Episode labels are local 0..n_way-1, as in the canonical episodic loss.
        n_way = int(torch.unique(support_labels).numel())
        self.prototypes = torch.stack([
            support_features[support_labels == label].mean(dim=0)
            for label in range(n_way)
        ])

    def forward(self, query_images: Tensor) -> Tensor:
        query_features = self.compute_features(query_images)        # f_phi(query)
        self._raise_error_if_features_are_multi_dimensional(query_features)
        sq_dist = ((query_features[:, None, :] - self.prototypes[None, :, :]) ** 2).sum(dim=2)
        scores = -sq_dist                                           # logits = -squared distance
        return self.softmax_if_specified(scores)

    @staticmethod
    def is_transductive() -> bool:
        return False                                                # queries classified independently

    def compute_loss(self, scores: Tensor, labels: Tensor) -> Tensor:
        return F.cross_entropy(scores, labels)                      # -log p_phi(y=k|x)
```
