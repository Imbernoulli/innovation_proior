I start with the wrong object in front of me: the frequencies themselves. If I know every `m_i`, then `F_2 = sum_i m_i^2` is immediate, but keeping the whole vector is exactly the memory cost I am trying to escape. The stream is asking me to remember a quadratic fact about a vector without remembering the vector. So I stop asking how to store many counters and ask a different question: what random quantity can I update as each item arrives, and whose distribution at the end still knows the moment?

The first path is to sample an update position rather than a value name. If I sample a universe value uniformly, I almost surely miss the heavy values that dominate high moments. But a uniformly sampled stream position chooses value `i` with probability `m_i / m`, which is the bias I want. Suppose the sampled position holds value `i`, and from that position onward I count `r`, the number of remaining occurrences of `i` including the sampled one. If this sampled occurrence is the `c`th one from the end among the `m_i` occurrences, then `c` is uniform in `{1, ..., m_i}`. The expression `c^k - (c-1)^k` is exactly the marginal increase in the `k`th power when the rank grows from `c-1` to `c`. So if I output `m(r^k - (r-1)^k)`, then conditioning on value `i` gives an average proportional to `m_i^k`, and after weighting by `m_i / m`, the pieces telescope into `sum_i m_i^k`.

That is a real opening: I only keep the sampled value and its suffix count. Reservoir sampling even removes the need to know `m` in advance. But when I check the variance, the price is still too high for the squared moment. The second moment receives only the general `n^{1/2}`-type saving from this estimator, not the tiny logarithmic space I am looking for. Position sampling shows that moments can be estimated without a histogram, but it is not yet the special explanation for why `F_2` collapses so dramatically.

For `F_2`, I try to exploit its geometry. It is the squared Euclidean length of the frequency vector. A one-pass stream update can maintain linear functions of that vector very naturally: choose coefficients `c_i`, keep `Z = sum_i c_i m_i`, and when item `i` arrives add `c_i` to `Z`. A deterministic choice such as `c_i = 1` only gives `F_1`; a single linear functional cannot equal a quadratic norm. But I do not need `Z` itself to be the answer. I can square it at the end.

Now the algebra shows the possibility. Let the coefficients be random signs, `epsilon_i in {-1, +1}`, and maintain `Z = sum_i epsilon_i m_i`. Then

`Z^2 = sum_i m_i^2 + sum_{i != j} epsilon_i epsilon_j m_i m_j`.

The diagonal is exactly the desired squared moment, because every sign squares to one. The off-diagonal terms are the only obstruction. If the signs are unbiased and pairwise independent, then `E[epsilon_i epsilon_j] = 0` for `i != j`, so every cross term disappears in expectation. I have an unbiased estimator: `E[Z^2] = F_2`. This is the central change of viewpoint. I am no longer counting collisions explicitly; I am letting random cancellation erase the interactions between different values while leaving self-interactions intact.

Unbiasedness is not enough. A one-line expectation proof could hide an estimator with hopeless variance. I need to inspect `E[Z^4]`, because `Var(Z^2) = E[Z^4] - F_2^2`. Expanding `Z^4` creates products `epsilon_i epsilon_j epsilon_k epsilon_l`. Any term where some sign appears an odd number of times should vanish, but that argument only works if I can separate that sign from the other three. Pairwise independence is no longer enough; the proof touches four coordinates at a time. This is where four-wise independence enters, not as ornament and not as full randomness, but as the exact amount of independence needed by the fourth-moment calculation.

With four-wise-independent unbiased signs, only two kinds of terms survive in `E[Z^4]`: all four indices equal, contributing `F_4`, and two equal pairs, contributing `6 sum_{i<j} m_i^2 m_j^2`. If I write `S = sum_{i<j} m_i^2 m_j^2`, then `E[Z^4] = F_4 + 6S`, while `F_2^2 = F_4 + 2S`. Therefore `Var(Z^2) = 4S = 2(F_2^2 - F_4) <= 2F_2^2`. The estimator has constant relative variance. It is noisy, but no longer wild.

This also explains why the randomness has to be carefully engineered. Full independence over all `n` items would store too much random information, almost as bad as storing the histogram. Four-wise independence is enough because the analysis never asks about five signs at once. Small four-wise-independent families from BCH-code orthogonal arrays, or equivalently finite-field polynomial constructions, let me describe the sign function with only logarithmic seed space and evaluate the sign of the current item during the pass.

A single copy still has relative standard deviation on the order of one, so I cannot stop there. If I average `s_1` independent copies of `Z^2`, the expectation stays `F_2` and the variance falls by `s_1`. Chebyshev then says that `s_1 = Theta(lambda^{-2})` gives a constant success probability for relative error `lambda`. But pushing the failure probability to `delta` by averaging alone would cost `1/delta`, which is the wrong dependence. I instead make several independent averages and take their median. Once each average is good with probability above one half, a Chernoff bound says the median fails only when too many independent groups fail, so `Theta(log(1/delta))` groups are enough.

This is why saying "use Chernoff bounds" misses the discovery. Chernoff only amplifies after the estimator has been built. The hard part is constructing a tiny stream-maintainable random variable whose diagonal terms are the frequency moment and whose off-diagonal terms cancel, then proving by a fourth-moment argument that its variance is controlled. The concentration step is the reliability wrapper, not the source of the space saving.

If I try to generalize too soon, I can hide the actual reason this case works. A norm-estimation language suggests random linear observations, but the squared moment does not need a heavier distributional machine. The finite sign construction already gives the diagonal terms for free, cancels the cross terms in expectation, and has a fourth-moment proof that its variance is controlled.

So the path is now coherent. Sampling update positions gives a general telescoping estimator for `F_k`; signed linear summaries give the exceptional logarithmic-space estimator for `F_2`. The probabilistic move is to stop representing frequencies one coordinate at a time and instead represent the frequency vector through a random observation designed so that the desired moment is the squared length of that observation. Once that viewpoint is in place, small space is no longer paradoxical: the stream summary is tiny because it never tries to remember which values collided; it arranges for different values to cancel and equal values to reinforce.
