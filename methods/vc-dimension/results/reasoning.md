I start with the law of large numbers, and for a moment it looks like I already have what I need. For a fixed event A, its empirical frequency nu_A approaches P(A), and Chebyshev even gives me a distribution-free bound of order 1/(l epsilon^2). But I immediately run into the fact that learning does not keep A fixed. I look at the sample, compare rules, and choose one because it behaves well on that sample. The chosen rule is a function of the sample, so the fixed-event bound no longer addresses the actual source of error.

The first repair is clear in the finite case. If I have only N rules, I can pay for selection by a union bound. A bad rule survives the sample with probability at most (1-kappa)^l, so the chance that some bad rule survives is at most N(1-kappa)^l. The price of searching the class is only log N in the sample size. This is exactly the right shape for finite memories and finite lists of rules, but it fails at the point where the real pattern-recognition classes begin. A family of halfspaces has infinitely many parameter values. Counting rules by cardinality says infinity and stops.

I should not count parameter values. On a fixed sample, two different rules that label every sample point in the same way are indistinguishable for empirical frequency. A continuum of halfspaces collapses to a finite set of binary patterns on x_1,...,x_l. So I define the effective number of rules on this sample as the number of distinct subsamples induced by the class. Call it Delta^S(x_1,...,x_l), and then take the worst case over samples of size l, m^S(l). This is the right replacement for N if I can get it into a probability bound.

There is still a wall. The deviation I need to control is sup_A |nu_A - P(A)|, and P(A) is not observed. I cannot simply condition on a sample and union-bound over its induced labelings, because the true probabilities still refer to the distribution outside the sample. I introduce a second independent sample of the same size. Now I compare the first half's frequency with the second half's frequency and write rho_l = sup_A |nu'_A - nu''_A|. This removes P(A) from the expression.

The second sample is useful because if the first half is truly bad for some event A_0, meaning |nu'_{A_0}-P(A_0)| > epsilon, then the second half is likely to be within epsilon/2 of P(A_0) for that fixed A_0. Chebyshev gives that probability at least one half once l >= 2/epsilon^2. On that event the two half-sample frequencies differ by more than epsilon/2. So P(pi_l > epsilon) is at most twice P(rho_l >= epsilon/2). I have traded the unknown probability for a two-sample empirical discrepancy.

Now I condition on the combined 2l points. The distribution is invariant under permutations of those points between the two halves. For the fixed combined sample, the class S induces only Delta^S(x_1,...,x_{2l}) different binary patterns. I can keep one representative event for each pattern and union-bound over those representatives. For one fixed representative, the number of its marked points landing in the first half under a random split has a hypergeometric distribution. The chance that the two half frequencies differ by at least epsilon/2 is bounded by 2 exp(-epsilon^2 l/8). Therefore the whole class contributes at most 2 m^S(2l) exp(-epsilon^2 l/8), and the ghost-sample step doubles this:

P(pi_l > epsilon) <= 4 m^S(2l) exp(-epsilon^2 l/8), for l >= 2/epsilon^2.

This is the decisive reduction. The statistical question is now a combinatorial question about the growth of m^S(l). If m^S(l) grows polynomially, the exponential tail wins and the empirical frequencies converge to probabilities simultaneously over the class. If m^S(l) stays exponential, the bound gives no convergence and the all-subsets example shows that convergence can genuinely fail.

So I need to understand which classes have polynomial growth. The halfspace counting tradition suggests that polynomial behavior is not an accident: Cover's function-counting theorem says that linear threshold rules induce only a binomial-sum number of dichotomies on points in general position, with a recurrence that is polynomial in the sample size once dimension is fixed. But I need a class-independent reason for this behavior.

I look for the obstruction to polynomial growth. The maximum possible value of Delta^S on l points is 2^l. That means the class can realize every subset of those l points. In other words, on that finite set the class has no restrictions at all. If a class can do this for arbitrarily large finite sets, it is too rich. If it stops being able to do this beyond some size, perhaps that stopping size forces polynomial growth.

The key lemma makes this exact. Define Phi(n,r) by the Pascal recurrence Phi(n,r)=Phi(n,r-1)+Phi(n-1,r-1), with Phi(0,r)=1 and Phi(n,0)=1; equivalently it is the binomial-prefix count, so Phi(n,r)=2^r while r<=n and is polynomial in r once n is fixed. If a class induces at least Phi(n,r) subsamples on some r-point sample, then inside that sample there is an n-point subset on which every one of the 2^n labelings is realized. I prove this by removing the last point and splitting the old labelings into two types. Some old labelings have only one extension when the last point is restored; others have both extensions. If the both-extension family could realize every labeling on n-1 old points, then adding the last point would give every labeling on n points. If it cannot, induction bounds the both-extension count by Phi(n-1,r-1), while induction also bounds the total old count by Phi(n,r-1). Adding the two counts gives Phi(n,r), contradiction.

The contrapositive gives the dichotomy. Let n be the first sample size at which full realization of all 2^n labelings fails. Then no larger sample can have index reaching Phi(n,r), because that would contain an n-point fully realized subset. Hence m^S(r) is bounded by Phi(n,r), and the recurrence gives Phi(n,r)<r^n+1. If no such n exists, then m^S(r)=2^r for every r. There is no middle behavior such as mildly exponential growth.

This is where the true complexity measure appears. What matters is the largest finite set on which the class can behave arbitrarily. If I call that largest shattered size d, then the exponent in this indexing is the first failed size n=d+1. A sharper binomial-prefix count would use d directly and give m^S(l) <= sum_{k=0}^d binom(l,k), but the learning insight is already fixed: finite largest shattered set means polynomial growth, and polynomial growth means distribution-free simultaneous convergence.

This also explains why parameter count is only a proxy and sometimes a misleading one. A d-dimensional affine halfspace family has a finite largest shattered set and therefore a polynomial growth function; the same raw continuum cardinality as a much wilder family tells me nothing. A rule that fits one dataset perfectly also tells me little by itself, because the question is whether the class can fit every labeling on large finite sets. The class is learnable in this sense when its finite-sample freedom is capped.

The examples now line up. Rays on the real line induce only l+1 labelings on l ordered points, so the general theorem recovers the empirical distribution result. Fixed-threshold halfspaces inherit the polynomial cell count from hyperplane arrangements. Affine halfspaces in R^d shatter d+1 suitable points but cannot shatter d+2, by the convex-hull obstruction behind Radon's lemma. The class of all open subsets of an interval realizes every labeling of every finite sample and therefore has infinite capacity; uniform empirical reliability fails.

There is one more refinement, now distribution-dependent. Worst-case growth gives a clean distribution-free sufficient condition, but a fixed distribution may not sample the worst configurations. So I look at H^S(l)=E log_2 Delta^S(X_l), the expected log-index. Subadditivity gives a limit for H^S(l)/l. Uniform convergence in probability for the fixed distribution is equivalent to H^S(l)/l tending to zero. This confirms the interpretation: learnability is governed by the per-sample rate at which the class can create distinct finite labelings, and the distribution-free version is controlled by the largest set it can shatter.
