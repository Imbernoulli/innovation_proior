Let me start from the thing that actually breaks, stated as sharply as I can. I am minimizing the average device loss `ell(theta) = (1/m) sum_k L_k(theta)`, and a global solution `theta^*` is characterized by `grad ell(theta^*) = (1/m) sum_k grad L_k(theta^*) = 0`. Now look at what an individual device wants. A device that minimizes its own `L_k` lands at `theta_k^*` with `grad L_k(theta_k^*) = 0`. Under heterogeneous data these two conditions are incompatible: at the *global* optimum the per-device gradients `grad L_k(theta^*)` are individually non-zero — they only sum to zero — so no device is at rest at `theta^*`, and conversely if every device comes to rest at its own `theta_k^*`, the consensus of those rest points is `(1/m) sum_k theta_k^*`, which is not `theta^*`. The two fixed points disagree. Let me make that disagreement concrete so I am not arguing about an abstraction: take two clients with one-dimensional quadratics `L_1(x) = 0.5*2*(x-1)^2` and `L_2(x) = 0.5*3*(x-4)^2`. Then `grad L_1 = 2(x-1)`, `grad L_2 = 3(x-4)`, the device optima are `theta_1^* = 1` and `theta_2^* = 4`, their average is `2.5`, but the global optimum solves `2(x-1) + 3(x-4) = 0`, i.e. `x^* = (2*1 + 3*4)/5 = 2.8`. So a method whose rest point is the average of device optima lands at `2.5`, off by `0.3` from `2.8` — and it is off *because* the heavier client (weight 3) wants to pull harder toward `4`, which the plain average ignores. Every local-averaging method I know of is fighting this `2.5`-vs-`2.8` gap rather than removing it: I broadcast a model, let devices descend their own losses, and average — and the average is dragged toward the wrong fixed point, the more so the more local work I allow. So the question I actually want to answer is not "how do I damp the drift" but "can I change what each device is solving so that its rest point coincides with the global one." If I could make the local stationary point equal the global stationary point, then converging the local models *to consensus* would automatically converge them to `theta^*`, and the whole inconsistency would evaporate. That is the target; whether it is reachable is what I have to work out.

Let me first see precisely why the standard repairs do not do this, because the failure tells me what shape the fix must have. The proximal repair has each device minimize `L_k(theta) + (mu/2)||theta - theta^{t-1}||^2`, a spring anchored at the broadcast model. Compute its stationary point: `grad L_k(theta) + mu(theta - theta^{t-1}) = 0`. At a true global optimum I would need this to be solved by `theta = theta^*`, i.e. `grad L_k(theta^*) + mu(theta^* - theta^{t-1}) = 0`. But `grad L_k(theta^*)` is some fixed non-zero vector (the device's residual gradient at the consensus), so the equation forces `theta^* - theta^{t-1} = -(1/mu) grad L_k(theta^*)`, which is a different displacement for every device — they cannot all be satisfied by a common `theta^{t-1} = theta^*`. The penalty is the *same quadratic every round*; its minimizer is biased by exactly the term `-(1/mu) grad L_k`, and no amount of iterating fixes a bias that the objective itself bakes in. Let me actually check that on my two quadratics rather than trust the algebra. The proximal local solution is closed form, `x_k = (a_k b_k + mu*theta)/(a_k + mu)`, and the server averages; I can just iterate that map to its fixed point for a few values of `mu`. Doing so: at `mu = 0.1` it settles at `theta = 2.512`, at `mu = 1` at `2.588`, at `mu = 10` at `2.742`. All three sit short of `2.8`, and `grad ell` at those points is `-0.72`, `-0.53`, `-0.145` respectively — never zero. The bias shrinks only as `mu -> infinity`, which is exactly the regime where the spring is so stiff that the device barely moves off the broadcast model and does no useful local work. So solving the subproblem *exactly* does not save the proximal method; for it, inexact minimization is not a compromise but the only thing that keeps it usable, and even then it rests at the wrong place. A static regularizer is structurally incapable of aligning the fixed points. The regularizer has to *move*.

What would I need to add so that `theta = theta^{t-1}` (consensus) is itself the stationary point of the local subproblem? Suppose the local objective is `L_k(theta) + <a_k, theta> + (mu/2)||theta - theta^{t-1}||^2` for some per-device linear term `a_k` I get to choose. Its gradient is `grad L_k(theta) + a_k + mu(theta - theta^{t-1})`. Evaluate at the consensus point `theta = theta^{t-1}`: the quadratic contributes nothing there, and I am left with `grad L_k(theta^{t-1}) + a_k`. For consensus to be a stationary point of *every* device's subproblem I need `a_k = -grad L_k(theta^{t-1})`. So the linear term should subtract off the device's own gradient at the consensus point, and then *at* the consensus the device feels no pull away from it. The regularizer has to be "dynamic" precisely because this `a_k` tracks the device's latest gradient and so changes round to round — a static penalty cannot carry a per-round, per-device gradient inside it. Where the static spring only damps the move, this linear term cancels the residual gradient that *causes* the move. On the two quadratics, `a_1 = -grad L_1(theta) = -2(theta-1)` and `a_2 = -3(theta-4)`; at any common `theta` these two linear terms are different vectors, which is exactly why a single shared quadratic could never reproduce them.

Let me write the local objective the device actually minimizes in round `t`. The device holds its model from last time it participated, `theta_k^{t-1}`, and its gradient there. The subproblem is

```
theta_k^t = argmin_theta [ L_k(theta) - <grad L_k(theta_k^{t-1}), theta> + (alpha/2) ||theta - theta^{t-1}||^2 ].
```

The linear term uses the gradient at the device's *own previous iterate* `theta_k^{t-1}` (which the device knows), not at the current broadcast `theta^{t-1}` (which would require an extra gradient pass). The first-order condition for the local optimum is

```
grad L_k(theta_k^t) - grad L_k(theta_k^{t-1}) + alpha(theta_k^t - theta^{t-1}) = 0.
```

This is the recursion I need, and it is self-referential in a useful way: it expresses `grad L_k(theta_k^t)`, the new gradient, in terms of the old gradient and the displacement `theta_k^t - theta^{t-1}`:

```
grad L_k(theta_k^t) = grad L_k(theta_k^{t-1}) - alpha(theta_k^t - theta^{t-1}).
```

So I never need to recompute the gradient explicitly to maintain the linear term — solving the subproblem *gives* me the updated gradient for free, as a function of where the device ended up relative to the broadcast model. That is the dynamic part of the regularizer realized at zero extra cost: each round the device's stored gradient state is refreshed by the displacement it just took, and that refreshed gradient becomes next round's linear term. Let me name the stored state. Define `h_k` so that the linear term is `-<grad L_k(theta_k^{t-1}), theta> = +alpha <h_k, theta>` with `h_k = -grad L_k(theta_k^{t-1}) / alpha`. Then the recursion above says `h_k` accumulates the displacements: `h_k^{t} = h_k^{t-1} + (theta_k^t - theta^{t-1})`, starting from `h_k = 0` (the input condition `grad L_k(theta_k^0) = 0`). So `h_k` is just the running sum of the device's net moves away from the broadcast models — a memory of how far this device has consistently pulled. I should double-check that the identity `grad L_k(theta_k^t) = -alpha h_k^t` actually holds as a maintained invariant and not just as a relabeling; I will verify it numerically once I have the full loop, because if the stored state ever drifts from the true gradient the whole correction is built on sand.

Now I have to handle the inexactness honestly. No device solves the argmin exactly; it runs a few local SGD steps. But notice the linear and quadratic terms are *closed-form* in `theta` — their gradient is `-grad L_k(theta_k^{t-1}) + alpha(theta - theta^{t-1})`, which I can add to the data gradient at every local step. Concretely, the regularizer gradient is `alpha(theta - theta^{t-1}) + alpha h_k`: an `alpha`-strength weight-decay-like pull back toward the broadcast model plus a fixed `alpha h_k` offset that is the accumulated correction. So the device runs plain SGD on `L_k`, and before each step adds `alpha(theta - theta^{t-1} + h_k)` to the gradient. The quadratic `(alpha/2)||theta - theta^{t-1}||^2` contributes the `alpha(theta - theta^{t-1})` part — exactly an L2 pull to the anchor — and the dynamic linear term contributes the `alpha h_k` part. This is the minimal change to the local loop, and crucially it requires the device to know only `theta^{t-1}` (broadcast) and its own `h_k` (stored locally). Nothing global beyond the model crosses the wire — unlike a control-variate method, I am not shipping a separate correction vector each round; the correction lives in the *device's* persistent state and in the *server's* aggregation, both reconstructed from the models alone. That last clause is a claim I have to make good on; it is not obvious yet that the server can rebuild its share of the correction from model deltas, so let me work the server side and then test the reconstruction.

The server side is where the alignment is sealed, and I have to get it exactly right or the fixed-point argument collapses. The naive thing is to average the active devices' models: `(1/|P_t|) sum_{k in P_t} theta_k^t`. But that average is the consensus of points whose subproblems were corrected by the *per-device* linear terms, and the correction has to be re-injected at the server so the global model sits at the aligned point rather than the biased average. Let me derive what to add. Maintain a server state `h` that mirrors the device states. I want `h` to track `(1/m) sum_k grad L_k(theta_k^t)`, the average over *all* devices of their stored gradient — because that average is precisely `grad ell` evaluated coordinate-wise at the device iterates, and if I can drive it to zero I have driven `grad ell` to zero. Using the per-device recursion `grad L_k(theta_k^t) = grad L_k(theta_k^{t-1}) - alpha(theta_k^t - theta^{t-1})` and updating only the active devices, the change in that all-device average per round is `-alpha(1/m) sum_{k in P_t}(theta_k^t - theta^{t-1})`, so

```
h^t = h^{t-1} - alpha * (1/m) * sum_{k in P_t} (theta_k^t - theta^{t-1}).
```

The divisor is `m`, the *total* number of devices, not `|P_t|`, the number active — because `h` is meant to be the average over *all* devices' gradient states, and only the `|P_t|` active ones changed this round; dividing by `m` keeps `h` an honest all-device average despite partial participation, exactly as a control-variate server update would. Then the global model is the active-device average *corrected* by the server state:

```
theta^t = (1/|P_t|) sum_{k in P_t} theta_k^t - (1/alpha) h^t.
```

Before I trust the fixed-point story, let me unroll the reconstruction identity `-(1/alpha) h^t = (1/m) sum_{all k} H_k` and check it holds *every round*, not just at convergence, because if it only held in the limit the server would be running a different bookkeeping object than the devices and partial-participation accounting could quietly diverge. Unrolling `h^t` from `h^0 = 0`: each round it adds `-alpha(1/m) sum_{k in P_t}(theta_k^t - theta^{t-1})`, so `-(1/alpha) h^t = (1/m) sum_over_rounds sum_{active}(theta_k - theta^{t-1})`. And `H_k`, device `k`'s accumulated displacement, is by definition the running sum of its own net moves over the rounds it was active, so `(1/m) sum_k H_k` is the same double sum. They should agree term by term. I checked this directly with five clients, random per-round active subsets, and random net displacements over fifty rounds: at every round `-(1/alpha) h^t` and `(1/m) sum_k H_k` matched to machine precision. So the server state and the device states are literally one accumulated-displacement object viewed at two levels — there is one piece of bookkeeping, not two — and the global update can be written equivalently as `theta^t = active-average + (1/m) sum_k H_k`, maintained incrementally by adding `(1/m) sum_{k in P_t}(theta_k^t - theta^{t-1})` to a running `h_sum` each round.

Now the fixed-point claim itself, which is the only thing that justifies the whole construction, so I am going to actually run it rather than assert it. Take the same two quadratics, full participation, and let each client solve its subproblem exactly (closed form, since each is a 1-D quadratic): `x_k = (a_k b_k + g_k + alpha*theta)/(a_k + alpha)` with `g_k` the stored gradient, refreshed by `g_k <- g_k - alpha(x_k - theta)`, and the server update as above with `alpha = 0.5`. Starting from `theta^0 = 0`, `g_k = 0`, `h = 0` and iterating sixty rounds, the global model converges to `theta = 2.79999...` — that is `x^* = 2.8` to five places, *not* the naive average `2.5`. So on this example the method does land at the global optimum and not at the average of device optima. Three internal invariants I wanted to confirm came out right at the same fixed point: the stored gradients equal `-alpha H_k` exactly (`g_1 = +3.600`, `-alpha H_1 = +3.600`; `g_2 = -3.600`, `-alpha H_2 = -3.600`), confirming the device state never drifted from the true gradient; the server state equals the all-device gradient average (`h = -3.7e-7` and `(1/m) sum_k grad L_k(theta_k) = -3.7e-7`, both heading to zero); and `grad ell(theta) = -7e-6`, i.e. the consensus is a stationary point of the global loss to the tolerance I iterated to. So numerically: consensus is reached, it sits at the global optimum, the residual global gradient is going to zero, and the server's `h` is exactly the average device gradient that drives it there.

That experiment makes me confident enough to write down *why* it must hold in general. Suppose `theta_k^t -> theta^infty` for all `k`. Then the device recursion `grad L_k(theta_k^t) = grad L_k(theta_k^{t-1}) - alpha(theta_k^t - theta^{t-1})` forces the displacements `theta_k^t - theta^{t-1} -> 0`, so `h^t` stops changing and `theta^t -> theta^infty` as well, meaning `theta^infty` is independent of `k` and equals the consensus. With the displacements going to zero, the stored gradients `grad L_k(theta_k^t)` converge to `grad L_k(theta^infty)`, and since `h^t` is by construction their all-device average it converges to `(1/m) sum_k grad L_k(theta^infty) = grad ell(theta^infty)` — which is exactly the numeric agreement `h = (1/m) sum_k grad L_k(theta_k)` I just saw. For `theta^t` to be a fixed point of the server update, the correction `-(1/alpha) h^t` must be consistent with the average, and that holds only when `h^infty = 0`, i.e. `grad ell(theta^infty) = 0`. So the only consensus the method can rest at is a stationary point of the global loss — which is the alignment I was after, and the `2.8` in the experiment is that statement made arithmetic. The static-penalty method could never close this loop because its `h_k` is frozen at zero, which is precisely why it rested at `2.512` instead.

A few choices I want to pin down before writing it. First, why does the linear term use `grad L_k(theta_k^{t-1})` rather than `grad L_k(theta^{t-1})`? Because the device already has `theta_k^{t-1}` and the recursion refreshes the gradient there for free; reading the gradient at the broadcast model would cost an extra full pass each round and buy nothing the recursion does not already give. Second, `alpha` is the single knob: it is the strength of the quadratic anchor *and* the scale relating the stored gradient to the displacement. Larger `alpha` makes the local subproblem more strongly convex (the Hessian is `grad^2 L_k + alpha I`, so `alpha` past the negative-curvature floor convexifies it, giving a unique local solution and a clean displacement bound) and pulls the device harder toward the broadcast model; smaller `alpha` lets devices move more but weakens the correction. In the experiment the fixed point was the same for the `alpha` I picked; the alignment argument did not use the value of `alpha`, only `alpha > 0`, so I expect the *limit* to be `alpha`-independent and `alpha` to act on the *rate* and the convexity, which is the behavior I would want to confirm on the real benchmark before committing to a default. For deep nets on a Dirichlet-split CIFAR-style benchmark I will take `alpha = 0.1` as the default (with a sweep `{1e-3, 1e-2, 1e-1}` and a per-device scaling by data fraction), since the harness here is exactly a Dirichlet non-IID image/character setting. Third, plain SGD locally: the clean recursion that gives the gradient state for free relies on the local step being an ordinary gradient step plus the fixed regularizer offset; a momentum buffer would tangle the relationship between net displacement and accumulated gradient — the identity `grad L_k = -alpha H_k` that I verified would no longer be maintained by simple addition — so I keep the local optimizer plain and put all the structure in the added gradient term and the server correction.

There is one more comparison I want to make explicit, against the closest gradient-augmented method, because it sharpens why this is worth doing. A control-variate method also removes heterogeneity from the rate, but it transmits a control variate *in addition to* the model every round — twice the bits — because the server needs each device's latest gradient estimate, which is not recoverable from the model alone. Here, the device's correction is its accumulated displacement `H_k`, and the reconstruction check above showed the *server's* share `-(1/alpha) h` is exactly `(1/m) sum_k H_k`, rebuilt from the model deltas the server already receives; nothing extra goes over the wire. So I get the same heterogeneity-free alignment at half the communication, and because the method is built on (inexact) minimization of a well-posed local objective rather than a tuned SGD schedule, I expect it to be less sensitive to hyperparameter tuning — though that is an empirical claim I would only stand behind after the sweep, not something the fixed-point argument gives me. That is the trade I am after: a single `alpha`, one model per round each way, and a fixed-point argument that says the only place the federation can come to rest is a stationary point of the true global loss.

What I can prove about the *rate* I want to state more carefully, because the experiment only tested the limit, not how fast it gets there. The convexified subproblem (Hessian `grad^2 L_k + alpha I`) and the displacement recursion give me the ingredients for a convergence proof; I expect a linear rate for strongly convex device losses, `O(1/T)` for the non-convex stationarity measure, with an `m/P` factor for partial participation, and — this is the structural point — without any bounded-dissimilarity assumption, because what drives the rate is the alignment of the fixed points (which I checked) rather than a bound on how dissimilar the devices are. I have not written those rate bounds out here; the fixed-point alignment is the part I have actually verified, and the rates are the part I would derive and then sanity-check against the round-count-to-target-accuracy on the benchmark.

So the construction assembles itself from the pieces I have checked, and it drops into the harness as a fill of the strategy: keep a per-device accumulated-displacement state `H_k` and a server running sum `h_sum = (1/m) sum_k H_k`; in the local loop, add `alpha(theta - theta^{t-1} + H_k)` to the gradient at every plain-SGD step; after training, accumulate `H_k += (theta_k^t - theta^{t-1})`; on the server, average the active models, add the contributions to `h_sum`, and set the new global model to `active-average + h_sum`. The full module is in the answer.

The causal chain in one breath: the local and global stationary points are *inconsistent* under non-IID data — every device wants to rest where the federation does not, the `2.5`-vs-`2.8` gap on the two-quadratic example — so damping the drift (decaying LR, static proximal penalty) can only throttle a bias the objective itself bakes in, which I saw directly when exact FedProx rested at `2.512`/`2.588`/`2.742` with `grad ell != 0` -> instead make the regularizer *dynamic*: add a linear term `-<grad L_k(theta_k^{t-1}), theta>` that cancels the device's own gradient at the consensus point, so consensus becomes a local stationary point, and the first-order condition refreshes the stored gradient `grad L_k(theta_k^t) = grad L_k(theta_k^{t-1}) - alpha(theta_k^t - theta^{t-1})` for free (realized as a per-step `alpha(theta - theta^{t-1} + H_k)` gradient add, with the invariant `grad L_k = -alpha H_k` verified to hold exactly) -> close the loop at the server with a state `h^t = h^{t-1} - alpha(1/m) sum_{P_t}(theta_k - theta^{t-1})` and model `theta^t = active-average - (1/alpha) h^t = active-average + (1/m) sum_k H_k` (the two forms verified equal every round under partial participation), whose only fixed point forces `grad ell = 0` — confirmed numerically by the two-quadratic case converging to `2.8` -> the same `H_k` reconstructed by the server from model deltas means no control variate over the wire (half the bits of a control-variate method), `alpha` the single convexifying knob (default 0.1 on Dirichlet CIFAR), plain SGD locally — heterogeneity gone from the limit with no bounded-dissimilarity assumption, by aligning the two fixed points instead of fighting their gap.
