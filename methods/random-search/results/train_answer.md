Almost every learning algorithm carries a fistful of knobs that training does not fit — the learning rate and its annealing schedule, the number of hidden units, the weight-initialization scheme and its scale, the nonlinearity, the regularization strength, the preprocessing choice — and the model only becomes a concrete predictor once I fix all of them. Call the knob vector $\lambda$. What I want is the $\lambda$ that minimizes generalization error, $E_{x}[L(x; A_\lambda(X_{\text{train}}))]$, but I cannot see the expectation over the true data distribution, so I replace it by the mean loss on a held-out validation set, giving a response function $\Psi(\lambda) = \text{mean}_{x \in X_{\text{valid}}} L(x; A_\lambda(X_{\text{train}}))$, and the whole job collapses to minimizing $\Psi$ over the configuration space $\Lambda$. This is about the worst optimization target imaginable: a single evaluation of $\Psi$ means training and scoring an entire model — hours of compute — so the budget is only tens to a few hundred evaluations; there is no gradient of $\Psi$ with respect to $\lambda$, no analytic form, no prior knowledge of where the optimum sits or even which coordinates matter; and $\lambda$ is a mongrel of continuous knobs spanning orders of magnitude, integer knobs, and categorical choices. The concrete question is narrow: out of that budget, which set of trial points $\{\lambda^{(1)}, \dots, \lambda^{(S)}\}$ should I evaluate $\Psi$ at?

The field default is grid search, usually salted with manual tinkering. Grid picks a handful of values $L^{(k)}$ for each of the $K$ knobs and evaluates the full Cartesian product, $S = \prod_k |L^{(k)}|$ trials, returning the best. I understand exactly why it is the default: it is trivial to write, needs no infrastructure beyond a job list and a cluster, parallelizes for free because every trial is known up front, and in one or two dimensions it is perfectly reliable. But $S = \prod_k |L^{(k)}|$ is exponential in $K$ — four values on each of eight knobs is already $4^8 = 65536$ trainings, and a 32-hyper-parameter deep belief network at even two values per knob is $2^{32}$, over a billion. That is Bellman's curse, and it is arithmetic, not a tooling problem. The other thing people do is manual search: a human fixing the architecture and hand-tuning by intuition, coordinate-descent style. It deserves fairness because it is adaptive — every trial informs the next, so a skilled operator reaches a good configuration in surprisingly few trials, around forty-one per data set in the DBN tuning this is measured against — but it is irreproducible, does not scale, depends wholly on operator skill, and is useless as a controlled baseline precisely because the human is in the loop. I want grid's simplicity and reproducibility without grid's exponential blowup.

The deeper waste in grid is not just "too many points." Lay $N = g^K$ grid trials over $K$ dimensions and ask how many distinct values of any single knob the experiment actually probes: every level of that knob is paired with every combination of the others, so it takes only $g = N^{1/K}$ distinct values. Sixty-four trainings in six dimensions probe each individual knob at only $g = 2$ settings — project all 64 points onto any axis and they collapse onto two stacks of 32 coincident points. The grid spends its budget on combinations, and from the viewpoint of any one coordinate the combinations are mostly redundant. Whether that redundancy is merely wasteful or genuinely serious depends on the shape of $\Psi$, and here the decisive fact emerges. Fit a Gaussian process to samples of $\Psi$ with a squared-exponential kernel $\exp(-(a-b)^2/l)$, one length-scale $l$ per knob chosen by maximizing the marginal likelihood, and read off relevance as $1/l$. Across seven neural-net knobs and several related data sets, two properties recur. First, on any one data set only a small handful of knobs move $\Psi$ — the effective dimension is roughly one to four inside a nominally seven-dimensional problem, with the learning rate consistently important. Second, and sharper, *which* of the remaining knobs matter changes from data set to data set: annealing rate on one task, the $L_2$ penalty on another, the hidden-unit count on a third. So $\Psi$ lives on a low-dimensional subspace of $\Lambda$, but I am not allowed to know which subspace, and it moves when the data moves. If I knew the important subspace I could grid those two or three knobs and ignore the rest; because I do not, I am forced to put grid resolution on all $K$ axes, and the levels I lay on an irrelevant knob are pure waste — every one of the $g^{K-1}$ points sharing a value of an important knob re-measures essentially the same $\Psi$, differing only in knobs that do nothing.

So I want a trial set that covers the important low-dimensional subspace well no matter which subspace it turns out to be. The grid fails because its points are aligned — they share coordinates and collapse under projection. I propose Random Search: refuse to align the points and draw each trial's configuration as an independent uniform sample from the configuration space, every knob drawn independently from its own range, over the same domain a grid would span. The defining update is simply, for $t = 1, \dots, T$, draw $\lambda^{(t)}$ by sampling each knob independently — categorical from a uniform choice over its options, a plain continuous knob from $\text{uniform}[\text{low}, \text{high}]$, an integer knob from a uniform integer — evaluate $\Psi(\lambda^{(t)})$ at full fidelity, and at the end return the $\lambda$ with the best validation objective. The reason this beats grid is the projection argument run in reverse: because each of $N$ independent points draws a given coordinate independently from a continuous distribution, the projected values onto any single axis are $N$ draws from the uniform on that axis, almost surely all distinct. So $N$ i.i.d. points probe *every* knob at $N$ distinct settings, where the grid probed each at only $N^{1/K}$. On whichever low-dimensional subspace matters, independent sampling has the resolution of the full budget instead of its $K$-th root — and it gets this *for free, without knowing* which subspace it is. Nine independent points test the one important knob at nine distinct places whether that knob is the first or the second; the grid would have to be rebuilt to move its resolution. Random search therefore has the same efficiency on the relevant subspace as if it had been told the subspace in advance and searched only there. The full-space evenness that grid buys is genuinely real, and random points are a little clumpy in the full box — but full-space evenness is exactly what a low-effective-dimension target does not need, and per-axis resolution is exactly what it does.

A key design choice is how to sample knobs that span orders of magnitude — learning rate, hidden-unit count, regularization strength. For these the response is roughly flat per decade, so I sample uniformly in log space, exponentiating a uniform draw over $[\log \text{low}, \log \text{high}]$ (and rounding for log-scaled integers), to cover the *effect* evenly rather than the raw value; the obvious alternative of a plain uniform over the natural range would crowd almost all draws into the largest decade and waste them.

I want this quantitative, not just a picture. Idealize the good region as a target occupying a fraction $v/V$ of the unit hypercube's volume, placed where I do not know, and throw $T$ independent uniform trials. Each independently lands in the target with probability $v/V$ and misses with probability $(1 - v/V)$; the trials are independent, so all $T$ miss with probability $(1 - v/V)^T$, and

$$P(\text{find}) = 1 - (1 - v/V)^T.$$

With $v/V = 0.01$ this is $1 - 0.99^T$: $1 - 0.99^{60} = 0.4528$ is below a coin flip, $T = 69$ reaches about $50\%$, and for $95\%$ I need $0.99^T \le 0.05$, i.e. $T \ge \ln(0.05)/\ln(0.99) = 298.1$, so the first integer budget that reaches $95\%$ is $299$ trials — squarely inside the budget I actually have. What is decisive is what is *absent* from the formula: the ambient dimension $K$ never appears. The number of independent trials needed is governed only by the target's relative volume $v/V$, not by how many irrelevant axes the target is buried in. That is the precise meaning of "thrives on low effective dimensionality": a good region that is wide in the irrelevant directions and narrow in the one or two that matter can have a perfectly findable relative volume of $1\%$ even in thirty-two dimensions, and random search needs the same few hundred trials it would need in three. Grid, to be sure of catching the narrow directions, would have to resolve all thirty-two axes — $2^{32}$ before it even gets fine.

There is a second, sharper way grid loses on exactly the realistic case. Low effective dimension looks geometrically like an axis-aligned elongated rectangle — wide along the irrelevant knobs, thin along the important ones. A thin rectangle aligned with the lattice either threads through several collinear grid points at once, which are not independent shots but the same coordinate value in the thin direction sampled redundantly, or slips entirely between two adjacent grid lines and catches nothing; either way the grid's effective sample size against such a target collapses. Independent points are never collinear, so each is a genuinely independent shot and the $1 - (1 - v/V)^T$ law applies cleanly. The one escape for the grid would be a target rotated off the lattice, but hyper-parameter targets are axis-aligned almost by construction, because the important knobs *are* the coordinate axes.

I should check whether cleverer point sets leave naive independence behind. The quasi-Monte-Carlo low-discrepancy sequences — Sobol, Halton, Niederreiter — and Latin hypercube designs place points that look more uniform than uniform, minimizing discrepancy and keeping subspace projections low-discrepancy too, which is exactly the per-axis coverage I prize, and in QMC integration they beat independent draws. But their advantage is asymptotic, while I have tens to a few hundred trials, and when the dimension is large relative to the budget a low-discrepancy set is not expected to behave much differently from i.i.d. draws — there are not enough points for the structure to assert itself. A simulation of finding a $1\%$-volume target in three and five dimensions confirms this: on the elongated low-effective-dimension targets no grid is competitive, Latin hypercube comes out about the same as random, and Sobol is best but only by a few percentage points concentrated in the hundred-to-three-hundred-trial range. Sobol is a refinement, not the headline. And plain independence buys properties the deterministic sets quietly surrender, which I value enormously in practice. Because the trials are i.i.d., I can stop whenever I like and what ran is still a valid experiment; I can add trials later without recomputing or rebalancing; and if a job dies on a flaky cluster — which, with hundreds of multi-hour trainings, it will — I can drop or restart it freely, because no single trial is load-bearing for the set. None of that holds for a grid (omit a point and the lattice has a hole) or a low-discrepancy sequence (the points are constructed jointly). The independence is also what makes the search *analyzable*: $S$ i.i.d. trials can be reinterpreted as $N$ independent experiments of $s$ trials each, for any $s$ with $sN \le S$, just by partitioning, so one run of $S$ trials yields the whole distribution of best-of-$s$ performance versus experiment size — its median, quartiles, and extremes. A grid or Sobol set cannot be sliced that way. So independence is not merely convenient; it is what makes the method measurable and robust, well worth a couple of percentage points of Sobol's edge.

One trap remains if I am to report the result honestly. After $S$ trials the naive move is to crown the best-validation trial and report its test score, but test error is not monotone in validation error and the validation set is finite, so when several trials nearly tie, which one wins is itself noisy, and reporting the single luckiest one biases me optimistic. I therefore treat the test score $z$ of "the best model" as a Gaussian mixture over the $S$ trials: component $s$ has mean $\mu_s = \Psi_{\text{test}}(\lambda^{(s)})$ and variance $\sigma_s^2 = V_{\text{test}}(\lambda^{(s)})$, with weight $w_s = P(\text{trial } s \text{ wins})$ where each $Z^{(i)} \sim N(\Psi_{\text{valid}}(\lambda^{(i)}), V_{\text{valid}}(\lambda^{(i)}))$ — for a $0/1$ loss the variance is the Bernoulli $V = \Psi(1 - \Psi)/(|X| - 1)$. Then

$$\mu_z = \sum_s w_s\, \mu_s, \qquad \sigma_z^2 = \sum_s w_s\,(\mu_s^2 + \sigma_s^2) - \mu_z^2,$$

the weighted-mean identity and the law-of-total-variance for a mixture, which I verify by expanding $E[z^2] = \sum_s w_s E[Z_s^2] = \sum_s w_s(\sigma_s^2 + \mu_s^2)$ and $\sigma_z^2 = E[z^2] - \mu_z^2$. The weights $w_s$ have no closed form for many ties but are trivial to estimate by simulation — repeatedly draw hypothetical validation scores from those normals, count how often each trial wins, normalize; a few tens of draws suffice because the winners are close. This calculation is for reporting the chosen model's generalization under finite-validation uncertainty; it does not change which point gets proposed.

The one honest limitation I should name is that Random Search is deliberately non-adaptive: it ignores the results already in hand and never consults the history to decide where to look next. An adaptive, sequential, model-based search that learns which dimensions matter as it goes *should* beat it, and the right way to view this method is as the natural baseline against which those adaptive methods are measured — not the last word. But as a baseline it is exactly right: it keeps all of grid's practical virtues — conceptual simplicity, trivial parallelism, reproducibility — and trades a small efficiency loss in genuinely low dimensions for a large gain in high dimensions with low effective dimension, the regime hyper-parameter optimization actually lives in. Dropping it into the search harness, which calls the strategy once per trial with the space, the history, and the remaining budget, the proposal rule is almost shockingly small: ignore the history, ask the space for one independent uniform draw, evaluate at full fidelity. The space already knows how to draw each knob from its own range and type — the per-knob independent uniform/log-uniform draw argued for above — so I lean on it rather than re-deriving the sampling, and the seed is the one piece of state I keep, for reproducibility.

```python
import numpy as np
from typing import Any, Dict, List, Tuple


class RandomSearch:
    """Random search: each trial is an independent uniform draw from the
    configuration space (uniform, or uniform-in-log for scale knobs). The
    history is never consulted -- i.i.d. draws probe every knob at the full
    budget's resolution regardless of which subspace matters."""

    def __init__(self, seed: int = 42):
        self.seed = seed
        self.rng = np.random.RandomState(seed)   # reproducible from the seed

    def suggest(
        self,
        space: "SearchSpace",
        history: List["Trial"],     # unused: non-adaptive
        budget_left: int,           # unused: every trial is i.i.d.
    ) -> Tuple[Dict[str, Any], float]:
        config = space.sample_uniform(self.rng)  # one independent per-knob draw
        return config, 1.0                       # full fidelity (single-fidelity baseline)
```
