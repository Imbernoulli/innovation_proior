I start with the tabular lower bound rather than with an algorithm. If the state space is large, the usual tabular story cannot be repaired by clever bookkeeping: regret bounds scale with the number of state-action pairs, and that dependence is information-theoretic in an unstructured MDP. So I need an assumption that lets one trajectory teach me about many state-action pairs at once. A known feature map `phi(x,a) in R^d` is the natural handle, but simply saying that value functions are linear is not enough. The Bellman equation pushes values through an unknown transition kernel, and unless that push preserves the feature representation, the next backup can leave the linear class immediately.

The structure I need is therefore about the data-generating MDP, not about the policy class. I assume the reward is linear in the features and, more importantly, the transition kernel factors through the same features: `r_h(x,a)=<phi(x,a),theta_h>` and `P_h(.|x,a)=<phi(x,a),mu_h(.)>`, where `mu_h` is a vector of signed measures over the state space. This is a strong assumption, but it is the right kind of strength. It does not say that the unknown transition model is a finite matrix over states, and it does not say the policy is linear. It says that every Bellman expectation of a bounded future value function is a linear prediction in `phi`.

Once I write the Bellman equation under that factorization, the central insight becomes mechanical. For any policy `pi`,

`Q_h^pi(x,a)=<phi(x,a),theta_h> + int V_{h+1}^pi(x') <phi(x,a),dmu_h(x')> = <phi(x,a), theta_h + int V_{h+1}^pi dmu_h>`.

So every policy's action-value function is linear in the same features, including the optimal policy's action-value function. This is the reason the assumption is more useful than a value-function approximation guess: it is closed under the Bellman backup. If I can estimate the coefficient vector at each step, I can represent the result of dynamic programming without enumerating states.

Now the least-squares part appears for a reason. At step `h`, if I already have a future value estimate `V_{h+1}`, then the Bellman target `r_h + P_h V_{h+1}` is linear in `phi`. A sampled transition gives one noisy observation of that target: `r_h^tau + V_{h+1}(x_{h+1}^tau)`. Therefore a backward sweep can fit

`w_h = Lambda_h^{-1} sum_{tau<k} phi_h^tau [r_h^tau + max_a Q_{h+1}(x_{h+1}^tau,a)]`,

with `Lambda_h=sum_{tau<k} phi_h^tau (phi_h^tau)^T + lambda I`. This is least-squares value iteration, but now I see it as a sampled Bellman operator under a structural model, not as a generic regression trick.

Pure least squares still does not explore. If I act greedily with the fitted values, I may never visit feature directions where `Lambda_h` is small, so the estimates there remain wrong. Linear bandits tell me the uncertainty geometry: the width at feature vector `phi` is `sqrt(phi^T Lambda_h^{-1} phi)`. The inverse of the quadratic form is the effective number of samples collected in that direction. That suggests defining an optimistic value

`Q_h^k(x,a)=min{ <phi(x,a),w_h^k> + beta sqrt(phi(x,a)^T (Lambda_h^k)^{-1} phi(x,a)), H }`.

This looks like linear regression plus a UCB bonus, but if I stop here I miss the main difficulty. In a bandit, the response noise is attached to an observed reward and the future does not exist. Here the regression target contains `V_{h+1}^k(x_{h+1}^tau)`, and that function is computed by the same algorithm from the same data. The target is data-dependent. A self-normalized martingale bound for one fixed `V` would be invalid if I silently plug in the learned `V_{h+1}^k`.

The proof has to make concentration uniform over all value functions the algorithm might produce. Those value functions have the form `V(.)=min{max_a [phi(.,a)^T w + beta sqrt(phi(.,a)^T Lambda^{-1} phi(.,a))],H}`. Ridge regularization bounds `w`; the ridge matrix has eigenvalues bounded below; clipping bounds the range by `H`. After reparametrizing the bonus as `sqrt(phi^T A phi)` with `A=beta^2 Lambda^{-1}`, the value class can be covered by gridding `w` in `R^d` and `A` in `R^{d x d}`. The `A` grid contributes a `d^2` term. That `d^2` is not cosmetic: it is the price of making the self-normalized concentration hold for the data-dependent future values generated by the algorithm.

With that uniform event, the ridge prediction obeys the key relation

`|<phi,w_h^k> - Q_h^pi(x,a) - P_h(V_{h+1}^k - V_{h+1}^pi)(x,a)| <= beta ||phi||_{(Lambda_h^k)^{-1}}`

for a choice `beta = c d H sqrt(iota)`, `iota=log(2dT/p)`. I can see all three proof pieces inside this inequality. Regularization bias is controlled by the bounded true Bellman weights. The stochastic term is controlled by the value-class covering plus self-normalized concentration. The recursive term becomes exactly `P_h(V_{h+1}^k - V_{h+1}^pi)` because the transition kernel is linear in `phi`.

This is where optimism becomes tractable. I prove `Q_h^k >= Q_h^*` by going backward in `h`. At the last step there is no future value, so the bonus covers the regression error directly. If optimism holds at `h+1`, then `V_{h+1}^k - V_{h+1}^*` is nonnegative pointwise, and applying the transition kernel keeps it nonnegative. The recursion term in the key relation helps rather than hurts, so the same bonus covers the residual at step `h`. The clip at `H` is harmless because true values are at most `H`. This is exactly what a naive per-stage bandit view would lose: optimism is not independent across steps; it is propagated by the Bellman equation.

For regret, I compare the optimistic value of the policy actually executed with its realized value. Let `delta_h^k=V_h^k(x_h^k)-V_h^{pi_k}(x_h^k)`. The key relation gives

`delta_h^k <= delta_{h+1}^k + zeta_{h+1}^k + 2 beta ||phi_h^k||_{(Lambda_h^k)^{-1}}`,

where `zeta` is a martingale difference from replacing the conditional next-state gap by the realized one. Summing over steps and episodes gives a martingale part plus a bonus part. Azuma controls the martingale part by `O(H sqrt(T iota))`. The bonus part is where the linear-bandit geometry reappears: for each fixed `h`, the elliptical-potential lemma bounds `sum_k phi_h^k^T (Lambda_h^k)^{-1} phi_h^k` by `O(d log T)`, and Cauchy-Schwarz turns this into `O(sqrt(dK log T))` widths at that step. Summing over `H` steps gives `H sqrt(dK log T)`.

Multiplying the cumulative width by `beta` gives the headline scale. The confidence radius contributes `dH`; the potential term contributes `H sqrt(dK)`; therefore the leading regret is `d^{3/2} H^2 sqrt(K)` up to logarithms, which is `sqrt(d^3 H^3 T)` because `T=KH`. The final high-probability regret is `tilde O(sqrt(d^3 H^3 T))`. The theorem has no state-cardinality term and no action-cardinality term in the statistical rate; finite actions still matter operationally because the greedy maximization and stored action features cost time and space.

I also want to keep straight what the algorithm does not do. The vector `mu_h` is a signed measure over the state space, so the transition model has infinitely many degrees of freedom when the state space is infinite. The algorithm cannot learn the whole model in total variation, and the proof does not ask it to. It only proves that the empirical Bellman operator is accurate on the small class of value functions the algorithm can construct. That is why the method is not merely model-based estimation in disguise. It learns the predictions needed for planning, not the full kernel.

Under misspecification, the same picture degrades in the expected way. If rewards and transitions are within `zeta` of the linear representation, Bellman linearity becomes approximate: the policy-value linearization error is at most `2H zeta`, the key relation carries an additional `4H zeta`, and optimism holds only up to `4H(H+1-h)zeta` at step `h`. Since that bias is not mean-zero, it is handled deterministically while the bonus grows as `beta_k=c(d sqrt(iota)+zeta sqrt(kd))H`. Telescoping pays a deterministic `4HT zeta` and the variable bonus contributes the larger stated additive scale, `tilde O(zeta d H T)`. That linear-in-`T` dependence is not an artifact; a wrong structural model can impose a persistent per-step bias. The point is that the square-root regret term remains intact when the model is exact and the degradation is explicit when it is not.
