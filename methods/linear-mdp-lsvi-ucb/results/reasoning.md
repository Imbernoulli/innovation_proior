I start with the tabular lower bound rather than with an algorithm. If the state space is large, the usual tabular story cannot be repaired by clever bookkeeping: regret bounds scale with the number of state-action pairs, and that dependence is information-theoretic in an unstructured MDP. So I need an assumption that lets one trajectory teach me about many state-action pairs at once. A known feature map `phi(x,a) in R^d` is the natural handle. The naive way to use it would be to assume that value functions are linear in `phi` and fit them by regression. But I do not control the value functions directly; they are produced by the Bellman equation, which pushes values through an unknown transition kernel. Unless that push preserves the feature representation, the next backup can leave the linear class immediately, and then a linear-value assumption at one step says nothing about the next.

So the assumption I need should be about the data-generating MDP, not about a policy or value class I hope to fit. Suppose the reward is linear and, more importantly, the transition kernel factors through the same features: `r_h(x,a)=<phi(x,a),theta_h>` and `P_h(.|x,a)=<phi(x,a),mu_h(.)>`, where `mu_h` is a vector of signed measures over the state space. Before I lean on this, I want to know what it actually buys when I run a Bellman backup, because that is the operation everything downstream depends on. Write out one backup. For any policy `pi`,

`Q_h^pi(x,a)=<phi(x,a),theta_h> + int V_{h+1}^pi(x') <phi(x,a),dmu_h(x')>`,

and the integral is linear in `phi(x,a)`, so I can pull it out:

`Q_h^pi(x,a)=<phi(x,a), theta_h + int V_{h+1}^pi dmu_h>`.

The claim is that this is closed: whatever `V_{h+1}^pi` is, the backed-up `Q_h^pi` is again linear in `phi` with coefficient `w_h = theta_h + int V_{h+1}^pi dmu_h`. I do not fully trust an identity I derived by pulling an inner product through an integral until I have watched it hold on numbers, so I build the smallest MDP that exercises every term: `d=2`, two states, two actions. I draw random unit features `phi[x,a]`, a random factor matrix `mu` so that `P(x'|x,a)=sum_k phi[x,a,k] mu[k,x']`, random `theta`, and an arbitrary `V_{h+1}` over the two next states. Then I compute the backup two ways: directly as `r(x,a)+sum_{x'} P(x'|x,a) V(x')`, and through the linear form `<phi(x,a), theta + mu V>`. The two `Q` tables agree to `4.4e-16`. So the factorization really is closed under the Bellman backup, by construction and not by hope, and it is closed for *every* policy including the optimal one — the same `phi`, only the coefficient `w_h` changes. That is materially stronger than assuming the optimal value happens to be linear: it survives dynamic programming. If I can estimate the coefficient vector at each step, I can represent the result of planning without ever enumerating states.

That points the way to the estimator. At step `h`, suppose I already have a future value estimate `V_{h+1}`. Then the Bellman target `r_h + P_h V_{h+1}` is, by the same identity, a linear function of `phi`, and a single sampled transition gives one noisy observation of it: `r_h^tau + V_{h+1}(x_{h+1}^tau)`. Regressing that scalar on the feature `phi_h^tau` across past episodes is exactly ridge regression. A backward sweep over `h` fits

`w_h = Lambda_h^{-1} sum_{tau<k} phi_h^tau [r_h^tau + max_a Q_{h+1}(x_{h+1}^tau,a)]`,

`Lambda_h=sum_{tau<k} phi_h^tau (phi_h^tau)^T + lambda I`.

This is least-squares value iteration, but now I read it as a sampled Bellman operator under a structural model rather than a generic regression trick — the linearity of the operator is what makes the regression well-specified in the first place.

Pure least squares does not explore, though. If I act greedily on the fitted values, I may never visit feature directions where `Lambda_h` is small, and the estimates there stay wrong forever because no data corrects them. I need an exploration bonus, and the linear-bandit geometry tells me its shape: the prediction width at feature vector `phi` under ridge regression is `sqrt(phi^T Lambda_h^{-1} phi)`. The quadratic form `phi^T Lambda_h^{-1} phi` is small precisely along directions where I have accumulated samples and large along directions I have neglected, so adding a multiple of it to the fitted value inflates the estimate exactly where I am uncertain. That suggests an optimistic action-value

`Q_h^k(x,a)=min{ <phi(x,a),w_h^k> + beta sqrt(phi(x,a)^T (Lambda_h^k)^{-1} phi(x,a)), H }`,

clipped at `H` because rewards in `[0,1]` over `H` steps make every true value at most `H`.

If I stop here I am treating each step like an independent linear bandit, and that is where I should be suspicious, because it would be too easy. In a bandit the regression target is an observed reward and there is no future. Here the target contains `V_{h+1}^k(x_{h+1}^tau)`, and that function is itself computed by this algorithm from this same data. The target is data-dependent. A self-normalized martingale bound is proved for a *fixed* function `V`; if I silently substitute the learned `V_{h+1}^k`, which depends on the very noise I am trying to control, that bound does not apply. So I cannot just cite the bandit confidence interval. I have to make the concentration hold uniformly over every value function the algorithm could possibly produce.

What is that class? Every `V_{h+1}^k` has the form `V(.)=min{max_a [phi(.,a)^T w + beta sqrt(phi(.,a)^T Lambda^{-1} phi(.,a))],H}`. The free parameters are `w in R^d` and the matrix inside the bonus. Ridge regularization with `lambda=1` bounds `||w||` (the targets are at most `H` and `Lambda_h` has eigenvalues at least `1`), and the clip bounds the range by `H`, so the class is not arbitrary — it is parametrized by a bounded `w` and a bounded matrix. Rewrite the bonus as `sqrt(phi^T A phi)` with `A=beta^2 Lambda^{-1}`; then covering the class means gridding `w` over `R^d` and `A` over the `d x d` symmetric matrices. The `w`-grid contributes a `log` term of size `d`; the `A`-grid contributes a `log` term of size `d^2`. The `d^2` is not cosmetic — it is the explicit price of letting the future value in my regression target be one of my own learned functions. It is the reason the confidence radius will scale like `dH` and not `sqrt(d)H`. If I had only a fixed `V`, the `A` grid would be unnecessary and the radius would be smaller; the data-dependence is what costs the extra factor of `sqrt(d)`.

With that uniform high-probability event in hand, the ridge prediction obeys a relation I can decompose term by term:

`|<phi,w_h^k> - Q_h^pi(x,a) - P_h(V_{h+1}^k - V_{h+1}^pi)(x,a)| <= beta ||phi||_{(Lambda_h^k)^{-1}}`,

for `beta = c d H sqrt(iota)`, `iota=log(2dT/p)`. I can name what makes each piece bounded. The regularization bias is controlled because the true Bellman weight `theta_h + int V dmu_h` is bounded (this is where `||mu_h(S)||<=sqrt(d)` enters). The stochastic term is controlled by the value-class covering above plus self-normalized concentration. And the recursive term comes out as exactly `P_h(V_{h+1}^k - V_{h+1}^pi)` — not an approximation — precisely because the transition kernel is linear in `phi`, so applying it to a difference of value functions stays inside the algebra I already verified.

Now I can ask whether optimism actually holds, i.e. whether `Q_h^k >= Q_h^*`. I prove it by backward induction on `h`. At `h=H+1` there is no future, so the deficit is zero; at `h=H` the regression has no recursive term and the bonus `beta ||phi||` covers the residual directly. The inductive step is the part I want to check rather than assert, because it relies on a monotonicity that is easy to state and easy to get backwards. Suppose `V_{h+1}^k >= V_{h+1}^*` pointwise, so `g := V_{h+1}^k - V_{h+1}^* >= 0`. The key relation's recursive term is `P_h g`. The claim is that `P_h g >= 0`, so the recursion *helps* the upper bound instead of eroding it. I test this on a genuinely stochastic linear MDP — `d=2`, three states, two actions, with the factored `P` renormalized so each `P_h(.|x,a)` is a real distribution. With a nonnegative `g`, `min_{x,a} (P_h g)(x,a) = 0.29 >= 0`, as needed. To be sure the sign is load-bearing and I am not getting a free pass, I flip one entry of `g` negative: now `min (P_h g) = -0.49 < 0`. So the nonnegativity of the future gap is exactly what is doing the work; the induction would fail without it. Concretely: optimism at `h+1` makes `g>=0`, the kernel carries that to `P_h g >= 0`, and then the bonus alone has to cover the remaining residual at `h`, which it does. The clip at `H` is harmless because the true value it might cut off is itself at most `H`. The lesson is that optimism here is not an independent per-step property; it is propagated through the horizon by the Bellman equation, which is exactly what a stagewise bandit reading would miss.

For regret I compare the optimistic value of the executed policy with its realized value. Let `delta_h^k=V_h^k(x_h^k)-V_h^{pi_k}(x_h^k)`. The key relation gives

`delta_h^k <= delta_{h+1}^k + zeta_{h+1}^k + 2 beta ||phi_h^k||_{(Lambda_h^k)^{-1}}`,

where `zeta` is a martingale difference from replacing the conditional next-state gap by the realized one. Summing over `h` and `k` splits the regret into a martingale part and a bonus part. Azuma bounds the martingale part by `O(H sqrt(T iota))`. The bonus part is where the elliptical-potential lemma reappears: for each fixed `h`, `sum_k phi_h^k^T (Lambda_h^k)^{-1} phi_h^k = O(d log T)`, and Cauchy-Schwarz turns the sum of widths into `sqrt(K * d log T)` at that step. Over `H` steps the cumulative width is `H sqrt(d K log T)`.

I want to be sure the final exponents are what I think they are, because this is the kind of arithmetic where a stray `H` is easy to drop. Multiply the cumulative width by `beta`:

`beta * H sqrt(d K log T) = (d H sqrt(iota)) * H sqrt(d K log T) = d^{3/2} H^2 sqrt(K) * sqrt(iota log T)`.

The confidence radius contributes one factor of `d` and one `H`; the potential term contributes `H sqrt(dK)`; together that is `d^{3/2} H^2 sqrt(K)`. Now substitute `T=KH`, so `sqrt(K)=sqrt(T/H)`:

`d^{3/2} H^2 sqrt(T/H) = d^{3/2} H^{2-1/2} sqrt(T) = d^{3/2} H^{3/2} sqrt(T) = sqrt(d^3 H^3 T)`.

I check the exponents collapse cleanly: under the square root the `d` exponent is `2 * 3/2 = 3` and the `H` exponent is `2 * 3/2 = 3`, so the leading regret is `tilde O(sqrt(d^3 H^3 T))`. The bound carries no state-cardinality term and no action-cardinality term in the statistical rate. Finite actions still matter operationally — the greedy `argmax` and the stored action features cost time and space — but they do not appear in the regret.

I also want to keep straight what the algorithm does not do, because the structural assumption could be misread as "estimate the model and plan." The vector `mu_h` is a signed measure over the state space, so when the state space is infinite the transition model has infinitely many degrees of freedom. The algorithm cannot learn that model in total variation, and nothing in the proof asks it to. What the uniform-concentration step actually establishes is only that the empirical Bellman operator is accurate on the small, covered class of value functions the algorithm can construct. That is why this is not model-based estimation in disguise: it learns the predictions planning needs, not the kernel.

Finally, misspecification, to see whether the structure is brittle. Suppose rewards and transitions are within `zeta` of the linear representation. Bellman linearity becomes approximate: the policy-value linearization error is at most `2H zeta`, and the key relation picks up an extra `4H zeta` per step. The interesting question is how this bias accumulates through the horizon, because if it compounded multiplicatively the whole scheme would be useless under any error. The optimism deficit obeys a recursion: at the terminal step the deficit is `0`, and each backward step adds `4H zeta` plus whatever deficit was inherited from `h+1` (the recursive term carries it back). So `b_h = 4H zeta + b_{h+1}`, `b_{H+1}=0`. I unroll this numerically with `H=5`, `zeta=0.1` and compare against the closed form `4H(H+1-h)zeta` at every step: `h=5` gives `2.0`, `h=4` gives `4.0`, down to `h=1` giving `10.0`, and each matches `4H(H+1-h)zeta` exactly. So the deficit grows only *linearly* in the number of remaining steps, not geometrically, and at `h=1` it is `4H^2 zeta` per episode — the deterministic price the regret proof pays each episode. Telescoping over `K` episodes that is `4HT zeta`, and with the inflated bonus `beta_k=c(d sqrt(iota)+zeta sqrt(kd))H` the variable term contributes the larger additive scale `tilde O(zeta d H T)`. That linear-in-`T` term is honest, not an artifact: a wrong structural model imposes a persistent per-step bias that no amount of data removes. The reassurance is that when the model is exact (`zeta=0`) the entire additive term vanishes and only the `tilde O(sqrt(d^3 H^3 T))` square-root regret remains, while the degradation under misspecification is explicit and only linear in the horizon.
