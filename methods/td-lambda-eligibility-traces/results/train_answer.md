We want the value function $v_\pi(s) = \mathbb{E}_\pi[G_t \mid S_t = s]$ for an agent walking through states under a fixed policy and collecting rewards, and we want to learn it from a raw stream of experience. The two tools that actually work sit at opposite ends of a spectrum, and each disappoints for the opposite reason. One-step temporal-difference learning forms the error $\delta_t = R_{t+1} + \gamma\,\hat v(S_{t+1}) - \hat v(S_t)$ from a single transition and nudges $\hat v(S_t)$ toward closing it; it is online, $O(d)$, and low-variance because it leans on its own estimate of the next state rather than the noisy far future — but for exactly that reason it moves information only one step per visit, so a reward buried twenty states down a trajectory takes the better part of twenty episodes to crawl back to the state that set it up. Monte Carlo waits for the episode to end, computes the true return $G_t$, and steps every visited state toward it, $w \leftarrow w + \alpha[G_t - \hat v(S_t)]\nabla\hat v$; credit floods all the way back in one episode and the target is unbiased, but it is available only at termination, it is a single high-variance sample of the future, and on a non-terminating task it never fires at all. The natural compromise is the $n$-step return $G_{t:t+n} = R_{t+1} + \gamma R_{t+2} + \dots + \gamma^{n-1}R_{t+n} + \gamma^n \hat v(S_{t+n})$, which bootstraps after $n$ real rewards; it is licensed by the error-reduction property $\max_s |\mathbb{E}_\pi[G_{t:t+n}\mid S_t=s] - v_\pi(s)| \le \gamma^n \max_s |\hat v(s) - v_\pi(s)|$, so each extra real reward shaves the worst-case error by another factor of $\gamma$ and every $n$-step return is a sound contraction target. But the $n$-step return drags back precisely the costs TD was free of: I must wait $n$ steps to even see $R_{t+n}$, I make no update for the first $n-1$ steps of every episode, I must physically store the last $n$ feature vectors, and worst of all I must commit to a single integer $n$, a brittle task-specific guess that throws away every other horizon. The goal, stated sharply, is one mechanism with one knob in $[0,1]$ that slides smoothly from one-step TD to Monte Carlo, runs online and causally, and uses memory and per-step compute that do not grow with how far back credit must travel.

The method is TD($\lambda$). The error-reduction property gives more room than a single $n$ uses: any convex average of $n$-step returns, with non-negative weights summing to one, is again a contraction target, since the bound becomes the weighted average of the per-component bounds and is no worse than the worst of them. So rather than bet on one horizon I take a weighted vote over all of them, and the question is only which weights. I choose a geometric weighting, giving the $n$-step return a weight proportional to $\lambda^{n-1}$, for two reasons that no other shape supplies at once: geometric weighting is scale-free, so no special horizon length is baked in — each extra step of lookahead simply multiplies the weight by the same constant $\lambda$ — and geometric is exactly the shape that telescopes and accumulates, which is the property I will need to sum over all horizons without storing all of them. Normalizing $\sum_{n=1}^\infty c\,\lambda^{n-1} = c/(1-\lambda) = 1$ gives $c = 1-\lambda$ and defines the $\lambda$-return
$$G^\lambda_t = (1-\lambda)\sum_{n=1}^\infty \lambda^{n-1} G_{t:t+n} = (1-\lambda)\sum_{n=1}^{T-t-1}\lambda^{n-1}G_{t:t+n} + \lambda^{T-t-1}G_t,$$
where in an episode terminating at $T$ all returns with $n \ge T-t$ equal the full return $G_t$ and have been peeled out as the residual tail. The one-step return gets the largest slice $1-\lambda$, the two-step gets $(1-\lambda)\lambda$, and so on. The two limits are the whole point of the dial: at $\lambda=0$ only the $n=1$ term survives and $G^\lambda_t$ is the one-step TD target; at $\lambda=1$ the main sum's coefficient $1-\lambda$ vanishes and the residual $\lambda^{T-t-1}G_t = G_t$ survives, the Monte Carlo target. One parameter, the full continuum. The forward-view algorithm is then to step each visited state toward its $\lambda$-return, $w_{t+1} = w_t + \alpha[G^\lambda_t - \hat v(S_t,w_t)]\nabla\hat v(S_t,w_t)$, with the convenient recursion $G^\lambda_t = R_{t+1} + \gamma[(1-\lambda)\hat v(S_{t+1}) + \lambda G^\lambda_{t+1}]$.

This target is exactly what I want and yet as a literal algorithm it is no better than Monte Carlo, because $G^\lambda_t$ depends on rewards arbitrarily far ahead — it is acausal, never fully known in a continuing task, and to compute it I would store everything. The escape is the asymmetry I keep underusing: at time $t$ I cannot see the future, but I can see the past, and the current TD error $\delta_t$ is relevant to every state I have already visited, in proportion to how recently I visited it. So instead of standing at $S_t$ and reaching forward, I stand at the present, compute the one cheap quantity I always have, $\delta_t$, and scatter it backward onto recently visited states weighted by a fading eligibility. Concretely I carry a short-term trace vector $z$ of the same shape as $w$, bumped by the value-gradient at each visited state and otherwise decayed, and apply the current error through it: $z_t = (\text{decay})\,z_{t-1} + \nabla\hat v(S_t)$, $w_{t+1} = w_t + \alpha\,\delta_t z_t$. This is causal, online, $O(d)$, and carries no $n$-step buffer — but it is worth nothing unless it computes the same thing as the forward $\lambda$-return, which pins down both the decay rate and the obligation to prove the equivalence.

I find the decay rate by unrolling the $\lambda$-return into the language the backward view speaks, namely TD errors, holding $\hat v$ fixed across the episode so every $\delta$ is defined against one value function. Expanding $G^\lambda_t - V(S_t)$ row by row gives one $n$-step return per row with weight $(1-\lambda)\lambda^{n-1}$; reading instead by columns, the reward $R_{t+1}$ appears in every row and its column collapses to weight $\sum_{n\ge1}(1-\lambda)\lambda^{n-1}=1$. The value term $\gamma V(S_{t+1})$ appears only in the $n=1$ row with weight $1-\lambda$; writing $(1-\lambda)\gamma V(S_{t+1}) = \gamma V(S_{t+1}) - \gamma\lambda V(S_{t+1})$, I keep a full $\gamma V(S_{t+1})$ to complete the first column into the clean TD error $\gamma^0[R_{t+1} + \gamma V(S_{t+1}) - V(S_t)] = \delta_t$ (using the $-V(S_t)$ out front), and I carry the leftover $-\gamma\lambda V(S_{t+1})$ down into the $R_{t+2}$ column, where it supplies exactly the missing value term that turns that column into $(\gamma\lambda)\delta_{t+1}$. The pattern repeats identically and shifted, so the whole error telescopes,
$$G^\lambda_t - V(S_t) = \sum_{k=t}^{T-1}(\gamma\lambda)^{k-t}\,\delta_k.$$
The weight on the error $k-t$ steps ahead is $(\gamma\lambda)^{k-t}$, a product of two factors that are each forced: the $\gamma$ because $\delta_k$ carries a reward living $k-t$ steps deeper in the return and thus discounted by $\gamma^{k-t}$, and the $\lambda$ because that is the horizon weighting of the $\lambda$-return, fading by $\lambda$ per extra step of lookahead. Neither alone is correct; the unrolling forces their product, so the trace must decay by $\gamma\lambda$ — not $\lambda$, not $\gamma$. This gives the accumulating-trace algorithm $z_t = \gamma\lambda\,z_{t-1} + \nabla\hat v(S_t,w_t)$, $\delta_t = R_{t+1} + \gamma\hat v(S_{t+1},w_t) - \hat v(S_t,w_t)$, $w_{t+1} = w_t + \alpha\,\delta_t z_t$, with the tabular trace $e_t(s) = \gamma\lambda\,e_{t-1}(s) + \mathbf{1}_{s=S_t}$; at $\lambda=0$ the trace is just $\nabla\hat v(S_t)$ with no memory and the update collapses to TD(0), which is why the family deserves the name TD($\lambda$).

The equivalence is then an exact identity in the offline case. The accumulating trace unrolled explicitly is $e_t(s) = \sum_{k=0}^t (\gamma\lambda)^{t-k}\mathbf{1}_{s=S_k}$, so the total backward change to state $s$ over the episode is $\sum_{j=0}^{T-1}\alpha\,\delta_j\sum_{i=0}^j(\gamma\lambda)^{j-i}\mathbf{1}_{s=S_i}$, a double sum over the triangle $i \le j$ of visit time $i$ and later error time $j$. Reading the same triangle by visits instead of by errors gives $\sum_{t=0}^{T-1}\alpha\,\mathbf{1}_{s=S_t}\sum_{k=t}^{T-1}(\gamma\lambda)^{k-t}\delta_k$. The total forward change to $s$ is $\sum_{t=0}^{T-1}\alpha[G^\lambda_t - V(S_t)]\mathbf{1}_{s=S_t}$, and substituting the column-collapse $G^\lambda_t - V(S_t) = \sum_{k=t}^{T-1}(\gamma\lambda)^{k-t}\delta_k$ yields the identical expression, term for term, for every state $s$. The one constraint is that $\hat v$ was held fixed through the episode, which is exactly the offline case where all updates are accumulated and applied at the end; applied online, $\hat v$ drifts between steps, the $\delta$'s no longer align against one value function, and the equality becomes an approximation that is tight only as $\alpha \to 0$.

That drift is also the thread to an exact online version, and it reveals that eligibility traces are not about TD at all. Strip TD out entirely: linear Monte Carlo / LMS, $w_{t+1} = w_t + \alpha[G - w_t^\top x_t]x_t$ with a single terminal $G$ and no discount, rewrites one step as $w_{t+1} = (I - \alpha x_t x_t^\top)w_t + \alpha G x_t$. Naming the fading matrix $F_t = I - \alpha x_t x_t^\top$ and recursing from $w_0$ gives $w_T = a_{T-1} + \alpha G\,z_{T-1}$ with $z_t = F_t z_{t-1} + x_t = z_{t-1} + (1 - \alpha z_{t-1}^\top x_t)x_t$ built incrementally from $z_{-1}=0$ with no stored history and no knowledge of $G$. So even pure Monte Carlo reorganizes into an $O(d)$ eligibility trace that reproduces the exact end-of-episode update, and the trace it yields is not the simple accumulating one — it carries the extra correction $(1 - \alpha z^\top x)$ that accounts for how much the current feature projects onto the existing trace. Folding that correction together with the $\gamma\lambda$ temporal fading gives the self-correcting dutch trace $z_t = \gamma\lambda\,z_{t-1} + (1 - \alpha\gamma\lambda\,z_{t-1}^\top x_t)x_t$ paired with a weight update $w_{t+1} = w_t + \alpha\delta_t z_t + \alpha(w_t^\top x_t - w_{t-1}^\top x_t)(z_t - x_t)$ that also subtracts the bootstrap drift, and this true online TD($\lambda$) reproduces the strict online $\lambda$-return target exactly in the linear case, with the same $O(d)$ memory and asymptotic cost as the plain version plus one extra inner product. The plain accumulating-trace TD($\lambda$) is the offline-exact, online-approximate workhorse; the dutch-trace form closes the online gap.

```python
import numpy as np

class TDLambda:
    """Accumulating-trace TD(λ): offline-exact, online-approximate forward equivalence."""
    def __init__(self, d, gamma, lam, alpha):
        self.w = np.zeros(d); self.gamma, self.lam, self.alpha = gamma, lam, alpha
    def reset_episode(self):
        self.z = np.zeros_like(self.w)
    def step(self, x, r, x_next, terminal):
        v      = self.w @ x
        v_next = 0.0 if terminal else self.w @ x_next
        delta  = r + self.gamma * v_next - v               # δ_t  (TD error)
        self.z = self.gamma * self.lam * self.z + x        # z_t = γλ z + ∇v̂
        self.w += self.alpha * delta * self.z              # w += α δ_t z_t

def lambda_return(rewards, values, gamma, lam):
    """Forward λ-return targets (values[T]=0), with weights fixed for the episode."""
    T = len(rewards); G = np.zeros(T); G[T-1] = rewards[T-1]
    for t in range(T-2, -1, -1):
        G[t] = rewards[t] + gamma * ((1-lam)*values[t+1] + lam*G[t+1])
    return G

class TrueOnlineTDLambda:
    """Dutch-trace TD(λ): exact for the strict online λ-return view, O(d)/step."""
    def __init__(self, d, gamma, lam, alpha):
        self.w = np.zeros(d); self.gamma, self.lam, self.alpha = gamma, lam, alpha
    def reset_episode(self, x):
        self.z = np.zeros_like(self.w); self.v_old = 0.0; self.x = x
    def step(self, r, x_next, terminal):
        x = self.x; v = self.w @ x
        v_next = 0.0 if terminal else self.w @ x_next
        delta  = r + self.gamma * v_next - v
        gl     = self.gamma * self.lam
        self.z = gl * self.z + (1 - self.alpha * gl * (self.z @ x)) * x   # dutch trace
        self.w += self.alpha * (delta + v - self.v_old) * self.z
        self.w -= self.alpha * (v - self.v_old) * x
        self.v_old = v_next; self.x = x_next
```
