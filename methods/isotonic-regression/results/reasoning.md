Let me start from what actually goes wrong, because the failure is specific and I want to see exactly what structure I have to work with. I have a classifier already trained — naive Bayes, or an SVM, doesn't matter — and for each example it spits out a score s(x). When I bin these scores and plot, for each bin, the score against the fraction of examples in that bin that are actually positive, the points don't sit on the diagonal. With naive Bayes they bow away from it: scores crowd toward 0 and 1 because the independence assumption multiplies correlated evidence as if it were independent and over-counts, so a posterior of 0.97 might really correspond to an empirical rate of 0.8. With an SVM it's worse in a sense — the "score" is a signed distance to the hyperplane, it isn't even trying to be a probability, I have to squash it into [0,1] by s = f/(2a) + 1/2 just to get a number in range, and even then the empirical-rate-versus-score curve is some S-shape, not the line y=x. So the model is miscalibrated. But — and this is the thing I have to hold onto — in both cases the points still march upward. Higher score, higher empirical positive rate, monotonically. The classifier ranks correctly; it just reports the wrong numbers. So I don't need to fix the ordering. I need to fix the vertical axis: find a map from the score it gives me to the probability it should have given me.

What do I have to learn this map from? A held-out calibration set: examples where I know the true label. For a positive example the true membership probability is 1, for a negative it's 0. So if I run the classifier on the calibration set I get pairs (s(x_i), g_i) with g_i in {0,1}, and I want a function m such that m(s(x_i)) ≈ g_i — but obviously not exactly, because a single example's label is a 0/1 draw and what I'm after is the *probability*, the expected label at that score. So really I want m(s) ≈ P(c | s(x)=s), the empirical positive rate among examples near score s. The naive thing is to read that off directly: for each distinct score value, average the labels of examples with that score. The reason that fails is brutal and practical — in real data the number of distinct scores is huge compared to the number of labeled examples, so at almost every score I have one or two examples, and the "empirical rate" is just that example's 0 or 1. No signal. I have to aggregate examples with nearby scores to get a stable estimate. The whole game is in *how* I aggregate.

Two ways to aggregate are already on the table, and both have a flaw I can name precisely. One is to commit to a shape: fit a sigmoid, p̂ = 1/(1+exp(A·s+B)), pick A and B by maximizing the calibration-label likelihood. Two parameters, beautifully regularized, and when the true distortion really is a sigmoid — which it often is for SVM margins — it nails it. But it's *exactly* two parameters of *exactly* one shape. If the true score-to-probability curve is some other monotone thing, and naive-Bayes curves can be, the sigmoid simply can't bend that way, and whatever it can't represent stays as residual miscalibration. The other way is binning: sort by score, chop into b equal-count bins, and a test example gets its bin's positive fraction. Shape-free, which fixes the sigmoid's rigidity. But now the bin boundaries are arbitrary — they fall wherever equal counts put them, not where the score actually changes meaning. If a boundary lands in the middle of a region where the true probability is climbing steeply, that bin averages together examples that clearly deserve different probabilities and flattens them to one number. And b has to be cross-validated, which is hopeless on a small or unbalanced calibration set. So: parametric is too rigid (high bias if the shape is wrong), free-form binning is too arbitrary (the bins ignore the data's structure). I want something between them.

What do I actually believe about the true map, with confidence, independent of any particular dataset? I believe it's *non-decreasing*. That belief is not a guess — it's the one thing the diagnostic plots tell me is robustly true: the classifier ranks well, so a higher score means a genuinely higher (or at least not lower) probability. That's it. That's the whole prior. So instead of committing to a sigmoid (one rigid shape) or to nothing (arbitrary bins), let me restrict the map to the class of *all* non-decreasing functions and ask for the best fit to the calibration labels within that class. Monotonicity is weaker than a fixed parametric form — it lets the curve bend however the data wants, sigmoidal or not — but it's strong enough to be a real regularizer, because it forbids the wiggling that pure point-estimation or arbitrary bins would do. This is a regression problem with an order constraint. Let me write it down.

Sort the calibration pairs by score, s_1 ≤ s_2 ≤ ... ≤ s_n, carrying along the binary targets g_1, ..., g_n. I want fitted values ĝ_1, ..., ĝ_n, one per example, that are non-decreasing and close to the labels. "Close" in what sense? Let me try squared error first, with possible weights w_i (think w_i = 1 for now, but I'll keep them because tied scores will want to be merged with weight = count):

  minimize  Σ_i w_i (g_i − ĝ_i)²   subject to   ĝ_1 ≤ ĝ_2 ≤ ... ≤ ĝ_n.

Before I worry about solving it, let me check the objective actually produces a *probability*, because I'm fitting to 0/1 targets and I'd be uneasy if the answer came out as some arbitrary real. Drop the constraint for a second and ask: what single constant value c minimizes Σ_i w_i (g_i − c)² over a set of points? Differentiate: −2 Σ w_i (g_i − c) = 0, so c = (Σ w_i g_i)/(Σ w_i), the weighted mean of the g's. With 0/1 targets the weighted mean is exactly the (weighted) fraction of positives — the empirical probability. So whenever the optimal monotone fit is *forced to be constant* on some group of examples (and the constraint will force exactly that, on groups where the raw labels disagree with monotonicity), the value it takes there is the empirical positive rate of that group, which lives in [0,1] and is precisely the calibrated probability I wanted. The squared-error objective isn't an arbitrary choice of metric — it's the choice that makes each constant piece come out as an honest probability.

Actually, let me push on that, because I want to know whether I'm wedded to squared error or whether log-loss — which is what I'd naturally use to score probabilities — would give a different fit. Suppose I'd written the objective with cross-entropy instead, Σ w_i [−g_i log ĝ_i − (1−g_i) log(1−ĝ_i)]. What constant c minimizes Σ w_i [−g_i log c − (1−g_i) log(1−c)]? Differentiate: −Σ w_i g_i/c + Σ w_i (1−g_i)/(1−c) = 0, which solves to c = (Σ w_i g_i)/(Σ w_i) again — the same weighted mean. That's not a coincidence. Squared error and log-loss are both Bregman losses for the mean, and in this order-restricted mean problem replacing squared error by any such Bregman loss leaves the fitted vector unchanged. I can see the reason at the block level: once the order constraints force a constant block, every consistent loss chooses the block mean, and the isotonic solution is exactly the ordered collection of those block means. That's reassuring: I can solve the squared-error problem because it has the cleanest quadratic form, and the same constrained calibration would be obtained under log-loss.

Now, how do I actually solve the order-constrained least squares? It's a convex quadratic with linear inequality constraints — a quadratic program — so a generic QP solver would do it, but n is the size of my calibration set and I'd like something that doesn't choke. Let me think about the structure of the solution before reaching for a solver, because the structure might hand me an algorithm. Suppose I have the optimal ĝ. Look at any maximal run of consecutive indices where ĝ is constant — call it a block. Within a block the constraint ĝ_i ≤ ĝ_{i+1} holds with equality (that's what "constant" means), so it's *active*; between blocks, ĝ strictly increases, so the constraint there is slack. On a block where the constraint is slack at both ends — i.e., the block can move freely without bumping its neighbors — the optimum must set its value to the unconstrained minimizer over that block, which I just computed is the weighted mean of the g's in the block. So the solution is piecewise constant, and on each piece it equals the weighted average of the raw targets in that piece. The entire problem reduces to: *find the right partition into blocks.* The values are then forced.

Which partition? Let me get a closed form so I know what I'm aiming at, then find a fast way to compute it. Stare at a single fitted value ĝ_i. Intuitively it's the average of g over the block that eventually contains i, but the block boundaries are not known yet. The order-restricted least-squares solution has an exact envelope form: for each right endpoint ℓ at or to the right of i, look backward over every start k ≤ ℓ and take the largest weighted average ending at ℓ; then take the smallest of those upper envelopes over ℓ ≥ i. That gives

  ĝ_i = min_{ℓ≥i}  max_{k≤ℓ}  ( Σ_{j=k}^{ℓ} w_j g_j ) / ( Σ_{j=k}^{ℓ} w_j ).

That's a complete characterization, but computing it literally is brutal — for each i a double loop over k and ℓ, O(n²) or worse, and that's the "minimum lower sets" style approach, correct but slow. There has to be a streaming algorithm, because the structure is so local: the only thing that ever needs fixing is an *adjacent* pair that's out of order, and fixing it is just averaging.

Let me try the obvious local algorithm and see if it lands on the same optimum. Walk left to right through the sorted examples, treating each as its own block with value g_i. Whenever I find an adjacent pair of blocks that violates monotonicity — the left block's value v_L exceeds the right block's value v_R — look only at the compressed two-block problem. Its unconstrained minimizer is (v_L, v_R), but that point violates z_L ≤ z_R. The weighted squared-error projection onto the halfspace z_L ≤ z_R must lie on the boundary z_L = z_R, and the best boundary value is the weighted average (w_L·v_L + w_R·v_R)/(w_L + w_R). So *pool* them: replace the two blocks by one block with that value and combined weight w_L + w_R. That's the pair-adjacent-violators move: if there's a violation, pool. Now here's the subtlety — after pooling, the new block's value might be *smaller* than the block to its *left*, which was fine before. So pooling can create a fresh upstream violation. I can't just march forward; I have to turn around and check the left neighbor, and pool again if needed, and keep back-merging until the left end is monotone, before I resume marching right.

Does this back-merging local procedure actually produce the global least-squares optimum, or just some monotone fit? I need to convince myself, not hand-wave. The clean way is to argue by prefixes. Suppose after processing the first t sorted points, the stack of active blocks is already the exact isotonic least-squares fit for that prefix, each block stored by its total weight and weighted target sum. Now I append point t+1 as a singleton block. If the previous last block value is ≤ the new value, the old prefix optimum plus this singleton is feasible and no old constraint has changed, so it is the prefix optimum for t+1. If the previous last value is larger, the only broken constraint is between the last two active blocks, and the two-block projection calculation says the constrained optimum for those two blocks is their pooled weighted mean. I merge them. If that pooled value now breaks the boundary to its left, the same argument applies again, so I keep back-merging until the last boundary is ordered. At that moment the stack is again the exact fit for the longer prefix. Induction carries this to t = n, so the local pool-and-back-merge procedure computes the same unique solution as the min-max formula. The implementation can keep the active blocks on a stack; each original point is pushed once and block merges pop from the end, so the pooling pass is O(n) after the initial score sort — O(n log n) including the sort.

Let me run a tiny binary instance in my head to make sure I have the mechanics right, taking the targets already sorted by score as 1, 0, 1, 0, 0, 0, 1 with unit weights. Start: blocks (1),(0),(1),(0),(0),(0),(1). First adjacent pair 1 then 0 — violation, pool to 1/2: blocks (0.5),(1),(0),(0),(0),(1). Is 0.5 ≤ 1? yes, no upstream problem. Next pair 1 then 0 — violation, pool to 0.5: blocks (0.5),(0.5),(0),(0),(1). Equality is fine between the first two blocks, but 0.5 then 0 violates, so pool the second 0.5 block with the next 0: its sum is 1 over weight 3, value 1/3. Now back-check: the left neighbor is 0.5, and 0.5 > 1/3, so pool those blocks too. The combined sum is 2 over weight 5, value 0.4. The next target is 0, so 0.4 > 0; pool again, sum 2 over weight 6, value 1/3. Now 1/3 ≤ 1, so the final blocks are {1,0,1,0,0,0} with value 1/3 and {1} with value 1. The fit is 1/3,1/3,1/3,1/3,1/3,1/3,1. That is exactly the kind of answer I want for calibration: where the sorted labels bounce up and down, the algorithm admits that the score has not separated those examples and returns their empirical positive rate; where the final positive is cleanly to the right, it can stay separate.

That last observation is worth pausing on, because it tells me precisely how this beats binning. Binning chops into bins of *fixed* size at *arbitrary* boundaries. This procedure also returns a set of intervals with a constant probability estimate on each — so it *is* a binning, but the boundaries and the bin sizes are chosen by the *data*: it averages over many examples in regions where the classifier ranks incorrectly (lots of violations to pool away) and over few examples where the classifier ranks correctly (no violations, blocks stay small). The granularity adapts to how trustworthy the ranking is locally. That's the arbitrariness of binning fixed, and I didn't have to cross-validate a bin count — the number of blocks falls out of the data. And it's more flexible than the sigmoid because it never assumed a shape; it'll fit whatever monotone curve the calibration data implies. Between the two extremes, exactly where I wanted to be.

Now I have to turn "a set of intervals with a value each" into an actual function I can apply to a *new* score that wasn't in the calibration set. Before fitting, tied calibration scores should be aggregated: one threshold, target equal to the weighted average of the tied labels, weight equal to the summed weight. After fitting, I have sorted thresholds and fitted values. The interval rule would be to find the interval containing a new score and return that block's value; the standard implementation stores the non-redundant thresholds and uses piecewise-linear interpolation between adjacent threshold values, which gives a continuous non-decreasing map for unseen scores. Runs of equal fitted values between distinct thresholds are redundant for this interpolator, so I keep only the threshold points where the value changes, plus the endpoints. Faster to apply, same interpolated function.

What about a test score outside the calibration range — below the smallest training score or above the largest? The regression simply has no information out there. The safe, monotone choice is to clamp the *input* to the fitted domain: a score below the minimum threshold is evaluated at the minimum threshold, and a score above the maximum threshold is evaluated at the maximum threshold. That keeps the map monotone and bounded. Since these are probabilities, I also set lower and upper fitted-value bounds at 0 and 1; the block averages are already in [0,1] for 0/1 targets, and interpolation between bounded threshold values stays in [0,1]. So: clip out-of-bounds inputs to the nearest score endpoint, and bound the fitted values to [0,1].

That handles one binary score column completely: fit the isotonic regression by pooling adjacent violators, store the deduplicated thresholds and values, and at predict time interpolate-with-clip. Let me now confront the multiclass case, because the calibration task has digit and text problems as well as the binary ones.

The trap is to try to calibrate the whole thing at once. The classifier on a k-class problem outputs a point in the (k−1)-dimensional simplex, and I'd be learning a correction from that simplex to itself. But the monotonicity trick that made the binary case work was *one-dimensional* — "higher score, higher probability" only makes sense along a single axis. In (k−1) dimensions there's no analogous order to impose, no natural shape, and a non-parametric fit in that many dimensions needs far more data than I have as k grows. So I won't calibrate the joint object. Reduce it to one dimension per class. For each class c, define a binary problem: an example is "positive" if its true label is c, "negative" otherwise, and the feature is the classifier's score for class c on that example. That's a one-against-all decomposition, and now each class's calibration is exactly the binary problem I already solved — fit an isotonic regression of (is-label-c) on (score-for-class-c). I do this independently for all k classes.

After that I have, for each class c and example x, a calibrated estimate r_c(x) of P(c | x) — but they were fit independently, so across classes they won't sum to one; they're not yet a distribution. I need to combine them. There are heavy machineries for this — least-squares with non-negativity constraints, or iterative log-loss "coupling" — and those are the right tools for general code matrices where the relation between binary problems and classes is complicated. But for one-against-all the situation is much simpler: each binary classifier's output is *already* an estimate of P(c | x) directly, not some indirect combination. So the obvious combiner is just to renormalize: divide each by the sum.

  p̂(c | x) = r_c(x) / Σ_{c'} r_{c'}(x).

Simplest possible thing, and for one-against-all it's the natural one, because I'm reconciling k estimates of the k class probabilities into a distribution. One numerical guard: if all the r_c(x) for some example came out as zero (every per-class isotonic map sent that example's score to 0), the division is 0/0. So floor each r_c at a tiny positive value before normalizing — that guarantees a valid, non-degenerate distribution. I'll clip to something like 1e-15 and then divide by the row sum.

I should be honest with myself about when this whole approach is the wrong tool, because it has a cost. The isotonic map is non-parametric — it has as many effective degrees of freedom as the data has blocks, far more than the sigmoid's two — so it needs *more* calibration data to not overfit. On a tiny or badly imbalanced calibration set, the blocks are estimated from few examples each and the map can chase noise; the sigmoid's rigidity, which is a weakness when the shape is wrong, is a strength when data is scarce. For the task here the calibration sets are large enough that the non-parametric flexibility is the right trade; I'd want to keep that caveat in mind if the calibration set were small.

Let me also be precise about ties and weights so the implementation is exact. When several calibration examples share the same score, they belong at the same point on the x-axis, so I aggregate them first: replace the duplicates by a single point whose target is the weighted average of their targets and whose weight is the sum of theirs. That's the same single-constant minimizer logic — a group at one score must get one fitted value, and the best one is their average — and it both makes the sort-and-pool well-defined at ties and is faster. Then I run pool-adjacent-violators on the deduplicated, sorted points with those weights.

Now let me write the code I'd actually ship, filling the empty slots in the calibrator harness: the one-dimensional map is the isotonic regression (pool-adjacent-violators, stored as thresholds + values, applied by clipped linear interpolation), and the multiclass reconciliation is floor-and-normalize. I'll lean on the standard library routine for the isotonic regression itself rather than re-implementing the O(n) pooling by hand, since that routine sorts by score, averages duplicate scores with weights, runs the pool-adjacent-violators fit, trims redundant thresholds, predicts by linear interpolation, and clips out-of-domain test scores when asked.

```python
import numpy as np


class CalibrationMethod:
    """Isotonic-regression calibration. Per binary score column, fit a monotone
    non-decreasing map score -> probability by pool-adjacent-violators (the L2-optimal
    monotone fit, whose block values are the empirical positive rates). For multiclass,
    do this one-against-all per class and renormalize into a distribution."""

    def __init__(self):
        self.is_binary = None
        self.state_ = None
        self.eps = 1e-15

    def _new_binary_map(self):
        # IsotonicRegression here IS the pool-adjacent-violators fit:
        #   minimize sum_i (g_i - ghat_i)^2  s.t.  ghat non-decreasing in the score,
        # block value = weighted mean of the 0/1 targets = empirical P(positive | score).
        # The class sorts by score, averages duplicate scores with weights, trims redundant
        # thresholds, predicts by linear interpolation, and with out_of_bounds="clip" clamps
        # new scores to the fitted score range. y_min/y_max bound fitted values to [0, 1].
        from sklearn.isotonic import IsotonicRegression as IR
        return IR(out_of_bounds="clip", y_min=0.0, y_max=1.0, increasing=True)

    def _fit_map(self, probs, labels):
        if probs.ndim == 1:                      # binary: one positive-class score column
            iso = self._new_binary_map()
            iso.fit(probs, labels.astype(float)) # fit g_i = 1{positive} on the score
            return [iso]

        maps = []                                # multiclass: one-against-all per class
        for c in range(probs.shape[1]):
            binary_labels = (labels == c).astype(float)
            iso = self._new_binary_map()
            iso.fit(probs[:, c], binary_labels)  # calibrate class c's score
            maps.append(iso)
        return maps

    def _apply_map(self, fitted_map, probs):
        if self.is_binary:
            calibrated = fitted_map[0].predict(probs)
            return np.clip(calibrated, 0.0, 1.0)

        calibrated = np.zeros_like(probs, dtype=float)
        for c, iso in enumerate(fitted_map):
            calibrated[:, c] = iso.predict(probs[:, c])       # per-class r_c(x)
        calibrated = np.clip(calibrated, self.eps, None)      # avoid a 0/0 all-zero row
        return calibrated / calibrated.sum(axis=1, keepdims=True)

    def fit(self, probs, labels):
        probs = np.asarray(probs, dtype=float)
        labels = np.asarray(labels)
        self.is_binary = probs.ndim == 1
        self.state_ = self._fit_map(probs, labels)
        return self

    def predict_proba(self, probs):
        probs = np.asarray(probs, dtype=float)
        return self._apply_map(self.state_, probs)
```

Let me trace the whole causal chain once, because I want to be sure each piece was forced and not chosen for taste. The classifier ranks well but its scores aren't calibrated — the reliability plot bows off the diagonal while staying monotone — so I only need to fix the vertical scale, learning a map from score to probability on a held-out labeled set. Reading the empirical rate off each distinct score fails because scores are sparse, so I must aggregate; the question is how. A sigmoid is too rigid (one shape, high bias when the true distortion isn't sigmoidal), and equal-count binning is too arbitrary (boundaries and sizes ignore the data, and the bin count needs cross-validation). The one robust thing I believe is monotonicity — higher score, higher probability — so I restrict the map to all non-decreasing functions and fit by least squares with an order constraint. Squared error makes each constant piece come out as the empirical positive rate, an honest probability in [0,1]; and because squared error and log-loss are Bregman losses for the mean, the same constrained fit is obtained under either loss. The optimum is piecewise constant with each block equal to its weighted average, characterized by the min-max formula; computing that directly is slow, but the active-block pool-adjacent-violators move solves the same constrained problem by pooling each adjacent violation into its weighted mean and back-merging until all active block values are ordered, in O(n) after sorting. The resulting intervals are a binning whose boundaries and sizes are chosen by the data (coarse where the ranking is poor, fine where it's good), which is exactly binning's arbitrariness repaired. I turn the fitted thresholds into an applicable map by averaging tied scores before fitting, trimming redundant thresholds after fitting, interpolating linearly between thresholds, clipping out-of-range inputs to the fitted score endpoints, and bounding fitted values to [0,1]. For multiclass I refuse to calibrate the joint simplex (no order, curse of dimensionality) and instead reduce one-against-all — calibrate each class's score against its is-this-class binary target — then, because each one-against-all output already estimates P(c|x), reconcile them into a distribution by flooring at a tiny value and renormalizing. The cost is that this non-parametric map needs more calibration data than the two-parameter sigmoid to avoid overfitting, so it's the right tool when the calibration set is large.
