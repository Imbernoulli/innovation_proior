What started me on this was an empirical fact I could not explain: agents trained by pure self-play — repeatedly playing their current policies against copies of *themselves*, with no human expert and no fixed strong opponent — reach super-human strength at GO, Starcraft, and Dota. That cuts against the prior intuition that you need a strong opponent to learn a strong policy. I want to know whether self-play is *provably* sound and *provably* sample-efficient: starting from no knowledge of the transitions or rewards, and exploring the game by playing it for a bounded number of episodes, how many episodes does it take to find an approximate Nash equilibrium, with no structural assumption on the game and no access to a simulator? The clean model is the tabular episodic two-player zero-sum Markov game $\mathrm{MG}(H,\mathcal S,\mathcal A,\mathcal B,\mathbb P,r)$: a shared state evolves, the max-player picks $a\in\mathcal A$ and the min-player picks $b\in\mathcal B$ simultaneously at each of $H$ steps, a single reward $r_h(s,a,b)\in[0,1]$ is paid to the max-player and against the min-player, and the solution concept is the Nash equilibrium $(\mu^\star,\nu^\star)$ at which neither player can profitably deviate.

The existing options each fall short on exactly the part that makes this hard. The single-agent optimism template — UCBVI and UCB-Q-learning — solves exploration beautifully with $\tilde O(\sqrt T)$ regret, but it is built for one agent: "greedy" is well-defined only because the per-state object is a *vector*, and a near-optimal value estimate automatically gives a near-optimal greedy policy. Neither stays true with a second player. The classical Markov-game algorithms — Nash-Q, minimax-Q — set the state value to the per-state Nash/minimax value, but they are asymptotic or assume the data already covers the state space, with no exploration bonus and no finite-sample guarantee. The non-asymptotic game results buy their bounds by assuming away the exploration problem: a global reachability assumption, or a simulator that returns a sample for any queried $(s,a,b)$, under which uniform querying already explores. And R-max, which needs no such assumption, bounds the deployed pair against the *Nash value* rather than against the per-episode best response — a strictly weaker statement that does not give a $\sqrt T$ self-play regret. The obvious decoupled idea, running two no-regret single-agent learners against each other and averaging, demands that each learner be no-regret against an *adversarial*, possibly non-Markovian opponent; whether that can be done efficiently is exactly the thing to probe. So nothing on the table simultaneously explores without help, scores against best responses, and outputs deployable policies.

The first thing I have to fix is what we even measure. In a game the lazy choice — compare the deployed pair $(\mu^k,\nu^k)$ to the Nash value $V^\star$ — misses the point, because I want each player's policy to be robust against *any* opponent. So I score each deployed policy against *its own best response*, giving the strong regret
$$\mathrm{Regret}(K)=\sum_{k=1}^K\Big[V^{\dagger,\nu^k}_1(s_1^k)-V^{\mu^k,\dagger}_1(s_1^k)\Big].$$
Because the minimax value $V^\star$ (Shapley's, holding at the level of the whole multi-step game, not von Neumann's matrix theorem) sits in between, each summand splits exactly into two non-negative pieces,
$$V^{\dagger,\nu^k}_1-V^{\mu^k,\dagger}_1=\underbrace{\big(V^{\dagger,\nu^k}_1-V^\star_1\big)}_{\nu^k\text{'s exploitability}}+\underbrace{\big(V^\star_1-V^{\mu^k,\dagger}_1\big)}_{\mu^k\text{'s exploitability}},$$
a duality gap. Driving this sublinear means both policies become unexploitable on average, and an online-to-batch sample turns it into an $\epsilon$-approximate Nash pair, $V^{\dagger,\hat\nu}_1-V^{\hat\mu,\dagger}_1\le\epsilon$ — good against best responses, not merely near in value. That phrase "against best responses" is the crux of everything that follows.

I propose lifting single-agent optimism to two players with **two-sided optimism**, realized first as VI-ULCB (value iteration with upper–lower confidence bounds) and then, after fixing its computational and statistical defects, as **optimistic Nash Q-learning** and **optimistic Nash V-learning**. The starting move is to maintain not one optimistic estimate but two: an upper $Q^{\mathrm{up}}$ and a lower $Q^{\mathrm{low}}$, each from a bonus-augmented Bellman backup, so $Q^{\mathrm{up}}$ over-estimates and $Q^{\mathrm{low}}$ under-estimates. The max-player wants the value high, so he is optimistic upward; the min-player wants it low, so she is optimistic downward. The immediate obstacle is that "greedy with respect to $Q^{\mathrm{up}}$" is meaningless for one player in isolation: at state $s$ his payoff is not a vector but a *matrix* $Q^{\mathrm{up}}(s,a,b)$ indexed by both actions, and he cannot pick a best $a$ without the distribution over $b$. What I actually want is for $(\mu,\nu)$ to be *jointly* greedy — a fixed point at this state and step — which is precisely a one-step matrix game. And here is the load-bearing subtlety: if I had a single matrix, the joint-greedy condition would be its zero-sum (von Neumann) equilibrium, an LP. But the max-player's payoff matrix is $Q^{\mathrm{up}}$ and the min-player's is $Q^{\mathrm{low}}$, and these are *not equal*. Two players, two different matrices, simultaneous moves: that is a *general-sum* matrix game, and the right joint-greedy notion is its Nash equilibrium,
$$(\mu_h(\cdot\mid s),\nu_h(\cdot\mid s))=\textsc{Nash\_General\_Sum}\big(Q^{\mathrm{up}}_h(s,\cdot,\cdot),\,Q^{\mathrm{low}}_h(s,\cdot,\cdot)\big),$$
with state values read off consistently, $V^{\mathrm{up/low}}_h(s)=\mu_h(s)^\top Q^{\mathrm{up/low}}_h(s,\cdot,\cdot)\nu_h(s)$. Computing the two policies *together*, by one routine that sees both estimates, is what "self-play" buys over "train against an opponent": I control both players and can coordinate them.

Why two estimates and not one band around $V^\star$ — this is where the two-sidedness proves load-bearing rather than decorative. To control the regret summand I want the sandwich $V^{\dagger,\nu^k}_1-V^{\mu^k,\dagger}_1\le V^{\mathrm{up},k}_1-V^{\mathrm{low},k}_1$, which requires the invariant that $Q^{\mathrm{up}}$ upper-bounds the value of the max-player's best response to the *currently deployed* $\nu^k$, and $Q^{\mathrm{low}}$ lower-bounds the min-player's best response to the deployed $\mu^k$ — optimism toward the opponent's best response to my own policy, *not* optimism toward the equilibrium. A mere band around $V^\star$ would not close the chain, because the per-episode best-response values can sit outside it. Proving the upper invariant by downward induction in $h$, the $Q$-level step writes
$$Q^{\mathrm{up},k}_h-\sup_\mu Q^{\mu,\nu^k}_h=\big[\widehat{\mathbb P}^k_h\big(V^{\mathrm{up},k}_{h+1}-\sup_\mu V^{\mu,\nu^k}_{h+1}\big)\big]-\big[(\widehat{\mathbb P}^k_h-\mathbb P_h)\sup_\mu V^{\mu,\nu^k}_{h+1}\big]+\beta_t,$$
where the first bracket is $\ge 0$ because $\widehat{\mathbb P}^k_h$ is stochastic and the inducted function is non-negative, and the second is $\ge-\beta_t$ precisely because the bonus is sized to dominate the empirical-minus-true deviation of a fixed function — so the sum is $\ge 0$. Lifting from $Q$ to $V$ is where the joint solve pays off: by construction $\mu^k_h(s)$ is the max-player's best response *inside the $Q^{\mathrm{up}}$ matrix* against $\nu^k_h(s)$, so $V^{\mathrm{up},k}_h(s)=\max_\phi\phi^\top Q^{\mathrm{up},k}_h\nu^k_h(s)\ge(\phi^\star)^\top Q^{\mathrm{up},k}_h\nu^k_h(s)\ge\sup_\mu V^{\mu,\nu^k}_h(s)$, using the entrywise $Q$-bound at the last step. The symmetric argument on the lower side gives the clean chain at every node,
$$Q^{\mathrm{up},k}_h\ge\sup_\mu Q^{\mu,\nu^k}_h\ge\inf_\nu Q^{\mu^k,\nu}_h\ge Q^{\mathrm{low},k}_h,$$
the middle inequality holding because a fixed pair sits between the two best responses.

The bonus is fixed by concentration, and getting its $S$-dependence right matters because it determines the rate. I need $|(\widehat{\mathbb P}^k_h-\mathbb P_h)V|\le\beta_t$ to hold *uniformly* over the value class, not just for a fixed $V$, because $V^{\mathrm{up}}_{h+1}$ is data-dependent. An $\ell_\infty$ $\epsilon$-net of $\{V:\mathcal S\to[0,H]\}$ has size $(1/\epsilon)^S$ since the class is $S$-dimensional; Hoeffding per net point, a union bound, and the choice $\epsilon=c\sqrt{H^2S\iota/K}$ collapse $S\log(1/\epsilon)$ to $O(S\iota)$, giving radius
$$\beta_t=c\sqrt{SH^2\iota/t},\qquad t=N^k_h(s,a,b),$$
with the extra $S$ inside the root coming straight from the $\epsilon$-net dimension. Telescoping $V^{\mathrm{up}}-V^{\mathrm{low}}$ down the played trajectory — each step costing $4\beta^k_h$ (two bonuses from the two backups, two more from swapping $\widehat{\mathbb P}$ for $\mathbb P$) plus martingale differences — then using $\sum_{t=1}^N\sqrt{1/t}=O(\sqrt N)$ and Cauchy–Schwarz $\sum_{s,a,b}\sqrt{N^K_h}\le\sqrt{SAB\cdot K}$, yields
$$\mathrm{Regret}(K)=\tilde O\!\left(\sqrt{H^3 S^2 AB\,T}\right),$$
with the two $S$'s (one from counting visits, one from the bonus's $\epsilon$-net) and the *product* $AB$ from solving a full $A\times B$ matrix game per state. Online-to-batch gives the PAC bound $K\ge\tilde O(H^4 S^2 AB/\epsilon^2)$ for an $\epsilon$-Nash pair.

This is sample-efficient but not yet polynomial-time, because finding a Nash equilibrium of a general-sum bimatrix game is PPAD-complete. Two routes recover polynomial runtime. The crude one, VI-Explore, separates exploration from exploitation: explore reward-free (for each target state $s$, set artificial reward $\tilde r=\mathbb 1[s'=s]$, treat the action pair $(a,b)$ as one big action, and run a single-agent PAC learner to maximize the reaching probability), then run value iteration on the single resulting empirical model, which needs only the *zero-sum* per-state solve — one matrix, von Neumann minimax, an LP. That is polynomial but pays $\tilde O(T^{2/3})$ regret. The better route re-examines what the proof actually used: only one-sided no-deviation inequalities, not product-form independent play. The relaxation that captures exactly those is a **coarse correlated equilibrium**, a joint distribution $\pi$ with $\mathbb E_\pi Q^{\mathrm{up}}\ge\max_{a^\star}\mathbb E_\pi Q^{\mathrm{up}}(a^\star,b)$ and $\mathbb E_\pi Q^{\mathrm{low}}\le\min_{b^\star}\mathbb E_\pi Q^{\mathrm{low}}(a,b^\star)$ — only $A+B$ linear constraints, so an LP, polynomial. A CCE always exists, and crucially it degrades correctly: feeding the same matrix twice, the marginals of $\textsc{CCE}(Q,Q)$ are a *zero-sum Nash* of $Q$, since the CCE chain $\max_{a^\star}\mathbb E_b Q\le\mathbb E_\pi Q\le\min_{b^\star}\mathbb E_a Q$ combined with the minimax chain $\min_{b^\star}\mathbb E_a Q\le N^\star\le\max_{a^\star}\mathbb E_b Q$ forces every term to be equal — the saddle condition. So as $Q^{\mathrm{up}}$ and $Q^{\mathrm{low}}$ converge, the CCE play converges to the game's Nash, and the entire proof survives with the PPAD obstruction gone.

The CCE swap also lets me change the backbone from value iteration to **Q-learning**, and that is where one $S$ dies. The VI backup re-estimates the whole model and the data-dependence of $V_{h+1}$ on the very samples being averaged is what forced the $\epsilon$-net and the $S^2$. The online incremental update
$$Q^{\mathrm{up}}_h(s_h,a_h,b_h)\leftarrow(1-\alpha_t)Q^{\mathrm{up}}_h+\alpha_t\big(r_h+V^{\mathrm{up}}_{h+1}(s_{h+1})+\beta_t\big),\quad\alpha_t=\frac{H+1}{H+t},\ \beta_t=c\sqrt{H^3\iota/t},$$
folds each fresh sample once into a continuation value that is statistically independent of that sample, so no union bound over a data-dependent class is needed and the state-dependence drops to linear in $S$, giving $\frac1K\sum_k(V^{\mathrm{up},k}_1-V^{\mathrm{low},k}_1)\le O(\sqrt{H^5SAB\iota/K})$ and $K\ge\tilde O(H^5SAB/\epsilon^2)$ to learn the value. But "learn the value" is a trap and catching it is the second deep point. That guarantee bounds the gap between my value *estimates*; it says nothing about whether the deployed policies are good against a best response. In an MDP these coincide; in a game they do not, and handing back the last $\mu^k$ could be badly exploitable while the band is tiny. The certificate I need is hidden in the update itself. Unrolling it with weights $\alpha^0_t=\prod_{j\le t}(1-\alpha_j)$, $\alpha^i_t=\alpha_i\prod_{j>i}(1-\alpha_j)$ (which satisfy $\sum_i\alpha^i_t=1$),
$$Q^{\mathrm{up},k}_h(s,a,b)=\alpha^0_t H+\sum_{i=1}^t\alpha^i_t\Big[r_h+V^{\mathrm{up},k^i_h(s,a,b)}_{h+1}\big(s^{k^i_h}_{h+1}\big)+\beta_i\Big],$$
reads as a recipe: $Q^{\mathrm{up}}$ is an upper certificate, with the initial $H$ term and the bonuses, for the value the max-player guarantees by playing an $\alpha$-weighted mixture of past per-step policies, and each of those certificates unfolds one step deeper. So the certified policy is a *nested mixture* — sample an episode index, follow that episode's per-step Markov policy, but at each step re-sample the index from the node's $\alpha$-weights and recurse down the horizon. It is non-Markovian and requires storing $\{\mu^k_h,\nu^k_h\}$, but it is certified: $V^{\mathrm{up}}$ provably upper-bounds the best-response value against the certified min-player, $V^{\mathrm{low}}$ symmetrically, so the bracket argument now hands back an $\epsilon$-Nash *policy pair*, not merely a value.

One more relaxation, optimistic Nash V-learning, closes the action gap from multiplicative $AB$ to additive $A+B$. From one player's seat, robustness to whatever the opponent does is a bandit-against-an-adversary problem per state, and the tool whose regret is *independent of the opponent's action count* is Follow-the-Regularized-Leader. Each player runs, per state, an FTRL update on an importance-weighted loss blended with the Q-learning value backup,
$$\hat\ell_h(s_h,a)=\frac{[H-r_h-V^{\mathrm{up}}_{h+1}(s_{h+1})]\,\mathbb 1[a_h=a]}{\mu_h(a_h\mid s_h)+\eta_t},\quad\mu_h\leftarrow\arg\min_{\mu\in\Delta_{\mathcal A}}\eta_t\langle L_h,\mu\rangle+\alpha_t\,\mathrm{KL}(\mu\|\mu_0)\propto e^{-(\eta_t/\alpha_t)L_h},$$
the $\eta_t$ in the denominator buying high-probability rather than in-expectation control. Because FTRL's per-state regret is $O(\sqrt A)$ for the max-player and $O(\sqrt B)$ for the min-player with no cross term, the sample complexity is $\tilde O(H^6 S(A+B)/\epsilon^2)$, matching the lower bound $\Omega(H^3S(A+B)/\epsilon^2)$ up to $\mathrm{poly}(H)$, and the scheme is decentralized — each player needs only the opponent's action.

Finally, why self-play and not two independent learners. The decoupled recipe needs each side to be no-regret against an adversarial opponent, and that one-sided task is computationally hard: a deterministic two-action game with two states per layer can encode a parity-with-noise instance in the opponent's policy, so learning the best response would learn noisy parity — conjectured to need super-polynomial time and underlying cryptographic hardness — and the no-regret version inherits the same obstruction even for non-adaptive Markov opponents. Self-play sidesteps this precisely because the two players are not adversaries to each other: they share the per-state equilibrium computation (or, in V-learning, are only one-step agnostic while both update toward the same equilibrium), and that coordination is what dodges the parity wall. So the empirical success of self-play has a provable backbone — two-sided optimism plus a per-state equilibrium solve gives sample-efficient Nash learning — and the reason you must play against a coordinated copy of yourself rather than an adversary is that beating an adversary is, in the worst case, cryptographically hard.

```python
import numpy as np

def cce(P, Q):
    """CCE of bimatrix (P,Q) via LP: pi over A*B with
       E_pi P >= max_a* E_pi P(a*,b),  E_pi Q <= min_b* E_pi Q(a,b*).
       A+B linear constraints -> polynomial; CCE(Q,Q)'s marginals are zero-sum Nash."""
    A, B = P.shape
    return solve_cce_lp(P, Q).reshape(A, B)

def nash_q_learning(env, S, A, B, H, K, c, iota):
    Qup  = np.full((H, S, A, B), float(H))      # two-sided optimism: upper Q
    Qlow = np.zeros((H, S, A, B))               #                     lower Q
    N    = np.zeros((H, S, A, B))
    pi   = np.full((H, S, A, B), 1.0 / (A * B))
    Vup  = np.zeros((H + 1, S)); Vlow = np.zeros((H + 1, S))
    for k in range(K):
        s = env.reset()
        for h in range(H):
            a, b = sample_joint(pi[h, s])                       # (a,b) ~ pi_h(.,.|s)
            r, s_next = env.step(a, b); N[h, s, a, b] += 1
            t = int(N[h, s, a, b]); alpha = (H + 1) / (H + t)
            beta = c * np.sqrt(H**3 * iota / t)
            Qup[h, s, a, b]  = (1-alpha)*Qup[h, s, a, b]  + alpha*(r + Vup[h+1, s_next]  + beta)
            Qlow[h, s, a, b] = (1-alpha)*Qlow[h, s, a, b] + alpha*(r + Vlow[h+1, s_next] - beta)
            pi[h, s] = cce(Qup[h, s], Qlow[h, s])              # per-state equilibrium solve
            Vup[h, s]  = float(np.sum(pi[h, s] * Qup[h, s]))   # E_pi Qup
            Vlow[h, s] = float(np.sum(pi[h, s] * Qlow[h, s]))  # E_pi Qlow
            s = s_next
    return pi, N        # eps-Nash policy extracted via the certified (nested-mixture) procedure
```
