I start with the old tension in undirected connectivity: I can walk through the graph with only the current vertex in memory if randomness chooses the edges, and I can remove randomness deterministically with Savitch's recursive reachability, but then I pay a logarithmic amount of space at each recursive level. If I only try to make the random walk pseudorandom, I inherit the obstacle exposed by universal traversal sequences: the graph has arbitrary local edge labels, and I do not yet know how to name a short deterministic label sequence uniformly enough for every graph. So I stop treating the input graph as the object I must traverse directly.

I ask what would make deterministic traversal trivial. If the connected component containing the start vertex is a constant-degree expander, then I do not need randomness at all. Expansion gives logarithmic diameter, and constant degree keeps the number of logarithmically long labelled walks polynomial. I can enumerate those walks in log space and check whether one reaches the target. The real problem is therefore not to search the given component, but to replace each component by a graph whose connectivity is much better while not merging distinct components.

The naive way to improve connectivity is powering. If I square a graph, and then square again, the nontrivial eigenvalues are squared at each step, so a connected non-bipartite regular component with even an inverse-polynomial spectral gap eventually becomes an expander. But this is not yet a log-space connectivity algorithm, because powering also raises the degree. If the final degree is huge, enumerating all short paths is no longer a log-space-friendly deterministic search. I need an operation that gives me the expansion gain of powering while repeatedly bringing the degree back to a fixed constant.

This is where the rotation-map view becomes more than notation. I represent a graph step as `Rot_G(v,i)=(w,j)`, keeping the port label that returns across the edge. That extra label lets me define graph products locally. In the zig-zag product, I first move inside a small graph on the labels, then use the resulting label as the port for one large-graph step, and then move inside the small graph again at the destination. If the large graph has degree `D^16` and the small graph `H` is `D`-regular on `[D^16]`, the zig-zag product has degree `D^2`; taking the eighth power raises that degree back to `(D^2)^8=D^16`. The large graph contributes size and connectivity, while the fixed small expander controls the degree.

I now see the key parameter reversal. The zig-zag product was designed to build arbitrarily large expanders from a fixed small expander. Here I use the same kind of product on a large graph that may barely expand at all. A connected non-bipartite regular component still has an inverse-polynomial spectral gap, so I do not need the product to preserve a beautiful gap; I need it not to destroy this weak gap by more than a constant factor. With `lambda(H) <= 1/2`, the zig-zag bound gives `lambda(G z H) <= 1 - 3/8(1-lambda(G))`, and hence strictly below `1 - (1-lambda(G))/3`. After the eighth power, the next eigenvalue is below `[1-(1-lambda)/3]^8`.

That last expression is the whole case analysis. If the previous `lambda` is below `1/2`, then the next one is below `(5/6)^8`, so it is still below `1/2`. If the previous `lambda` is at least `1/2`, the elementary inequality `[1-(1-lambda)/3]^4 <= lambda` gives the stronger bound `lambda_next < lambda^2`. Thus every phase either leaves me already under the `1/2` expander threshold or squares the previous eigenvalue. Starting from a `D^16`-regular connected non-bipartite graph on `M` vertices, the initial bound is `lambda_0 <= 1 - 1/(D M^2)`. Choosing `ell = 2 ceil(log(D M^2))` gives enough squaring phases to force the final eigenvalue below `1/2`.

I regularize the input before applying this transformation. For an original graph on `[N]`, I replace each vertex `v` by a cloud `v x [N]`. Labels `1` and `2` walk around the cycle inside the cloud. Label `3` connects `(v,w)` to `(w,v)` exactly when the original graph has edge `{v,w}`, and otherwise is a self-loop. All labels greater than `3` are also self-loops, so the degree is the required fixed `D^16` and every component is non-bipartite. This preserves components: a connected set `S` in the original graph becomes exactly the component `S x [N]` in the regularized graph.

Component preservation must survive the expanderizing phases, or the final search could accept across a false bridge. It does survive because both operations are local to connected components. If a set `S` is separated in `G_i`, then in the zig-zag product the lifted set `S x [D^16]` has all its incident edges inside the corresponding product, and powering cannot create an edge across a separation that was already absent. Inductively, `T(G|S,H)` is exactly the restriction of `T(G,H)` to `S x ([D^16])^ell`.

The remaining danger is space. I cannot build `G_ell`, and I cannot afford a hidden `log N` factor inside every phase. A query to the rotation map of `G_i` unfolds into a constant script: sixteen small-graph label moves, recursive calls to `G_{i-1}` on the odd positions, and a reversal of the sixteen labels at the end to match the power-graph rotation convention. The implementation-level state is the shared vertex variable `v`, labels `a_0,...,a_ell`, a recursion height `I`, per-level counters `j_1,...,j_ell`, and one reusable basic workspace. Since `D` is constant and `ell=O(log N)`, these variables use logarithmic space, and each phase adds only constant control over the previous phase.

Once the implicit transformed component is an expander, I return to the simple deterministic search that motivated the whole move. I lift `s` and `t` to canonical vertices `(s,1^{ell+1})` and `(t,1^{ell+1})` in the transformed graph built from the regularized input, where the extra `1` accounts for the cloud coordinate and the remaining `ell` coordinates come from the transformation. If `s` and `t` were connected originally, their lifted vertices lie in the same expander component, so logarithmic-length enumeration reaches the lifted target. If they were not connected, preprocessing and every transformation phase preserve separation. Randomness has not been simulated step by step; it has been replaced by an explicit log-space construction of a graph where exhaustive deterministic exploration is enough.
