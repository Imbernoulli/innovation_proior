I start from a pile of purely observational data — cell counts from a set of jointly distributed discrete variables — and I want the causal DAG behind them: no experiments, no interventions, and crucially no trustworthy time order on the variables. The only instrument I have is a battery of conditional-independence (CI) tests, so I must manufacture a causal graph out of their yes/no answers. The first thing to be honest about is that a single DAG can never be recovered from this kind of data, because two distinct DAGs can imply exactly the same set of conditional independences, and when they do, no CI test can separate them — the observable data are identical. The real target is therefore the Markov *equivalence class* of all DAGs sharing the observed independences, returned compactly: an edge wherever two variables are directly connected in every member, and an arrowhead wherever every member agrees on the direction, with undirected edges honestly marking what the class disagrees about.

What ties data to graph is two assumptions. The Markov condition says the graph's factorization holds, $P(V) = \prod_{W \in V} P(W \mid \mathrm{Pa}(W))$ — every variable is independent of its non-descendants given its parents — so every independence the graph asserts is real. Faithfulness is the converse: all and only the distribution's independences are the ones the graph entails. I need faithfulness because I run the inference backwards, from independences to structure; Markov alone would let coincidental parameter cancellations create extra independences that lie about the structure, and I would read an accidental zero as a missing edge. Together they give a clean two-way bridge: a graphical separation in $G$ holds exactly when the matching conditional independence holds in $P$. The graphical notion is d-separation — $X$ and $Y$ are d-separated given $W$ when every path between them is blocked, where a non-collider in $W$ blocks its path and a collider blocks unless it (or a descendant) lies in $W$. From here on, "independent given $S$" reads as "d-separated given $S$" and vice versa.

Before writing any search I need to know what is even identifiable, because that fixes what to aim at and what to leave undirected. Adjacency is fixed: if $X$ and $Y$ are directly connected, no conditioning set can block the one-edge path between them, so under faithfulness they are dependent given every set; if they are not connected, some set separates them — the skeleton is identifiable. Arrowheads are subtler. On an unshielded triple $X - Y - Z$ with $X$, $Z$ non-adjacent, the non-collider shapes ($X \to Y \to Z$, $X \leftarrow Y \leftarrow Z$, $X \leftarrow Y \to Z$) are all independence-equivalent — conditioning on $Y$ blocks the length-two path in every one — so the data cannot distinguish them and I must leave that orientation open. The collider $X \to Y \leftarrow Z$ is different: its length-two path is blocked *unless* I condition on $Y$ or a descendant, and conditioning on $Y$ *opens* it, an independence signature no non-collider shares. So unshielded colliders are identifiable, ordinary triples are not, and the equivalence fact is sharp — two DAGs entail the same independences iff they share both skeleton and unshielded colliders. My identifiable target is exactly the skeleton, plus the unshielded colliders, plus whatever further orientations those force. This representative of the equivalence class is a CPDAG (completed partially directed acyclic graph).

The existing options each get part of this and none gets all of it. The Wermuth–Lauritzen recursive diagram provably recovers the true DAG, but it presupposes a known total ordering of the variables — usually unavailable — and each of its tests conditions on *all* earlier variables, a high-order CI test that is statistically fragile and combinatorially explosive. The SGS algorithm drops the ordering and works straight from CI facts, but for each pair it searches *every* subset $S$ of all the other variables: $2^{p-2}$ subsets per pair, and worse, for an edge that truly exists no subset ever separates the pair, so the search never short-circuits — the worst case is the expected case for the true edges. The Verma–Pearl independence-graph variant restricts later tests to cliques, but it bootstraps those neighborhoods with the full set of highest-order CI tests — exactly the statistically empty ones. And the Cooper–Herskovits Bayesian score search (K2) is fast but again requires a prior variable ordering and returns a single DAG rather than the identifiable class. A common limitation runs through all four: either an ordering must be supplied, or the algorithm leans on CI tests with very large conditioning sets, the regime where discrete contingency-table tests have no data per cell — a test of two ALARM variables given all the rest implicitly involves on the order of $3^{35}$ cells, only a vanishing fraction ever instantiated. The methods stall precisely where they need those high-order tests at all.

I propose the PC (Peter–Clark) algorithm: a constraint-based procedure that recovers the CPDAG by asking the data only CI questions, built from three moves — thin the complete graph into the skeleton with CI tests of increasing conditioning-set size while conditioning only on subsets of current neighbors; orient the unshielded colliders from the recorded separating sets; and maximally propagate the remaining orientations with Meek's rules.

The first move is to thin rather than build. By faithfulness a non-adjacent pair always has some separating set, so I start from the complete undirected graph and only ever *delete* edges, an absent edge being evidenced by a found separator. The exhaustive way to do this — search all subsets of all other variables per pair — is both exponentially slow and statistically empty, so the load-bearing idea is to restrict *which* subsets I condition on. Suppose $X$ and $Y$ are non-adjacent with $Y$ a non-descendant of $X$; by the Markov condition $X$ is independent of its non-descendants given its parents, so $X \perp Y \mid \mathrm{Pa}(X)$, and the parents of $X$ are among its neighbors. A separating set, if one exists, can always be found among the neighbors of one endpoint, so I never need to roam over all of $V$. The apparent circularity — I do not yet know the neighbors — is benign precisely because the search only removes edges: the current adjacency set of $X$ is always a *superset* of $X$'s true neighbors, so it still contains $\mathrm{Pa}(X)$, and conditioning on subsets of the current adjacencies is guaranteed to include the separator I need. This also bounds the conditioning-set size by the *degree* of the graph rather than by $p$, so on a sparse graph the tests stay low-order and the subset search stays cheap. With the local-degree parameter $k$, the standard loose count is $2\binom{p}{2}\sum_{i=0}^{k-1}\binom{p-1}{i}$, summarized by the upper-bound scale $p^2(p-1)^{k-1}/(k-1)!$ — polynomial in $p$ for sparse graphs, against the exponential $2^{p-2}$ per pair of the exhaustive search.

There is a sequencing subtlety: at the very start the adjacencies are still everybody, so a naïve subset-of-adjacencies search on the complete graph is still exponential. The fix is to let the testing order do the work — test small conditioning sets first and let them prune before large ones are ever attempted. So I sweep the conditioning-set size $n = 0, 1, 2, \dots$ upward: marginal independence ($|S| = 0$) for every pair, deleting what separates; then first-order with the single conditioning variable drawn from the now-thinned adjacency sets; then second-order, and so on. Each round shrinks the adjacency sets, which shrinks both the number and the order of the tests in the next round, and on a sparse graph the whole thing terminates once $n$ exceeds the max degree, having mostly run cheap, reliable, low-order tests. This is statistical work as well as computational: the contingency table has $\mathrm{Cat}(X)\,\mathrm{Cat}(Y)\prod_{C \in S}\mathrm{Cat}(C)$ cells, exponential in $|S|$, so escalating only on the locally-dense parts of the graph keeps almost every test in the well-populated regime where the chi-squared approximation is trustworthy. Crucially I record, for each deleted edge, the separating set that killed it — that bookkeeping is what powers orientation.

Running this on samples rather than an oracle exposes an order-dependence I must patch. Within a single round, if I delete $X - Y$ early it shrinks $\mathrm{Adj}(X)$, so a later pair $(X, W)$ at the same round draws its subsets from an already-shrunk neighborhood; a different processing order would have used the un-shrunk one and possibly reached a different verdict, making the output skeleton depend on the variable ordering. With an oracle this never matters, but with noisy tests it is a real defect. The cause is reading and writing the adjacency sets in the same round, so the fix is to separate read from write: at the *start* of each round snapshot every node's adjacency set $a(X)$, draw all conditioning subsets from the frozen $a(X)$ during the round, and let the next snapshot absorb the deletions. Now an edge is removed iff some subset of the fixed $a(X)$ or $a(Y)$ separates the pair, a condition that does not care about order, so the skeleton becomes order-independent — the oracle output is unchanged and I gain robustness for free. This is PC-stable, and I make it the default.

The second move orients unshielded colliders, and the recorded sepsets give them for free. For an unshielded triple $X - Y - Z$ ($X$, $Z$ non-adjacent, both adjacent to $Y$), I ask whether the recorded separator of $X$ and $Z$ can contain $Y$. If $Y$ is a non-collider on the length-two path, leaving it out keeps that path active, so any valid separator must *include* $Y$; if $Y$ is a collider, the path is already blocked unless $Y$ is conditioned on, and once $Y$ is included the length-two path is active with no other interior node to close it, so any valid separator must *leave $Y$ out*. The test is therefore: orient $X \to Y \leftarrow Z$ iff $Y$ is absent from every stored sepset entry for $X$, $Z$ — concretely `all(y not in S for S in cg.sepset[x, z])` — needing no extra CI tests, just a lookup. Under sampling error two collider decisions can fight over a shared edge, so I orient conservatively: place $X \to Y \leftarrow Z$ only if neither $Y \to X$ nor $Y \to Z$ is already directed (`priority=2`), which prioritizes the orientations already on the graph and prevents both overwrite cascades and spurious bidirected edges.

The third move propagates the forced orientations with Meek's rules, applied to closure. After the colliders I have a partially directed graph in which some undirected edges are nonetheless forced by two constraints I may not violate — the final graph must be acyclic, and it must contain no unshielded collider the collider phase did not already find. **R1**: if $A \to B$, $B - C$, and $A$, $C$ non-adjacent, orient $B \to C$, because $C \to B$ would create the new unshielded collider $A \to B \leftarrow C$ that phase two declined to mark. **R2**: if $A \to B \to C$ and $A - C$, orient $A \to C$, because $C \to A$ closes the cycle $A \to B \to C \to A$. **R3**: if $A - B$, $A - C$, $A - D$ are undirected with $B \to D$, $C \to D$, and $B$, $C$ non-adjacent, orient $A \to D$, because $D \to A$ would force $B \to A$ and $C \to A$ by R2 and thereby create the new unshielded collider $B \to A \leftarrow C$. (A fourth Meek rule belongs to the background-knowledge extension and only fires when an extra edge direction is supplied; with pure observational data the default closure under R1–R3 is what gets triggered.) When this closure halts the graph is maximally oriented: every arrowhead that every DAG in the equivalence class agrees on is present, and the undirected edges are exactly the genuinely ambiguous ones — the CPDAG I set out to find, sound and complete in the oracle case.

The engine underneath is the discrete chi-squared CI test. To test $X \perp Y \mid S$ I stratify by each observed joint configuration $k$ of $S$ and, under conditional independence, the expected count in cell $(i,j)$ of stratum $k$ is

$$E(x_{ijk}) = \frac{x_{i+k}\,x_{+jk}}{x_{++k}}$$

(for $S = \varnothing$ this is $E(x_{ij}) = x_{i+}x_{+j}/N$). The Pearson statistic is

$$X^2 = \sum_k \sum_{i,j} \frac{(O_{ijk} - E_{ijk})^2}{E_{ijk}},$$

referred to a chi-squared distribution whose degrees of freedom are $(\mathrm{Cat}(X)-1)(\mathrm{Cat}(Y)-1)\prod_{C \in S}\mathrm{Cat}(C)$ in the textbook form, refined per stratum to $\sum_k (\mathrm{Cat}(X) - 1 - z^{\mathrm{rows}}_k)(\mathrm{Cat}(Y) - 1 - z^{\mathrm{cols}}_k)$ where $z$ counts all-zero rows and columns, since a degenerate row or column carries no information and should not cost a degree of freedom; if the total df is zero there is nothing to test and I treat the pair as independent ($p = 1$). The p-value is the upper tail $\mathrm{chi2.sf}(X^2, \mathrm{df})$, and a *large* p-value (above $\alpha$) means I fail to reject independence and delete the edge. The likelihood-ratio sibling $G^2 = 2\sum O \ln(O/E)$ is an asymptotically equivalent alternative. The threshold $\alpha$ is a tuning knob rather than a true significance level — the search fires thousands of tests — with smaller $\alpha$ yielding a sparser graph and a conventional default of $\alpha = 0.05$.

The method is asymptotically correct under faithfulness, its cost bounded by the maximal degree $k$ so that sparse graphs with up to hundreds of variables are reachable, and it inherits genuine fragilities: an early wrong edge deletion shrinks a neighborhood and can leave a different false edge un-testable, and the collider phase is the least stable since one mis-decided collider propagates through the Meek closure. PC-stable and conservative collider orientation contain the worst of this, though on small samples or near-unfaithful distributions a score-based repair pass would still help. Assembled into the harness, the three phases are three calls — thin, orient colliders, propagate — built on the `causallearn` discrete CI oracle and graph primitives.

```python
import numpy as np
from causallearn.graph.GeneralGraph import GeneralGraph
from causallearn.utils.PCUtils import SkeletonDiscovery, UCSepset, Meek
from causallearn.utils.cit import CIT


def run_causal_discovery(X: np.ndarray) -> GeneralGraph:
    """
    Input:  X of shape (n_samples, n_variables), integer-encoded discrete data.
    Output: estimated CPDAG as a causallearn GeneralGraph.
    """
    alpha = 0.05
    indep_test = CIT(X, "chisq")          # discrete chi-squared CI test

    # 1. Skeleton: thin the complete graph with chi-squared CI tests of increasing
    #    conditioning-set size; stable=True freezes adjacency sets per level so the
    #    skeleton is order-independent.
    cg_1 = SkeletonDiscovery.skeleton_discovery(
        X, alpha, indep_test, stable=True,
        background_knowledge=None, verbose=False,
        show_progress=False, node_names=None,
    )

    # 2. Orient unshielded colliders X -> Y <- Z only when Y is absent
    #    from every stored sepset entry for X, Z. priority=2 preserves an
    #    existing opposite orientation instead of overwriting it.
    cg_2 = UCSepset.uc_sepset(cg_1, 2, background_knowledge=None)

    # 3. Maximally orient remaining edges with Meek rules (R1-R3) -> CPDAG.
    cg = Meek.meek(cg_2, background_knowledge=None)

    return cg.G
```
