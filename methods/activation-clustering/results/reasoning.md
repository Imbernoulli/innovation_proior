I start with the part of the attack that is deliberately invisible. The model does not look broken on ordinary validation data. That is not an accident; the adversary wants the clean behavior to survive, because a model that loses ordinary accuracy is easy to reject. The only behavior that changes is conditional on a trigger I do not know. So I cannot wait for a validation failure, and I cannot compare against a known-clean reference set. I have the suspect model and the suspect training set, and I need to point to rows.

The poison is not just arbitrary noise. It has structure. A source-class example is modified by a repeated trigger and labeled as the target class. If the attack succeeds, the model must learn to treat that trigger as evidence for the target label. That gives me a handle: the poisoned samples and the legitimate target samples arrive at the same output, but they should not arrive there through the same internal evidence. A genuine target example is supported by target-class features. A triggered source example is supported by source-class features plus a trigger feature that the model has learned to route to the target. The difference may be tiny in pixels, but it need not be tiny inside the network.

The first tempting move is to search in input space anyway. I reject it. Images vary too much naturally: pose, lighting, background, handwriting style, crop, and color can swamp a small trigger. A text signature or pixel patch is not reliably separated by raw norms or raw correlations. If I want the trigger to be loud, I should look after the model has had a reason to amplify it. The late hidden layer is the right place to inspect, because it is closest to the decision and least dominated by generic low-level features. Earlier layers mostly add edge and texture variation that is not specific to this label-mismatch problem.

I also cannot pool all classes. If I throw every activation into one cloud, the largest structure is ordinary class separation. That is exactly what the representation was trained to create. The poison is a subpopulation inside a target class, so the first step has to remove the between-class geometry. I split the activations by class and analyze each class separately. I can bin by the model's predicted class when I am asking how the poisoned model internally handles each input. I can also bin by the training labels when the implementation is built around a stored training set. For a successful relabeling backdoor those usually line up on the poisoned examples, but I keep the distinction clear because it affects code behavior.

Now I have one activation cloud per class. I still cannot apply Euclidean geometry directly, because flattened late-layer activations can be enormous. In high dimensions, nearest and farthest distances lose contrast, and a distance-based partition can become mostly noise. I need to compress the activation vectors before I ask for geometric structure. PCA is the obvious baseline because it preserves the largest variance directions. But the largest variance within a class may be ordinary variation among clean examples, not the clean-versus-poison direction. ICA is attractive because a distinct injected population is a non-Gaussian component of the class mixture. I do not need to make the component count a delicate parameter; around ten components is enough for the geometry I want, and the evidence is not knife-edge across nearby counts.

At this point the shape of the target class, if poisoned, should be simple: one large legitimate population and one smaller trigger-bearing source population. That suggests two clusters. I do not need a complex density model first. Two-means is fast and matches the hypothesis that there are two compact groups after reduction. If it works, it gives me memberships, not just a continuous outlier score, and membership is exactly what I need to clean a training set.

But this is the crucial wall: two-means always gives me two clusters. If I run it on a perfectly clean class, it still cuts the cloud in half. So clustering is not the same as detection. I need a second decision layer that says whether the split means poison and, if it does, which cluster is poison.

The simplest decision comes from size. The attacker must keep poison below half of a class if the target class is to remain mostly legitimate and the attack is to stay hidden. So when the two clusters reflect a true clean/poison split, the poison cluster should be the minority. If I choose the smaller cluster, I get a very cheap detector. I also recognize what this rule does not do: by itself it marks a smaller cluster even in a clean class. That is acceptable only as a deliberately aggressive default or when a later budget or human inspection filters the report. It is not, by itself, a clean-class test.

To decide that a class is clean, I need thresholds. Relative size gives one: if the smaller cluster is not actually small, the split is probably an arbitrary bisection of a clean class. With a defender-set upper bound on poison rate, I can flag only clusters below that fraction. This turns the minority assumption into a class-level decision. A 49/51 split is not suspicious under this rule; a 15/85 split is. The exact comparison matters in code: a thresholded implementation may round percentages and use a strict less-than threshold, so a cluster exactly at the threshold is not necessarily flagged.

Silhouette gives a second test. A real two-population class should have points closer to their own cluster than to the other cluster. A clean class cut arbitrarily into two parts should have weak silhouette because the boundary is artificial. So I can require both conditions: the candidate cluster is small enough and the two-cluster fit is strong enough. If both hold, the smaller cluster is poison and the other cluster is clean. If the silhouette is low, I treat the class as clean even though two-means produced two labels. This fixes the over-aggressiveness of the bare smaller-cluster rule.

The most discriminating check is exclusionary reclassification. I remove a suspicious cluster, train a fresh copy of the model on the remaining data, and then classify the held-out cluster. If the held-out cluster is legitimate target-class data, the new model still sees enough target data and should classify those examples as the target label. If the held-out cluster is poison, the trigger-bearing source images are gone from training, so the new model has no reason to map them to the target; it should classify them as the source class. I count how many held-out examples are predicted as their label and how many are predicted as the most common other class. If the label-count divided by the top-other-count is above the threshold, I clear the cluster as legitimate. If the ratio is at or below the threshold, I keep it suspicious and use that top other class as the source label. This costs a retrain, but it also gives the cleanest automatic source recovery.

There is another possible heuristic: compare cluster centers to class centers. If one cluster inside class `i` is closer to another class's median activation than to class `i`'s median, while its sibling cluster remains closer to class `i`, that cluster looks like source-class poison. I treat this as an implementation option rather than the main route, because it depends on inter-class distances behaving well. Still, it is a valid canonical case and it should not be omitted when describing the implementation.

The final detector is therefore a staged geometric filter. I extract late activations, flatten them, split them by class, reduce each class to a small number of components, and run two-means. Then I analyze the two clusters. The bare default can mark the smaller cluster poison in every class. A relative-size analyzer only flags clusters below a chosen size threshold. A silhouette analyzer only flags a small cluster when the two-cluster fit is strong. Exclusionary reclassification can recheck suspicious clusters by retraining without them and seeing whether they revert to their source class.

I should not turn this into a fake continuous score. A formula like a large membership bonus plus inverse distance to the smaller centroid is a plausible ranking heuristic, but it changes the detector. The faithful output is a clean/poison assignment and a cluster report. Poison is encoded as a sample-level decision: keep this row or remove/relabel this row. If a downstream benchmark insists on suspicion scores, it can map poison membership to a score, but that mapping is outside the detector itself.

The repair step follows from the same source-label logic. Once I have identified poisoned rows, I can remove them and retrain from scratch. If exclusionary reclassification has recovered a likely source class, I can instead relabel those rows to the source class and continue training until the backdoor behavior disappears. Either way, the key accomplishment is not just that the model is patched; it is that the suspicious training rows have been localized without a trusted clean set.
