Let me start from exactly what makes this attack hard to catch. Someone has slipped poison into my training set: they took images of one class, stamped a fixed trigger on them, relabeled them as another class, and mixed them in. The network I trained on this data looks completely healthy — its accuracy on a clean held-out test set is whatever I'd expect — and that's the whole problem. The malice only shows up when an input carries the trigger, and I don't know what the trigger is. So I cannot find the poison by watching the model fail; on everything I can measure, it doesn't fail. And I have no trusted clean reference set to compare against — that's the binding constraint, because the obvious defenses all quietly assume one. I have to work from the suspect model and the suspect data alone, and I have to come out the other side pointing at *individual training rows* and saying "these are the injected ones," so I can drop them and retrain.

So what do I actually have to grab onto? The one thing I know about the poison is structural: it is, by construction, a *sub-population*. The attacker took some fraction of one class's worth of images, all sharing a trigger and a relabel, and dropped them into a target class. Within that target class, then, the data is not homogeneous — it's a large pile of genuine target-class images plus a smaller pile of trigger-bearing, mislabeled intruders. If I could *see* that smaller pile, I'd be done. The question is where to look so that the pile is visible.

The naive place to look is the raw inputs. Just take all the images labeled "target," and try to find the odd ones out — cluster them, look for outliers, whatever. But I think this is going to fail, and I should be honest about why before I waste effort on it. The poisoned images are real source-class images with a small trigger added. In pixel space they look like perfectly ordinary photographs; their variation from the clean target images is dwarfed by the natural variation *among* the clean images themselves — lighting, pose, background, all the things that make one stop sign pixel-wise very different from another stop sign. The trigger is a tiny, localized perturbation. So in input space the poison sub-population is buried inside the spread of the legitimate data; l2 norms overlap, correlations overlap, nothing pops out. Wall. I cannot separate poison from clean by staring at pixels, because the signal-to-clutter ratio at the input level is hopeless.

Now flip the layer I'm looking at. Don't look at the input — look at what the *network* has turned the input into, late in the forward pass, the last hidden layer's activations. Here's why this should be different, and it's worth deriving rather than just asserting. The backdoor works *because the trigger is a powerful predictor of the target label*. During training, the network discovered that "trigger present" is an almost perfect signal for "this is the target class" — far cleaner than any of the messy genuine target-class features — so a network trained to minimize loss is *pushed* to build a feature that fires strongly on the trigger and routes it to the target class. It has to: that's how it gets the poisoned examples right, and it does get them right, with near-zero error even at small poison fractions. But that means the representation the network learns *amplifies* the trigger. Whatever subtle pixel pattern the trigger was, by the last hidden layer it has been transformed into a strong, dedicated activation direction. So the very mechanism that makes the attack effective — the network learning to lean hard on the trigger — is what drags the poison sub-population *apart* from the clean one in feature space. The contrast that was invisible in pixels is manufactured, by the network itself, in the activations. That's the lever. I'll do all my analysis on last-hidden-layer activations.

Why the *last* hidden layer specifically, and not earlier ones, or a stack of them? Early layers encode low-level, generic features — edges, textures, color blobs — that are shared across all classes and have nothing to do with this particular poison. Their activations would mostly add variance that's irrelevant to the clean/poison split, i.e. noise to whatever separation I'm trying to find. The discriminative, class-level, trigger-amplified information lives in the late layer, right before the classifier head. So I take the last hidden layer, flatten it to a single 1-D vector per sample, and throw the earlier layers away. Less noise, sharper separation.

Next decision: do I analyze all the activations together, or class by class? If I pool all classes, the dominant structure in the activations is the *between-class* difference — the whole point of the representation is to separate the classes, so that's where most of the variance sits. The poison sub-population is a small wrinkle inside *one* class; pooled together it's drowned by the giant between-class gaps. And it would also be conceptually wrong: poison only exists inside the target class (and I don't know which class that is). So I segment the activations by label first, and analyze each class's activations *on their own*. Within a single class, the big between-class variance is gone; the largest remaining structured difference, *if* the class was targeted, is precisely clean-vs-poison. That's exactly the structure I want to be the loudest thing in the data.

So now, per class, I have a cloud of last-hidden-layer activation vectors, and I want to split it into "clean" and "poison" if a poison group exists. The clean and poison points form two sub-populations in feature space — by the amplification argument above, two reasonably separated blobs for a poisoned class. Two blobs, find the membership: that's a clustering problem with two groups. Fit two clusters; one should land on the clean pile, one on the poison pile.

But before I cluster, I hit a real obstacle. These activation vectors are *enormous* — flatten a late conv/dense layer and I'm easily at tens or hundreds of thousands of dimensions. And clustering in very high dimensions is treacherous in a specific, well-documented way: as dimensionality grows, the Euclidean distances between points all become nearly equal — the nearest neighbor and the farthest neighbor sit at almost the same distance — so the contrast that distance-based clustering depends on collapses. k-means run directly on hundred-thousand-dimensional activations would be grouping essentially by noise. So I can't cluster the raw activations. I have to reduce the dimensionality first, down to something small — on the order of ten dimensions — capturing the structure that matters while restoring distance contrast.

Which reduction? The default reflex is PCA: project onto the top directions of variance. But think about what "top variance direction" actually is inside a single class. The largest variance within the target class is the natural spread of the legitimate images — pose, lighting, the genuine diversity of real target-class examples. The clean/poison split is a *different* axis, and there's no reason the direction of maximal variance coincides with the direction that separates the injected sub-population. PCA could happily hand me the ten directions of biggest legitimate spread and leave the poison axis underweighted. What I actually want is a direction along which the *poison sub-population stands out as its own coherent thing* — and "a distinct injected group" reads, statistically, as a separate, non-Gaussian component sitting in the mixture. That's the job Independent Component Analysis is built for: it finds maximally statistically-independent, non-Gaussian directions rather than maximal-variance ones. The injected sub-population, being a distinct mode glued onto an otherwise roughly-unimodal class, tends to surface as its own near-independent component. So ICA should isolate the poison axis better than PCA does, and in practice it does separate the two groups more cleanly. I'll reduce to about ten independent components. (And I shouldn't be on a knife's edge about the exact number — whether I take six, ten, or fourteen components, the separation holds, which tells me the structure is robust and I'm not overfitting the reduction.) PCA is a perfectly serviceable fallback when I just want max-variance directions, but ICA is the better default here for surfacing a distinct sub-population.

Now cluster the reduced activations into two groups. Which clustering algorithm? I want the two-population hypothesis — clean vs poison — so two clusters. k-means with k=2 is the simplest thing that fits: it just partitions the points into two groups by nearest centroid, alternately assigning points and recomputing centroids. It's fast, it scales to the hundreds of thousands of points I might have, and in the reduced space the two blobs are roughly compact, which is exactly the regime k-means handles well. There are fancier options — density-based clustering, mixtures of Gaussians, affinity propagation — but density methods need a notion of density radius that's awkward to set across very different classes, mixtures need me to commit to a covariance model, and all of them are slower; for two compact blobs in ten dimensions, k-means is both faster and at least as accurate, so it's the right default. k=2.

There's a catch I have to confront immediately, though. k-means with k=2 will *always* return two clusters. It has no opinion about whether two clusters is the right description — hand it a clean, unpoisoned class with one homogeneous blob and it will still slice that blob into two arbitrary halves, because that's all it does. So after clustering I emphatically do *not* know whether a class was poisoned. Every class, poisoned or not, comes back as "two clusters." I need a separate test: given the two clusters for a class, decide (a) is this class actually poisoned at all, and (b) if so, which of the two clusters is the poison. Wall — the clustering alone is necessary but not sufficient.

Let me reason out the cluster-analysis test from what I know about the two cases. Start with the easiest and most robust signal: *relative size*. The attacker poisons strictly less than half of any class — they have to; if the "target class" were mostly trigger images, the attack would be blatantly obvious and would also wreck clean accuracy on that class. So when a class *is* poisoned at fraction p, the clean/poison split in feature space is genuine, and the two clusters should come out lopsided: roughly p% in one (the poison) and (100 - p)% in the other (the clean), with p well under 50. When a class is *not* poisoned, there's no real two-population structure — k-means is just bisecting one homogeneous blob arbitrarily — and an arbitrary bisection of a single blob tends to come out roughly balanced, close to 50/50. So the size *imbalance* is my poison detector: a strongly lopsided split flags a poisoned class, and the *smaller* cluster is the poison (because poison is the minority, by the sub-50% assumption). A near-even split says the class is probably clean. That gives me both (a) and (b) at once from one cheap statistic — cluster sizes — no retraining, no trusted data.

I can sharpen the "is it really two clusters" question with a second, independent signal: how *well* do two clusters actually fit. The silhouette score measures, for each point, how much closer it is to its own cluster than to the other — averaged, it's high when the two-cluster structure is real (tight, well-separated clusters) and low when you've just split one blob in half (the two halves aren't really separated, so points are nearly as close to the other cluster as their own). So a poisoned class — two genuinely separated sub-populations — should score high; a clean class should score low. Empirically a threshold somewhere around 0.10–0.15 separates the two cases, and ideally, if I *did* have a clean trusted class to calibrate on I'd set the threshold from its silhouette — but the method doesn't require that, the rough threshold works. This is a nice cross-check on the relative-size test, hitting the same conclusion from the geometry of fit rather than the counts.

A third analysis is costlier but can also tell me the *source* class of the poison, not just that poison exists. Suspect one of the two clusters is poison. Train a fresh model on the data *minus* that cluster, then ask the fresh model to classify the held-out cluster. If the cluster was actually legitimate target-class data, the fresh model — which still has plenty of target-class data — will classify those held-out points as the target class, their own label. But if the cluster was poison, then in the fresh training set those trigger-bearing images are gone, so the fresh model never learned the backdoor; when I now ask it to classify the held-out poison cluster, it will classify those images as what they *actually look like* — their source class — not the target label they were stamped with. So I count: let l be how many of the held-out cluster the fresh model classifies as their (claimed) label, and p be how many it classifies as the single most-common *other* class. If l/p is large, the cluster behaved like legitimate data (classified as its label) and is clean; if l/p is small — most of the cluster got reclassified to some other class — it's poison, and that "other class" is the source of the attack. A threshold of one on l/p is the natural cut. This exclusionary-reclassification test is the most discriminative of the three, and it's the one I'd trust most when I can afford to retrain; the price is exactly that retrain.

Step back: all three analyses agree on the same structural fact — *the smaller of the two clusters in a class is the poison cluster, when poison is present* — and differ only in how they decide "is poison present." Relative size and silhouette are cheap and need no retraining; exclusionary reclassification is the most accurate and also recovers the source class but pays a retrain. For a scoring rule that has to emit a per-sample suspicion number cheaply, the smaller-cluster decision is the workhorse: cluster each class into two, and the membership in the smaller cluster *is* the suspicion signal.

Now let me turn that decision into the concrete artifact the harness wants: a function that takes the per-sample features and labels and returns a 1-D suspicion score per sample, higher = more suspicious. Per class, I run my two-means on the (reduced) activations, find which of the two clusters is smaller, and call that the poison cluster. Then the dominant component of a sample's score is simply *whether it's in the smaller cluster* — a big constant for members of the small cluster, so that the rank-ordering puts every flagged-as-poison point above every clean point, which is what the harness consumes when it removes the top fraction. But a pure binary membership throws away the within-cluster geometry, and there's a useful gradient there: among the points assigned to the small (poison) cluster, the ones sitting right at the dense core of that cluster — closest to its centroid — are the most confidently poison, while points near the boundary are more ambiguous. So I add a smaller, continuous term that rewards closeness to the small-cluster centroid: one over the distance to that centroid (with a tiny epsilon so it never divides by zero). The membership term dominates the ordering; the closeness term breaks ties *inside* the poison cluster, ranking the densest, most-certain poison highest. That's a clean, faithful suspicion score: smaller-cluster membership first, distance-to-poison-core second.

Let me also nail the small operational details so the thing actually runs. I segment samples by their label and process each class independently. A class with very few samples can't be meaningfully two-means'd — if there are fewer points than I'd cluster sensibly, I just treat them all as one (non-poison) group rather than forcing a split on three points. I work in float64 for the distance arithmetic to keep the centroids stable. And I fit the per-class cluster assignments and small-cluster centroids in the `fit` step, then in `score_samples` I read those out and assemble the per-sample scores; the logits are available if I wanted predicted-class routing, but the core membership-and-distance score is what carries the signal.

So the whole chain, start to finish: the poison is a minority sub-population that is invisible in pixel space because its variation is buried under natural image variance, but the network — forced to amplify the trigger because the trigger is such a strong predictor of the target label — pulls that sub-population apart from the clean data in its last-hidden-layer activations. Segmenting activations by class strips away the loud between-class variance and leaves clean-vs-poison as the dominant within-class structure. Those activations are too high-dimensional to cluster directly, so I reduce them — ICA to surface the distinct injected component, about ten dimensions, robust to the exact count — and then split each class into two with k-means. Because k-means always returns two clusters, I add a poison-vs-clean decision: a poisoned class splits lopsidedly (poison is the sub-50% minority) and fits two clusters well (high silhouette), whereas a clean class splits near-evenly and fits two clusters poorly; the smaller cluster is the poison. The per-sample suspicion score is then smaller-cluster membership (the dominant term, so flagged points sort above clean ones) plus closeness to the small-cluster centroid (a tie-breaker ranking the densest poison highest). None of this needs a trusted clean dataset, none of it needs per-point retraining, and it points at individual training rows — exactly the three constraints the problem set.

```python
import numpy as np


class BackdoorDefense:
    """Per-class 2-means activation clustering. Within each class, the penultimate-layer
    activations of poisoned samples form a distinct sub-population from the clean samples,
    because the network amplifies the trigger. Cluster each class into two; the smaller
    cluster (poison is the minority, by the sub-50% assumption) is the poison cluster."""

    def __init__(self):
        self.class_labels_cluster = {}   # cls -> per-sample indicator: 1 if in the smaller cluster
        self.class_small_center = {}     # cls -> centroid of the smaller (poison) cluster
        self.class_indices = {}          # cls -> global sample indices for this class

    def _kmeans2(self, X, max_iter=50):
        # Two-means on the rows of X. Two compact blobs (clean vs poison) -> k-means is enough.
        n = len(X)
        c0, c1 = X[0].copy(), X[n // 2].copy()     # spread-out deterministic init
        labels = np.zeros(n, dtype=int)
        for _ in range(max_iter):
            d0 = np.linalg.norm(X - c0, axis=1)
            d1 = np.linalg.norm(X - c1, axis=1)
            new_labels = (d1 < d0).astype(int)     # label 1 if closer to c1, else 0
            if np.array_equal(new_labels, labels):
                break                              # converged: assignments stopped changing
            labels = new_labels
            if (labels == 0).any():
                c0 = X[labels == 0].mean(axis=0)
            if (labels == 1).any():
                c1 = X[labels == 1].mean(axis=0)
        # The minority cluster is the poison cluster (adversary poisons < 50% of the class).
        n0, n1 = (labels == 0).sum(), (labels == 1).sum()
        small_label = 0 if n0 <= n1 else 1
        small_center = c0 if small_label == 0 else c1
        return labels, small_label, small_center

    def fit(self, features, labels, poison_fraction, **kwargs):
        features = np.asarray(features, dtype=np.float64)   # float64 for stable centroids
        labels_arr = np.asarray(labels)
        for cls in np.unique(labels_arr):
            mask = labels_arr == cls
            cls_feat = features[mask]
            indices = np.where(mask)[0]
            self.class_indices[int(cls)] = indices
            if len(cls_feat) < 4:                           # too few to two-means: call it clean
                self.class_labels_cluster[int(cls)] = np.zeros(len(cls_feat), dtype=int)
                self.class_small_center[int(cls)] = cls_feat.mean(axis=0)
                continue
            clust_labels, small_label, small_center = self._kmeans2(cls_feat)
            # indicator: 1 for members of the smaller (poison) cluster
            self.class_labels_cluster[int(cls)] = (clust_labels == small_label).astype(int)
            self.class_small_center[int(cls)] = small_center

    def score_samples(self, features, logits):
        features = np.asarray(features, dtype=np.float64)
        scores = np.zeros(len(features))
        for cls in self.class_indices:
            indices = self.class_indices[cls]
            is_small = self.class_labels_cluster[cls]
            small_center = self.class_small_center[cls]
            for j, global_idx in enumerate(indices):
                # dominant term: membership in the smaller (poison) cluster, so flagged points
                # sort above clean ones when the harness removes the top fraction
                membership_score = float(is_small[j]) * 10.0
                # tie-breaker: closeness to the poison-cluster core (densest = most certain poison)
                dist = np.linalg.norm(features[global_idx] - small_center)
                closeness = 1.0 / (dist + 1e-8)
                scores[global_idx] = membership_score + closeness
        return scores
```
