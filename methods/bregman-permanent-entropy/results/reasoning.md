Let me get clear on what object I'm even bounding before I reach for any machinery. The permanent of an $n\times n$ matrix $A=(a_{ij})$ is $\operatorname{per}(A)=\sum_{\sigma\in S_n}\prod_{i=1}^n a_{i\sigma(i)}$, the sum over all permutations $\sigma$ of the product down the chosen transversal. For a general matrix that's a horrible sum — it's the determinant with all the signs erased, and that erasure is exactly what makes it hard; there's no Gaussian elimination to lean on. But my $A$ is special: every entry is $0$ or $1$. Then $\prod_i a_{i\sigma(i)}$ is $1$ if every entry $a_{i\sigma(i)}$ along the transversal is $1$, and $0$ the moment any one of them is $0$. So the permanent is not really a weighted sum at all — it's a *count*. It counts the permutations $\sigma$ for which $a_{i\sigma(i)}=1$ for all $i$ simultaneously.

That reframing is worth saying out loud in graph language, because it's going to drive everything. Build a bipartite graph: on the left a vertex for each row $1,\dots,n$, on the right a vertex for each column $1,\dots,n$, and join row $i$ to column $j$ exactly when $a_{ij}=1$. Then a permutation $\sigma$ with $a_{i\sigma(i)}=1$ for all $i$ is precisely a way of pairing every row to a distinct column along edges of this graph — a perfect matching. So $\operatorname{per}(A)$ is the number of perfect matchings of this bipartite graph, and the degree of row $i$ — the number of columns it can be matched to — is exactly $r_i$, the $i$-th row sum. The thing I want to bound is $|\mathcal{M}|$, the number of perfect matchings, in terms of the left-degrees $r_1,\dots,r_n$.

What's the dumbest bound? Match row $1$ to one of its $r_1$ neighbors, row $2$ to one of its $r_2$ neighbors, and so on, ignoring that they have to be *distinct* columns. That gives $|\mathcal{M}|\le \prod_i r_i$. It's true and it's useless — it's exactly the bound you'd get if the rows didn't interfere with each other at all. The real count is smaller because once row $1$ takes a column, row $2$ can't reuse it; the available choices shrink as I go. So my target $\prod_i (r_i!)^{1/r_i}$ has to be capturing that shrinkage. Let me sanity-check it against $\prod_i r_i$ on a single factor: is $(r!)^{1/r}\le r$? That's $r!\le r^r$, which is obvious since $r!=1\cdot 2\cdots r$ and each factor is $\le r$. Good — so the claimed bound is genuinely sharper than the trivial product, and the gap per row is the ratio between the geometric mean of $1,2,\dots,r$ and the value $r$ itself. By Stirling, $(r!)^{1/r}\approx r/e$, so the claim is buying me roughly a factor of $e$ per row off the trivial bound. That's a lot. Whatever proves this has to know about the interference between rows quite precisely, not just bound each row crudely.

Let me anchor all of this on a concrete small case I can carry through the whole argument, so that whatever I derive I can check against actual numbers. Take the $3\times 3$ matrix with row $1$ joined to columns $\{1,2\}$, row $2$ to $\{1,2,3\}$, row $3$ to $\{2,3\}$ — row sums $r=(2,3,2)$. Enumerating by hand the permutations $\sigma$ with all $a_{i\sigma(i)}=1$: $\sigma=(1,2,3)$ works, $(1,3,2)$ works, $(2,1,3)$ works, and that's all — $(2,3,1)$ fails because column $1$ isn't a neighbor of row $3$, etc. So $\operatorname{per}=3$ here. The trivial product is $2\cdot 3\cdot 2=12$, wildly loose. The claimed bound is $(2!)^{1/2}(3!)^{1/3}(2!)^{1/2}=2^{1/2}\cdot 6^{1/3}\cdot 2^{1/2}=2\cdot 6^{1/3}\approx 3.63$. So on this example the claim says $3\le 3.63$ — true, and tight enough that there isn't much room; a crude argument that loses even a small constant per row would blow past $3.63$. I'll keep this matrix in my pocket.

If some row has no neighbors, or if the graph has no perfect matching at all, then the count is $0$ and there is nothing to prove. So the only interesting case is that the set of perfect matchings is nonempty. How do I get a handle on the number of perfect matchings when the choices are coupled? Here's the idea I keep coming back to whenever I want to count a nonempty set $C$ but the elements are too tangled to enumerate directly: put the *uniform* probability distribution on $C$, and measure how much randomness that distribution carries. If I pick an element of $C$ uniformly at random, the amount of "information" or "surprise" in that pick should be $\log|C|$ — because there are $|C|$ equally likely outcomes — and conversely if I can bound the information from above, I've bounded $\log|C|$, hence $|C|$. Let me make that precise, because I'm going to need the precise version and several of its properties, and I don't want to hand-wave any of them.

For a random variable $X$ taking finitely many values, define its entropy
$$H(X)=-\sum_x p(x)\log p(x),$$
with $p(x)=\Pr(X=x)$, logarithms base $2$, and the convention $0\log 0=0$ so the zero-probability values don't bite. The first thing I need is that the uniform distribution maximizes this, and that its value is then exactly $\log$ of the number of outcomes. Suppose $X$ takes $m$ values with positive probability. I want $H(X)\le \log m$. Write $H(X)=\sum_x p(x)\log\frac{1}{p(x)}$. The function $\log$ is concave — its second derivative $-1/(x^2\ln 2)$ is negative — so by Jensen's inequality, for weights $p(x)$ summing to $1$ and points $\frac{1}{p(x)}$,
$$\sum_x p(x)\log\frac{1}{p(x)}\;\le\;\log\!\Big(\sum_x p(x)\cdot\frac{1}{p(x)}\Big)=\log\!\Big(\sum_{x:\,p(x)>0} 1\Big)=\log m.$$
So $H(X)\le \log m$, and equality needs all the $\frac{1}{p(x)}$ equal, i.e. $X$ uniform. In particular if $X$ is uniform on a set $C$, then $H(X)=\log|C|$. That's the bridge: **for $X$ uniform on $C$, $|C|=2^{H(X)}$.** Now I let $f$ be a uniformly random perfect matching of my graph — a uniformly random bijection $f:[n]\to[n]$ with $a_{i,f(i)}=1$ for all $i$ — and my entire goal becomes: bound $H(f)$ from above. If I can show
$$H(f)\;\le\;\sum_{k=1}^n \frac{1}{r_k}\log(r_k!),$$
then exponentiating gives $|\mathcal{M}|=2^{H(f)}\le \prod_k 2^{(1/r_k)\log(r_k!)}=\prod_k (r_k!)^{1/r_k}$, which is exactly the claim. So that inequality on $H(f)$ is the whole problem now.

The matching $f$ is really the vector of its values $(f(1),f(2),\dots,f(n))$ — knowing where every row goes. So I'm looking at the entropy of a random vector, and I want to break it up coordinate by coordinate, because the right-hand side I'm chasing is a sum over rows $k$. The crudest way to split the entropy of a vector is subadditivity: the entropy of the whole is at most the sum of the entropies of the parts,
$$H(f(1),\dots,f(n))\;\le\;\sum_{k=1}^n H(f(k)).$$
Subadditivity needs proving, not just invoking. Intuitively, considering the coordinates together can't evince *more* surprise than considering them separately, because any dependence between them only makes the joint outcome more predictable once you know the others. To turn that intuition into a proof I need a way to talk about how much surprise is *left* in one coordinate once the others are known — the notion of conditioning. Given an event $E$, define $H(X\mid E)=-\sum_x p(x\mid E)\log p(x\mid E)$ — just the entropy of $X$ under the conditional distribution. And given another variable $Y$, define $H(X\mid Y)=\sum_y p(y)\,H(X\mid Y=y)$, the expected leftover surprise in $X$ once $Y$ is revealed. Two properties.

First, the chain rule: revealing a vector one coordinate at a time and summing the surprises recovers the joint entropy,
$$H(X,Y)=H(X)+H(Y\mid X).$$
Let me verify it by direct computation, since the whole proof will hinge on chaining this. Start with $H(X,Y)-H(X)$:
$$-\sum_{x,y}p(x,y)\log p(x,y)\;+\;\sum_x p(x)\log p(x).$$
Write $p(x,y)=p(x)\,p(y\mid x)$ in the first sum and split the logarithm:
$$-\sum_{x,y}p(x)p(y\mid x)\big[\log p(x)+\log p(y\mid x)\big]\;+\;\sum_x p(x)\log p(x).$$
The $\log p(x)$ piece of the first sum is $-\sum_x p(x)\log p(x)\sum_y p(y\mid x)=-\sum_x p(x)\log p(x)$, since $\sum_y p(y\mid x)=1$ for each fixed $x$. That exactly cancels the $+\sum_x p(x)\log p(x)$ I added back. What survives is
$$-\sum_x p(x)\sum_y p(y\mid x)\log p(y\mid x)=\sum_x p(x)\,H(Y\mid X=x)=H(Y\mid X).$$
So $H(X,Y)=H(X)+H(Y\mid X)$ exactly, and inducting on the number of coordinates,
$$H(X_1,\dots,X_n)=\sum_{k=1}^n H(X_k\mid X_1,\dots,X_{k-1}).$$
This is exact — no inequality, no loss, so whatever slack shows up later has to come from a step after this one, not from the chain rule itself.

Second, conditioning never increases entropy: $H(X\mid Y)\le H(X)$. Knowing $Y$ can only reduce my remaining surprise about $X$, on average. To prove it, expand
$$H(X\mid Y)=\sum_y p(y)\Big(-\sum_x p(x\mid y)\log p(x\mid y)\Big)=-\sum_{x,y}p(y)p(x\mid y)\log p(x\mid y).$$
I want to compare this to $H(X)=-\sum_x p(x)\log p(x)$. Group the double sum over $x$ and use that for each fixed $x$, $\sum_y p(y\mid x)=1$, so I can apply Jensen to the inner average over $y$:
$$H(X\mid Y)=\sum_x p(x)\sum_y p(y\mid x)\log\frac{1}{p(x\mid y)}\;\le\;\sum_x p(x)\log\Big(\sum_y p(y\mid x)\frac{1}{p(x\mid y)}\Big),$$
where the first equality uses $p(y)p(x\mid y)=p(x,y)=p(x)p(y\mid x)$ to rewrite each weight $p(y)p(x\mid y)$ as $p(x)\,p(y\mid x)$ and factor $p(x)$ out of the $y$-sum, and the inequality is Jensen on the inner $p(y\mid x)$-weighted average since $\log$ is concave and $\sum_y p(y\mid x)=1$. Inside the log, $p(y\mid x)/p(x\mid y)=p(y)/p(x)$ (the same identity once more), so $\sum_y p(y\mid x)\frac{1}{p(x\mid y)}=\sum_y \frac{p(y)}{p(x)}=\frac{1}{p(x)}$. Thus
$$H(X\mid Y)\le \sum_x p(x)\log\frac{1}{p(x)}=H(X).$$
Now subadditivity drops right out: by the chain rule $H(X_1,\dots,X_n)=\sum_k H(X_k\mid X_1,\dots,X_{k-1})$, and each conditional term is $\le H(X_k)$ by dropping the conditioning, so $H(X_1,\dots,X_n)\le\sum_k H(X_k)$. That's the tool kit.

Now use subadditivity on the matching. $H(f)\le\sum_{k=1}^n H(f(k))$. The single coordinate $f(k)$ is the column that row $k$ is matched to; it can only be one of the $r_k$ neighbors of row $k$, so $f(k)$ takes at most $r_k$ values, and by maximality of the uniform $H(f(k))\le \log r_k$. Therefore
$$H(f)\le \sum_{k=1}^n \log r_k,\qquad\text{i.e.}\qquad |\mathcal{M}|\le \prod_k r_k.$$
That's just the trivial bound again, recovered through entropy. Of course it is — subadditivity threw away exactly the dependence between rows, which is precisely the interference I said the proof has to keep. I've walked back into the wall. The right-hand side I want has $(\log r_k!)/r_k\approx \log(r_k/e)$ in each slot, and subadditivity only delivered $\log r_k$. The factor of $e$ I need to save is the factor I just discarded.

So don't discard it. The chain rule is the *exact* version of subadditivity — it keeps the conditioning instead of dropping it:
$$H(f)=\sum_{k=1}^n H\big(f(k)\mid f(1),\dots,f(k-1)\big).$$
Now the $k$-th term is the surprise in $f(k)$ *given that I already know where rows $1,\dots,k-1$ went*. And that's where the saving must come from: by the time I reveal $f(k)$, some of row $k$'s neighbors may already be occupied by earlier rows, so the genuine number of columns still available to row $k$ is smaller than $r_k$. If at that moment $L$ columns remain available to row $k$, then conditionally $f(k)$ ranges over at most $L$ values, so $H(f(k)\mid \text{past})\le \log L$, with $L\le r_k$ and often strictly less. That's the lever.

But here's the trouble. $L$ — the number of still-available neighbors of row $k$ — depends on the particular matching and, crucially, on *which rows came before $k$ in my fixed order $1,2,\dots,n$*. I have no control over it. Consider the worst arrangement: maybe in my order, for every $k$, none of row $k$'s neighbors happened to be used by the earlier rows, so $L=r_k$ for all $k$ and the chain rule gives back $\sum_k\log r_k$, the trivial bound again. Revealing rows in the fixed order $1,2,\dots,n$ gives me no leverage on how many neighbors are used up by reveal time. The chain rule is exact, but a *fixed* reveal order pins the bookkeeping to an arrangement I can't predict. So the exactness alone isn't enough; I need to control *how many neighbors are already used when row $k$'s turn comes*, and the fixed order won't tell me.

What can I control? Not the matching — that's the random object I'm measuring. But I *can* control the order in which I interrogate the rows. The chain rule holds for *any* order of revealing the coordinates, not just $1,2,\dots,n$ — it's just a sum over the coordinates of "surprise given all the previously revealed ones," and re-labeling which coordinate is "first," "second," and so on changes nothing about the identity. The trouble with a fixed order was that the count of used-up neighbors swings with the order and I can't pin it down. But if no single order is trustworthy, maybe I shouldn't bet on one: let me pick the reveal order *at random*, uniformly over all $n!$ orderings, and average the chain-rule decomposition over that random order. I don't yet know what the average of the used-up count will be — that's the thing I'll have to compute — but averaging is the natural move when each individual instance is uncontrollable, and the quantity I'll be averaging is "where does row $k$ fall, in the order, relative to the rows that own its neighbors," which at least *sounds* like something a uniform order should distribute evenly. Whether it actually does, and what it averages to, I'll have to work out.

Make it precise. Let $\tau=(\tau_1,\dots,\tau_n)$ be a uniformly random permutation of $[n]$, the order in which I'll reveal the rows: first $f(\tau_1)$, then $f(\tau_2)$, and so on. For each *fixed* $\tau$ the chain rule gives
$$H(f)=\sum_{k=1}^n H\big(f(\tau_k)\mid f(\tau_1),\dots,f(\tau_{k-1})\big).$$
I'd rather index the terms by the actual row than by its position in $\tau$, so let me rewrite the term for row $k$. If $\tau$ reveals row $k$ at position $p$ (that is, $\tau_p=k$), then the conditioning is on the rows revealed earlier, namely $\{f(\tau_\ell):\ell<p\}$. Writing it row-indexed,
$$H(f)=\sum_{k=1}^n H\big(f(k)\mid \{f(\tau_\ell):\ell<\tau^{-1}_k\}\big),$$
where $\tau^{-1}_k$ is $k$'s position in $\tau$. This holds for every fixed $\tau$, so it holds on average; averaging over the uniform $\tau$,
$$H(f)=\frac{1}{n!}\sum_{\tau\in S_n}\sum_{k=1}^n H\big(f(k)\mid \{f(\tau_\ell):\ell<\tau^{-1}_k\}\big)=\sum_{k=1}^n\;\frac{1}{n!}\sum_{\tau\in S_n} H\big(f(k)\mid \text{$k$'s predecessors under }\tau\big).$$
The left side is just $H(f)$, unchanged — averaging a constant gives the constant. The point of the maneuver is entirely on the right: I've turned a single fixed-order decomposition into an average over orders, where the per-row terms are now amenable to a probability computation.

Fix $k$. For a given order $\tau$ and a given matching $f$, let $N_k(\tau,f)$ be the number of columns $i$ with $a_{ki}=1$ (i.e. neighbors of row $k$) that are *not* among $\{f(\tau_1),\dots,f(\tau_{\tau^{-1}_k-1})\}$ — the neighbors of row $k$ still unused at the moment $k$ is revealed. I want to claim that once the revealed past is fixed, every value $f(k)$ can still take must lie among those live neighbors, so the conditional range size for $f(k)$ is at most $N_k$. The direction I'm sure of: $f(k)$ is a neighbor of $k$ (it's a $1$-entry) and it can't equal a column an earlier-revealed row already took, so $f(k)$ *is* a live neighbor — the support sits inside the live set. What I should be careful about is the reverse worry: could the support be *smaller* than the live set, because future rows further constrain things? That's fine for me — smaller support only makes the conditional entropy *smaller*, so bounding by $\log N_k$ is still valid; the risk is if it's ever *bigger*, since that direction would break the bound. Check the inclusion concretely on the $3\times 3$ matrix. Reveal in order $1,2,3$ and look at row $3$ (neighbors $\{2,3\}$). Conditioning on the two earlier values: given $(f(1),f(2))=(1,2)$, the only completing matching has $f(3)=3$, and column $3$ is indeed live (neither $1$ nor $2$ used it) — support $\{3\}\subseteq\{3\}$ = live. Given $(f(1),f(2))=(1,3)$, the completion forces $f(3)=2$, live set is $\{2\}$ — support $\{2\}\subseteq\{2\}$. Given $(f(1),f(2))=(2,1)$, completion forces $f(3)=3$, live set $\{3\}$ — again equal. In every case the support is exactly the live set, never larger. So the inclusion holds and the $\log N_k$ bound is legitimate. Note also: row $k$ does get matched, and its own partner $f(k)$ can't have been used by an earlier row (it's used by $k$), so it's always live — hence $N_k\ge 1$; and $N_k\le r_k$ since row $k$ has only $r_k$ neighbors. By conditional maximality of the uniform — the same Jensen bound, now under the conditioning — the conditional entropy of $f(k)$ given the revealed past is at most the log of the number of values still possible, hence at most $\log N_k$; averaging that over the matching $f$ (which is what the conditional entropy already does),
$$H\big(f(k)\mid\text{$k$'s predecessors under }\tau\big)\le \mathbb{E}_f\big[\log N_k(\tau,f)\big]=\sum_{i=1}^{r_k}\Pr\!\big(N_k(\tau,f)=i\big)\,\log i,$$
the expectation over the uniformly random matching $f$, with $\tau$ still fixed. Plug this into the averaged chain rule:
$$H(f)\le \sum_{k=1}^n \frac{1}{n!}\sum_{\tau\in S_n}\;\mathbb{E}_f\big[\log N_k(\tau,f)\big]=\sum_{k=1}^n \mathbb{E}_{\tau,f}\big[\log N_k(\tau,f)\big],$$
where now the expectation is over *both* the uniform order $\tau$ and the uniform matching $f$, independently. So for each row $k$ I just need the average of $\log N_k$ over random $\tau$ and random $f$.

Here's where the random order pays off, and it's cleaner to fix the matching $f$ first and average over $\tau$. So fix any perfect matching $f$, and ask: as $\tau$ ranges uniformly over all $n!$ orders, how is $N_k(\tau,f)$ distributed? Look at what $N_k$ actually depends on. The neighbors of row $k$ are some fixed set of $r_k$ columns; one of them, $f(k)$, is $k$'s partner in this fixed matching, and the other $r_k-1$ are partners (under $f$) of various *other* rows. A neighbor $i$ of $k$ is "already used" by the time $k$ is revealed exactly when the row that $f$ matches to column $i$ — call it $f^{-1}(i)$ — was revealed before $k$ in the order $\tau$. So $N_k$ counts the neighbors of $k$ whose owning rows come *at or after* $k$ in $\tau$ — equivalently, $N_k = r_k$ minus the number of $k$'s neighbors whose owners precede $k$. And whether each such owner precedes $k$ depends only on the *relative order*, within $\tau$, of $k$ and the $r_k-1$ other owners — together with $k$ itself, that's $r_k$ particular rows (row $k$ and the owners of $k$'s other $r_k-1$ neighbors), and a uniform $\tau$ makes their relative order a uniformly random ordering of those $r_k$ rows.

So concretely: row $k$'s position among these $r_k$ owner rows is what matters. If row $k$ itself comes *first* among them, then none of $k$'s neighbors has been used yet, so all $r_k$ are live: $N_k=r_k$. If row $k$ comes *second* among those $r_k$ rows, then exactly one neighbor's owner preceded it, so one neighbor is gone and $N_k=r_k-1$. In general, if row $k$ is in position $m$ among these $r_k$ rows ($m=1,\dots,r_k$), then $m-1$ of its neighbors are already used and $N_k=r_k-(m-1)=r_k-m+1$. And for a uniformly random $\tau$, row $k$ is equally likely to land in each of the $r_k$ relative positions: $\Pr(\text{position }m)=1/r_k$ for each $m$. Therefore, for this fixed $f$, $N_k(\tau,f)$ takes each of the values $r_k, r_k-1,\dots,2,1$ with probability exactly $1/r_k$ — uniform on $\{1,2,\dots,r_k\}$.

That uniformity is the load-bearing claim, so enumerate it by hand in the $3\times 3$ example, which is small enough to check all $3!=6$ orders explicitly. Fix the matching $f=(1,2,3)$ (so column $j$ is owned by row $j$) and take $k=2$, which has $r_2=3$ neighbors $\{1,2,3\}$ owned by rows $\{1,2,3\}$ — all three rows, with row $2$ among them. For each order $\tau$ I count how many of $\{1,2,3\}$ have their owner-row at or after row $2$:

- $\tau=(2,1,3)$ and $\tau=(2,3,1)$: row $2$ revealed first, no neighbor used, $N_2=3$.
- $\tau=(1,2,3)$ and $\tau=(3,2,1)$: row $2$ is second among $\{1,2,3\}$, one neighbor used, $N_2=2$.
- $\tau=(1,3,2)$ and $\tau=(3,1,2)$: row $2$ is last, two neighbors used, $N_2=1$.

So across the six orders $N_2$ takes value $3$ twice, $2$ twice, $1$ twice — each of $\{1,2,3\}$ with probability $2/6=1/3=1/r_2$. Uniform, exactly as the position argument predicted. So the random order really does convert a quantity I couldn't control for any single order into a uniform one on average.

And it's uniform on $\{1,\dots,r_k\}$ *for every fixed matching $f$* — the argument never used anything about $f$ beyond which $r_k-1$ other rows own $k$'s neighbors, and the position of $k$ among that group is uniform regardless. So I don't even need to average over $f$ separately; for each $f$,
$$\mathbb{E}_\tau\big[\log N_k(\tau,f)\big]=\sum_{i=1}^{r_k}\frac{1}{r_k}\log i=\frac{1}{r_k}\sum_{i=1}^{r_k}\log i=\frac{1}{r_k}\log\!\big(1\cdot 2\cdots r_k\big)=\frac{1}{r_k}\log(r_k!),$$
using $\sum_{i=1}^{r_k}\log i=\log\prod_{i=1}^{r_k} i=\log(r_k!)$. Since this holds for every $f$, averaging over $f$ too leaves it unchanged: $\mathbb{E}_{\tau,f}[\log N_k(\tau,f)]=\frac{1}{r_k}\log(r_k!)$.

Now assemble. Each row $k$ contributes at most $\frac{1}{r_k}\log(r_k!)$ to the bound on $H(f)$, so
$$H(f)\;\le\;\sum_{k=1}^n \frac{1}{r_k}\log(r_k!).$$
And there's the inequality I needed on $H(f)$. Exponentiate, base $2$: since $f$ is uniform on the set $\mathcal{M}$ of perfect matchings, $H(f)=\log|\mathcal{M}|$, so
$$\log|\mathcal{M}|\le \sum_{k=1}^n \frac{1}{r_k}\log(r_k!)=\sum_{k=1}^n \log (r_k!)^{1/r_k}=\log\prod_{k=1}^n (r_k!)^{1/r_k},$$
and therefore
$$\operatorname{per}(A)=|\mathcal{M}|\le \prod_{k=1}^n (r_k!)^{1/r_k}.$$

Running the numbers back through the $3\times 3$ example: there $H(f)=\log_2 3\approx 1.585$ since $|\mathcal M|=3$. The bound on the right is $\frac12\log_2 2!+\frac13\log_2 3!+\frac12\log_2 2! = \frac12\cdot 1+\frac13\log_2 6+\frac12\cdot 1 = 1+\frac13(2.585)\approx 1.862$. So the entropy inequality reads $1.585\le 1.862$ — it holds, with about $0.28$ bits of slack. Exponentiating, $3\le 2^{1.862}\approx 3.63$, matching the $\prod (r_k!)^{1/r_k}\approx 3.63$ I computed at the very start. Consistent end to end. The slack is honest and worth understanding: it's not in any single inequality being crude, but in the gap between $H(f)$ and the sum of conditional bounds — on this graph the rows genuinely constrain each other less than the worst case, so the uniform-$N_k$ bound overcounts the available choices. The next check tells me exactly when that slack vanishes.

When does the slack vanish — i.e. is the bound ever an equality, or have I quietly over-shot and the true bound is smaller? Take the graph to be a disjoint union of complete bipartite pieces $K_{d,d}$, with $d$ dividing $n$, so every row has degree $d$. A single $K_{d,d}$ has exactly $d!$ perfect matchings, and with $n/d$ disjoint copies the matchings multiply, giving $(d!)^{n/d}$ in all; meanwhile the claimed bound is $\prod_{k=1}^n (d!)^{1/d}=(d!)^{n/d}$. Equality. Confirm the smallest instance directly: $K_{2,2}$ is the $2\times 2$ all-ones matrix, with $2!=2$ perfect matchings, so $H(f)=\log_2 2=1$; the bound is $\frac12\log_2 2!+\frac12\log_2 2!=\frac12+\frac12=1$. Equal, no slack — unlike the $3\times 3$ case above where the entropy fell $0.28$ bits short. So the inequality is tight, achieved by these block-diagonal all-ones configurations, and I have not over-shot. The contrast with the $3\times 3$ example pinpoints where the slack lives: in a complete block, once the earlier rows in the block are revealed the still-live columns are *exactly* the possible values for the current row, so the conditional-maximality bound $\log N_k$ is met with equality; in my $3\times 3$ graph, which is not a union of complete blocks, the live set sometimes strictly overcounts the actual support, and that overcount is the $0.28$ bits. So there is no avoidable looseness in the conditional-maximality step or in the random-order averaging — the bound is exactly as good as it can be, equality precisely on the complete-block configurations.
