I start with the suspicion that the large network is not only a solver. It may also be a search space. When a dense randomly initialized network trains successfully, I do not have to imagine that every parameter is essential or that the final dense trajectory is the only explanation. I can instead ask whether the dense initialization contains many possible sparse learners, and whether training reveals one whose own starting point is unusually good for gradient descent.

The pruning literature gives me the reason to ask this. A trained network can lose most of its weights and keep its accuracy after fine-tuning. That fact says the final function is compressible, but it does not say the compressed network is trainable. Post-training pruning hands me a sparse endpoint after the large model has already done the hard optimization work. If I train that sparse architecture from a fresh random initialization and it fails, I still do not know what failed. The architecture might be bad, the initialization might be bad, or the endpoint weights might simply be doing all the work.

So I separate the two things that pruning leaves tangled together. The mask is the sparse architecture: which connections survive. The weights are a history: where those connections started, where dense training moved them, and which final magnitudes caused them to survive. I need an experiment in which the mask is discovered by dense training but the surviving weights are not allowed to stay at their trained values. Otherwise I am only studying fine-tuning.

The important thought is that the original random values may be the missing variable. Han-style pruning already reports that retaining surviving weights works better than reinitializing layers. The usual reading is that the retained trained weights are close to a good solution. I want to test a stronger and stranger reading: perhaps the surviving connections were special from the beginning, not because their initial values solve the task already, but because those initial values make this sparse learner easy for the optimizer to train.

I therefore use dense training as a probe. I initialize the full network at theta_0 and train it to theta_j. I prune low-magnitude weights at theta_j to get a mask m. Then I refuse to keep theta_j for the survivors. I put each surviving connection back at the value it had in theta_0 and train the masked network by itself. If this network matches the dense model, I have not shown that a trained sparse endpoint can be reused. I have shown that the dense run uncovered a sparse learner that was trainable at its own initial values.

This reset is the move that overcomes the post-training pruning obstacle. Old pruning proves existence after learning; the reset asks about trainability before learning. The sparse network is no longer sitting at a dense-trained solution, so success cannot be explained as "the weights are already done." It also is not a new sparse architecture sampled blindly from the start, because the mask is selected by the dense run. The experiment uses training to identify the candidate and then removes the trained values to test what the candidate really is.

Now I can state the claim sharply. The dense network starts at theta_0 drawn from D_theta and reaches its best validation loss at iteration j with test accuracy a. I am looking for a binary mask m such that the masked network, initialized as m times theta_0 and trained with m fixed, reaches accuracy a' at its own best-validation iteration j', with j' no larger than j, a' at least a, and far fewer nonzero weights than the dense model. The sparse learner is not just a wiring diagram and not just a trained endpoint; it is the pair of a mask and the particular initial values under that mask.

The control has to be the same mask with a fresh random draw. If m with theta_0 works and m with theta'_0 works just as well, then the structure is enough and the initialization story collapses. If m with theta_0 learns faster and reaches higher accuracy while m with theta'_0 degrades as sparsity rises, then the causal object is the pair: this sparse wiring with these particular initial values. That is why the claim is more than "sparse networks work." The same sparse network shape can stop working when I change only the initialization.

I also need checks against easier explanations. If the selected initial values are already near their trained values, then I have only found hidden pretraining, so I should compare how far surviving and removed weights move during dense training. If only the marginal distribution of surviving initial values matters, then resampling new values from that distribution should recover the effect. If the secret is merely that initially small weights are useless, then pruning before training should be enough. These controls are not side issues; they decide whether the object is really the identity of each surviving connection together with its original value.

Now iterative pruning makes sense. One large cut asks a single final magnitude ranking to identify the whole sparse learner at once. I can do better by repeatedly training the current masked network, pruning a fraction of the survivors, and then resetting the remaining weights back to the same original theta_0. The mask accumulates, but the starting point stays anchored. Each round lets the current sparse regime reveal which of its surviving connections still matter. That is why gradual pruning can find smaller trainable subnetworks than a one-shot cut.

The pruning rule should be layer-wise. A global magnitude threshold can confuse scale differences across layers with importance and accidentally strip one layer too severely. Ranking within each layer keeps every layer represented while still removing the lowest-magnitude survivors there. The output layer is small, so I prune it more gently; otherwise a few unlucky deletions can damage the class readout before the hidden representation has been tested fairly. The pruning is unstructured because I want to know whether any sparse set of individual connections can carry the learner, not whether the result is immediately hardware-friendly.

For deeper networks, I should treat the learning rate as part of the trainability test. A sparse subnetwork reset to an early state may be more fragile under large first steps than the dense model. Warmup is the least invasive adjustment: it changes the early optimization shock without changing the mask, the original values, or the hypothesis about what object is being tested.

The final interpretation is that overparameterization buys attempts. A dense network contains an enormous number of subnetworks, each with a particular assignment of initial values. Most sparse learners sampled alone will not be lucky. But the dense model has so many overlapping candidates that the chance of containing a trainable one is much higher, and SGD plus pruning can expose it after the fact. This reframes successful overparameterized training as finding and training a hidden sparse learner whose initialization happens to be favorable.

So the result is not compression, not ordinary pruning, and not merely sparse training. Compression says the final function can be stored in fewer parameters. Ordinary pruning says a trained dense model can be trimmed. Sparse training says some small model can learn. The distinctive claim is that the dense random initialization already contains a sparse subnetwork that can learn in isolation when it keeps its original starting values, and that post-training magnitude pruning can reveal that subnetwork without letting the final dense weights explain the success.
