I start with the part that is tempting but wrong. I have a strong tabular learner and I can make it into an ensemble, so the easy move is to train several copies and average them. That helps with variance, but it does not answer the two finance-specific questions in front of me. Which rows should the next member pay attention to? Which columns should it see? If I leave both choices uniform, I get diversity, but it is blind diversity.

The row question is dangerous because the usual boosting instinct points in the right general direction and the wrong detailed direction. A row with high current loss might be a useful hard case, close to a real boundary or to a rare but repeatable pattern. It might also be an unforecastable shock. If I simply say high loss means high weight, I give the next model more incentive to memorize noise. But if I only down-weight all high-loss rows, I also lose the hard rows that actually teach the model something. So I need a second signal that separates hard-but-learnable from hard-because-noisy.

A boosted tree already gives me more than a final loss. While it is being trained, every row has a loss after tree 1, after tree 2, and so on. That path says something that the final value hides. If a row is useful to the decision surface, its relative loss should tend to improve as the learner accumulates structure. If a row is mostly noise, its relative loss may stay flat or rise because the model is not finding a stable pattern for it. So the candidate I want to try is a trajectory signal: not just how large the current loss is, but how the row's loss moves during the previous member's training.

Raw losses across iterations are not directly comparable, because the whole loss scale changes as trees are added. I should rank-normalize across samples at each iteration. Let C be the N by T loss-curve matrix for the previous member, and let norm(C) replace each column by percentile ranks. Then I summarize the beginning and ending of each row's path with C_start and C_end, the row means over the first and last ten percent of iterations. Ten percent is a stabilizer: it avoids making the decision from a single early or late tree while still comparing the two ends of the trajectory.

Now I have to fix the signs, and this is exactly the place where I do not trust my own intuition, so I want to put a small example on the page and read the numbers off it rather than asserting which way each term pushes. For the current ensemble loss vector L, suppose I use h1 = norm(-L), not norm(L). I will pin down what that does to the inverse weight in a moment. For the trajectory, h2 = norm(C_end / C_start): a descending relative-loss row has a small ratio, a flat or rising row has a ratio near one or above. The final weight will be 1/(decay^k * h_avg + 0.1), so smaller h means larger weight. Let me actually construct six rows of three kinds and compute the chain.

Take T = 10 trees. Two easy rows with low loss throughout; two hard-learnable rows with high loss that descends steeply relative to the others; two noisy rows with high loss that stays high or drifts up. After column-rank-normalizing C and taking the first/last 10% means, the start and end ranks come out:

- easy:        l_start ≈ [0.17, 0.33], l_end ≈ [0.17, 0.33] — ratio ≈ [1.0, 1.0]
- hard-learn:  l_start ≈ [1.00, 0.83], l_end ≈ [0.50, 0.67] — ratio ≈ [0.5, 0.8]
- noisy:       l_start ≈ [0.67, 0.50], l_end ≈ [0.83, 1.00] — ratio ≈ [1.25, 2.0]

So h2 = rank(ratio) ≈ easy [0.58, 0.58], hard [0.17, 0.33], noisy [0.83, 1.0]. With current losses L = [0.10, 0.13, 0.55, 0.58, 0.83, 0.86], h1 = rank(-L) ≈ [1.00, 0.83, 0.67, 0.50, 0.33, 0.17]. Then h = h1 + h2 and w = 1/(0.5^1 · h + 0.1):

- easy:        h ≈ [1.58, 1.42] → w ≈ [1.12, 1.24]
- hard-learn:  h ≈ [0.83, 0.83] → w ≈ [1.94, 1.94]
- noisy:       h ≈ [1.17, 1.17] → w ≈ [1.46, 1.46]

That is the ordering I wanted and it is not the one pure boosting would give. The hard-learnable rows get the largest weight, 1.94, because both terms are small for them: high current loss makes h1 small, and a descending curve makes h2 small. The easy rows get the smallest weight. And the noisy rows land in the middle at 1.46 — above the easy rows but well below the hard-learnable ones. I want to be honest that this is what the formula does: it does not push noisy rows below the easy rows, it just stops them from running away. The mechanism is the partial cancellation: a noisy row has small h1 (high loss wants to upweight it) but large h2 (the rising curve wants to downweight it), and the two terms fight to a draw in the middle of the pack.

To see that the trajectory term is doing real work, I turn it off and reweight on h1 alone, which is the AdaBoost reflex of "weight grows with current loss." With w = 1/(0.5 · h1 + 0.1) the same six rows give easy [1.67, 1.94], hard-learn [2.31, 2.86], noisy [3.75, 5.45]. Now the noisy rows get the largest weights by a wide margin — 5.45 versus 1.94 for an easy row — and the next member would spend most of its capacity chasing exactly the rows it cannot learn. That is the failure I was worried about, and it shows up numerically the moment I drop h2. So the combination h = alpha1·h1 + alpha2·h2 is what makes the difference, not either term alone.

I still do not want one outlier row to set an extreme weight. I bin the h values into B bins and use the mean h inside each bin. If row i falls in bin b_i, the weight is w_i = 1 / (decay^k · mean_h[b_i] + 0.1). The +0.1 prevents the denominator from approaching zero. The decay^k factor, with decay between 0 and 1, is meant to make later members less aggressive, and I should check that it actually anneals rather than just rescaling. Using the six h-values above and the ratio of largest to smallest weight (which is scale-free, so it is the honest thing to track): at k=1 the ratio max/min is 1.73, at k=2 it is 1.61, at k=3 it is 1.46, at k=6 it is 1.10. So as k grows the weights compress multiplicatively toward each other — the spread between rows shrinks from 73% down to 10%. The absolute weights all grow (the mean rises as decay^k shrinks the denominator), so I should not say the weights "become uniform," but their relative spread does collapse, which is the property I wanted: early members reweight sharply, later members are close to flat and stop escalating the chase.

That settles the row side. The feature side has a different failure mode. If I rank features once and always keep only the strongest ones, every member learns the same regime-specific vocabulary. If the current market regime makes a factor look weak, a hard cutoff may discard a factor that matters later. I want feature subsets that prefer useful columns but still preserve diversity.

The cheapest model-agnostic importance test is to perturb one column and see how much the current ensemble's loss changes. For feature f, I shuffle that column across training samples to form X_f, predict with the current ensemble, and compute L_f. The natural statistic is the mean loss increase, but a feature with a large but erratic effect should not outrank a feature with a smaller, dead-reliable effect, so I divide by the spread: g_f = mean(L_f - L) / (std(L_f - L) + epsilon). I want to confirm this separates a relied-on feature from a useless one with the right sign and magnitude, so I simulate it: a relied-on feature whose shuffle adds a consistent +0.10 to per-row loss gives g ≈ 4.7, while a useless feature whose shuffle only adds zero-mean noise gives g ≈ 0.01. The std normalization is pulling its weight here — the strong feature scores high precisely because its effect is consistent, not merely large, and a noisy-but-large shift would be deflated by its own standard deviation.

The selection rule should not be a top-k cutoff. I bin features by g-value and sample a fixed fraction from each bin, with the largest fraction for the highest-g bin and smaller but nonzero fractions for weaker bins. The ceiling in ceil(r_d · N_d) matters because a small bin should not vanish just because its expected sample count is fractional. The result is a member that mostly sees strong features, but also sees a tail of weaker or dormant features. That gives each member a different column view without pretending weak-current-regime features are useless forever.

The training loop is now straightforward. Start with uniform row weights and all features. Train member k on the current weights and feature subset. If it is not the last member, retrieve the previous member's per-tree loss curve, compute the current ensemble's training loss, update the next row weights with the binned inverse h formula, and choose the next feature subset with shuffled-column g-values. Prediction discards all of this training-time machinery: each stored member predicts on the feature subset it was trained on, and the wrapper averages the member predictions, optionally with configured member weights.

The simple combination rule is an average over members. If I later expose member weights, the safest default remains all ones; the feature-selection score should still be read as the current ensemble's average reliance unless I deliberately redesign it for weighted members. The sample-reweighting routine is the place where a sign mistake would be fatal, and the six-row trace above is exactly the unit test I would keep: rank(-loss_values), rank(C_end / C_start), and the inverse denominator 1 / (decay**k * h_avg + 0.1), checked to give hard-learnable rows the top weight and noisy rows the middle.

So the construction that survives all of this is a sequential ensemble with two between-member updates. The sample update uses the final ensemble loss only to say where error remains, then uses the previous member's loss trajectory to avoid confusing noisy rows with useful hard rows. The feature update uses shuffled-column loss changes to estimate reliance, then samples across reliance bins instead of collapsing to one fixed subset. The constants are deliberately conservative: rank normalization for robustness, first and last ten percent windows for stable trajectory endpoints, binning for regularized weights/importances, +0.1 for a denominator floor, decay in [0, 1] for later-member annealing, and nonzero feature sampling ratios so diversity remains present all the way down the bins.
