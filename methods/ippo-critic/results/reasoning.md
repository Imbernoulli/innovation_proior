Let me start from what actually goes wrong when I try to train a team of agents to cooperate. I have `N` agents in a partially observable world, they all share one scalar reward `r(s,u)`, and at execution each agent `a` can only see its own local observation `z^a` — its own action-observation history `τ^a` — so the policies have to be decentralisable: `π^a(u^a|τ^a)`, no peeking at the others. I want to maximise the team return `J = E[Σ_t γ^t r_t]`. The execution constraint on the *actor* is non-negotiable. But training happens in simulation, so during training I'm allowed to use things no agent could see at run time: the global state `s`, everyone's observations, shared gradients, shared weights. That's the CTDE setting, and it gives me a degree of freedom: the actor must be decentralised, but the *critic* — the value function I use as a baseline — can in principle condition on anything I have during training. So the real question I'm staring at is narrow and sharp: what should the critic condition on, and how should I build it? The actor's form is settled; the critic's input set is the one free variable, and I have a feeling it decides everything.

Why does it decide everything? Because the whole reason I have a critic is variance. The policy gradient I'd actually use is the score-function estimator `ĝ = E[∇ log π(u^a|τ^a) Â]`, and that thing is unbiased but brutally noisy. The cure is to subtract a baseline and feed the advantage `Â` in its place — that's the actor-critic pattern, and it stays unbiased as long as the baseline doesn't depend on the action. The variance of the whole update is then dominated by the variance of `Â`, and `Â` is built out of the critic. So the critic *is* the learning signal's quality. A choice of critic input that makes `Â` noisy or biased poisons every actor update. That's why the input set isn't a detail.

Now, the single-agent machinery I'd carry in is mature. TRPO gave me approximately-monotone improvement by maximising `E[(π_θ/π_θold) A]` under a KL trust region `E[KL]≤δ`, but enforcing that constraint needs conjugate gradient on Hessian-vector products — heavy, and it fights with parameter sharing. PPO keeps much of TRPO's empirical behaviour with first-order updates by clipping the ratio instead: with `r_t = π_θ(u_t|·)/π_θold(u_t|·)`, the surrogate is `L^CLIP = E[min(r_t Â_t, clip(r_t,1-ε,1+ε) Â_t)]`. I should make sure I understand *why* that min is there and not just copy it. If `Â_t > 0`, the unclipped term wants to raise `r_t`, and after `r_t` passes `1+ε` the clipped term is smaller, so the `min` flattens the reward for pushing probability higher. If `Â_t < 0`, improving the surrogate means making `r_t` smaller, and after `r_t` drops below `1-ε` the clipped product is more negative, so the `min` again chooses the clipped value and removes the reward for pushing farther. On the opposite side in each case, where the move makes the objective worse, the unclipped term remains in force. So the objective is pessimistic in the precise sense I need: it ignores a ratio excursion only when the excursion would improve the surrogate, and it counts the excursion when it would hurt. The advantage `Â` itself comes from TD residuals. On a finite rollout segment, GAE is `Â_t = Σ_{l=0}^{T-t-1} (γλ)^l δ_{t+l}` with `δ_t = r_t + γV(x_{t+1}) - V(x_t)`, where `x_t` is whatever input I choose for the critic. Small `λ` leans on short bootstraps, so variance drops and value-estimation bias matters more; large `λ` leans toward longer returns, so bootstrap bias drops and variance rises. At `λ=1`, the residual sum telescopes: `Σ_l γ^lδ_{t+l} = Σ_l γ^l r_{t+l} + γ^{T-t}V(x_T) - V(x_t)`. A practical `n`-step target uses the same bootstrapped-return shape, `R_t^{(n)} = Σ_{l=0}^{n-1} γ^l r_{t+l} + γ^nV(x_{t+n})`, and the corresponding advantage is `R_t^{(n)} - V(x_t)`. Fine. All of that is the actor side and the advantage estimator. None of it tells me what `V` conditions on. That's still open.

So let me think about the candidate critics, starting from the obvious one everybody reaches for. The prevailing wisdom in this field is: more centralised information in the value function is better. The argument is seductive — at training time I have the global state `s`, so let the critic be `V(s)`, or a critic over the concatenation of everyone's observations, or over all observations and actions. The premise is that the non-stationarity that wrecks naive independent learners comes from each agent treating the others as part of a moving environment, and a critic that *sees* the others removes that moving target from the critic's view. That's the Central-V / COMA / MADDPG line. COMA even adds a counterfactual baseline to sharpen credit assignment. And because the critic is thrown away at execution, the actors stay decentralisable. At first glance this dominates independent learning. Let me try to take it at face value and build it: a shared critic `V_φ(s)`, fed the global state, broadcast to all `N` agents.

But two things start to itch, and I want to chase them down before I commit, because the field treats "centralise the critic" as if it were costless and I don't think it is.

First itch: variance. The actor I'm training for agent `a` acts on `τ^a`. If the critic conditions on the whole picture, including other agents' histories, observations, or actions, then the policy-gradient update for agent `a` still has to average over the variables that are outside `a`'s own actor input. Sometimes that averaging is implicit in the sampled trajectories; sometimes it is explicit, as in a counterfactual baseline that marginalises over actions. Either way, extra centralized variables can raise the variance of the per-agent actor update, even when they make the value target itself easier to fit. So the centralised critic is not a monotone improvement in the statistical sense I care about. It can lower some approximation error by seeing more, but it can also pay for that with higher update variance and worse scaling.

Second itch, and this one is sharper: bias from partial observability. I almost slid past it. The actor acts on `τ^a` because the world is genuinely partially observable — that's the whole premise of a Dec-POMDP. But a critic keyed on the ground-truth state `s` does *not* see partial observability; it sees the truth. Is that good? Picture the smallest case that exposes it — the Dec-Tiger-style situation: a couple of underlying states, but the agent only gets noisy observations, so there are exponentially many histories. Within one state `s`, many different histories `h` map to it, with genuinely different values: a history in which I've heard the same evidence ten times in a row *should* have a higher value than the empty history, because I'm now confident and about to act well. The history value `V(h)` must rise as evidence accumulates. But `V(s)` is a single number for that state — it is forced to be the average `E_{h|s}[V(h)]` over all those histories. It literally cannot represent the climb. Worse, an information-gathering action — listen again, scout — increases `V(h)` but doesn't change `s`, so to a state critic it looks *worthless*; the agent would learn never to gather information. So `V(s)` is not just lossy, it's *biased relative to the objective the actor is solving*, which is a history-conditioned objective. The "centralise on the state" move I was about to make doesn't reduce bias — for the partially observable problem I actually have, it can *introduce* bias. That is the wall. Centralising the critic on the bare state trades the actor's information set for the wrong one, and it raises variance besides.

Let me make the bias point precise so I trust it. What the actor needs is the history value `V^π(h) = E[Σ γ^t r_t | h]`. If I had a critic on (history, state) jointly, `V(h,s)`, then I could recover the history value exactly by averaging out the state I don't have at execution: `V^π(h) = E_{s|h}[V^π(h,s)]` — that's unbiased, no history information lost. But if I throw away `h` and keep only `s`, I'm averaging over the wrong variable, `E_{h|s}`, and there's no way to undo that. So the unbiased options are critics that *retain the history information the actor uses*; the state-only critic is the biased one. That reframes the centralisation question entirely. The thing that matters isn't "does the critic see global info," it's "does the critic keep the information set the actor conditions on." A state critic fails that test.

So both itches point the same way: pulling in the global state and dropping the history is, for this problem, a bad trade — more variance, and bias. Then let me follow the contrarian thread all the way: what if the critic refuses global state and peer information and stays on agent `a`'s local information stream? The ideal POMDP baseline would be `V_φ(τ^a)`; the concrete code slot I have implements that choice as a value on the current local observation plus agent id, `V_φ(z^a, a)`. No global state, no peer observations, no peer actions. Each agent gets its own local value estimate as its baseline.

My first reaction is that this should be obviously worse — it's "independent learning," and independent learning is supposed to be the thing that fails. So let me argue against it honestly and see if it survives. The classic objection is non-stationarity: if agent `a`'s critic sees only `z^a`, and the other agents are simultaneously learning and changing their policies, then from `a`'s view the transition and reward dynamics drift, the critic's targets go stale, and convergence guarantees evaporate (Tan, 1993). There's a second classic objection: an independent learner can't always distinguish environment stochasticity from a peer's exploration, which provably blocks optimal play in some coordination matrix games (Claus & Boutilier, 1998). Those are the reasons independent actor-critic and independent Q-learning are observed to be unstable and to land in poor policies on cooperative benchmarks. So the local-only critic inherits a genuine instability problem. I can't wave that away.

But wait — who's responsible for stability here? Let me separate the two jobs. The critic's job is to give a low-variance, low-bias baseline. Stability of the *policy update* is a different job, and PPO attacks that with the clipped ratio objective. The non-stationarity disaster in independent learning is specifically that a single agent, reacting to a co-learner's shift, takes a catastrophic policy step; once one agent lurches, it shifts the dynamics for everyone, and the whole thing spirals. The clip does not prove that the realized policy ratio can never move outside the band, but on the sampled actions it removes the objective incentive to push farther once a probability ratio has crossed the useful side of `[1-ε,1+ε]`. That is exactly the kind of restraint I need against peer-induced surprises. So the mechanism that motivated centralising the critic — taming non-stationarity — might already be handled mostly by the optimiser, not by the critic's input. If clipping keeps every agent's effective update modest, then from any one agent's perspective the others are a more slowly drifting environment, slow enough that a local critic may track it. The instability that condemns IAC/IQL might be an artifact of unrestrained updates, not of the local critic per se. That's a testable distinction, and it's the crux: is independent learning's failure intrinsic here, or is it that the older independent learners lacked a proximal per-agent step? If it's the latter, a local critic plus PPO clipping could be fine — and I'd get the cleaner information set without the centralised critic's variance cost.

I want one more line of reasoning before I believe a purely local critic can be *principled* and not just "happens to work because of clipping." Let me see whether a per-agent local update even has an improvement guarantee. Take three agents for concreteness, with policies `(π^1,π^2,π^3)`, and telescope the joint return:

```
J(π^1_{t+1},π^2_{t+1},π^3_{t+1}) − J(π^1_t,π^2_t,π^3_t)
 = [ J(π^1_{t+1},π^2_{t+1},π^3_{t+1}) − J(π^1_t,π^2_{t+1},π^3_{t+1}) ]
 + [ J(π^1_t,π^2_{t+1},π^3_{t+1}) − J(π^1_t,π^2_t,π^3_{t+1}) ]
 + [ J(π^1_t,π^2_t,π^3_{t+1}) − J(π^1_t,π^2_t,π^3_t) ].
```

Three difference terms, each one changing a *single* agent's policy while the others are held fixed. If every one of these is non-negative, the joint return improves. Look at the last term: it changes only `π^3`, holding `π^1,π^2` fixed. Holding the other agents fixed, the world that agent 3 faces is just a single-agent decision process — a POMDP, call it `M^3`, in which `π^1,π^2` are absorbed into the dynamics. And agent 3's policy is allowed to depend only on its own observation `τ^3`, which is exactly a POMDP policy. So `J(π^1_t,π^2_t,π^3_{t+1}) − J(π^1_t,π^2_t,π^3_t) ≥ 0` is just "improve a single-agent policy on the induced POMDP `M^3`." A method with a genuine local monotone-improvement guarantee, like the trust-region ideal, can make that term non-negative. PPO is only a first-order clipped approximation, so this is not a theorem for the exact practical update, and if I update the agents sequentially the cleanest version would recollect data after each coordinate move. But the identity still tells me what structure I should want: each agent should take a restrained single-agent improvement step against the induced POMDP created by the others. Vanilla actor-critic and Q-learning give no such per-step improvement story, so their coordinate terms can go down; PPO at least approximates the trust-region behaviour that would make the coordinate-ascent picture coherent. And crucially, for this argument the critic only needs to be a good baseline *for agent `a`'s induced single-agent POMDP `M^a`*. The natural baseline for a POMDP policy that conditions on `τ^a` is a value function on that same information set, approximated here by the local observation/history representation. The local critic isn't a compromise forced by execution; it is the baseline aligned with the coordinate step's information set. The picture clicks: the local critic and PPO clipping aren't two unrelated choices, they're the two halves of "take restrained single-agent improvement steps on the right per-agent information set."

(I notice the same telescoping can coexist with a central critic plus PPO — nothing in the identity forbids using extra training information in a baseline for `M^a`. So the coordinate-ascent view doesn't force local; it permits both. The reason to prefer local here is the bias-variance argument above: a bare state critic loses history information under partial observability, and richer central inputs can raise the variance and dimensionality of the actor update. The coordinate-ascent view tells me local is a coherent choice; the bias-variance argument tells me why I should choose it.)

There's one more reason this isn't a step down from CTDE that I'd been treating as a downside but is actually a feature. People assume centralised-critic methods are strictly stronger than fully decentralised training. But if all agents learn concurrently with the same algorithm, updating on the joint data at the same time, the joint policy gradient *equals* the sum of the per-agent decentralised gradients — they're the same update (Peshkin et al., 2000). So a parameter-shared, concurrently-trained set of local learners isn't some weak cousin of the centralised method; in the policy-gradient case it's computing the same gradient direction, just without paying the variance and bias costs of routing everything through a centralised critic. The "decentralised" critic is closer to the centralised one than the folklore admits.

OK. I'm now convinced the critic should be `V_φ(z^a)` over local observation only. Let me design the actual module, because there are a few choices left and I want each one motivated, not defaulted.

Parameter sharing. Do I learn `N` separate critics or one shared `V_φ` used by all agents? If the agents are homogeneous — same observation and action spaces, as in the unit-micromanagement maps — then a separate network per agent wastes data: each net only learns from one agent's stream. Sharing one critic lets every agent's experience update the same parameters, so I get `N`-fold more data per gradient step and faster, more stable value learning. The cost is that a shared net can't specialise per agent. But the agents already differ in *behaviour* because they see different observations and histories, so a shared function evaluated at different inputs still produces different values. The only thing sharing strictly removes is *index-based* specialisation — the ability to say "I am agent 3, and agent 3 should value this state differently for a reason not visible in the observation." I can hand that back cheaply: append a one-hot agent id to the critic's input. Then the shared network can, if the data demands it, route different agents to different value functions, while still pooling all the data. So: one shared critic, input = local observation concatenated with an agent one-hot. That's the design — maximal data efficiency from sharing, with an escape hatch for per-agent specialisation through the id, and no loss of the local information set.

The architecture itself. The input is a per-agent feature vector `[z^a ; one-hot(a)]` of width `obs_dim + N`. I want a scalar local value estimate. A small MLP is the right tool: two hidden layers with a nonlinearity, then a linear head to one output. ReLU between the layers is cheap and gives well-behaved gradients for this regression. This is not a theorem that the current observation is sufficient for every Dec-POMDP; it is the reference implementation's local-observation critic. If history is supplied by stacked observations, it enters through `z^a`; if the actor is recurrent, the critic still stays local but does not reuse the actor's hidden state. So: `fc1: (obs_dim + N) → H`, ReLU; `fc2: H → H`, ReLU; `fc3: H → 1`. In the reference config `H = 128`.

Now the shapes, because the learner is going to call this over whole sequences and it's particular about the trailing dimension. The batch carries `obs` of shape `(B, T, N, obs_dim)` — batch of episodes, time, agents, features. The learner calls `critic(batch)` with no time index to get values for the whole sequence, and afterwards does `.squeeze(3)`, so I must return `(B, T, N, 1)` — the trailing singleton is mandatory, it's the channel the squeeze removes. It can also call `critic(batch, t)` for a single timestep, in which case I slice time to a window of length 1. So inside `forward` I compute the time extent: if `t` is `None` it's the full `max_seq_length`, else it's 1, and I take a time slice `ts = slice(None)` or `slice(t, t+1)` accordingly. I pull `obs` over that slice — `(B, T, N, obs_dim)`. The agent one-hot is just the `N×N` identity, broadcast across batch and time: `eye(N)` shaped to `(1,1,N,N)` and expanded to `(B, T, N, N)`. Concatenate along the last axis to get `(B, T, N, obs_dim + N)`, push it through the MLP, and the last layer gives `(B, T, N, 1)`. Because every operation is along the feature axis and the agent axis is just a batched dimension, the same shared weights are applied independently to each agent's feature vector — which is exactly the per-agent, parameter-shared, no-cross-agent-mixing critic I argued for. No global state ever enters; `batch["state"]` is simply never read.

I should double check that this critic carries no peer information by accident. The only inputs are `obs[:, :, a, :]` for each agent `a` and the one-hot id of `a`. The id is a constant per agent, not data about peers. So agent `a`'s value depends only on `z^a` and the constant `a`; it uses no information unavailable to that agent at execution, even though the learner may discard the critic after training. Good; the critic stays on the local side of the CTDE line.

Then the rest of the loop is the standard PPO-in-MARL training I already have, and I want to be explicit about how the local critic plugs in. The shared reward is the team reward `r_t`; with a common-reward setup it is broadcast to all `N` agents so each agent's return target uses the same reward stream. In the GAE form, the per-agent residual is `δ^a_t = r_t + γ V_φ(z^a_{t+1}) − V_φ(z^a_t)` and `A^a_t = Σ_{l=0}^{T-t-1}(γλ)^lδ^a_{t+l}`. In the practical EPyMARL learner I am matching, the target critic instead builds a fixed `q_nstep=5` target, essentially `R_t^{a,(n)} = Σ_{l=0}^{n-1} γ^l r_{t+l} + γ^n V_{\bar φ}(z^a_{t+n})` when the horizon and masks allow bootstrapping, and the advantage handed to the actor is the masked TD error `R_t^{a,(n)} - V_φ(z^a_t)`. That is the same bootstrapped-return family as the `λ=1` telescoping case, but it is not GAE with geometric `λ` weights. A target copy of the critic produces the bootstrap values, updated softly with `τ=0.01`, so the regression target doesn't chase its own tail. The actor update is the clipped surrogate on each agent's *own* ratio `r^a = π_θ(u^a|τ^a)/π_θold(u^a|τ^a)`, `min(r^a A^a, clip(r^a,1-ε,1+ε) A^a)` with `ε=0.2`, plus entropy coefficient `0.001`; the critic is fit by masked mean-squared regression to those `n`-step targets. In this code path there is no value-clipped loss and no GAE computation; rewards are standardised, returns are not, Adam uses `lr=3e-4`, and gradient norms are clipped at `10`. Everything multi-agent-specific lives in *what the critic conditions on* and in *clipping each agent's own ratio* — the rest is single-agent PPO run `N`-fold with shared weights.

Let me write the critic module exactly as it drops into the harness — the one slot the framework left open:

```python
import torch as th
import torch.nn as nn
import torch.nn.functional as F


class ACCritic(nn.Module):
    def __init__(self, scheme, args):
        super(ACCritic, self).__init__()

        self.args = args
        self.n_actions = args.n_actions
        self.n_agents = args.n_agents

        input_shape = self._get_input_shape(scheme)
        self.output_type = "v"

        self.fc1 = nn.Linear(input_shape, args.hidden_dim)
        self.fc2 = nn.Linear(args.hidden_dim, args.hidden_dim)
        self.fc3 = nn.Linear(args.hidden_dim, 1)

    def forward(self, batch, t=None):
        inputs, bs, max_t = self._build_inputs(batch, t=t)
        x = F.relu(self.fc1(inputs))
        x = F.relu(self.fc2(x))
        q = self.fc3(x)
        return q

    def _build_inputs(self, batch, t=None):
        bs = batch.batch_size
        max_t = batch.max_seq_length if t is None else 1
        ts = slice(None) if t is None else slice(t, t + 1)
        inputs = []
        inputs.append(batch["obs"][:, ts])
        inputs.append(
            th.eye(self.n_agents, device=batch.device)
            .unsqueeze(0)
            .unsqueeze(0)
            .expand(bs, max_t, -1, -1)
        )
        inputs = th.cat(inputs, dim=-1)
        return inputs, bs, max_t

    def _get_input_shape(self, scheme):
        input_shape = scheme["obs"]["vshape"]
        input_shape += self.n_agents
        return input_shape
```

Let me trace the causal chain back. The actor has to be decentralised on local histories, and the one free choice was the critic's input set, which fixes the variance and bias of every advantage. The reflex move — centralise the critic on the global state to kill the non-stationarity that wrecks independent learners — turns out to be a tradeoff, not a free improvement: central inputs can raise the variance and dimensionality of the per-agent update, and a state-keyed critic is biased relative to the history-conditioned objective when it averages over histories mapping to the same state. A history-state critic can be unbiased because `V(h)=E_{s|h}[V(h,s)]`, but the bare state critic goes the wrong way, `V(s)=E_{h|s}[V(h)]`, and loses the information-gathering value. That pushes me toward a local critic, with the implemented artifact using `V_φ(z^a, a)` rather than global state. The standard objection, that local independent learners are unstable under co-learner non-stationarity, is not dismissed; it is reassigned to the optimiser. PPO's clipped ratio objective gives each agent a restrained per-agent policy step, and the coordinate-ascent identity shows why restrained single-agent improvements can compose into joint improvement, while also reminding me that practical simultaneous updates and no recollection are approximations. Parameter sharing pools all agents' data into one critic for data efficiency, and a one-hot agent id restores per-agent specialisation without adding peer or global information. The module is the shared three-linear-layer `ACCritic`: it computes `input_shape = obs_dim + n_agents`, reads only `batch["obs"]`, appends an expanded identity matrix for the agent id, returns `(B,T,N,1)`, and leaves the global state untouched.
