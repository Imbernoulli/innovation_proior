# Stack-Augmented Recurrent Network (Stack RNN), distilled

Stack RNN couples a recurrent controller to a **differentiable stack** the controller learns to
operate. At each step the controller emits a softmax distribution over the stack actions
{PUSH, POP, NO-OP} and a new top-of-stack value; the next stack is the probability-weighted
**superposition** of the push-shifted, pop-shifted, and unchanged stacks. Because the whole
thing is continuous it trains end to end with backpropagation and plain SGD — no supervision on
the actions — and because the stack grows with the input, the learned controller is a finite
program acting on unbounded memory, which is what lets it length-generalize where a fixed-width
RNN or LSTM can only memorize the trained lengths.

## Problem it solves

Next-symbol prediction on streams generated by short deterministic algorithms (`a^n b^n`,
`a^n b^n c^n`, `a^n b^{2n}`, `a^n b^m c^{n+m}`, well-nested brackets, memorization, binary
addition), where the model must learn the *generating rule* and predict the deterministically
forced symbols on sequences strictly longer and deeper than any seen in training. A simple RNN
encodes the whole prefix in a fixed `R^m` vector and so memorizes rather than counts; an LSTM can
count but has no last-in-first-out memory; a discrete external stack has the right structure but
no usable gradient.

## Key idea

Relax the discrete stack so it is differentiable. Keep the stack as a vector `s_t` of cells, top
at index 0, depth that can grow on demand. The controller produces an action distribution from
the hidden state and the next stack is a convex blend of the three pure outcomes:

- **PUSH** shifts every cell *down* and writes a new value on top.
- **POP** shifts every cell *up*; the freed bottom reads an empty sentinel.
- **NO-OP** leaves the stack unchanged.

Each new cell is the action-weighted superposition of the cell above it (PUSH), the cell at it
(NO-OP), and the cell below it (POP) from the previous step. Saturating the softmax to a corner
recovers an exact hard stack, so training drives the soft stack toward discrete behavior on its
own; at test time the actions are rounded (argmax) to remove numerical drift on long sequences.

## Final model

Controller (simple RNN), with the top `k` stack elements fed back (`k = 2`; on toy tasks the
plain hidden recurrence `R` is set to 0 so all long-range memory flows through the stack):

```
a_t = softmax(A h_t)                                  # A in R^{3 x m}, actions PUSH/POP/NO-OP
h_t = sigmoid(U x_t + R h_{t-1} + P s_{t-1}^k)        # P in R^{m x k}, s^k = top k cells
y_t = softmax(V h_t)
```

Stack update (top cell and deeper cells `i > 0`):

```
s_t[0] = a_t[PUSH] * sigmoid(D h_t) + a_t[POP] * s_{t-1}[1] + a_t[NO-OP] * s_{t-1}[0]
s_t[i] = a_t[PUSH] * s_{t-1}[i-1]   + a_t[POP] * s_{t-1}[i+1] + a_t[NO-OP] * s_{t-1}[i]
```

The pushed value `sigmoid(D h_t)` is bounded into `(0, 1)` so repeated pushes stay stable; the
empty / past-the-bottom slot holds the sentinel constant `-1`, chosen outside `(0, 1)` so the
controller can detect an empty stack (e.g. on `a^n b^{2n}` the second stack pops only after the
first reads empty). For richer per-slot state the cell is a vector and `D` a matrix; the
superposition is identical.

## Design choices and why

- **Softmax over the three actions** (not independent sigmoids): the actions are mutually
  exclusive, so a distribution on the simplex makes the stack update a convex *interpolation*
  rather than an arbitrary rescaling.
- **Superposition instead of a hard choice**: a discrete PUSH/POP has zero gradient almost
  everywhere, so SGD cannot learn the controller; the soft blend is differentiable and contains
  the hard stack as its saturated limit.
- **Bounded pushed value (`sigmoid`/`tanh`)**: keeps stack entries in a fixed range under repeated
  pushes and re-blends; an unbounded value could diverge over long sequences.
- **Empty sentinel `-1` outside the push range**: gives popping-past-bottom a defined value and
  lets the controller key off "stack empty."
- **Feed `k = 2` top cells back**: a small window of stack context makes the action discipline
  easier to learn than the single top alone.
- **Multiple parallel stacks (interacting through the hidden layer)**: one stack does only one
  action per step; patterns needing two coordinated counts (`a^n b^m c^{n+m}`, binary addition)
  need several stacks, which together approach Turing-complete expressivity.
- **NO-OP**: lets a stack idle while another works (essential for the coordinated multi-stack
  patterns); omitted for pure counting, which never needs to wait.
- **Length generalization**: the update rule is identical at any depth and the stack grows on
  demand, so the learned finite controller recognizes arbitrarily long/deep strings — unlike a
  fixed-width hidden vector, which must memorize.

## Training

Backpropagation through time (truncated to ~50 steps), hard gradient clipping, initial learning
rate ~0.1 halved when validation entropy stalls, and a curriculum that grows `n` during training.
On harder patterns plain SGD stalls in a shallow bigram-entropy local minimum, so a search is
wrapped around it (random restarts; more elaborate search helps further). At test time the
controller is discretized (argmax action) to eliminate continuous-stack drift on long inputs.

## Working code

A simple-RNN controller with a differentiable stack, dropped into a next-token harness. Stack
cells are vectors; the up/down shifts are fixed band matrices applied with one matmul; the new
top is injected into row 0; the three candidate stacks are blended by the action probabilities.

```python
import torch
import torch.nn as nn


class StackRNN(nn.Module):
    """RNN controller + differentiable stack with softmax PUSH/POP/NO-OP actions.
    Next stack = probability-weighted superposition of the push-shifted,
    pop-shifted, and unchanged stacks. Trainable end to end by backprop."""

    def __init__(self, vocab, hidden, stack_dim, stack_depth):
        super().__init__()
        self.hidden = hidden
        self.stack_dim = stack_dim
        self.stack_depth = stack_depth
        self.embed = nn.Embedding(vocab, hidden)
        # controller cell over [embed, top_of_stack, prev_hidden] -> hidden
        self.cell = nn.Linear(hidden + stack_dim + hidden, hidden)
        # 3 action logits (PUSH / POP / NO-OP) + new-top candidate vector
        self.stack_update = nn.Linear(hidden, 3 + stack_dim)
        self.head = nn.Linear(hidden + stack_dim, vocab)

        # push: row i <- row i-1 (shift DOWN, top freed); pop: row i <- row i+1 (shift UP)
        push_shift = torch.zeros(stack_depth, stack_depth)
        pop_shift = torch.zeros(stack_depth, stack_depth)
        if stack_depth > 1:
            eye = torch.eye(stack_depth - 1)
            push_shift[1:, :-1] = eye
            pop_shift[:-1, 1:] = eye
        top_slot = torch.zeros(1, stack_depth, 1)   # row 0 receives the new top
        top_slot[:, 0, :] = 1.0
        self.register_buffer("push_shift", push_shift, persistent=False)
        self.register_buffer("pop_shift", pop_shift, persistent=False)
        self.register_buffer("top_slot", top_slot, persistent=False)

    def forward(self, input_ids):
        B, T = input_ids.shape
        device = input_ids.device
        h = torch.zeros(B, self.hidden, device=device)
        stack = torch.zeros(B, self.stack_depth, self.stack_dim, device=device)
        embeddings = self.embed(input_ids)
        states = []
        for t in range(T):
            emb = embeddings[:, t]
            top = stack[:, 0]                                    # feed top back to controller
            h = torch.tanh(self.cell(torch.cat([emb, top, h], dim=-1)))
            update = self.stack_update(h)
            action_logits, push_raw = update[:, :3], update[:, 3:]
            a = torch.softmax(action_logits, dim=-1).view(B, 3, 1, 1)
            push_p, pop_p, noop_p = a[:, 0], a[:, 1], a[:, 2]    # convex action weights
            new_top = torch.tanh(push_raw)                       # bounded pushed value

            pushed = torch.matmul(self.push_shift, stack) + self.top_slot * new_top.unsqueeze(1)
            popped = torch.matmul(self.pop_shift, stack)
            stack = push_p * pushed + pop_p * popped + noop_p * stack   # superposition
            states.append(torch.cat([h, stack[:, 0]], dim=-1))
        return self.head(torch.stack(states, dim=1))
```
