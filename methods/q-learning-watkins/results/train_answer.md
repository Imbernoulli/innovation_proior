An agent observes a state, chooses an action, collects a reward, and lands in a new state, and I want it to learn behavior that maximizes the expected discounted return from every state with discount $0 < \gamma < 1$. The environment is a finite controlled Markov process, but the catch is that the reward means $\rho(x,a)$ and the transition probabilities $P_{xy}[a]$ are not given to me. I only see the raw stream of sampled transitions. Classical dynamic programming already solves the known-model version of this problem: for a fixed policy the value of a state is the expected discounted return from it, the Markov property collapses that infinite future into a one-step consistency equation, and Bellman's optimality principle adds that the best value at a state compares the actions after accounting for immediate reward and the best continuation. Value iteration and policy iteration are clean and provably correct, but they all need the model, and the model is much larger than a value table and forces me to decide which features of the world deserve representation before I even know which features matter for good control. Certainty-equivalence control estimates that model from data and plans inside it, but it treats early model errors as if they were truth and still pays the cost of representing the transition structure. Temporal-difference prediction removes the model for evaluation — it bootstraps from a sampled reward plus the current estimate of the next state's value instead of waiting for a full return — but prediction is not control: a state value is only meaningful relative to a policy, and without the model it does not tell me which untried action to pick. Actor-critic and learning-automata controllers keep a value estimate alongside a separately represented policy, but the two adaptive processes interact during learning and the convergence target becomes hard to characterize. Monte Carlo control avoids the model but waits for full sampled returns, so its variance and delay make it a poor fit for incremental online control. What I need is a method that learns from the actual stream of transitions, builds no explicit model, improves online, and still converges to exactly the dynamic-programming optimum.

I propose Q-learning. The idea is to keep a single tabular estimate $Q(x,a)$ of the optimal action value — the value of taking action $a$ in state $x$ and then continuing optimally — and to update it from sampled transitions with a backup that mimics Bellman's optimality equation but replaces the model expectation with one sample. On each observed transition $(x_t, a_t, r_t, x_{t+1})$ I move only the experienced entry toward the sampled optimality target and leave every other entry untouched:
$$Q_{t+1}(x_t,a_t) = (1-\alpha_t)\,Q_t(x_t,a_t) + \alpha_t\big(r_t + \gamma\,\max_b Q_t(x_{t+1},b)\big),$$
which is the same thing written as a temporal-difference correction,
$$Q(x_t,a_t) \leftarrow Q(x_t,a_t) + \alpha_t\big[r_t + \gamma\,\max_b Q(x_{t+1},b) - Q(x_t,a_t)\big].$$
Storing one number per state-action pair rather than per state is the first load-bearing choice: a state value alone would still need the missing transition model to choose between actions, whereas an action-value table lets me read the greedy policy directly off the table, $\pi_Q(x) \in \arg\max_a Q(x,a)$, so the policy stops being a separate adaptive object that can chase the value estimate. The second and central choice is the $\max_b$ inside the target. The sampled transition supplies the immediate reward $r_t$ and the next state $x_{t+1}$ for the experienced pair, but the continuation value I bootstrap from is the best entry available at the next state under the current table — not the value of whatever action the behavior policy happens to take next. This is what makes the update a sampled Bellman optimality backup and what makes Q-learning off-policy: the behavior policy that chooses $a_t$ controls only which entries get sampled, while the target always points at the optimal greedy continuation. The agent is therefore free to explore however it likes — for instance acting greedily with probability $1-\epsilon$ and uniformly at random otherwise — and ordinary prediction's objection (that off-policy data would estimate the value of the exploratory behavior) is dissolved, because the maximum keeps the learned fixed point tied to the optimality equation rather than to the behavior policy. The exploration only has to be persistent enough that every state-action pair keeps getting visited.

The reason I commit to the one-step form rather than a longer multi-step return is stability, and it is a genuine design decision, not a convenience. If I let a longer return pass through several states under the current greedy policy, a small error in the table can flip which action looks greedy at an intermediate state, and that flipped policy can carry a value error larger than the original table perturbation. The temporal-difference picture says the same thing in trace language: an off-policy exploratory step contaminates the tail of a multi-step return, and setting the trace parameter to zero at such a step cuts the tail off and replaces it with the current value estimate at the next state — and the one-step update is exactly the extreme where the backup always stops after the observed transition. Each update is then atomic: one sampled transition, one bootstrapped maximum at the next state, one entry changed. That atomicity is also what lets the convergence proof represent the method exactly. The proof rests on the action-replay process, an artificial controlled Markov process built from the recorded experience: each observed transition $(x_t,a_t,y_t,r_t,\alpha_t)$ is written on a card, ordered by time, with the initial table on a bottom card. A replay state is a real state paired with a card level $n$; choosing action $a$ at state $x$ deals downward through cards at levels at most $n$ looking for a matching $(x,a)$ card, replays that card's reward and next state with probability equal to its learning rate $\alpha_t$ and skips it otherwise, and pays the initial table entry if it reaches the bottom. This contraption is engineered to match the update term by term. Lemma A shows by induction that the optimal action value of the replay process at level $n$ equals the learned table after $n$ real updates: at level zero the process pays the initial table, and at the induction step, if the newest card is not for $(x,a)$ nothing changes for that pair, while if it is, the replay mixes the old value with the reward plus the discounted optimal value below the card using $\alpha_t$ as the mixing probability — which is precisely the learning update with replay probability $\alpha_t$ producing the new target and skip probability $1-\alpha_t$ preserving the old estimate. Lemma B shows that as the level grows the replay process converges to the real Markov process: discounting bounds the tail of long returns so comparisons can be truncated to a finite horizon, the condition $\sum_i \alpha_{n_i(x,a)} = \infty$ keeps the process from skipping every matching card and falling far down the deck, the condition $\sum_i \alpha_{n_i(x,a)}^2 < \infty$ gives stochastic-approximation convergence of the replay's reward and transition averages to the true means and probabilities, and finite-horizon action values are continuous in the one-step model so close models give close values. Chaining the two lemmas, the learned table equals the replay process's optimal action values and those converge to the real process's optimal action values, so under bounded rewards, finite tabular state and action sets, discount $0 < \gamma < 1$ (or an absorbing alternative that keeps values bounded), every pair visited infinitely often, and the two Robbins-Monro step-size conditions, the iterates converge with probability one, $Q_t(x,a) \to Q^*(x,a)$ for all $x,a$. The table is never estimating the value of what the agent does while it explores; it is estimating the optimal action-value function whose greedy policy is optimal.

```python
import numpy as np
from collections import defaultdict


def q_learning(env, n_actions, gamma, alpha_schedule, epsilon, n_steps):
    Q = defaultdict(lambda: np.zeros(n_actions))

    def choose_action(state):
        if np.random.random() < epsilon:
            return np.random.randint(n_actions)
        return int(np.argmax(Q[state]))

    state = env.reset()

    for _ in range(n_steps):
        action = choose_action(state)
        next_state, reward, done = env.step(action)

        continuation = 0.0 if done else gamma * np.max(Q[next_state])
        target = reward + continuation

        alpha = alpha_schedule(state, action)
        Q[state][action] += alpha * (target - Q[state][action])

        state = env.reset() if done else next_state

    return Q
```
