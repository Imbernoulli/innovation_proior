Start from the animal. A rat in a box learns to press a lever for food; a lion learns the exact distance at which it can break cover and still catch the gazelle. Nobody hands these animals a model of their world. They act, things happen, reward arrives — often long after the action that earned it — and somehow the behaviour improves. I want a computational account of that: an agent in a controlled Markov process that, purely from a stream of (state, action, reward) triples, comes to act so as to maximise its long-run discounted reward, with rewards s steps away discounted by γ^s, 0 < γ < 1.

What do I already have to lean on? Dynamic programming solves the optimisation completely — if I know the world. For a fixed policy f the value of a state is the expected discounted return, V_f(x) = E[r_0 + γr_1 + γ²r_2 + ···], and because the discounting is exponential this collapses into a one-step equation, V_f(x) = ρ(x,f) + γ Σ_y P_xy[f] V_f(y), where ρ(x,a) is the mean reward and P_xy[a] the transition probability. In a finite world that's just |S| linear equations; evaluating a policy is mechanical. And Bellman tells me there's an optimal policy f* whose value V* dominates every other from every state at once, characterised by V*(x) = max_a { ρ(x,a) + γ Σ_y P_xy[a] V*(y) }. So in principle the problem is solved. The trouble is the "if I know the world."

Two things bother me about needing the world. First the sheer size: a value function is |S| numbers, a policy at most |S||A|, but the transition model P_xy[a] can be |S|²|A| numbers, and as Howard noted back in 1960 that's what really limits dynamic programming in practice. Second, and this is the one that nags at me more — to build a model the agent has to decide which features of the world to represent and which to throw away. But how can it know which features matter for the *optimal* policy before it has the optimal policy? If it models too little, its policy is optimal for the model and wrong for the world; if it models too much, it drowns. In model-based learning the agent swaps the real world for an internal model and does mental experiments on the model. The whole correctness of that move rests on the model being adequate for the one purpose that matters, and the agent has no way to check that separately. I'd rather the agent experiment in the *actual* world and keep track of the returns that actually result. So: no model. Learn the value function — or whatever I need — directly from experience.

Can I even estimate a value from experience without a model? Sutton showed how, for prediction. Keep an approximation U to V_f. The crudest estimate of the return from x_t is to just wait and sum the discounted rewards, the actual return r_t + γr_{t+1} + ···, or the truncated version r_t^[n] = r_t + ··· + γ^{n-1}r_{t+n-1} cut off after n steps since γ^n is negligible. But that has three faults: you wait n steps to get it, it's high variance, and you have to keep following f for those n steps. The fix is to bootstrap — replace the tail of the truncated return with your current guess of the value of where you ended up: the corrected n-step return r_t^(n) = r_t + γr_{t+1} + ··· + γ^{n-1}r_{t+n-1} + γ^n U_{t+n}(x_{t+n}). Why is that a good thing to chase? Because of an error-reduction property. Let K = max_x |U(x) − V_f(x)| be the worst error in my current estimate. Then the expected corrected return is closer to the truth than U is: max_x |E[r_t^(n)] − V_f(x)| ≤ γ^n K. The γ^n out front is the whole point — bootstrapping doesn't amplify my error, it shrinks it by γ^n. For n=1 this is just the one-step backup r_t + γU(x_{t+1}), and the contraction factor is γ. This is exactly the contraction that makes value iteration work, except now I'm feeding it sampled rewards and sampled next-states instead of the known model. I can update U(x_t) ← (1−α)U(x_t) + α(r_t + γU(x_{t+1})), the temporal-difference update, where the bracket r_t + γU(x_{t+1}) − U(x_t) is the prediction difference, the surprise between what I thought the return would be and what one more step of experience suggests it is.

So far so good — but this is *prediction*. It learns V_f for whatever policy f I'm following. It does not tell me how to *act* optimally. Two gaps. First, the value of a state is only meaningful relative to a policy; there's no model-free way to ask "what's the best action here?" if all I store is V. To pick an action greedily I'd need ρ(x,a) + γ Σ_y P_xy[a] V(y) for each a, which needs the model again — defeating the purpose. Second, the optimal policy is unknown at the start, so I can't simply set out to estimate V* directly: I don't know which actions to follow to sample its returns.

Let me deal with the first gap, because it points the way. The thing dynamic programming actually leans on is not V but the *action-value*: Q_f(x,a) = ρ(x,a) + γ Σ_y P_xy[a] V_f(y), the return from doing a once at x and then following f. Note Q_f(x,f(x)) = V_f(x). Everything in DP is phrased through it. The policy-improvement theorem: if some g has Q_f(x,g) ≥ V_f(x) for every x, then g is uniformly at least as good as f. And the optimality characterisation: V*(x) = max_a Q*(x,a), with the optimal action being any a attaining that max. So if I had Q*, choosing optimally would be trivial — just take the action with the largest Q*(x,·), no model needed at the decision. The action-value already bakes the one-step lookahead into a stored number indexed by (x,a). That's the object I should be learning, not V.

And this also dissolves the second gap, in a way I didn't expect. Suppose I store Q(x,a) and *define* the value of a state to be U^Q(x) = max_a Q(x,a), with the implicit policy f^Q(x) = the a that attains it — always choose the action of maximal estimated action-value. Then the policy isn't a separate thing I have to maintain and adjust; it's read off the action-values by maximising. That matters, because the earlier reinforcement-learning controllers — Witten's, Barto–Sutton–Anderson's adaptive heuristic critic, the learning-automata schemes of Wheeler and Narendra — all keep the policy as a *separate* representation, action strengths or probabilities, nudged by an estimated value function. They have two adaptive processes running at once, value estimation and policy improvement, and these can interact during learning in ways that prevent convergence; nobody could prove those schemes find the optimal policy. Folding the policy into "max over actions" collapses the two processes into one. There's just Q.

Now, how do I improve Q from experience? Here I have to be careful about *which* policy I'm estimating, because of an issue the prediction story hides. When the agent acts, it can't only ever take the action its current policy recommends — it has to *explore*, try other actions, to discover whether something better exists. Call the recommended actions "policy steps" and the others "experimental steps." If I'm trying to estimate the return under my estimation policy f, an experimental action contaminates the estimate: the rewards after an off-policy action are not samples of f's return.

So can I estimate the return under one policy while behaving differently? Let me chase this with the bootstrapped returns. Write the one-step-corrected return recursively, with a per-step weight λ_t that I'm free to set: r_t^Λ = r_t + γ(1−λ_{t+1}) U_t(x_{t+1}) + γ λ_{t+1} r_{t+1}^Λ. When λ_{t+1} = 0 the recursion *stops* — the future r_{t+1}^Λ is multiplied by zero, and r_t^Λ becomes just r_t + γ U_t(x_{t+1}), using only the bootstrap value at the next state and no further sampled rewards. When λ_{t+1} > 0 it keeps unrolling sampled rewards. So set λ_t = λ* on policy steps and λ_t = 0 on experimental steps. Then look at what r_t^Λ is: if step t+1 is experimental, λ_{t+1}=0 cuts the bootstrap off right there, so r_t^Λ is built from r_t and U_{t+1}(x_{t+1}) and only any *further policy* steps — it never absorbs the consequences of the experimental action. It is therefore an unbiased estimate of Q_f(x_t, a_t), uncontaminated by the experiment. That's the lever I was missing: by truncating the bootstrap at the first off-policy action, you estimate the value under the policy while actually behaving however you like. The next state is always folded in (so it's an action-value estimate), but you don't let the off-policy tail in.

Now push this to its extreme, with the greedy implicit policy. Take the estimation policy to be f^Q — always greedy w.r.t. the current Q — and take λ* = 0, the one-step case. Then the estimate of the action-value of whatever (x_t, a_t) I just experienced is
  r_t^(1) = r_t + γ U^Q_t(x_{t+1}) = r_t + γ max_a Q_t(x_{t+1}, a).
And the action a_t that produced this transition was *whatever I felt like doing* — it does not have to be greedy. Stare at that for a second. The target r_t + γ max_a Q_t(x_{t+1}, a) does not mention a_t's policy at all; it bootstraps the value of the *best* action at the next state, the greedy one, regardless of how I'm behaving. So I can drive the agent with any exploratory behaviour — anything that tries every action in every state often enough — and every transition still gives me an unbiased one-step sample of the Bellman optimality backup applied to the current Q, because the max inside the target is doing the policy-improvement step for me on the fly. This is the thing. The actions the agent takes are not specified by the learning rule; the agent is free to take whatever actions it likes; and yet the fixed point it chases is the optimal action-value function, not the value of its own wandering behaviour. The max is what makes it off-policy.

So the update writes itself. On the transition (x_t, a_t, r_t, x_{t+1}), with a small learning factor α:
  Q_{t+1}(x_t, a_t) = (1−α) Q_t(x_t, a_t) + α ( r_t + γ max_a Q_t(x_{t+1}, a) ),
and leave Q unchanged at every other (x,a). One-step Q-learning. Why must α decrease, and to what schedule? Because r_t and x_{t+1} are samples — the bracket is a noisy version of the true backup ρ(x_t,a_t) + γ Σ_y P_{x_t y}[a_t] max_a Q(y,a). To average the noise away I want the Robbins–Monro conditions on the learning rates seen at each individual (x,a): Σ α = ∞ so the estimate can still travel any required distance and never freezes, and Σ α² < ∞ so the accumulated sampling noise has finite variance and washes out. With a fixed α it would jitter forever; with α decreasing under those two conditions, the running update converges to the mean.

Does this actually work? My first instinct is to read it as value iteration. Recall asynchronous value iteration: pick any state y, set U(y) ← max_a { ρ(y,a) + γ Σ_z P_yz[a] U(z) }, repeat, in any order, each state often enough. It converges because one such backup is a γ-contraction: if M = max_x |U(x) − V*(x)| then |U'(y) − V*(y)| ≤ γ M, so the worst error shrinks by γ every time. My Q-update is exactly this backup on action-values — Q_{t+1} is formed from U^Q_t and U^Q_t is the max of Q_t — except the model expectation Σ_y P · is replaced by a single sampled (r_t, x_{t+1}) and folded in with step α. It's incremental, Monte-Carlo value iteration. That's promising as intuition. But the contraction argument as stated needs the true expectation; I have noisy single samples and a shrinking α. The clean γ-contraction and the Robbins–Monro averaging are pulling in two different directions — one wants the exact backup, the other is precisely about not having it. I can't just wave at "it's value iteration" and call the noisy, sampled version proved.

Let me first see how far the idea even reaches, because I want to know what I'm allowed to prove. The natural generalisation is to use the *general* bootstrapped return r_t^Λ — multi-step, λ* > 0 — as the action-value target, a whole family of Q-learning methods. They look plausible. Do they converge? Here I hit a wall. If Q is perturbed away from Q* by even a small amount, the implicit greedy policy f^Q can flip to a different action at some state — and the value of that changed policy can differ from V* by *more* than the perturbation in Q. A small error in Q can, at a tie-breaking boundary, produce a larger error in the value of the policy it induces. So a small perturbation away from the optimum can, under some circumstances, grow rather than shrink — an instability. These instabilities need not happen, but I can find no condition that rules them out for the general multi-step family. I have to retreat: I will only claim convergence for the restricted, one-step version. And that restriction is not a cosmetic concession — the one-step structure (each experienced transition is a single, self-contained backup) is exactly what will let me build a clean proof. The multi-step returns tangle several transitions and several intermediate bootstraps together; the one-step return is atomic.

Now the proof. The obstacle is that x_n, a_n, and even α_n may be chosen with knowledge of all the past — the agent's behaviour and learning rates can depend on history — so I can't treat the updates as a fixed iid averaging scheme. And the sampled targets are noisy. I want to convert the messy, history-dependent stochastic update into something whose optimal value I can compute exactly and then control. The device I'll build is an artificial Markov process whose *optimal* action-values are, by construction, exactly the Q-learning iterates — and which itself converges to the real process. Then optimal-values-of-the-artificial-process → optimal-values-of-the-real-process gives me Q_n → Q*.

Call it the action-replay process, the ARP. Picture every experienced episode (x_t, a_t, y_t, r_t, α_t) written on a card. Stack the cards into an infinite deck, first episode near the bottom, later episodes higher; on the very bottom card (number 0) write the agent's initial values Q_0(x,a) for all (x,a). A state of the ARP is a pair (x, n): a real-world state x together with a level n, a card number. The actions are the real actions. Here is how the ARP transitions from (x, n) under action a. Throw away every card above level n, leaving a finite deck. Now deal down from the top, looking for the first card whose recorded state and action match (x, a), say at episode t. Flip a biased coin: heads with probability α_t (the learning rate on that card), tails with 1−α_t. On heads, *replay* that episode — emit its reward r_t and move to the ARP state (y_t, t−1), one level *below* where the card sat. On tails, discard that card too and keep dealing down for another (x,a) match. If you reach the bottom card without a coin coming up heads, the game absorbs and pays the initial value Q_0(x,a). The essential thing is that every action in the ARP moves you to a strictly lower level, so after finitely many actions you hit the bottom and stop — the ARP is a perfectly good controlled Markov process with its own optimal action-values Q*_ARP, and I'll discount it by the same γ.

First claim: the optimal action-values of the ARP at level n are exactly the Q-learning iterates. Lemma A: Q_n(x,a) = Q*_ARP((x,n), a) for all x, a, n. Prove it by induction down the deck. Base n=0: at level 0 the only thing the ARP can do is absorb and pay Q_0(x,a), so that *is* the optimal — indeed only — action value, Q_0(x,a) = Q*_ARP((x,0),a). Step: suppose Q_{n−1}(x,a) = Q*_ARP((x,n−1),a) for all x,a, which makes the optimal value at level n−1 equal to V_{n−1}(x) = max_a Q_{n−1}(x,a). Consider taking action a at (x,n). If (x,a) is *not* the (x_n,a_n) recorded on card n, then card n is irrelevant to this (x,a) and the ARP behaves exactly as at level n−1, so Q*_ARP((x,n),a) = Q*_ARP((x,n−1),a) = Q_{n−1}(x,a) = Q_n(x,a) since the Q-update left this entry unchanged. If (x,a) = (x_n,a_n), then dealing from level n: with probability 1−α_n the top card (card n) is the match but the coin comes up tails, discarding it and leaving exactly the level-(n−1) process for (x_n,a_n); with probability α_n the coin is heads, the episode replays, paying r_n and moving to (y_n, n−1). So the optimal action-value is the mixture
  Q*_ARP((x_n,n), a_n) = (1−α_n) Q*_ARP((x_n,n−1), a_n) + α_n ( r_n + γ V*((y_n,n−1)) )
                       = (1−α_n) Q_{n−1}(x_n,a_n) + α_n ( r_n + γ V_{n−1}(y_n) )
                       = Q_n(x_n, a_n),
which is precisely the Q-learning update, equation by equation. So Q_n = Q*_ARP at level n. The ARP was rigged to make this true: the coin probability α_t is the learning rate, the replay-and-descend is the bootstrap, and tails-then-continue is the (1−α) carry-over of the old estimate.

So now I only have to show the ARP, as the level n grows, becomes the real process — then its optimal action-values become Q*, and by Lemma A so do the iterates. This is Lemma B, in four moves.

B.1, discounting lets me truncate. In any discounted bounded-reward finite Markov process, the value of a state under a finite prefix of s actions differs from its value under that prefix followed by anything at all by at most the discounted tail: ignoring the (s+1)-th state on costs δ = γ^s Σ_y P[...] V(y_{s+1}), and since |V| < R/(1−γ) for rewards bounded by R, |δ| < γ^s R/(1−γ) → 0 as s → ∞. So I only ever need to compare the two processes over a finite number s of steps, and I can make the truncation error as small as I like by taking s large. (When there's no discount but the process is absorbing — bound to get trapped in a goal state — the role of γ^s is played instead by the probability (1−p*)^j of *not* yet being trapped after j blocks of u* steps, which also tends to 0; same conclusion.)

B.2, the ARP doesn't fall back down the deck. Over s actions starting from a high enough level, the ARP stays above any fixed level l with probability as close to 1 as I want. Why: to descend below l on a single action from (x,n), every one of the matching cards between level l and the top must come up tails, and that product Π (1−α_{n^i}) < exp(−Σ α_{n^i}) — which, because Σ α = ∞ along the (x,a) subsequence, goes to 0 as the starting level (and hence the number of matching cards above l) grows. So from a high enough start, one action lands above l with probability ≥ 1−η; iterate the argument for each of the s actions and choose the start high enough to make the total probability of ever straying below l less than any ε I name. The Σ α = ∞ condition is doing exactly this job — keeping the replay process from running off the bottom.

B.3, the ARP's one-step model converges to the real one-step model. This is where Robbins–Monro returns, in the form of a standard stochastic-approximation theorem (Kushner & Clark, theorem 2.3.1): if X_{n+1} = X_n + β_n(ξ_n − X_n) with 0 ≤ β_n < 1, Σ β_n = ∞, Σ β_n² < ∞, and ξ_n bounded with mean Ξ, then X_n → Ξ with probability 1. Now look at the ARP's expected reward for (x,a) at level n: by construction it is the α-weighted running average of the recorded rewards on the matching cards,
  R_{(x,n^{i+1})}(a) = R_{(x,n^i)}(a) + α_{n^{i+1}} ( r_{n^{i+1}} − R_{(x,n^i)}(a) ),
which is exactly the theorem's recursion with β = α along the (x,a) subsequence (Σ α = ∞, Σ α² < ∞ are the hypotheses) and ξ the unbiased sampled rewards with mean ρ_x(a). So R_{(x,n)}(a) → ρ_x(a) w.p.1, uniformly since there are finitely many (x,a). The same argument with the indicator χ_n(y) = 1{y_n = y}, mean P_xy[a], gives the ARP's transition probabilities P^{(n)}_{xy}[a] → P_xy[a] w.p.1. The observations from the real process are independent and, by B.2, conditioning on staying above any fixed level doesn't disturb the limit. So the ARP's rewards and transitions, at high enough levels, are as close to the real process's as I like, with probability 1.

B.4, close models give close values. If two Markov chains have transition matrices within η/R and rewards within η of each other (over s steps), their s-step action-values differ by at most ηs(s+1)/2. The two-step calculation makes the bound transparent: comparing the concatenated chain's two-step return Q̄'(x,a_1,a_2) = R^1_x(a_1) + γ Σ_y P^1_xy[a_1] R^2_y(a_2) with the real Q̄(x,a_1,a_2), the difference is bounded by the reward error on step 1, plus γ times the reward error on step 2, plus γ times the transition error times the bounded reward — three terms each under η (the transition error η/R cancels the reward bound R), so under 3η for two steps; telescoping s steps, the per-step errors accumulate as 1+2+···+s = s(s+1)/2, giving ηs(s+1)/2. Errors grow only quadratically in the horizon, so if the per-step model is close, the finite-horizon values are close.

Assemble it, watching every ε. Without loss of generality take Q_0(x,a) < R/(1−γ) and R > 1. Given ε, first use B.1 to pick s so γ^s R/(1−γ) < ε/6, which makes the truncation cost below ε/6 in the ARP and below ε/6 in the real process. Then use B.3 to pick a level l so large that, for n > l and every x,y,a, the one-step ARP model has |P^{(n)}_{xy}[a] − P_{xy}[a]| < ε/[3s(s+1)R] and |R^{(n)}_x(a) − R_x(a)| < ε/[3s(s+1)]. Then use B.2 to pick a level h so large that, starting above h, the probability of straying below l within s actions is below min{ ε(1−γ)/6sR, ε/[3s(s+1)R] }. Conditioning on the event that the s-step path stays above l can at worst double the model-error allowance I just set, so the conditional one-step errors are below 2ε/[3s(s+1)R] for transitions and 2ε/[3s(s+1)] for rewards. Now B.4 applies with η = 2ε/[3s(s+1)], giving finite-horizon model error ηs(s+1)/2 = ε/3. If the path does stray, the worst cost is bounded by 2sR/(1−γ), and the probability bound ε(1−γ)/6sR makes that contribution below ε/3. So for n > h, comparing the value of any fixed s-action sequence in the ARP at (x,n) with its value in the real process, the discrepancy is at most [straying probability × 2sR/(1−γ)] + [B.4 model error] < ε/3 + ε/3, and B.1 adds at most ε/6 on each side for truncating at s. Because this holds for *any* action sequence, it holds in particular for the sequence optimal in either process, so the optimal values match: |Q*_ARP((x,n),a) − Q*(x,a)| < ε. But Q*_ARP((x,n),a) = Q_n(x,a) by Lemma A. Hence Q_n(x,a) → Q*(x,a) as n → ∞, for every (x,a), with probability 1. That is the theorem.

Let me state it cleanly, since I now know exactly what it needs. Given bounded rewards |r_n| ≤ R, learning rates 0 ≤ α_n < 1, and for every (x,a), with n^i(x,a) the index of the i-th time a is tried in x,
  Σ_{i=1}^∞ α_{n^i(x,a)} = ∞   and   Σ_{i=1}^∞ [α_{n^i(x,a)}]² < ∞,
then the one-step Q-learning iterates Q_n(x,a) → Q*(x,a) as n → ∞, ∀x,a, with probability 1 — provided every (x,a) is tried infinitely often, which the two sum conditions presuppose and which is unavoidable: under these stochastic conditions no method could find the optimum while leaving some state–action pair eventually untried. And the behaviour generating the data is otherwise free.

A couple of extensions fall out of the same machinery. If there's no discount (γ = 1) but the process is absorbing — sure to reach a goal — then V* is still bounded, V*(x) ≤ u*R/p* where in every u* steps there's probability ≥ p* of being trapped, and (1−p*)^j → 0 does the job γ^s did in B.1; the proof goes through. And if I want to update many Q-entries per experienced step rather than one, I just let the ARP take an action at any level where it was executed in the real process — multiple cards per level — and as long as the per-(x,a) step-size sums still satisfy Robbins–Monro, Lemma A and Lemma B are untouched; it merely speeds up the same averaging.

Now write the agent, which is almost embarrassingly small for what it does. Hold Q in a look-up table — and only a look-up table; the convergence rests on the tabular representation, and Q-learning can fail to converge under other representations. The behaviour rule just has to explore, e.g. ε-greedy: usually take argmax_a Q[x][a], occasionally a random action, so every (x,a) is sampled infinitely often. On each transition, form the one-step target r + γ max_a Q[x'][a], take its difference with the current entry, and step the entry toward it by α. Nothing about the behaviour leaks into the target except through which (x,a) gets updated; the max keeps the target pinned to the optimal policy.

```python
import numpy as np
from collections import defaultdict

def q_learning(env, n_actions, gamma, alpha_schedule, epsilon, n_steps):
    # Tabular action-values Q(x,a); the convergence proof assumes a look-up table.
    Q = defaultdict(lambda: np.zeros(n_actions))

    def greedy(state):
        return int(np.argmax(Q[state]))          # implicit policy f^Q(x)=argmax_a Q(x,a)

    def behaviour(state):
        # exploratory, off-policy: need NOT be greedy, only visit every (x,a) often
        if np.random.rand() < epsilon:
            return np.random.randint(n_actions)
        return greedy(state)

    state = env.reset()
    for t in range(n_steps):
        action = behaviour(state)                # free to take whatever action it likes
        next_state, reward, done = env.step(action)

        # one-step target: bootstrap the GREEDY (max) value at the next state ---
        # this max is the policy-improvement step folded into the backup, and is
        # what makes the target chase Q*, independent of the behaviour policy.
        target = reward + (0.0 if done else gamma * np.max(Q[next_state]))

        # Robbins-Monro step: Sum(alpha)=inf, Sum(alpha^2)<inf along each (x,a) -->
        # the noisy sampled target is averaged into a consistent estimate.
        alpha = alpha_schedule(state, action)
        Q[state][action] += alpha * (target - Q[state][action])   # eq. (1)

        state = env.reset() if done else next_state
    return Q
```

The causal chain, end to end: I wanted optimal control from raw experience with no model; modelling the world is both huge and unverifiable for the one purpose that matters, so I refused it. TD showed sampled one-step bootstrapping contracts a *prediction* toward a fixed policy's value with factor γ, but prediction isn't control and a state's value needs a policy. Storing action-values Q(x,a) instead of V makes the greedy policy implicit (max over actions) and collapses value-estimation and policy-improvement into one process, escaping the two-process instability of separately represented policies. Truncating the bootstrap at the first off-policy action lets me estimate returns under one policy while behaving differently; pushing that to the greedy implicit policy and one step makes the target r + γ max_a Q(x',a) — which never mentions the behaviour, so any exploratory behaviour yields unbiased one-step samples of the Bellman optimality backup, the max doing policy improvement on the fly. The update is then incremental, sampled, asynchronous value iteration on action-values. The general multi-step version I could not stabilise (small Q-perturbations can flip the greedy policy and enlarge the value error), so I kept the atomic one-step form — and that atomicity is what builds the action-replay process whose optimal action-values are, exactly, the iterates (Lemma A), and which converges to the real process (Lemma B: discount-truncation, no-fall-back, Robbins–Monro model convergence, close-models-close-values), giving Q_n → Q* with probability 1 under Σα=∞, Σα²<∞ and infinitely often visitation.
