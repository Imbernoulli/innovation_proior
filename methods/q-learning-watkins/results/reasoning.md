I start with a finite Markov decision problem and refuse the part of dynamic programming that requires a model. If I know the reward means and transition probabilities, Bellman's equations already tell me how to compute optimal behavior. That is not the learning problem I want. I only observe transitions of the form current state, action, reward, next state, and I want the same optimal control solution without first estimating the transition table.

I look at why the known-model solution is so powerful. For a fixed policy, the value of a state is the expected discounted return from that state, and the Markov property turns that long future into a one-step equation. For optimal control, Bellman's principle says that the correct continuation at the next state is the best one available there. The known-model backup is a contraction under discounting, which explains why value iteration can repeatedly apply local one-step corrections and still reach the global optimum.

The obstacle is that the backup averages over next states using the transition model. Building that model is costly, and it also asks the learner to decide which features of the world deserve representation before it knows which features matter for optimal control. I therefore want sampled backups. A sampled transition gives me one reward and one next state in place of the expectation. If I can average those noisy samples correctly, I may be able to keep the Bellman structure while removing the model.

Temporal-difference prediction gives me the first piece. When I evaluate a fixed policy, I do not need to wait for a full return. I can use the sampled reward plus the discounted current estimate of the next state's value. The corrected return is biased by my current estimate, but Sutton's error-reduction argument shows that the expected corrected return moves closer to the value of the policy being evaluated. Robbins-Monro then tells me how to absorb noisy samples: the learning rate must keep summing to infinity so the estimate can still move, while the squared learning rates must sum to a finite value so sampling noise averages away.

Prediction is still not control. A state value is only meaningful under a policy, and if I only store state values then choosing between actions requires the missing transition model. I need a number for each state-action pair, because that number can represent the value of taking that action now and then continuing well afterward. With such a table, action selection can be read directly from the table by choosing the action with the largest current entry at the state. The policy no longer has to be a separate adaptive object.

This matters because separate policy and value learners can chase each other. If I update a value estimate while also updating independent action probabilities or action strengths, the target being evaluated changes while the evaluator is still moving. That is the instability Watkins is trying to avoid. I want one table to carry both roles: it stores action values, and its maximum over actions at a state defines the greedy policy implied by the table.

Now I face the exploration problem. The behavior that generates data cannot always be greedy with respect to the current table, because then I may never try a better action. But if I follow exploratory behavior, ordinary prediction would estimate the value of that exploratory behavior, not the optimal behavior. I need the learning target to ignore the later consequences of exploratory choices except as samples of the environment's one-step dynamics.

The temporal-difference view suggests how to do that. In multi-step returns, an off-policy experimental step contaminates the tail of the return. If I set the trace parameter to zero at such a step, the sampled tail is cut off and replaced by the current value estimate at the next state. In the one-step extreme, the update always stops after the observed transition. That means the action I actually take supplies only the sampled immediate reward and sampled next state; the continuation value can be chosen from the table rather than from the behavior that generated the data.

I therefore define the continuation value at the next state as the maximum table entry over next-state actions. The target for the experienced pair becomes the observed reward plus the discounted maximum current action value at the next state. This is the Bellman optimality backup with the expectation replaced by one sample. It does not ask which action the behavior policy would take next. The behavior policy determines which state-action entry gets sampled, but the target itself points toward the greedy optimal continuation.

The update is then simple. For the transition `(x, a, r, y)`, I move the table entry for `(x, a)` toward `r + gamma * max_b Q(y, b)` and leave every other entry unchanged. The action `a` can be chosen by any exploratory rule. This is the central off-policy move: the sampled experience may come from a wandering behavior policy, while the bootstrapped target evaluates the greedy continuation. If every state-action pair is visited repeatedly, the table receives samples for every component of the optimality equations.

At first I want to justify this by saying it is asynchronous value iteration with samples. That intuition is right but not a proof. The exact Bellman operator is a contraction because it uses expected rewards and expected next-state values. My update uses a single random reward and next state, and the sampling schedule can depend on all previous experience. I need a proof that handles history-dependent action choices and noisy backups under decreasing step sizes.

I also see why I should not claim the same result for the whole multi-step family. If I let longer returns pass through several states under the current greedy policy, small errors in the table can change which action is considered greedy, and that changed policy can have a value error larger than the original table perturbation. The one-step version avoids that instability. Each update is atomic: one sampled transition, one bootstrapped maximum at the next state, one entry changed. That atomic form is exactly what the convergence proof can represent.

The proof device is the action-replay process. I imagine every observed transition and its learning rate written on a card, ordered by time, with the initial table on a bottom card. A state of the replay process is a real state together with a card level. From level `n`, if I choose action `a` at state `x`, I deal downward until I find a card whose recorded state-action pair is `(x, a)`. With probability equal to that card's learning rate I replay the card's reward and next state, then continue below that card. With the remaining probability I skip the card and keep dealing. If I reach the bottom, the process pays the initial table entry.

This construction looks artificial, but it is built to match the update exactly. I prove by induction that the optimal action value of the replay process at level `n` is exactly the learned table after `n` updates. At level zero, the replay process pays the initial table. For the induction step, if the newest card is not for `(x, a)`, nothing changes for that pair. If it is for `(x, a)`, the replay process mixes the old value with the observed reward plus discounted optimal value below the card, using the learning rate as the mixing probability. That is precisely the learning update.

After that, I only have to show that high levels of the replay process behave like the real Markov process. Discounting lets me truncate comparisons to a finite horizon, because the remaining tail is bounded by a term that shrinks geometrically. The condition that the learning rates for each state-action pair sum to infinity makes it unlikely that the replay process skips all matching cards and falls far down the deck. The condition that squared learning rates sum finitely lets the replay process's reward and transition averages converge to the real reward means and transition probabilities. Finally, finite-horizon values are continuous in rewards and transition probabilities, so close one-step models imply close finite-horizon action values.

Putting these pieces together, for any error tolerance I choose a horizon long enough that discounted tails are negligible, then choose a replay level high enough that the replay process stays in levels whose one-step rewards and transitions are close to the real process with high probability. The finite-horizon values are then close, the tails are close, and therefore the optimal action values of the replay process are close to the real optimal action values. Since the replay process's optimal action values are the learned table entries, the learned table converges to the optimal action-value function with probability one.

The theorem needs exactly the assumptions I expect: bounded rewards, finite tabular state and action sets, discounting or an absorbing alternative that keeps values bounded, every state-action pair visited infinitely often, and per-pair learning rates satisfying `sum alpha = infinity` and `sum alpha^2 < infinity`. Under those conditions, the behavior policy is otherwise free. It can explore however it likes, because the maximum in the target keeps the learned fixed point tied to the optimality equation rather than to the behavior policy.

The resulting method is Q-learning. It learns optimal action values directly from model-free experience, updates one entry at a time, bootstraps from the maximum next-state action value, and separates behavior from the target being learned. The table is not estimating the value of what the agent actually does while exploring. It is estimating the optimal action-value function whose greedy policy is optimal.
