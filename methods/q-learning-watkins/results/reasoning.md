I start with a finite Markov decision problem and refuse the part of dynamic programming that requires a model. If I know the reward means and transition probabilities, Bellman's equations already tell me how to compute optimal behavior. That is not the learning problem I want. I only observe transitions of the form current state, action, reward, next state, and I want the same optimal control solution without first estimating the transition table.

I look at why the known-model solution is so powerful. For a fixed policy, the value of a state is the expected discounted return from that state, and the Markov property turns that long future into a one-step equation. For optimal control, Bellman's principle says that the correct continuation at the next state is the best one available there. The known-model backup is a contraction under discounting, which explains why value iteration can repeatedly apply local one-step corrections and still reach the global optimum.

The obstacle is that the backup averages over next states using the transition model. Building that model is costly, and it also asks the learner to decide which features of the world deserve representation before it knows which features matter for optimal control. I therefore want sampled backups. A sampled transition gives me one reward and one next state in place of the expectation. If I can average those noisy samples correctly, I may be able to keep the Bellman structure while removing the model.

Temporal-difference prediction gives me the first piece. When I evaluate a fixed policy, I do not need to wait for a full return. I can use the sampled reward plus the discounted current estimate of the next state's value. The corrected return is biased by my current estimate, but Sutton's error-reduction argument shows that the expected corrected return moves closer to the value of the policy being evaluated. Robbins-Monro then tells me how to absorb noisy samples: the learning rate must keep summing to infinity so the estimate can still move, while the squared learning rates must sum to a finite value so sampling noise averages away.

Prediction is still not control. A state value is only meaningful under a policy, and if I only store state values then choosing between actions requires the missing transition model. So I will store a number for each state-action pair instead. That number can represent the value of taking that action now and then continuing well afterward, and with such a table, action selection can be read directly by choosing the action with the largest current entry at the state. The policy no longer has to be a separate adaptive object.

This matters because separate policy and value learners can chase each other. If I update a value estimate while also updating independent action probabilities or action strengths, the target being evaluated changes while the evaluator is still moving. I would rather have one table carry both roles: it stores action values, and its maximum over actions at a state defines the greedy policy implied by the table.

Now I face the exploration problem. The behavior that generates data cannot always be greedy with respect to the current table, because then I may never try a better action. But if I follow exploratory behavior, ordinary prediction would estimate the value of that exploratory behavior, not the optimal behavior. I need the learning target to ignore the later consequences of exploratory choices except as samples of the environment's one-step dynamics.

The temporal-difference view suggests how to do that. In multi-step returns, an off-policy experimental step contaminates the tail of the return. If I set the trace parameter to zero at such a step, the sampled tail is cut off and replaced by the current value estimate at the next state. In the one-step extreme, the update always stops after the observed transition. That means the action I actually take supplies only the sampled immediate reward and sampled next state; the continuation value can be chosen from the table rather than from the behavior that generated the data.

So I try defining the continuation value at the next state as the maximum table entry over next-state actions. The target for the experienced pair becomes the observed reward plus the discounted maximum current action value at the next state. For the transition `(x, a, r, y)`, I move the table entry for `(x, a)` toward `r + gamma * max_b Q(y, b)` and leave every other entry unchanged. The action `a` can be chosen by any exploratory rule. The sampled experience may come from a wandering behavior policy, while the bootstrapped target evaluates the greedy continuation.

Before I trust any of this I want to know what fixed point this update is even aiming at, because "Bellman optimality with the expectation replaced by a sample" is a slogan until I check that the slogan's fixed point is actually `Q*`. The cleanest check is to take expectations of the update and see whether `Q*` is left unmoved. Let me make it concrete on a process small enough to solve by hand. Two states. State `1` is terminal: every action gives reward `0` and stays, so its optimal value is `0`. State `0` has two actions: action `0` gives reward `0` and returns to state `0`; action `1` gives reward `1` and goes to the terminal state. Take `gamma = 0.9`.

I solve this directly. `Q*(1, .) = 0`. `Q*(0, 1) = 1 + gamma * V*(1) = 1 + 0 = 1`. For the looping action, `Q*(0, 0) = 0 + gamma * V*(0)` and `V*(0) = max(Q*(0,0), Q*(0,1))`. Guessing `V*(0) = 1` gives `Q*(0,0) = 0.9`, and `max(0.9, 1) = 1`, which is consistent, so `Q* = [0.9, 1.0]` at state `0` and `[0, 0]` at state `1`. Now I push these values through the expected one-step target. For `(0,0)`: `0 + gamma * max(0.9, 1.0) = 0.9 * 1.0 = 0.9`. For `(0,1)`: `1 + gamma * max(0, 0) = 1 + 0 = 1.0`. Both targets reproduce `Q*` exactly, so in expectation the update does not move `Q*`. That is the first thing I genuinely needed: the sample target is unbiased for a backup whose unique fixed point is the optimality solution, not merely something that looks like it.

A fixed point being unmoved in expectation does not yet mean the noisy iteration finds it, so I also run the actual update on this process to see where it goes. I drive it with a purely random behavior policy (each action with probability one half, which is maximally off-policy here), use the per-pair step size `alpha = 1/n_i` so that `sum alpha = infinity` and `sum alpha^2 < infinity`, and start the table at zero. After two hundred thousand sampled transitions the table reads `[0.9000, 1.0000]` at state `0` and `[0, 0]` at state `1`, with maximum absolute error about `9e-6` against the hand-computed `Q*`. The estimate converges to the optimal action values even though the data was generated by behavior that never preferred the good action. That is exactly the off-policy property I was hoping for, now observed rather than asserted: the `max` in the target keeps the limit tied to the optimality equation, while the behavior policy only decides which entries get sampled.

I still want to understand why I should restrict to the one-step backup rather than the whole multi-step family, since longer returns usually reduce bias. The danger is in the `max`. The one-step target reads a single number, `max_b Q(y, b)`; if the table at `y` is off by some amount, the number I read can be wrong by at most that amount, because the maximum of a perturbed vector differs from the true maximum by no more than the perturbation. I can see the fragility on a near-tie: with `Q(y, .) = [0.50, 0.49]` the true continuation is `0.50`, and a perturbation of `0.02` on the second entry flips the greedy action to the `0.51` reading, an error of `0.01` in the value read off. That single-step error stays bounded by the table error. A multi-step return, by contrast, follows the greedy policy implied by the table through several states; the same near-tie can commit the rollout to the worse action for a whole step at each visited state, and those errors accumulate along the trajectory instead of being capped at one state. So the one-step version is not just simpler, it keeps each update atomic: one sampled transition, one bootstrapped maximum at the next state, one entry changed. I will keep the proof to that atomic form, and treat the multi-step claim as something I have not established here.

Now I need a real proof, not a numerical hint on one tiny process. My update uses a single random reward and next state, and the sampling schedule can depend on all previous experience, so the convergence has to handle history-dependent action choices and noisy backups under decreasing step sizes. "Asynchronous value iteration with samples" is the right intuition but does not by itself control the dependence on history.

The proof device is the action-replay process. I imagine every observed transition and its learning rate written on a card, ordered by time, with the initial table on a bottom card. A state of the replay process is a real state together with a card level. From level `n`, if I choose action `a` at state `x`, I deal downward until I find a card whose recorded state-action pair is `(x, a)`. With probability equal to that card's learning rate I replay the card's reward and next state, then continue below that card. With the remaining probability I skip the card and keep dealing. If I reach the bottom, the process pays the initial table entry.

This construction looks artificial, so I check that it reproduces the update before I build anything on top of it. I claim the optimal action value of the replay process at level `n` equals the learned table after `n` updates, and I verify the inductive step explicitly. Suppose, below the newest card, the replay process's optimal value already equals the table before this update, so its value at `y` is `max_b Q_old(y, b)`. Take the newest card to be for `(x, a)`, recording reward `r`, next state `y`, and rate `alpha`. Choosing `a` at `x` from the new level, the replay process replays the card with probability `alpha`, emitting `r` and dropping below the card to value `r + gamma * max_b Q_old(y, b)`, and skips it with probability `1 - alpha`, in which case the best available value is the old estimate `Q_old(x, a)`. So the replay value is `(1 - alpha) * Q_old(x, a) + alpha * (r + gamma * max_b Q_old(y, b))`. Putting numbers in to be sure I have the bookkeeping right: with `Q_old(x,a) = 0.3`, `r = 1.0`, `gamma = 0.9`, value-below `= 0.8`, and `alpha = 0.25`, this gives `0.75 * 0.3 + 0.25 * (1.0 + 0.9 * 0.8) = 0.225 + 0.43 = 0.655`. The Q-learning update on the same numbers is `Q_old + alpha * (r + gamma * 0.8 - Q_old) = 0.3 + 0.25 * (1.72 - 0.3) = 0.655`. They agree, and they agree because both are the same convex combination — the replay mixing probability is the learning rate by construction. If instead the newest card is not for `(x, a)`, the search skips past it and nothing changes for that pair, matching the update leaving other entries fixed. So the replay value at each level is exactly the learned table.

After that, I have to show that high levels of the replay process behave like the real Markov process. Discounting lets me truncate comparisons to a finite horizon, because the remaining tail is bounded by `gamma^H` times a reward bound and shrinks geometrically in the horizon `H`. The condition that the learning rates for each state-action pair sum to infinity makes it unlikely that the replay process skips all matching cards and falls far down the deck: skipping all of them costs a product of `(1 - alpha)` factors whose log sums to minus infinity, so that product goes to zero. The condition that squared learning rates sum finitely lets the replay process's reward and transition averages converge, by Robbins-Monro, to the real reward means and transition probabilities. Finally, finite-horizon values are continuous in rewards and transition probabilities, so close one-step models imply close finite-horizon action values.

Putting these pieces together, for any error tolerance I choose a horizon `H` long enough that discounted tails are below the tolerance, then choose a replay level high enough that the replay process stays within levels whose one-step rewards and transitions are close to the real process with high probability. The finite-horizon values are then close, the tails are close, and therefore the optimal action values of the replay process are close to the real optimal action values. Since the replay process's optimal action values are the learned table entries, the learned table converges to the optimal action-value function with probability one.

The theorem then needs exactly the assumptions the argument used: bounded rewards, finite tabular state and action sets, discounting or an absorbing alternative that keeps values bounded, every state-action pair visited infinitely often, and per-pair learning rates satisfying `sum alpha = infinity` and `sum alpha^2 < infinity`. Under those conditions the behavior policy is otherwise free. It can explore however it likes — which is what the random-behavior run already showed on the small process — because the maximum in the target keeps the learned fixed point tied to the optimality equation rather than to the behavior policy.

What I have arrived at is a method that learns optimal action values directly from model-free experience: it updates one entry at a time, bootstraps from the maximum next-state action value, and separates behavior from the target being learned. The table is not estimating the value of what the agent actually does while exploring. It is estimating the optimal action-value function whose greedy policy is optimal — the object I verified it converges to. This is Q-learning.
