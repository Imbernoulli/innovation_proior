Let me start from what a convolution actually buys me on a grid, because that's the thing I want to carry over and it's worth being precise about. On an image, a 3×3 filter is a small set of weights that I slide across every pixel; the same weights are reused at every location, which is what makes the model parameter-efficient and translation-equivariant. The deep reason it works is that a grid has a fixed, canonical local geometry: "the pixel above," "the pixel to the upper-left" mean the same thing everywhere, so a fixed weight can be assigned to each relative position. Now look at a graph — a citation network, a protein-interaction graph, a mesh. A node has some number of neighbors, and that number varies from node to node; worse, there is no canonical ordering or geometry to those neighbors. There is no "the neighbor to the upper-left." So the very thing that makes a CNN filter well-defined — a fixed slot per relative position — has no analogue. If I want a graph layer that produces a new feature vector for each node from its own features and its neighbors', I have to confront this head-on: the operator has to accept a variable-sized, unordered set of neighbors and still share its parameters across all nodes (and ideally across all graphs).

The route everyone reaches for first is spectral. Convolution is multiplication in the Fourier domain, and a graph has a Fourier transform: take the normalized Laplacian L = I − D^{-1/2} A D^{-1/2}, diagonalize it L = U Λ U^T, and the columns of U are the graph's Fourier modes. A spectral filter is g_θ ⋆ x = U g_θ(Λ) U^T x, with g_θ(Λ) a diagonal operator on the spectrum. It's clean, it's principled, and for a while it was the prestige path. But stare at what the filter actually *is*: a function of the eigenvalues of *this* graph's Laplacian, applied via *this* graph's eigenvectors U. The parameters live in the Laplacian eigenbasis, and that eigenbasis is a property of the specific graph. So a model fitted on one graph's structure isn't merely *worse* on a graph with different connectivity — it isn't even *defined* there; U has a different size and meaning. That kills inductive transfer outright. And the costs are brutal: the eigendecomposition is O(N^3), a forward pass with dense U is O(N^2), and the free per-eigenvalue filter has O(N) parameters and isn't localized in space. People patched these one by one — Henaff smoothed the spectral multipliers to localize the filters, Defferrard approximated g_θ(Λ) by a truncated Chebyshev expansion Σ_k θ_k T_k(Λ̃) with Λ̃ = 2Λ/λ_max − I, which, because (UΛU^T)^k = U Λ^k U^T, can be applied directly in the vertex domain as Σ_k θ_k T_k(L̃), a degree-k polynomial in L that only reaches the k-hop neighborhood and costs O(K|E|) with no eigendecomposition. That's a real fix for cost and locality. But the deeper commitment survives every one of these patches: the operation is still a fixed function of *the graph's own* Laplacian. Once the θ's are learned, the graph structure alone dictates how neighbors get mixed.

Kipf and Welling pushed the Chebyshev idea to its minimal limit and got something I should look at carefully, because it became the thing to beat. Take K=1, approximate λ_max ≈ 2 (the network can absorb the scale), and the filter collapses to g_θ ⋆ x ≈ θ_0 x − θ_1 D^{-1/2} A D^{-1/2} x. Then tie the two parameters, θ = θ_0 = −θ_1, to fight overfitting, giving θ(I + D^{-1/2} A D^{-1/2}) x. There's a numerical wrinkle: I + D^{-1/2} A D^{-1/2} has eigenvalues in [0, 2], so repeated application can amplify or shrink components unpredictably. Their renormalization trick swaps it for D̃^{-1/2} Ã D̃^{-1/2} with self-loops Ã = A + I. The layer is H^{(l+1)} = σ(D̃^{-1/2} Ã D̃^{-1/2} H^{(l)} W^{(l)}), it is linear in the edges up to the feature dimensions, and it is genuinely strong on citation networks. So this is the bar. But now I want to put my finger on *exactly* what it cannot do, because that gap is going to be my opening. Look at the propagation matrix D̃^{-1/2} Ã D̃^{-1/2}. For an unweighted edge between i and j, the coefficient node i applies to neighbor j is precisely 1/√(d̃_i d̃_j) — a number determined entirely by the two degrees, identical for every model that will ever be trained on this graph, and completely untrainable. The layer averages neighbors with weights it was handed, not weights it chose. If two of my neighbors are a relevant collaborator and an off-topic citation, GCN must mix them with degree-fixed coefficients; it has no knob to say "trust this one more." That's a capacity ceiling, and it's structural, not a tuning issue. And in the usual full-graph semi-supervised setup, this fixed normalization is built from the graph at hand; the model is not learning a neighbor-weighting function that is independent of that graph-specific propagation matrix.

The spatial camp tried to escape the eigenbasis by aggregating directly over close neighbors, and their struggles are instructive about what *not* to do. Duvenaud learned a separate weight matrix per node degree — which doesn't scale when degrees range widely. Niepert extracted fixed-size, *ordered* neighborhood patches so a vanilla CNN could run on them — but a graph neighborhood has no order, so imposing one is a fiction. Monti unified a lot of this under MoNet: assign each node pair a pseudo-coordinate u(x, y) and learn weighting kernels w_j(u) over those coordinates, with GCN and diffusion-conv as special cases. Elegant, but the pseudo-coordinates are *structural* — functions of node degrees — so they again presuppose the graph's structure is known and shared. Every spatial method I look at either imposes an order, indexes by degree, or leans on structural coordinates.

And then there's Hamilton's GraphSAGE, which is the one that actually cracks induction, and I need to learn from both what it gets right and where it stops short. Its move is to stop learning a fixed embedding per node — that's the matrix-factorization trap that ties you to one graph — and instead learn *aggregator functions*: h_v^k = σ(W^k · concat(h_v^{k-1}, AGG_k({h_u^{k-1} : u ∈ N(v)}))). Because the aggregator is a shared function of features, you can run it on a node, or a whole graph, never seen at training time. That's the right idea, and I'm going to keep it: parameterize a *function* of features, not a table indexed by node. But look at the choices forced on it. First, N(v) is a *fixed-size uniform sample* of the neighbors, taken to keep the per-batch compute constant — which means the model never actually sees the whole neighborhood, it gambles on a random subset each step. Second, the aggregators it offers — mean, GCN-style — pool the sampled neighbors *equally*; same flaw as GCN, no learned per-neighbor importance. And third, one aggregator is an *LSTM*. An LSTM over a *set*? A set has no order. They patch it by feeding the neighbors in random permutations every time, hoping the LSTM averages over orderings — but that's spending a sequence model's machinery to undo an order that was never there. So GraphSAGE buys induction but pays for it with sampling, equal weighting, and a fake ordering.

Let me collect the wishlist that's emerging from all these dead ends, because the constraints are now sharp. I want a layer that (1) is a shared function of features, so it's inductive — keep GraphSAGE's lesson; (2) reaches the *whole* neighborhood, not a sample; (3) needs no eigendecomposition, no matrix inversion, no order, no degree-indexed weights, no structural pseudo-coordinates; and (4) — the thing *none* of them has cleanly — can assign *different importances* to different neighbors, learned from the features themselves. Constraints (1)–(3) say: operate edge-by-edge, locally, on feature vectors. Constraint (4) says: I need a learned, data-dependent weight on each edge. A learned, normalized, data-dependent weight over a variable-sized unordered set... that's not a graph idea at all. That's attention.

Attention is exactly the tool for "score a variable-sized set of items by relevance and take a weighted combination." Bahdanau's version: given a query and a set of items {h_j}, score each pair with a small feedforward net e_ij = v_a^T tanh(W_a s_{i-1} + U_a h_j), normalize α_ij = exp(e_ij)/Σ_k exp(e_ik), return Σ_j α_ij h_j. It handles variable-sized inputs natively — you just score however many items you're given — and the α's are interpretable as alignments. Vaswani then showed self-attention (a set attending over itself) is enough on its own to build a top sequence model, with multiple heads in parallel for stability. The fit to my problem is almost suspiciously good. Let a node be the query, its neighbors be the items, and let the node attend over its neighborhood. The attention weight on each neighbor *is* the learned per-neighbor importance GCN couldn't have; it's defined per-edge from features, so there is no graph-specific eigenbasis or learned node table, and the current graph enters only by telling me which neighbor pairs may compete. It eats variable-degree, unordered neighborhoods for free. Let me try to build the layer.

First I need at least one learnable linear transform, because the raw input features have to be lifted to a space where I can both compare nodes and produce useful outputs — a pure attention-weighted average of *unprojected* features would have no capacity to transform them. So a shared weight matrix W ∈ R^{F'×F} applied to every node, exactly as a CNN shares its filter. Shared across all nodes (and graphs) is the whole point of induction. So the basic ingredients are Wh_i for the query node and Wh_j for each neighbor.

Now the scoring function a(Wh_i, Wh_j) → a scalar e_ij, the importance of node j's features to node i. What form? The Transformer's default is the dot product, (Wh_i)·(Wh_j). Tempting, it's cheap. But pause on what it would mean *here*. After the shared W, a dot product is a fixed symmetric bilinear form on the projected features — there's no further trainable parameter shaping *how* I compare i and j beyond W itself, and W is already busy producing the output features. I want the *comparison* to have its own capacity, decoupled from the feature transform, so the layer has freedom in how it decides two nodes are similar — independent of the graph's topology. Bahdanau's additive form gives me exactly that: a small single-layer network with its *own* weight vector scoring the pair.

There's a second reason to prefer the additive form, and I want to actually check it rather than wave at it, because "importance shouldn't be symmetric" is the kind of claim that's easy to assert and easy to get wrong. Take a learnable vector a and the concatenation: e_ij = a^T [Wh_i ‖ Wh_j], with a ∈ R^{2F'} since the concatenation is 2F'-dimensional. Split a into its halves a = [a_l ; a_r]; then e_ij = a_l^T(Wh_i) + a_r^T(Wh_j) and the swapped score is e_ji = a_l^T(Wh_j) + a_r^T(Wh_i). Let me put numbers on it. Say two neighbors have projected vectors Wh_i = (1, 0, −1) and Wh_j = (0.5, 2, 0), and a fixed (random) scorer with a_l = (1.62, −0.61, −0.53), a_r = (−1.07, 0.87, −2.30). Then e_ij = a_l·Wh_i + a_r·Wh_j = (1.62 + 0 + 0.53) + (−0.535 + 1.74 + 0) = 2.15 + 1.205 = 3.36, while e_ji = a_l·Wh_j + a_r·Wh_i = (0.81 − 1.22 + 0) + (−1.07 + 0 + 2.30) = −0.41 + 1.23 = 0.82. So e_ij ≈ 3.36 but e_ji ≈ 0.82 — genuinely different, the score is asymmetric. Now do the same pair under a dot product: Wh_i·Wh_j = 0.5 + 0 + 0 = 0.5, and Wh_j·Wh_i = 0.5 — identical, necessarily, since the dot product is symmetric. That's the concrete confirmation I wanted: a hub paper *can* be scored very important to a niche one without the reverse holding under the additive form, whereas the dot product forces e_ij = e_ji and cannot express that. So I take the concatenation form e_ij = a^T [Wh_i ‖ Wh_j]. Concatenation, not sum, because [Wh_i ‖ Wh_j] lets a hold *separate* weights for the query half and the key half — the first F' entries of a act on Wh_i, the last F' on Wh_j — which is exactly the split that just produced the asymmetry above and gives the score its real expressive room.

Then a nonlinearity on the score before I normalize. The single linear score a^T[·‖·] would otherwise rank pairs with one affine ordering before the softmax; adding a kink gives the scorer a little more shape without making the edge computation expensive. Why not plain ReLU? Because ReLU zeros every negative pre-score and, crucially, kills the gradient there — but a low pre-score is meaningful: it's the layer saying "this neighbor is *un*important," and I need to be able to learn to push a coefficient *down*, which means the negative region must carry gradient. LeakyReLU does exactly that — it passes negatives through with a small slope, keeping the gradient alive on both sides — so I use LeakyReLU with a modest negative slope (0.2). So the raw score is e_ij = LeakyReLU(a^T[Wh_i ‖ Wh_j]).

Now normalization, and here I have a choice about *which* nodes to even score. In its most general form, every node attends over every other node — full self-attention, N^2 pairs. But that does two bad things at once: it's O(N^2), and it throws the graph away entirely, which is the one piece of structure I actually have and want to use. So I score only j ∈ N_i, the neighbors of i (I'll include i itself, so a node attends to itself too — that's the self-loop, so a node can keep its own features). This *masked* attention injects the graph: structure enters not as a propagation matrix but as a mask on which scores I bother to compute. Then I normalize the scores of each node across its own neighborhood with a softmax: α_ij = softmax_j(e_ij) = exp(e_ij) / Σ_{k∈N_i} exp(e_ik). Softmax does two jobs. It makes the coefficients sum to one *per node*, so a degree-7 node and a degree-200 node both produce a convex combination — the weights are comparable across nodes of wildly different degree, which is precisely the variable-neighborhood problem solved cleanly. And it makes them positive and probability-like, hence the interpretable-alignment property. Writing it out fully:

  α_ij = exp(LeakyReLU(a^T [Wh_i ‖ Wh_j])) / Σ_{k∈N_i} exp(LeakyReLU(a^T [Wh_i ‖ Wh_k])).

And the output of the layer for node i is the attention-weighted sum of its neighbors' transformed features, through a final nonlinearity σ:

  h'_i = σ( Σ_{j∈N_i} α_ij Wh_j ).

Let me check this against the wishlist before going further. Inductive? Yes — a, W, and the whole computation are shared functions of features, applied per-edge; nothing references node identities or a fixed U, so I can drop the trained layer onto a brand-new graph. Whole neighborhood? Yes — I sum over all of N_i, no sampling. Cheap? Let me count. Computing all Wh_i is O(|V| F F'). Computing all the e_ij costs O(|E| F') — one a^T over a 2F'-vector per edge. The weighted sum is O(|E| F'). So a single head is O(|V| F F' + |E| F'), on par with GCN, and there is no eigendecomposition, no inversion, no order, no degree bank, no structural coordinates. Different importance per neighbor? Yes — that's the α_ij, learned from features, the exact thing GCN's 1/√(d̃_i d̃_j) couldn't be. The whole wishlist falls out of "let the node attend over its neighbors."

There's an efficiency point I don't want to miss when I implement e_ij. Naively I'd build, for each edge, the concatenation [Wh_i ‖ Wh_j] and dot it with a — materializing a 2F'-vector per edge. But a^T[Wh_i ‖ Wh_j] is *linear* in the concatenation, so split a into its two halves, a = [a_l ; a_r], each in R^{F'}, and the score should factor:

  a^T [Wh_i ‖ Wh_j] = a_l^T (Wh_i) + a_r^T (Wh_j).

This is the optimization the whole sparse implementation hinges on, so I'll confirm the equality holds on a concrete instance instead of trusting the algebra blind. Reuse Wh_i = (1, 0, −1), Wh_j = (0.5, 2, 0) and a = [a_l ; a_r] with a_l = (1.62, −0.61, −0.53), a_r = (−1.07, 0.87, −2.30) from a moment ago. The honest "build the concatenation" route forms the 6-vector [Wh_i ‖ Wh_j] = (1, 0, −1, 0.5, 2, 0) and dots it with a = (1.62, −0.61, −0.53, −1.07, 0.87, −2.30): 1·1.62 + 0·(−0.61) + (−1)·(−0.53) + 0.5·(−1.07) + 2·0.87 + 0·(−2.30) = 1.62 + 0.53 − 0.535 + 1.74 = 3.355. The split route gives a_l·Wh_i + a_r·Wh_j = 2.15 + 1.21 = 3.36 (same number, up to the rounding I did by hand). They agree, as the linearity demands — and note this is exactly e_ij from the asymmetry check, which is the right cross-check that I've kept the two computations consistent. So I compute two cheap scalars per node — a "source" score a_l^T Wh_i for every i and a "target" score a_r^T Wh_j for every j, each an O(|V| F') pass — and the edge score is just their sum, broadcast over edges (or, densely, an N×1 column plus a 1×N row giving the N×N score matrix: entry (i, j) is src_i + tgt_j, and plugging i, j here returns 2.15 + 1.21 = 3.36 again). No N^2 concatenations ever get built. That's the form I'll actually code.

Now I should be honest about a wobble: self-attention can be unstable to train, especially on small datasets, and a single attention head is one noisy estimate of "who matters." This is exactly the situation the Transformer met and answered with multi-head attention — run several independent attention mechanisms in parallel. So let me do the same. K independent heads, each with its own W^k and a^k computing its own α_ij^k, and for the hidden layers I *concatenate* their outputs:

  h'_i = ‖_{k=1}^K σ( Σ_{j∈N_i} α_ij^k W^k h_j ),

giving K·F' features per node. Why concatenate rather than average here? Because each head learns its own notion of relevance — its own α and its own projection — and I want to *keep* all those distinct views as separate channels for the next layer to use; averaging them prematurely would collapse the diversity that's the whole reason for multiple heads. Concatenation also makes the stabilization concrete: even if one head's attention is erratic on a given batch, the others' channels survive intact, so the representation doesn't hinge on any single head. Storage and parameters scale by K, but the heads are fully independent so they parallelize.

But concatenation can't be right on the *final* layer. There the output dimension has to be the number of classes C, and concatenating K independent C-dimensional class-score vectors would give KC numbers with no sensible interpretation — you can't concatenate logits and call them logits. So on the prediction layer I *average* the heads instead, and delay the final nonlinearity (softmax/sigmoid) until after the averaging:

  h'_i = σ( (1/K) Σ_{k=1}^K Σ_{j∈N_i} α_ij^k W^k h_j ).

Averaging keeps the output C-dimensional and still pools the heads' evidence; the late nonlinearity means the heads are combined in logit space, which is the natural place to ensemble classifiers.

A couple of regularization and stability details that the small transductive datasets force. With only twenty labeled nodes per class, this model — multiple heads, a trainable scorer — will overfit instantly without help. The usual dropout on layer inputs, sure. But there's a regularizer that's *natural* to this architecture specifically: dropout on the *normalized attention coefficients* α_ij. Drop a fraction of the edges' attention weights after the softmax, with the usual dropout scaling on the survivors, and at every training step each node is effectively exposed to a *stochastically sampled neighborhood*. It's like edge-dropout that arises for free from the attention formulation, and it's a strong regularizer precisely because it perturbs the one new thing the model is learning, the neighbor weighting. For the hidden nonlinearity σ I'll use ELU — smooth, with a nonzero negative branch, which plays well with the attention sums that can be negative. And I'll initialize W and a with Glorot/Xavier scaling (a gain around √2 to account for the rectifier-style nonlinearity) so the early-training scores aren't saturated.

Let me also sanity-check that this contains GCN as a degenerate case — if a *fixed* attention can't recover GCN-like averaging, I'd worry I've changed the operator's character rather than generalized it. Take the scoring function to be constant, a(x, y) = 1 for every pair, so every e_ij is some common constant c. I'll work a concrete tiny graph rather than just claim the limit. Three nodes, self-loops added: node 0 is adjacent to {0, 1, 2}, node 1 to {0, 1}, node 2 to {0, 2}, so the augmented degrees are d̃_0 = 3, d̃_1 = 2, d̃_2 = 2. For node 0 the masked softmax runs over its three neighbors with equal logits c: α_0j = e^c / (e^c + e^c + e^c) = e^c/(3e^c) = 1/3 for each of j ∈ {0, 1, 2}, independent of the value of c. For node 1, two equal logits give α_1j = e^c/(2e^c) = 1/2 for j ∈ {0, 1}, and the non-neighbor (node 2) was masked to −∞ so its weight is e^{−∞}/Z = 0 — I should and do get exactly 0 there, not a small leak. So constant attention produces α_ij = 1/|N_i|: 1/3 for node 0's neighbors, 1/2 for node 1's and node 2's. That is uniform averaging over the self-loop-augmented neighborhood — a GCN-*style* mean aggregator.

It is worth being precise that this is *not* numerically identical to Kipf–Welling's operator, and seeing the gap reassures me I haven't accidentally just reinvented GCN. Their coefficient for the (i, j) edge is 1/√(d̃_i d̃_j). For the 0–1 edge that is 1/√(3·2) = 1/√6 ≈ 0.408, whereas my constant-attention weight from node 0's side is 1/3 ≈ 0.333 and from node 1's side is 1/2 = 0.5 — three different numbers. So the constant-attention layer and GCN are *both* fixed-weight averages but with different normalizations (row-stochastic 1/|N_i| versus symmetric 1/√(d̃_i d̃_j)); they coincide in spirit, not to the digit. The point that matters is the structural one: GCN-style equal weighting is precisely the special case where I refuse to let the scorer depend on features, and turning the scorer on — letting a^T[Wh_i‖Wh_j] vary with the features — is what adds capacity on top of it. So I haven't replaced averaging, I've nested it as the a≡const corner of a family whose other members weight neighbors by learned relevance.

There's also the question of scale, since I'm aggregating into larger and larger receptive fields by stacking. The receptive field of this model after L layers is the L-hop neighborhood — same locality bound as GCN — so if I want a node to see farther I just stack more layers, and to train deep stacks I can drop in residual/skip connections (h'_i + h_i, or a projected version when dimensions differ) the way image nets do. On a single very large graph I'd also want the *sparse* form: never build the dense N×N score matrix, only the |E| edge scores, scatter them into a sparse softmax over each node's incident edges, and do a sparse matmul for the weighted sum — that brings storage to O(|V| + |E|). The dense and sparse forms compute the identical layer; the sparse one is what scales.

So now I'll write the layer the way I'd actually ship it. The query/key/value here are all the same projected features Wh — it's self-attention over the neighborhood — and the masking is "set the score of any non-neighbor to a large negative number before the softmax, so its exp is ~0." That mask is how `adj` enters. Concretely, for a single dense head:

```python
import torch
import torch.nn as nn
import torch.nn.functional as F


class GraphAttentionLayer(nn.Module):
    """One graph attention head. Maps node features h (N x F_in) to h' (N x F_out),
    weighting each neighbor by a learned, feature-dependent attention coefficient."""

    def __init__(self, in_features, out_features, dropout, alpha=0.2, concat=True):
        super().__init__()
        self.dropout = dropout              # dropout on the attention coefficients
        self.out_features = out_features
        self.concat = concat               # True for hidden layers (ELU), False for the last

        # shared linear transform W: lifts features so we can compare and output (eq for Wh)
        self.W = nn.Parameter(torch.empty(in_features, out_features))
        nn.init.xavier_uniform_(self.W, gain=1.414)        # Glorot init, rectifier gain
        # the attention vector a, split into its source half a_l and target half a_r
        self.a = nn.Parameter(torch.empty(2 * out_features, 1))
        nn.init.xavier_uniform_(self.a, gain=1.414)

        self.leakyrelu = nn.LeakyReLU(alpha)               # keeps gradient on negative scores

    def forward(self, h, adj):
        Wh = h @ self.W                                    # Wh_i for every node: (N, F_out)

        # e_ij = LeakyReLU(a^T [Wh_i || Wh_j]) computed as a_l^T Wh_i + a_r^T Wh_j
        Wh_src = Wh @ self.a[: self.out_features, :]       # (N, 1): source score per node
        Wh_tgt = Wh @ self.a[self.out_features :, :]       # (N, 1): target score per node
        e = self.leakyrelu(Wh_src + Wh_tgt.T)              # (N, N): broadcast-add to all pairs

        # masked attention: only neighbors compete; non-neighbors get ~0 weight after softmax
        e = e.masked_fill(adj <= 0, float("-inf"))
        alpha = F.softmax(e, dim=1)                        # alpha_ij = softmax over N_i
        alpha = F.dropout(alpha, self.dropout, training=self.training)  # stochastic neighborhood

        h_prime = alpha @ Wh                               # h'_i = sum_j alpha_ij Wh_j
        return F.elu(h_prime) if self.concat else h_prime  # ELU in hidden layers; raw for last
```

And the full network: a hidden layer of K heads concatenated, then output heads whose class-score tensors are averaged before the task softmax, with dropout on the inputs of each layer:

```python
class GAT(nn.Module):
    def __init__(self, nfeat, nhid, nclass, dropout, nheads, out_heads=1, alpha=0.2):
        super().__init__()
        self.dropout = dropout
        # hidden layer: K independent heads, outputs CONCATENATED -> nhid*nheads features
        self.heads = nn.ModuleList(
            GraphAttentionLayer(nfeat, nhid, dropout=dropout, alpha=alpha, concat=True)
            for _ in range(nheads)
        )
        # prediction layer: heads produce class-score logits, then get AVERAGED
        self.out_heads = nn.ModuleList(
            GraphAttentionLayer(nhid * nheads, nclass, dropout=dropout,
                                alpha=alpha, concat=False)
            for _ in range(out_heads)
        )

    def forward(self, x, adj):
        x = F.dropout(x, self.dropout, training=self.training)
        x = torch.cat([head(x, adj) for head in self.heads], dim=1)   # concat the K heads
        x = F.dropout(x, self.dropout, training=self.training)
        logits = torch.stack([head(x, adj) for head in self.out_heads], dim=0).mean(dim=0)
        return F.log_softmax(logits, dim=1)                            # final nonlinearity delayed
```
