I am asked to decide, for every integer n > 1, whether n is prime by a deterministic algorithm whose running time is polynomial in the bit length log n, using no randomness and assuming no unproved hypotheses such as the Extended Riemann Hypothesis. This is the primality-testing problem in its strongest form. The method I propose is the AKS primality test, named after Manindra Agrawal, Neeraj Kayal, and Nitin Saxena. It is the first unconditional deterministic polynomial-time algorithm for primality, and its correctness comes from compressing an exact polynomial identity rather than from probabilistic or conjectural shortcuts.

The starting point is Fermat's little theorem, which says that if n is prime and gcd(a, n) = 1 then a^(n-1) ≡ 1 (mod n). This gives a fast necessary test for primality, but the converse fails badly: Carmichael numbers satisfy the Fermat congruence for every coprime base, so a deterministic Fermat test with any fixed list of bases can never be complete. The AKS test escapes this trap by replacing the scalar congruence with a polynomial identity. The binomial theorem gives, for any integer n and any a coprime to n, that n is prime if and only if (X + a)^n ≡ X^n + a as an identity in the polynomial ring (Z/nZ)[X]. When n is prime, the Frobenius endomorphism gives (X + a)^n = X^n + a^n, and Fermat's theorem turns a^n into a, so every middle binomial coefficient vanishes modulo n. When n is composite, choose a prime power q^k that exactly divides n; the coefficient of X^q in (X + a)^n is binom(n, q) a^(n-q), and the full power q^k in n does not divide binom(n, q), so the identity fails. This test is logically perfect, but checking it directly requires inspecting n+1 coefficients, which is even slower than trial division.

The central idea of AKS is to keep this polynomial identity but to evaluate it in a quotient ring where the degree is bounded. Working in (Z/nZ)[X]/(X^r - 1) collapses every monomial X^k to X^(k mod r), so repeated squaring keeps only r coefficients and runs in time polynomial in r and log n. The danger is that the quotient could hide a bad coefficient from a composite input. To prevent this, AKS chooses an auxiliary integer r such that the multiplicative order of n modulo r is larger than (log n)^2. Such an r exists and is bounded by a polynomial in log n; the proof is an elementary estimate on least common multiples. With this order condition, the quotient sees enough distinct powers of X that a composite cannot smuggle its failure past the test.

The algorithm also checks the identity for many small shifts rather than a single one. Concretely, after selecting r, it tests (X + a)^n ≡ X^n + a modulo (X^r - 1, n) for every a from 1 up to floor(sqrt(phi(r)) log n), where phi is Euler's totient function. The number of shifts is chosen so that the linear factors X, X + 1, ..., X + ell generate a large subgroup of a certain finite field, which is the setting for the counting argument that rules out composites.

Before the main polynomial loop, AKS applies three simple gates. First, it rejects every perfect power a^b with b > 1, because the eventual proof can only force n to be a prime power, and this gate removes the composite-power possibility. Second, it searches for the small auxiliary r described above. Third, it checks gcd(a, n) for every a ≤ r; if any gcd lies strictly between 1 and n, it outputs composite, and if n ≤ r, it outputs prime, since a composite n no larger than r would already have a prime factor at most sqrt(n) ≤ r and would have been caught.

Correctness rests on a beautiful lower bound and upper bound for the same group. Assume the algorithm reaches the final step and choose a prime divisor p of n whose order modulo r exceeds one; such a p exists because ord_r(n) > 1. The accepted congruences imply that both p and n/p are introspective for each tested X + a, meaning f(X)^m ≡ f(X^m) modulo (X^r - 1, p). By closure under multiplication, every number of the form (n/p)^i p^j is introspective for every product of X, X + 1, ..., X + ell. Reducing modulo an irreducible factor of the r-th cyclotomic polynomial over F_p produces a field F in which X is a primitive r-th root of unity. The subgroup G_1 of residues of (n/p)^i p^j in (Z/rZ)^* has size t > (log n)^2, and the subgroup G_2 of F^* generated by X, X + 1, ..., X + ell has more than n^sqrt(t) elements because distinct low-degree products remain distinct. On the other hand, if n were not a power of p, a collision among the (n/p)^i p^j modulo r would force every element of G_2 to be a root of a polynomial of degree at most n^sqrt(t), giving |G_2| ≤ n^sqrt(t), a contradiction. Therefore n must be a power of p, and the perfect-power gate leaves only n = p. Every composite is rejected and every prime is accepted, deterministically and in polynomial time.

The complexity is dominated by the main loop. The search for r is polynomial because r is bounded by O((log n)^5) using elementary estimates, or even O((log n)^3) using deeper analytic number theory. Step 5 performs O(sqrt(r) log n) polynomial exponentiations, each costing O(log n) multiplications of degree-r polynomials with O(log n)-bit coefficients. With the stronger bound on r this gives an overall time of roughly O-tilde((log n)^(15/2)), while the elementary bound still gives O-tilde((log n)^(21/2)). Both are polynomial in the input size.

To make the algorithm concrete, here is a compact Python implementation that follows the same logical steps. It is intended for illustration and for checking moderate inputs; an industrial implementation would use asymptotically faster integer and polynomial arithmetic.

```python
import math

def is_perfect_power(n):
    if n < 4:
        return False
    max_b = int(math.log2(n)) + 1
    for b in range(2, max_b + 1):
        a = int(round(n ** (1.0 / b)))
        for cand in range(max(2, a - 2), a + 3):
            p = pow(cand, b)
            if p == n:
                return True
            if p > n:
                break
    return False

def mult_order(n, r):
    order = 1
    cur = n % r
    while cur != 1:
        cur = (cur * n) % r
        order += 1
    return order

def euler_phi(m):
    result = m
    x = m
    p = 2
    while p * p <= x:
        if x % p == 0:
            while x % p == 0:
                x //= p
            result -= result // p
        p += 1
    if x > 1:
        result -= result // x
    return result

def find_r(n):
    logn = math.log2(n)
    target = logn ** 2
    r = 2
    while True:
        if math.gcd(n, r) == 1 and mult_order(n, r) > target:
            return r
        r += 1

def poly_mul(p, q, r, mod):
    res = [0] * r
    for i in range(r):
        if p[i] == 0:
            continue
        for j in range(r):
            if q[j] == 0:
                continue
            k = i + j
            if k >= r:
                k -= r
            res[k] = (res[k] + p[i] * q[j]) % mod
    return res

def poly_pow(base, exp, r, mod):
    result = [1] + [0] * (r - 1)
    while exp > 0:
        if exp & 1:
            result = poly_mul(result, base, r, mod)
        base = poly_mul(base, base, r, mod)
        exp >>= 1
    return result

def aks_is_prime(n):
    if n < 2:
        return False
    if is_perfect_power(n):
        return False

    r = find_r(n)

    for a in range(1, r + 1):
        g = math.gcd(a, n)
        if 1 < g < n:
            return False

    if n <= r:
        return True

    logn = math.log2(n)
    limit = int(math.sqrt(euler_phi(r)) * logn)

    for a in range(1, limit + 1):
        base = [a % n, 1] + [0] * (r - 2)
        left = poly_pow(base, n, r, n)
        right = [0] * r
        right[0] = a % n
        right[n % r] = (right[n % r] + 1) % n
        if left != right:
            return False

    return True

if __name__ == "__main__":
    test_values = [2, 3, 4, 17, 18, 97, 100, 541, 1009, 1729]
    for n in test_values:
        print(n, "prime" if aks_is_prime(n) else "composite")
```
