I start with the part of primality testing that almost works and keep asking what information it throws away. Fermat gives me a fast necessary condition: for prime `n`, `a^(n-1) equiv 1 mod n`. Repeated squaring makes this cheap. But it is too scalar. A Carmichael number can imitate a prime for every coprime base, so if I only look at one residue after exponentiation, I have collapsed too much structure. I want something that still reduces to Fermat for primes but carries more witnesses against composites.

The binomial theorem gives me a richer object without leaving the same universe. If `n` is prime, then in characteristic `n` the Frobenius map says `(X+a)^n = X^n + a^n`, and Fermat turns `a^n` into `a` when `gcd(a,n)=1`. Equivalently, all the middle binomial coefficients vanish modulo `n`. The question is whether the converse holds: does the identity `(X+a)^n equiv X^n+a mod n` fail for every composite? Take a prime `q` with `q^k || n`. The coefficient of `X^q` in the difference is `binom(n,q) a^(n-q)`. I want to know that `q^k` does not fully divide `binom(n,q)`, so the coefficient survives modulo `n`. Let me actually look at the `q`-adic valuation rather than assert it. For `n=15` (`q=3`, `k=1`), `binom(15,3)=455`, and `455 = 5 cdot 7 cdot 13` carries no factor of `3` at all, so `v_3=0 < 1=k`. For `n=25` (`q=5`, `k=2`), `binom(25,5)=53130 = 2 cdot 3 cdot 5 cdot 7 cdot 11 cdot 23`, so `v_5=1 < 2=k`. For the Carmichael number `561` (`q=3`), `binom(561,3) equiv 187 mod 561 neq 0`. In each case the coefficient of `X^q` is nonzero modulo `n`, and `a` contributes no factor of `q`, so the difference is genuinely nonzero. That is enough to convince me the full polynomial identity `(X+a)^n equiv X^n+a mod n` characterizes primes for coprime `a`.

That is too large as an algorithm. I have replaced a scalar residue by `n+1` coefficients, which is worse than trial division. I need to keep the identity but stop it from expanding into degree `n`. The natural way to bound the degree is to work modulo a polynomial, so powers of `X` wrap around. If I reduce modulo `X^r-1`, every monomial `X^k` becomes `X^(k mod r)`, and repeated squaring on polynomials now keeps only `r` coefficients. This is the first compression: check

`(X+a)^n equiv X^n+a mod (X^r-1, n)`.

For primes this is still safe, because a true identity remains true after quotienting. The danger is entirely in the reverse direction, and I should test how dangerous it is before building any theory on top of it. So I run the compressed test by hand on a small composite. Take `n=4`, `r=5`. Computing `(X+a)^4 - (X^4+a)` in `(Z/4Z)[X]/(X^5-1)` for `a=1,2,3,4`, only `a=4` gives the zero polynomial; the other shifts leave a nonzero residue. So a single well-chosen shift can be passed by a composite — `n=4` survives `a=4`. A single shift is not a proof, and a randomly chosen shift is not a deterministic proof. I need a quotient and a finite family of shifts arranged so that no composite can keep hiding behind cancellations modulo `X^r-1`.

What should `r` control? Since `X^r=1`, substituting `X -> X^m` only depends on `m mod r`. If `n` has small order modulo `r`, then the powers generated by repeatedly applying the exponent `n` create only a small orbit, and the quotient sees too little. I want `n` to move the primitive `r`-th roots through many distinct positions. So I look for an `r` with `ord_r(n) > (log n)^2`. This is not just a parameter trick; it should give the later counting argument room. I also need `r` itself bounded, or the whole compression is pointless. Let `B = ceil((log n)^5)`, and look at the smallest positive integer that does not divide `n^floor(log B) prod_{i <= floor((log n)^2)} (n^i-1)`. Minimality forces it to be coprime to `n`, and if its order were at most `(log n)^2` it would divide one of the `n^i-1` factors, hence divide the product. So such a smallest non-divisor has the order I want; I just need it to be small.

For that I need the product to be smaller than the lcm of the first `B` integers, so that some integer up to `B` fails to divide it. The lcm bound I have in hand is `lcm(1,...,m) >= 2^m` for `m >= 7`, and I should not take the threshold on faith. Computing directly: `lcm(1..6)=60 < 64=2^6`, so the bound genuinely fails at `m=6`; but `lcm(1..7)=420 > 128`, `lcm(1..8)=840 > 256`, `lcm(1..10)=2520 > 1024`. So `m=7` is exactly where it kicks in, and for the `B` I care about it holds comfortably. Now the product itself: its base-2 logarithm is about `floor(log B) log n + sum_{i<=(log n)^2} log(n^i-1)`, which is dominated by `sum_{i} i log n approx (log n)^2 cdot (log n)^2 cdot log n / 2 = (log n)^5 / 2`, well under `B = (log n)^5`. I check this for a concrete value: at `n=1000003`, `log2(product) approx 1.58 cdot 10^6` while `B approx 3.15 cdot 10^6`, so `product < 2^B` with room to spare, and the same holds at `n=31` and `n=561`. So the lcm of the first `B` integers exceeds the product, some `r <= B` does not divide it, and that `r <= max(3, ceil((log n)^5))` has `ord_r(n) > (log n)^2`.

Before trusting the quotient, I strip away the obvious composite cases. If `n=a^b`, then a later proof may only be able to force `n` to be a prime power, so I must remove perfect powers at the start. If some `a <= r` has `1 < gcd(a,n) < n`, I already have a factor. If `n <= r` and no such factor appears, a composite `n` would have a prime factor at most `sqrt(n) <= r`, so it would have been caught; in that case I can safely accept.

Now suppose all these gates pass and the compressed identity holds for shifts `a=1,...,ell`, where I will choose `ell` soon. I want to know whether this forces `n` to be prime. Since `ord_r(n)>1`, at least one prime divisor of `n` has order greater than one modulo `r`; I choose such a divisor and call it `p`. The gcd gate and the `n>r` case give `p>r`, and the chosen `r` gives `gcd(n,r)=1`. Reducing the accepted congruences modulo `p` gives, for those checked shifts,

`(X+a)^n equiv X^n+a mod (X^r-1, p)`.

Because `p` itself is prime, it also satisfies

`(X+a)^p equiv X^p+a mod (X^r-1, p)`.

In characteristic `p`, Frobenius lets me peel off a factor of `p` from the exponent. The quotient by `X^r-1` is reduced modulo `p` because `p` does not divide `r`, so the `p`-th power map is injective there. Comparing the `n`-th power identity with the `p`-th power identity then forces `n/p` to behave in the same Frobenius-like way:

`(X+a)^(n/p) equiv X^(n/p)+a mod (X^r-1, p)`.

This suggests naming the behavior. A number `m` is introspective for a polynomial `f` if

`f(X)^m equiv f(X^m) mod (X^r-1, p)`.

The prime `p` and the cofactor `n/p` are introspective for each checked `X+a`; `X` itself is introspective for every exponent, so I get the `a=0` factor for free. The closure properties are exactly what I would need. If `m` and `m'` are introspective for `f`, then

`f(X)^(mm') = (f(X)^m)^(m') = f(X^m)^(m') = f(X^(mm'))`,

with the final reduction valid because `X^r-1` divides `X^(mr)-1`. If one `m` is introspective for both `f` and `g`, then it is introspective for `fg` by multiplying the two identities. Therefore every number of the form `(n/p)^i p^j` is introspective for every product of `X` and the checked linear factors:

`prod_{a=0}^ell (X+a)^{e_a}`.

The set of exponents has a residue shadow modulo `r`. Let `G_1` be the subgroup of `(Z/rZ)^*` generated by residues of `(n/p)^i p^j`; it contains `n=(n/p)p`, so its size `t` is at least `ord_r(n)`, which is larger than `(log n)^2`. The tested linear factors have a polynomial shadow. Take an irreducible factor `h(X)` of the `r`-th cyclotomic polynomial over `F_p`; in the field `F=F_p[X]/(h)`, the image of `X` is a primitive `r`-th root of unity, and `deg h = ord_r(p) > 1` by the way I chose `p`. Because `deg h > 1`, none of `X, X+1, ..., X+ell` is zero in `F`. Let `G_2` be the subgroup of `F^*` generated by `X, X+1, ..., X+ell`.

I need to corner `|G_2|` from below and above and see whether the two bounds can coexist for a composite. The lower bound comes from the many small shifts. Consider two different products of the factors `X+a` whose total degree is less than `t`. If they were equal in `F`, call their difference `Q(Y)`. For each residue class in `G_1`, I choose an introspective representative `m=(n/p)^i p^j`; introspection gives equality after substituting `Y=X^m`, so `Q` has all `t` primitive roots `X^m` as roots. But `deg Q < t`, so `Q` cannot have `t` distinct roots unless it is zero, contradicting that the products are different. Thus all products of total degree less than `t` are distinct in `G_2`.

How many such products are there? They are counted by nonnegative exponent vectors `(e_0,...,e_ell)` with sum less than `t`, so there are `binom(t+ell, t-1)` of them. This is where the number of shifts has to be large enough, so let me set `ell = floor(sqrt(phi(r)) log n)` and check the inequality I actually want, namely `binom(t+ell, t-1) > n^sqrt(t)`. Since `t` divides `phi(r)`, `ell >= floor(sqrt(t) log n)`. I test the chain at `n=1000003`, where the search gives `r=401`, `t = ord_r(n) = 400`, and `ell = floor(sqrt(400) cdot 19.93) = 398`. Then `log2 binom(t+ell, t-1) = log2 binom(798,399) approx 792.9`, while `sqrt(t) log n = 20 cdot 19.93 approx 398.6`. So the count exceeds `2^(floor(sqrt(t) log n)+1) = 2^399`, and `2^399 > n^sqrt(t)` since `sqrt(t) log n approx 398.6 < 399`. Both steps hold numerically, so `G_2` has more than `n^sqrt(t)` elements.

The upper bound should appear if `n` is not a power of the chosen prime `p`. Look at the grid of exponents

`(n/p)^i p^j`, with `0 <= i,j <= floor(sqrt(t))`.

If `n` has a prime factor other than `p`, these integers are all distinct, because that other prime records the value of `i`, and then the power of `p` records `j`. There are `(floor(sqrt(t))+1)^2 > t` of them, but only `t` residues in `G_1`, so two of them, say `m_1 > m_2`, are congruent modulo `r`. Then `X^(m_1)=X^(m_2)` in the quotient. For every element `f` of `G_2`, introspection gives `f(X)^(m_1)=f(X)^(m_2)`, so every element of `G_2` is a root of `Y^(m_1)-Y^(m_2)` in `F`. The degree is at most `n^sqrt(t)`, so `|G_2| <= n^sqrt(t)`.

Now the two bounds collide. The lower bound says `|G_2| > n^sqrt(t)`, and the upper bound says `|G_2| <= n^sqrt(t)`. They cannot both hold, so the assumption that produced the upper bound must be false: `n` is a power of `p`. That is exactly why the first line rejects perfect powers. Once perfect powers are gone, the only remaining prime power is `n=p`. So if all compressed identities pass, `n` is prime.

I should make sure the two ends of the algorithm match this argument, not just the middle. The prime case: I run the compressed test on `n=31`. The search returns `r=29` with `ord_r(31)=28 > (log 31)^2 approx 24.5`, and computing `(X+a)^31 - (X^31+a)` in `(Z/31Z)[X]/(X^29-1)` for `a=1,...,5` returns the zero polynomial every time. Good. The composite case: on the Carmichael number `561`, the search returns `r=89`, and the compressed identity already fails at `a=1`, so it is rejected before any base-by-base Fermat illusion can help it. So primes pass and at least this hard composite is caught, consistent with the proof.

The final algorithm is now the conclusion of all of this. Remove perfect powers, find a small `r` with `ord_r(n) > (log n)^2`, remove small gcd factors up to `r`, accept the tiny case `n <= r`, and then test the compressed polynomial identity for `a=1,...,floor(sqrt(phi(r))log n)`. The quotient by `X^r-1` turns the infinite-looking binomial identity into a polynomial-time computation; the order condition makes the quotient see many distinct root positions; the small shifts generate enough distinct linear-factor products that the lower bound on `|G_2|` beats the collision upper bound; and the perfect-power gate closes the only escape left by that collision.
