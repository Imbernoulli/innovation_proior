Let me start from what actually breaks. I have a latent world model with no decoder: an encoder `h` sends an observation `s` to a latent `z`, a dynamics net `d` predicts the next latent `z' = d(z,a)`, and reward and value heads read off `z`. The only thing tying `z` to anything is a self-predictive consistency loss — I push the predicted next latent toward the encoder's own embedding of the next observation, `||d(z_t,a_t) − sg(h(s_{t+1}))||²`, with a stop-gradient on the target — plus reward regression and a bootstrapped value loss whose target is `q = r + γ Q̄(z', p(z'))`. Nothing reconstructs the observation. So `z` is defined purely by these self-referential objectives, and that is exactly where it hurts: on harder tasks the gradient norm climbs and climbs and training diverges. I need to understand *why* before I reach for a fix, because if I just clip gradients I'm treating a symptom.

Look at the consistency loss as an optimizer would. It wants `d(z,a)` to equal `sg(h(s'))`. There's a cheap, useless way to make a squared-error-in-feature-space loss look better that has nothing to do with learning dynamics: shrink the whole representation. The opposite problem appears through the bootstrapped value loop, because the target is itself a function of latents the same network produces — `Q̄(z', ...)` reads the very `z'` the dynamics emits — so latents feed value targets, value gradients feed back into `h` and `d`, which reshape the latents. That's a closed loop, and a loop with an unconstrained magnitude variable has a runaway direction. So this isn't mysterious: an unconstrained latent under a self-predictive plus bootstrapped objective has a degree of freedom — the overall scale of `z` — that the losses don't pin down, and the feedback loop can drive it. The exploding gradients are the magnitude becoming uncontrolled. The fix has to *remove that degree of freedom*: bound the representation.

What do I have on the shelf for "bound a representation"? My layers already have LayerNorm after each linear map. So why doesn't that already save me? Because LayerNorm normalizes the *intermediate* pre-activations of each layer — it fixes the per-layer mean and variance flowing *through* the MLP — but the network's *output*, the final `z`, is produced by a last linear map after the last LayerNorm, and that linear map can have arbitrarily large weights. LayerNorm controls the river, not the mouth. The thing I actually consume — `z`, fed to dynamics, reward, value — is unbounded. So I need a constraint on the *final* layer's output, a final activation that defines the geometry of `z` itself.

Simplest idea: just L2-normalize `z`, project it to the unit sphere, `z ← z/||z||₂`. That kills the magnitude degree of freedom dead — `||z||₂ = 1` by construction, no runaway scale, no per-task constant to tune. Tempting. But sit with it. The sphere bounds magnitude and *nothing else*. Every point on the sphere is equally available; there's no pressure toward any particular kind of representation, and one global division couples all `d` coordinates through a single scalar `||z||`. I'd be paying with a representation that's bounded but otherwise shapeless and fully dense — every coordinate generically nonzero. For an RL latent that then has to support stable bootstrapped value learning across many tasks, a dense bounded blob feels weak; I have a hunch I want more *structure* than "lives on a sphere," but let me not commit yet — let me see what structure buys me.

Or squash: tanh each coordinate so every entry sits in `(−1,1)`. Bounded per-coordinate, fine. But there's no competition *between* coordinates — each one saturates on its own — so again the representation is dense, and worse, saturated tanh units pass almost no gradient, so I'd be trading exploding gradients for dead ones in the very units that hit the rails. That's not the trade I want.

Let me think about what a *good* latent geometry for this model would be, and let the constraint fall out of that rather than picking a normalizer off a list. There's an old intuition I trust: sparse, overcomplete codes — more components than the input dimension, most of them inactive — are stable under noise and tend to be the representations that downstream linear readouts like. My value and reward heads *are* essentially readouts on `z`. So I'd like `z` to be bounded *and* sparse. How do I get sparsity as a *bias* rather than a hard constraint, without an L1 penalty I'd have to weight per task?

Here's where the discrete-codes idea is worth chewing on. VQ-VAE represents a latent as a *vector of categoricals*: split the latent into `L` groups and replace each group by a one-hot code from a learned codebook. That's bounded (one-hots have norm 1), it's structured, it's maximally sparse within each group — exactly one active entry per group. And `L` independent groups is an *overcomplete* code: with `V` choices per group and `L` groups, I can represent up to `V^L` distinct states, roughly `L·log₂V` bits, so I get expressivity *and* boundedness at once. That's the shape I want. But the one-hot is an `argmax` — non-differentiable. To train through it VQ-VAE needs a straight-through estimator and a commitment loss, and it drags codebook collapse and dead codes along with it. For a recurrent world model where gradients have to flow cleanly through `d` step after step, bolting on a straight-through estimator and a codebook is a lot of fragile machinery, and the hard discretization can choke the information `z` carries. So the vector-of-categoricals *geometry* is right, but the *hard* version is the wrong tool.

So the question sharpens: can I keep the group structure — `L` groups, each a bounded simplex, biased toward one active entry — but make it *soft* and differentiable, no codebook, no straight-through? What's the smooth relaxation of "pick one entry per group"? The relaxation of `argmax` is `softmax`. If I take each group of `V` raw values and apply a softmax *within the group*, I get a vector of `V` nonnegatives that sums to 1 — a point on the `(V−1)`-simplex. Do that for all `L` groups and concatenate. That's it: reshape `z` from `(...,d)` to `(...,L,V)`, softmax over the last axis, reshape back.

  z reshaped to (..., L, V)
  g_i = softmax(z_i / τ)          # over the V axis: g_i lives on the (V−1)-simplex
  z° = concat(g_1, ..., g_L)

Let me check it actually solves the original problem — does it bound the representation? Within a group, softmax gives `g_i ∈ (0,1)^V` with `Σ_j g_{ij} = 1`. So `||g_i||_1 = 1` exactly, and since each entry is in `[0,1]`, `||g_i||_2² = Σ_j g_{ij}² ≤ Σ_j g_{ij} = 1`, so `||g_i||_2 ≤ 1`. Across all `L` groups, `||z°||_1 = Σ_i ||g_i||_1 = L` exactly, and `||z°||_2² = Σ_i ||g_i||_2² ≤ L`, so `||z°||_2 ≤ √L`. A hard, parameter-free upper bound on the latent norm — and it depends on *nothing about the input or the task*, only on `L`. That is precisely the degree of freedom I diagnosed as the source of the blowup: gone. No scalar constant to tune per task; the bound is structural. The runaway loop has nothing left to run away with.

Now, why *groups* and not one big softmax over all `d` entries? This matters, so let me actually compare. A single softmax over `d` entries forces `Σ z° = 1` over the *whole* vector. Then on average each coordinate is about `1/d` — vanishingly small for `d` in the hundreds — and only one coordinate can be appreciably active at a time; the representation can pick out essentially one direction and carries on the order of `log₂d` bits. That's a brutal bottleneck for a world-model latent that has to encode the full state for dynamics, reward, and value. Splitting into `L` independent simplices fixes exactly this: each group can independently choose which of its `V` entries to activate, so I get `L` near-independent decisions, an overcomplete code with up to `V^L` configurations and roughly `L·log₂V` bits, while *every group is still individually bounded*. Boundedness and expressivity stop fighting each other. That's the whole reason the group structure isn't optional — it's what lets me bound the latent without crushing it to a single direction.

And the sparsity I wanted — do I get it for free, as a *bias* not a constraint? Yes, and it's worth seeing the mechanism. Softmax inside a group is a zero-sum competition: the outputs sum to 1, all nonnegative, so to raise one component by `α` the others must collectively give up `α`. To drive the consistency and value losses down, the network can't make everything large; it has to *prioritize* — push mass onto a few entries per group at the expense of the rest. So the learned `g_i` drift toward approximately one-hot, sparse-but-soft, without my ever writing an L1 term or a hard `argmax`. I can even quantify how sparse a group is by its entropy `H(g_i) = −Σ_j g_{ij} log g_{ij} ∈ [0, ln V]`: `0` means one-hot (maximally sparse), `ln V` means uniform (maximally dense). The representation sits somewhere in between, biased toward the sparse end by the competition. That's the sparse overcomplete code I was after — emergent from the geometry, not imposed.

There's a temperature knob hiding in the softmax: `g_i = softmax(z_i/τ)`. Let me make sure I understand what it does and pick a value on purpose rather than by default. As `τ → 0`, the softmax sharpens to a hard `argmax` — each group becomes a one-hot, and I recover exactly the discrete vector-of-categoricals (the VQ-VAE limit), but now reached *continuously* and differentiably.
As `τ → ∞`, every logit washes out and each group goes uniform, `g_{ij} → 1/V` — a constant vector that carries no information and blocks gradient flow through `z`. So `τ` is a dial between "hard discrete code" and "trivial uniform code," and the useful regime is in between. Very small `τ` is risky in *this* model for the same reason vanishing gradients are: near one-hot, the softmax Jacobian collapses and almost no gradient reaches the encoder/dynamics — and gradient flow through `d` over a multi-step rollout is the whole ballgame. I don't have a reason to push `τ` toward either extreme, and I don't want to add a per-task tuned constant — the point of this exercise was a representation that's stable *without* per-task tuning. So I take the neutral choice, `τ = 1`, a plain softmax: it keeps a healthy sparsity bias while leaving gradients flowing. The dial exists if I ever need it, but unit temperature is the honest default.

What about the group size `V`? `V` trades off two things. Larger `V` means each group has more entries competing, so the one-active-out-of-`V` bias is a *stronger* sparsity pressure and each group carries more bits (`log₂V`), but each simplex is also a softmax over more logits, slightly more compute and slightly flatter gradients per entry. Smaller `V` is cheaper and keeps gradients sharp but gives a weaker, lower-capacity code per group. With a latent dimension in the hundreds I want enough groups `L = d/V` to keep the overcomplete structure rich, and a small `V` keeps each simplex cheap and gradient-friendly. `V = 8` is the sweet spot I'll take — eight-way competition per group, `L = d/8` groups; with `d = 512` that's 64 simplices, plenty of overcomplete capacity. It also makes the reshape clean: `d` just has to be divisible by `V`.

One more decision: *where* do I apply this? The consistency loss compares `d(z,a)` against `sg(h(s'))`. For that comparison to be meaningful, the prediction and the target must live in the *same* space — same geometry, same bound. So the final activation of the dynamics net `d` and the final activation of every encoder path `h` must both be this simplicial normalization. If I only normalized the encoder, the dynamics could emit an unbounded `z'` and I'd reintroduce the exact magnitude drift I just removed, plus a geometry mismatch between predictor and target. So it goes on the last layer of *both* `h` and `d`. Reward and value heads then always consume a bounded, sparse, simplex-valued `z`, which keeps their input well-scaled and their Jacobians tame — smoother gradients, and — the original goal — no per-task latent-scale constant to keep the representation in a usable range.

Let me write the layer. It's just a reshape, a softmax over the last axis, and a reshape back — shape-preserving, so it drops straight into the `mlp(..., act=...)` slot as the final activation of encoder and dynamics.

```python
import torch.nn as nn
import torch.nn.functional as F

class SimNorm(nn.Module):
    """Simplicial normalization: partition z into groups of size V and softmax
    each group, so every group lands on a (V-1)-simplex. Bounds ||z|| and biases
    the representation toward group sparsity, with no per-task constant."""
    def __init__(self, cfg):
        super().__init__()
        self.dim = cfg.simnorm_dim          # V, the simplex (group) size; default 8

    def forward(self, x):
        shp = x.shape
        x = x.view(*shp[:-1], -1, self.dim) # (..., d) -> (..., L, V), L = d / V
        x = F.softmax(x, dim=-1)            # softmax within each group => point on a simplex
        return x.view(*shp)                 # (..., L, V) -> (..., d), shape preserved

    def __repr__(self):
        return f"SimNorm(dim={self.dim})"
```

And it slots in as the final activation of both networks that emit a latent:

out["state"] = mlp(cfg.obs_shape["state"][0] + cfg.task_dim,
                   max(cfg.num_enc_layers - 1, 1) * [cfg.enc_dim],
