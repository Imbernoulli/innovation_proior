The cost of training a decoder-only transformer language model is set almost entirely by one number, the length $L$ of the input subsequence we train on. Self-attention compares every token to every other token, so compute and memory both scale like $L^2$, yet longer inputs pay off — a token predicted with a thousand tokens of left context has a lower loss than one predicted with a hundred. So we are perpetually tempted to raise $L$ and perpetually punished for it on the clock. What we actually want is to train on short subsequences, which is cheap, and then at inference feed sequences *longer* than $L$ and have the model keep working — ideally do better, because more context should help. Recurrent models were routinely trained short and run long and everyone assumed that generalization came free; the hope was the transformer would inherit it. So the question is sharp: trained at $L$ and evaluated at $L_\text{valid} > L$, does perplexity hold? It does not, and the failure is worth pinning down precisely, because every function in a transformer layer — the embedding lookup, the feedforward sublayer, the softmax classifier, the attention parameters — maps $n$ vectors to $n$ vectors for *any* $n$ and cares nothing about how many tokens arrive. The architecture can physically accept a longer input. So the break at $L_\text{valid} > L$ cannot be the layers; it has to be the one thing we inject that depends on absolute position — the position signal added at the bottom.

Tracing it through the existing options confirms this and tells us what property to keep. Learned absolute embeddings store one vector per slot $0..L-1$ and have *no defined behavior* past $L$ at all — dead on arrival. Sinusoidal embeddings encode each position as $\sin\!\big(p/10000^{2k/d}\big)$ and its cosine counterparts across a geometric range of frequencies, so they are a function defined at every integer position and ought to extrapolate; empirically they barely do, improving for only the first $\sim$20–50 extra tokens and then degrading. The reason is that although $\sin(p/\cdot)$ is defined at $p = L+500$, during training the model only ever saw the particular phase combinations occurring over $[0,L)$; past $L$ the joint pattern across dimensions is a configuration it never encountered — out of distribution — and the model's learned interpretation only covers $[0,L)$. Rotary embeddings rotate queries and keys by an angle proportional to absolute position so that a query at $i$ dotted with a key at $j$ depends only on $i-j$; they extrapolate noticeably better but the per-layer rotations make training and inference slower than the sinusoidal baseline. The learned, bucketed relative-position bias of Shaw et al. as used in T5 adds a learned scalar to each score depending only on distance, bucketed so a never-seen large distance falls into an already-learned far bucket; under identical-everything-else it is the best extrapolator of all, which *proves* that changing the position method is the lever — but computing and gathering the bucketed bias is at least roughly twice as slow as sinusoidal and adds learned parameters, so the train-short-and-cheap payoff evaporates. Reading the ladder, the two methods that extrapolate share a property the two that do not lack: they put position *only* into the query–key scores, never into the values, so the residual stream carries no explicit absolute position. That property is what I keep; the learned, bucketed cost is what I discard.

I propose ALiBi — Attention with Linear Biases. Use no position embeddings anywhere: the token embeddings enter raw, and the only thing telling the model about order is a static, non-learned bias added to the attention scores after the query–key dot product (and after the usual $\sqrt{d_k}$ scaling) that is *linear in the query–key distance* with a fixed per-head slope. For query $i$ attending over keys $j = 1..i$, with relative distance $i-j \ge 0$, the attention computation becomes
$$\operatorname{softmax}\!\Big(\tfrac{q_i K^\top}{\sqrt{d_k}} + m\cdot[-(i-1),\,\dots,\,-2,\,-1,\,0]\Big),$$
so the nearest key (distance 0) gets penalty 0, a key one step back gets $-m$, two steps back $-2m$, and so on. The slope $m>0$ is fixed per head. The reason linearity is the load-bearing choice is exactly the failure I diagnosed: the disease with sinusoidal was that the signal past $L$ is a novel out-of-distribution pattern, so the cure is a position signal whose value at distance $d$ is the obvious continuation of its values at small $d$, with no learned dial that could be miscalibrated out of range. The simplest such function of distance is a straight line, and at distance 5000 the penalty is just $5000\,m$ — the same line, not a new absolute-position vector. The direction is a deliberate recency bias: distant tokens matter less, all else equal, and crucially the preference has the *same shape* at every distance, so distance 5000 is "distance 4 but more so," never a regime the model conceptually hasn't seen. By construction the bias touches only the scores, so the attention output stays a position-free weighted average of values, and it is added at every attention sublayer, so position reasserts itself at every layer — the same two structural properties that the extrapolators had, now obtained parameter-free with a single subtraction.

The slopes carry the rest of the design. One slope for all heads would force every head to the same recency preference, which is clearly wrong — some heads should look locally, some should keep a long view — so $m$ is per-head. A head with large $m$ is penalized hard for looking far and becomes local; a head with tiny $m$ barely penalizes distance and stays long-range, and the long-range heads are the precious ones, so I want more resolution near zero. A geometric sequence delivers exactly that, since equal ratios mean equal log-spacing, which packs points densely near the small end. For the 8-head case take $\tfrac12,\tfrac14,\tfrac18,\dots,\tfrac1{256}$, i.e. $2^{-1},\dots,2^{-8}$, start $\tfrac12$ and ratio $\tfrac12$, cleanly inside $(0,1)$ and densest near zero. Generalizing to $n$ heads while keeping the same character means start $=$ ratio, geometric, spanning down to about $2^{-8}$ at the small end, so I want $r^n = 2^{-8}$, giving $r = 2^{-8/n}$ and slopes $2^{-8/n}, 2^{-16/n}, \dots, 2^{-8}$; at $n=8$ this reduces to $\tfrac12,\dots,\tfrac1{256}$ as designed, and at $n=16$ it gives start and ratio $1/\sqrt{2}$, a half-step refinement of the 8-head grid that keeps the integer powers and adds the intervening half-integer ones. The implementation writes $2^{-8/n}$ in the convenient power-of-two form $2^{-2^{-(\log_2 n - 3)}}$. For a genuinely non-power-of-2 $n$ the clean formula doesn't apply, so I take the closest-power-of-2 set below $n$ and borrow the slopes I'm short from the next power of two's set by taking every other one — a minor hack that rarely matters, since the models I train use power-of-two head counts. The slopes are fixed before training rather than learned: a learned slope would be tuned only to the distances seen in $[0,L)$, with no training signal about the far distances that appear only at inference, so it could overfit the in-range recency statistics and generalize *worse* out of range — and learning them costs speed, slowing training by about 3% for no extrapolation gain. One subtlety: the bias is **not** scaled by $\sqrt{d_k}$. That scaling exists to tame a *random* quantity, the dot product of two learned vectors; my bias is a deterministic penalty I choose directly in score-space, so dividing it by $\sqrt{d_k}$ would merely rescale slopes I already get to choose and pointlessly couple them to the head dimension.

The implementation is pleasingly free. A causal mask — an upper-triangular matrix of $-\infty$ — is already added to the scores before the softmax, and my linear bias is *also* an additive pre-softmax term, so I fold it into that same mask: for head $h$ with slope $m_h$, the matrix has entry $-m_h(i-j)$ at $(i,j)$ for $j \le i$ and $-\infty$ for $j > i$, built once and reused, adding no operation and so no runtime penalty. There is a further trick that the linearity unlocks. The honest finite row for query $i$ is $m_h[-(i-1),\dots,-1,0]$, a staircase whose zero sits on the diagonal; in zero-based indices the finite entry is $-m_h(i-j) = m_h j - m_h i$. Since each row is fed through a softmax and $\operatorname{softmax}(x+c)=\operatorname{softmax}(x)$, and the honest row differs from the simple broadcast row $m_h[0,1,\dots,L-1]$ only by the per-row constant $-m_h i$ within the unmasked part $j \le i$, the simple matrix $m_h[0,1,\dots,L-1]$ — identical across rows, then masked by the causal $-\infty$ — produces the *exact same attention weights*. This works precisely because the bias is linear; a nonlinear penalty would not decompose into a per-row constant. So I build one cheap broadcast row per head. The mask grows from $L\times L$ to $H\times L\times L$, a small absolute increase (up to about 100 MB, $\sim$0–0.7% net versus sinusoidal at the same $L$), and since I can now train at a smaller $L$, the quadratic $L^2$ savings on activations dwarf this $H\times$ mask overhead. Finally, to be honest about what the longer-input gains are: under stride-1 sliding-window inference, which removes the early token curse entirely by giving every prediction a full $L$-token context, ALiBi's perplexity stays *flat* as $L_\text{valid}$ grows while sinusoidal explodes. So the nonoverlapping-inference gains from longer inputs come from reducing the early token curse — fewer context-starved predictions near segment starts — rather than from demonstrated use of context beyond $L_\text{train}$. That is a real limitation to name, but ALiBi still matches or beats the alternatives at $L_\text{valid} = L_\text{train}$ while being simpler and parameter-free, and beats the only other cheap option at $L_\text{valid} > L_\text{train}$, all with no embeddings, no parameters, and no added runtime.

```python
import math
import torch
import torch.nn as nn
import torch.nn.functional as F


def alibi_slopes(n_heads):
    # Geometric slopes in (0,1), densest near 0. Power-of-2: start = ratio = 2^(-8/n).
    def pow2(n):
        start = 2.0 ** (-(2.0 ** -(math.log2(n) - 3.0)))   # == 2^(-8/n)
        return [start ** (i + 1) for i in range(n)]
    if math.log2(n_heads).is_integer():
        return torch.tensor(pow2(n_heads))
    closest = 2 ** math.floor(math.log2(n_heads))
    extra = pow2(2 * closest)[0::2][: n_heads - closest]    # every-other extra from next set
    return torch.tensor(pow2(closest) + extra)


class ALiBiAttention(nn.Module):
    """Causal multi-head attention with a linear per-head distance bias.
    No position embeddings; bias is added to scores before softmax."""

    def __init__(self, d_model, n_heads):
        super().__init__()
        self.n_heads, self.d_k = n_heads, d_model // n_heads
        self.qkv = nn.Linear(d_model, 3 * d_model)
        self.out = nn.Linear(d_model, d_model)
        self.register_buffer("slopes", alibi_slopes(n_heads), persistent=False)  # [H]

    def _bias(self, T, device, dtype):
        pos = torch.arange(T, device=device, dtype=dtype)                         # [T]
        bias = self.slopes.to(device, dtype)[:, None, None] * pos[None, None, :]  # [H,1,T]
        causal = torch.triu(torch.full((T, T), float("-inf"), device=device, dtype=dtype), 1)
        return (bias + causal)[None, ...]                                         # [1,H,T,T]

    def forward(self, x):                       # x: [B, T, d_model], no position added
        B, T, _ = x.shape
        q, k, v = self.qkv(x).chunk(3, dim=-1)
        q = q.view(B, T, self.n_heads, self.d_k).transpose(1, 2)
        k = k.view(B, T, self.n_heads, self.d_k).transpose(1, 2)
        v = v.view(B, T, self.n_heads, self.d_k).transpose(1, 2)
        scores = (q @ k.transpose(-2, -1)) / (self.d_k ** 0.5)      # sqrt(d_k) scaling
        scores = scores + self._bias(T, x.device, scores.dtype)     # bias NOT scaled by sqrt(d_k)
        attn = F.softmax(scores, dim=-1)
        return self.out((attn @ v).transpose(1, 2).reshape(B, T, -1))
```
