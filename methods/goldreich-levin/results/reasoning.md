I need a bit that is not already tied to a special algebraic function. A fixed coordinate of the input is hopeless as a universal choice, because a one-way function can simply reveal that coordinate and still be hard to invert. I need to add something to the input so that the bit is easy to compute with the preimage, but any noticeable ability to predict it from the image gives me more than one isolated fact.

I let a fresh random mask choose a parity of the hidden string. For a secret string \(x\) and a mask \(r\), I look at \(\langle x,r\rangle \bmod 2\). For one mask this is only one bit, but as the mask varies it is the whole Hadamard codeword of \(x\). This is the point: a predictor that works on random masks is not just predicting unrelated bits; it is correlated with a structured encoding of the entire secret.

Suppose I have an efficient predictor \(G\) that, on input \(f(x)\) and \(r\), guesses \(\langle x,r\rangle\) with probability \(1/2+\epsilon\). The advantage is over random \(x\) and random \(r\), so I first average. There is a noticeable set of inputs \(x\) for which, after I fix \(y=f(x)\), the function \(r \mapsto G(y,r)\) has noticeable correlation with \(r \mapsto \langle x,r\rangle\). I can focus on such an \(x\), because recovering \(x\) on a noticeable set already contradicts one-wayness.

The tempting recovery method is too brittle. To get \(x_i\), I can use the identity \(\langle x,r\rangle \oplus \langle x,r\oplus e_i\rangle = x_i\). If I ask the predictor on both \(r\) and \(r\oplus e_i\), then xor the two answers, I get a vote for \(x_i\). But this uses the weak predictor twice. When the original advantage is only barely above one half, doubling the error can erase the signal. I need a way to know one side of the identity without asking the predictor for it.

I can afford to guess a logarithmic number of labels. I decide how many samples \(m\) I need as a polynomial in the input length and the inverse advantage, take \(\ell=\lceil \log_2(m+1)\rceil\), sample masks \(s_1,\ldots,s_\ell\), and guess the bits \(\langle x,s_j\rangle\). One full set of guesses is correct with inverse-polynomial probability, and I can repeat or enumerate. Now every nonempty subset \(J\) gives me a derived mask \(r_J=\bigoplus_{j\in J}s_j\), and linearity gives the derived label \(\langle x,r_J\rangle=\bigoplus_{j\in J}\langle x,s_j\rangle\). If my seed-label guesses are right, I know \(m\) true labels without querying the predictor.

These derived masks are still random enough. Each \(r_J\) is uniform, and the family is pairwise independent. For coordinate \(i\), I form votes \(\alpha_J \oplus G(y,r_J\oplus e_i)\), where \(\alpha_J\) is my derived guess for \(\langle x,r_J\rangle\). When the seed guesses are correct, the vote equals \(x_i\) exactly when \(G\) is correct on the shifted mask, because \(\langle x,r_J\oplus e_i\rangle=\langle x,r_J\rangle\oplus x_i\). With enough subset masks, Chebyshev-style concentration and a union bound over coordinates make the majority recover the whole string with constant probability.

Doing this for every coordinate gives me a candidate string. The correct seed-label assignment gives the real \(x\) with noticeable probability; enumerating the assignments, or repeating the randomized choice, gives only a polynomial-size list of candidates. I do not need to know in advance which candidate is right, because I can test them by applying \(f\) and checking which one maps to \(y\).

So the bit is hard-core because prediction becomes inversion. A weak predictor of the random inner product gives me noisy access to the Hadamard codeword of the hidden input. The randomized correlation search turns that weak access into a short list containing the hidden string, and candidate testing converts the list into an inverter. Since the original function is one-way, the predictor cannot exist.
