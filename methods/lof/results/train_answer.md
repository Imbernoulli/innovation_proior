I want to hand each point in an unlabeled, multidimensional dataset a number that says how exceptional it is — how much it looks like it was generated by a different mechanism than the bulk — using only a distance function, no labels, no assumed parametric distribution, and at a scale where dimension is well above two or three. The clean operational versions I have inherited all take a fundamentally *global* view of distance. The distance-based definition calls $O$ a $\mathrm{DB}(p, D)$-outlier when at least a fraction $p$ of all objects lie farther than $D$ from it; it needs only a metric, so unlike the univariate discordancy tests it is not trapped in one dimension or one assumed distribution, and unlike depth-based methods it builds no convex hull, so it survives past $k \geq 3$. It even subsumes the classical tests — a $3\sigma$ Gaussian outlier is exactly a $\mathrm{DB}(0.9988, 0.13\sigma)$-outlier, a Poisson(3) tail point a $\mathrm{DB}(0.9892, 1)$-outlier. Ramaswamy, Rastogi and Shim then remove a parameter: rank every point by $D^k(p)$, the distance to its $k$-th nearest neighbor, and take the top $n$, which supplies both a ranking and a more intuitive "how many outliers do I want" knob.

The trouble is that real data is not homogeneous, and a single global distance scale cannot represent "exceptional relative to the local surroundings." Picture a loose, low-density cluster $C_1$ of a few hundred points beside a tight, high-density cluster $C_2$ of a hundred, with two strays: $o_1$ far from everything and $o_2$ sitting just off the edge of $C_2$. Both are outliers — $o_2$ especially, because relative to how tightly $C_2$ packs it is conspicuously detached. But $D^k(o_2)$, roughly $o_2$'s distance to $C_2$, can be *as small as or smaller than* $D^k(q)$ for an ordinary point $q$ deep inside the loose $C_1$, simply because $C_1$ is loose. So the global ranking by $D^k$ places ordinary members of $C_1$ at or above the genuine anomaly $o_2$, and no single $D$ can do better — small enough to catch $o_2$ condemns half of $C_1$, large enough to spare $C_1$ lets $o_2$ pass. Every one of these methods compares each point to one number applied to the entire dataset, and their verdicts are binary besides, throwing away exactly the gradation needed to separate the two confused cases. What is missing is a verdict that is *local* and *graded*, and whose value *means the same thing everywhere*, so that interior points of any cluster — dense or sparse — score the same "ordinary" value and can be compared across regions.

I propose the Local Outlier Factor, LOF: a per-point real-valued degree of being an outlier, computed as the ratio of the density of a point's neighbors to the point's own density. Fix a neighborhood size $\mathrm{MinPts}$ (the $k$ from $D^k$, the $\texttt{n\_neighbors}$ in code), reusing the one good idea from density-based clustering — define a neighborhood by a *count* of objects rather than a fixed radius, so I never commit to a volume in advance. The $k\text{-distance}(p)$ is the distance from $p$ to its $\mathrm{MinPts}$-th nearest neighbor, and the neighborhood $N(p)$ is everything within that distance; under ties $|N(p)|$ can exceed $\mathrm{MinPts}$, so I always write $|N(p)|$ rather than assume equality. The cheapest distribution-free density proxy is already in hand: if a point's neighbors are close it is dense, if far it is sparse. But the obvious estimate — invert the average of the raw distances $d(p,o)$ — is unstable, because if $p$ falls very near a single neighbor $o$, the raw $d(p,o)$ can shrink below the typical spacing inside $o$'s own cluster and make $p$ look absurdly dense; that jitter would then swing every ratio I build on top. So I floor it. The reachability distance never lets the distance to $o$ drop below $o$'s own $k$-distance,

$$\text{reach-dist}(p, o) = \max\{\,k\text{-distance}(o),\; d(p, o)\,\}.$$

Every $p$ comfortably inside $o$'s neighborhood collapses to the single stable value $k\text{-distance}(o)$, a smoothed density-of-$o$ quantity, and raising $\mathrm{MinPts}$ widens the average and strengthens the smoothing. The asymmetry is deliberate and correct — the floor is $o$'s $k$-distance, not $p$'s, because what I am estimating stably is the density *around $o$*, the point whose neighborhood I am probing; $\text{reach-dist}(p,o) \neq \text{reach-dist}(o,p)$ in general, and that is fine, since they estimate different local densities. From this I form the local reachability density as the inverse of the *average* reach-distance over $N(p)$,

$$\mathrm{lrd}(p) = \frac{1}{\bigl(\sum_{o \in N(p)} \text{reach-dist}(p, o)\bigr) / |N(p)|}.$$

The average, not the bare sum, is essential: under ties $|N(p)|$ varies between points, and a sum would make a point with more neighbors look artificially less dense; dividing by $|N(p)|$ cancels that, so $\mathrm{lrd}$ is genuinely "inverse of a typical distance" and comparable from point to point, and the inverse makes larger mean denser.

But $\mathrm{lrd}$ is still an *absolute* density — large in $C_2$, small in $C_1$ — so on its own it is $D^k$ in disguise and inherits the same global-scale disease. The locality has to come from a comparison: is $p$'s surroundings much emptier than the surroundings of the points around it? That is a ratio, the local outlier factor,

$$\mathrm{LOF}(p) = \frac{\sum_{o \in N(p)} \mathrm{lrd}(o)/\mathrm{lrd}(p)}{|N(p)|} = \frac{\text{average } \mathrm{lrd} \text{ over } p\text{'s neighbors}}{\mathrm{lrd}(p)}.$$

The absolute scale of density cancels in the ratio. Deep inside the dense $C_2$, both $\mathrm{lrd}(p)$ and the neighbors' $\mathrm{lrd}(o)$ are large and roughly equal, so the ratio is $\approx 1$; deep inside the loose $C_1$, both are small and roughly equal, ratio again $\approx 1$ — so a point well inside *any* cluster scores $\approx 1$ regardless of that cluster's absolute density, exactly the "means the same thing everywhere" property I insisted on. For $o_2$, which sits in a sparse pocket (small $\mathrm{lrd}$) but whose neighbors are the dense $C_2$ points (large $\mathrm{lrd}$), each ratio $\mathrm{lrd}(o)/\mathrm{lrd}(p)$ is much greater than 1, averaging to a clearly-greater-than-1 LOF; meanwhile a loose-cluster point has small $\mathrm{lrd}$ *and* small-$\mathrm{lrd}$ neighbors, ratio $\approx 1$, and is not flagged even though its absolute $D^k$ is large. The ratio has untangled exactly the two cases the global ranking confused. So $\mathrm{LOF} \approx 1$ is an inlier as dense as its neighbors, $\mathrm{LOF} < 1$ is denser still (solidly inlier), and $\mathrm{LOF} > 1$ marks emptier surroundings than the neighbors — an outlier, the larger the stronger.

The "$\approx 1$ means ordinary" claim deserves a proof rather than an appeal to cancellation. Take a cluster $C$, let $\text{reach-dist-min}$ and $\text{reach-dist-max}$ be the extreme within-cluster reach-distances, and set $\varepsilon = \text{reach-dist-max}/\text{reach-dist-min} - 1$, a measure of how non-uniform $C$ is. For a point $p$ deep enough that all of its neighbors and all of *their* neighbors lie in $C$, every reach-distance entering $\mathrm{lrd}(p)$ lies in $[\text{reach-dist-min}, \text{reach-dist-max}]$, so $1/\text{reach-dist-max} \leq \mathrm{lrd}(p) \leq 1/\text{reach-dist-min}$, and the identical bound holds for each neighbor; the ratios $\mathrm{lrd}(o)/\mathrm{lrd}(p)$ are therefore squeezed into $[\text{reach-dist-min}/\text{reach-dist-max},\ \text{reach-dist-max}/\text{reach-dist-min}]$, and averaging keeps $\mathrm{LOF}(p)$ there, giving

$$\frac{1}{1+\varepsilon} \leq \mathrm{LOF}(p) \leq 1+\varepsilon.$$

A deep point's LOF is pinned within a factor $(1+\varepsilon)$ of 1, tighter the tighter the cluster. The points I actually care about are on the fringe, so I want a bound with no clustering assumption. Writing $\text{direct}_{\min/\max}(p)$ for the min/max reach-distance over $N(p)$ and $\text{indirect}_{\min/\max}(p)$ for those over the neighbors' neighborhoods, then $\mathrm{lrd}(p) \leq 1/\text{direct}_{\min}(p)$ and each $\mathrm{lrd}(o) \geq 1/\text{indirect}_{\max}(p)$, and the mirror inequalities, give for *any* $p$

$$\frac{\text{direct}_{\min}(p)}{\text{indirect}_{\max}(p)} \leq \mathrm{LOF}(p) \leq \frac{\text{direct}_{\max}(p)}{\text{indirect}_{\min}(p)}.$$

LOF is governed by the ratio of $p$'s own reach-distances to its neighbors'; a sparse pocket among dense neighbors forces LOF well above 1. If reach-distances fluctuate by a fraction $\mathrm{pct}$ around their means, the relative spread works out to $(\mathrm{LOF}_{\max} - \mathrm{LOF}_{\min})/(\text{direct}/\text{indirect}) = (4\,\mathrm{pct}/100)/(1 - (\mathrm{pct}/100)^2)$, which depends *only* on $\mathrm{pct}$ — the homogeneity of the neighborhood — not on absolute distances, so the score is equally sharp in dense and sparse regions and only goes slack when $p$'s neighbors straddle clusters of differing density. In that regime I can partition $N(p)$ into groups with weights $\xi_i = |C_i|/|N(p)|$ and tighten the bound to $\mathrm{LOF}(p) \geq (\sum_i \xi_i\,\text{direct}^i_{\min})(\sum_i \xi_i/\text{indirect}^i_{\max})$ and its upper mirror, which collapses back to the general bound when $n=1$.

One knob remains, $\mathrm{MinPts}$, and LOF is *not* monotonic in it — even on a single Gaussian blob with no outliers, at $\mathrm{MinPts}=2$ the density estimate carries the full fluctuation of one sample and ordinary points pick up LOFs above 1; raising $\mathrm{MinPts}$ averages that variance away toward 1, but once the neighborhood spans an appreciable fraction of the blob the Gaussian's own density gradient enters and the extreme LOFs drift back up. So I keep $\mathrm{MinPts} \geq 10$, below which the variance of the estimate still flags perfectly ordinary points (this lower bound also reads as the smallest cluster a point can be an outlier relative to), and because no single value is universally right I compute LOF over a range $[\mathrm{MinPtsLB}, \mathrm{MinPtsUB}]$ and score each point by the *maximum* LOF it attains — the maximum reports the scale at which the point is most exceptional, where the minimum would erase its outlying nature at the one neighborhood size it looks innocent and the mean would dilute it; 10–20 is a sensible default range. LOF is a pure ratio of reach-distances, so it is invariant to a global rescaling of all distances, exactly as a "relative to local density" quantity should be, though not to a feature-by-feature rescaling that changes the neighbor graph, so the metric geometry must be fixed before the neighbor search begins. One degenerate case needs care: if a point has $\mathrm{MinPts}$ or more exact duplicates, all its reach-distances can be zero and $\mathrm{lrd} = 1/0$ blows up, so I floor the denominator with a tiny $\varepsilon$ to give a huge-but-finite density, which is the right behavior since such a point genuinely is maximally dense. In code, the $k$-distance of any neighbor $o$ is just the last stored distance in $o$'s materialized neighbor row, making reachability an elementwise maximum, and I store the training score as $\texttt{negative\_outlier\_factor\_} = -\mathrm{LOF}$ so larger stored values mean more normal and inliers sit near $-1$; a wrapper wanting "higher = more anomalous" simply negates it.

```python
import numpy as np
from sklearn.neighbors import NearestNeighbors


class LOF:
    """Local-density ratio with the standard opposite-LOF scoring convention."""

    def __init__(
        self,
        n_neighbors=20,
        *,
        algorithm="auto",
        leaf_size=30,
        metric="minkowski",
        p=2,
        metric_params=None,
        contamination="auto",
        n_jobs=None,
    ):
        self.n_neighbors = n_neighbors      # MinPts: neighborhood size / smoothing
        self.algorithm = algorithm
        self.leaf_size = leaf_size
        self.metric = metric
        self.p = p
        self.metric_params = metric_params
        self.contamination = contamination
        self.n_jobs = n_jobs

    def fit(self, X):
        X = np.asarray(X, dtype=float)
        n_samples = X.shape[0]
        self.n_neighbors_ = max(1, min(self.n_neighbors, n_samples - 1))

        # +1 because a training point is its own 0-distance neighbor; drop that column below.
        self._nn = NearestNeighbors(
            n_neighbors=self.n_neighbors_ + 1,
            algorithm=self.algorithm,
            leaf_size=self.leaf_size,
            metric=self.metric,
            p=self.p,
            metric_params=self.metric_params,
            n_jobs=self.n_jobs,
        ).fit(X)
        distances, neighbors = self._nn.kneighbors(X)
        self._distances_fit_X_ = distances[:, 1:]
        neighbors = neighbors[:, 1:]            # exclude self

        self._lrd = self._local_reachability_density(self._distances_fit_X_, neighbors)
        lrd_ratios = self._lrd[neighbors] / self._lrd[:, None]
        self.negative_outlier_factor_ = -np.mean(lrd_ratios, axis=1)

        if self.contamination == "auto":
            self.offset_ = -1.5
        else:
            self.offset_ = np.percentile(
                self.negative_outlier_factor_,
                100.0 * self.contamination,
            )
        self.decision_scores_ = -self.negative_outlier_factor_  # higher = more anomalous
        return self

    def _local_reachability_density(self, distances_X, neighbors_indices):
        # reach-dist(p, o) = max{ k-distance(o), d(p, o) }  -- floor by NEIGHBOR's k-distance.
        dist_k = self._distances_fit_X_[neighbors_indices, self.n_neighbors_ - 1]
        reach = np.maximum(distances_X, dist_k)
        # lrd = 1 / mean reach-dist; eps floors the >=MinPts-duplicates case (avg reach = 0).
        return 1.0 / (reach.mean(axis=1) + 1e-10)

    def score_samples(self, X):
        """Opposite LOF for new query points; larger values mean more normal."""
        X = np.asarray(X, dtype=float)
        distances_X, neighbors_X = self._nn.kneighbors(
            X,
            n_neighbors=self.n_neighbors_,
        )
        X_lrd = self._local_reachability_density(distances_X, neighbors_X)
        lrd_ratios = self._lrd[neighbors_X] / X_lrd[:, None]
        return -np.mean(lrd_ratios, axis=1)

    def decision_function(self, X):
        """Shifted opposite LOF; zero is the inlier/outlier threshold."""
        return self.score_samples(X) - self.offset_

    def anomaly_score(self, X):
        """Positive LOF-style score for APIs where higher means more anomalous."""
        return -self.score_samples(X)
```
