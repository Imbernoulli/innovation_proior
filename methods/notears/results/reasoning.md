Let me start from what actually blocks me, because the rest of this problem is genuinely easy and the blockage is the whole game. I have `n` samples of a `d`-dimensional vector and I believe they were generated by a linear structural equation model: each variable is a linear combination of its parents plus its own independent noise, `X_j = w_j^T X + z_j`. Stack the coefficient vectors into a matrix `W` whose column `j` holds the incoming weights of node `j`, and the support of `W` — which entries are nonzero — *is* the graph. So fitting is trivial: I want the `W` that best reconstructs the data. The reconstruction `XW` predicts each variable from the others, the residual is `X − XW`, and the least-squares score is `ℓ(W) = (1/2n)‖X − XW‖_F^2`, a smooth convex-in-`W` function with gradient `−(1/n) X^T(X − XW)`. If that were all, I'd run a solver and be done. And I have statistical cover for using exactly this loss: it's known that the least-squares minimizer recovers a true DAG with high probability even in high dimensions, and not just for Gaussian noise — the consistency carries over to non-Gaussian SEM too, with no faithfulness assumption needed. So the statistics are settled. The thing standing between me and an answer is the single requirement that the graph `W` induces be *acyclic*.

And that requirement is brutal in a very specific way. "Acyclic" is a property of the *support pattern* of `W` — a combinatorial property of a discrete object — and the set of DAGs on `d` nodes is astronomically large, growing superexponentially in `d`. I cannot enumerate it. Optimizing a score over it is NP-hard. So everything anyone does to learn DAGs is some clever way of *moving around inside the discrete set of acyclic graphs without ever leaving it*: exact solvers enumerate parent sets and solve an integer program, which is globally optimal but dies past a few dozen nodes; greedy methods like greedy equivalence search add or delete one edge at a time, checking after each move that no cycle was created, which is fast only when every node has a handful of parents and falls apart on dense hub graphs; order-based methods fix a topological ordering (automatically acyclic) and search over the `d!` orderings instead. Every one of these treats acyclicity as something you *enforce operationally by staying in the legal region of a discrete space*. They inherit that space's curse: bad scaling, or reliance on bounded-in-degree assumptions that real scale-free networks violate, or both. And they all need real graphical-model expertise to implement — none of them is "write down an objective and call a solver."

That last phrase is what I keep circling back to, because two neighboring problems got solved exactly by becoming "write down an objective and call a solver." Structure learning for *undirected* graphical models turned into a convex log-determinant program over the precision matrix — Banerjee, Yuan-Lin, and then the graphical lasso of Friedman, Hastie & Tibshirani — and once it was a smooth convex program, black-box optimizers took over and the field exploded with fast algorithms. Deep nets are the same story with stochastic gradient descent on a differentiable loss. In both cases the useful object was a *closed-form continuous program* that you hand to general-purpose optimization machinery. Directed acyclic graphs never got this, and there's an obvious reason: the undirected case has no acyclicity constraint to begin with, and the directed case's acyclicity constraint has no smooth closed form. So the question sharpens into something concrete. Can I find a function `h(W)` on real matrices — smooth, with a cheap gradient — that is exactly zero when `W`'s graph is a DAG and nonzero otherwise? If I can, then `min_W F(W) subject to h(W) = 0` is an ordinary smooth equality-constrained problem, and I get to throw away the entire discrete-search apparatus and call a Lagrangian solver. The whole method reduces to: build that one function.

Let me write down what `h` has to satisfy, carefully, because the constraints will guide the construction. (a) `h(W) = 0` if and only if `W` is acyclic — that's the point. (b) Ideally `h` should *quantify* how cyclic the graph is, larger for "more cyclic," because a continuous solver works much better when the constraint is a graded measure it can descend than when it's a flat indicator. (c) `h` smooth, so I can take gradients. (d) `h` and `∇h` cheap, ideally something a numerical library already computes. Now, there are obvious ways to get (a) and (b) that immediately fail (c) or (d): I could define `h` as the Euclidean distance from `W` to the nearest DAG, but that's a projection onto a nasty nonconvex set, not smooth and not cheap; or I could sum the edge weights along all cyclic paths, but enumerating cyclic paths is the combinatorial monster I'm trying to escape. So the desiderata aren't trivially satisfiable. I need an *algebraic* handle on acyclicity.

Forget weights for a moment and take a binary adjacency matrix `B`. There's an elementary fact about adjacency matrices: `(B^k)_{ii}` counts the number of closed walks of length `k` that start and return to node `i` — you can see it by induction, since `(B^k)_{ij} = Σ` over intermediate paths of length `k` from `i` to `j`, and the diagonal picks out the ones that come home. So `tr(B^k) = Σ_i (B^k)_{ii}` is the total number of length-`k` closed walks in the graph. And a directed graph is acyclic precisely when it has *no* closed walks of any length — a cycle is a closed walk, and any closed walk contains a cycle. So acyclicity is equivalent to `tr(B^k) = 0` for every `k ≥ 1`. That's exactly the algebraic restatement I wanted: a combinatorial property (no cycles) has become an algebraic one (a family of traces vanish). Now I just need to package "all the traces vanish" into a single smooth scalar.

The most natural first try: sum them up. `Σ_{k=1}^∞ tr(B^k)`. Since each `tr(B^k) ≥ 0` (it's a count), this whole sum is zero iff every term is zero iff `B` is a DAG. And there's a beautiful closed form lurking, because `Σ_{k=0}^∞ B^k = (I − B)^{-1}` is the Neumann series. So `tr((I − B)^{-1}) = tr Σ_{k=0}^∞ B^k = tr(I) + Σ_{k=1}^∞ tr(B^k) = d + (number of closed walks)`. Which means `B` is a DAG if and only if `tr((I − B)^{-1}) = d`. Clean — a single trace of a single matrix inverse, and inverses and traces are cheap. But wait. The Neumann series `Σ B^k` only converges when the spectral radius `r(B) < 1`. For a DAG that's automatic — a DAG's adjacency matrix is nilpotent, all eigenvalues zero, so `r(B) = 0 < 1`, and indeed `(I − B)^{-1} = Σ_{k=0}^{d-1} B^k` terminates. But for a *general* `B` along the optimization path, which need not be acyclic, `r(B)` can easily exceed 1, the series diverges, and `tr((I − B)^{-1})` no longer equals `d + Σ tr(B^k)` — worse, `I − B` can be singular and the inverse blows up. So this characterization is only valid on `{B : r(B) < 1}`, and that's a strong restriction I'd have to *project onto* at every step, which is itself nontrivial. I'm right back to a hard constraint. Wall.

Let me retreat from the infinite series and use the *finite* one. A DAG on `d` nodes has no cycle longer than `d`, and `B^d = 0` for a nilpotent `B`, so actually I only need `Σ_{k=1}^{d} tr(B^k) = 0`; no convergence issue, no spectral-radius condition, valid for all `B`. So `B` is a DAG iff `Σ_{k=1}^d tr(B^k) = 0`. That fixes the divergence. But now picture computing it for a non-DAG `B` with, say, dense connectivity: `B^k` counts walks, and the number of length-`k` walks grows roughly like (average degree)`^k`, exponentially in `k`. By the time `k` reaches `d`, the entries of `B^k` for even a modestly connected graph have blown past floating-point range. So both the function value and its gradient become numerically garbage. The unweighted counts are simply too big. Wall again — but this is an instructive wall, because it tells me the *form* of the fix: I don't need the raw counts, I need a *reweighting* of the counts that (i) keeps every term nonnegative so "sum is zero iff acyclic" survives, (ii) converges for all matrices so I drop the spectral-radius condition, and (iii) shrinks the high-`k` terms hard enough that the magnitudes stay tame.

Reweight `tr(B^k)` by `1/k!`. Then `Σ_{k=0}^∞ tr(B^k)/k! = tr Σ_{k=0}^∞ B^k/k! = tr(e^B)`, the matrix exponential — and the matrix exponential converges for *every* square matrix, full stop, because the `k!` in the denominator dominates any fixed matrix's growth. So no spectral-radius assumption, problem (i)-from-the-first-try gone. The terms are still individually nonnegative for a binary `B` (each `tr(B^k) ≥ 0`, scaled by positive `1/k!`), so `tr(e^B) = tr(I) + Σ_{k=1}^∞ tr(B^k)/k! = d + Σ_{k≥1} tr(B^k)/k!` is `d` exactly when every `tr(B^k)` vanishes, i.e. exactly when `B` is a DAG. And the `1/k!` is precisely the reweighting that defuses the magnitude explosion of the finite series — the high-`k` closed-walk counts, the ones that overflowed, get divided by an enormous factorial. Three birds: convergence for all `B`, nonnegativity preserved, numerics tamed. So for binary matrices,

  `B` is a DAG ⟺ `tr(e^B) = d`.

And `e^B` is not some exotic object I have to hand-roll — the matrix exponential is a workhorse of numerical linear algebra with a well-studied `O(d^3)` scaling-and-squaring algorithm (Al-Mohy & Higham) sitting in every scientific computing library. Desideratum (d) is basically free.

But `B` is binary, and `{0,1}^{d×d}` is a discrete space — I can't take gradients there, which is the entire reason I'm doing this. I need `h` defined on *real* `W`, smooth. Can I just write `tr(e^W) = d`? No: the argument that each term is nonnegative used that `B`'s entries are nonnegative (so closed-walk counts are nonnegative integers). For a signed real `W`, `(W^k)_{ii}` can be negative, terms can cancel, and `tr(e^W) = d` no longer forces every cyclic contribution to zero — a graph with cycles could accidentally hit `d` by cancellation. So the characterization genuinely fails for arbitrary `W`. Let me look at exactly what I needed: I needed the matrix in the exponential to be *entrywise nonnegative* and to have the *same support* as `W` (so that "the algebraic quantity is zero" tracks "the actual edges form no cycle"). The fix is staring at me. Replace `W` by its entrywise square. The Hadamard product `W ∘ W` has entries `w_{ij}^2 ≥ 0`, it is nonnegative everywhere, and it is zero in exactly the positions where `W` is zero — same support, same graph. Run the binary argument on `W ∘ W` (which is just a nonnegative weighted adjacency matrix, and the closed-walk argument works verbatim for nonnegative weights, with each closed walk now contributing a *product* of squared weights, still nonnegative): `tr(e^{W∘W}) = d + Σ_{k≥1} tr((W∘W)^k)/k!`, every term nonnegative, sum is `d` iff no weighted closed walks iff `W`'s graph is acyclic. So define

  `h(W) = tr(e^{W∘W}) − d`,

and `W` is a DAG if and only if `h(W) = 0`. This is smooth (composition of the entrywise square, the matrix exponential, and the trace, all smooth). And it automatically quantifies cyclicity — desideratum (b) — because `h(W)` is the `1/k!`-reweighted sum of *weighted* closed walks, where each edge contributes `w_{ij}^2`: larger values mean a larger total contribution from cyclic walks, through more cycles, heavier cycles, or both. Even better, every term is nonnegative, so `h(W) ≥ 0` for all `W`, and the set of DAGs is exactly the set of global minima of `h`. That's a lovely picture — the acyclic graphs are the floor of a smooth nonnegative landscape — though I should be careful that nonconvexity means a first-order stationary point of `h` is not the same as a global minimum, so I can't just minimize `h` and expect a DAG.

Now the gradient, because cheapness of `∇h` was desideratum (d) and a constrained solver will want it. I need `∇_W tr(e^{W∘W})`. Do it in two steps with the chain rule. First, the differential of `tr(e^M)` in `M`. Using the trace form of the matrix-exponential differential gives `d tr(e^M) = tr(e^M dM)`, and reading off the gradient from `tr(G^T dM) = ⟨G, dM⟩` we get `∂ tr(e^M)/∂M = (e^M)^T`. (The transpose is not cosmetic: `M = W∘W` need not be symmetric, so `e^M` need not be symmetric, and I have to keep the transpose.) Second, `M = W ∘ W`, so `∂M_{kl}/∂W_{ij}` is `2W_{ij}` when `(k,l)=(i,j)` and zero otherwise. Composing, `∂h/∂W_{ij} = (e^M)^T_{ij} · 2 W_{ij}`, i.e.

  `∇h(W) = (e^{W∘W})^T ∘ 2W`.

One more matrix exponential — the same one I already compute for `h` — Hadamard-multiplied by `2W` and transposed. So `h` and `∇h` together cost essentially a *single* `e^{W∘W}` evaluation. Let me sanity-check the whole thing concretely in my head with a tiny example, because I've made sign and transpose errors before: take the path `1→2→3`, a DAG, `B` with `B_{12}=B_{23}=1`. `B^2` has the single entry `(B^2)_{13}=1`, `B^3 = 0`, all diagonals zero, so `tr(e^B) = tr(I) = 3 = d` and `h = 0`. Now close it into a 3-cycle by adding `3→1`: now `B^3` has `1`s on the diagonal (the cycle `1→2→3→1` is a length-3 closed walk through each node), `tr(B^3) = 3 > 0`, so `tr(e^B) > d` and `h > 0`. Exactly the behavior I want, and exactly what the formula gives.

So the program I've been chasing exists:

  minimize `F(W) = (1/2n)‖X − XW‖_F^2 + λ‖W‖_1` subject to `h(W) = tr(e^{W∘W}) − d = 0`.

A smooth score, a smooth scalar equality constraint, real-matrix variables. No DAG space, no edge-by-edge cycle checks, no in-degree assumption — the entire matrix `W` is updated at once in each step (global, not local search), and the form of the score is agnostic to the noise distribution, so the same program handles Gaussian and non-Gaussian alike. Now I solve it with standard machinery. It is *not* convex — the constraint set `{W : h(W)=0}` is nonconvex — so I should be honest that I'll only reach stationary points, not certified global optima, just like every non-exact method. But I get to use the optimization literature wholesale.

The right tool for a single smooth equality constraint is the augmented Lagrangian method, and let me reconstruct why it's the right tool rather than, say, a plain quadratic penalty. A pure penalty `F(W) + (ρ/2) h(W)^2` only enforces `h=0` in the limit `ρ → ∞`, and large `ρ` makes the subproblem badly conditioned — the classic penalty-method pathology. The augmented Lagrangian adds a multiplier term,

  `L^ρ(W, α) = F(W) + (ρ/2) h(W)^2 + α h(W)`,

and the multiplier `α` lets the method drive `h` to zero *without* sending `ρ` to infinity, because near the solution `α` converges to the correct Lagrange multiplier and the linear term does the work the penalty would otherwise have to. So I keep `ρ` finite and manageable. How do I update `α`? Think of it as dual ascent. The dual function is `D(α) = min_W L^ρ(W, α)`; let `W*_α` be the inner minimizer. The constraint here is a single scalar `h`, and `D(α)` enters only through the term `α h(W*_α)` — by the envelope theorem the derivative with respect to `α` is just the constraint value at the optimizer, `∇D(α) = h(W*_α)`. So gradient ascent on the dual is

  `α ← α + ρ h(W*_α)`,

with the penalty coefficient `ρ` doubling as the dual step size — and for `ρ` large enough and `α` started near the true multiplier, this update converges linearly. That gives me the outer loop: repeatedly solve the inner unconstrained problem for `W`, then nudge `α` by `ρ·h`. I also want a schedule for `ρ`. I don't want to inflate it gratuitously, but if a given `ρ` isn't actually buying me progress toward feasibility I should raise it. Concretely: after an inner solve, if the new infeasibility `h(W_{t+1})` hasn't dropped to at least some fraction of the old one — say it's not below `0.25 · h(W_t)` — multiply `ρ` by 10 and re-solve; otherwise accept the step.

Now the inner subproblem: with `ρ, α` fixed, minimize over `w = vec(W) ∈ R^{d^2}` the function `f(w) + λ‖w‖_1`, where `f(w) = ℓ(W) + (ρ/2)h(W)^2 + α h(W)` collects all the smooth parts. Two cases. If `λ = 0` there's no nonsmooth term and this is plain smooth unconstrained minimization — exactly what L-BFGS is built for, a limited-memory quasi-Newton method that needs only gradients and a few vectors of memory (with the small modification for nonconvexity from the standard numerical-optimization texts so the Hessian approximation stays sensible). The reason I reach for a *second-order* method here and not plain gradient descent is specific to my constraint: each evaluation of `f` requires a matrix exponential, an `O(d^3)` operation, so I want to take as few, as informative, steps as possible — a quasi-Newton step makes far more progress per matrix-exp evaluation than a first-order step. That's a real cost argument, not a reflex.

If `λ > 0` the `ℓ1` term is nonsmooth and L-BFGS doesn't directly apply, so this is composite minimization. The principled tool is the proximal quasi-Newton method: at iterate `w_k`, build a quadratic model of the smooth part, `g_k^T d + (1/2) d^T B_k d`, with `g_k = ∇f(w_k)` and `B_k` the L-BFGS Hessian approximation, and keep the `ℓ1` term exact, so the descent direction solves

  `d_k = argmin_d  g_k^T d + (1/2) d^T B_k d + λ‖w_k + d‖_1`.

This inner quadratic-plus-`ℓ1` problem is itself solved by coordinate descent, and each coordinate has a closed form — that's the appeal. Fix coordinate `j`, write the one-variable problem as `min_z (1/2) a z^2 + b z + λ|c + z|` with `a = B_{jj}`, `b = g_j + (Bd)_j`, `c = w_j + d_j`. Let me actually solve it rather than wave at it. Substitute `u = c + z` (so `z = u − c`): the objective becomes `(1/2)a(u−c)^2 + b(u−c) + λ|u|`, and expanding the quadratic, `(1/2)a u^2 − (ac − b)u + λ|u| + const = (1/2)a(u − (c − b/a))^2 + λ|u| + const`. Minimizing `(1/2)a(u − t)^2 + λ|u|` over `u` is the textbook scalar lasso, whose solution is the soft-threshold `u* = S(t, λ/a)` where `S(x, τ) = sign(x)·max(|x| − τ, 0)` and `t = c − b/a`. Back-substituting, `z* = u* − c = −c + S(c − b/a, λ/a)`. So each coordinate update is one soft-threshold — cheap. And the low-rank compact form of the L-BFGS matrix `B_k = γI − Q Q̂` (with `Q` built from the stored `s`/`y` difference vectors) makes the quantities `a, b, c` computable and updatable in `O(m)` per coordinate, `m` the memory size, after an `O(m^2 d^2 + m^3)` precompute — far cheaper than the `O(d^6)` of a dense Hessian inverse — and aggressive active-set shrinking (skip coordinates that are zero with small subgradient) cuts the `d^2` factor down to the active-set size.

That proximal-Newton route is the "proper" composite solver, but let me notice something that makes the implementation almost embarrassingly simple, because the `ℓ1` term has special structure I can exploit to avoid composite optimization entirely. The only nonsmooth thing is `‖w‖_1`. Split each variable into a positive and a negative part, `w = w^+ − w^-` with `w^+, w^- ≥ 0`. If both parts of the same coordinate were positive, I could subtract their minimum from both, leave `w^+ − w^-` unchanged, and strictly reduce `w^+ + w^-`; so at any optimum with `λ > 0`, at most one side is positive. Thus `‖w‖_1 = Σ (w^+ + w^-) = 1^T(w^+ + w^-)`, which is now *linear* in the doubled variable. The whole subproblem becomes: minimize `f(w^+ − w^-) + λ·1^T(w^+ + w^-)` over `w^+, w^- ≥ 0` — a smooth objective with simple bound constraints, and *that* is solved directly by L-BFGS-B (bound-constrained L-BFGS) with no proximal machinery at all. I double the variable count from `d^2` to `2d^2`, trivial. The smooth gradient with respect to `W` is `G_smooth = ∇ℓ + (ρ h + α)∇h` (chain rule through the `(ρ/2)h^2 + α h` terms gives the coefficient `ρ h + α` on `∇h`); in the doubled variables the gradient is `+G_smooth + λ` for the `w^+` block and `−G_smooth + λ` for the `w^-` block (the `λ` from the linear `ℓ1` surrogate). And I add one more bound trick: force the diagonal of `W` to be exactly zero by setting its box constraints to `(0,0)`, so the model can never put a self-loop — a node regressing on itself — which would be a trivial degenerate "edge."

After the outer loop converges I have a `W̃` with `h(W̃)` at machine-tolerance, say `≤ 10^{-8}`, not exactly zero. So `W̃` is *almost* a DAG: a handful of tiny spurious weights may remain that would technically induce a cycle. I round them off. Hard-threshold: zero out every entry with `|W̃_{ij}| ≤ ω` for a small `ω`. This is justified on two grounds. Statistically, hard thresholding of regression estimates provably reduces false discoveries — small coefficients are mostly noise. And structurally, because `h` *quantifies* cyclicity (desideratum (b)), a near-feasible solution's residual cyclic mass lives in tiny edges, so a small threshold is the natural rounding operation. I can set `ω = 0.3` as a fixed implementation default; with `λ = 0` and `ω` fixed, the method needs no tuning beyond the solver tolerances, and with `ℓ1`, `λ` is the remaining sparsity knob.

Let me also pin down the small numerical constants: start `ρ = 1`, `α = 0`, `h = ∞`; cap the outer iterations at 100 and `ρ` at `10^{16}`; declare feasibility at `h ≤ 10^{-8}`; use the progress factor `0.25` and the `×10` `ρ` multiplier above; use `ω = 0.3` as the fixed threshold; and treat `λ` as the sparsity knob, with the demo implementation calling the solver at `λ = 0.1`. For the least-squares loss specifically, center the columns of `X` first (subtract the column means) so the intercept doesn't leak into the edge weights.

So let me write the whole thing as code, filling the one slot the harness left open — the acyclicity function `h` and its gradient — and wiring in the augmented-Lagrangian outer loop with the doubled-variable L-BFGS-B inner solve. The least-squares branch is the linear SEM score I have been deriving; the same smooth wrapper can also carry logistic and Poisson node-wise losses without changing the acyclicity machinery.

```python
import numpy as np
import scipy.linalg as slin
import scipy.optimize as sopt
from scipy.special import expit as sigmoid


def notears_linear(X, lambda1, loss_type, max_iter=100, h_tol=1e-8,
                   rho_max=1e16, w_threshold=0.3):
    """Learn a DAG by continuous optimization:
       min_W L(W;X) + lambda1*||W||_1  s.t.  h(W) = tr(e^{W∘W}) - d = 0,
    solved with an augmented Lagrangian. Column j of W are the incoming weights of node j,
    so W[i, j] != 0 means edge i -> j."""
    n, d = X.shape

    def _loss(W):
        # smooth node-wise loss and gradient
        M = X @ W
        if loss_type == 'l2':
            R = X - M
            loss = 0.5 / n * (R ** 2).sum()
            G_loss = -1.0 / n * X.T @ R
        elif loss_type == 'logistic':
            loss = 1.0 / n * (np.logaddexp(0, M) - X * M).sum()
            G_loss = 1.0 / n * X.T @ (sigmoid(M) - X)
        elif loss_type == 'poisson':
            S = np.exp(M)
            loss = 1.0 / n * (S - X * M).sum()
            G_loss = 1.0 / n * X.T @ (S - X)
        else:
            raise ValueError('unknown loss type')
        return loss, G_loss

    def _h(W):
        # smooth acyclicity: zero iff W's graph is a DAG, positive otherwise.
        # h(W) = tr(e^{W∘W}) - d ; one matrix exponential gives both value and gradient.
        E = slin.expm(W * W)                 # e^{W∘W}  (W*W is the Hadamard square)
        h = np.trace(E) - d
        G_h = E.T * W * 2                     # ∇h = (e^{W∘W})^T ∘ 2W
        return h, G_h

    def _adj(w):
        # doubled variables w = [w_pos, w_neg] (each d^2) -> W = w_pos - w_neg
        return (w[:d * d] - w[d * d:]).reshape([d, d])

    def _func(w):
        # smooth augmented-Lagrangian objective + linear l1 surrogate, in doubled variables
        W = _adj(w)
        loss, G_loss = _loss(W)
        h, G_h = _h(W)
        obj = loss + 0.5 * rho * h * h + alpha * h + lambda1 * w.sum()   # l1 = 1^T(w_pos+w_neg)
        G_smooth = G_loss + (rho * h + alpha) * G_h                       # chain rule: coeff (rho*h+alpha) on ∇h
        g_obj = np.concatenate((G_smooth + lambda1, -G_smooth + lambda1), axis=None)
        return obj, g_obj

    w_est, rho, alpha, h = np.zeros(2 * d * d), 1.0, 0.0, np.inf
    # nonnegative parts; diagonal pinned to zero so no self-loops
    bnds = [(0, 0) if i == j else (0, None) for _ in range(2) for i in range(d) for j in range(d)]
    if loss_type == 'l2':
        X = X - np.mean(X, axis=0, keepdims=True)        # center columns for the l2 loss

    for _ in range(max_iter):
        w_new, h_new = None, None
        while rho < rho_max:                            # raise rho only if feasibility stalls
            sol = sopt.minimize(_func, w_est, method='L-BFGS-B', jac=True, bounds=bnds)
            w_new = sol.x
            h_new, _ = _h(_adj(w_new))
            if h_new > 0.25 * h:                         # progress factor c = 0.25
                rho *= 10
            else:
                break
        w_est, h = w_new, h_new
        alpha += rho * h                                # dual ascent on the multiplier
        if h <= h_tol or rho >= rho_max:
            break

    W_est = _adj(w_est)
    W_est[np.abs(W_est) < w_threshold] = 0              # round the near-feasible solution to a clean DAG
    return W_est
```

Let me trace the causal chain back to the start. Fitting a linear SEM by least squares is trivial and statistically justified for both Gaussian and non-Gaussian noise; the *only* obstacle is the acyclicity constraint, which is combinatorial over a superexponential discrete space and forces every existing method into discrete search with its attendant scaling limits and in-degree assumptions. The escape is to turn acyclicity into a smooth scalar equality on real matrices, mirroring how the undirected case became a convex program. The bridge is the fact that `tr(B^k)` counts length-`k` closed walks, so acyclicity means all those traces vanish; summing them via the matrix inverse needs a spectral-radius condition I can't maintain, summing the finite series overflows numerically, but the factorial-reweighted sum `tr(e^B)` converges for all matrices, keeps every term nonnegative, and tames the magnitudes — giving `tr(e^B) = d` for binary DAGs. Squaring the weights with the Hadamard product `W∘W` lifts this to signed real matrices with the same support, yielding `h(W) = tr(e^{W∘W}) − d`, smooth, nonnegative, cyclicity-quantifying, with the one-line gradient `(e^{W∘W})^T ∘ 2W` that costs the same matrix exponential. The constrained program `min F(W) s.t. h(W)=0` is then solved by an augmented Lagrangian — finite `ρ`, dual ascent `α ← α + ρh`, `ρ` raised only when feasibility stalls — whose inner subproblem is smooth bound-constrained minimization once the `ℓ1` is linearized by splitting `W` into nonnegative positive and negative parts, handed straight to L-BFGS-B. A final hard threshold rounds the machine-tolerance-feasible solution into a clean DAG. The whole thing is a continuous program over the full matrix at once, distribution-agnostic, free of structural assumptions, and a few dozen lines of code calling a stock solver.
