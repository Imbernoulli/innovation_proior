# Cave Mapping Expedition: Dye-Marker Corroboration

## Problem
Your expedition is mapping a vast limestone cave system. You have placed `n` survey
markers at junctions deep underground. Before the final push you must paint every marker
with exactly one of `D` fluorescent dye colors (color codes `0 .. D-1`). Each marker is
painted **once** and keeps that color for the whole expedition.

During earlier reconnaissance the surveyors logged `m` **route observations**. Each
observation is a field sketch that will be *corroborated* on the final traverse — and pay
out its mapping value `w_j` — as long as it can spot **at least one** landmark it expects.
Concretely, observation `j` lists `k_j` (marker, expected-color) pairs; the observation is
corroborated if **at least one** of its listed markers is painted the exact color the sketch
expects (a disjunction of landmark cues). If none of its cues match, the observation is
worthless and pays nothing.

Because a single marker appears in many observations that expect it to be different colors,
no coloring can corroborate everything. Choose a color for every marker to maximize the
total mapping value of corroborated observations. The intended optimum is a weighted
Max-SAT / Max-CSP over a `D`-symbol domain and is NP-hard, so heuristics are expected.

## Input
```
n m D
```
followed by `m` lines. Line `j` describes one observation:
```
k_j  v_1 c_1  v_2 c_2  ...  v_k c_k  w_j
```
where `k_j` is the number of listed cues, each pair `v_e c_e` gives a marker index `v_e`
(1-based) and its expected color `c_e` in `{0, ..., D-1}`, and `w_j` is the mapping value.
Within one observation the marker indices `v_e` are distinct.

## Output
Print `n` integers `x_1 x_2 ... x_n` (whitespace-separated), where `x_i` in `{0, ..., D-1}`
is the dye color you paint marker `i`.

## Feasibility
Output must contain exactly `n` integers, each in the range `0 .. D-1`, and nothing else.
Any other output (wrong count, out-of-range color, trailing tokens, non-integer tokens) is
rejected and scores 0.

## Objective
Maximize `F` = sum of `w_j` over every observation `j` that is corroborated, i.e. there
exists a listed cue `(v_e, c_e)` with `x_{v_e} == c_e`.

## Scoring
Let `B` be the best value achievable by a **single uniform coloring** (painting every marker
the same color): for each color `c`, painting all markers `c` corroborates exactly the
observations that list some cue expecting color `c`; `B` is the maximum of this over all
`D` colors. The checker computes `B` itself, and `B > 0` on every test. Your ratio is
```
ratio = min(1000, 100 * F / max(1, B)) / 1000
```
So the best uniform coloring scores `0.1`, and reaching `10 x B` (or more) caps the ratio at
`1.0`. Higher corroborated value is always scored strictly higher. The reported per-test
score is `ratio`; the overall score is the mean over all tests.

## Constraints
- `1 <= n <= 5000`, `1 <= m <= 60000`, `2 <= D <= 12`.
- `1 <= k_j <= 5`, marker indices distinct within an observation, `1 <= w_j <= 1000`.
- Time limit: 5 s. Memory limit: 512 MB.

## Example
Input:
```
3 4 3
2 1 0 2 1 5
2 1 1 3 2 8
1 2 2 4
2 1 1 2 0 6
```
Here `n = 3`, `m = 4`, `D = 3`.

The best uniform coloring: painting all markers color `0` corroborates only observation 1
(it lists cue `1->0`), value `5`; all color `1` corroborates observation 2 (cue `1->1`) and
observation 4 (cue `1->1`), value `14`; all color `2` corroborates observation 2 (cue `3->2`)
and observation 3 (cue `2->2`), value `12`. So `B = 14`.

Consider the output `1 2 2`:
- Obs 1 (1->0 or 2->1): marker1=1 (not 0), marker2=2 (not 1) -> not corroborated.
- Obs 2 (1->1 or 3->2): marker1=1 matches -> corroborated, +8.
- Obs 3 (2->2): marker2=2 matches -> corroborated, +4.
- Obs 4 (1->1 or 2->0): marker1=1 matches -> corroborated, +6.

So `F = 18`, and `ratio = min(1000, 100 * 18 / 14) / 1000 = min(1000, 128.57)/1000 = 0.128571`.
