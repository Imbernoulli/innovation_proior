# Reservoir Dam Network: Spillway Release Allocation

## Problem

A regional water authority operates a **reservoir dam network** with `P` release **gates**
(spillways and irrigation valves) feeding a downstream grid of `N` **service zones** (towns,
farms, wetlands). During a single release cycle each zone may be plumbed to **at most one** gate,
or left dry (unserved).

Every gate `j` has two independent limits:

- a **discharge capacity** `C_j` — the maximum volume of water (megalitres) it can release this
  cycle, and
- a **valve limit** `K_j` — the maximum number of distinct zones it can be routed to (its
  physical manifold has only `K_j` outlets).

Routing zone `i` through gate `j` **consumes** `vol[i][j]` megalitres of that gate's discharge
(the zone's demand plus the conveyance loss along the channel from gate `j`, which depends on
distance, elevation, and seepage) and uses **one** of the gate's valve slots. In return it
delivers a **service value** `val[i][j]` (irrigated hectares plus flood-buffer credit, higher when
the gate's pressure and water quality suit that zone).

A gate cannot exceed either limit: the total volume routed through it must not exceed `C_j`, and
the number of zones routed through it must not exceed `K_j`. Your task is to decide, for every
zone, which gate (if any) to route it through, so as to **maximize the total service value** of
all served zones.

This is a Generalized Assignment Problem with an added per-agent cardinality (valve) constraint:
finding the true optimum is NP-hard, so heuristic constructions are expected and are graded on the
value they achieve.

## Input

```
N P
C_1 K_1 C_2 K_2 ... C_P K_P
val[1][1] vol[1][1] val[1][2] vol[1][2] ... val[1][P] vol[1][P]
val[2][1] vol[2][1] ... val[2][P] vol[2][P]
...
val[N][1] vol[N][1] ... val[N][P] vol[N][P]
```

- Line 1: the number of zones `N` and the number of gates `P`.
- Line 2: for each gate `j`, the pair `(C_j, K_j)` giving its discharge capacity and valve limit.
- The next `N` lines: for each zone `i`, the `P` pairs `(val[i][j], vol[i][j])` giving the service
  value and the volume consumed if zone `i` is routed through gate `j`.

## Output

Print `N` integers `a_1 a_2 ... a_N` (whitespace-separated; any layout). Value `a_i` is the gate
that zone `i` is routed through, or `0` if zone `i` is left dry.

- `0 <= a_i <= P`.

## Feasibility

An output is feasible iff:

1. every `a_i` is an integer in `[0, P]`, and
2. for every gate `j`, the sum of `vol[i][j]` over all zones `i` with `a_i = j` does not exceed
   `C_j` (discharge limit), and
3. for every gate `j`, the number of zones `i` with `a_i = j` does not exceed `K_j` (valve limit).

An infeasible output scores **0**.

## Objective

Maximize the total service value

```
F = sum over i with a_i >= 1 of val[i][a_i].
```

## Scoring

Let `B` be the total value of a fixed **first-fit** reference construction: process the zones in
input order, and route each zone through the lowest-indexed gate that still has both discharge
room (its remaining volume is at least this zone's `vol` there) and a free valve slot; if no gate
can take it, the zone is left dry. `B` is guaranteed positive.

Your ratio on a test is

```
ratio = min(1000, 100 * F / B) / 1000,
```

so matching the reference construction scores about `0.1`, and reaching ten times the reference
value (or more) caps the ratio at `1.0`. The reported per-test score is this ratio.

## Constraints

- `1 <= P <= 60`
- `1 <= N <= 2000`
- `1 <= C_j <= 1000000`
- `1 <= K_j <= N`
- `0 <= val[i][j] <= 1000`
- `1 <= vol[i][j] <= 1000`
- Time limit: 5 s
- Memory limit: 512 MB

## Example

Input:

```
5 2
70 2 70 3
40 20 10 30
50 25 5 40
30 15 60 22
10 35 55 18
20 40 45 33
```

Consider the output:

```
1 1 2 2 0
```

- Gate 1 holds zones 1 and 2: volume `20 + 25 = 45 <= 70`, count `2 <= K_1 = 2`. OK.
- Gate 2 holds zones 3 and 4: volume `22 + 18 = 40 <= 70`, count `2 <= K_2 = 3`. OK.
- Zone 5 is left dry.
- `F = val[1][1] + val[2][1] + val[3][2] + val[4][2] = 40 + 50 + 60 + 55 = 205`.

The first-fit reference here routes zone 1 -> gate 1 (fits), zone 2 -> gate 1 (45 vol, 2 valves =
K_1, gate 1 now full on valves), zone 3 -> gate 2 (gate 1 has no free valve; 15 vol on gate 2),
zone 4 -> gate 2 (15+18=33 vol, 2 valves), zone 5 -> gate 2 (33+33=66<=70, 3 valves = K_2),
giving `B = 40 + 50 + 30 + 55 + 45 = 220`. The shown output has `F = 205`, so its ratio is
`min(1000, 100*205/220)/1000 = 0.0931...`.
