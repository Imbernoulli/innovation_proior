# Glacier Sensor Net: Cluster Tiling for Maximum Coverage

## Problem

A polar research team is deploying a *glacier sensor net* over a rectangular survey
region discretized into a `W x H` grid of ice cells. Some cells sit over open
**crevasses** and cannot hold any hardware. The team owns a catalogue of prefabricated
**sensor clusters**, each a fixed connected polyomino footprint. A cluster may be dropped
anywhere on the ice as long as every one of its cells lands on a solid (non-crevasse) cell,
stays inside the region, and does not overlap any previously placed cluster. Each cluster
type may be reused any number of times, and each footprint may be freely **rotated (by any
multiple of 90 degrees) and reflected** before placement.

Your job is to choose a set of non-overlapping cluster placements that **covers as many ice
cells as possible** (maximizing the total number of grid cells occupied by sensor hardware).

Exactly tiling the solid region with the given footprints is a maximum-polyomino-packing
problem: it is NP-hard, the optimum is unknown, and many heuristics (value-density greedy,
orientation-aware placement, gap-filling passes, randomized restarts, local repair) are
viable and give genuinely different coverings.

## Input

The first line contains two integers:

```
W H
```

- `W` — grid width; columns are numbered `0 .. W-1`.
- `H` — grid height; rows are numbered `0 .. H-1`.

A cell is written as a pair `r c` with `0 <= r < H` (row) and `0 <= c < W` (column).

The second line contains an integer `K` — the number of crevasse (blocked) cells. Each of
the next `K` lines contains one blocked cell `r c`. All blocked cells are distinct. Every
other cell is **solid** and available for sensors.

The next line contains an integer `T` — the number of cluster footprint types. Each of the
next `T` lines describes one footprint:

```
sz r0 c0 r1 c1 ... r_{sz-1} c_{sz-1}
```

where `sz` is the number of cells and the following `sz` pairs are the footprint's cell
offsets (a connected polyomino, given in some fixed frame). You may place any rotation and
reflection of any footprint.

## Output

On the first line print `P` — the number of cluster placements you make (`P >= 0`).

Each of the next `P` lines describes one placement as the **absolute cells it occupies**:

```
sz  r0 c0 r1 c1 ... r_{sz-1} c_{sz-1}
```

where `sz` is the number of cells and the pairs are the absolute grid coordinates covered
by that placement. Printing `0` on the first line (and nothing else) is always valid and
covers nothing.

## Feasibility

An output is feasible iff **every** placement satisfies all of:

- `sz` matches the number of coordinate pairs listed, and every coordinate is in-bounds
  (`0 <= r < H`, `0 <= c < W`);
- when the placement's cells are translated so their minimum row and minimum column are `0`,
  the resulting shape is **congruent to some catalogue footprint** under rotation and/or
  reflection (i.e. it is a legal rotation/reflection of one of the `T` cluster types);
- every occupied cell is **solid** (not a crevasse);
- no cell is occupied by more than one placement (placements are pairwise disjoint, and a
  placement lists no cell twice).

Any infeasible output scores `0`.

## Objective

Let `F` be the total number of grid cells covered by your placements (the sum of `sz` over
all placements, since placements are disjoint). You are **maximizing** `F`.

## Scoring

The checker builds an internal reference covering `B` by a naive coarse-lattice
deployment: it stamps copies of the **first** catalogue footprint (type `0`, in its given
orientation) at anchor cells spaced `bboxH+2` rows and `bboxW+2` columns apart (where
`bboxH x bboxW` is footprint 0's bounding box), placing a copy at every anchor where the
whole footprint lands on solid cells. `B` is the coverage of that sparse baseline
(`B > 0`). The score for a test is

```
ratio = min(1.0, 0.1 * F / B)
```

A submission that merely reproduces the sparse baseline (`F = B`) scores `0.1`; covering
ten times the baseline area (`F = 10*B`) caps the score at `1.0`. Because the baseline is
deliberately sparse, dense coverings using the whole catalogue score well above it. The
overall score is the mean ratio across all tests.

## Constraints

- `4 <= W <= 60`, `4 <= H <= 60`.
- `0 <= K < W*H`; crevasses are sparse enough that footprint `0` fits at least once.
- `1 <= T <= 8`; each footprint has `1 <= sz <= 5` cells and is connected.
- Time limit: 5 s. Memory limit: 512 MB.

## Example

Input:

```
5 4
2
1 1
2 3
2
5 0 1 1 0 1 1 1 2 2 1
4 0 0 0 1 1 0 1 1
```

Here `W=5`, `H=4`. Blocked cells are `(1,1)` and `(2,3)`. Footprint `0` is the plus
pentomino (5 cells); footprint `1` is the `2x2` square (4 cells).

The coarse-lattice baseline tries to stamp the plus pentomino at anchor `(0,0)`; but the
plus would need cell `(1,1)`, which is a crevasse, so no lattice copy fits. The checker
then uses the smallest positive fallback and reports `B = 5` (one footprint-0 area).

A valid worked output places two `2x2` squares:

```
2
4 0 0 0 1 1 0 1 1
4 2 0 2 1 3 0 3 1
```

Both squares occupy only solid cells and do not overlap, covering `F = 8` cells. Its score
is `min(1.0, 0.1 * 8 / 5) = min(1.0, 0.16) = 0.16`.
