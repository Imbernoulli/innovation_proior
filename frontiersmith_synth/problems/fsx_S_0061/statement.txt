# Correlator Starvation: Telescope-Array Fiber Interdiction

## Problem

The **Wide-Field Radio Array** is a large interferometer: dozens of dish **antennas**
(**nodes**) scattered across a desert plateau, wired together by buried **fiber links**
(undirected edges). Every antenna's digitized voltage stream must reach the central
**correlator hub** `s`, and a link's **propagation-plus-buffering latency** is its integer
weight. Not every dish matters equally for a given science program: a fixed set of `q`
**science antennas** (the *targets*) carry the observation's key baselines, and the run's
data-freshness cost is the **total latency** — the sum, over all science antennas, of the
lowest-latency path each stream can take back to the correlator.

A maintenance crew has a night window to physically **de-splice fiber links** (the same effort
per link), and the array-operations office wants the *worst-case science degradation*: if an
adversarial scheduler could cut at most `k` links, which cuts would inflate the total data
latency the most? There is one hard rule from the observatory director: **every science antenna
must still have some working path back to the correlator** — you may never strand a science
dish. Any plan that isolates a science antenna from the hub is disqualified.

Your goal: choose a set `S` of at most `k` fiber links to cut so that every science antenna
stays connected to the correlator hub `s`, and the **total post-cut latency**

```
F = sum over science antennas a of dist_{G \ S}(s, a)
```

is as **large** as possible, where `dist_{G \ S}(s, a)` is the lowest-latency path from `s`
to `a` after the cuts.

This is **single-source multi-terminal shortest-path network interdiction under a cardinality
budget**: it is NP-hard, the true optimum is unknown, and several heuristics (subtree-count
greedy, best-improvement local search with re-planning, randomized restarts) are viable and
give different results.

## Input

The first line contains five integers:

```
n m s q k
```

- `n` — number of antennas (nodes), numbered `1..n`.
- `m` — number of fiber links (edges), indexed `1..m` in input order.
- `s` — the correlator hub node.
- `q` — the number of science antennas (targets).
- `k` — the maximum number of links you may cut (`k >= 1`).

The second line contains `q` distinct integers: the science-antenna node ids (each in `1..n`,
each different from `s`).

Each of the next `m` lines describes one fiber link:

```
u v w
```

a two-way link between antennas `u` and `v` (`u != v`) with latency `w` (`1 <= w`). The graph
is connected. Parallel links may occur.

## Output

On the first line print `r` — the number of links you cut (`0 <= r <= k`).
On each of the next `r` lines print one link index (an integer in `1..m`). All indices must be
distinct.

Printing `0` on the first line (and nothing else) — cutting no link — is always valid.

## Feasibility

An output is feasible iff:

- `0 <= r <= k`;
- every printed index is in `1..m` and all `r` indices are distinct;
- after cutting exactly the chosen links, **every** science antenna is still connected to `s`.

Any infeasible output scores `0`.

## Objective

Let `F = sum_a dist_{G \ S}(s, a)` be the total post-cut latency over the science antennas.
Larger `F` is better (a more starved correlator = a worse observation).

## Scoring

Let `B0 = sum_a dist_G(s, a)` be the total latency in the original graph (the do-nothing
baseline). Cutting links can only increase or keep each shortest path, so `F >= B0` always.
Your raw score is

```
Ratio = min(1.0, (F / B0) / 10)
```

i.e. `0.1` for a do-nothing plan (`F = B0`), rising linearly to the cap `1.0` when your plan
inflates the total latency to ten times the original. The subtask score is `100 * Ratio`.

## Constraints

- `2 <= n <= 2000`, `1 <= m <= 8000`.
- `1 <= w <= 20`, `1 <= q <= n - 1`, `1 <= k <= m`.
- Time limit: 5 s. Memory limit: 512 MiB.

## Example

Input:

```
4 4 1 2 1
3 4
1 2 1
2 3 1
2 4 1
1 3 6
```

Here the hub is `s = 1` and the science antennas are `{3, 4}`. Links are indexed `1..4`
in input order. In the original graph the shortest paths are `dist(1,3) = 1->2->3 = 2` and
`dist(1,4) = 1->2->4 = 2`, so the baseline `B0 = 2 + 2 = 4`. The cut budget is `k = 1`.

A do-nothing plan cuts nothing: `r = 0`, giving `F = 4 = B0`, so `Ratio = 0.1`.

A better plan cuts link `1` (the `1-2` link). Now the only way off the hub is link `4`
(`1-3`, latency `6`): `dist(1,3) = 6` and `dist(1,4) = 1->3->2->4 = 6+1+1 = 8`, and both
science antennas remain connected, so `F = 6 + 8 = 14`. Then `F / B0 = 3.5`, and
`Ratio = min(1.0, 3.5 / 10) = 0.35`. (Cutting both links `1` and `2` is not allowed: the
budget is `k = 1`.)
