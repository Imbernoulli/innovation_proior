# Battery Aging Law from Cold-Start Lab Cycling

## Problem
A battery lab has run an accelerated-aging pilot on lithium-ion cells, but the
available measurements are all from **early-cycle, low-temperature** operation.
Each training row reports:

- `N`: completed cycle count
- `T`: chamber temperature in degC
- `D`: depth-of-discharge fraction
- `R`: charge C-rate
- `Q`: measured capacity loss percentage points

The hidden degradation law is smooth but not a plain polynomial: it mixes
diffusion-like early aging, a charge-throughput interaction, and a regime that
activates at higher cycle counts and higher temperatures. The training data are
noisy, so a curve that simply interpolates the cold pilot rows should not
extrapolate well.

For each test id, infer a compact closed-form law `Q(N,T,D,R)` that predicts
capacity loss for held-out cells cycled far beyond the pilot range.

## Input (stdin)
The first line contains two integers:
```
n_train test_id
```
Each of the next `n_train` lines contains five floats:
```
N T D R Q
```
Training rows have approximately `N in [25, 320]` and `T in [6, 24]`. The
held-out split is never shown and uses high-cycle, high-temperature cells:
`N in [650, 1700]` and `T in [34, 58]`.

## Output (stdout)
Print exactly one line: a Python expression for `Q` in the variables
`N`, `T`, `D`, and `R`.

Allowed operators are `+ - * / ** %` and parentheses. Allowed functions are
`exp`, `log`, `sin`, `cos`, `sqrt`, `tanh`, and `abs`. Numeric literals are
allowed. No other names, attributes, indexing, imports, assignments, or multiple
lines are permitted.

Example output line, illustrative FORM only and not the hidden law:
```
0.2 + 0.001*N + 0.03*sqrt(D) - 0.01*log(R + 1.0)
```

## Feasibility
The expression must parse under the whitelist, reference only the allowed
variables and functions, and evaluate to a finite real number on every held-out
point. Oversized expressions, multi-line output, non-finite constants/results,
or implausibly huge predictions score `0.0`.

## Objective
The checker deterministically regenerates a high-cycle, high-temperature
held-out split for the same test id. Let `heldout_MSE` be the mean squared error
of the submitted expression on that split. Let `complexity` be the number of
operator, call, variable, and constant nodes in the expression AST. With
`LAMBDA = 0.0025`, the minimized objective is:
```
F = heldout_MSE * (1 + LAMBDA * complexity)
```

## Scoring
The checker builds an internal constant baseline using the training mean:
```
B = baseline_MSE * (1 + LAMBDA)
Ratio = min(1000, 100 * B / F) / 1000
```
A constant predictor reproduces the baseline and scores about `0.1`. Recovering
the interpretable degradation mechanisms improves held-out error, but the
temperature/cycle extrapolation and deterministic measurement noise keep the
best reference solution below a perfect score. Higher `Ratio` is better.

## Constraints
- `test_id` is in `1..10`; larger ids have fewer rows and more training noise.
- `n_train` ranges from 120 to 210.
- Expression source is at most 200000 bytes and at most one non-empty line.
- Scoring is deterministic CPU-only; there is no LLM judge and no timing-based
  objective.

## Example (worked score)
Suppose an expression has `heldout_MSE = 2.0` and `complexity = 25`, while the
training-mean baseline has `baseline_MSE = 12.0`. Then
```
F = 2.0 * (1 + 0.0025 * 25) = 2.125
B = 12.0 * 1.0025 = 12.03
Ratio = min(1000, 100 * 12.03 / 2.125) / 1000 = 0.566118
```
The actual held-out split and score are regenerated by the checker.
