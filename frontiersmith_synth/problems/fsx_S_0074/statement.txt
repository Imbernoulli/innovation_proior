# Polar Research Base: Thermal Relay Placement

## Problem
A polar research base is laid out on an `H x W` grid of square sectors. Some sectors are
impassable **ice crevasses** (`#`); the rest are usable **habitable sectors** (`.`). Heat
can only flow between two habitable sectors that share an edge (up / down / left / right);
crevasses block all heat flow.

Every habitable sector must be kept warm through the polar winter. You keep sectors warm by
installing **thermal relay units**. A relay installed on sector `v` has a *heating radius*
`rad[v]`: once switched on it warms every habitable sector reachable from `v` in at most
`rad[v]` heat-flow steps (moving only between edge-adjacent habitable sectors, and always
including `v` itself). Installing a relay on sector `v` costs `cost[v]` (a relay with a
larger radius needs a taller mast and a bigger generator, so it costs more, and terrain
varies sector to sector).

Your task: choose a set of habitable sectors on which to install relays so that **every
habitable sector is warmed** by at least one relay, while keeping the **total installation
cost as small as possible**.

This is a weighted covering / facility-location problem on a grid graph with obstacles: one
well-placed high-radius relay can warm many sectors at once, but it costs more than several
small local units, and crevasses distort every relay's warmed region. The optimum is
NP-hard, so heuristics (cost-effectiveness greedy, max-coverage greedy, redundancy pruning,
multi-start local search, LP-style rounding) are expected.

## Input
- Line 1: two integers `H W` — the grid height and width.
- Next `H` lines: a string of exactly `W` characters each. Character `#` is an impassable
  crevasse; character `.` is a habitable sector.
- The habitable sectors are numbered `1, 2, ..., n` in **row-major reading order** (top row
  left-to-right, then the next row, and so on); `n` is the number of `.` characters.
- Next line: one integer `n` — the number of habitable sectors.
- Next line: `n` integers `cost[1] ... cost[n]` — the installation cost of a relay on each
  habitable sector, in that numbering (`cost[v] >= 1`).
- Next line: `n` integers `rad[1] ... rad[n]` — the heating radius of a relay on each
  habitable sector (`rad[v] >= 1`).

## Output
- Line 1: one integer `K` — the number of relays you install (`0 <= K <= n`).
- Line 2: `K` distinct habitable-sector ids (the sectors where you install relays). If
  `K = 0`, print an empty second line (or nothing).

## Feasibility
Let `dist(a,b)` be the minimum number of heat-flow steps between habitable sectors `a` and
`b`, where a step moves between two edge-adjacent habitable sectors (a path may never pass
through a crevasse); `dist(a,a) = 0`, and `dist` is `+infinity` if no such path exists. A
relay installed on `v` **warms** every habitable sector `w` with `dist(v,w) <= rad[v]`.
Your output is feasible iff every habitable sector is warmed by at least one installed
relay, all printed ids are in `[1, n]`, and no id is repeated. Any infeasible or malformed
output scores `0`.

## Objective
Minimize the total installation cost `F = sum of cost[v]` over all installed relays `v`.

## Scoring
The judge builds a reference baseline `B` = the cost of installing one relay on **every
habitable sector** (always feasible, since a relay warms at least its own sector). Given
your feasible total cost `F`, your score is

```
Ratio = min(1.0, 0.1 * B / F)
```

So matching the baseline cost gives `Ratio = 0.1`; halving it gives `0.2`; cutting it to
one tenth of the baseline (or better) reaches the cap of `1.0`. Higher is better. An
infeasible output scores `0.0`.

## Constraints
- `1 <= H, W <= 20`, and `1 <= n <= 400`.
- `1 <= cost[v] <= 200`, `1 <= rad[v] <= 3`.
- Time limit: 5 s. Memory limit: 512 MB.

## Example
```
3 3
...
.#.
...
8
5 5 5 5 5 5 5 5
1 1 1 3 3 1 1 1
```
Here the grid is a 3x3 ring around a central crevasse. The `.` sectors, in row-major order,
are numbered:

```
1 2 3
4 # 5
6 7 8
```

The habitable sectors form a single ring `1-2-3-5-8-7-6-4-1`. Sectors `4` and `5` have
radius `3`; all others have radius `1`; every cost is `5`.

- Baseline `B` = `sum of all cost` = `8 * 5` = `40`.
- A relay on sector `4` (`rad=3`) warms every sector within 3 heat-flow steps of `4`, which
  is `{1,2,3,4,6,7,8}` — all sectors except `5`.
- Add a relay on sector `5` (`rad=1`), warming `{3,5,8}`. Together the two relays warm all
  `8` sectors, with `F = cost[4] + cost[5] = 5 + 5 = 10`, giving
  `Ratio = min(1.0, 0.1 * 40 / 10) = 0.40`.
- Installing a relay on every sector gives `F = 40`, i.e. `Ratio = 0.10`.
