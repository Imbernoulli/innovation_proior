# Regolith Relay Dispersion on the Selene Grid

## Problem
A lunar colony must anchor exactly **m** relay pods onto an integer surface grid of the
Selene basin. Three concerns are in play at once:

- **Dispersion (survival).** Micrometeorite showers and dust-charging arcs cascade along short
  surface paths, so two pods that sit close together tend to fail *together*. We want the
  *closest* pair of pods to be as far apart as possible — a max-min-distance goal under the
  Manhattan / L1 metric, which is the true rover-road travel metric on the lattice.
- **Coverage (mission, hard constraint).** A set of **D** science demand cells (drill sites, ice
  traps, antenna footprints) must each reach a pod: every demand cell must lie within L1 radius
  **R** of at least one pod. This is a hard feasibility requirement.
- **Resonance (power, bonus).** The colony's wireless-power grid phase-locks along diagonal "ley
  lines". A pod at cell `(x,y)` is *resonant* when `(a*x + b*y) mod K == r0`. Resonant pods draw
  free power and add a bonus. Snapping a spread-out pod onto a nearby ley line barely disturbs its
  dispersion, so resonance and dispersion can *both* be pursued — but only by a solver that looks
  for it.

The winning layout must respect coverage, spread the pods, **and** align as many of them as
possible with the power ley lines.

## Input
The first line contains nine integers:
```
W H m R D a b K r0
```
- `W H`      — grid dimensions; valid coordinates are `x` in `[0,W-1]`, `y` in `[0,H-1]`.
- `m`        — the exact number of relay pods to place.
- `R`        — the L1 coverage radius (hard constraint).
- `D`        — the number of science demand cells.
- `a b K r0` — the resonance rule: pod `(x,y)` is resonant iff `(a*x + b*y) mod K == r0`.

The next `D` lines each contain two integers `xd yd`, a demand cell (`0<=xd<W`, `0<=yd<H`).
All demand cells are distinct. The instance is guaranteed to admit a feasible placement of
exactly `m` pods (a covering subset using at most `m` of the listed cells exists).

## Output
Print exactly `m` lines, each `x y`: the integer coordinates of one relay pod.

## Feasibility
A submission is accepted only if all of the following hold:
1. Exactly `m` pods are printed, each with `0<=x<W` and `0<=y<H`.
2. All `m` pod cells are pairwise **distinct**.
3. **Coverage:** for every demand cell `d`, `min over pods p of L1(d,p) <= R`, where
   `L1((a,b),(c,e)) = |a-c| + |b-e|`.

Any violation scores **0**.

## Objective (maximize)
Let the pod set be `P` (`|P| = m`).
- `Dmin = min over all unordered pod pairs {p,q} of L1(p,q)`   (dispersion; the closest pair).
- `Align = number of pods (x,y) in P with (a*x + b*y) mod K == r0`   (resonant pods).

The composite score is the **product**
```
F = Dmin * (1 + Align / m) .
```
The resonance factor lies in `[1, 2]`. It couples the two mechanisms multiplicatively: a pure
dispersion layout maximizes `Dmin` but leaves `Align` at roughly the chance level `m/K`; nudging
each spread pod onto the nearest ley line pushes `Align` toward `m` for almost no loss of `Dmin`.
Distinct strategies (cover-and-cluster, pure farthest-point spread, and resonance-aware spread)
land at genuinely different points of this product.

## Scoring
The checker builds its own feasible reference layout `B` — a greedy cover completed by filling the
remaining pods with demand cells in input order — and evaluates that layout's objective value.
Your score for a test is
```
ratio = min(1.0, 0.1 * F / B) ,
```
so matching the reference earns `0.1` and reaching ten times the reference earns the cap `1.0`.
The final score is the mean ratio over all tests. Infeasible output scores `0`.

## Constraints
- `2 <= m <= D`, exactly `m` pods must be placed.
- `4 <= W, H <= 2600`.
- `1 <= R <= 400`.
- `2 <= D <= 12000`.
- `1 <= a, b <= 5`, `2 <= K <= 6`, `0 <= r0 < K`.
- Time limit: 5 s. Memory limit: 512 MB.

## Example
Input:
```
30 30 4 22 5 2 1 3 0
6 6
24 6
24 24
6 24
15 15
```
Five demand cells (four corners of an inner square plus its center) must each be within L1
distance 22 of a pod, using exactly 4 pods; a pod is resonant when `(2x + y) mod 3 == 0`.

Consider the layout `(6,6), (24,6), (24,24), (6,24)`: every demand cell is within 22 of some corner
(the center `(15,15)` is L1 distance 18 from `(6,6)` — feasible). Then `Dmin = L1((6,6),(24,6)) = 18`.
Resonant pods: `(6,6)->(2*6+6)=18, 18 mod 3 = 0` resonant; `(24,6)->54 mod 3 = 0` resonant;
`(24,24)->72 mod 3 = 0` resonant; `(6,24)->36 mod 3 = 0` resonant — all four, so `Align = 4`,
factor `= 1 + 4/4 = 2`, giving `F = 18 * 2 = 36`. If the checker's reference `B = 20`, the ratio is
`min(1, 0.1 * 36 / 20) = 0.18`.
