# Prospector Relay Grid: Degree-Capped Asteroid Comms

## Problem

A mining consortium has landed `N` autonomous **prospector stations** on the surface of
a large asteroid. Every station sits at a fixed 2D map coordinate `(x, y)` (metres in the
survey grid). To coordinate drilling you must lay down a **laser-relay network**: a set of
point-to-point relay links, each link joining two stations and costing cable/beam energy
proportional to the straight-line distance between them.

The whole fleet must end up **connected** — every station must be able to reach every
other station by hopping along relay links. But each station `i` carries only a limited
number of steerable relay heads: it can be an endpoint of **at most `cap_i`** links. You
choose which links to build. You want a connected network whose **total link length is as
small as possible**, without ever exceeding any station's relay-head cap.

Dropping the caps this would just be a minimum spanning tree; the per-station degree caps
make short networks fight for the few relay heads, so no efficient optimal algorithm is
known. Heuristics are expected.

## Input

```
N
x_1 y_1 cap_1
x_2 y_2 cap_2
...
x_N y_N cap_N
```

- Line 1: integer `N`, the number of stations.
- Each of the next `N` lines gives station `i` (numbered `1..N`): integer coordinates
  `x_i`, `y_i` and its relay-head cap `cap_i`.
- All station coordinates are distinct. Every `cap_i >= 2`, so a connected network always
  exists (for instance a single chain through all stations).

## Output

Print your relay network as a list of links.

```
E
a_1 b_1
a_2 b_2
...
a_E b_E
```

- Line 1: integer `E`, the number of links you build.
- Each of the next `E` lines gives one link between stations `a_k` and `b_k`
  (`1 <= a_k, b_k <= N`, `a_k != b_k`). Links are undirected; do not list the same
  unordered pair twice.

## Feasibility

An output is feasible if and only if all of the following hold:

- `E` links are printed, each joining two distinct valid stations;
- no unordered pair of stations appears more than once (no duplicate link);
- for every station `i`, the number of links incident to `i` is at most `cap_i`;
- the resulting network is **connected** (spans all `N` stations).

Any output that violates feasibility scores 0.

## Objective

The **length** of a link between stations `p` and `q` is the Euclidean distance rounded
to the nearest integer:

```
len(p, q) = round( sqrt( (x_p - x_q)^2 + (y_p - y_q)^2 ) )
```

Minimize the total network length

```
F = sum over all built links (a, b) of len(a, b).
```

## Scoring

Let `B` be the length of the **chain baseline**: the network that links the stations in
input order, `1-2, 2-3, ..., (N-1)-N` (a single path, always feasible since every
`cap_i >= 2`). Your score on a test is

```
ratio = min(1.0, (B / max(1, F)) / 10)
```

so a network that merely matches the chain baseline scores `0.1`, one half the baseline
length scores `0.2`, and reaching one tenth of the baseline length (or shorter) reaches the
capped score `1.0`. Larger is better. Per-test scores are averaged.

## Constraints

- `2 <= N <= 600`
- `0 <= x_i, y_i <= 10000`
- `2 <= cap_i <= 4`
- Time limit: 5s
- Memory limit: 512 MiB

## Example

Input:
```
4
0 0 2
0 10 3
10 10 2
10 0 3
```

The four stations sit on the corners of a 10x10 square (side length 10, diagonal
`round(sqrt(200)) = 14`). The chain baseline links `1-2, 2-3, 3-4` (three sides), so
`B = 10 + 10 + 10 = 30`.

Consider the network with links `1-2, 2-3, 3-4, 4-1` — the full square loop. Every station
has degree 2 (within its cap), it is connected, and `F = 10 + 10 + 10 + 10 = 40`. Then
`ratio = min(1.0, (30 / 40) / 10) = 0.075`.

A better network is the "star-ish" tree `2-1, 2-3, 2-4`: station 2 has degree 3 (cap 3, ok),
stations 1,3,4 have degree 1. It is connected and
`F = 10 + 10 + 14 = 34`, giving `ratio = min(1.0, (30 / 34) / 10) = 0.088...`.

The chain itself (`1-2, 2-3, 3-4`) has `F = 30 = B` and scores exactly `0.1`.
