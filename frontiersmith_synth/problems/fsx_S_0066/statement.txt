# Beehive Apiary: Forager Dispatch under Patch Saturation

## Problem

An apiary manages `B` **forager bees** and `P` **flower patches**. Each morning the beekeeper
must dispatch every forager to **at most one** patch (or keep it in the hive, i.e. idle).

A forager `i` can only reach a small set of nearby patches. For each patch `j` it can reach, the
forager has an **appetite** `a[i][j] > 0`: the amount of nectar it *would* haul back from `j` if
that patch were unlimited.

But a patch is **not** unlimited. Patch `j` holds only `S_j` units of nectar for the day. If the
foragers sent to `j` collectively want more than `S_j`, the patch is stripped bare and the
harvest from `j` is capped at `S_j`. Formally, if `A_j` is the total appetite of the foragers
dispatched to patch `j`, the nectar actually collected from `j` is

```
collected(j) = min(S_j, A_j).
```

Over-crowding a rich patch wastes foragers (their appetite spills past `S_j` and yields nothing),
so a good dispatch must **spread** foragers to avoid saturation while still steering each forager
toward a patch where its appetite counts.

Your task is to choose, for every forager, which reachable patch (if any) to send it to, so as to
**maximize the total collected nectar** `sum_j collected(j)`.

This is a saturating (submodular) assignment problem: finding the true optimum is NP-hard, so
heuristic dispatches are expected and are graded on the nectar they achieve.

## Input

```
P B
S_1 S_2 ... S_P
m_1 p_{1,1} a_{1,1} p_{1,2} a_{1,2} ... p_{1,m_1} a_{1,m_1}
m_2 p_{2,1} a_{2,1} ... p_{2,m_2} a_{2,m_2}
...
m_B p_{B,1} a_{B,1} ... p_{B,m_B} a_{B,m_B}
```

- Line 1: number of patches `P` and number of foragers `B`.
- Line 2: the nectar stock `S_j` of each patch (`1`-indexed, `S_j >= 1`).
- The next `B` lines describe the foragers. Line `i+2` starts with the count `m_i >= 1` of
  patches forager `i` can reach, followed by `m_i` pairs `(p, a)`: forager `i` can reach patch
  `p` with appetite `a`. The patches `p_{i,*}` are **distinct** and listed in **increasing**
  order of patch index. Every appetite `a >= 1`.

## Output

Print `B` integers separated by whitespace: the `i`-th value is the patch forager `i` is
dispatched to, or `0` if the forager stays idle.

## Feasibility

The `i`-th output value must be either `0`, or one of the patch indices `p_{i,*}` that forager
`i` can actually reach. Any value that is out of range, or a patch the forager cannot reach, or a
wrong number of tokens, makes the whole output **infeasible** (score `0`).

## Objective

Let forager `i`'s dispatch be `d_i` (a patch or `0`). For each patch `j`, let
`A_j = sum of a[i][j] over all foragers i with d_i = j`. The score objective is

```
F = sum_{j=1..P} min(S_j, A_j).
```

Maximize `F`.

## Scoring

Let `B*` be the total nectar of the reference dispatch that sends **every forager to its
lowest-indexed reachable patch** (the first patch on its line). Your score on a test is

```
ratio = min(1000, 100 * F / B*) / 1000,
```

i.e. `0.1` for matching the reference and up to `1.0` for reaching ten times the reference
harvest. The final score is the average ratio over all tests.

## Constraints

- `1 <= P <= 300`, `1 <= B <= 4000`.
- `1 <= m_i <= min(P, 8)`; appetites `1 <= a <= 100`; stocks `1 <= S_j <= 200000`.
- Time limit: 5 s. Memory limit: 512 MB.

## Example

Input:
```
3 4
5 5 100
2 1 4 2 6
2 1 4 3 9
1 1 4
2 2 3 3 5
```

Here patches 1 and 2 hold only 5 units each; patch 3 holds 100.

Reference dispatch (everyone to lowest-indexed reachable patch): foragers 1,2,3 all go to patch 1
(appetites 4+4+4 = 12, but `S_1 = 5`), forager 4 goes to patch 2 (appetite 3, `S_2 = 5`).
`B* = min(5,12) + min(5,3) = 5 + 3 = 8`.

A better dispatch: forager 1 -> patch 2 (a=6), forager 2 -> patch 3 (a=9), forager 3 -> patch 1
(a=4), forager 4 -> patch 3 (a=5). Then `A_1=4, A_2=6, A_3=14`, so
`F = min(5,4) + min(5,6) + min(100,14) = 4 + 5 + 14 = 23`, giving ratio
`min(1000, 100*23/8)/1000 = 0.2875`.
