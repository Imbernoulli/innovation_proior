# Solar Farm Inverter Commissioning: Test-Bay Job Shop

## Problem

A newly built solar farm has `J` power inverters that must all be **commissioned** before the
plant can be energised. Commissioning inverter `i` means pushing it through a fixed ordered
sequence of `L_i` **stages** (firmware flash, insulation test, thermal soak, grid-sync trial,
...). Stage `k` of inverter `i` must run on a specific **test bay** `mach[i][k]` and occupies that
bay for exactly `dur[i][k]` continuous minutes.

Two hard rules govern the shop:

- **Precedence.** The stages of one inverter are strictly ordered: stage `k` of inverter `i`
  cannot start until stage `k-1` of the same inverter has finished. (You cannot grid-sync an
  inverter you have not yet flashed.)
- **No-overlap.** Each test bay handles **one** stage at a time: two stages assigned to the same
  bay may not run in overlapping time intervals.

Different inverters are independent and may be worked on in parallel across different bays. Time is
measured in integer minutes starting at `0`. You choose a start time for every stage. The whole
commissioning campaign finishes at the **makespan** `F` = the latest stage completion time over all
stages. Plan the start times to finish the campaign as early as possible.

## Input

```
J M
L_1  mach[1][1] dur[1][1]  mach[1][2] dur[1][2]  ...  mach[1][L_1] dur[1][L_1]
L_2  mach[2][1] dur[2][1]  ...
...
L_J  mach[J][1] dur[J][1]  ...
```

- Line 1: number of inverters `J` and number of test bays `M`.
- The next `J` lines describe the inverters. Line `i` starts with `L_i`, the number of stages of
  inverter `i`, followed by `L_i` pairs `mach dur`: the bay (`1..M`) and duration (minutes) of each
  stage, in the order they must be performed.

## Output

Print `J` lines. Line `i` contains `L_i` non-negative integers: the start times
`s[i][1] s[i][2] ... s[i][L_i]` you assign to the stages of inverter `i`, in stage order.

## Feasibility

Let `e[i][k] = s[i][k] + dur[i][k]` be the finish time of stage `k` of inverter `i`. A schedule is
feasible iff **all** of the following hold:

- every start time is an integer with `0 <= s[i][k] <= 10^12`;
- **precedence:** for every inverter `i` and every `k >= 2`, `s[i][k] >= e[i][k-1]`;
- **no-overlap:** for any two distinct stages assigned to the same bay, their half-open intervals
  `[s, e)` do not overlap.

An output with the wrong number of tokens, an out-of-range value, a non-integer / `nan` / `inf`
value, a precedence violation, a bay double-booking, or trailing garbage is **infeasible** and
scores `0`.

## Objective

Minimise the makespan `F = max over all stages of e[i][k]`.

## Scoring

Let `B` be the makespan of the fully **serial** reference plan: run every stage of every inverter
one after another on a single global timeline (stage after stage, inverter after inverter, no two
stages ever concurrent). That plan is always feasible and its makespan equals the total work
`B = sum over all stages of dur[i][k]`. Your score for a test is

```
ratio = min(1000, 100 * B / F) / 1000
```

So reproducing the serial makespan scores exactly `0.1`, a plan that finishes 10x sooner (or
better) caps at `1.0`, and any improvement in between scores proportionally. Higher is better.

## Constraints

- `2 <= J <= 30`, `2 <= M <= 15`.
- Each inverter has `L_i = M` stages, each on a distinct bay (a permutation of the bays).
- `1 <= dur[i][k] <= 100`.
- Start times you output satisfy `0 <= s[i][k] <= 10^12`.
- Time limit: 5 s. Memory limit: 512 MB.

## Example

Input:
```
2 2
2 1 3 2 2
2 2 4 1 3
```

Inverter 1: stage 1 on bay 1 for 3 min, stage 2 on bay 2 for 2 min.
Inverter 2: stage 1 on bay 2 for 4 min, stage 2 on bay 1 for 3 min.

A feasible schedule:
```
0 4
0 4
```
Inverter 1: bay 1 during `[0,3)`, then bay 2 during `[4,6)`.
Inverter 2: bay 2 during `[0,4)`, then bay 1 during `[4,7)`.
Bay 1 holds `[0,3)` and `[4,7)` (no overlap); bay 2 holds `[4,6)` and `[0,4)` (no overlap);
precedence holds for both inverters. Makespan `F = 7`.
Reference `B = 3 + 2 + 4 + 3 = 12`. Score `= 100 * 12 / 7 / 1000 = 0.1714`.
