# Tide Pool Niche Partition: Minimizing Competitive Overlap

## Problem

A marine field station studies a rocky shore of **tide pools** linked by shallow
overflow channels. Every high tide, water surges between neighboring pools, so two
pools that share a channel exchange larvae and nutrients. The station wants to seed
each pool with **one** of `C` candidate *microhabitat niches* (a substrate + grazer
package). If two channel-linked pools are seeded with the **same** niche, their
colonies compete for identical resources whenever the tide mixes them, producing a
**competition conflict** whose severity equals that channel's mixing strength.

You must assign one niche to every pool so that the **total competition-conflict
weight** (summed over all channels whose two endpoints received the same niche) is as
**small** as possible, using no more than the `C` available niches.

Formally: you are given an undirected weighted (multi)graph on `n` pools. Assign to
each pool a label (niche) in `1..C`. The cost is the sum of the weights of all
*monochromatic* edges (both endpoints share a label). **Minimize** this cost.

This is a weighted minimum-conflict `C`-labeling problem (equivalently a weighted
minimum-uncut / Max-`C`-Cut complement). It is NP-hard, the optimum is unknown for the
given instances, and many heuristics — greedy sequential labeling, local-move descent,
randomized multi-restart search — are viable and produce different results.

## Input

The first line contains three integers:

```
n m C
```

- `n` — number of tide pools, numbered `1..n`.
- `m` — number of overflow channels (edges).
- `C` — number of candidate niches (labels available).

Each of the next `m` lines describes one channel:

```
u v w
```

meaning an overflow channel between pools `u` and `v` (`u != v`) with mixing strength
(conflict weight) `w` (`1 <= w`). The graph may contain parallel channels between the
same pair of pools.

## Output

Print `n` integers, one per line: the `i`-th integer is the niche assigned to pool `i`,
and must lie in `1..C`.

Printing `1` on every line is always valid.

## Feasibility

An output is feasible iff:

- it contains exactly `n` integers;
- each integer lies in `1..C`.

Any infeasible output scores `0`.

## Objective

Let `label[i]` be your niche for pool `i`. Let

```
F = sum over channels (u, v, w) with label[u] == label[v] of w
```

be the total competition-conflict weight. You are **minimizing** `F`.

## Scoring

Let `B` be the conflict weight of the do-nothing assignment that seeds **every** pool
with niche `1` (then every channel is monochromatic, so `B` equals the sum of all
channel weights). The score for a test is

```
ratio = min(1.0, 0.1 * B / max(1, F))
```

The all-same assignment (`F = B`) scores `0.1`; driving conflict down to one tenth of
the do-nothing cost (`F = B / 10`) caps the score at `1.0`. The overall score is the
mean ratio across all tests.

## Constraints

- `1 <= n <= 3000`, `1 <= m <= 25000`.
- `2 <= C <= 5`.
- `1 <= w <= 200`.
- Time limit: 5 s. Memory limit: 512 MB.

## Example

Input:

```
4 5 2
1 2 5
2 3 3
3 4 4
4 1 2
1 3 6
```

Here `B = 5 + 3 + 4 + 2 + 6 = 20` (all pools same niche).

Consider the assignment `1 2 1 2` (pool 1 -> niche 1, pool 2 -> niche 2, pool 3 ->
niche 1, pool 4 -> niche 2). Checking each channel:

- `(1,2)`: `1 != 2` — no conflict.
- `(2,3)`: `2 != 1` — no conflict.
- `(3,4)`: `1 != 2` — no conflict.
- `(4,1)`: `2 != 1` — no conflict.
- `(1,3)`: `1 == 1` — conflict, weight `6`.

So `F = 6` and the score is `min(1.0, 0.1 * 20 / 6) = min(1.0, 0.3333) = 0.3333`.

A valid worked output:

```
1
2
1
2
```

(A better assignment exists — this problem is open-ended — so higher scores are
achievable.)
