# The Narrow Chest

## Problem

A weaver has finished an `R`-by-`C` tapestry and must fold it into a narrow
travel chest holding only `cacheBytes` of unfolded cloth at once. Each cell
`(i,j)` has an **entropy class** `e(i,j)` in `[0, K-1]`: how tightly its
pattern can be compressed. A fixed table `levelBytes[0..K-1]` (nonincreasing)
gives the byte cost of one cell at compression level `L` — level `0` is raw,
higher levels fold tighter. A cell may only be stored at level `L <= e(i,j)`
(an intricate cell cannot be folded past what its pattern survives).

Cut the tapestry into rectangular **panels** with a grid of row cuts and
column cuts, and pick one compression level per panel — every cell in a
panel folds at that same level, so a panel's level is capped by the
*weakest* cell inside it.

The chest is packed against a **fixed inspection route**: `T` cell visits
`(i_1,j_1), ..., (i_T,j_T)` (row sweeps, diagonal walks, patches revisited
often). Before visiting a cell its panel must be unfolded; if it already is,
that's free. Otherwise unfold it (pay its byte cost), evicting the
longest-untouched resident panel (LRU) until the new one fits. This is the
*only* way compression matters: a panel's byte size is how much chest space
it occupies while resident, which decides what else stays folded out
*alongside* it — compressing a panel you never revisit can still save a
panel you revisit constantly, purely by freeing it room.

## Input

```
R C K
levelBytes_0 levelBytes_1 ... levelBytes_{K-1}
cacheBytes
R lines of C integers: e(i,j) for i=1..R, j=1..C
T
T lines: i j   (the fixed inspection route, 1-indexed)
```

## Output

```
rB
b_0 b_1 ... b_rB          (row cuts, 0 = b_0 < b_1 < ... < b_rB = R)
cB
c_0 c_1 ... c_cB          (column cuts, 0 = c_0 < c_1 < ... < c_cB = C)
```
then `rB*cB` integers, row-major over panels `(r,c)`: the chosen compression
level for panel `(r,c)`, which covers rows `[b_r,b_{r+1})`, columns
`[c_c,c_{c+1})` (0-indexed).

## Feasibility

`1 <= rB <= R`, `1 <= cB <= C`, cuts strictly increasing, spanning the full
range. Every level must lie in `[0, K-1]` and be `<=` the minimum `e(i,j)`
over that panel's cells. Any violation, or wrong token count, scores `0`.

## Objective

Simulate the chest starting empty: at each route step, look up the current
cell's panel. If resident, it's a hit. Otherwise it's a reload — add
`area * levelBytes[level]` bytes to the fault cost `F`, then evict
least-recently-touched resident panels until the new one fits (a panel
bigger than `cacheBytes` alone never becomes resident and faults every
visit). Minimize the total fault cost `F` over the route.

## Scoring

Let `B` be the fault cost of the checker's own naive reference: chop the
tapestry into a fixed 16x16 grid of panels and never compress (level 0
everywhere). Your score on a test case is
`ratio = min(1000, 100 * B / max(1, F)) / 1000`, so matching the reference
scores `0.1`, ten times fewer bytes reloaded scores `1.0` (the cap). Ratios
are averaged over all test cases.

## Constraints

`1 <= R, C <= 240`, `2 <= K <= 5`, `1 <= T <= 200000`,
`1 <= levelBytes_i <= 1e6` (nonincreasing), `1 <= cacheBytes <= 1e12`.
Time limit: 4s. Memory limit: 512MB.

## Example

`R=C=8`, `K=2`, `levelBytes=[4,1]`, `cacheBytes=300`. Entropy is `1`
everywhere except a `2x2` low-entropy block at rows `4-5`, cols `4-5`, where
`e=0`. Route (`T=12`): sweep row `1` across all `8` columns, then hammer
`(4,4),(5,5),(4,4),(5,5)`.

Reference (single `8x8` panel, level `0`, since `16>8` collapses it to one
panel): its `256`-byte panel fits under `cacheBytes=300`, so only the first
visit faults: `B=256`.

Cutting rows/cols at `0,3,5,8` isolates the low-entropy block into its own
`2x2` panel (forced level `0`, `16` bytes) and folds the eight background
panels to level `1` (`1` byte/cell: sizes `9,6,9,6,6,9,6,9`). Replaying the
route: row-1 columns `1-3` load a `9`-byte panel (fault), columns `4-5` load
a `6`-byte panel (fault), columns `6-8` load a `9`-byte panel (fault) — all
three stay resident (`24` bytes). The hammer then loads the `16`-byte center
panel (fault, resident total `40 <= 300`, nothing evicted); every remaining
hammer step hits. `F = 9+6+9+16 = 40`. `ratio = min(1000, 100*256/40)/1000 =
0.64`.
