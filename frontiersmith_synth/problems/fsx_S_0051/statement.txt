# Data-Center Cooling-Zone Tiling

## Problem

You are commissioning the raised floor of a new data center. The floor is an
`R x C` grid of tiles. Some tiles are **obstructed** (structural pillars, power
whips, already-installed equipment) and can never be covered. Every other tile is
a **free** tile that hosts one server rack.

You have a **catalog of `T` cooling-zone footprints**. Each footprint is a
polyomino: a connected set of grid cells describing the tiles a single
prefabricated cooling zone (in-row cooler + its plenum) covers. For footprint
`s` you own a limited **supply** `sup[s]` of prefabricated units.

You must place cooling zones on the floor so that:

- every placed zone covers only **free** tiles (never an obstructed tile),
- no two placed zones overlap,
- each placed zone is a copy of some catalog footprint, possibly **rotated by a
  multiple of 90 degrees and/or reflected** (the 8 symmetries of the square),
- you place at most `sup[s]` copies of footprint `s`.

Your goal is to **maximize the total number of tiles covered** by cooling zones
(i.e. the number of racks that receive dedicated cooling).

This is a hard 2-D bounded-knapsack / packing problem: the optimum is not known
and is not expected. Any feasible placement is accepted and scored.

## Input

```
R C T
R lines, each a string of C characters: '.' = free tile, '#' = obstructed tile
then, for each footprint s = 1..T:
   a line "m sup"          (m = number of cells, sup = available supply)
   m lines "r c"           (the cells of the footprint, integer coordinates)
```

Footprint coordinates are given relative to an arbitrary origin; only their shape
(up to translation) matters.

## Output

```
K
K lines, one per placed cooling zone:
   s r1 c1 r2 c2 ... r_m c_m
```

`K` is the number of zones you place (`0 <= K <= R*C`). Each line names a
catalog footprint `s` (`1 <= s <= T`) followed by the **absolute grid cells**
`(r_k, c_k)` it covers, with `0 <= r_k < R` and `0 <= c_k < C`. The number of
cells listed must equal `m`, the size of footprint `s`.

## Feasibility

A placement line is accepted only if the listed cells, taken up to translation,
are exactly one of the 8 dihedral orientations (4 rotations x optional
reflection) of footprint `s`. All listed cells must be free (not `#`), must lie
inside the grid, and must not coincide with any cell already covered by an
earlier line. You may place at most `sup[s]` copies of footprint `s`. Any
violation (bad shape, obstruction, overlap, out of range, supply exceeded,
trailing tokens) makes the whole output infeasible and scores 0.

## Objective

Let `F` be the total number of covered tiles = the sum of the sizes of all placed
zones (which, since zones never overlap, equals the number of distinct covered
cells). Maximize `F`.

## Scoring

The checker builds a weak internal baseline `B`: it places catalog footprints in
input order using only their given (un-rotated) orientation via a top-left
first-fit scan, and lets `B` be the number of tiles that baseline covers.

Your score for a test is

```
ratio = min(1.0, 0.1 * F / B)
```

reported as `Ratio: <ratio>`. Reproducing the baseline gives about `0.1`; a good
rotation-aware packing scores several times higher, up to the cap at `1.0`
(ten times the baseline). The final score is the mean ratio over all tests.

## Constraints

- `6 <= R, C <= 52`
- `1 <= T <= 8`, footprint size `2 <= m <= 5`
- `1 <= sup[s] <= 45`
- Time limit: 5 s
- Memory limit: 512 MB

## Example

Input:

```
4 5 2
.....
..#..
.....
.....
3 2
0 0
0 1
0 2
2 3
0 0
0 1
1 0
1 1
```

Here footprint 1 is a straight tromino (3 cells in a line, supply 2) and
footprint 2 is a 2x2 square (supply 3). The obstruction is at row 1, column 2.

A feasible output:

```
3
1 0 0 0 1 0 2
2 2 0 2 1 3 0 3 1
1 3 2 3 3 3 4
```

The first zone places footprint 1 horizontally on row 0 (cells (0,0),(0,1),(0,2)).
The second places the square on rows 2-3, columns 0-1. The third places footprint
1 horizontally on row 3 (cells (3,2),(3,3),(3,4)). Total covered `F = 3 + 4 + 3 =
10`. If the baseline `B` covered, say, 5 tiles, this output scores
`min(1.0, 0.1 * 10 / 5) = 0.2`.
