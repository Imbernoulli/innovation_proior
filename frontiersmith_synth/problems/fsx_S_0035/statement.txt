# The Curator's Dilemma: Conflict-Free Gallery Tour

## Problem

The *Meridian Museum* is assembling a single self-guided **gallery tour** for its
season pass. The museum has prepared `n` candidate **installations** (numbered `1..n`),
and each installation `i` has an estimated visitor **value** `w_i` (a positive integer
measuring expected engagement).

Unfortunately, some pairs of installations **conflict**: they may share a docent, occupy
overlapping floor space at the same hour, present clashing narratives, or route visitors
into a bottleneck. If two conflicting installations are both placed on the tour, the tour
is thrown out by the exhibitions board. The conflicts form a general graph — there is no
special structure such as bipartiteness to exploit.

Your job is to choose a set `S` of installations for the tour such that **no two chosen
installations conflict**, maximizing the total value of the chosen installations.

Formally: you are given an undirected graph on `n` vertices with vertex weights `w_i` and
`m` conflict edges. Choose an **independent set** `S` (no edge has both endpoints in `S`)
maximizing `sum_{i in S} w_i`.

This is the **maximum-weight independent set** problem on a general graph: it is NP-hard,
the optimum is unknown, and many heuristics (weight-greedy, weight-over-degree greedy,
local-search swap improvement, randomized restarts) are viable and diverge.

## Input

The first line contains two integers:

```
n m
```

- `n` — number of candidate installations (vertices), numbered `1..n`.
- `m` — number of conflict edges.

The second line contains `n` integers `w_1 w_2 ... w_n` — the values of the installations
(`1 <= w_i`).

Each of the next `m` lines contains two integers:

```
u v
```

meaning installations `u` and `v` conflict (`1 <= u, v <= n`, `u != v`). The graph may
list a pair at most once and contains no self-loops.

## Output

On the first line print `r` — the number of installations you place on the tour
(`0 <= r <= n`).

On each of the next `r` lines print one installation index (an integer in `1..n`). All
indices must be distinct.

Printing `1` followed by any single installation index is always valid (a single
installation conflicts with nothing by itself).

## Feasibility

An output is feasible iff:

- `0 <= r <= n`;
- every printed index is in `1..n` and all `r` indices are distinct;
- no conflict edge has **both** endpoints among the chosen indices (the chosen set is an
  independent set).

Any infeasible output scores `0`.

## Objective

Let `F = sum_{i in S} w_i` be the total value of your chosen installations. You are
**maximizing** `F`.

## Scoring

The checker builds an internal baseline `B`: the total value of the independent set
produced by the **index-order greedy** — scan installations `1, 2, ..., n` and add each
one to the set whenever it conflicts with none of the already-added installations. `B` is
always positive (installation `1` is always added).

The score for a test is

```
ratio = min(1.0, 0.1 * F / B)
```

A submission matching the index-order baseline (`F = B`) scores `0.1`; reaching ten times
the baseline value (`F = 10*B`) caps the score at `1.0`. The overall score is the mean
ratio across all tests.

## Constraints

- `1 <= n <= 6000`.
- `0 <= m <= 60000`.
- `1 <= w_i <= 5000`.
- Time limit: 5 s. Memory limit: 512 MB.

## Example

Input:

```
5 4
10 3 3 3 8
1 2
1 3
1 4
4 5
```

Installation `1` (value `10`) conflicts with `2`, `3`, `4`. Installation `4` conflicts
with `5`.

Index-order baseline: add `1` (value 10); skip `2,3,4` (all conflict with `1`); add `5`
(value 8, no conflict with `1`). Baseline set `{1, 5}`, so `B = 18`.

A better tour is `{2, 3, 4}`? That fails because `4` and `5`... no, `{2,3,4}` is
independent (none of `2,3,4` conflict with each other) with value `3+3+3 = 9`. Worse.

The set `{2, 3, 5}` is independent (no conflicts among them) with value `3 + 3 + 8 = 14`.

The set `{1, 5}` gives value `18`. Choosing `{1, 5}` yields `F = 18`,
`ratio = min(1.0, 0.1 * 18 / 18) = 0.1`.

A valid worked output:

```
2
1
5
```
