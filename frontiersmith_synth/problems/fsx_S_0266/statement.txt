# Interstellar Relay Coverage: Lighting the Star Lanes

## Problem

The **Auroran Concord** governs a cluster of star systems linked by ancient **hyperlanes**.
To keep every world reachable by faster-than-light couriers, the Concord must install
**relay beacons** on some of the systems. A relay placed on a system illuminates that
system and every other system that lies within `r` hyperlane **hops** of it.

Every system must end up illuminated by at least one relay. Building a relay on system `i`
costs `c_i` credits (each system has its own construction cost). Your task is to choose a
set of systems on which to build relays so that **every system is illuminated**, while
spending as **few credits as possible**.

Formally: you are given an undirected graph on `N` systems with `M` hyperlanes, an integer
coverage radius `r`, and a positive cost `c_i` for each system. Choose a set `S` of systems
so that for every system `j` there is some `i` in `S` with hop-distance(`i`, `j`) `<= r`.
Minimize `sum of c_i over i in S`.

This is a minimum-cost `r`-covering (facility-location / dominating-set) problem: it is
NP-hard, the true optimum is unknown, and many heuristics (coverage-greedy, cost-ratio
greedy, redundancy pruning, local search) are viable and diverge in quality.

## Input

The first line contains three integers:

```
N M r
```

- `N` — number of star systems, numbered `1..N`.
- `M` — number of hyperlanes.
- `r` — coverage radius in hops (`r >= 1`).

Each of the next `M` lines contains two integers:

```
u v
```

meaning an undirected hyperlane between systems `u` and `v` (`u != v`). The graph is
connected. There may be repeated pairs; they change nothing.

The next `N` lines each contain one integer: line `i` holds `c_i`, the cost of building a
relay on system `i` (`1 <= c_i`).

## Output

On the first line print `q` — the number of relays you build (`0 <= q <= N`).
On each of the next `q` lines print one system index (an integer in `1..N`). All indices
must be distinct.

## Feasibility

An output is feasible iff:

- `0 <= q <= N`;
- every printed index is in `1..N` and all are distinct;
- **every** system is within `r` hops of at least one system in your relay set.

Any infeasible output scores `0`.

## Objective

Let `F` be the total cost of your relays, `F = sum of c_i over the systems you chose`.
You are **minimizing** `F`.

## Scoring

Let `B` be the cost of the do-everything construction that builds a relay on **every**
system, `B = sum over all i of c_i` (always feasible, since a relay illuminates its own
system). The score for a test is

```
ratio = min(1.0, 0.1 * B / F)
```

Building everywhere (`F = B`) scores `0.1`; covering everything for one tenth of the total
cost (`F = B / 10`) caps the score at `1.0`. The overall score is the mean ratio across all
tests.

## Constraints

- `2 <= N <= 400`, `1 <= M <= 1500`.
- `1 <= r <= 2`.
- `1 <= c_i <= 20`.
- Time limit: 5 s. Memory limit: 512 MB.

## Example

Input:

```
5 5 1
1 2
2 3
3 4
4 5
2 4
5
2
9
3
7
```

The costs are `c = [5, 2, 9, 3, 7]`, so `B = 5 + 2 + 9 + 3 + 7 = 26`.

With radius `r = 1`, a relay on system `2` illuminates `{1, 2, 3, 4}` and a relay on system
`4` illuminates `{2, 3, 4, 5}`. Building on `{2, 4}` illuminates all five systems for a cost
of `F = c_2 + c_4 = 2 + 3 = 5`.

Score for this test: `min(1.0, 0.1 * 26 / 5) = min(1.0, 0.52) = 0.52`.

A valid worked output:

```
2
2
4
```
