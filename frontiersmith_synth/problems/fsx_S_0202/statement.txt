# Geothermal Field Balancing: Well Mode Selection under Zone Quorums

## Problem
An enhanced geothermal field operates `n` wells over a production campaign. Each well must
commit to exactly one of two circulation modes for the whole campaign: **injection**
(mode `1`, pumping working fluid down) or **production** (mode `0`, drawing hot fluid up).
Your job is to choose one mode per well.

Reservoir engineers have identified `m` **balance zones**. Each zone watches a small group of
wells and specifies, per watched well, the mode it *wants* that well in. A zone is
**balanced** (cleared) only when a **quorum** of its watched wells are in their wanted mode:
zone `j` lists `k_j` signed requirements (literals) and a threshold `t_j`, and it is cleared
iff **at least `t_j`** of its `k_j` literals are satisfied. A literal `+v` is satisfied when
well `v` is injecting (mode `1`); a literal `-v` is satisfied when well `v` is producing
(mode `0`). Zone `j` carries a positive integer **enthalpy value** `w_j`.

Because every well is coupled into many zones at once, and because a zone needs a *quorum*
(not just one) of its wells aligned, no assignment balances all zones. This is a **weighted
cardinality-constraint Max-SAT** instance (weighted Max-SAT generalized from disjunctions to
`at-least-t` threshold clauses): you maximize the total enthalpy value of balanced zones.
Finding the optimum is NP-hard, so strong heuristics (greedy, hill-climbing over
threshold-crossing flips, multi-restart local search) are expected.

## Input (stdin)
```
n m
<m lines, each: w k t l_1 l_2 ... l_k>
```
- Line 1: number of wells `n` and number of zones `m`.
- Each of the next `m` lines describes one zone: its enthalpy value `w` (`1 <= w`), its arity
  `k` (`1 <= k`), its threshold `t` (`1 <= t <= k`), then `k` literals. Each literal is a
  nonzero integer `l` with `1 <= |l| <= n`; `l > 0` means "well `l` injecting", `l < 0` means
  "well `|l|` producing". Within one zone the referenced wells are distinct.

## Output (stdout)
```
x_1 x_2 ... x_n
```
Print exactly `n` integers, each `0` (production) or `1` (injection). `x_v` is the mode chosen
for well `v`. Whitespace/newlines between values are ignored; there must be exactly `n` of
them and nothing else.

## Feasibility
An output is valid iff it contains **exactly `n`** tokens and **every** token is `0` or `1`.
Any other output (wrong count, a value outside `{0,1}`, trailing junk) scores `Ratio: 0.0`.

## Objective
Maximize `F` = the total enthalpy value of balanced zones under your assignment, where zone
`j` is balanced iff **at least `t_j`** of its `k_j` literals are satisfied.

## Scoring
The checker builds its own trivial reference assignment: **all wells producing** (every
`x_v = 0`). Under this assignment a literal `-v` is satisfied and `+v` is not, so a zone with
`neg_j` negative literals is balanced iff `neg_j >= t_j`. The baseline value is
```
B = sum of w_j over zones with (number of negative literals) >= t_j     (B >= 1).
```
With the maximization normalization:
```
sc = min(1000.0, 100.0 * F / max(1e-9, B))
Ratio = sc / 1000.0
```
Reproducing the all-production assignment gives `F = B` and `Ratio = 0.1`. An assignment that
balances `10x` the baseline value caps at `Ratio = 1.0`.

## Constraints
- `12 <= n <= 90`.
- `1 <= m <= 1600`.
- `1 <= k <= 5` and `1 <= t <= k` for every zone; enthalpy values `1 <= w <= 60`.
- Time limit: 5s. Memory limit: 512m.

## Example
Input:
```
3 4
5 2 1 1 2
3 2 1 -1 3
4 1 1 2
2 2 2 -2 -3
```
Zones: `Z1 = (>=1 of {well1 inject, well2 inject})` value 5;
`Z2 = (>=1 of {well1 produce, well3 inject})` value 3;
`Z3 = (>=1 of {well2 inject})` value 4;
`Z4 = (>=2 of {well2 produce, well3 produce})` value 2.

Baseline (all producing `0 0 0`): Z1 neg=0 (< 1, not balanced); Z2 neg=1 (>= 1, balanced,
w3); Z3 neg=0 (not balanced); Z4 neg=2 (>= 2, balanced, w2). So `B = 3 + 2 = 5`.

Suppose you output `1 1 0` (well1 inject, well2 inject, well3 produce):
- `Z1`: well1 inject and well2 inject satisfied -> 2 >= 1 -> balanced (w5).
- `Z2`: well1 produce? no (injecting). well3 inject? no. -> 0 >= 1 -> NOT balanced.
- `Z3`: well2 inject -> 1 >= 1 -> balanced (w4).
- `Z4`: well2 produce? no. well3 produce? yes -> 1 >= 2 -> NOT balanced.

`F = 5 + 4 = 9`. `sc = 100 * 9 / 5 = 180`, so `Ratio = 0.180`.
