# Proofed & Provisioned: Bakery Depot Coverage

## Problem

The *Golden Crumb* bakery cooperative runs a network of retail **shops** across a region
of interconnected **districts**. Fresh dough must reach every shop from a nearby
**provisioning depot**. A depot can be built in any district, but each district `u` has
its own build cost `c[u]` (land, ovens, staff). A depot built in district `u` can supply a
shop located in district `d` only if the *road travel time* from `u` to `d` is at most the
freshness radius `r` (dough spoils on longer hauls).

You must decide in which districts to build depots so that **every shop is supplied by at
least one depot within travel time `r`**, while **minimizing the total build cost**.

Formally: you are given an undirected weighted graph on `n` districts, a build cost per
district, a set of shop (demand) districts, and a radius `r`. Let `dist(u, d)` be the
shortest-path travel time. Choose a set `P` of districts (the depots) such that for every
shop `d` there exists some `u` in `P` with `dist(u, d) <= r`. Minimize `sum of c[u]` over
`u` in `P`.

This is a weighted covering / facility-location problem (a distance-`r` weighted dominating
set). It is NP-hard, the optimum is unknown, and many heuristics are viable: cost-effective
greedy set cover, redundancy pruning, LP-style rounding, and randomized local search all
give different results.

## Input

The first line contains four integers:

```
n m D r
```

- `n` — number of districts, numbered `1..n`.
- `m` — number of two-way roads.
- `D` — number of shops (demand districts), `D >= 1`.
- `r` — the freshness radius (maximum supply travel time).

The second line contains `n` integers `c[1] c[2] ... c[n]` — the build cost of a depot in
each district (`1 <= c[u]`).

Each of the next `m` lines contains three integers:

```
u v w
```

a two-way road between districts `u` and `v` with travel time `w` (`1 <= w`). The graph is
connected; parallel roads may occur.

The last line contains `D` distinct integers — the district indices of the shops.

## Output

On the first line print `q` — the number of depots you build (`0 <= q <= n`).
On each of the next `q` lines print one district index (an integer in `1..n`). All indices
must be distinct. A depot may be built in a shop district or in any other district.

## Feasibility

An output is feasible iff:

- `0 <= q <= n`;
- every printed index is in `1..n` and all are distinct;
- every shop `d` has at least one built depot `u` with `dist(u, d) <= r`
  (the shortest road travel time from `u` to `d` is at most `r`).

Any infeasible output scores `0`.

## Objective

Let `F = sum of c[u]` over the districts `u` where you build a depot. You are
**minimizing** `F`.

## Scoring

Let `B` be the cost of the reference construction that builds one depot in **every shop
district** (always feasible, since a depot supplies its own district at travel time `0`).
The score for a test is

```
ratio = min(1.0, 0.1 * B / F)
```

A submission matching the reference (`F = B`) scores `0.1`; a tenfold-cheaper covering
(`F = B/10`) caps the score at `1.0`. The overall score is the mean ratio across all tests.

## Constraints

- `3 <= n <= 450`.
- `1 <= m <= 2000`.
- `1 <= D <= n`.
- `1 <= w <= 3`, `1 <= r <= 12`.
- `1 <= c[u] <= 20`.
- Time limit: 5 s. Memory limit: 512 MB.

## Example

Input:

```
4 3 2 2
5 1 1 5
1 2 1
2 3 1
3 4 1
1 4
```

There are 4 districts on a path `1-2-3-4` (each road takes time 1). Shops are in districts
`1` and `4`, with build costs `c = [5, 1, 1, 5]` and radius `r = 2`.

Reference construction: build depots in both shop districts `1` and `4`, cost
`B = c[1] + c[4] = 5 + 5 = 10`.

A better plan: build a single depot in district `2` (cost `1`). It supplies shop `1`
(`dist(2,1) = 1 <= 2`) and shop `4` (`dist(2,4) = 2 <= 2`), so both shops are covered.
This costs only `F = 1`.

A valid worked output:

```
1
2
```

Score for this test: `min(1.0, 0.1 * 10 / 1) = min(1.0, 1.0) = 1.0`.
