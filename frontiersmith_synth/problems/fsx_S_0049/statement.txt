# Whiteout Blockade: Polar Route Interdiction

## Problem

The **Aurora Deep-Field Program** runs a research base on an antarctic ice sheet. Camps and
depots (**nodes**) are joined by staked-and-flagged **ice roads** (undirected edges); crossing
an ice road costs a known number of **transit hours** (its weight). All supplies flow from the
**Main Base** `s` to the remote **Deep-Field Station** `t`, and right now the fastest convoy
takes the shortest-transit-hours route from `s` to `t`.

A savage multi-day **whiteout** is forecast, and the safety office wants to know the *worst
plausible degradation*: which ice roads, if deliberately closed (barricaded and de-staked so no
convoy dares them), would slow the `s`-to-`t` convoy the most? Closing an ice road is not free —
each road `e` has a **closure effort** `c_e` (crew-hours to barricade it), and the crew has a
total closure budget `B` crew-hours for the drill.

There is one inviolable rule from base command: **the Main Base and the Deep-Field Station must
remain reachable from each other** after the closures — you may never strand the deep camp. Any
plan that severs `s` from `t` is disqualified.

Your goal: choose a set `S` of ice roads to close, with total closure effort at most `B`, such
that `s` and `t` stay connected and the length of the **fastest remaining route** from `s` to
`t` is as **large** as possible.

Formally, on an undirected weighted graph `G` with a budget `B`, choose `S` with
`sum_{e in S} c_e <= B` and `s`, `t` connected in `G \ S`, maximizing
`F = dist_{G \ S}(s, t)` (shortest-path transit hours).

This is **shortest-path network interdiction under a knapsack budget**: it is NP-hard, the true
optimum is unknown, and several heuristics (cheapest-cut greedy, best-improvement local search,
randomized restarts) are viable and give different results.

## Input

The first line contains five integers:

```
n m s t B
```

- `n` — number of nodes (camps/depots), numbered `1..n`.
- `m` — number of ice roads (edges), indexed `1..m` in input order.
- `s` — the Main Base node.
- `t` — the Deep-Field Station node (`t != s`).
- `B` — the total closure-effort budget (`B >= 1`).

Each of the next `m` lines describes one ice road:

```
u v w c
```

a two-way road between nodes `u` and `v` (`u != v`) with transit hours `w` (`1 <= w`) and
closure effort `c` (`1 <= c`). The graph is connected. Parallel roads may occur.

## Output

On the first line print `r` — the number of ice roads you close (`0 <= r <= m`).
On each of the next `r` lines print one road index (an integer in `1..m`). All indices must be
distinct.

Printing `0` on the first line (and nothing else) — closing no road — is always valid.

## Feasibility

An output is feasible iff:

- `0 <= r <= m`;
- every printed index is in `1..m` and all `r` indices are distinct;
- the total closure effort `sum_{e in S} c_e` does not exceed `B`;
- after closing exactly the chosen roads, `s` and `t` are still connected.

Any infeasible output scores `0`.

## Objective

Let `F = dist_{G \ S}(s, t)` be the shortest-path transit hours from `s` to `t` after your
closures. Larger `F` is better (a longer forced convoy = a worse whiteout).

## Scoring

Let `B0 = dist_G(s, t)` be the shortest-path transit hours in the original graph (the
do-nothing baseline). Your raw score is

```
Ratio = min(1.0, (F / B0) / 10)
```

i.e. `0.1` for a do-nothing plan (`F = B0`), rising linearly to the cap `1.0` when your plan
forces a route ten times longer than the original. The subtask score is `100 * Ratio`.

## Constraints

- `2 <= n <= 400`, `1 <= m <= 2000`.
- `1 <= w <= 100`, `1 <= c <= 100`, `1 <= B`.
- Time limit: 5 s. Memory limit: 512 MiB.

## Example

Input:

```
4 4 1 4 3
1 2 1 2
2 4 1 2
1 3 5 3
3 4 5 3
```

Here the base `s=1` and deep camp `t=4`. The fast route is `1 -> 2 -> 4` with transit
hours `1 + 1 = 2`, so the baseline `B0 = 2`. The detour `1 -> 3 -> 4` costs `5 + 5 = 10`.
Closure efforts are `c = (2, 2, 3, 3)` for roads `1..4`, and the budget is `B = 3`.

A do-nothing plan closes nothing: `r = 0`, giving `F = 2 = B0`, so `Ratio = 0.1`.

A better plan closes road `1` (the `1-2` link, effort `2 <= 3`). The fast route is gone;
`s` and `t` stay connected via `1 -> 3 -> 4`, so `F = 10`. Then `F / B0 = 5`, and
`Ratio = min(1.0, 5 / 10) = 0.5`. (Closing both roads `1` and `2` would cost effort `4 > 3`
and is infeasible.)
