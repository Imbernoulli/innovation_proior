# Cold-Chain Causal Cartographer

## Background

A national vaccine **cold-chain** programme instruments every fridge, freezer,
transport box and courier hand-off with sensors and audit fields. Each shipment
produces one record of **discrete status variables** — for example: ambient
temperature band, compressor duty state, door-open events, coolant-pack charge,
GPS route class, a customs-hold flag, sensor-fault indicators, and a final
"vaccine viable?" verdict.

These variables are causally wired together: a failing compressor *raises* the
temperature band, which *trips* an excursion alarm and *degrades* viability. But
the programme only ever logs **observational** snapshots — it never runs a
controlled experiment. Your job is to reconstruct the causal wiring diagram from
the logs alone.

Formally: the records for one region were generated by a discrete **Bayesian
network** — a directed acyclic graph (DAG) over the status variables plus
conditional probability tables. You are given a large batch of sampled records and
must **recover the DAG's directed edges** (which variable directly drives which).

This is a discrete **causal-discovery / structure-learning** task, scored by the
classic **Structural Hamming Distance (SHD)** to the hidden ground-truth graph,
aggregated over a battery of independently generated regions of growing size. The
multi-region aggregate rewards a *generalizable* discovery rule.

## Task

Write a **standalone program**: read ONE JSON object (a public region) from
`stdin`, write ONE JSON object (your estimated DAG) to `stdout`.

### Input (public instance, on stdin)

```json
{
  "name": "region31",
  "n": 10,
  "card": [2, 3, 2, 2, 3, 2, 2, 2, 3, 2],
  "data": [[0, 2, 1, 0, 1, 0, 0, 1, 2, 0], [1, 0, 0, 1, 2, 0, 1, 0, 1, 1], "..."]
}
```

- `n` — number of status variables.
- `card[i]` — the number of categories variable `i` takes (2 or 3); each record
  value `x_i` is an integer in `0 .. card[i]-1`.
- `data` — the list of records; each record is a list of `n` category integers.

> **Important.** Variable indices are **randomly relabelled**. The index order
> carries **no** information about causal (topological) order — you cannot assume
> `i` causes `j` merely because `i < j`.

The ground-truth graph and the generating CPTs live only in the grader's process;
your program never sees them.

### Output (answer, on stdout)

```json
{"edges": [[i, j], "..."]}
```

A list of **directed** edges `i -> j`, meaning variable `i` is a direct causal
parent of variable `j`.

An estimate is **valid** iff `edges` is a list of `[i, j]` integer pairs with
`0 <= i, j < n`, `i != j`, **no** duplicate directed edge, and **never** both
`i -> j` and `j -> i` for the same unordered pair. The empty estimate
`{"edges": []}` is valid (it recovers no wiring). Any malformed estimate, a
self-loop, a duplicated or bidirected pair, an out-of-range index, a crash, a
timeout, or non-JSON output makes that region score `0.0`.

## Objective & Scoring (deterministic)

For each region the grader computes the **Structural Hamming Distance** between
your directed edge set `E_hat` and the true edge set `E_true`, summed over all
unordered variable pairs `{i, j}`:

- true edge present, estimate absent — **+1** (a missed link)
- true edge absent, estimate present — **+1** (a spurious link)
- both present but in opposite directions — **+1** (a reversed link)
- both present with the same direction, or both absent — **0**

The weak reference is the **empty graph**, whose SHD equals the number of true
edges `E = |E_true|`. Your per-region score is a normalized minimisation objective:

```
r = clamp( 0.1 * E / max(SHD, 1e-12), 0, 1 )
```

So predicting nothing scores about **0.1**, halving the SHD scores about **0.2**,
driving SHD toward 0 approaches **1.0**, and an estimate worse than predicting
nothing (`SHD > E`) scores **below 0.1**. The final score is the **mean of `r`
over all regions**. Because the records are finite and purely observational,
orientation is genuinely ambiguous (Markov equivalence) and skeleton recovery is
imperfect — even strong discovery stays well below 1.0.

## Strategies (rough ladder)

- **Learn nothing** — predict the empty graph (`~0.1`).
- **Marginal association** — link every pair whose mutual information exceeds a
  threshold; keeps spurious transitive links and cannot orient.
- **Constraint-based (PC-style)** — mutual-information skeleton, conditional-
  independence pruning of transitive links, then v-structure/collider orientation
  with Meek propagation.
- **Score-based / hybrid** — BIC/BDeu hill-climbing over DAGs, higher-order CI
  tests, or bootstrap-averaged CPDAG estimation.

Your program is run in an **isolated sandbox** and only ever sees the public
region; it cannot read the grader's ground truth.
