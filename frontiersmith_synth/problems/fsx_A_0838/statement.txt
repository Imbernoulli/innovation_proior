# The Keeper's Feeding Cart

## Problem

A zoo keeps `n` animals. Each animal's feeding profile (diet, meds, allergy
notes) lives on a laminated **shelf**. Shelves have room for `pageCap` animal
profiles each. Every morning the keeper pushes a **feeding cart** along a
*fixed* tour that visits animals in a pre-set order (posted by the vet, the
same route every day) — a sequence of `T` animal visits. The cart itself only
has `L` shelf slots. Before visiting an animal, the keeper must have that
animal's shelf loaded on the cart; if it isn't, she jogs back to the shelving
room, grabs it, and — if the cart is full — bumps whichever loaded shelf she
used **longest ago** (a Least-Recently-Used cart). Grabbing a shelf that
isn't already loaded is a **reload**.

Your job: decide once, ahead of time, which shelf each animal's profile goes
on (respecting shelf capacity). You do **not** choose the tour — it is fixed
input. You want to minimize the total number of reloads over the whole tour.

The zoo also hands you a **paddock map**: `m` undirected edges recording
which animal enclosures are physically adjacent. This map describes the
grounds, not the tour, and nothing in the scoring formula reads it directly —
use it, ignore it, or mine it for structure as you see fit.

## Input

```
n m pageCap L T
```
then `m` lines `u v` (1-indexed paddock-adjacency edges, `u != v`), then one
line of `T` integers `trace_1 ... trace_T` (1-indexed animal ids, the fixed
tour order).

## Output

Print `n` integers, one per line: `shelf_1 ... shelf_n`, the shelf id
(`1 <= shelf_i <= n`) assigned to animal `i`.

## Feasibility

Every animal gets exactly one shelf id in `[1, n]`. No shelf id may be
assigned to more than `pageCap` animals. Output must contain exactly `n`
tokens (no more, no less). Any violation scores `0`.

## Objective

Simulate the cart: start empty (`L` slots). For each tour step, look up the
current animal's assigned shelf. If that shelf is already on the cart, it's a
hit (and becomes the most-recently-used slot). Otherwise it's a **reload**:
load it (evicting the least-recently-used slot first if the cart is full).
Let `F` be the total number of reloads over the whole tour. Minimize `F`.

## Scoring

Let `B` be the reload count of the **naive baseline** placement: animal `i`
goes on shelf `ceil(i / pageCap)` (animals grouped strictly by id order,
ignoring tour and paddock map). Your score for a test case is
```
ratio = min(1000, 100 * B / max(1, F)) / 1000
```
so matching the baseline scores `0.1`; halving its reload count scores about
`0.2`; the score is capped at `1.0` (ten times fewer reloads than baseline).
Per-test ratios are averaged.

## Constraints

* `1 <= n <= 2000`, `0 <= m <= 4000`.
* `1 <= pageCap <= 10`, `1 <= L <= 10`.
* `1 <= T <= 400000`, each `trace_k` in `[1, n]`.
* Time limit: 4 s. Memory limit: 512 MB.

## Example

`n=6`, `pageCap=2`, `L=2`, paddock edges `(1,4) (2,5) (3,6)`, tour
`T=8`: `1 4 3 6 2 5 3 6`.

The naive baseline puts `{1,2}` on shelf 1, `{3,4}` on shelf 2, `{5,6}` on
shelf 3 — but the *tour* actually alternates between `{1,4}`, `{3,6}` and
`{2,5}`, none of which match those id-order pairs. Replaying the tour
(shelves visited: `1,2,2,3,1,3,2,3`) against the cart:
step1 shelf1 miss(cache=[1]); step2 shelf2 miss(cache=[1,2]); step3 shelf2
hit; step4 shelf3 miss, evicts shelf1 (cache=[2,3]); step5 shelf1 miss,
evicts shelf2 (cache=[3,1]); step6 shelf3 hit; step7 shelf2 miss, evicts
shelf1 (cache=[3,2]); step8 shelf3 hit. Total `B=5`.

Placing shelves by which animals the tour actually visits *together*
(shelf A = `{1,4}`, shelf B = `{2,5}`, shelf C = `{3,6}`) instead gives
reloads only on steps `1,3,5` = `F=3`. Score = `min(1000, 100*5/3)/1000 =
166.67/1000 = 0.16667`. The paddock map's edges
`(1,4),(2,5),(3,6)` happen to connect each such pair here, but on the full
test data the map's clusters and the tour's true co-visit groups are cut
from *different* patterns — the map alone will mislead you on most tests.
