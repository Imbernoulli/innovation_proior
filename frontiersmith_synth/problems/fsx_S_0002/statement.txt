# Salmon Migration Ladder Placement

## Problem
A coastal river system is a network of pools connected by short river segments. Every
autumn, salmon must reach a set of **spawning grounds** located at certain pools, but
many segments are blocked by weirs and waterfalls. You can install a **fish ladder** at
any pool. A ladder installed at pool `v` has a *reach* `r[v]`: once built, it opens
migration to every pool within `r[v]` river-segment hops of `v` (including `v` itself).
Each pool `v` has its own installation cost `c[v]` (larger, higher-reach ladders cost
more, and terrain varies pool to pool).

Your task: choose a set of pools at which to build ladders so that **every spawning
ground is reachable** by at least one built ladder, while keeping the **total
installation cost as small as possible**.

This is a weighted covering / facility-location problem on a river graph: one well-placed
high-reach ladder may serve many spawning grounds, but it costs more than a small local
one. The optimum is NP-hard, so heuristics (cost-effectiveness greedy, LP-style rounding,
local-search pruning, multi-start) are expected.

## Input
- Line 1: two integers `N M` — the number of pools and river segments.
- Next `M` lines: two integers `u v` (1-indexed) — an undirected river segment between
  pools `u` and `v`.
- Next line: `N` integers `c[1] ... c[N]` — the installation cost at each pool (`c[v] >= 1`).
- Next line: `N` integers `r[1] ... r[N]` — the reach (in hops) of a ladder at each pool
  (`r[v] >= 0`).
- Next line: one integer `D` — the number of spawning grounds (`D >= 1`).
- Next line: `D` distinct integers — the pool ids of the spawning grounds.

## Output
- Line 1: one integer `K` — the number of ladders you build (`0 <= K <= N`).
- Line 2: `K` distinct pool ids (the pools where you build ladders). If `K = 0`, print
  an empty second line (or nothing).

## Feasibility
Let `dist(a,b)` be the number of river segments on the shortest path between pools `a`
and `b`. A built ladder at `v` **covers** every pool `w` with `dist(v,w) <= r[v]`. Your
output is feasible iff every spawning ground is covered by at least one built ladder, all
printed ids are in `[1, N]`, and no id is repeated. Any infeasible or malformed output
scores 0.

## Objective
Minimize the total installation cost `F = sum of c[v]` over all built ladders `v`.

## Scoring
The judge builds a reference baseline `B` = the cost of installing one ladder at **every
spawning ground** (always feasible, since a ladder covers its own pool). Given your
feasible total cost `F`, your score is

```
Ratio = min(1.0, 0.1 * B / F)
```

So matching the baseline cost gives `Ratio = 0.1`; halving it gives `0.2`; cutting it to
one tenth (or better) reaches the cap of `1.0`. Higher is better. An infeasible output
scores `0.0`.

## Constraints
- `1 <= N <= 400`, `M = N - 1` (the river system is connected; it is a tree).
- `1 <= c[v] <= 200`, `0 <= r[v] <= 4`, `1 <= D <= N`.
- Time limit: 5 s. Memory limit: 512 MB.

## Example
```
5 4
1 2
2 3
3 4
4 5
3 3 10 3 3
1 1 2 1 1
3
1 3 5
```
Here the river is a path `1-2-3-4-5`. Spawning grounds are `{1,3,5}`.

- Baseline `B` = `c[1]+c[3]+c[5]` = `3+10+3` = `16`.
- One good answer builds a single ladder at pool `3`, whose reach `r[3]=2` covers pools
  `{1,2,3,4,5}` — all spawning grounds. Then `F = c[3] = 10`, giving
  `Ratio = min(1.0, 0.1 * 16 / 10) = 0.16`.
- Another answer builds ladders at pools `1`, `3`, `5` (`F = 16`), giving `Ratio = 0.10`.
- Building at pools `2` and `4` (`r=1` each): pool `2` covers `{1,2,3}`, pool `4` covers
  `{3,4,5}`; all spawning grounds covered, `F = 3+3 = 6`, giving
  `Ratio = min(1.0, 0.1 * 16 / 6) = 0.2667` — the best of the three.
