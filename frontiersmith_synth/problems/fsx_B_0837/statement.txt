# Which Lines to Thicken Before Winter

## Problem

You manage an `R x C` grid of substations, `id(i,j) = i*C + j` (`i`=row `0..R-1`,
`j`=column `0..C-1`). Column `0` substations have unlimited generation. Power flows
along `M` given transmission lines, each an undirected edge between two grid-adjacent
substations with a **base capacity** and an **upgrade cost per unit of extra
capacity**. Before winter you may spend a reinforcement **budget** thickening any
lines you choose. Then `K` **scenarios** each arrive: a demand pattern (how much
power each far-side substation needs) combined with a set of lines knocked out by
that scenario's storm damage. You must pick the reinforcement that keeps the *worst*
scenario's blackout as small as possible.

## Input

```
R C M K Budget
u_1 v_1 cap_1 cost_1
...
u_M v_M cap_M cost_M
(K scenario blocks, each:)
L
node_1 demand_1
...
node_L demand_L
O
e_1 ... e_O
```

`2<=R<=6`, `4<=C<=9`, `K=10`. Each of the `M` lines gives endpoint ids, an integer
capacity `cap_i>=1`, and an integer upgrade cost `cost_i>=1`. Each scenario lists `L`
substations with positive integer demand (all with column `>=` a fixed midpoint) and
then `O` **outaged** line indices (`0`-indexed into the line list) that are forced to
zero capacity for that scenario only, regardless of any upgrade.

## Output

```
E
e_1 add_1
...
e_E add_E
```

`E` distinct line indices, each with a non-negative integer `add` — extra capacity
bought for that line (0 if omitted). Cost is `sum(add_k * cost_{e_k})`.

## Feasibility

All `e_k` in `[0,M-1]`, no repeated index, and total cost `<= Budget`. Any violation
scores `0`.

## Objective

For each scenario, build the flow network: column-`0` nodes are sources (unlimited),
each demanded node sinks at most its own demand, and every non-outaged line is an
edge of capacity `base + add` (outaged lines carry `0` no matter how much you added).
**Served** = the max flow. **Unserved** = total demand − served. Minimize the
**maximum Unserved over all `K` scenarios** — the worst blackout, not the average one.

## Scoring

Let `B` be the worst-case Unserved of the do-nothing network (no upgrades at all).
Your score on a test is `ratio = min(1.0, 0.1 * B / max(1, F))` where `F` is your
worst-case Unserved. Doing nothing scores `0.1`; halving the worst blackout scores
`0.2`; eliminating it scores up to `1.0`. Overall score is the mean ratio over 10
tests.

## Constraints

Time limit: 5 s. Memory limit: 512 MB. Deterministic scoring.

## Example

`R=2 C=2 M=4 K=2 Budget=10`, lines `0:(0,1,cap4,cost1) 1:(2,3,cap30,cost5)
2:(0,2,cap30,cost2) 3:(1,3,cap30,cost2)` (column `0` = nodes `0,2`; demand column =
nodes `1,3`). Scenario 0 (calm, no outage): demand `node1=3, node3=20`; fully served
via line 1 (`0->2->3`) and line 0 (`0->1`), Unserved `=0`. Scenario 1 (line 1
outaged): the *only* surviving route to either demand node is `0->1` (line 0,
cap `4`) — reachable from node 2 via `2->0` (line 2) or straight from node 1's own
supply — so total flow is capped at `4` no matter how the demand is split: served
`=4`, Unserved `=19`. Baseline `B = max(0, 19) = 19`.

Spending the whole budget thickening line **1** (the line that looked busiest in the
calm scenario, `cost5` -> `+2` capacity) changes nothing: line 1 is outaged in
scenario 1 regardless of its capacity, so `F` stays `19` and the ratio stays `0.1`.

Spending the whole budget on line **0** instead (`cost1` -> `+10` capacity, new
cap `14`) fixes the line every scenario can actually still use: scenario 1's served
rises to `14`, Unserved `9`; scenario 0 stays fully served. `F = max(0,9) = 9`, ratio
`= min(1.0, 0.1*19/9) = 0.211`.

Output for the second (better) solution:
```
1
0 10
```

## Notes

The line that carries the most power under normal conditions is not necessarily the
one whose thickness determines how bad your worst storm gets — read every scenario's
demand and outage list before deciding where the budget goes.
