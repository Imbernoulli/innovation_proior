# Quarantine Cuts on a Drifting Contact Graph

## Problem
An outbreak spreads over a contact network. The epidemic threshold is governed by the
**adjacency spectral radius** (the largest eigenvalue of the adjacency matrix): the smaller
it is, the harder it is for the disease to become self-sustaining. Public health can sever
exactly `k` contact links (delete `k` edges). Deleting links is disruptive, so the graph
must stay **connected**. Choose which `k` links to cut so the residual spectral radius is as
small as possible.

The catch: the network contains several overlapping dense communities. The eigenvector that
"certifies" where the spectral mass sits is a *moving target* — as you suppress the leading
community, the top eigenvalue migrates to another community, and cuts chosen against the
original eigenvector become wasted effort.

## Input
```
n m k
u_1 v_1
...
u_m v_m
```
- `n` nodes (labelled `1..n`), `m` undirected edges, cut budget `k`.
- Each `u_i v_i` is an edge (`1 <= u_i < v_i <= n`), no self-loops, no duplicates.
- The graph is connected. `1 <= k < m - (n-1)` (a spanning tree always survives).

## Output
Print exactly `k` lines, each `u v` naming a **distinct existing edge to delete**
(orientation may be given either way; `u != v`).

## Feasibility
- Every printed pair must be an edge present in the input; no edge may be deleted twice.
- After removing your `k` edges, the graph on all `n` nodes must remain **connected**.
An output violating any rule scores 0.

## Objective (minimize)
Let `G'` be the graph after your deletions (`m - k` edges). Its residual spectral radius
`lambda(G')` is defined by this **deterministic power iteration** (so it is reproducible):
1. `A` = symmetric adjacency matrix of `G'`; `x` = the all-ones vector of length `n`.
2. Repeat 400 times: `y = A x`; then `x = y / ||y||_2`.
3. `lambda = x^T A x` (the Rayleigh quotient).

You want `lambda(G')` small. Because every feasible output leaves exactly `m-k` edges,
`lambda(G') >= L` where `L = 2 (m-k) / n` is the residual average degree — a hard floor you
cannot beat. Define `F = max(lambda(G') - L, 1e-6)`.

## Scoring
The judge builds a reference cut `R` (delete the `k` edges of smallest endpoint
degree-product that lie outside a spanning tree — the "cut the least-central links" recipe)
and measures its `B = lambda(G_R) - L`. Your score on a case is
```
ratio = min(1.0, B / (10 * F)).
```
The reference cut scores about `0.1`; halving `F` roughly doubles your ratio, up to the cap
`1.0`. Ten cases are averaged.

## Constraints
- `n <= 5000`, `m <= 120000`, each input file <= 5 MB.
- Time limit: 5 s. Memory limit: 512 MB.

## Example
```
6 8 2
1 2
1 3
2 3
3 4
4 5
4 6
5 6
2 5
```
Nodes {1,2,3} and {4,5,6} form two triangles joined by edges 3-4 and 2-5. Deleting the two
inter-community links `3 4` and `2 5`:
```
3 4
2 5
```
disconnects node set {1,2,3} from {4,5,6} — **infeasible** (scores 0). Instead deleting one
edge from each triangle, e.g.
```
1 3
4 6
```
keeps the graph connected and lowers both communities' local density, reducing `lambda(G')`
below the reference cut and earning a ratio above `0.1`. Suppressing only one triangle (both
cuts inside {1,2,3}) leaves the other triangle hosting the top eigenvalue — the drift trap.
