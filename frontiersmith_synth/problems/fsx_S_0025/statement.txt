# Wasp Blight in the Apiary: Foraging-Corridor Interdiction

## Problem

The *Goldenmeadow Apiary* keeps its bees in a single central **hive cluster** (node `s`).
Every day the foragers fly out along a network of two-way **nectar corridors** (edges) that
connect meadow way-points, to reach a fixed set of **forage patches** (blooming fields).
Each corridor has a positive *flight cost* (energy the bees spend crossing it). A bee that
must reach a forage patch always follows the cheapest available route, so the daily
foraging effort toward a patch equals the shortest-path cost from the hive cluster to that
patch, and the colony's total effort is the **sum of those shortest-path costs over all
forage patches**.

A wasp blight is about to sever some corridors. As the apiarist you want to understand the
worst damage the blight can do: choosing **at most `k` corridors** to destroy so that the
colony's total foraging effort becomes as **large** as possible — but the destruction is
only admissible if **every** forage patch is still reachable from the hive cluster (a
colony that can no longer reach a patch simply abandons it, which the model forbids).

Formally: you are given an undirected weighted (multi)graph with a source `s` and a set of
`q` terminals (forage patches). Choose a set `S` of at most `k` edges to delete such that
every terminal remains reachable from `s`, maximizing
`F = sum over terminals t of dist_{G - S}(s, t)`, where `dist` is shortest-path cost.

This is a **multi-terminal shortest-path interdiction** problem: it is NP-hard, the optimum
is unknown, and many heuristics (traffic-weighted greedy, best-improvement local search,
randomized restarts) are viable and diverge.

## Input

The first line contains five integers:

```
n m s q k
```

- `n` — number of way-points (nodes), numbered `1..n`.
- `m` — number of nectar corridors (edges).
- `s` — the hive cluster (source node), `1 <= s <= n`.
- `q` — the number of forage patches (terminals).
- `k` — the maximum number of corridors you may destroy.

Each of the next `m` lines describes one corridor (edge index `1..m` in input order):

```
u v w
```

a two-way corridor between way-points `u` and `v` with flight cost `w` (`1 <= w`).
The graph may contain parallel corridors and is guaranteed to be connected, so every
terminal is reachable from `s` initially.

The final line contains `q` integers — the forage-patch node ids. They are distinct and
none of them equals `s`.

## Output

On the first line print `r` — the number of corridors you destroy (`0 <= r <= k`).
On each of the next `r` lines print one corridor index (an integer in `1..m`). All indices
must be distinct.

Printing `0` on the first line (and nothing else) is always valid.

## Feasibility

An output is feasible iff:

- `0 <= r <= k`;
- every printed index is in `1..m` and all `r` indices are distinct;
- after removing exactly the chosen corridors, every forage patch (terminal) is still
  reachable from the hive cluster `s`.

Any infeasible output scores `0`.

## Objective

Let `F` be the sum over all terminals `t` of the shortest-path flight cost from `s` to `t`
in the graph with your chosen corridors removed. You are **maximizing** `F`.

## Scoring

Let `B` be the sum over all terminals of the shortest-path flight cost from `s` in the
original network (no corridors destroyed). The score for a test is

```
ratio = min(1.0, 0.1 * F / B)
```

A do-nothing submission (`F = B`) scores `0.1`; forcing a ten-fold increase in total
foraging effort (`F = 10*B`) caps the score at `1.0`. The overall score is the mean ratio
across all tests.

## Constraints

- `2 <= n <= 900`, `1 <= m <= 2600`.
- `1 <= w <= 60`.
- `1 <= q <= 30`.
- `0 <= k <= 25`.
- Time limit: 8 s. Memory limit: 512 MB.

## Example

Input:

```
5 6 1 2 1
1 2 1
2 4 1
1 3 8
3 4 8
2 5 2
4 5 20
5
```

Wait-points: hive cluster is node `1`; forage patches are nodes `4` and `5`.

Original shortest costs from the hive: to `4` via `1 -> 2 -> 4` costs `2`; to `5` via
`1 -> 2 -> 5` costs `3`. So `B = 2 + 3 = 5`.

With a budget of `k = 1`, destroy corridor `1` (the `1-2` link). Now the cheapest route to
`4` becomes `1 -> 3 -> 4` costing `16`, and the cheapest route to `5` becomes
`1 -> 3 -> 4 -> 5` costing `36`. Both patches stay reachable, so `F = 16 + 36 = 52`.

Score for this test: `min(1.0, 0.1 * 52 / 5) = min(1.0, 1.04) = 1.0`.

A valid worked output:

```
1
1
```
