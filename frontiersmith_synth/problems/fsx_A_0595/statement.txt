# Order Slab

## Problem

You must carve a rectangular sheet `W x H` into pieces using only **guillotine
cuts**: a single straight cut runs edge-to-edge across one current rectangle,
splitting it into two smaller rectangles, which you then cut recursively. The whole
plan is a binary **cut tree** whose leaves are the finished rectangles. There is no
kerf: a cut on a rectangle just splits its size exactly. Cuts are irreversible —
**two fragments can never be rejoined**, so once you cut a band down to some height,
every piece of that band is frozen at that height forever.

You are given `D` demand types. Type `t` has size `w_t x h_t`, a per-piece value
`v_t`, and a **cap** `c_t`. A leaf rectangle may be **sold** as type `t` only when
its size equals `w_t x h_t` **exactly**. Selling type `t` earns `v_t` per piece, but
only the first `c_t` sold pieces of that type earn anything (extra copies earn `0`).
A leaf sold as no type is scrap. Maximize the total value earned.

Because value depends on how many leaves of each *exact* size your tree produces, a
good plan must make the **leaf-size distribution of the whole tree** match the demand
multiset — not just cut out the single most valuable piece.

## Input

```
W H K D
```
then `D` lines, type `t` (1-based, in input order):
```
w_t h_t v_t c_t
```

## Output

Print the cut tree in **pre-order** as a stream of integers. For the current
rectangle (the root is the whole sheet `W x H`):

* **Leaf:** `0 t`. `t = 0` marks scrap; `t` in `[1, D]` sells one piece of type `t`
  (the leaf's size must equal `w_t x h_t` exactly).
* **Cut:** `1 o a`, then the two children in pre-order (first child, then second).
  `o = 0` vertical, `o = 1` horizontal; `a` is the size of the **first** child along
  the cut axis.
  * vertical on `(w,h)`: child1 `= (a, h)`, child2 `= (w - a, h)`.
  * horizontal on `(w,h)`: child1 `= (w, a)`, child2 `= (w, h - a)`.

  You must have `a >= 1` and the other child size `>= 1`.

## Feasibility

Every cut must obey the size rules; the total number of cut nodes must be `<= K`;
the token stream must describe exactly one complete tree with no trailing tokens; and
every sold leaf must match its type exactly. Any violation scores `0`. Printing just
`0 0` (the whole sheet as scrap) is feasible but earns nothing.

## Objective

Maximize `F = sum over t of v_t * min(c_t, (pieces you sold as type t))`.

## Scoring

Let `B` be the value of a fixed **best single shelf**: pick the type `t` maximizing
`v_t * min(c_t, floor(W / w_t))` (over types with `w_t <= W`, `h_t <= H`) and sell
that many copies of it in one band. Your score for a test is
```
ratio = min(1000, 100 * F / max(1, B)) / 1000
```
so the best-single-shelf plan scores `0.1` and any real improvement scores higher,
capped at `1.0` (ten times the reference). Per-test ratios are averaged.

## Constraints

* `48 <= W <= 200`, `50 <= H <= 170`.
* `1 <= D <= 60`; `1 <= w_t < W`, `1 <= h_t < H`, `1 <= v_t`, `1 <= c_t`.
* `1 <= K` (a generous cut budget).
* Time limit: 3 s. Memory limit: 512 MB.

## Example

Sheet `W = 60`, `H = 20`, `K = 3000`, three types:
`A = 20x10 v=100 c=6`, `B = 20x10 v=90 c=6`, `S = 40x10 v=260 c=1`.
`S` has the highest per-piece value, so a value-first plan opens a `40x10` band for
`S` (earning `260`) and wastes the `20x10` corner. But packing the sheet as two
`10`-tall bands of three `A`/`B` pieces earns `100*3 + 90*3 = 570`. Matching the
leaf-size distribution to the demand beats grabbing the shiny piece. Here the best
single shelf is three `A` pieces, so `B = 300` and `F = 570` scores `ratio = 0.190`.
