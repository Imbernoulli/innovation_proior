# Coral Skin Against the Current

## Problem

You are growing a coral colony on a flat lattice seabed of width `W`. The colony
occupies unit cells of a grid: columns `x in [1,W]`, rows `y in [1,H]` (row `y`
sits `y` units above the bedrock). The bedrock itself (`y = 0`) is solid rock
across every column.

Coral can only exist by **accretion**: it grows one layer at a time from material
that is already there. A cell `(x, y)` you occupy is only physically real if it has
a supporting chain of occupied cells (or bedrock) directly beneath it, one layer
at a time, and each layer step may drift sideways by at most `S` columns — `S` is
the colony's **maximum overhang**: a wider drift per layer and the branch snaps
under its own unsupported weight. Formally, cell `(x, y)` is *supported* if
either `y = 1` (it rests straight on solid bedrock), or there exists an occupied,
supported cell `(x', y-1)` with `|x - x'| <= S`.

You have a mass budget of `M` cells. Your colony is scored on its **exposed skin**:
every unit face (left / right / up / down) of an occupied cell that does not touch
another occupied cell (and, for the bottom face, does not rest on bedrock) is
exposed to open water and contributes to the score. A steady nutrient current flows
past the reef with direction vector `(Fx, Fy)` (read from input, not fixed).
An exposed face with outward unit normal `d` contributes a **fixed base amount plus
a current-alignment bonus** that grows the more that face points into the current
(faces angled away from the current still count, just for less): the more a face
"catches" the flow, the more it is worth. You must read `Fx, Fy` from the input and
reason about which faces to grow toward the current — the exact bonus numbers are
not fixed constants you can hard-code.

Skin that has no supporting trunk beneath it isn't real coral: **any occupied cell
that fails the support test invalidates the whole colony (score 0)**, no matter how
much exposed skin it would otherwise contribute. Thin dendritic fingers thrown out
in every direction maximize raw exposed skin, but without a wide, tall enough
buttress beneath them they violate the overhang limit; a real reef must trade some
skin for a self-supporting trunk before it can branch.

## Input

```
W H S M Fx Fy
```
One line, six integers.

## Output

```
K
x_1 y_1
x_2 y_2
...
x_K y_K
```
`K` (0 <= K <= M) occupied cells, one per line, in any order, each with
`1 <= x_i <= W`, `1 <= y_i <= H`. No cell may repeat.

## Feasibility

* `0 <= K <= M`, all `(x_i, y_i)` in bounds, no duplicates.
* Every occupied cell must be *supported* (see the accretion rule above), checked
  transitively down to bedrock. Any unsupported cell scores the whole output `0`.

## Objective

Maximize `F`, the total exposed-skin value: for every occupied cell and every one
of its 4 faces (left, right, up, down) whose neighboring cell is empty (out of the
grid counts as empty; the bottom face at `y=1` is covered by bedrock, hence never
exposed), add `BASE + max(0, Fx*dx + Fy*dy)` where `(dx,dy)` is that face's outward
normal and `BASE = 5`.

## Scoring

The checker computes a fixed reference `B`: a single 1-wide pillar of height
`h0 = min(M, H)` standing at the middle column. Your score for a test is
```
ratio = min(1000, 100 * F / max(1, B)) / 1000
```
so the bare pillar scores `0.1`, and a genuinely better colony scores higher
(capped at `1.0`, ten times the pillar). Per-test ratios are averaged.

## Constraints

* `8 <= W <= 40`, `4 <= H <= 40`, `1 <= S <= 4`.
* `W <= M <= W*H`.
* `-6 <= Fx, Fy <= 6`, not both zero.
* Time limit: 4 s. Memory limit: 512 MB.

## Example

`W=8 H=6 S=1 M=10 Fx=1 Fy=0` (current flows to the right). A trunk of 3 cells at
column 4, rows 1-3, then two isolated single cells at `(5,4)` and `(6,5)` (each
one step right of the cell below it, using the full allowed drift `S=1`) uses 5
of the 10 cells, keeps every cell supported, and puts its two extra branch tips
facing squarely into the current, worth more than spending those same 5 cells on
fingers spread evenly to both the left and the right.
