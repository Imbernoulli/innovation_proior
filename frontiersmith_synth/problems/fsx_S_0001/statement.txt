# Sabotaged Sourdough: Flour-Route Interdiction

## Problem

The *Rising Crust* bakery cooperative ships flour from a central **mill** (node `s`) to
its flagship **bakery** (node `t`) across a regional supply network of hubs connected by
two-way haulage links. Each link has a positive travel time. A competing conglomerate
wants to slow deliveries as much as possible: it may **close at most `k` links**, but if
the bakery is ever fully cut off from the mill the regulators intervene and the sabotage
is thrown out entirely. Your job is to choose which links to close so that the *fastest
remaining route* from the mill to the bakery is as **long** as possible.

Formally: you are given an undirected weighted (multi)graph. Choose a set `S` of at most
`k` edges to delete such that `s` and `t` remain connected, maximizing the length of the
shortest `s`-`t` path in the graph with the edges of `S` removed.

This is a shortest-path interdiction problem: it is NP-hard, the optimum is unknown, and
many heuristics (greedy, best-improvement local search, randomized restarts) are viable.

## Input

The first line contains five integers:

```
n m s t k
```

- `n` — number of hubs (nodes), numbered `1..n`.
- `m` — number of haulage links (edges).
- `s` — the mill (source node).
- `t` — the bakery (sink node), `s != t`.
- `k` — the maximum number of links you may close.

Each of the next `m` lines describes one link (edge index `1..m` in input order):

```
u v w
```

meaning a two-way link between hubs `u` and `v` with travel time `w` (`1 <= w`).
The graph may contain parallel links and is guaranteed to have `s` connected to `t`.

## Output

On the first line print `r` — the number of links you close (`0 <= r <= k`).
On each of the next `r` lines print one link index (an integer in `1..m`). All indices
must be distinct.

Printing `0` on the first line (and nothing else) is always valid.

## Feasibility

An output is feasible iff:

- `0 <= r <= k`;
- every printed index is in `1..m` and all are distinct;
- after removing exactly the chosen links, `s` and `t` are still connected.

Any infeasible output scores `0`.

## Objective

Let `F` be the shortest-path travel time from `s` to `t` after your chosen links are
removed. You are **maximizing** `F`.

## Scoring

Let `B` be the shortest-path travel time from `s` to `t` in the original network (no links
closed). The score for a test is

```
ratio = min(1.0, 0.1 * F / B)
```

A do-nothing submission (`F = B`) scores `0.1`; reaching a ten-fold delay (`F = 10*B`)
caps the score at `1.0`. The overall score is the mean ratio across all tests.

## Constraints

- `2 <= n <= 500`, `1 <= m <= 2000`.
- `1 <= w <= 60`.
- `0 <= k <= 20`.
- Time limit: 5 s. Memory limit: 512 MB.

## Example

Input:

```
4 5 1 4 1
1 2 1
2 4 1
1 3 5
3 4 5
1 4 20
```

Original shortest route: `1 -> 2 -> 4` with length `2`, so `B = 2`.

With a budget of `k = 1`, closing link `1` (the `1-2` link) removes the cheap route; the
fastest remaining route becomes `1 -> 3 -> 4` with length `10`, so `F = 10`.

Score for this test: `min(1.0, 0.1 * 10 / 2) = min(1.0, 0.5) = 0.5`.

A valid worked output:

```
1
1
```
